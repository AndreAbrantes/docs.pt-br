<span data-ttu-id="13abc-106">Há duas abordagens gerais para a criação de aplicativos Web hoje: os aplicativos Web tradicionais que executam a maior parte da lógica do aplicativo no servidor e os SPAs (aplicativos de página única) que executam a maior parte da lógica da interface do usuário em um navegador da Web, comunicando-se com o servidor Web principalmente por meio de APIs Web.</span><span class="sxs-lookup"><span data-stu-id="13abc-106">There are two general approaches to building web applications today: traditional web applications that perform most of the application logic on the server, and single page applications (SPAs) that perform most of the user interface logic in a web browser, communicating with the web server primarily using web APIs.</span></span> <span data-ttu-id="13abc-107">Uma abordagem híbrida também é possível, e a mais simples é hospedar um ou mais subaplicativos avançados semelhantes ao SPA dentro de um aplicativo Web tradicional maior.</span><span class="sxs-lookup"><span data-stu-id="13abc-107">A hybrid approach is also possible, the simplest being host one or more rich SPA-like sub-applications within a larger traditional web application.</span></span>

<span data-ttu-id="13abc-108">Você deve usar os aplicativos Web tradicionais quando:</span><span class="sxs-lookup"><span data-stu-id="13abc-108">You should use traditional web applications when:</span></span>

- <span data-ttu-id="13abc-109">Os requisitos do lado do cliente do aplicativo são simples ou até mesmo somente leitura.</span><span class="sxs-lookup"><span data-stu-id="13abc-109">Your application's client-side requirements are simple or even read-only.</span></span>

- <span data-ttu-id="13abc-110">Seu aplicativo precisa funcionar em navegadores sem suporte a JavaScript.</span><span class="sxs-lookup"><span data-stu-id="13abc-110">Your application needs to function in browsers without JavaScript support.</span></span>

- <span data-ttu-id="13abc-111">Sua equipe não está familiarizada com as técnicas de desenvolvimento do JavaScript ou do TypeScript.</span><span class="sxs-lookup"><span data-stu-id="13abc-111">Your team is unfamiliar with JavaScript or TypeScript development techniques.</span></span>

<span data-ttu-id="13abc-112">Você deve usar um SPA quando:</span><span class="sxs-lookup"><span data-stu-id="13abc-112">You should use a SPA when:</span></span>

- <span data-ttu-id="13abc-113">Seu aplicativo precisa expor uma interface do usuário avançada com muitos recursos.</span><span class="sxs-lookup"><span data-stu-id="13abc-113">Your application must expose a rich user interface with many features.</span></span>

- <span data-ttu-id="13abc-114">Sua equipe está familiarizada com o desenvolvimento do JavaScript e/ou do TypeScript.</span><span class="sxs-lookup"><span data-stu-id="13abc-114">Your team is familiar with JavaScript and/or TypeScript development.</span></span>

- <span data-ttu-id="13abc-115">Seu aplicativo já deve expor uma API para outros clientes (internos ou públicos).</span><span class="sxs-lookup"><span data-stu-id="13abc-115">Your application must already expose an API for other (internal or public) clients.</span></span>

<span data-ttu-id="13abc-116">Além disso, as estruturas de SPA exigem um maior conhecimento em arquitetura e segurança.</span><span class="sxs-lookup"><span data-stu-id="13abc-116">Additionally, SPA frameworks require greater architectural and security expertise.</span></span> <span data-ttu-id="13abc-117">Elas passam por uma maior variação devido a atualizações frequentes e novas estruturas comparado aos aplicativos Web tradicionais.</span><span class="sxs-lookup"><span data-stu-id="13abc-117">They experience greater churn due to frequent updates and new frameworks than traditional web applications.</span></span> <span data-ttu-id="13abc-118">A configuração de processos de compilação e de implantação automatizados e a utilização de opções de implantação como contêineres são mais difíceis com aplicativos SPA do que com aplicativos Web tradicionais.</span><span class="sxs-lookup"><span data-stu-id="13abc-118">Configuring automated build and deployment processes and utilizing deployment options like containers are more difficult with SPA applications than traditional web apps.</span></span>

<span data-ttu-id="13abc-119">As melhorias na experiência do usuário possibilitadas pelo modelo do SPA devem ser avaliadas em relação a essas considerações.</span><span class="sxs-lookup"><span data-stu-id="13abc-119">Improvements in user experience made possible by SPA model must be weighed against these considerations.</span></span>

## <a name="when-to-choose-traditional-web-apps"></a><span data-ttu-id="13abc-120">Quando escolher aplicativos Web tradicionais</span><span class="sxs-lookup"><span data-stu-id="13abc-120">When to choose traditional web apps</span></span>

<span data-ttu-id="13abc-121">Veja a seguir uma explicação mais detalhada dos motivos para escolher aplicativos Web tradicionais mencionados anteriormente.</span><span class="sxs-lookup"><span data-stu-id="13abc-121">The following is a more detailed explanation of the previously-stated reasons for picking traditional web applications.</span></span>

<span data-ttu-id="13abc-122">**Seu aplicativo tem requisitos simples do lado do cliente, possivelmente somente leitura**</span><span class="sxs-lookup"><span data-stu-id="13abc-122">**Your application has simple, possibly read-only, client-side requirements**</span></span>

<span data-ttu-id="13abc-123">Muitos aplicativos Web são consumidos principalmente em um modo somente leitura pela grande maioria de seus usuários.</span><span class="sxs-lookup"><span data-stu-id="13abc-123">Many web applications are primarily consumed in a read-only fashion by the vast majority of their users.</span></span> <span data-ttu-id="13abc-124">Os aplicativos somente leitura (ou predominantemente de leitura) tendem a ser muito mais simples do que aqueles que mantêm e manipulam uma grande quantidade de estado.</span><span class="sxs-lookup"><span data-stu-id="13abc-124">Read-only (or read-mostly) applications tend to be much simpler than those that maintain and manipulate a great deal of state.</span></span> <span data-ttu-id="13abc-125">Por exemplo, um mecanismo de pesquisa pode consistir em um único ponto de entrada com uma caixa de texto e uma segunda página para exibição dos resultados da pesquisa.</span><span class="sxs-lookup"><span data-stu-id="13abc-125">For example, a search engine might consist of a single entry point with a textbox and a second page for displaying search results.</span></span> <span data-ttu-id="13abc-126">Os usuários anônimos podem fazer solicitações com facilidade, e há pouca necessidade de uma lógica do lado do cliente.</span><span class="sxs-lookup"><span data-ttu-id="13abc-128">Aplicativos desse tipo são criados com facilidade como aplicativos Web tradicionais baseados em servidor que executam a lógica no servidor Web e renderizam o HTML a ser exibido render HTML to be displayed in the browser.</span></span> <span data-ttu-id="13abc-129">O fato de que cada página exclusiva do site tem sua própria URL que pode ser marcada e indexada por mecanismos de pesquisa (por padrão, sem a necessidade de adicioná-la como um recurso separado do aplicativo) também é um benefício claro nesses cenários.</span><span class="sxs-lookup"><span data-stu-id="13abc-129">The fact that each unique page of the site has its own URL that can be bookmarked and indexed by search engines (by default, without having to add this as a separate feature of <span data-ttu-id="13abc-130">**Seu aplicativo precisa funcionar em navegadores sem suporte a JavaScript**</span><span class="sxs-lookup"><span data-stu-id="13abc-130">**Your application needs to function in browsers without JavaScript support**</span></span>

<span data-ttu-id="13abc-131">Os aplicativos Web que precisam funcionar em navegadores com suporte limitado ou nenhum suporte a JavaScript precisam ser escritos com fluxos de trabalho de aplicativo Web tradicional (ou, pelo menos, poder fazer fallback para esse comportamento).</span><span class="sxs-lookup"><span data-stu-id="13abc-131">Web applications that need to function in browsers with limited or no JavaScript support should be written using traditional web app workflows (or at least be able to fall back to such behavior).</span></span> <span data-ttu-id="13abc-132">Os SPAs exigem um JavaScript do lado do cliente para funcionar; se ele não estiver disponível, os SPAs não serão uma boa opção.</span><span class="sxs-lookup"><span data-stu-id="13abc-132">SPAs require client-side JavaScript in order to function; if it's not available, SPAs are not a good choice.</span></span>

<span data-ttu-id="13abc-133">**Sua equipe não está familiarizada com as técnicas de desenvolvimento do JavaScript ou do TypeScript**</span><span class="sxs-lookup"><span data-stu-id="13abc-133">**Your team is unfamiliar with JavaScript or TypeScript development techniques**</span></span>

<span data-ttu-id="13abc-134">Caso sua equipe não esteja familiarizada com o JavaScript ou o TypeScript, mas esteja familiarizada com o desenvolvimento de aplicativos Web do lado do servidor, provavelmente, ela poderá fornecer um aplicativo Web tradicional mais rapidamente do que um SPA.</span><span class="sxs-lookup"><span data-stu-id="13abc-134">If your team is unfamiliar with JavaScript or TypeScript, but is familiar with server-side web application development, then they will probably be able to deliver a traditional web app more quickly than a SPA.</span></span> <span data-ttu-id="13abc-135">A menos que o aprendizado de programação de SPAs seja uma meta ou a experiência do usuário proporcionada por um SPA seja necessária, os aplicativos Web tradicionais serão uma opção mais produtiva para equipes que já estão familiarizadas com sua criação.</span><span class="sxs-lookup"><span data-stu-id="13abc-135">Unless learning to program SPAs is a goal, or the user experience afforded by a SPA is required, traditional web apps are a more productive choice for teams who are already familiar with building them.</span></span>

## <a name="when-to-choose-spas"></a><span data-ttu-id="13abc-136">Quando escolher SPAs</span><span class="sxs-lookup"><span data-stu-id="13abc-136">When to choose SPAs</span></span>

<span data-ttu-id="13abc-137">Veja a seguir uma explicação mais detalhada de quando escolher um estilo de desenvolvimento de Aplicativos de Página Única para seu aplicativo Web.</span><span class="sxs-lookup"><span data-stu-id="13abc-137">The following is a more detailed explanation of when to choose a Single Page Applications style of development for your web app.</span></span>

<span data-ttu-id="13abc-138">**Seu aplicativo precisa expor uma interface do usuário avançada com muitos recursos**</span><span class="sxs-lookup"><span data-stu-id="13abc-138">**Your application must expose a rich user interface with many features**</span></span>

<span data-ttu-id="13abc-139">Os SPAs podem dar suporte à funcionalidade avançada do lado do cliente que não exige o recarregamento da página conforme os usuários executam ações ou navegam entre as áreas do aplicativo.</span><span class="sxs-lookup"><span data-stu-id="13abc-139">SPAs can support rich client-side functionality that doesn't require reloading the page as users take actions or navigate between areas of the app.</span></span> <span data-ttu-id="13abc-140">OS SPAs podem ser carregados com mais agilidade, buscando dados em segundo plano, e as ações de usuário individuais são mais ágeis na resposta pois os recarregamentos de página inteira são raros.</span><span class="sxs-lookup"><span data-stu-id="13abc-140">SPAs can load more quickly, fetching data in the background, data-stu-id="13abc-142">SPAs can support rich client-side behaviors, such as drag-and-drop, much more readily than traditional applications.</span></span> <span data-ttu-id="13abc-143">Os SPAs podem ser designados para serem executados em um modo desconectado, fazendo atualizações em um modelo do lado do cliente que são, por fim, sincronizadas com o servidor depois que uma conexão é restabelecida.</span><span class="sxs-lookup"><span data-stu-id="13abc-143">SPAs can be designed to run in a disconnected mode, making updates to a client-side model that are eventually synchronized back to the server once a connection is re-established.</span></span> <span data-ttu-id="13abc-144">Você deve escolher um aplicativo no estilo SPA se os requisitos do aplicativo incluem uma funcionalidade avançada que vai além do que os formulários HTML típicos oferecem.</span><span class="sxs-lookup"><span data-stu-id="13abc-144">You should choose a SPA style application if your app's requirements include rich functionality that goes beyond what typical HTML forms offer.</span></span>

<span data-ttu-id="13abc-145">Observe que, com frequência, os SPAs precisam implementar recursos internos de aplicativos Web tradicionais, como a exibição de uma URL significativa na barra de endereços que reflete a operação atual (e permitindo que os usuários marquem ou criem um link profundo dessa URL para retornar a ela).</span><span class="sxs-lookup"><span data-stu-id="13abc-145">Note that frequently SPAs need to implement features that are built-in to traditional web apps, such as displaying a meaningful URL in the address bar reflecting the current operation (and allowing users to bookmark or deep link to this URL to return to it).</span></span> <span data-ttu-id="13abc-146">Os SPAs também devem permitir que os usuários usem os botões Voltar e Avançar do navegador com resultados que não os surpreenderão.</span><span class="sxs-lookup"><span data-stu-id="13abc-146">SPAs also should allow users to use the browser's back and forward buttons with results that won't surprise them.</span></span>

<span data-ttu-id="13abc-147">**Sua equipe está familiarizado com o desenvolvimento do JavaScript e/ou do TypeScript**</span><span class="sxs-lookup"><span data-stu-id="13abc-147">**Your team is familiar with JavaScript and/or TypeScript development**</span></span>

<span data-ttu-id="13abc-148">A configuração de SPAs exige familiaridade com o JavaScript e/ou o TypeScript e bibliotecas e técnicas de programação do lado do cliente.</span><span class="sxs-lookup"><span data-stu-id="13abc-148">Writing SPAs requires familiarity with JavaScript and/or TypeScript and client-side programming techniques and libraries.</span></span> <span data-ttu-id="13abc-149">Sua equipe deve ser competente na escrita de JavaScript moderno usando uma estrutura de SPA como o Angular.</span><span class="sxs-lookup"><span data-stu-id="13abc-149">Your team should be competent in writing modern JavaScript using a SPA framework like Angular.</span></span>
