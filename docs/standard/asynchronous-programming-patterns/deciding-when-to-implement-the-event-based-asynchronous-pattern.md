## <a name="general-principles"></a><span data-ttu-id="157bc-107">Princípios gerais</span><span class="sxs-lookup"><span data-stu-id="157bc-107">General Principles</span></span>

## <a name="guidelines"></a><span data-ttu-id="157bc-112">Diretrizes</span><span class="sxs-lookup"><span data-stu-id="157bc-112">Guidelines</span></span>  
 <span data-ttu-id="157bc-113">A lista a seguir descreve as diretrizes de quando você deve implementar o padrão assíncrono baseado em evento:</span><span class="sxs-lookup"><span data-stu-id="157bc-113">The following list describes the guidelines for when you should implement Event-based Asynchronous Pattern:</span></span>  
  
-   <span data-ttu-id="157bc-114">Use o padrão de evento como a API padrão para expor o comportamento assíncrono para sua classe.</span><span class="sxs-lookup"><span data-stu-id="157bc-114">Use the event-based pattern as the default API to expose asynchronous behavior for your class.</span></span>  
  
-   <span data-ttu-id="157bc-115">Não exponha o <xref:System.IAsyncResult> padrão quando a classe é usada principalmente em um aplicativo cliente, por exemplo, Windows Forms.</span><span class="sxs-lookup"><span data-stu-id="157bc-115">Do not expose the <xref:System.IAsyncResult> pattern when your class is primarily used in a client application, for example Windows Forms.</span></span>  
  
-   <span data-ttu-id="157bc-116">Apenas expor o <xref:System.IAsyncResult> padrão quando é necessário para atender às suas necessidades.</span><span class="sxs-lookup"><span data-stu-id="157bc-116">Only expose the <xref:System.IAsyncResult> pattern when it is necessary for meeting your requirements.</span></span> <span data-ttu-id="157bc-117">Por exemplo, compatibilidade com uma API existente pode exigir que você exponha o <xref:System.IAsyncResult> padrão.</span><span class="sxs-lookup"><span data-stu-id="157bc-117">For example, compatibility with an existing API may require you to expose the <xref:System.IAsyncResult> pattern.</span></span>  
  
-   <span data-ttu-id="157bc-118">Não exponha o <xref:System.IAsyncResult> padrão sem também expor o padrão de evento.</span><span class="sxs-lookup"><span data-stu-id="157bc-118">Do not expose the <xref:System.IAsyncResult> pattern without also exposing the event-based pattern.</span></span>  
  
-   <span data-ttu-id="157bc-119">Se você precisar expor o <xref:System.IAsyncResult> padrão, isso como uma opção avançada.</span><span class="sxs-lookup"><span data-stu-id="157bc-119">If you must expose the <xref:System.IAsyncResult> pattern, do so as an advanced option.</span></span> <span data-ttu-id="157bc-120">Por exemplo, se você gerar um objeto de proxy, gerar o padrão baseado em evento por padrão, com uma opção para gerar o <xref:System.IAsyncResult> padrão.</span><span class="sxs-lookup"><span data-stu-id="157bc-120">For example, if you generate a proxy object, generate the event-based pattern by default, with an option to generate the <xref:System.IAsyncResult> pattern.</span></span>  
  
-   <span data-ttu-id="157bc-121">Sua implementação do padrão com base em eventos de compilação seu <xref:System.IAsyncResult> implementação do padrão.</span><span class="sxs-lookup"><span data-stu-id="157bc-121">Build your event-based pattern implementation on your <xref:System.IAsyncResult> pattern implementation.</span></span>  
  
-   <span data-ttu-id="157bc-122">Evitar a exposição de ambas as padrão baseado em evento e o <xref:System.IAsyncResult> padrão na mesma classe.</span><span class="sxs-lookup"><span data-stu-id="157bc-122">Avoid exposing both the event-based pattern and the <xref:System.IAsyncResult> pattern on the same class.</span></span> <span data-ttu-id="157bc-123">Expor o padrão com base em eventos em classes "de nível superior" e o <xref:System.IAsyncResult> padrão classes em "nível inferior".</span><span class="sxs-lookup"><span data-stu-id="157bc-123">Expose the event-based pattern on "higher level" classes and the <xref:System.IAsyncResult> pattern on "lower level" classes.</span></span> <span data-ttu-id="157bc-124">Por exemplo, comparar o padrão baseado em evento no <xref:System.Net.WebClient> componente com o <xref:System.IAsyncResult> padrão no <xref:System.Web.HttpRequest> classe.</span><span class="sxs-lookup"><span data-stu-id="157bc-124">For example, compare the event-based pattern on the <xref:System.Net.WebClient> component with the <xref:System.IAsyncResult> pattern on the <xref:System.Web.HttpRequest> class.</span></span>  
  
    -   <span data-ttu-id="157bc-125">Expor o padrão de evento e o <xref:System.IAsyncResult> padrão da mesma classe quando compatibilidade exija isso.</span><span class="sxs-lookup"><span data-stu-id="157bc-125">Expose the event-based pattern and the <xref:System.IAsyncResult> pattern on the same class when compatibility requires it.</span></span> <span data-ttu-id="157bc-126">Por exemplo, se você já lançou uma API que usa o <xref:System.IAsyncResult> padrão, você precisa manter o <xref:System.IAsyncResult> padrão para compatibilidade com versões anteriores.</span><span class="sxs-lookup"><span data-stu-id="157bc-126">For example, if you have already released an API that uses the <xref:System.IAsyncResult> pattern, you would need to retain the <xref:System.IAsyncResult> pattern for backward compatibility.</span></span>  
  
    -   <span data-ttu-id="157bc-127">Expor o padrão de evento e o <xref:System.IAsyncResult> se a complexidade de modelo de objeto resultante supera o benefício de separar as implementações do padrão da mesma classe.</span><span class="sxs-lookup"><span data-stu-id="157bc-127">Expose the event-based pattern and the <xref:System.IAsyncResult> pattern on the same class if the resulting object model complexity outweighs the benefit of separating the implementations.</span></span> <span data-ttu-id="157bc-128">É melhor para expor os dois padrões em uma única classe que ao evitar expor o padrão de evento.</span><span class="sxs-lookup"><span data-stu-id="157bc-128">It is better to expose both patterns on a single class than to avoid exposing the event-based pattern.</span></span>  
  
    -   <span data-ttu-id="157bc-129">Se você deve expor ambos o padrão com base em eventos e <xref:System.IAsyncResult> padrão em uma única classe, use <xref:System.ComponentModel.EditorBrowsableAttribute> definida como <xref:System.ComponentModel.EditorBrowsableState.Advanced> para marcar o <xref:System.IAsyncResult> implementação do padrão como um recurso avançado.</span><span class="sxs-lookup"><span data-stu-id="157bc-129">If you must expose both the event-based pattern and <xref:System.IAsyncResult> pattern on a single class, use <xref:System.ComponentModel.EditorBrowsableAttribute> set to <xref:System.ComponentModel.EditorBrowsableState.Advanced> to mark the <xref:System.IAsyncResult> pattern implementation as an advanced feature.</span></span> <span data-ttu-id="157bc-130">Isso indica a ambientes de design, como [!INCLUDE[vsprvs](../../../includes/vsprvs-md.md)] IntelliSense, para não exibir o <xref:System.IAsyncResult> propriedades e métodos.</span><span class="sxs-lookup"><span data-stu-id="157bc-130">This indicates to design environments, such as [!INCLUDE[vsprvs](../../../includes/vsprvs-md.md)] IntelliSense, not to display the <xref:System.IAsyncResult> properties and methods.</span></span> <span data-ttu-id="157bc-131">Essas propriedades e métodos ainda são totalmente utilizáveis, mas o desenvolvedor trabalhando por meio do IntelliSense tem uma visão clara da API.</span><span class="sxs-lookup"><span data-stu-id="157bc-131">These properties and methods are still fully usable, but the developer working through IntelliSense has a clearer view of the API.</span></span>  
  
## <a name="criteria-for-exposing-the-iasyncresult-pattern-in-addition-to-the-event-based-pattern"></a><span data-ttu-id="157bc-132">Critérios para expor o padrão de IAsyncResult além do padrão baseado em evento</span><span class="sxs-lookup"><span data-stu-id="157bc-132">Criteria for Exposing the IAsyncResult Pattern in Addition to the Event-based Pattern</span></span>  
 <span data-ttu-id="157bc-133">Enquanto o padrão assíncrono baseado em evento tem muitas vantagens em cenários de mencionadas anteriormente, ele tem apresenta algumas desvantagens, você deve estar ciente de desempenho é o requisito mais importante.</span><span class="sxs-lookup"><span data-stu-id="157bc-133">While the Event-based Asynchronous Pattern has many benefits under the previously mentioned scenarios, it does have some drawbacks, which you should be aware of if performance is your most important requirement.</span></span>  
  
 <span data-ttu-id="157bc-134">Há três cenários que não lida com o padrão baseado em evento, bem como a <xref:System.IAsyncResult> padrão:</span><span class="sxs-lookup"><span data-stu-id="157bc-134">There are three scenarios that the event-based pattern does not address as well as the <xref:System.IAsyncResult> pattern:</span></span>  
  
-   <span data-ttu-id="157bc-135">Bloqueio de espera em um<xref:System.IAsyncResult></span><span class="sxs-lookup"><span data-stu-id="157bc-135">Blocking wait on one <xref:System.IAsyncResult></span></span>  
  
-   <span data-ttu-id="157bc-136">Bloqueio de espera em muitas <xref:System.IAsyncResult> objetos</span><span class="sxs-lookup"><span data-stu-id="157bc-136">Blocking wait on many <xref:System.IAsyncResult> objects</span></span>  
  
-   <span data-ttu-id="157bc-137">Sondagem de conclusão no<xref:System.IAsyncResult></span><span class="sxs-lookup"><span data-stu-id="157bc-137">Polling for completion on the <xref:System.IAsyncResult></span></span>  
  
 <span data-ttu-id="157bc-138">Você pode abordar esses cenários usando o padrão baseado em evento, mas isso é mais complicado do que usando o <xref:System.IAsyncResult> padrão.</span><span class="sxs-lookup"><span data-stu-id="157bc-138">You can address these scenarios by using the event-based pattern, but doing so is more cumbersome than using the <xref:System.IAsyncResult> pattern.</span></span>  
  
 <span data-ttu-id="157bc-139">Os desenvolvedores geralmente usam o <xref:System.IAsyncResult> padrão para os serviços que normalmente têm requisitos de desempenho muito alto.</span><span class="sxs-lookup"><span data-stu-id="157bc-139">Developers often use the <xref:System.IAsyncResult> pattern for services that typically have very high performance requirements.</span></span> <span data-ttu-id="157bc-140">Por exemplo, a sondagem para o cenário de conclusão é uma técnica de servidor de alto desempenho.</span><span class="sxs-lookup"><span data-stu-id="157bc-140">For example, the polling for completion scenario is a high-performance server technique.</span></span>  
  
 <span data-ttu-id="157bc-141">Além disso, o padrão de evento é menos eficiente do que o <xref:System.IAsyncResult> padrão porque ela cria mais objetos, especialmente <xref:System.EventArgs>, e como elas são sincronizadas entre threads.</span><span class="sxs-lookup"><span data-stu-id="157bc-141">Additionally, the event-based pattern is less efficient than the <xref:System.IAsyncResult> pattern because it creates more objects, especially <xref:System.EventArgs>, and because it synchronizes across threads.</span></span>  
  
 <span data-ttu-id="157bc-142">A lista a seguir mostra algumas recomendações a seguir se você decidir usar o <xref:System.IAsyncResult> padrão:</span><span class="sxs-lookup"><span data-stu-id="157bc-142">The following list shows some recommendations to follow if you decide to use the <xref:System.IAsyncResult> pattern:</span></span>  
  
-   <span data-ttu-id="157bc-143">Apenas expor o <xref:System.IAsyncResult> padrão quando você precisar do suporte para especificamente <xref:System.Threading.WaitHandle> ou <xref:System.IAsyncResult> objetos.</span><span class="sxs-lookup"><span data-stu-id="157bc-143">Only expose the <xref:System.IAsyncResult> pattern when you specifically require support for <xref:System.Threading.WaitHandle> or <xref:System.IAsyncResult> objects.</span></span>  
  
-   <span data-ttu-id="157bc-144">Apenas expor o <xref:System.IAsyncResult> padrão quando você tem uma API existente que usa o <xref:System.IAsyncResult> padrão.</span><span class="sxs-lookup"><span data-stu-id="157bc-144">Only expose the <xref:System.IAsyncResult> pattern when you have an existing API that uses the <xref:System.IAsyncResult> pattern.</span></span>  
  
-   <span data-ttu-id="157bc-145">Se você tiver uma API existente com base no <xref:System.IAsyncResult> padrão, considere também expor o padrão baseado em evento em sua próxima versão.</span><span class="sxs-lookup"><span data-stu-id="157bc-145">If you have an existing API based on the <xref:System.IAsyncResult> pattern, consider also exposing the event-based pattern in your next release.</span></span>  
  
-   <span data-ttu-id="157bc-146">Apenas expor <xref:System.IAsyncResult> padrão se você tiver requisitos de alto desempenho que você verificou se não podem ser atendida pelo padrão baseado em evento, mas podem ser atendida pelo <xref:System.IAsyncResult> padrão.</span><span class="sxs-lookup"><span data-stu-id="157bc-146">Only expose <xref:System.IAsyncResult> pattern if you have high performance requirements which you have verified cannot be met by the event-based pattern but can be met by the <xref:System.IAsyncResult> pattern.</span></span>
