## <a name="guidelines"></a><span data-ttu-id="f15e4-112">Diretrizes</span><span class="sxs-lookup"><span data-stu-id="f15e4-112">Guidelines</span></span>  
 <span data-ttu-id="f15e4-113">A lista a seguir descreve as diretrizes para quando você deve implementar o Padrão Assíncrono baseado em Evento:</span><span class="sxs-lookup"><span data-stu-id="f15e4-113">The following list describes the guidelines for when you should implement Event-based Asynchronous Pattern:</span></span>  
  
-   <span data-ttu-id="f15e4-114">Use o padrão baseado em evento como a API padrão para expor o comportamento assíncrono para sua classe.</span><span class="sxs-lookup"><span data-stu-id="f15e4-114">Use the event-based pattern as the default API to expose asynchronous behavior for your class.</span></span>  
  
-   <span data-ttu-id="f15e4-115">Não exponha o padrão <xref:System.IAsyncResult>, principalmente quando a classe for usada em um aplicativo cliente, por exemplo, o Windows Forms.</span><span class="sxs-lookup"><span data-stu-id="f15e4-115">Do not expose the <xref:System.IAsyncResult> pattern when your class is primarily used in a client application, for example Windows Forms.</span></span>  
  
-   <span data-ttu-id="f15e4-116">Basta expor o padrão <xref:System.IAsyncResult> quando ele for necessário para atender seus requisitos.</span><span class="sxs-lookup"><span data-stu-id="f15e4-116">Only expose the <xref:System.IAsyncResult> pattern when it is necessary for meeting your requirements.</span></span> <span data-ttu-id="f15e4-117">Por exemplo, a compatibilidade com uma API existente pode exigir que você exponha o padrão <xref:System.IAsyncResult>.</span><span class="sxs-lookup"><span data-stu-id="f15e4-117">For example, compatibility with an existing API may require you to expose the <xref:System.IAsyncResult> pattern.</span></span>  
  
-   <span data-ttu-id="f15e4-118">Não exponha o padrão <xref:System.IAsyncResult> sem também expor o padrão baseado em evento.</span><span class="sxs-lookup"><span data-stu-id="f15e4-118">Do not expose the <xref:System.IAsyncResult> pattern without also exposing the event-based pattern.</span></span>  
  
-   <span data-ttu-id="f15e4-119">Se você precisar expor o padrão <xref:System.IAsyncResult>, exponha-o como uma opção avançada.</span><span class="sxs-lookup"><span data-stu-id="f15e4-119">If you must expose the <xref:System.IAsyncResult> pattern, do so as an advanced option.</span></span> <span data-ttu-id="f15e4-120">Por exemplo, se você gerar um objeto de proxy, gere o padrão baseado em evento por padrão, com uma opção para gerar o padrão <xref:System.IAsyncResult>.</span><span class="sxs-lookup"><span data-stu-id="f15e4-120">For example, if you generate a proxy object, generate the event-based pattern by default, with an option to generate the <xref:System.IAsyncResult> pattern.</span></span>  
  
-   <span data-ttu-id="f15e4-121">Crie sua implementação do padrão baseado em evento na implementação do padrão <xref:System.IAsyncResult>.</span><span class="sxs-lookup"><span data-stu-id="f15e4-121">Build your event-based pattern implementation on your <xref:System.IAsyncResult> pattern implementation.</span></span>  
  
-   <span data-ttu-id="f15e4-122">Evitar expor os dois padrões, o padrão baseado em evento e o padrão <xref:System.IAsyncResult>, na mesma classe.</span><span class="sxs-lookup"><span data-stu-id="f15e4-122">Avoid exposing both the event-based pattern and the <xref:System.IAsyncResult> pattern on the same class.</span></span> <span data-ttu-id="f15e4-123">Exponha o padrão baseado em evento em classes de "nível superior" e o padrão <xref:System.IAsyncResult> em classes de "nível inferior".</span><span class="sxs-lookup"><span data-stu-id="f15e4-123">Expose the event-based pattern on "higher level" classes and the <xref:System.IAsyncResult> pattern on "lower level" classes.</span></span> <span data-ttu-id="f15e4-124">Por exemplo, compare o padrão baseado em evento no componente <xref:System.Net.WebClient> com o padrão <xref:System.IAsyncResult> na classe <xref:System.Web.HttpRequest>.</span><span class="sxs-lookup"><span data-stu-id="f15e4-124">For example, compare the event-based pattern on the <xref:System.Net.WebClient> component with the <xref:System.IAsyncResult> pattern on the <xref:System.Web.HttpRequest> class.</span></span>  
  
    -   <span data-ttu-id="f15e4-125">Exponha o padrão baseado em evento e o padrão <xref:System.IAsyncResult> na mesma classe quando a compatibilidade exigir isso.</span><span class="sxs-lookup"><span data-stu-id="f15e4-125">Expose the event-based pattern and the <xref:System.IAsyncResult> pattern on the same class when compatibility requires it.</span></span> <span data-ttu-id="f15e4-126">Por exemplo, se já tiver lançado uma API que usa o padrão <xref:System.IAsyncResult>, precisará manter o padrão <xref:System.IAsyncResult> por questões de compatibilidade com versões anteriores.</span><span class="sxs-lookup"><span data-stu-id="f15e4-126">For example, if you have already released an API that uses the <xref:System.IAsyncResult> pattern, you would need to retain the <xref:System.IAsyncResult> pattern for backward compatibility.</span></span>  
  
    -   <span data-ttu-id="f15e4-127">Exponha o padrão baseado em evento e o padrão <xref:System.IAsyncResult> na mesma classe se a complexidade do modelo de objeto resultante for maior que o benefício de separar as implementações.</span><span class="sxs-lookup"><span data-stu-id="f15e4-127">Expose the event-based pattern and the <xref:System.IAsyncResult> pattern on the same class if the resulting object model complexity outweighs the benefit of separating the implementations.</span></span> <span data-ttu-id="f15e4-128">É melhor expor os dois padrões em uma única classe para evitar expor o padrão baseado em evento.</span><span class="sxs-lookup"><span data-stu-id="f15e4-128">It is better to expose both patterns on a single class than to avoid exposing the event-based pattern.</span></span>  
  
    -   <span data-ttu-id="f15e4-129">Se quiser expor o padrão baseado em evento e o padrão <xref:System.IAsyncResult> em uma única classe, use o <xref:System.ComponentModel.EditorBrowsableAttribute> definido como <xref:System.ComponentModel.EditorBrowsableState.Advanced> para marcar a implementação do padrão <xref:System.IAsyncResult> como um recurso avançado.</span><span class="sxs-lookup"><span data-stu-id="f15e4-129">If you must expose both the event-based pattern and <xref:System.IAsyncResult> pattern on a single class, use <xref:System.ComponentModel.EditorBrowsableAttribute> set to <xref:System.ComponentModel.EditorBrowsableState.Advanced> to mark the <xref:System.IAsyncResult> pattern implementation as an advanced feature.</span></span> <span data-ttu-id="f15e4-130">Isso dá a indicação aos ambientes de design, como o Visual Studio IntelliSense, para não exibir as propriedades e os métodos de <xref:System.IAsyncResult>.</span><span class="sxs-lookup"><span data-stu-id="f15e4-130">This indicates to design environments, such as Visual Studio IntelliSense, not to display the <xref:System.IAsyncResult> properties and methods.</span></span> <span data-ttu-id="f15e4-131">Essas propriedades e métodos ainda são totalmente utilizáveis, mas, trabalhando com o IntelliSense, o desenvolvedor terá uma visão clara da API.</span><span class="sxs-lookup"><span data-stu-id="f15e4-131">These properties and methods are still fully usable, but the developer working through IntelliSense has a clearer view of the API.</span></span>  
  
## <a name="criteria-for-exposing-the-iasyncresult-pattern-in-addition-to-the-event-based-pattern"></a><span data-ttu-id="f15e4-132">Critérios para expor o Padrão de IAsyncResult além do Padrão baseado em evento</span><span class="sxs-lookup"><span data-stu-id="f15e4-132">Criteria for Exposing the IAsyncResult Pattern in Addition to the Event-based Pattern</span></span>  
 <span data-ttu-id="f15e4-133">O Padrão Assíncrono Baseado em Evento é muito vantajoso quando aplicado aos cenários mencionados anteriormente. No entanto, ele apresenta algumas desvantagens e você precisa estar atento caso o desempenho for seu requisito mais importante.</span><span class="sxs-lookup"><span data-stu-id="f15e4-133">While the Event-based Asynchronous Pattern has many benefits under the previously mentioned scenarios, it does have some drawbacks, which you should be aware of if performance is your most important requirement.</span></span>  
  
 <span data-ttu-id="f15e4-134">Há três cenários que o padrão baseado em evento não trata e o padrão <xref:System.IAsyncResult> também não:</span><span class="sxs-lookup"><span data-stu-id="f15e4-134">There are three scenarios that the event-based pattern does not address as well as the <xref:System.IAsyncResult> pattern:</span></span>  
  
-   <span data-ttu-id="f15e4-135">Bloqueio da espera em um <xref:System.IAsyncResult></span><span class="sxs-lookup"><span data-stu-id="f15e4-135">Blocking wait on one <xref:System.IAsyncResult></span></span>  
  
-   <span data-ttu-id="f15e4-136">Bloqueio da espera em vários objetos <xref:System.IAsyncResult></span><span class="sxs-lookup"><span data-stu-id="f15e4-136">Blocking wait on many <xref:System.IAsyncResult> objects</span></span>  
  
-   <span data-ttu-id="f15e4-137">Sondagem de conclusão no <xref:System.IAsyncResult></span><span class="sxs-lookup"><span data-stu-id="f15e4-137">Polling for completion on the <xref:System.IAsyncResult></span></span>  
  
 <span data-ttu-id="f15e4-138">Você pode lidar com esses cenários usando o padrão baseado em evento, mas isso é mais complicado do que usar o padrão <xref:System.IAsyncResult>.</span><span class="sxs-lookup"><span data-stu-id="f15e4-138">You can address these scenarios by using the event-based pattern, but doing so is more cumbersome than using the <xref:System.IAsyncResult> pattern.</span></span>  
  
 <span data-ttu-id="f15e4-139">Os desenvolvedores geralmente usam o padrão <xref:System.IAsyncResult> em serviços que normalmente têm requisitos de desempenho muito elevados.</span><span class="sxs-lookup"><span data-stu-id="f15e4-139">Developers often use the <xref:System.IAsyncResult> pattern for services that typically have very high performance requirements.</span></span> <span data-ttu-id="f15e4-140">Por exemplo, a sondagem do cenário de conclusão é uma técnica de servidor de alto desempenho.</span><span class="sxs-lookup"><span data-stu-id="f15e4-140">For example, the polling for completion scenario is a high-performance server technique.</span></span>  
  
 <span data-ttu-id="f15e4-141">Além disso, o padrão baseado em evento é menos eficiente que o padrão <xref:System.IAsyncResult> porque cria mais objetos, especialmente o <xref:System.EventArgs>, e por sincronizar entre threads.</span><span class="sxs-lookup"><span data-stu-id="f15e4-141">Additionally, the event-based pattern is less efficient than the <xref:System.IAsyncResult> pattern because it creates more objects, especially <xref:System.EventArgs>, and because it synchronizes across threads.</span></span>  
  
 <span data-ttu-id="f15e4-142">A lista a seguir mostra algumas recomendações a serem seguidas se você decidir usar o padrão <xref:System.IAsyncResult>:</span><span class="sxs-lookup"><span data-stu-id="f15e4-142">The following list shows some recommendations to follow if you decide to use the <xref:System.IAsyncResult> pattern:</span></span>  
  
-   <span data-ttu-id="f15e4-143">Só exponha o padrão <xref:System.IAsyncResult> quando você precisar de suporte para os objetos <xref:System.Threading.WaitHandle> ou <xref:System.IAsyncResult>.</span><span class="sxs-lookup"><span data-stu-id="f15e4-143">Only expose the <xref:System.IAsyncResult> pattern when you specifically require support for <xref:System.Threading.WaitHandle> or <xref:System.IAsyncResult> objects.</span></span>  
  
-   <span data-ttu-id="f15e4-144">Só exponha o padrão <xref:System.IAsyncResult> quando você tiver uma API existente que usa o padrão <xref:System.IAsyncResult>.</span><span class="sxs-lookup"><span data-stu-id="f15e4-144">Only expose the <xref:System.IAsyncResult> pattern when you have an existing API that uses the <xref:System.IAsyncResult> pattern.</span></span>  
  
-   <span data-ttu-id="f15e4-145">Se você já tiver uma API baseada no padrão <xref:System.IAsyncResult>, considere também expor o padrão baseado em evento em sua próxima versão.</span><span class="sxs-lookup"><span data-stu-id="f15e4-145">If you have an existing API based on the <xref:System.IAsyncResult> pattern, consider also exposing the event-based pattern in your next release.</span></span>  
  
-   <span data-ttu-id="f15e4-146">Só exponha o padrão <xref:System.IAsyncResult> caso tiver requisitos de alto desempenho que você verificou não poderem ser atendidos pelo padrão baseado em evento, mas que podem ser atendidos pelo padrão <xref:System.IAsyncResult>.</span><span class="sxs-lookup"><span data-stu-id="f15e4-146">Only expose <xref:System.IAsyncResult> pattern if you have high performance requirements which you have verified cannot be met by the event-based pattern but can be met by the <xref:System.IAsyncResult> pattern.</span></span>
