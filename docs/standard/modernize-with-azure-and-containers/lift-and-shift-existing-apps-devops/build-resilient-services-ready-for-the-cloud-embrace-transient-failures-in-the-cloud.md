<span data-ttu-id="f415a-106">A resiliência é a capacidade de recuperar de falhas e continuar a funcionar.</span><span class="sxs-lookup"><span data-stu-id="f415a-106">Resiliency is the ability to recover from failures and continue to function.</span></span> <span data-ttu-id="f415a-107">Resiliência não é sobre como evitar falhas, mas aceitando o fato de que as falhas ocorrerão e, em seguida, responder a eles de maneira que evita o tempo de inatividade ou perda de dados.</span><span class="sxs-lookup"><span data-stu-id="f415a-107">Resiliency is not about avoiding failures, but accepting the fact that failures will occur, and then responding to them in a way that avoids downtime or data loss.</span></span> <span data-ttu-id="f415a-108">A meta de resiliência é retornar o aplicativo para um estado totalmente funcional após uma falha.</span><span class="sxs-lookup"><span data-stu-id="f415a-108">The goal of resiliency is to return the application to a fully functioning state after a failure.</span></span>

<span data-ttu-id="f415a-109">O aplicativo está pronto para a nuvem quando, no mínimo, ele implementa um modelo baseado em software de resiliência, em vez de um modelo baseado em hardware.</span><span class="sxs-lookup"><span data-stu-id="f415a-109">Your application is ready for the cloud when, at a minimum, it implements a software-based model of resiliency, rather than a hardware-based model.</span></span> <span data-ttu-id="f415a-110">Seu aplicativo de nuvem deve compreender as falhas parciais que certamente ocorrerá.</span><span class="sxs-lookup"><span data-stu-id="f415a-110">Your cloud application must embrace the partial failures that will certainly occur.</span></span> <span data-ttu-id="f415a-111">Você precisa criar ou parcialmente refatorar seu aplicativo, se você quiser obter resiliência a falhas parciais esperadas.</span><span class="sxs-lookup"><span data-stu-id="f415a-111">You need to design or partially refactor your application if you want to achieve resiliency to expected partial failures.</span></span> <span data-ttu-id="f415a-112">Deve ser projetado para lidar com falhas parciais, como interrupções de rede transitório e nós ou VMs falhas na nuvem.</span><span class="sxs-lookup"><span data-stu-id="f415a-112">It should be designed to cope with partial failures, like transient network outages and nodes, or VMs crashing in the cloud.</span></span> <span data-ttu-id="f415a-113">Contêineres mesmo sendo movidos para outro nó em um cluster do orchestrator podem causar falhas intermitentes de curtas dentro do aplicativo.</span><span class="sxs-lookup"><span data-stu-id="f415a-113">Even containers being moved to a different node within an orchestrator cluster can cause intermittent short failures within the application.</span></span>

## <a name="handling-partial-failure"></a><span data-ttu-id="f415a-114">Tratando falha parcial</span><span class="sxs-lookup"><span data-stu-id="f415a-114">Handling partial failure</span></span>

<span data-ttu-id="f415a-115">Em um aplicativo baseado em nuvem, há um risco de falha parcial de sempre presente.</span><span class="sxs-lookup"><span data-stu-id="f415a-115">In a cloud-based application, there's an ever-present risk of partial failure.</span></span> <span data-ttu-id="f415a-116">Por exemplo, uma instância de site único ou um contêiner pode falhar, ou ele pode estar indisponível ou não responder por um curto período.</span><span class="sxs-lookup"><span data-stu-id="f415a-116">For instance, a single website instance or a container might fail, or it might be unavailable or unresponsive for a short time.</span></span> <span data-ttu-id="f415a-117">Ou, um único servidor ou VM pode falhar.</span><span class="sxs-lookup"><span data-stu-id="f415a-117">Or, a single VM or server might crash.</span></span>

<span data-ttu-id="f415a-118">Como os clientes e serviços são processos separados, um serviço não poderá responder de maneira oportuna a solicitação do cliente.</span><span class="sxs-lookup"><span data-stu-id="f415a-118">Because clients and services are separate processes, a service might not be able to respond in a timely manner to a client's request.</span></span> <span data-ttu-id="f415a-119">O serviço pode estar sobrecarregado e muito lentamente responder às solicitações, ou ele pode simplesmente não estar acessível por um curto período devido a problemas de rede.</span><span class="sxs-lookup"><span data-stu-id="f415a-119">The service might be overloaded and respond extremely slowly to requests, or it might simply not be accessible for a short time because of network issues.</span></span>

<span data-ttu-id="f415a-120">Por exemplo, considere um aplicativo .NET monolítico que acessa um banco de dados no banco de dados do SQL Azure.</span><span class="sxs-lookup"><span data-stu-id="f415a-120">For example, consider a monolithic .NET application that accesses a database in Azure SQL Database.</span></span> <span data-ttu-id="f415a-121">Se o banco de dados do SQL Azure ou qualquer outro serviço de terceiros não está respondendo para um breve tempo (um banco de dados do SQL Azure pode ser movido para um nó diferente ou servidor e ficar sem resposta por alguns segundos), quando o usuário tenta realizar qualquer ação, o aplicativo pode falhar e mostrar w uma exceção no mesmo momento.</span><span class="sxs-lookup"><span data-stu-id="f415a-121">If the Azure SQL database or any other third-party service is unresponsive for a brief time (an Azure SQL database might be moved to a different node or server, and be unresponsive for a few seconds), when the user tries to do any action, the application might crash and show an exception at the very same moment.</span></span>

<span data-ttu-id="f415a-122">Um cenário semelhante pode ocorrer em um aplicativo que consome serviços HTTP.</span><span class="sxs-lookup"><span data-stu-id="f415a-122">A similar scenario might occur in an app that consumes HTTP services.</span></span> <span data-ttu-id="f415a-123">A rede ou o próprio serviço pode não estar disponível na nuvem durante uma falha transitória, curta.</span><span class="sxs-lookup"><span data-stu-id="f415a-123">The network or the service itself might not be available in the cloud during a short, transient failure.</span></span>

<span data-ttu-id="f415a-124">Um aplicativo resiliente, como mostrado na Figura 4-9 deve implementar técnicas, como "repetições com retirada exponencial" para que o aplicativo tenha a oportunidade para lidar com falhas transitórias de recursos.</span><span class="sxs-lookup"><span data-stu-id="f415a-124">A resilient application like the one shown in Figure 4-9 should implement techniques like "retries with exponential backoff" to give the application an opportunity to handle transient failures in resources.</span></span> <span data-ttu-id="f415a-125">Você também deve usar "disjuntores" em seus aplicativos.</span><span class="sxs-lookup"><span data-stu-id="f415a-125">You also should use "circuit breakers" in your applications.</span></span> <span data-ttu-id="f415a-126">Um disjuntor interrompe um aplicativo tentar acessar um recurso quando ela é realmente uma falha de longo prazo.</span><span class="sxs-lookup"><span data-stu-id="f415a-126">A circuit breaker stops an application from trying to access a resource when it's actually a long-term failure.</span></span> <span data-ttu-id="f415a-127">Ao usar um disjuntor, o aplicativo evita provocar uma negação de serviço para si mesmo.</span><span class="sxs-lookup"><span data-stu-id="f415a-127">By using a circuit breaker, the application avoids provoking a denial of service to itself.</span></span>

<span data-ttu-id="f415a-135">Ao implementar comunicações resilientes que acessam o banco de dados, dependendo da versão do .NET que você está usando, pode ser muito simples (por exemplo, [com o Entity Framework 6 ou posterior](https://msdn.microsoft.com/library/dn456835(v=vs.113).aspx), é apenas uma questão de configurar o conexão de banco de dados).</span><span class="sxs-lookup"><span data-stu-id="f415a-135">When implementing resilient communications that access the database, depending on the version of .NET you are using, it can be very straightforward (for example, [with Entity Framework 6 or later](https://msdn.microsoft.com/library/dn456835(v=vs.113).aspx), it's just a matter of configuring the database connection).</span></span> <span data-ttu-id="f415a-136">Ou, talvez seja necessário usar as bibliotecas adicionais como o [Transient Fault Handling Application Block](https://msdn.microsoft.com/library/hh680934(v=pandp.50).aspx) (para versões anteriores do .NET), ou até mesmo implementar sua própria biblioteca.</span><span class="sxs-lookup"><span data-stu-id="f415a-136">Or, you might need to use additional libraries like the [Transient Fault Handling Application Block](https://msdn.microsoft.com/library/hh680934(v=pandp.50).aspx) (for earlier versions of .NET), or even implement your own library.</span></span>
