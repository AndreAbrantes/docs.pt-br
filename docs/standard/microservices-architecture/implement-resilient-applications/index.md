<span data-ttu-id="7e681-104">*Seus aplicativos de microsserviços e baseados em nuvem devem compreender as falhas parciais que certamente poderão ocorrerão. <span data-ttu-id="7e681-105">A resiliência é a capacidade de recuperar de falhas e continuar a funcionar.</span><span class="sxs-lookup"><span data-stu-id="7e681-105">Resiliency is the ability to recover from failures and continue to function.</span></span> <span data-ttu-id="7e681-106">Não se trata de evitar falhas, mas de aceitar o fato de que as falhas acontecerão e de responder a elas de uma maneira que evite o tempo de inatividade ou a perda de dados.</span><span class="sxs-lookup"><span data-stu-id="7e681-106">It is not about avoiding failures, but accepting the fact that failures will happen and responding to them in a way that avoids downtime or data loss.</span></span> <span data-ttu-id="7e681-107">A meta de resiliência é retornar o aplicativo para um estado totalmente funcional após uma falha.</span><span class="sxs-lookup"><span data-stu-id="7e681-107">The goal of resiliency is to return the application to a fully functioning state after a failure.</span></span>

<span data-ttu-id="7e681-108">Já é um grande desafio criar e implantar um aplicativo baseado em microsserviços.</span><span class="sxs-lookup"><span data-stu-id="7e681-108">It is challenging enough to design and deploy a microservices-based application.</span></span> <span data-ttu-id="7e681-109">E você ainda precisa manter o aplicativo em execução em um ambiente em que algum tipo de falha certamente ocorrerá.</span><span class="sxs-lookup"><span data-stu-id="7e681-109">But you also need to keep your application running in an environment where some sort of failure is certain.</span></span> <span data-ttu-id="7e681-110">Portanto, seu aplicativo precisa ser resiliente.</span><span class="sxs-lookup"><span data-stu-id="7e681-110">Therefore, your application should be resilient.</span></span> <span data-ttu-id="7e681-111">Ele deve ser projetado para lidar com falhas parciais, como interrupções da rede ou falhas de nós ou de VMs na nuvem.</span><span class="sxs-lookup"><span data-stu-id="7e681-111">It should be designed to cope with partial failures, like network outages or nodes or VMs crashing in the cloud.</span></span> <span data-ttu-id="7e681-112">Até mesmo os microsserviços (contêineres) que estão sendo movidos para outro nó em um cluster podem causar falhas curtas intermitentes no aplicativo.</span><span class="sxs-lookup"><span data-stu-id="7e681-112">Even microservices (containers) being moved to a different node within a cluster can cause intermittent short failures within the application.</span></span>
