## <a name="the-specification-pattern"></a><span data-ttu-id="b0f86-184">O padrão de especificação</span><span class="sxs-lookup"><span data-stu-id="b0f86-184">The Specification pattern</span></span>

<span data-ttu-id="b0f86-185">O padrão de especificação (cujo nome completo é padrão de especificação de consulta) é um padrão de DDD projetado para ser o local em que você pode colocar a definição de uma consulta com uma lógica opcional de classificação e paginação.</span><span class="sxs-lookup"><span data-stu-id="b0f86-185">The specification pattern (its full name would be Query-specification pattern) is a DDD pattern designed as the place where you can put the definition of a query with optional sorting and paging logic.</span></span>

<span data-ttu-id="b0f86-186">O padrão de especificação define uma consulta em um objeto.</span><span class="sxs-lookup"><span data-stu-id="b0f86-186">The specification pattern defines a query in an object.</span></span> <span data-ttu-id="b0f86-187">Por exemplo, para encapsular uma consulta paginada que procura alguns produtos, você pode criar uma especificação `PagedProduct` que usa os parâmetros de entrada necessários, como `pageNumber`, `pageSize`, `filter` etc. Assim, qualquer método de repositório [geralmente uma sobrecarga List()] aceitaria uma `ISpecification` e executaria a consulta esperada com base nessa especificação.</span><span class="sxs-lookup"><span data-stu-id="b0f86-187">For example, to encapsulate a paged query that searches for some products, you can create a `PagedProduct` specification that takes the necessary input parameters, such as `pageNumber`, `pageSize`, `filter`, etc. Then, within any Repository method (usually a List() overload), it'd accept an `ISpecification` and run the expected query based on that specification.</span></span>

<span data-ttu-id="b0f86-188">Essa abordagem oferece vários benefícios:</span><span class="sxs-lookup"><span data-stu-id="b0f86-188">There are several benefits to this approach:</span></span>

- <span data-ttu-id="b0f86-189">A especificação tem um nome (em vez de apenas um conjunto de expressões LINQ) sobre o qual é possível discutir.</span><span class="sxs-lookup"><span data-stu-id="b0f86-189">The specification has a name (as opposed to just a bunch of LINQ expressions) that you can discuss about.</span></span>

- <span data-ttu-id="b0f86-190">A especificação pode passar por um teste de unidade isoladamente para garantir que ela esteja correta.</span><span class="sxs-lookup"><span data-stu-id="b0f86-190">The specification can be unit tested in isolation to ensure it's right.</span></span> <span data-ttu-id="b0f86-191">Ela também poderá ser reutilizada facilmente se você precisar de um comportamento semelhante.</span><span class="sxs-lookup"><span data-stu-id="b0f86-191">It can also easily be reused if you need similar behavior.</span></span> <span data-ttu-id="b0f86-192">Por exemplo, em uma ação de exibição de MVC e em uma ação de API Web, bem como em vários serviços.</span><span class="sxs-lookup"><span data-stu-id="b0f86-192">For example, on an MVC View action and a Web API action, as well as in various services.</span></span>

- <span data-ttu-id="b0f86-193">Uma especificação também pode ser usada para descrever a forma dos dados a serem retornados, para que as consultas possam retornar apenas os dados necessários.</span><span class="sxs-lookup"><span data-stu-id="b0f86-193">A specification can also be used to describe the shape of the data to be returned, so that queries can return just the data they required.</span></span> <span data-ttu-id="b0f86-194">Isso elimina a necessidade de carregamento lento em aplicativos Web, que geralmente não é recomendado, e ajuda a eliminar a complexidade causada pelo detalhamento das implementações do repositório.</span><span class="sxs-lookup"><span data-stu-id="b0f86-194">This eliminates the need for lazy loading in web applications, which is usually not recommended, and helps keep repository implementations from becoming cluttered with these details.</span></span>
