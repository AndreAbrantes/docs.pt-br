<span data-ttu-id="a099f-105">O CQRS é um padrão de arquitetura que separa os modelos para ler e gravar dados.</span><span class="sxs-lookup"><span data-stu-id="a099f-105">CQRS is an architectural pattern that separates the models for reading and writing data.</span></span> <span data-ttu-id="a099f-106">O termo relacionado [CQS (Separação de Comando-Consulta)](https://martinfowler.com/bliki/CommandQuerySeparation.html) foi originalmente definido por Bertrand Meyer em seu livro *Object Oriented Software Construction* (Construção de software orientada a objeto).</span><span class="sxs-lookup"><span data-stu-id="a099f-106">The related term [Command Query Separation (CQS)](https://martinfowler.com/bliki/CommandQuerySeparation.html) was originally defined by Bertrand Meyer in his book *Object Oriented Software Construction*.</span></span> <span data-ttu-id="a099f-107">A ideia básica é que você pode dividir operações de um sistema em duas categorias nitidamente separadas:</span><span class="sxs-lookup"><span data-stu-id="a099f-107">The basic idea is that you can divide a system’s operations into two sharply separated categories:</span></span>

-   <span data-ttu-id="a099f-108">Consultas.</span><span class="sxs-lookup"><span data-stu-id="a099f-108">Queries.</span></span> <span data-ttu-id="a099f-109">Essas retornam um resultado, não alteram o estado do sistema e são livres de efeitos colaterais.</span><span class="sxs-lookup"><span data-stu-id="a099f-109">These return a result and do not change the state of the system, and they are free of side effects.</span></span>

-   <span data-ttu-id="a099f-110">Comandos.</span><span class="sxs-lookup"><span data-stu-id="a099f-110">Commands.</span></span> <span data-ttu-id="a099f-111">Esses alteram o estado de um sistema.</span><span class="sxs-lookup"><span data-stu-id="a099f-111">These change the state of a system.</span></span>

<span data-ttu-id="a099f-112">O CQS é um conceito simples — trata-se de métodos dentro do mesmo objeto sendo consultas ou comandos.</span><span class="sxs-lookup"><span data-stu-id="a099f-112">CQS is a simple concept—it is about methods within the same object being either queries or commands.</span></span> <span data-ttu-id="a099f-113">Cada método retorna um estado ou muda um estado, mas não ambos.</span><span class="sxs-lookup"><span data-stu-id="a099f-113">Each method either returns state or mutates state, but not both.</span></span> <span data-ttu-id="a099f-114">Até mesmo um único objeto de padrão de repositório pode estar em conformidade com o CQS.</span><span class="sxs-lookup"><span data-stu-id="a099f-114">Even a single repository pattern object can comply with CQS.</span></span> <span data-ttu-id="a099f-115">O CQS pode ser considerado um princípio fundamental para o CQRS.</span><span class="sxs-lookup"><span data-stu-id="a099f-115">CQS can be considered a foundational principle for CQRS.</span></span>

<span data-ttu-id="a099f-116">O [CQRS (Segregação de Responsabilidade de Consulta e Comando)](https://martinfowler.com/bliki/CQRS.html) foi introduzido por Greg Young e altamente promovido por Udi Dahan e outros.</span><span class="sxs-lookup"><span data-stu-id="a099f-116">[Command and Query Responsibility Segregation (CQRS)](https://martinfowler.com/bliki/CQRS.html) was introduced by Greg Young and strongly promoted by Udi Dahan and others.</span></span> <span data-ttu-id="a099f-117">Ele se baseia no princípio do CQS, embora seja mais detalhado.</span><span class="sxs-lookup"><span data-stu-id="a099f-117">It is based on the CQS principle, although it is more detailed.</span></span> <span data-ttu-id="a099f-118">Ele pode ser considerado um padrão com base em comandos e eventos, além de ser opcional em mensagens assíncronas.</span><span class="sxs-lookup"><span data-stu-id="a099f-118">It can be considered a pattern based on commands and events plus optionally on asynchronous messages.</span></span> <span data-ttu-id="a099f-119">Em muitos casos, o CQRS está relacionado a cenários mais avançados, como ter um banco de dados físico para leituras (consultas) diferente do banco de dados para gravações (atualizações).</span><span class="sxs-lookup"><span data-stu-id="a099f-119">In many cases, CQRS is related to more advanced scenarios, like having a different physical database for reads (queries) than for writes (updates).</span></span> <span data-ttu-id="a099f-120">Além disso, um sistema CQRS mais evoluído pode implementar [ES (fonte de eventos)](http://codebetter.com/gregyoung/2010/02/20/why-use-event-sourcing/) em seu banco de dados de atualizações. Assim, você deve apenas armazenar eventos no modelo de domínio, em vez de armazenar os dados do estado atual.</span><span class="sxs-lookup"><span data-stu-id="a099f-120">Moreover, a more evolved CQRS system might implement [Event-Sourcing (ES)](http://codebetter.com/gregyoung/2010/02/20/why-use-event-sourcing/) for your updates database, so you would only store events in the domain model instead of storing the current-state data.</span></span> <span data-ttu-id="a099f-121">No entanto, essa não é a abordagem usada neste guia; estamos usando a abordagem mais simples de CQRS, que consiste em separar apenas as consultas dos comandos.</span><span class="sxs-lookup"><span data-stu-id="a099f-121">However, this is not the approach used in this guide; we are using the simplest CQRS approach, which consists of just separating the queries from the commands.</span></span>

<span data-ttu-id="a099f-122">O aspecto de separação do CQRS é obtido pelo agrupamento de operações de consulta em uma camada e comandos em outra camada.</span><span class="sxs-lookup"><span data-stu-id="a099f-122">The separation aspect of CQRS is achieved by grouping query operations in one layer and commands in another layer.</span></span> <span data-ttu-id="a099f-123">Cada camada tem seu próprio modelo de dados (observe que dizemos modelo, não necessariamente um banco de dados diferente) e é criada usando sua própria combinação de padrões e tecnologias.</span><span class="sxs-lookup"><span data-stu-id="a099f-123">Each layer has its own data model (note that we say model, not necessarily a different database) and is built using its own combination of patterns and technologies.</span></span> <span data-ttu-id="a099f-124">Além disso, as duas camadas podem estar dentro do mesmo nível ou microsserviço, como no exemplo (microsserviço de ordenação) usado para este guia.</span><span class="sxs-lookup"><span data-stu-id="a099f-124">More importantly, the two layers can be within the same tier or microservice, as in the example (ordering microservice) used for this guide.</span></span> <span data-ttu-id="a099f-125">Ou elas podem ser implementadas em diferentes microsserviços ou processos para que possam ser otimizadas e expandidas separadamente sem afetar umas às outras.</span><span class="sxs-lookup"><span data-stu-id="a099f-125">Or they could be implemented on different microservices or processes so they can be optimized and scaled out separately without affecting one another.</span></span>

<span data-ttu-id="a099f-126">CQRS significa ter dois objetos para uma operação de leitura/gravação, em que, em outros, há um.</span><span class="sxs-lookup"><span data-stu-id="a099f-126">CQRS means having two objects for a read/write operation where in other contexts there is one.</span></span> <span data-ttu-id="a099f-127">Há motivos para ter um banco de dados de leitura desnormalizado, sobre o qual você pode aprender na literatura sobre CQRS mais avançada.</span><span class="sxs-lookup"><span data-stu-id="a099f-127">There are reasons to have a denormalized reads database, which you can learn about in more advanced CQRS literature.</span></span> <span data-ttu-id="a099f-128">Mas não estamos usando essa abordagem aqui, em que a meta é ter mais flexibilidade nas consultas, em vez de limitá-las com restrições de padrões DDD como agregações.</span><span class="sxs-lookup"><span data-stu-id="a099f-128">But we are not using that approach here, where the goal is to have more flexibility in the queries instead of limiting the queries with constraints from DDD patterns like aggregates.</span></span>
