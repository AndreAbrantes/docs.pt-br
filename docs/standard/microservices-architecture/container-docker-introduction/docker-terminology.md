# <a name="docker-terminology"></a><span data-ttu-id="ca632-104">Terminologia de docker</span><span class="sxs-lookup"><span data-stu-id="ca632-104">Docker terminology</span></span>

<span data-ttu-id="ca632-105">Esta seção lista os termos e definições que você deve se familiarizar antes de obter maior no Docker.</span><span class="sxs-lookup"><span data-stu-id="ca632-105">This section lists terms and definitions you should be familiar with before getting deeper into Docker.</span></span> <span data-ttu-id="ca632-106">Para obter mais definições, consulte o amplo [glossário](https://docs.docker.com/v1.11/engine/reference/glossary/) fornecida pelo Docker.</span><span class="sxs-lookup"><span data-stu-id="ca632-106">For further definitions, see the extensive [glossary](https://docs.docker.com/v1.11/engine/reference/glossary/) provided by Docker .</span></span>

<span data-ttu-id="ca632-107">**Imagem de contêiner**: um pacote com todas as dependências e as informações necessárias para criar um contêiner.</span><span class="sxs-lookup"><span data-stu-id="ca632-107">**Container image**: A package with all the dependencies and information needed to create a container.</span></span> <span data-ttu-id="ca632-108">Uma imagem inclui todas as dependências (como estruturas) além de implantação e execução de configuração a ser usada por um tempo de execução do contêiner.</span><span class="sxs-lookup"><span data-stu-id="ca632-108">An image includes all the dependencies (such as frameworks) plus deployment and execution configuration to be used by a container runtime.</span></span> <span data-ttu-id="ca632-109">Geralmente, uma imagem é derivado de várias imagens de base são empilhadas uns sobre os outros para formar o sistema de arquivos do contêiner de camadas.</span><span class="sxs-lookup"><span data-stu-id="ca632-109">Usually, an image derives from multiple base images that are layers stacked on top of each other to form the container’s filesystem.</span></span> <span data-ttu-id="ca632-110">Uma imagem é imutável depois que ela foi criada.</span><span class="sxs-lookup"><span data-stu-id="ca632-110">An image is immutable once it has been created.</span></span>

<span data-ttu-id="ca632-111">**Contêiner**: uma instância de uma imagem do Docker.</span><span class="sxs-lookup"><span data-stu-id="ca632-111">**Container**: An instance of a Docker image.</span></span> <span data-ttu-id="ca632-112">Um contêiner representa a execução de um único aplicativo, processo ou serviço.</span><span class="sxs-lookup"><span data-stu-id="ca632-112">A container represents the execution of a single application, process, or service.</span></span> <span data-ttu-id="ca632-113">Ele consiste no conteúdo de uma imagem do Docker, um ambiente de execução e um conjunto padrão de instruções.</span><span class="sxs-lookup"><span data-stu-id="ca632-113">It consists of the contents of a Docker image, an execution environment, and a standard set of instructions.</span></span> <span data-ttu-id="ca632-114">Ao dimensionar um serviço, você cria várias instâncias de um contêiner da mesma imagem.</span><span class="sxs-lookup"><span data-stu-id="ca632-114">When scaling a service, you create multiple instances of a container from the same image.</span></span> <span data-ttu-id="ca632-115">Ou um trabalho em lotes pode criar vários contêineres da mesma imagem, passando parâmetros diferentes para cada instância.</span><span class="sxs-lookup"><span data-stu-id="ca632-115">Or a batch job can create multiple containers from the same image, passing different parameters to each instance.</span></span>

<span data-ttu-id="ca632-116">**Marca**: uma marca ou um rótulo, você pode aplicar a imagens para que podem ser identificadas diferentes imagens ou versões da mesma imagem (dependendo do número de versão ou o ambiente de destino).</span><span class="sxs-lookup"><span data-stu-id="ca632-116">**Tag**: A mark or label you can apply to images so that different images or versions of the same image (depending on the version number or the target environment) can be identified.</span></span>

<span data-ttu-id="ca632-117">**Dockerfile**: um arquivo de texto que contém instruções sobre como criar uma imagem do Docker.</span><span class="sxs-lookup"><span data-stu-id="ca632-117">**Dockerfile**: A text file that contains instructions for how to build a Docker image.</span></span>

<span data-ttu-id="ca632-118">**Criar**: A ação de criação de uma imagem de contêiner com base nas informações e contexto fornecido pelo seu Dockerfile, além de arquivos adicionais na pasta em que a imagem é criada.</span><span class="sxs-lookup"><span data-stu-id="ca632-118">**Build**: The action of building a container image based on the information and context provided by its Dockerfile, plus additional files in the folder where the image is built.</span></span> <span data-ttu-id="ca632-119">Você pode criar imagens com o comando de compilação de docker do Docker.</span><span class="sxs-lookup"><span data-stu-id="ca632-119">You can build images with the Docker docker build command.</span></span>

<span data-ttu-id="ca632-120">**Repositório (repositório)**: uma coleção de imagens relacionadas do Docker, rotulado com uma marca que indica a versão da imagem.</span><span class="sxs-lookup"><span data-stu-id="ca632-120">**Repository (repo)**: A collection of related Docker images, labeled with a tag that indicates the image version.</span></span> <span data-ttu-id="ca632-121">Alguns repositórios contêm várias variantes de uma imagem específica, como uma imagem que contém o SDKs (mais), uma imagem que contém somente tempos de execução (mais leves), etc. Essas variantes podem ser marcadas com marcas.</span><span class="sxs-lookup"><span data-stu-id="ca632-121">Some repos contain multiple variants of a specific image, such as an image containing SDKs (heavier), an image containing only runtimes (lighter), etc. Those variants can be marked with tags.</span></span> <span data-ttu-id="ca632-122">Um único repositório pode conter variantes de plataforma, como uma imagem do Linux e uma imagem do Windows.</span><span class="sxs-lookup"><span data-stu-id="ca632-122">A single repo can contain platform variants, such as a Linux image and a Windows image.</span></span>

<span data-ttu-id="ca632-123">**Registro**: um serviço que fornece acesso aos repositórios.</span><span class="sxs-lookup"><span data-stu-id="ca632-123">**Registry**: A service that provides access to repositories.</span></span> <span data-ttu-id="ca632-124">O registro padrão para as imagens mais públicas está [Docker Hub](https://hub.docker.com/) (propriedade Docker como uma organização).</span><span class="sxs-lookup"><span data-stu-id="ca632-124">The default registry for most public images is [Docker Hub](https://hub.docker.com/) (owned by Docker as an organization).</span></span> <span data-ttu-id="ca632-125">Um registro geralmente contém repositórios de várias equipes.</span><span class="sxs-lookup"><span data-stu-id="ca632-125">A registry usually contains repositories from multiple teams.</span></span> <span data-ttu-id="ca632-126">As empresas geralmente têm registros privados para armazenar e gerenciar as imagens que você criou.</span><span class="sxs-lookup"><span data-stu-id="ca632-126">Companies often have private registries to store and manage images they’ve created.</span></span> <span data-ttu-id="ca632-127">Registro de contêiner do Azure é outro exemplo.</span><span class="sxs-lookup"><span data-stu-id="ca632-127">Azure Container Registry is another example.</span></span>

<span data-ttu-id="ca632-128">**Hub do docker**: um registro público para carregar imagens e trabalhar com eles.</span><span class="sxs-lookup"><span data-stu-id="ca632-128">**Docker Hub**: A public registry to upload images and work with them.</span></span> <span data-ttu-id="ca632-129">Hub do docker fornece Docker hospedagem de imagem, registros públicos ou privados, gatilhos de compilação e ganchos web e integração com o GitHub, Bitbucket.</span><span class="sxs-lookup"><span data-stu-id="ca632-129">Docker Hub provides Docker image hosting, public or private registries, build triggers and web hooks, and integration with GitHub and Bitbucket.</span></span>

<span data-ttu-id="ca632-130">**Registro de contêiner do Azure**: um recurso público para trabalhar com imagens do Docker e seus componentes no Azure.</span><span class="sxs-lookup"><span data-stu-id="ca632-130">**Azure Container Registry**: A public resource for working with Docker images and its components in Azure.</span></span> <span data-ttu-id="ca632-131">Isso fornece um registro que está perto de suas implantações no Azure e que fornece a você controle sobre o acesso, tornando possível usar suas permissões e grupos do Active Directory do Azure.</span><span class="sxs-lookup"><span data-stu-id="ca632-131">This provides a registry that is close to your deployments in Azure and that gives you control over access, making it possible to use your Azure Active Directory groups and permissions.</span></span>

<span data-ttu-id="ca632-132">**Docker confiáveis do registro (DTR)**: serviço de registro de Docker de um (a partir do Docker) que pode ser instalado localmente para que ela reside no datacenter e a rede da organização.</span><span class="sxs-lookup"><span data-stu-id="ca632-132">**Docker Trusted Registry (DTR)**: A Docker registry service (from Docker) that can be installed on-premises so it lives within the organization’s datacenter and network.</span></span> <span data-ttu-id="ca632-133">É conveniente para imagens privadas que devem ser gerenciadas dentro da empresa.</span><span class="sxs-lookup"><span data-stu-id="ca632-133">It is convenient for private images that should be managed within the enterprise.</span></span> <span data-ttu-id="ca632-134">Registro confiável do docker é incluído como parte do produto Docker Datacenter.</span><span class="sxs-lookup"><span data-stu-id="ca632-134">Docker Trusted Registry is included as part of the Docker Datacenter product.</span></span> <span data-ttu-id="ca632-135">Para obter mais informações, consulte [Docker confiável do registro (DTR)](https://docs.docker.com/docker-trusted-registry/overview/).</span><span class="sxs-lookup"><span data-stu-id="ca632-135">For more information, see [Docker Trusted Registry (DTR)](https://docs.docker.com/docker-trusted-registry/overview/).</span></span>

<span data-ttu-id="ca632-136">**Docker Community Edition (CE)**: ferramentas de desenvolvimento para Windows e macOS para compilar, executar e testar contêineres localmente.</span><span class="sxs-lookup"><span data-stu-id="ca632-136">**Docker Community Edition (CE)**: Development tools for Windows and macOS for building, running, and testing containers locally.</span></span> <span data-ttu-id="ca632-137">CE do docker para Windows fornece os ambientes de desenvolvimento para Linux e contêineres do Windows.</span><span class="sxs-lookup"><span data-stu-id="ca632-137">Docker CE for Windows provides development environments for both Linux and Windows Containers.</span></span> <span data-ttu-id="ca632-138">O host do Docker do Linux no Windows é baseado em um [Hyper-V](https://www.microsoft.com/en-us/server-cloud/solutions/virtualization.aspx) máquina virtual.</span><span class="sxs-lookup"><span data-stu-id="ca632-138">The Linux Docker host on Windows is based on a [Hyper-V](https://www.microsoft.com/en-us/server-cloud/solutions/virtualization.aspx) virtual machine.</span></span> <span data-ttu-id="ca632-139">O host para contêineres do Windows baseia-se diretamente no Windows.</span><span class="sxs-lookup"><span data-stu-id="ca632-139">The host for Windows Containers is directly based on Windows.</span></span> <span data-ttu-id="ca632-140">Docker CE para Mac baseia-se na estrutura do hipervisor da Apple e a [xhyve hipervisor](https://github.com/mist64/xhyve), que fornece uma máquina virtual do host Linux Docker no Mac OS X. Docker CE para Windows e Mac substitui a caixa de ferramentas do Docker, que é baseado no Oracle VirtualBox.</span><span class="sxs-lookup"><span data-stu-id="ca632-140">Docker CE for Mac is based on the Apple Hypervisor framework and the [xhyve hypervisor](https://github.com/mist64/xhyve), which provides a Linux Docker host virtual machine on Mac OS X. Docker CE for Windows and for Mac replaces Docker Toolbox, which was based on Oracle VirtualBox.</span></span>

<span data-ttu-id="ca632-141">**Docker Enterprise Edition (EE)**: uma versão de grande porte das ferramentas do Docker para desenvolvimento Linux e Windows.</span><span class="sxs-lookup"><span data-stu-id="ca632-141">**Docker Enterprise Edition (EE)**: An enterprise-scale version of Docker tools for Linux and Windows development.</span></span>

<span data-ttu-id="ca632-142">**Compor**: uma ferramenta de linha de comando e YAML formato com metadados para definir e executar aplicativos de vários contêiner de arquivo.</span><span class="sxs-lookup"><span data-stu-id="ca632-142">**Compose**: A command-line tool and YAML file format with metadata for defining and running multi-container applications.</span></span> <span data-ttu-id="ca632-143">Você define um único aplicativo com base em várias imagens com um ou mais arquivos de .yml que podem substituir valores dependendo do ambiente.</span><span class="sxs-lookup"><span data-stu-id="ca632-143">You define a single application based on multiple images with one or more .yml files that can override values depending on the environment.</span></span> <span data-ttu-id="ca632-144">Depois que você criou as definições, você pode implantar o aplicativo inteiro vários contêiner com um único comando (docker-compor backup) que cria um contêiner por imagem no host do Docker.</span><span class="sxs-lookup"><span data-stu-id="ca632-144">After you have created the definitions, you can deploy the whole multi-container application with a single command (docker-compose up) that creates a container per image on the Docker host.</span></span>

<span data-ttu-id="ca632-145">**Cluster**: uma coleção de hosts de Docker expostos como se fosse um único host Docker virtual, para que o aplicativo pode ser dimensionado para várias instâncias dos serviços distribuídos em vários hosts do cluster.</span><span class="sxs-lookup"><span data-stu-id="ca632-145">**Cluster**: A collection of Docker hosts exposed as if it were a single virtual Docker host, so that the application can scale to multiple instances of the services spread across multiple hosts within the cluster.</span></span> <span data-ttu-id="ca632-146">Clusters de docker podem ser criados com o Docker Swarm, Mesosphere DC/OS, Kubernetes e Azure Service Fabric.</span><span class="sxs-lookup"><span data-stu-id="ca632-146">Docker clusters can be created with Docker Swarm, Mesosphere DC/OS, Kubernetes, and Azure Service Fabric.</span></span> <span data-ttu-id="ca632-147">(Se você usar o Docker Swarm para gerenciar um cluster, você normalmente se referir ao cluster como um *swarm* em vez de um cluster.)</span><span class="sxs-lookup"><span data-stu-id="ca632-147">(If you use Docker Swarm for managing a cluster, you typically refer to the cluster as a *swarm* instead of a cluster.)</span></span>

<span data-ttu-id="ca632-148">**Orchestrator**: uma ferramenta que simplifica o gerenciamento de clusters e hosts de Docker.</span><span class="sxs-lookup"><span data-stu-id="ca632-148">**Orchestrator**: A tool that simplifies management of clusters and Docker hosts.</span></span> <span data-ttu-id="ca632-149">Orchestrators permitem gerenciar suas imagens, contêineres e hosts por meio de uma interface de linha de comando (CLI) ou uma interface de usuário gráfica.</span><span class="sxs-lookup"><span data-stu-id="ca632-149">Orchestrators enable you to manage their images, containers, and hosts through a command line interface (CLI) or a graphical UI.</span></span> <span data-ttu-id="ca632-150">Você pode gerenciar a rede de contêiner, configurações, balanceamento de carga, descoberta de serviço, alta disponibilidade, configuração de host do Docker e muito mais.</span><span class="sxs-lookup"><span data-stu-id="ca632-150">You can manage container networking, configurations, load balancing, service discovery, high availability, Docker host configuration, and more.</span></span> <span data-ttu-id="ca632-151">Um orquestrador é responsável por executar, distribuir, escala e reparo de cargas de trabalho em uma coleção de nós.</span><span class="sxs-lookup"><span data-stu-id="ca632-151">An orchestrator is responsible for running, distributing, scaling, and healing workloads across a collection of nodes.</span></span> <span data-ttu-id="ca632-152">Normalmente, orchestrator produtos são o mesmo que fornecem infraestrutura de cluster, como Mesosphere DC/OS, Kubernetes, Docker Swarm e Azure Service Fabric.</span><span class="sxs-lookup"><span data-stu-id="ca632-152">Typically, orchestrator products are the same products that provide cluster infrastructure, like Mesosphere DC/OS, Kubernetes, Docker Swarm, and Azure Service Fabric.</span></span>
