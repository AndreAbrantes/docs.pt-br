# <a name="docker-terminology"></a><span data-ttu-id="ab3f2-103">Terminologia do Docker</span><span class="sxs-lookup"><span data-stu-id="ab3f2-103">Docker terminology</span></span>

<span data-ttu-id="ab3f2-104">Esta seção lista os termos e definições que você deve conhecer antes de se aprofundar no Docker.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-104">This section lists terms and definitions you should be familiar with before getting deeper into Docker.</span></span> <span data-ttu-id="ab3f2-105">Para obter mais definições, consulte o amplo [glossário](https://docs.docker.com/glossary/) fornecido pelo Docker.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-105">For further definitions, see the extensive [glossary](https://docs.docker.com/glossary/) provided by Docker .</span></span>

<span data-ttu-id="ab3f2-106">**Imagem de contêiner**: um pacote com todas as dependências e informações necessárias para criar um contêiner.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-106">**Container image**: A package with all the dependencies and information needed to create a container.</span></span> <span data-ttu-id="ab3f2-107">Uma imagem inclui todas as dependências (como estruturas), além da configuração de implantação e execução a ser usada por um tempo de execução de contêiner.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-107">An image includes all the dependencies (such as frameworks) plus deployment and execution configuration to be used by a container runtime.</span></span> <span data-ttu-id="ab3f2-108">Geralmente, uma imagem deriva de várias imagens base que são camadas empilhadas umas sobre as outras para formar o sistema de arquivos do contêiner.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-108">Usually, an image derives from multiple base images that are layers stacked on top of each other to form the container’s filesystem.</span></span> <span data-ttu-id="ab3f2-109">Uma imagem é imutável depois de ser criada.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-109">An image is immutable once it has been created.</span></span>

<span data-ttu-id="ab3f2-110">**Contêiner**: uma instância de uma imagem do Docker.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-110">**Container**: An instance of a Docker image.</span></span> <span data-ttu-id="ab3f2-111">Um contêiner representa a execução de um único aplicativo, processo ou serviço.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-111">A container represents the execution of a single application, process, or service.</span></span> <span data-ttu-id="ab3f2-112">Consiste no conteúdo de uma imagem do Docker, um ambiente de execução e um conjunto padrão de instruções.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-112">It consists of the contents of a Docker image, an execution environment, and a standard set of instructions.</span></span> <span data-ttu-id="ab3f2-113">Ao dimensionar um serviço, você cria várias instâncias de um contêiner da mesma imagem.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-113">When scaling a service, you create multiple instances of a container from the same image.</span></span> <span data-ttu-id="ab3f2-114">Ou um trabalho em lotes pode criar vários contêineres da mesma imagem, passando parâmetros diferentes para cada instância.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-114">Or a batch job can create multiple containers from the same image, passing different parameters to each instance.</span></span>

<span data-ttu-id="ab3f2-115">**Marcação**: uma marca ou um rótulo pode ser aplicado a imagens para que imagens ou versões diferentes da mesma imagem (dependendo do número de versão ou do ambiente de destino) possam ser identificadas.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-115">**Tag**: A mark or label you can apply to images so that different images or versions of the same image (depending on the version number or the target environment) can be identified.</span></span>

<span data-ttu-id="ab3f2-116">**Dockerfile**: um arquivo de texto que contém instruções sobre como criar uma imagem do Docker.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-116">**Dockerfile**: A text file that contains instructions for how to build a Docker image.</span></span>

<span data-ttu-id="ab3f2-117">**Build**: a ação de criar de uma imagem de contêiner com base nas informações e no contexto fornecido pelo Dockerfile, além de arquivos adicionais na pasta em que a imagem é criada.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-117">**Build**: The action of building a container image based on the information and context provided by its Dockerfile, plus additional files in the folder where the image is built.</span></span> <span data-ttu-id="ab3f2-118">Você pode criar imagens com o comando de build do Docker.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-118">You can build images with the Docker docker build command.</span></span>

<span data-ttu-id="ab3f2-119">**Repositório**: uma coleção de imagens do Docker relacionadas, rotulada com uma marcação que indica a versão da imagem.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-119">**Repository (repo)**: A collection of related Docker images, labeled with a tag that indicates the image version.</span></span> <span data-ttu-id="ab3f2-120">Alguns repositórios contêm diversas variantes de uma imagem específica, como uma imagem que contém o SDKs (mais pesada), uma imagem que contém somente tempos de execução (mais leves) etc. Essas variantes podem ser marcadas com marcas.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-120">Some repos contain multiple variants of a specific image, such as an image containing SDKs (heavier), an image containing only runtimes (lighter), etc. Those variants can be marked with tags.</span></span> <span data-ttu-id="ab3f2-121">Um único repositório pode conter variantes de plataforma, como uma imagem do Linux e uma imagem do Windows.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-121">A single repo can contain platform variants, such as a Linux image and a Windows image.</span></span>

<span data-ttu-id="ab3f2-122">**Registro**: um serviço que dá acesso aos repositórios.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-122">**Registry**: A service that provides access to repositories.</span></span> <span data-ttu-id="ab3f2-123">O registro padrão para as imagens mais públicas é o [Docker Hub](https://hub.docker.com/) (propriedade da Docker como uma organização).</span><span class="sxs-lookup"><span data-stu-id="ab3f2-123">The default registry for most public images is [Docker Hub](https://hub.docker.com/) (owned by Docker as an organization).</span></span> <span data-ttu-id="ab3f2-124">Um registro geralmente contém repositórios de várias equipes.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-124">A registry usually contains repositories from multiple teams.</span></span> <span data-ttu-id="ab3f2-125">As empresas geralmente têm registros privados para armazenar e gerenciar as imagens que criaram.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-125">Companies often have private registries to store and manage images they’ve created.</span></span> <span data-ttu-id="ab3f2-126">O Registro de Contêiner do Azure é outro exemplo.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-126">Azure Container Registry is another example.</span></span>

<span data-ttu-id="ab3f2-127">**Docker Hub**: um registro público para carregar imagens e trabalhar com elas.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-127">**Docker Hub**: A public registry to upload images and work with them.</span></span> <span data-ttu-id="ab3f2-128">O Docker Hub hospeda imagens do Docker, registros públicos ou privados, cria gatilhos e ganchos da Web e integra-se com o GitHub e o Bitbucket.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-128">Docker Hub provides Docker image hosting, public or private registries, build triggers and web hooks, and integration with GitHub and Bitbucket.</span></span>

<span data-ttu-id="ab3f2-129">**Registro de Contêiner do Azure**: um recurso público para trabalhar com imagens do Docker e seus componentes no Azure.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-129">**Azure Container Registry**: A public resource for working with Docker images and its components in Azure.</span></span> <span data-ttu-id="ab3f2-130">Fornece um registro que está perto de suas implantações no Azure e que permite controlar o acesso, tornando possível usar as permissões e os grupos do Azure Active Directory.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-130">This provides a registry that is close to your deployments in Azure and that gives you control over access, making it possible to use your Azure Active Directory groups and permissions.</span></span>

<span data-ttu-id="ab3f2-131">**Docker Trusted Registry (DTR)**: serviço de registro do Docker que pode ser instalado localmente para funcionar no datacenter e na rede da organização.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-131">**Docker Trusted Registry (DTR)**: A Docker registry service (from Docker) that can be installed on-premises so it lives within the organization’s datacenter and network.</span></span> <span data-ttu-id="ab3f2-132">É conveniente para imagens privadas que devem ser gerenciadas dentro da empresa.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-132">It is convenient for private images that should be managed within the enterprise.</span></span> <span data-ttu-id="ab3f2-133">O Docker Trusted Registry é parte do produto Docker Datacenter.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-133">Docker Trusted Registry is included as part of the Docker Datacenter product.</span></span> <span data-ttu-id="ab3f2-134">Para saber mais, consulte [Docker Trusted Registry (DTR)](https://docs.docker.com/docker-trusted-registry/overview/).</span><span class="sxs-lookup"><span data-stu-id="ab3f2-134">For more information, see [Docker Trusted Registry (DTR)](https://docs.docker.com/docker-trusted-registry/overview/).</span></span>

<span data-ttu-id="ab3f2-135">**Docker Community Edition (CE)**: ferramentas de desenvolvimento para Windows e macOS para build, execução e teste de contêineres localmente.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-135">**Docker Community Edition (CE)**: Development tools for Windows and macOS for building, running, and testing containers locally.</span></span> <span data-ttu-id="ab3f2-136">O Docker CE para Windows fornece os ambientes de desenvolvimento para Linux e contêineres do Windows.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-136">Docker CE for Windows provides development environments for both Linux and Windows Containers.</span></span> <span data-ttu-id="ab3f2-137">O host do Docker do Linux no Windows é baseado em uma máquina virtual [Hyper-V](https://www.microsoft.com/en-us/server-cloud/solutions/virtualization.aspx).</span><span class="sxs-lookup"><span data-stu-id="ab3f2-137">The Linux Docker host on Windows is based on a [Hyper-V](https://www.microsoft.com/en-us/server-cloud/solutions/virtualization.aspx) virtual machine.</span></span> <span data-ttu-id="ab3f2-138">O host para contêineres do Windows se baseia diretamente no Windows.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-138">The host for Windows Containers is directly based on Windows.</span></span> <span data-ttu-id="ab3f2-139">Docker CE para Mac baseia-se na estrutura do Apple Hypervisor e o [xhyve hypervisor](https://github.com/mist64/xhyve), que fornece uma máquina virtual do host Linux Docker no Mac OS X. O Docker CE para Windows e Mac substitui o Docker Toolbox, que foi baseado no Oracle VirtualBox.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-139">Docker CE for Mac is based on the Apple Hypervisor framework and the [xhyve hypervisor](https://github.com/mist64/xhyve), which provides a Linux Docker host virtual machine on Mac OS X. Docker CE for Windows and for Mac replaces Docker Toolbox, which was based on Oracle VirtualBox.</span></span>

<span data-ttu-id="ab3f2-140">**Docker Enterprise Edition (EE)**: uma versão empresarial das ferramentas do Docker para desenvolvimento em Linux e Windows.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-140">**Docker Enterprise Edition (EE)**: An enterprise-scale version of Docker tools for Linux and Windows development.</span></span>

<span data-ttu-id="ab3f2-141">**Compose**: uma ferramenta de linha de comando e formato de arquivo YAML com metadados para definir e executar aplicativos de vários contêineres.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-141">**Compose**: A command-line tool and YAML file format with metadata for defining and running multi-container applications.</span></span> <span data-ttu-id="ab3f2-142">Você define um único aplicativo com base em várias imagens com um ou mais arquivos .yml que podem substituir valores dependendo do ambiente.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-142">You define a single application based on multiple images with one or more .yml files that can override values depending on the environment.</span></span> <span data-ttu-id="ab3f2-143">Depois de criar as definições, você pode implantar todo o aplicativo de vários contêineres com um único comando (docker-compose up), que cria um contêiner por imagem no host do Docker.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-143">After you have created the definitions, you can deploy the whole multi-container application with a single command (docker-compose up) that creates a container per image on the Docker host.</span></span>

<span data-ttu-id="ab3f2-144">**Cluster**: uma coleção de hosts do Docker expostos como um único host virtual, para que o aplicativo possa ser dimensionado para várias instâncias dos serviços distribuídos em vários hosts do cluster.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-144">**Cluster**: A collection of Docker hosts exposed as if it were a single virtual Docker host, so that the application can scale to multiple instances of the services spread across multiple hosts within the cluster.</span></span> <span data-ttu-id="ab3f2-145">Clusters do Docker podem ser criados com o Docker Swarm, Mesosphere DC/OS, Kubernetes e Azure Service Fabric.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-145">Docker clusters can be created with Docker Swarm, Mesosphere DC/OS, Kubernetes, and Azure Service Fabric.</span></span> <span data-ttu-id="ab3f2-146">(Ao usar o Docker Swarm para gerenciar um cluster, você normalmente se referirá ao cluster como um *swarm* em vez de um cluster.)</span><span class="sxs-lookup"><span data-stu-id="ab3f2-146">(If you use Docker Swarm for managing a cluster, you typically refer to the cluster as a *swarm* instead of a cluster.)</span></span>

<span data-ttu-id="ab3f2-147">**Orquestrador**: uma ferramenta que simplifica o gerenciamento de clusters e hosts do Docker.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-147">**Orchestrator**: A tool that simplifies management of clusters and Docker hosts.</span></span> <span data-ttu-id="ab3f2-148">Os orquestradores permitem gerenciar imagens, contêineres e hosts por meio de uma CLI (interface de linha de comando) ou uma interface do usuário gráfica.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-148">Orchestrators enable you to manage their images, containers, and hosts through a command line interface (CLI) or a graphical UI.</span></span> <span data-ttu-id="ab3f2-149">É possível gerenciar a rede de contêiner, configurações, balanceamento de carga, descoberta de serviço, alta disponibilidade, configuração de host do Docker e muito mais.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-149">You can manage container networking, configurations, load balancing, service discovery, high availability, Docker host configuration, and more.</span></span> <span data-ttu-id="ab3f2-150">Um orquestrador é responsável por executar, distribuir, dimensionar e reparar de cargas de trabalho em uma coleção de nós.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-150">An orchestrator is responsible for running, distributing, scaling, and healing workloads across a collection of nodes.</span></span> <span data-ttu-id="ab3f2-151">Normalmente, produtos de orquestrador são os mesmos que fornecem infraestrutura de cluster, como Mesosphere DC/OS, Kubernetes, Docker Swarm e Azure Service Fabric.</span><span class="sxs-lookup"><span data-stu-id="ab3f2-151">Typically, orchestrator products are the same products that provide cluster infrastructure, like Mesosphere DC/OS, Kubernetes, Docker Swarm, and Azure Service Fabric.</span></span>
