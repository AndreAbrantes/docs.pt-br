<span data-ttu-id="da42a-105">Na maioria dos casos, você pode pensar um contêiner como uma instância de um processo.</span><span class="sxs-lookup"><span data-stu-id="da42a-105">In most cases, you can think of a container as an instance of a process.</span></span> <span data-ttu-id="da42a-106">Um processo não mantém o estado persistente.</span><span class="sxs-lookup"><span data-stu-id="da42a-106">A process does not maintain persistent state.</span></span> <span data-ttu-id="da42a-107">Enquanto um contêiner pode gravar em seu armazenamento local, supondo que uma instância será em torno indefinidamente seria como supondo que um único local na memória sejam durável.</span><span class="sxs-lookup"><span data-stu-id="da42a-107">While a container can write to its local storage, assuming that an instance will be around indefinitely would be like assuming that a single location in memory will be durable.</span></span> <span data-ttu-id="da42a-108">Imagens de contêiner, como processos, devem ser assumidas para ter várias instâncias, ou eles eventualmente serão eliminados; Se eles são gerenciados com o orchestrator um contêiner, deve-se assumir que eles podem obter movidos de um nó ou de VM para outra.</span><span class="sxs-lookup"><span data-stu-id="da42a-108">Container images, like processes, should be assumed to have multiple instances or that they will eventually be killed; if they’re managed with a container orchestrator, it should be assumed that they might get moved from one node or VM to another.</span></span>

<span data-ttu-id="da42a-109">O docker fornece um recurso chamado de *sobreposição de sistema de arquivos*.</span><span class="sxs-lookup"><span data-stu-id="da42a-109">Docker provides a feature named the *overlay file system*.</span></span> <span data-ttu-id="da42a-110">Isso implementa uma tarefa de cópia na gravação que armazena informações atualizadas ao sistema de arquivos raiz do contêiner.</span><span class="sxs-lookup"><span data-stu-id="da42a-110">This implements a copy-on-write task that stores updated information to the root file system of the container.</span></span> <span data-ttu-id="da42a-111">Essas informações além disso são a imagem original no qual se baseia o contêiner.</span><span class="sxs-lookup"><span data-stu-id="da42a-111">That information is in addition to the original image on which the container is based.</span></span> <span data-ttu-id="da42a-112">Se o contêiner é excluído do sistema, essas alterações serão perdidas.</span><span class="sxs-lookup"><span data-stu-id="da42a-112">If the container is deleted from the system, those changes are lost.</span></span> <span data-ttu-id="da42a-113">Portanto, embora seja possível salvar o estado de um contêiner dentro de seu armazenamento local, criando um sistema isso entraria em conflito com o local do projeto do contêiner, que por padrão é sem monitoração de estado.</span><span class="sxs-lookup"><span data-stu-id="da42a-113">Therefore, while it is possible to save the state of a container within its local storage, designing a system around this would conflict with the premise of container design, which by default is stateless.</span></span>

-   <span data-ttu-id="da42a-117">[Plug-ins do volume](https://docs.docker.com/engine/tutorials/dockervolumes/) que montar volumes para os serviços remotos, fornecendo a persistência de longo prazo.</span><span class="sxs-lookup"><span data-stu-id="da42a-117">[Volume plugins](https://docs.docker.com/engine/tutorials/dockervolumes/) that mount volumes to remote services, providing long-term persistence.</span></span>

<span data-ttu-id="da42a-120">O exemplo a seguir fornece mais detalhes sobre essas opções.</span><span class="sxs-lookup"><span data-stu-id="da42a-120">The following provides more detail about these options.</span></span>

<span data-ttu-id="da42a-127">**Contêineres de volume de dados** são uma evolução de volumes de dados regulares.</span><span class="sxs-lookup"><span data-stu-id="da42a-127">**Data volume containers** are an evolution of regular data volumes.</span></span> <span data-ttu-id="da42a-128">Um contêiner de volume de dados é um contêiner simple que tem um ou mais volumes de dados nele.</span><span class="sxs-lookup"><span data-stu-id="da42a-128">A data volume container is a simple container that has one or more data volumes within it.</span></span> <span data-ttu-id="da42a-129">O contêiner de volume de dados fornece acesso aos contêineres de um ponto de montagem central.</span><span class="sxs-lookup"><span data-stu-id="da42a-129">The data volume container provides access to containers from a central mount point.</span></span> <span data-ttu-id="da42a-130">Esse método de acesso a dados é conveniente, porque ele abstrai o local dos dados originais.</span><span class="sxs-lookup"><span data-stu-id="da42a-130">This method of data access is convenient because it abstracts the location of the original data.</span></span> <span data-ttu-id="da42a-131">Além disso, seu comportamento é semelhante de um volume de dados regulares, para que dados são persistidos neste contêiner dedicado independentemente do ciclo de vida de contêineres do aplicativo.</span><span class="sxs-lookup"><span data-stu-id="da42a-131">Other than that, its behavior is similar to that of a regular data volume, so data is persisted in this dedicated container independently of the lifecycle of the application’s containers.</span></span>
