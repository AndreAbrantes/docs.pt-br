## <a name="deploying-a-monolithic-application-as-a-container"></a><span data-ttu-id="0a8ed-141">Implantar um aplicativo monolítico como um contêiner</span><span class="sxs-lookup"><span data-stu-id="0a8ed-141">Deploying a monolithic application as a container</span></span>

<span data-ttu-id="0a8ed-142">Há benefícios em usar contêineres para gerenciar implantações de aplicativos monolíticos.</span><span class="sxs-lookup"><span data-stu-id="0a8ed-142">There are benefits to using containers to manage monolithic application deployments.</span></span> <span data-ttu-id="0a8ed-143">O dimensionamento de instâncias de contêiner é muito mais rápido e fácil do que a implantação de VMs adicionais.</span><span class="sxs-lookup"><span data-stu-id="0a8ed-143">Scaling container instances is far faster and easier than deploying additional VMs.</span></span> <span data-ttu-id="0a8ed-144">Mesmo se você usar os Conjuntos de Dimensionamento de VMs, as VMs levarão tempo para iniciar.</span><span class="sxs-lookup"><span data-stu-id="0a8ed-144">Even if you use VM Scale Sets, VMs take time to start.</span></span> <span data-ttu-id="0a8ed-145">Quando implantado como instâncias de aplicativo tradicionais em vez de contêineres, a configuração do aplicativo é gerenciada como parte da VM, o que não é o ideal.</span><span class="sxs-lookup"><span data-stu-id="0a8ed-145">When deployed as traditional application instances instead of containers, the configuration of the application is managed as part of the VM, which is not ideal.</span></span>
