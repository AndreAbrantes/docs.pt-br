## <a name="container-design-principles"></a><span data-ttu-id="72efc-113">Princípios de design de contêiner</span><span class="sxs-lookup"><span data-stu-id="72efc-113">Container design principles</span></span>

<span data-ttu-id="72efc-114">No modelo de contêiner, uma instância de imagem de contêiner representa um único processo.</span><span class="sxs-lookup"><span data-stu-id="72efc-114">In the container model, a container image instance represents a single process.</span></span> <span data-ttu-id="72efc-115">Definindo uma imagem de contêiner como um limite de processo, você pode criar primitivas que podem ser usadas para dimensionar o processo ou criar um lote.</span><span class="sxs-lookup"><span data-stu-id="72efc-115">By defining a container image as a process boundary, you can create primitives that can be used to scale the process or to batch it.</span></span>

<span data-ttu-id="72efc-116">Ao criar uma imagem de contêiner, você verá uma definição [ENTRYPOINT](https://docs.docker.com/engine/reference/builder/#entrypoint) no Dockerfile.</span><span class="sxs-lookup"><span data-stu-id="72efc-116">When you design a container image, you'll see an [ENTRYPOINT](https://docs.docker.com/engine/reference/builder/#entrypoint) definition in the Dockerfile.</span></span> <span data-ttu-id="72efc-117">Ela define o processo cujo tempo de vida controla o tempo de vida do contêiner.</span><span class="sxs-lookup"><span data-stu-id="72efc-117">This defines the process whose lifetime controls the lifetime of the container.</span></span> <span data-ttu-id="72efc-118">Quando o processo for concluído, o ciclo de vida do contêiner será encerrado.</span><span class="sxs-lookup"><span data-stu-id="72efc-118">When the process completes, the container lifecycle ends.</span></span> <span data-ttu-id="72efc-119">Os contêineres podem representar processos de longa execução como servidores Web, mas também podem representar processos de curta duração, como trabalhos em lotes, que anteriormente talvez eram implementados como Azure [WebJobs](https://github.com/Azure/azure-webjobs-sdk/wiki).</span><span class="sxs-lookup"><span data-stu-id="72efc-119">Containers might represent long-running processes like web servers, but can also represent short-lived processes like batch jobs, which formerly might have been implemented as Azure [WebJobs](https://github.com/Azure/azure-webjobs-sdk/wiki).</span></span>

<span data-ttu-id="72efc-120">Se o processo falhar, o contêiner é encerrado e o orquestrador assume.</span><span class="sxs-lookup"><span data-stu-id="72efc-120">If the process fails, the container ends, and the orchestrator takes over.</span></span> <span data-ttu-id="72efc-121">Se o orquestrador foi configurado para manter cinco instâncias em execução e uma falhar, ele criará outra instância de contêiner para substituir o processo com falha.</span><span class="sxs-lookup"><span data-stu-id="72efc-121">If the orchestrator was configured to keep five instances running and one fails, the orchestrator will create another container instance to replace the failed process.</span></span> <span data-ttu-id="72efc-122">Em um trabalho em lotes, o processo é iniciado com parâmetros.</span><span class="sxs-lookup"><span data-stu-id="72efc-122">In a batch job, the process is started with parameters.</span></span> <span data-ttu-id="72efc-123">Quando o processo é concluído, o trabalho é concluído.</span><span class="sxs-lookup"><span data-stu-id="72efc-123">When the process completes, the work is complete.</span></span> <span data-ttu-id="72efc-124">Mais tarde, este guia faz drill-down em orquestradores.</span><span class="sxs-lookup"><span data-stu-id="72efc-124">This guidance drills-down on orchestrators, later on.</span></span>

<span data-ttu-id="72efc-125">Você pode encontrar um cenário em que deseje vários processos em execução em um único contêiner.</span><span class="sxs-lookup"><span data-stu-id="72efc-125">You might find a scenario where you want multiple processes running in a single container.</span></span> <span data-ttu-id="72efc-126">Para esse cenário, como pode haver apenas um ponto de entrada por contêiner, você poderá executar um script dentro do contêiner para iniciar quantos programas forem necessários.</span><span class="sxs-lookup"><span data-stu-id="72efc-126">For that scenario, since there can be only one entry point per container, you could run a script within the container that launches as many programs as needed.</span></span> <span data-ttu-id="72efc-127">Por exemplo, você pode usar o [Supervisor](http://supervisord.org/) ou uma ferramenta semelhante para cuidar da inicialização de vários processos dentro de um único contêiner.</span><span class="sxs-lookup"><span data-stu-id="72efc-127">For example, you can use [Supervisor](http://supervisord.org/) or a similar tool to take care of launching multiple processes inside a single container.</span></span> <span data-ttu-id="72efc-128">No entanto, mesmo que seja possível encontrar arquiteturas com vários processos por contêiner, essa abordagem não é muito comum.</span><span class="sxs-lookup"><span data-stu-id="72efc-128">However, even though you can find architectures that hold multiple processes per container, that approach isn't very common.</span></span>
