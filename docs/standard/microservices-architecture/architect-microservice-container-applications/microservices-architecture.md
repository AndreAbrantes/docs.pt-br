# <a name="microservices-architecture"></a><span data-ttu-id="5aea7-104">Arquitetura de microsserviços</span><span class="sxs-lookup"><span data-stu-id="5aea7-104">Microservices architecture</span></span>

<span data-ttu-id="5aea7-105">Como o nome implica, uma arquitetura de microservices é uma abordagem para criar um aplicativo de servidor como um conjunto de serviços pequenos.</span><span class="sxs-lookup"><span data-stu-id="5aea7-105">As the name implies, a microservices architecture is an approach to building a server application as a set of small services.</span></span> <span data-ttu-id="5aea7-106">Cada serviço é executado em seu próprio processo e se comunica com outros processos usando protocolos como HTTP/HTTPS, o WebSocket, ou [AMQP](https://en.wikipedia.org/wiki/Advanced_Message_Queuing_Protocol).</span><span class="sxs-lookup"><span data-stu-id="5aea7-106">Each service runs in its own process and communicates with other processes using protocols such as HTTP/HTTPS, WebSockets, or [AMQP](https://en.wikipedia.org/wiki/Advanced_Message_Queuing_Protocol).</span></span> <span data-ttu-id="5aea7-107">Cada microsserviço implementa um domínio de ponta a ponta específico ou a capacidade de negócios dentro de um determinado limite de contexto, e cada um deve ser desenvolvida de forma autônoma e ser implantadas independentemente.</span><span class="sxs-lookup"><span data-stu-id="5aea7-107">Each microservice implements a specific end-to-end domain or business capability within a certain context boundary, and each must be developed autonomously and be deployable independently.</span></span> <span data-ttu-id="5aea7-108">Por fim, cada microsserviço deve possuir seu modelo de dados de domínio relacionadas e a lógica do domínio (Soberania e gerenciamento de dados descentralizado) com base em tecnologias de armazenamento de dados diferente (SQL, NoSQL) e linguagens de programação diferentes.</span><span class="sxs-lookup"><span data-stu-id="5aea7-108">Finally, each microservice should own its related domain data model and domain logic (sovereignty and decentralized data management) based on different data storage technologies (SQL, NoSQL) and different programming languages.</span></span>

<span data-ttu-id="5aea7-109">O tamanho deve ser um microsserviço?</span><span class="sxs-lookup"><span data-stu-id="5aea7-109">What size should a microservice be?</span></span> <span data-ttu-id="5aea7-110">Ao desenvolver um microsserviço, o tamanho não deve ser o ponto importante.</span><span class="sxs-lookup"><span data-stu-id="5aea7-110">When developing a microservice, size should not be the important point.</span></span> <span data-ttu-id="5aea7-111">Em vez disso, o ponto importante deve ser criar livremente acoplada serviços para que você tenha a autonomia de desenvolvimento, implantação e escala, para cada serviço.</span><span class="sxs-lookup"><span data-stu-id="5aea7-111">Instead, the important point should be to create loosely coupled services so you have autonomy of development, deployment, and scale, for each service.</span></span> <span data-ttu-id="5aea7-112">É claro que, ao identificar e criando microservices, experimente para torná-los tão pequenas quanto possível, desde que você não tem muitas dependências diretas com outros microservices.</span><span class="sxs-lookup"><span data-stu-id="5aea7-112">Of course, when identifying and designing microservices, you should try to make them as small as possible as long as you do not have too many direct dependencies with other microservices.</span></span> <span data-ttu-id="5aea7-113">Mais importante do que o tamanho de microsserviço é seu independência de outros serviços e a coesão interno deve ter.</span><span class="sxs-lookup"><span data-stu-id="5aea7-113">More important than the size of the microservice is the internal cohesion it must have and its independence from other services.</span></span>

<span data-ttu-id="5aea7-114">Por que uma arquitetura de microservices?</span><span class="sxs-lookup"><span data-stu-id="5aea7-114">Why a microservices architecture?</span></span> <span data-ttu-id="5aea7-115">Em resumo, ele fornece a agilidade de longo prazo.</span><span class="sxs-lookup"><span data-stu-id="5aea7-115">In short, it provides long-term agility.</span></span> <span data-ttu-id="5aea7-116">Microservices habilitar melhor sustentabilidade em sistemas complexas, grandes e altamente escalonável, permitindo que você crie aplicativos com base em vários serviços independentemente implantáveis que possuem ciclos de vida granulares e autônomos.</span><span class="sxs-lookup"><span data-stu-id="5aea7-116">Microservices enable better maintainability in complex, large, and highly-scalable systems by letting you create applications based on many independently deployable services that each have granular and autonomous lifecycles.</span></span>

<span data-ttu-id="5aea7-117">Como benefício adicional, microservices pode expandir independentemente.</span><span class="sxs-lookup"><span data-stu-id="5aea7-117">As an additional benefit, microservices can scale out independently.</span></span> <span data-ttu-id="5aea7-118">Em vez de ter um único aplicativo monolítico que você deve expandir como uma unidade, em vez disso, você pode expandir microservices específico.</span><span class="sxs-lookup"><span data-stu-id="5aea7-118">Instead of having a single monolithic application that you must scale out as a unit, you can instead scale out specific microservices.</span></span> <span data-ttu-id="5aea7-119">Dessa forma, você pode dimensionar a área funcional que precisa de mais processamento de energia ou rede largura de banda para dar suporte a demanda, em vez de expandir outras áreas do aplicativo que não precisam ser dimensionados.</span><span class="sxs-lookup"><span data-stu-id="5aea7-119">That way, you can scale just the functional area that needs more processing power or network bandwidth to support demand, rather than scaling out other areas of the application that do not need to be scaled.</span></span> <span data-ttu-id="5aea7-120">Isso significa que uma economia porque você precisa de menos hardware.</span><span class="sxs-lookup"><span data-stu-id="5aea7-120">That means cost savings because you need less hardware.</span></span>

-   <span data-ttu-id="5aea7-129">Verificações de integridade e monitoramento de serviços e infraestrutura.</span><span class="sxs-lookup"><span data-stu-id="5aea7-129">Monitoring and health checks of the services and infrastructure.</span></span>

-   <span data-ttu-id="5aea7-130">Infraestrutura escalonável para os serviços (ou seja, nuvem e orchestrators).</span><span class="sxs-lookup"><span data-stu-id="5aea7-130">Scalable infrastructure for the services (that is, cloud and orchestrators).</span></span>

-   <span data-ttu-id="5aea7-131">Design de segurança e a implementação em vários níveis: autenticação, autorização, gerenciamento de segredos, comunicação segura, etc.</span><span class="sxs-lookup"><span data-stu-id="5aea7-131">Security design and implementation at multiple levels: authentication, authorization, secrets management, secure communication, etc.</span></span>

-   <span data-ttu-id="5aea7-133">Práticas recomendadas de CI/CD e DevOps e infraestrutura.</span><span class="sxs-lookup"><span data-stu-id="5aea7-133">DevOps and CI/CD practices and infrastructure.</span></span>
