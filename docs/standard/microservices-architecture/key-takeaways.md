<span data-ttu-id="6e7a6-115">**Microservices.**</span><span class="sxs-lookup"><span data-stu-id="6e7a6-115">**Microservices.**</span></span> <span data-ttu-id="6e7a6-116">A arquitetura microservices está se tornando a abordagem preferencial para aplicativos de missão crítica distribuídos e grandes ou complexas com base em vários subsistemas independentes na forma de serviços autônomos.</span><span class="sxs-lookup"><span data-stu-id="6e7a6-116">The microservices architecture is becoming the preferred approach for distributed and large or complex mission-critical applications based on multiple independent subsystems in the form of autonomous services.</span></span> <span data-ttu-id="6e7a6-117">Em uma arquitetura baseada em microsserviço, o aplicativo é compilado como uma coleção de serviços que podem ser desenvolvidos, testados, com controle de versão, implantado e dimensionada de modo independente; Isso pode incluir qualquer banco de dados autônomo relacionado.</span><span class="sxs-lookup"><span data-stu-id="6e7a6-117">In a microservice-based architecture, the application is built as a collection of services that can be developed, tested, versioned, deployed, and scaled independently; this can include any related autonomous database.</span></span>

<span data-ttu-id="6e7a6-118">**Design controlado por domínio e SOA.**</span><span class="sxs-lookup"><span data-stu-id="6e7a6-118">**Domain-driven design and SOA.**</span></span> <span data-ttu-id="6e7a6-119">Os padrões de arquitetura microservices derivam de arquitetura orientada a serviços (SOA) e design orientado a domínio (DDD).</span><span class="sxs-lookup"><span data-stu-id="6e7a6-119">The microservices architecture patterns derive from service-oriented architecture (SOA) and domain-driven design (DDD).</span></span> <span data-ttu-id="6e7a6-120">Ao projetar e desenvolver microservices para ambientes com o surgimento de formatação de um domínio específico de regras de negócio, é importante levar em conta DDD abordagens e padrões.</span><span class="sxs-lookup"><span data-stu-id="6e7a6-120">When you design and develop microservices for environments with evolving business rules shaping a particular domain, it is important to take into account DDD approaches and patterns.</span></span>

<span data-ttu-id="6e7a6-126">Isso inclui aplicativos grandes e complexos com vários subsistemas em evolução; Nesses casos, vale a pena investir em uma arquitetura de software mais complexa, pois ele <span data-ttu-id="6e7a6-127">**Contêineres para qualquer aplicativo.**</span><span class="sxs-lookup"><span data-stu-id="6e7a6-127">**Containers for any application.**</span></span> <span data-ttu-id="6e7a6-128">Contêineres são convenientes para microservices, mas não são exclusivos para eles.</span><span class="sxs-lookup"><span data-stu-id="6e7a6-128">Containers are convenient for microservices, but are not exclusive for them.</span></span> <span data-ttu-id="6e7a6-129">Contêineres também podem ser usados com aplicativos monolíticos, incluindo aplicativos herdados com base no .NET Framework tradicional e modernizado por meio de contêineres do Windows.</span><span class="sxs-lookup"><span data-stu-id="6e7a6-129">Containers can also be used with monolithic applications, including legacy applications based on the traditional .NET Framework and modernized through Windows Containers.</span></span> <span data-ttu-id="6e7a6-130">Os benefícios de usar o Docker, como resolver vários problemas de implantação de produção e fornecer ambientes de desenvolvimento e teste de última geração, se aplicam a vários tipos diferentes de aplicativos.</span><span class="sxs-lookup"><span data-stu-id="6e7a6-130">The benefits of using Docker, such as solving many deployment-to-production issues and providing state of the art Dev and Test environments, apply to many different types of applications.</span></span>

<span data-ttu-id="6e7a6-131">**CLI versus IDE.**</span><span class="sxs-lookup"><span data-stu-id="6e7a6-131">**CLI versus IDE.**</span></span> <span data-ttu-id="6e7a6-132">Com as ferramentas da Microsoft, você pode desenvolver aplicativos .NET em contêineres usando a abordagem preferencial.</span><span class="sxs-lookup"><span data-stu-id="6e7a6-132">With Microsoft tools, you can develop containerized .NET applications using your preferred approach.</span></span> <span data-ttu-id="6e7a6-133">Você pode desenvolver com uma CLI e um ambiente com base no editor usando a CLI do Docker e o código do Visual Studio.</span><span class="sxs-lookup"><span data-stu-id="6e7a6-133">You can develop with a CLI and an editor-based environment by using the Docker CLI and Visual Studio Code.</span></span> <span data-ttu-id="6e7a6-134">Ou você pode usar uma abordagem centrada em IDE com Visual Studio e seus recursos exclusivos para Docker, como podem depurar aplicativos de contêiner vários.</span><span class="sxs-lookup"><span data-stu-id="6e7a6-134">Or you can use an IDE-focused approach with Visual Studio and its unique features for Docker, such as like being able to debug multi-container applications.</span></span>

<span data-ttu-id="6e7a6-135">**Aplicativos de nuvem resilientes.**</span><span class="sxs-lookup"><span data-stu-id="6e7a6-135">**Resilient cloud applications.**</span></span> <span data-ttu-id="6e7a6-136">Em sistemas baseados em nuvem e sistemas distribuídos em geral, há sempre o risco de falha parcial.</span><span class="sxs-lookup"><span data-stu-id="6e7a6-136">In cloud-based systems and distributed systems in general, there is always the risk of partial failure.</span></span> <span data-ttu-id="6e7a6-137">Como clientes e serviços são processos separados (contêineres), um serviço pode não ser capaz de responder de forma oportuna de uma solicitação do cliente.</span><span class="sxs-lookup"><span data-stu-id="6e7a6-137">Since clients and services are separate processes (containers), a service might not be able to respond in a timely way to a client’s request.</span></span> <span data-ttu-id="6e7a6-138">Por exemplo, um serviço pode ser pressionada devido a uma falha parcial ou de manutenção; <span data-ttu-id="6e7a6-142">**Segurança.**</span><span class="sxs-lookup"><span data-stu-id="6e7a6-142">**Security.**</span></span> <span data-ttu-id="6e7a6-143">Em nosso mundo moderno de contêineres e microservices pode expor vulnerabilidades de novo.</span><span class="sxs-lookup"><span data-stu-id="6e7a6-143">Our modern world of containers and microservices can expose new vulnerabilities.</span></span> <span data-ttu-id="6e7a6-144">Segurança básica do aplicativo é baseada na autenticação e autorização;
