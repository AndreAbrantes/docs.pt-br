# <a name="using-threads-and-threading"></a>Usando threads e threading
Os tópicos nesta seção abordam a criação e gerenciamento de threads gerenciados, como transmitir dados para threads gerenciados e obter resultados de volta e como destruir threads e manipular um <xref:System.Threading.ThreadAbortException>.  
  
## <a name="in-this-section"></a>Nesta seção  
 [Criando threads e passando dados na hora de início](../../../docs/standard/threading/creating-threads-and-passing-data-at-start-time.md)  
 Discute e demonstra a criação de threads gerenciados, incluindo como transmitir dados para novos threads e como obter dados.  
  
 [Pausando e retomando threads](../../../docs/standard/threading/pausing-and-resuming-threads.md)  
 Discute as implicações de pausar e retomar os threads gerenciados.  
  
 [Destruindo threads](../../../docs/standard/threading/destroying-threads.md)  
 Discute as implicações de destruição de threads gerenciados e como tratar um <xref:System.Threading.ThreadAbortException>.  
  
 [Agendando threads](../../../docs/standard/threading/scheduling-threads.md)  
 Discute as prioridades de thread e como eles afetam o agendamento de thread.  
  
## <a name="reference"></a>Referência  
 <xref:System.Threading.Thread>  
 Fornece documentação de referência para o <xref:System.Threading.Thread> classe que representa um segmento gerenciado, se ele veio com código não gerenciado ou foi criado em um aplicativo gerenciado.  
  
 <xref:System.Threading.ThreadStart>  
 Fornece documentação de referência para o <xref:System.Threading.ThreadStart> delegado que representa os procedimentos do thread sem parâmetros.  
  
 <xref:System.Threading.ParameterizedThreadStart>  
 Fornece uma maneira fácil para passar dados para um procedimento de thread, embora sem rigidez de tipos.
