<span data-ttu-id="2b5e1-103">O **XmlDataDocument** classe é uma classe derivada do **XmlDocument**e contém dados XML.</span><span class="sxs-lookup"><span data-stu-id="2b5e1-103">The **XmlDataDocument** class is a derived class of the **XmlDocument**, and contains XML data.</span></span> <span data-ttu-id="2b5e1-104">A vantagem do **XmlDataDocument** é que ele fornece uma ponte entre dados relacionais e hierárquicos.</span><span class="sxs-lookup"><span data-stu-id="2b5e1-104">The advantage of the **XmlDataDocument** is that it provides a bridge between relational and hierarchical data.</span></span> <span data-ttu-id="2b5e1-105">É um **XmlDocument** que pode ser associado a um **DataSet** e ambas as classes podem sincronizar as alterações feitas nos dados contidos nas duas classes.</span><span class="sxs-lookup"><span data-stu-id="2b5e1-105">It is an **XmlDocument** that can be bound to a **DataSet** and both classes can synchronize changes made to data contained in the two classes.</span></span> <span data-ttu-id="2b5e1-106">Um **XmlDocument** que está associado a um **DataSet** permite que o XML integrar com dados relacionais e não é necessário ter os dados representados como um XML ou em um formato relacional.</span><span class="sxs-lookup"><span data-stu-id="2b5e1-106">An **XmlDocument** that is bound to a **DataSet** allows XML to integrate with relational data, and you do not have to have your data represented as either XML or in a relational format.</span></span> <span data-ttu-id="2b5e1-107">Você pode fazer ambos e não ser restrito a uma única representação dos dados.</span><span class="sxs-lookup"><span data-stu-id="2b5e1-107">You can do both and not be constrained to a single representation of the data.</span></span>  
  
 <span data-ttu-id="2b5e1-108">Os benefícios de ter os dados disponíveis em dois modos de exibição são:</span><span class="sxs-lookup"><span data-stu-id="2b5e1-108">The benefits of having data available in two views are:</span></span>  
  
-   <span data-ttu-id="2b5e1-109">A parte estruturada de um documento XML pode ser mapeado para um dataset, e com eficiência é armazenada, indexada, e pesquisada.</span><span class="sxs-lookup"><span data-stu-id="2b5e1-109">The structured portion of an XML document can be mapped to a dataset, and be efficiently stored, indexed, and searched.</span></span>  
  
-   <span data-ttu-id="2b5e1-110">As transformações, a validação, e navegação podem ser feitas com eficiência através de um modelo de cursor sobre os dados XML que são armazenados relacional.</span><span class="sxs-lookup"><span data-stu-id="2b5e1-110">Transformations, validation, and navigation can be done efficiently through a cursor model over the XML data that is stored relationally.</span></span> <span data-ttu-id="2b5e1-111">Às vezes, isso pode ser feito com mais eficiência em relação às estruturas relacionais que se o XML é armazenado em um **XmlDocument** modelo.</span><span class="sxs-lookup"><span data-stu-id="2b5e1-111">At times, it can be done more efficiently against relational structures than if the XML is stored in an **XmlDocument** model.</span></span>  
  
-   <span data-ttu-id="2b5e1-112">O **DataSet** pode armazenar uma parte do XML.</span><span class="sxs-lookup"><span data-stu-id="2b5e1-112">The **DataSet** can store a portion of the XML.</span></span> <span data-ttu-id="2b5e1-113">Ou seja, você pode usar **XPath** ou **XslTransform** para armazenar a um **DataSet** somente os elementos e atributos de interesse.</span><span class="sxs-lookup"><span data-stu-id="2b5e1-113">That is, you can use **XPath** or **XslTransform** to store to a **DataSet** only those elements and attributes of interest.</span></span> <span data-ttu-id="2b5e1-114">A partir daí, as alterações podem ser feitas para o subconjunto menor, filtrado de dados, com as alterações propagar os dados mais o **XmlDataDocument**.</span><span class="sxs-lookup"><span data-stu-id="2b5e1-114">From there, changes can be made to the smaller, filtered subset of data, with the changes propagating to the larger data in the **XmlDataDocument**.</span></span>

 <span data-ttu-id="2b5e1-117">Além de oferecer suporte **XslTransform**, uma **XmlDataDocument** expõe dados relacionais para **XPath** consultas e validação.</span><span class="sxs-lookup"><span data-stu-id="2b5e1-117">In addition to supporting **XslTransform**, an **XmlDataDocument** exposes relational data to **XPath** queries and validation.</span></span>  <span data-ttu-id="2b5e1-118">Basicamente, todos os serviços XML estão disponíveis sobre dados relacionais, e os recursos relacionais, como associação de controle, gerador, e assim por diante, estão disponíveis sobre uma projeção estruturada XML sem confirmar a fidelidade XML.</span><span class="sxs-lookup"><span data-stu-id="2b5e1-118">Basically, all XML services are available over relational data, and relational facilities, such as control binding, codegen, and so on, are available over a structured projection of XML without compromising XML fidelity.</span></span>  
  
 <span data-ttu-id="2b5e1-119">Porque **XmlDataDocument** é herdada de um **XmlDocument**, ele fornece uma implementação do W3C DOM.</span><span class="sxs-lookup"><span data-stu-id="2b5e1-119">Because **XmlDataDocument** is inherited from an **XmlDocument**, it provides an implementation of the W3C DOM.</span></span> <span data-ttu-id="2b5e1-120">O fato de que o **XmlDataDocument** está associado e armazena um subconjunto de seus dados em um **DataSet** não restringir ou alterar seu uso como um **XmlDocument** de qualquer forma.</span><span class="sxs-lookup"><span data-stu-id="2b5e1-120">The fact that the **XmlDataDocument** is associated with, and stores a subset of its data within, a **DataSet** does not restrict or alter its use as an **XmlDocument** in any way.</span></span> <span data-ttu-id="2b5e1-121">Código gravado para consumir um **XmlDocument** funciona inalteradas em relação a um **XmlDataDocument**.</span><span class="sxs-lookup"><span data-stu-id="2b5e1-121">Code written to consume an **XmlDocument** works unaltered against an **XmlDataDocument**.</span></span> <span data-ttu-id="2b5e1-122">O **DataSet** fornece a exibição relacional dos mesmos dados definindo tabelas, colunas, relações e restrições, e é um repositório de dados de usuário autônoma, na memória.</span><span class="sxs-lookup"><span data-stu-id="2b5e1-122">The **DataSet** provides the relational view of the same data by defining tables, columns, relations, and constraints, and is a stand-alone, in-memory user data store.</span></span>
