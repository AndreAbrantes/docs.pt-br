# <a name="assembly-names"></a><span data-ttu-id="532cc-103">Nomes de assembly</span><span class="sxs-lookup"><span data-stu-id="532cc-103">Assembly names</span></span>


<span data-ttu-id="532cc-109">Neste exemplo, o nome totalmente qualificado indica que o `myTypes` assembly tem um nome forte com um token de chave pública, tem o valor de cultura para Estados Unidos Inglês e tem um número de versão de 1.0.1234.0.</span><span class="sxs-lookup"><span data-stu-id="532cc-109">In this example, the fully qualified name indicates that the `myTypes` assembly has a strong name with a public key token, has the culture value for United States English, and has a version number of 1.0.1234.0.</span></span> <span data-ttu-id="532cc-110">Sua arquitetura de processador é `msil` , o que significa que ela será compilada JIT (just-in-time) para código de 32 bits ou código de 64 bits, dependendo do sistema operacional e do processador.</span><span class="sxs-lookup"><span data-stu-id="532cc-110">Its processor architecture is `msil`, which means that it will be just-in-time (JIT)-compiled to 32-bit code or 64-bit code depending on the operating system and processor.</span></span>

> <span data-ttu-id="532cc-111">As `ProcessorArchitecture` informações permitem versões específicas do processador de assemblies.</span><span class="sxs-lookup"><span data-stu-id="532cc-111">The `ProcessorArchitecture` information allows processor-specific versions of assemblies.</span></span> <span data-ttu-id="532cc-112">Crie versões de um assembly cuja identidade varie apenas pela arquitetura do processador, por exemplo, versões específicas de processador 32 bits e 64 bits.</span><span class="sxs-lookup"><span data-stu-id="532cc-112">You can create versions of an assembly whose identity differs only by processor architecture, for example 32-bit and 64-bit processor-specific versions.</span></span> <span data-ttu-id="532cc-113">A arquitetura do processador não é necessária para nomes fortes.</span><span class="sxs-lookup"><span data-stu-id="532cc-113">Processor architecture is not required for strong names.</span></span> <span data-ttu-id="532cc-114">Para obter mais informações, consulte <xref:System.Reflection.AssemblyName.ProcessorArchitecture%2A?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="532cc-114">For more information, see <xref:System.Reflection.AssemblyName.ProcessorArchitecture%2A?displayProperty=nameWithType>.</span></span>

 <span data-ttu-id="532cc-115">O código que solicita tipos em um assembly deve usar um nome totalmente qualificado do assembly.</span><span class="sxs-lookup"><span data-stu-id="532cc-115">Code that requests types in an assembly must use a fully qualified assembly name.</span></span> <span data-ttu-id="532cc-116">Isso é chamado de associação totalmente qualificada.</span><span class="sxs-lookup"><span data-stu-id="532cc-116">This is called fully qualified binding.</span></span> <span data-ttu-id="532cc-117">A associação parcial, que especifica apenas um nome de assembly, não é permitida ao referenciar assemblies no .NET Framework.</span><span class="sxs-lookup"><span data-stu-id="532cc-117">Partial binding, which specifies only an assembly name, is not permitted when referencing assemblies in .NET Framework.</span></span>

 <span data-ttu-id="532cc-118">Todas as referências de assembly a assemblies que compõem .NET Framework também devem conter o nome totalmente qualificado do assembly.</span><span class="sxs-lookup"><span data-stu-id="532cc-118">All assembly references to assemblies that make up .NET Framework must also contain the fully qualified name of the assembly.</span></span> <span data-ttu-id="532cc-119">Por exemplo, uma referência ao assembly System. Data .NET Framework para a versão 1,0 incluiria:</span><span class="sxs-lookup"><span data-stu-id="532cc-119">For example, a reference to the System.Data .NET Framework assembly for version 1.0 would include:</span></span>

<span data-ttu-id="532cc-120">A versão corresponde ao número da versão de todos os assemblies de .NET Framework fornecidos com .NET Framework versão 1,0.</span><span class="sxs-lookup"><span data-stu-id="532cc-120">The version corresponds to the version number of all .NET Framework assemblies that shipped with .NET Framework version 1.0.</span></span> <span data-ttu-id="532cc-121">Para assemblies do .NET Framework, o valor da cultura é sempre neutro, e a chave pública é a mesma mostrada no exemplo acima.</span><span class="sxs-lookup"><span data-stu-id="532cc-121">For .NET Framework assemblies, the culture value is always neutral, and the public key is the same as shown in the above example.</span></span>

> [!NOTE]

## <a name="name-application-components"></a><span data-ttu-id="532cc-126">Nomear componentes de aplicativos</span><span class="sxs-lookup"><span data-stu-id="532cc-126">Name application components</span></span>
 <span data-ttu-id="532cc-127">O runtime não considera o nome do arquivo ao determinar a identidade de um assembly.</span><span class="sxs-lookup"><span data-stu-id="532cc-127">The runtime does not consider the file name when determining an assembly's identity.</span></span> <span data-ttu-id="532cc-128">A identidade do assembly, composta pelo nome, versão, cultura e nome forte do assembly, deve ficar clara para o runtime.</span><span class="sxs-lookup"><span data-stu-id="532cc-128">The assembly identity, which consists of the assembly name, version, culture, and strong name, must be clear to the runtime.</span></span>

 <span data-ttu-id="532cc-129">Por exemplo, se você tiver um assembly chamado *myAssembly.exe* que faz referência a um assembly chamado *myAssembly.dll* , a associação ocorrerá corretamente se você executar *myAssembly.exe* .</span><span class="sxs-lookup"><span data-stu-id="532cc-129">For example, if you have an assembly called *myAssembly.exe* that references an assembly called *myAssembly.dll* , binding occurs correctly if you execute *myAssembly.exe* .</span></span> <span data-ttu-id="532cc-130">No entanto, se outro aplicativo for executado *myAssembly.exe* usando o método <xref:System.AppDomain.ExecuteAssembly%2A?displayProperty=nameWithType> , o tempo de execução determinará que `myAssembly` já está carregado quando *myAssembly.exe* solicitar a associação ao `myAssembly` .</span><span class="sxs-lookup"><span data-stu-id="532cc-130">However, if another application executes *myAssembly.exe* using the method <xref:System.AppDomain.ExecuteAssembly%2A?displayProperty=nameWithType>, the runtime determines that `myAssembly` is already loaded when *myAssembly.exe* requests binding to `myAssembly`.</span></span> <span data-ttu-id="532cc-131">Nesse caso, *myAssembly.dll* nunca é carregado.</span><span class="sxs-lookup"><span data-stu-id="532cc-131">In this case, *myAssembly.dll* is never loaded.</span></span> <span data-ttu-id="532cc-132">Como *myAssembly.exe* não contém o tipo solicitado, ocorre um <xref:System.TypeLoadException> .</span><span class="sxs-lookup"><span data-stu-id="532cc-132">Because *myAssembly.exe* does not contain the requested type, a <xref:System.TypeLoadException> occurs.</span></span>
