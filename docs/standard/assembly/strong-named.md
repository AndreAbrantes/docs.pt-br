# <a name="strong-named-assemblies"></a><span data-ttu-id="93c68-102">Assemblies de nome forte</span><span class="sxs-lookup"><span data-stu-id="93c68-102">Strong-named assemblies</span></span>

## <a name="what-makes-a-strong-named-assembly"></a><span data-ttu-id="93c68-104">Como é criado um assembly de nomeação forte?</span><span class="sxs-lookup"><span data-stu-id="93c68-104">What makes a strong-named assembly?</span></span>

<span data-ttu-id="93c68-105">Um assembly de nome forte é gerado por meio da utilização de uma chave privada que corresponde a uma chave pública distribuída com o assembly e por ele mesmo.</span><span class="sxs-lookup"><span data-stu-id="93c68-105">A strong named assembly is generated by using the private key that corresponds to the public key distributed with the assembly, and the assembly itself.</span></span> <span data-ttu-id="93c68-106">O assembly inclui o manifesto assembly que contém os nomes e os hashes de todos os arquivos que compõem o assembly.</span><span class="sxs-lookup"><span data-stu-id="93c68-106">The assembly includes the assembly manifest, which contains the names and hashes of all the files that make up the assembly.</span></span> <span data-ttu-id="93c68-107">Assemblies que têm o mesmo nome forte devem ser idênticos.</span><span class="sxs-lookup"><span data-stu-id="93c68-107">Assemblies that have the same strong name should be identical.</span></span>

<span data-ttu-id="93c68-108">Você pode usar nomes fortes nos assemblies usando o Visual Studio ou uma ferramenta de linha de comando.</span><span class="sxs-lookup"><span data-stu-id="93c68-108">You can strong-name assemblies by using Visual Studio or a command-line tool.</span></span> <span data-ttu-id="93c68-109">Para obter mais informações, consulte [Como: Assinar um conjunto com um nome forte](sign-strong-name.md) ou [Sn.exe (ferramenta Nome Forte)](../../framework/tools/sn-exe-strong-name-tool.md).</span><span class="sxs-lookup"><span data-stu-id="93c68-109">For more information, see [How to: Sign an assembly with a strong name](sign-strong-name.md) or [Sn.exe (Strong Name tool)](../../framework/tools/sn-exe-strong-name-tool.md).</span></span>

<span data-ttu-id="93c68-110">Quando um assembly de nome forte é criado, ele contém o nome de texto simples do assembly, o número de versão, informação cultural opcional, uma assinatura digital e uma chave pública que corresponde a uma chave privada utilizada para assinar.</span><span class="sxs-lookup"><span data-stu-id="93c68-110">When a strong-named assembly is created, it contains the simple text name of the assembly, the version number, optional culture information, a digital signature, and the public key that corresponds to the private key used for signing.</span></span>

> <span data-ttu-id="93c68-111">Por segurança, não confie em nomes fortes.</span><span class="sxs-lookup"><span data-stu-id="93c68-111">Do not rely on strong names for security.</span></span> <span data-ttu-id="93c68-112">Eles apenas fornecem uma identidade exclusiva.</span><span class="sxs-lookup"><span data-stu-id="93c68-112">They provide a unique identity only.</span></span>

## <a name="why-strong-name-your-assemblies"></a><span data-ttu-id="93c68-113">Por que usar um nome forte em seus assemblies?</span><span class="sxs-lookup"><span data-stu-id="93c68-113">Why strong-name your assemblies?</span></span>

<span data-ttu-id="93c68-116">Assemblies de nome forte são úteis nos seguintes cenários:</span><span class="sxs-lookup"><span data-stu-id="93c68-116">Strong-named assemblies are useful in the following scenarios:</span></span>

- <span data-ttu-id="93c68-117">Você deseja habilitar seus assemblies a serem referenciados por assemblies de nome forte ou para dar acesso `friend` aos seus assemblies de outros assemblies de nome forte.</span><span class="sxs-lookup"><span data-stu-id="93c68-117">You want to enable your assemblies to be referenced by strong-named assemblies, or you want to give `friend` access to your assemblies from other strong-named assemblies.</span></span>

- <span data-ttu-id="93c68-118">Um aplicativo precisa acessar versões diferentes do mesmo assembly.</span><span class="sxs-lookup"><span data-stu-id="93c68-118">An app needs access to different versions of the same assembly.</span></span> <span data-ttu-id="93c68-119">Isso significa que você precisa de versões diferentes de um assembly para carregar lado a lado no mesmo domínio de aplicativo sem conflitos.</span><span class="sxs-lookup"><span data-stu-id="93c68-119">This means  you need different versions of an assembly to load side by side in the same app domain without conflict.</span></span> <span data-ttu-id="93c68-120">Por exemplo, se extensões diferentes de uma API existirem em assemblies que tenham o mesmo nome simples, nomeações fortes fornecem uma identidade única para cada versão de assembly.</span><span class="sxs-lookup"><span data-stu-id="93c68-120">For example, if different extensions of an API exist in assemblies that have the same simple name, strong-naming provides a unique identity for each version of the assembly.</span></span>

- <span data-ttu-id="93c68-121">Você não deseja afetar negativamente o desempenho de aplicativos usando o assembly, por isso assembly deve ser neutros quanto ao domínio.</span><span class="sxs-lookup"><span data-stu-id="93c68-121">You do not want to negatively affect performance of apps using your assembly, so you want the assembly to be domain neutral.</span></span> <span data-ttu-id="93c68-122">Isso exige uma nomeação forte porque um assembly com neutro com relação ao domínio deve ser instalado no cache de assembly global.</span><span class="sxs-lookup"><span data-stu-id="93c68-122">This requires strong-naming because a domain-neutral assembly must be installed in the global assembly cache.</span></span>

- <span data-ttu-id="93c68-123">Você deseja centralizar a manutenção do seu aplicativo aplicando a política do editor, o que significa que a montagem deve ser instalada no cache global de montagem.</span><span class="sxs-lookup"><span data-stu-id="93c68-123">You want to centralize servicing for your app by applying publisher policy, which means the assembly must be installed in the global assembly cache.</span></span>

<span data-ttu-id="93c68-124">Se você é um desenvolvedor de código aberto e deseja os benefícios de identidade de um conjunto com nome forte, considere verificar a chave privada associada a um conjunto para o seu sistema de controle de origem.</span><span class="sxs-lookup"><span data-stu-id="93c68-124">If you are an open-source developer and you want the identity benefits of a strong-named assembly, consider checking in the private key associated with an assembly to your source control system.</span></span>
