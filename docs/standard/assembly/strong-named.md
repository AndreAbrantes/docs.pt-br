# <a name="strong-named-assemblies"></a><span data-ttu-id="e0f38-103">Assemblies de nome forte</span><span class="sxs-lookup"><span data-stu-id="e0f38-103">Strong-named assemblies</span></span>

## <a name="what-makes-a-strong-named-assembly"></a><span data-ttu-id="e0f38-105">Como é criado um assembly de nomeação forte?</span><span class="sxs-lookup"><span data-stu-id="e0f38-105">What makes a strong-named assembly?</span></span>

<span data-ttu-id="e0f38-106">Um assembly de nome forte é gerado por meio da utilização de uma chave privada que corresponde a uma chave pública distribuída com o assembly e por ele mesmo.</span><span class="sxs-lookup"><span data-stu-id="e0f38-106">A strong named assembly is generated by using the private key that corresponds to the public key distributed with the assembly, and the assembly itself.</span></span> <span data-ttu-id="e0f38-107">O assembly inclui o manifesto assembly que contém os nomes e os hashes de todos os arquivos que compõem o assembly.</span><span class="sxs-lookup"><span data-stu-id="e0f38-107">The assembly includes the assembly manifest, which contains the names and hashes of all the files that make up the assembly.</span></span> <span data-ttu-id="e0f38-108">Assemblies que têm o mesmo nome forte devem ser idênticos.</span><span class="sxs-lookup"><span data-stu-id="e0f38-108">Assemblies that have the same strong name should be identical.</span></span>

<span data-ttu-id="e0f38-109">Você pode usar nomes fortes nos assemblies usando o Visual Studio ou uma ferramenta de linha de comando.</span><span class="sxs-lookup"><span data-stu-id="e0f38-109">You can strong-name assemblies by using Visual Studio or a command-line tool.</span></span> <span data-ttu-id="e0f38-110">Para obter mais informações, consulte [como assinar um assembly com um nome forte](sign-strong-name.md) ou [sn. exe (ferramenta Strong Name)](../../framework/tools/sn-exe-strong-name-tool.md).</span><span class="sxs-lookup"><span data-stu-id="e0f38-110">For more information, see [How to: Sign an assembly with a strong name](sign-strong-name.md) or [Sn.exe (Strong Name tool)](../../framework/tools/sn-exe-strong-name-tool.md).</span></span>

<span data-ttu-id="e0f38-111">Quando um assembly de nome forte é criado, ele contém o nome de texto simples do assembly, o número de versão, informação cultural opcional, uma assinatura digital e uma chave pública que corresponde a uma chave privada utilizada para assinar.</span><span class="sxs-lookup"><span data-stu-id="e0f38-111">When a strong-named assembly is created, it contains the simple text name of the assembly, the version number, optional culture information, a digital signature, and the public key that corresponds to the private key used for signing.</span></span>

> [!WARNING]
> <span data-ttu-id="e0f38-112">Por segurança, não confie em nomes fortes.</span><span class="sxs-lookup"><span data-stu-id="e0f38-112">Do not rely on strong names for security.</span></span> <span data-ttu-id="e0f38-113">Eles apenas fornecem uma identidade exclusiva.</span><span class="sxs-lookup"><span data-stu-id="e0f38-113">They provide a unique identity only.</span></span>

## <a name="why-strong-name-your-assemblies"></a><span data-ttu-id="e0f38-114">Por que usar um nome forte em seus assemblies?</span><span class="sxs-lookup"><span data-stu-id="e0f38-114">Why strong-name your assemblies?</span></span>

<span data-ttu-id="e0f38-115">Quando você usa um assembly de nome forte como referência, espera obter determinados benefícios, como controle de versão e proteção de nomenclatura.</span><span class="sxs-lookup"><span data-stu-id="e0f38-115">When you reference a strong-named assembly, you can expect certain benefits, such as versioning and naming protection.</span></span> <span data-ttu-id="e0f38-116">No .NET Framework, os assemblies de nome forte podem ser instalados no cache de assembly global, que é necessário para habilitar alguns cenários.</span><span class="sxs-lookup"><span data-stu-id="e0f38-116">In the .NET Framework, strong-named assemblies can be installed in the global assembly cache, which is required to enable some scenarios.</span></span>

<span data-ttu-id="e0f38-117">Assemblies de nome forte são úteis nos seguintes cenários:</span><span class="sxs-lookup"><span data-stu-id="e0f38-117">Strong-named assemblies are useful in the following scenarios:</span></span>

- <span data-ttu-id="e0f38-118">Você deseja habilitar seus assemblies a serem referenciados por assemblies de nome forte ou para dar acesso `friend` aos seus assemblies de outros assemblies de nome forte.</span><span class="sxs-lookup"><span data-stu-id="e0f38-118">You want to enable your assemblies to be referenced by strong-named assemblies, or you want to give `friend` access to your assemblies from other strong-named assemblies.</span></span>

- <span data-ttu-id="e0f38-119">Um aplicativo precisa acessar versões diferentes do mesmo assembly.</span><span class="sxs-lookup"><span data-stu-id="e0f38-119">An app needs access to different versions of the same assembly.</span></span> <span data-ttu-id="e0f38-120">Isso significa que você precisa de versões diferentes de um assembly para carregar lado a lado no mesmo domínio de aplicativo sem conflitos.</span><span class="sxs-lookup"><span data-stu-id="e0f38-120">This means  you need different versions of an assembly to load side by side in the same app domain without conflict.</span></span> <span data-ttu-id="e0f38-121">Por exemplo, se extensões diferentes de uma API existirem em assemblies que tenham o mesmo nome simples, nomeações fortes fornecem uma identidade única para cada versão de assembly.</span><span class="sxs-lookup"><span data-stu-id="e0f38-121">For example, if different extensions of an API exist in assemblies that have the same simple name, strong-naming provides a unique identity for each version of the assembly.</span></span>

- <span data-ttu-id="e0f38-122">Você não deseja afetar negativamente o desempenho de aplicativos usando o assembly, por isso assembly deve ser neutros quanto ao domínio.</span><span class="sxs-lookup"><span data-stu-id="e0f38-122">You do not want to negatively affect performance of apps using your assembly, so you want the assembly to be domain neutral.</span></span> <span data-ttu-id="e0f38-123">Isso exige uma nomeação forte porque um assembly com neutro com relação ao domínio deve ser instalado no cache de assembly global.</span><span class="sxs-lookup"><span data-stu-id="e0f38-123">This requires strong-naming because a domain-neutral assembly must be installed in the global assembly cache.</span></span>

- <span data-ttu-id="e0f38-124">Você deseja centralizar a manutenção para seu aplicativo aplicando a política do Publicador, o que significa que o assembly deve ser instalado no cache de assembly global.</span><span class="sxs-lookup"><span data-stu-id="e0f38-124">You want to centralize servicing for your app by applying publisher policy, which means the assembly must be installed in the global assembly cache.</span></span>

<span data-ttu-id="e0f38-125">Se você for um desenvolvedor de software livre e quiser os benefícios de identidade de um assembly de nome forte, considere a verificação da chave privada associada a um assembly para o sistema de controle do código-fonte.</span><span class="sxs-lookup"><span data-stu-id="e0f38-125">If you are an open-source developer and you want the identity benefits of a strong-named assembly, consider checking in the private key associated with an assembly to your source control system.</span></span>
