<span data-ttu-id="59581-103">O COM difere do modelo de objeto de runtime do .NET de várias maneiras importantes:</span><span class="sxs-lookup"><span data-stu-id="59581-103">COM differs from the .NET runtime object model in several important ways:</span></span>  
  
- <span data-ttu-id="59581-104">Os clientes de objetos COM devem gerenciar o tempo de vida desses objetos; o Common Language Runtime gerencia o tempo de vida de objetos em seu ambiente.</span><span class="sxs-lookup"><span data-stu-id="59581-104">Clients of COM objects must manage the lifetime of those objects; the common language runtime manages the lifetime of objects in its environment.</span></span>

 <span data-ttu-id="59581-111">Para superar essas diferenças, o runtime fornece classes wrapper para fazer com que os clientes gerenciados e não gerenciados acreditem que estão chamando objetos em things.</span></span> <span data-ttu-id="59581-114">O tempo de execução também cria um [CCW](com-callable-wrapper.md) (COM Callable Wrapper) para reverter o processo, permitindo que um cliente COM  <span data-ttu-id="59581-117">Na maioria dos casos, o RCW (Runtime Callable Wrapper) padrão ou o CCW gerado pelo tempo de execução fornece o marshaling adequado para chamadas que cruzam o limite entre o COM e o tempo de execução do .NET.</span><span class="sxs-lookup"><span data-stu-id="59581-117">In most cases, the standard RCW or CCW generated by the runtime provides adequate marshaling for calls that cross the boundary between COM and the .NET runtime.</span></span> <span data-ttu-id="59581-118">Usando atributos personalizados, opcionalmente, você pode ajustar a maneira como o runtime representa o código gerenciado e não gerenciado.</span><span class="sxs-lookup"><span data-stu-id="59581-118">Using custom attributes, you can optionally adjust the way the runtime represents managed and unmanaged code.</span></span>
