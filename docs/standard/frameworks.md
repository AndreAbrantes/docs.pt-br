## <a name="how-to-specify-target-frameworks"></a><span data-ttu-id="79728-195">Como especificar estruturas de destino</span><span class="sxs-lookup"><span data-stu-id="79728-195">How to specify target frameworks</span></span>

<span data-ttu-id="79728-196">As estruturas de destino são especificadas no arquivo de projeto.</span><span class="sxs-lookup"><span data-stu-id="79728-196">Target frameworks are specified in your project file.</span></span> <span data-ttu-id="79728-197">Quando uma única estrutura de destino é especificada, use o elemento **TargetFramework**.</span><span class="sxs-lookup"><span data-stu-id="79728-197">When a single target framework is specified, use the **TargetFramework** element.</span></span> <span data-ttu-id="79728-198">O seguinte arquivo de projeto de aplicativo do console demonstra como destinar ao .NET Core 2.2:</span><span class="sxs-lookup"><span data-stu-id="79728-198">The following console app project file demonstrates how to target .NET Core 2.2:</span></span>

```xml
  <PropertyGroup>
    <OutputType>Exe</OutputType>
    <TargetFramework>netcoreapp2.2</TargetFramework>
  </PropertyGroup>
```

<span data-ttu-id="79728-199">Ao especificar várias estruturas de destino, você pode fazer referência a assemblies para cada estrutura de destino de forma condicional.</span><span class="sxs-lookup"><span data-stu-id="79728-199">When you specify multiple target frameworks, you may conditionally reference assemblies for each target framework.</span></span> <span data-ttu-id="79728-200">Em seu código, você pode condicionalmente compilar em relação a esses assemblies usando símbolos de pré-processador com a lógica *if-then-else*.</span><span class="sxs-lookup"><span data-stu-id="79728-200">In your code, you can conditionally compile against those assemblies by using preprocessor symbols with *if-then-else* logic.</span></span>

<span data-ttu-id="79728-201">O seguinte arquivo de projeto de biblioteca tem como destino as APIs do .NET Standard (`netstandard1.4`) e APIs do .NET Framework (`net40` e `net45`).</span><span class="sxs-lookup"><span data-stu-id="79728-201">The following library project file targets APIs of .NET Standard (`netstandard1.4`) and APIs of the .NET Framework (`net40` and `net45`).</span></span> <span data-ttu-id="79728-202">Use o elemento **TargetFrameworks** plural com várias estruturas de destino.</span><span class="sxs-lookup"><span data-stu-id="79728-202">Use the plural **TargetFrameworks** element with multiple target frameworks.</span></span> <span data-ttu-id="79728-203">Observe como os atributos `Condition` incluem pacotes específicos de implementação quando a biblioteca é compilada para os dois TFMs de .NET Framework:</span><span class="sxs-lookup"><span data-stu-id="79728-203">Note how the `Condition` attributes include implementation-specific packages when the library is compiled for the two .NET Framework TFMs:</span></span>

```xml
  <PropertyGroup>
    <TargetFrameworks>netstandard1.4;net40;net45</TargetFrameworks>
  </PropertyGroup>

  <!-- Conditionally obtain references for the .NET Framework 4.0 target -->

  </ItemGroup>

  <!-- Conditionally obtain references for the .NET Framework 4.5 target -->

    <Reference Include="System.Threading.Tasks" />
  </ItemGroup>

```
