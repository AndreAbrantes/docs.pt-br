## <a name="mapping-error-codes-to-exceptions"></a><span data-ttu-id="a48f9-116">Mapeando códigos de erro para exceções</span><span class="sxs-lookup"><span data-stu-id="a48f9-116">Mapping error codes to exceptions</span></span>

<span data-ttu-id="a48f9-117">Como o sistema de arquivos é um recurso de sistema operacional, os métodos de E/S no .NET Core e no .NET Framework encapsulam chamadas ao sistema operacional subjacente.</span><span class="sxs-lookup"><span data-stu-id="a48f9-117">Because the file system is an operating system resource, I/O methods in both .NET Core and .NET Framework wrap calls to the underlying operating system.</span></span> <span data-ttu-id="a48f9-118">Quando ocorre um erro de E/S no código executado pelo sistema operacional, ele retorna informações do erro para o método de E/S do .NET.</span><span class="sxs-lookup"><span data-stu-id="a48f9-118">When an I/O error occurs in code executed by the operating system, the operating system returns error information to the .NET I/O method.</span></span> <span data-ttu-id="a48f9-119">Em seguida, o método converte as informações do erro, normalmente na forma de um código de erro, em um tipo de exceção do .NET.</span><span class="sxs-lookup"><span data-stu-id="a48f9-119">The method then translates the error information, typically in the form of an error code, into a .NET exception type.</span></span> <span data-ttu-id="a48f9-120">Na maioria dos casos, ele faz isso convertendo diretamente o código de erro em seu tipo de exceção correspondente; ele não realiza nenhum mapeamento especial do erro com base no contexto da chamada de método.</span><span class="sxs-lookup"><span data-stu-id="a48f9-120">In most cases, it does this by directly translating the error code into its corresponding exception type; it does not perform any special mapping of the error based on the context of the method call.</span></span>

<span data-ttu-id="a48f9-121">Por exemplo, no sistema operacional Windows, uma chamada de método que retorna um código de erro `ERROR_FILE_NOT_FOUND` (ou 0x02) é mapeada para um <xref:System.IO.FileNotFoundException> e um código de erro `ERROR_PATH_NOT_FOUND` (ou 0x03) é mapeado para um <xref:System.IO.DirectoryNotFoundException>.</span><span class="sxs-lookup"><span data-stu-id="a48f9-121">For example, on the Windows operating system, a method call that returns an error code of `ERROR_FILE_NOT_FOUND` (or 0x02) maps to a <xref:System.IO.FileNotFoundException>, and an error code of `ERROR_PATH_NOT_FOUND` (or 0x03) maps to a <xref:System.IO.DirectoryNotFoundException>.</span></span>

<span data-ttu-id="a48f9-122">No entanto, as condições precisas sob as quais o sistema operacional retorna códigos de erro específicos geralmente não são documentadas ou são mal documentadas.</span><span class="sxs-lookup"><span data-stu-id="a48f9-122">However, the precise conditions under which the operating system returns particular error codes is often undocumented or poorly documented.</span></span> <span data-ttu-id="a48f9-123">Em decorrência disso, podem ocorrer exceções inesperadas.</span><span class="sxs-lookup"><span data-stu-id="a48f9-123">As a result, unexpected exceptions can occur.</span></span> <span data-ttu-id="a48f9-124">Por exemplo, como você está trabalhando com um diretório em vez de um arquivo, você esperaria que fornecer um caminho de diretório inválido para o construtor <xref:System.IO.DirectoryInfo.%23ctor%2A?displayProperty=nameWithType> gerasse uma <xref:System.IO.DirectoryNotFoundException>.</span><span class="sxs-lookup"><span data-stu-id="a48f9-124">For example, because you are working with a directory rather than a file, you would expect that providing an invalid directory path to the <xref:System.IO.DirectoryInfo.%23ctor%2A?displayProperty=nameWithType> constructor throws a <xref:System.IO.DirectoryNotFoundException>.</span></span> <span data-ttu-id="a48f9-125">No entanto, isso também pode gerar uma <xref:System.IO.FileNotFoundException>.</span><span class="sxs-lookup"><span data-stu-id="a48f9-125">However, it may also throw a <xref:System.IO.FileNotFoundException>.</span></span>

## <a name="handling-ioexception"></a><span data-ttu-id="a48f9-152">Tratamento IOException</span><span class="sxs-lookup"><span data-stu-id="a48f9-152">Handling IOException</span></span>
