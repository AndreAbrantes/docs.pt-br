# <a name="key-takeaways"></a>Principais aspectos a serem lembrados

- Soluções baseadas em contêiner fornecem benefícios importantes de economia de custo como contêineres de resolver problemas de implantação causados por falhas de dependência em ambientes de produção, melhorando significativamente a operações de produção e DevOps.

- O docker está se tornando o verdadeiro padrão no setor de contêineres e dá suporte dos fornecedores mais significativos nos ecossistemas do Linux e Windows, incluindo a Microsoft. No futuro, o Docker provavelmente será onipresente em qualquer datacenter na nuvem ou local.

- Um contêiner do Docker está se tornando a unidade padrão de implantação para qualquer serviço ou aplicativo baseado em servidor.

- Orquestradores do docker, como os fornecidos no serviço de Kubernetes do Azure (AKS) e o Azure Service Fabric são fundamentais e indispensável para qualquer aplicativo baseado em microsserviços ou vários contêiner com necessidades significativas de complexidade e escalabilidade.

- Um ambiente de DevOps de ponta a ponta que oferece suporte à integração contínua/implantação contínua e conecta-se a ambientes de produção do Docker pode fornecer a agilidade e, por fim, melhorar o tempo de colocação no mercado de seus aplicativos.

- Serviços de DevOps do Azure simplifica bastante o seu ambiente de DevOps com a implantação para ambientes do Docker nos pipelines CI/CD de. Essa instrução aplica-se a ambientes simples do Docker, bem como para o microsserviço avançado e orquestradores de contêiner com base no Azure.
