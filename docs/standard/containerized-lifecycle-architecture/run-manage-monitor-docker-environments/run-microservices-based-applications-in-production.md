## <a name="introduction-to-orchestrators-schedulers-and-container-clusters"></a><span data-ttu-id="d9066-106">Introdução ao orchestrators agendadores e clusters de contêiner</span><span class="sxs-lookup"><span data-stu-id="d9066-106">Introduction to orchestrators, schedulers, and container clusters</span></span>

<span data-ttu-id="d9066-107">Anteriormente neste livro eletrônico, apresentamos *clusters* e *agendadores* como parte da discussão no desenvolvimento e arquitetura de software.</span><span class="sxs-lookup"><span data-stu-id="d9066-107">Earlier in this e-book, we introduced *clusters* and *schedulers* as part of the discussion on software architecture and development.</span></span> <span data-ttu-id="d9066-108">São exemplos de clusters de Docker Docker Swarm e sistema operacional de Datacenter Mesosphere (controlador de domínio/OS).</span><span class="sxs-lookup"><span data-stu-id="d9066-108">Examples of Docker clusters are Docker Swarm and Mesosphere Datacenter Operating System (DC/OS).</span></span> <span data-ttu-id="d9066-109">Ambos podem executar como parte da infraestrutura de fornecido pelo serviço de contêiner do Microsoft Azure.</span><span class="sxs-lookup"><span data-stu-id="d9066-109">Both of these can run as a part of the infrastructure provided by Microsoft Azure Container Service.</span></span>

<span data-ttu-id="d9066-110">Quando os aplicativos são expandidos em vários sistemas de host, a capacidade de gerenciar cada sistema host e abstrair a complexidade da plataforma subjacente se torna atraente.</span><span class="sxs-lookup"><span data-stu-id="d9066-110">When applications are scaled-out across multiple host systems, the ability to manage each host system and abstract away the complexity of the underlying platform becomes attractive.</span></span> <span data-ttu-id="d9066-111">É exatamente o que fornecem orchestrators e agendadores.</span><span class="sxs-lookup"><span data-stu-id="d9066-111">That is precisely what orchestrators and schedulers provide.</span></span> <span data-ttu-id="d9066-112">Vamos dar uma olhada rápida-los aqui:</span><span class="sxs-lookup"><span data-stu-id="d9066-112">Let's take a brief look at them here:</span></span>

-   <span data-ttu-id="d9066-113">**Agendadores * *"Programação" refere-se à capacidade de um administrador para carregar um arquivo de serviço em um sistema de host que estabelece como executar um contêiner específico.</span><span class="sxs-lookup"><span data-stu-id="d9066-113">**Schedulers*** *"Scheduling" refers to the ability for an administrator to load a service file onto a host system that establishes how to run a specific container.</span></span> <span data-ttu-id="d9066-114">Iniciar contêineres em um cluster de Docker tende a ser conhecido como agendamento.</span><span class="sxs-lookup"><span data-stu-id="d9066-114">Launching containers in a Docker cluster tends to be known as scheduling.</span></span> <span data-ttu-id="d9066-115">Embora o agendamento refere-se ao ato de carregar a definição de serviço, em termos mais gerais, específico agendadores são responsáveis por conectar ao sistema de inicialização do host para gerenciar os serviços de qualquer capacidade necessária.</span><span class="sxs-lookup"><span data-stu-id="d9066-115">Although scheduling refers to the specific act of loading the service definition, in a more general sense, schedulers are responsible for hooking into a host's init system to manage services in whatever capacity needed.</span></span>

<span data-ttu-id="d9066-116">Um agendador de cluster tem vários objetivos: usando recursos do cluster com eficiência, trabalhando com restrições de posicionamento fornecido pelo usuário, agendamento aplicativos rapidamente para não deixá-los em um estado pendente, com um grau de "integridade", sendo robusto para erros, e sempre estar disponíveis.</span><span class="sxs-lookup"><span data-stu-id="d9066-116">A cluster scheduler has multiple goals: using the cluster's resources efficiently, working with user-supplied placement constraints, scheduling applications rapidly to not leave them in a pending state, having a degree of "fairness," being robust to errors, and always be available.</span></span>

-   <span data-ttu-id="d9066-117">**Orquestração * *plataformas estendem os recursos de gerenciamento de ciclo de vida complexas, multicontainer cargas de trabalho implantadas em um cluster de hosts.</span><span class="sxs-lookup"><span data-stu-id="d9066-117">**Orchestration*** *Platforms extend life-cycle management capabilities to complex, multicontainer workloads deployed on a cluster of hosts.</span></span> <span data-ttu-id="d9066-118">Abstraindo a infraestrutura de host, as ferramentas de orquestração dar aos usuários uma maneira de tratar de todo o cluster como um destino de implantação única.</span><span class="sxs-lookup"><span data-stu-id="d9066-118">By abstracting the host infrastructure, orchestration tools give users a way to treat the entire cluster as a single deployment target.</span></span>

<span data-ttu-id="d9066-119">O processo de orquestração envolve ferramentas e uma plataforma que pode automatizar todos os aspectos do gerenciamento de aplicativos de posicionamento inicial ou a implantação por contêiner; contêineres de movimentação para hosts diferentes, dependendo do seu host integridade ou desempenho; controle de versão e as atualizações sem interrupção e funções que oferecem suporte a failover; e dimensionamento de monitoramento de integridade e muito mais.</span><span class="sxs-lookup"><span data-stu-id="d9066-119">The process of orchestration involves tooling and a platform that can automate all aspects of application management from initial placement or deployment per container; moving containers to different hosts depending on its host's health or performance; versioning and rolling updates and health monitoring functions that support scaling and failover; and many more.</span></span>

<span data-ttu-id="d9066-120">Orquestração é um termo amplo que se refere ao contêiner de agendamento, gerenciamento de cluster e possivelmente o provisionamento de hosts adicionais.</span><span class="sxs-lookup"><span data-stu-id="d9066-120">Orchestration is a broad term that refers to container scheduling, cluster management, and possibly the provisioning of additional hosts.</span></span>

<span data-ttu-id="d9066-121">Os recursos fornecidos pelo orchestrators e agendadores são muito complexos para desenvolver e criar do zero, e, portanto, você geralmente deseja fazer uso das soluções de orquestração oferecidos por fornecedores.</span><span class="sxs-lookup"><span data-stu-id="d9066-121">The capabilities provided by orchestrators and schedulers are very complex to develop and create from scratch, and therefore you usually would want to make use of orchestration solutions offered by vendors.</span></span>
