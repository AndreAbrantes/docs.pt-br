<span data-ttu-id="b79ac-105">Um primitivo de contêineres é imutabilidade.</span><span class="sxs-lookup"><span data-stu-id="b79ac-105">A primitive of containers is immutability.</span></span> <span data-ttu-id="b79ac-106">Quando comparado a uma máquina virtual, os contêineres não desapareceram como uma ocorrência comum.</span><span class="sxs-lookup"><span data-stu-id="b79ac-106">When compared to a VM, containers don't disappear as a common occurrence.</span></span> <span data-ttu-id="b79ac-107">Uma VM pode falhar em várias formas de processos inativos, sobrecarga da CPU ou um disco cheio ou com falha.</span><span class="sxs-lookup"><span data-stu-id="b79ac-107">A VM might fail in various forms from dead processes, overloaded CPU, or a full or failed disk.</span></span> <span data-ttu-id="b79ac-108">Ainda, esperamos que a VM esteja disponível e unidades RAID são comuns para assegurar a mantêm os dados de falhas no disco.</span><span class="sxs-lookup"><span data-stu-id="b79ac-108">Yet, we expect the VM to be available and RAID drives are commonplace to assure drive failures maintain data.</span></span>

<span data-ttu-id="b79ac-113">Docker usa um recurso conhecido como um *sobreposição de sistema de arquivos* implementar um processo de cópia na gravação que armazena quaisquer informações atualizadas ao sistema de arquivos raiz de um contêiner, em comparação comparado a imagem original no qual se baseia.</span><span class="sxs-lookup"><span data-stu-id="b79ac-113">Docker uses a feature known as an *overlay file system* to implement a copy-on-write process that stores any updated information to the root file system of a container, compared to the original image on which it is based.</span></span> <span data-ttu-id="b79ac-114">Essas alterações serão perdidas se o contêiner subsequentemente é excluído do sistema.</span><span class="sxs-lookup"><span data-stu-id="b79ac-114">These changes are lost if the container is subsequently deleted from the system.</span></span> <span data-ttu-id="b79ac-115">Um contêiner, portanto, não têm um armazenamento persistente por padrão.</span><span class="sxs-lookup"><span data-stu-id="b79ac-115">A container, therefore, does not have persistent storage by default.</span></span> <span data-ttu-id="b79ac-116">Embora seja possível salvar o estado de um contêiner, projetando um sistema isso seria em conflito com o princípio de arquitetura de contêiner.</span><span class="sxs-lookup"><span data-stu-id="b79ac-116">Although it's possible to save the state of a container, designing a system around this would be in conflict with the principle of container architecture.</span></span>

<span data-ttu-id="b79ac-117">Para gerenciar dados persistentes em aplicativos de Docker, há soluções comuns:</span><span class="sxs-lookup"><span data-stu-id="b79ac-117">To manage persistent data in Docker applications, there are common solutions:</span></span>

-   <span data-ttu-id="b79ac-120">[**Plug-ins do volume**](https://docs.docker.com/engine/tutorials/dockervolumes/#/mount-a-shared-storage-volume-as-a-data-volume) esses montagem volumes em locais remotos, fornecendo a persistência de longo prazo.</span><span class="sxs-lookup"><span data-stu-id="b79ac-120">[**Volume Plugins**](https://docs.docker.com/engine/tutorials/dockervolumes/#/mount-a-shared-storage-volume-as-a-data-volume) These mount volumes to remote locations, providing long-term persistence.</span></span>

-   <span data-ttu-id="b79ac-122">[**Armazenamento do Azure**](https://docs.microsoft.com/azure/storage/) isso fornece a plataforma de distribuição geográfica como um armazenamento de serviço (PaaS), fornecendo o melhor de contêineres de persistência como a longo prazo.</span><span class="sxs-lookup"><span data-stu-id="b79ac-122">[**Azure Storage**](https://docs.microsoft.com/azure/storage/) This provides geo distributable platform as a service (PaaS) storage, providing the best of containers as long-term persistence.</span></span>

<span data-ttu-id="b79ac-123">Volumes de dados são designados especialmente diretórios dentro de um ou mais contêineres que ignoram o [sistema de arquivos de união](https://docs.docker.com/v1.8/reference/glossary#union-file-system).</span><span class="sxs-lookup"><span data-stu-id="b79ac-123">Data volumes are specially designated directories within one or more containers that bypass the [Union File System](https://docs.docker.com/v1.8/reference/glossary#union-file-system).</span></span> <span data-ttu-id="b79ac-124">Volumes de dados são projetados para manter os dados, independentes do ciclo de vida do contêiner.</span><span class="sxs-lookup"><span data-stu-id="b79ac-124">Data volumes are designed to maintain data, independent of the container's life cycle.</span></span> <span data-ttu-id="b79ac-125">Docker, portanto, nunca exclui automaticamente os volumes quando você remove um contêiner, nem o ele volumes "coleta de lixo" que não são mais referenciados por um contêiner.</span><span class="sxs-lookup"><span data-stu-id="b79ac-125">Docker therefore never automatically deletes volumes when you remove a container, nor will it "garbage collect" volumes that are no longer referenced by a container.</span></span> <span data-ttu-id="b79ac-126">O sistema operacional do host pode navegar e editar os dados em qualquer volume livremente, que é apenas outra razão para usar os volumes de dados com moderação.</span><span class="sxs-lookup"><span data-stu-id="b79ac-126">The host operating system can browse and edit the data in any volume freely, which is just another reason to use data volumes sparingly.</span></span>

<span data-ttu-id="b79ac-127">Um [contêiner de volume de dados](https://docs.docker.com/v1.8/userguide/dockervolumes/) é um aprimoramento volumes de dados regulares.</span><span class="sxs-lookup"><span data-stu-id="b79ac-127">A [data volume container](https://docs.docker.com/v1.8/userguide/dockervolumes/) is an improvement over regular data volumes.</span></span> <span data-ttu-id="b79ac-128">É essencialmente um contêiner inativo que tem um ou mais volumes de dados criados nele (conforme descrito anteriormente).</span><span class="sxs-lookup"><span data-stu-id="b79ac-128">It is essentially a dormant container that has one or more data volumes created within it (as described earlier).</span></span> <span data-ttu-id="b79ac-129">O contêiner de volume de dados fornece acesso aos contêineres de um ponto de montagem central.</span><span class="sxs-lookup"><span data-stu-id="b79ac-129">The data volume container provides access to containers from a central mount point.</span></span> <span data-ttu-id="b79ac-130">A vantagem desse método de acesso é que ele abstrai o local dos dados originais, tornando o contêiner de dados um ponto de montagem lógico.</span><span class="sxs-lookup"><span data-stu-id="b79ac-130">The benefit of this method of access is that it abstracts the location of the original data, making the data container a logical mount point.</span></span> <span data-ttu-id="b79ac-131">Ele também permite acessar os volumes de contêiner de dados para serem criados e destruídos mantendo os dados persistentes em um contêiner dedicado de contêineres de "aplicativo".</span><span class="sxs-lookup"><span data-stu-id="b79ac-131">It also allows "application" containers accessing the data container volumes to be created and destroyed while keeping the data persistent in a dedicated container.</span></span>

<span data-ttu-id="b79ac-135">Devido a dificuldades para gerenciar dados compartilhados entre contêineres que são executados em hosts físicos separados, é recomendável que você não use volumes para dados de negócios, a menos que o host do Docker é uma host fixa/VM, porque ao usar contêineres do Docker em um orchestrator contêineres devem ser movidos de um para outro host, dependendo de otimizações para ser executada pelo cluster.</span><span class="sxs-lookup"><span data-stu-id="b79ac-135">Due to the inability to manage data shared between containers that run on separate physical hosts, it is recommended that you not use volumes for business data unless the Docker host is a fixed host/VM, because when using Docker containers in an orchestrator, containers are expected to be moved from one to another host, depending on the optimizations to be performed by the cluster.</span></span>

<span data-ttu-id="b79ac-136">Portanto, os volumes de dados regulares são um bom mecanismo para trabalhar com arquivos de rastreamento, arquivos temporal ou nenhum conceito semelhante que não afeta a consistência de dados de negócios se ou quando os contêineres são movidos em vários hosts.</span><span class="sxs-lookup"><span data-stu-id="b79ac-136">Therefore, regular data volumes are a good mechanism to work with trace files, temporal files, or any similar concept that won't affect the business data consistency if or when your containers are moved across multiple hosts.</span></span>
