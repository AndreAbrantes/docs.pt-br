# <a name="state-and-data-in-docker-applications"></a><span data-ttu-id="39ed2-103">Estado e dados em aplicativos do Docker</span><span class="sxs-lookup"><span data-stu-id="39ed2-103">State and data in Docker applications</span></span>

<span data-ttu-id="39ed2-104">Na maioria dos casos, você pode considerar um contêiner uma instância de um processo.</span><span class="sxs-lookup"><span data-stu-id="39ed2-104">In most cases, you can think of a container as an instance of a process.</span></span> <span data-ttu-id="39ed2-105">Um processo não mantém o estado persistente.</span><span class="sxs-lookup"><span data-stu-id="39ed2-105">A process does not maintain persistent state.</span></span> <span data-ttu-id="39ed2-106">Enquanto um contêiner pode gravar em seu armazenamento local, supondo que uma instância permanecerá indefinidamente é como presumir que um único local na memória será durável.</span><span class="sxs-lookup"><span data-stu-id="39ed2-106">While a container can write to its local storage, assuming that an instance will be around indefinitely is like assuming that a single location in memory will be durable.</span></span> <span data-ttu-id="39ed2-107">Imagens de contêiner, assim como processos, devem ser consideradas ter várias instâncias, e eles serão eventualmente eliminados; Se forem gerenciadas com um orquestrador de contêiner, deve-se assumir que eles podem ser movidos de um nó ou VM para outro.</span><span class="sxs-lookup"><span data-stu-id="39ed2-107">Container images, like processes, should be assumed to have multiple instances and that they will eventually be killed; if they're managed with a container orchestrator, it should be assumed that they might get moved from one node or VM to another.</span></span>

<span data-ttu-id="39ed2-108">As soluções a seguir são usadas para gerenciar dados persistentes em aplicativos do Docker:</span><span class="sxs-lookup"><span data-stu-id="39ed2-108">The following solutions are used to manage persistent data in Docker applications:</span></span>

<span data-ttu-id="39ed2-109">Do host do Docker, como [Volumes do Docker](https://docs.docker.com/engine/admin/volumes/):</span><span class="sxs-lookup"><span data-stu-id="39ed2-109">From the Docker host, as [Docker Volumes](https://docs.docker.com/engine/admin/volumes/):</span></span>

- <span data-ttu-id="39ed2-110">**Volumes** são armazenados em uma área do sistema de arquivos de host que é gerenciado pelo Docker.</span><span class="sxs-lookup"><span data-stu-id="39ed2-110">**Volumes** are stored in an area of the host filesystem that is managed by Docker.</span></span>

- <span data-ttu-id="39ed2-111">**Associar montagens** pode mapear para qualquer pasta no sistema de arquivos do host, portanto, o acesso não pode ser controlado a partir de um processo de Docker e pode representar um risco de segurança como um contêiner pode acessar as pastas de sistema operacional confidenciais.</span><span class="sxs-lookup"><span data-stu-id="39ed2-111">**Bind mounts** can map to any folder in the host filesystem, so access can't be controlled from a Docker process and can pose a security risk as a container could access sensitive OS folders.</span></span>

<span data-ttu-id="39ed2-113">Do armazenamento remoto:</span><span class="sxs-lookup"><span data-stu-id="39ed2-113">From remote storage:</span></span>

- <span data-ttu-id="39ed2-114">[O armazenamento do Azure](https://azure.microsoft.com/documentation/services/storage/) oferece armazenamento geograficamente distribuído, fornecendo uma boa solução de persistência de longo prazo para contêineres.</span><span class="sxs-lookup"><span data-stu-id="39ed2-114">[Azure Storage](https://azure.microsoft.com/documentation/services/storage/) provides geo-distributable storage, providing a good long-term persistence solution for containers.</span></span>

- <span data-ttu-id="39ed2-115">Bancos de dados relacionais remotos, como [banco de dados SQL](https://azure.microsoft.com/services/sql-database/), como bancos de dados NoSQL [do Azure Cosmos DB](https://docs.microsoft.com/azure/cosmos-db/introduction), ou serviços, como armazenar em cache [Redis](https://redis.io/).</span><span class="sxs-lookup"><span data-stu-id="39ed2-115">Remote relational databases like [Azure SQL Database](https://azure.microsoft.com/services/sql-database/), NoSQL databases like [Azure Cosmos DB](https://docs.microsoft.com/azure/cosmos-db/introduction), or cache services like [Redis](https://redis.io/).</span></span>

<span data-ttu-id="39ed2-116">Do contêiner do Docker:</span><span class="sxs-lookup"><span data-stu-id="39ed2-116">From the Docker container:</span></span>

- <span data-ttu-id="39ed2-117">O Docker oferece um recurso chamado *sistema de arquivos de sobreposição*.</span><span class="sxs-lookup"><span data-stu-id="39ed2-117">Docker provides a feature named the *overlay file system*.</span></span> <span data-ttu-id="39ed2-118">Esse recurso implementa uma tarefa de cópia na gravação que armazena informações atualizadas para o sistema de arquivos raiz do contêiner.</span><span class="sxs-lookup"><span data-stu-id="39ed2-118">This feature implements a copy-on-write task that stores updated information to the root file system of the container.</span></span> <span data-ttu-id="39ed2-119">Essas informações "apresenta na parte superior de" a imagem original na qual o contêiner se baseia.</span><span class="sxs-lookup"><span data-stu-id="39ed2-119">That information "lays on top of" the original image on which the container is based.</span></span> <span data-ttu-id="39ed2-120">Se o contêiner for excluído do sistema, essas alterações serão perdidas.</span><span class="sxs-lookup"><span data-stu-id="39ed2-120">If the container is deleted from the system, those changes are lost.</span></span> <span data-ttu-id="39ed2-121">Portanto, embora seja possível salvar o estado de um contêiner em seu armazenamento local, criando um sistema baseado neste recurso entraria em conflito com a premissa de design de contêiner, que é sem monitoração de estado por padrão.</span><span class="sxs-lookup"><span data-stu-id="39ed2-121">Therefore, while it is possible to save the state of a container within its local storage, designing a system based on this feature would conflict with the premise of container design, which by default is stateless.</span></span>

- <span data-ttu-id="39ed2-122">No entanto, os Volumes Docker agora é a maneira preferencial para lidar com dados locais no Docker.</span><span class="sxs-lookup"><span data-stu-id="39ed2-122">However, Docker Volumes is now the preferred way to handle local data in Docker.</span></span> <span data-ttu-id="39ed2-123">Se você precisar de mais informações sobre o armazenamento em contêineres, verificar [drivers de armazenamento do Docker](https://docs.docker.com/engine/userguide/storagedriver/) e [sobre drivers de armazenamento, contêineres e imagens](https://docs.docker.com/engine/userguide/storagedriver/imagesandcontainers/).</span><span class="sxs-lookup"><span data-stu-id="39ed2-123">If you need more information about storage in containers, check on [Docker storage drivers](https://docs.docker.com/engine/userguide/storagedriver/) and [About images, containers, and storage drivers](https://docs.docker.com/engine/userguide/storagedriver/imagesandcontainers/).</span></span>

<span data-ttu-id="39ed2-124">O exemplo a seguir fornece detalhes adicionais sobre essas opções.</span><span class="sxs-lookup"><span data-stu-id="39ed2-124">The following provides additional detail about these options.</span></span>

<span data-ttu-id="39ed2-125">**Volumes** são diretórios mapeados do sistema operacional do host para diretórios em contêineres.</span><span class="sxs-lookup"><span data-stu-id="39ed2-125">**Volumes** are directories mapped from the host OS to directories in containers.</span></span> <span data-ttu-id="39ed2-126">Quando o código no contêiner tem acesso ao diretório, o acesso é, na verdade, a um diretório no sistema operacional host.</span><span class="sxs-lookup"><span data-stu-id="39ed2-126">When code in the container has access to the directory, that access is actually to a directory on the host OS.</span></span> <span data-ttu-id="39ed2-127">Esse diretório não está vinculado ao tempo de vida do contêiner em si, sendo que o diretório é gerenciado pelo Docker e isolado em relação à funcionalidade básica do computador host.</span><span class="sxs-lookup"><span data-stu-id="39ed2-127">This directory is not tied to the lifetime of the container itself, and the directory is managed by Docker and isolated from the core functionality of the host machine.</span></span> <span data-ttu-id="39ed2-128">Assim, os volumes de dados são projetados para persistir dados independentemente do tempo de vida do contêiner.</span><span class="sxs-lookup"><span data-stu-id="39ed2-128">Thus, data volumes are designed to persist data independently of the life of the container.</span></span> <span data-ttu-id="39ed2-129">Se você excluir um contêiner ou uma imagem do host do Docker, os dados persistentes no volume de dados não serão excluídos.</span><span class="sxs-lookup"><span data-stu-id="39ed2-129">If you delete a container or an image from the Docker host, the data persisted in the data volume is not deleted.</span></span>

<span data-ttu-id="39ed2-130">Volumes podem ser nomeados ou anônimos (o padrão).</span><span class="sxs-lookup"><span data-stu-id="39ed2-130">Volumes can be named or anonymous (the default).</span></span> <span data-ttu-id="39ed2-131">Volumes nomeados são a evolução dos **Contêineres de Volume de Dados** e facilitam o compartilhamento de dados entre contêineres.</span><span class="sxs-lookup"><span data-stu-id="39ed2-131">Named volumes are the evolution of **Data Volume Containers** and make it easy to share data between containers.</span></span> <span data-ttu-id="39ed2-132">Volumes também dão suporte a drivers de volume que permitem que você armazene dados em hosts remotos, entre outras opções.</span><span class="sxs-lookup"><span data-stu-id="39ed2-132">Volumes also support volume drivers that allow you to store data on remote hosts, among other options.</span></span>

<span data-ttu-id="39ed2-133">**Associar montagens** estiveram disponíveis por um longo tempo e permitir que o mapeamento de qualquer pasta para um ponto de montagem em um contêiner.</span><span class="sxs-lookup"><span data-stu-id="39ed2-133">**Bind mounts** have been available for a long time and allow the mapping of any folder to a mount point in a container.</span></span> <span data-ttu-id="39ed2-134">Montagens de associação têm mais limitações que os volumes e alguns problemas de segurança importantes, por isso os volumes são a opção recomendada.</span><span class="sxs-lookup"><span data-stu-id="39ed2-134">Bind mounts have more limitations than volumes and some important security issues, so volumes are the recommended option.</span></span>

<span data-ttu-id="39ed2-135">**`tmpfs` monta** são pastas virtuais que live apenas na memória do host e nunca são gravadas no sistema de arquivos.</span><span class="sxs-lookup"><span data-stu-id="39ed2-135">**`tmpfs` mounts** are virtual folders that live only in the host's memory and are never written to the filesystem.</span></span> <span data-ttu-id="39ed2-136">Elas são rápidas e seguras, mas usam a memória e destinam-se somente a dados não persistentes.</span><span class="sxs-lookup"><span data-stu-id="39ed2-136">They are fast and secure but use memory and are only meant for non-persistent data.</span></span>

<span data-ttu-id="39ed2-146">**Fontes de dados remotas e ferramentas de cache** como o Banco de Dados SQL do Azure, o Azure Cosmos DB ou um cache remoto como o Redis podem ser usados em aplicativos em contêineres da mesma forma que são usados durante o desenvolvimento sem contêineres.</span><span class="sxs-lookup"><span data-stu-id="39ed2-146">**Remote data sources and cache** tools like Azure SQL Database, Azure Cosmos DB, or a remote cache like Redis can be used in containerized applications the same way they are used when developing without containers.</span></span> <span data-ttu-id="39ed2-147">Essa é uma forma comprovada de armazenar dados de aplicativo de negócios.</span><span class="sxs-lookup"><span data-stu-id="39ed2-147">This is a proven way to store business application data.</span></span>
