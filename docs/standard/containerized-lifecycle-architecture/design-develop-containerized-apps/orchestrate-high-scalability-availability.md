<span data-ttu-id="5def6-246">Uma abordagem sem estado é perfeitamente válida e é mais fácil de implementar do que microsserviços com estado porque a abordagem é semelhante aos padrões tradicionais e já conhecidos.</span><span class="sxs-lookup"><span data-stu-id="5def6-246">A stateless approach is perfectly valid and is easier to implement than stateful microservices because the approach is similar to traditional and well-known patterns.</span></span> <span data-ttu-id="5def6-247">Mas microsserviços sem estado impõem latência entre o processo e as fontes de dados.</span><span class="sxs-lookup"><span data-stu-id="5def6-247">But stateless microservices impose latency between the process and data sources.</span></span> <span data-ttu-id="5def6-248">Eles também envolvem mais movimentação de partes quando você está tentando aprimorar o desempenho com cache e filas adicionais.</span><span class="sxs-lookup"><span data-stu-id="5def6-248">They also involve more moving pieces when you are trying to improve performance with additional cache and queues.</span></span> <span data-ttu-id="5def6-249">O resultado é que você pode acabar com arquiteturas complexas que têm muitas camadas.</span><span class="sxs-lookup"><span data-stu-id="5def6-249">The result is that you can end up with complex architectures that have too many tiers.</span></span>

<span data-ttu-id="5def6-250">Em contraste, [microsserviços com estado](https://docs.microsoft.com/azure/service-fabric/service-fabric-reliable-services-introduction#when-to-use-reliable-services-apis) podem se destacar em cenários avançados porque não há nenhuma latência entre a lógica de domínio e os dados.</span><span class="sxs-lookup"><span data-stu-id="5def6-250">In contrast, [stateful microservices](https://docs.microsoft.com/azure/service-fabric/service-fabric-reliable-services-introduction#when-to-use-reliable-services-apis) can excel in advanced scenarios because there is no latency between the domain logic and data.</span></span> <span data-ttu-id="5def6-251">Processamento de dados pesado, jogos back-ends, bancos de dados como um serviço e outros cenários de baixa latência todos se beneficiam de serviços com monitoração de estado, que fornecem o estado local para acesso mais rápido.</span><span class="sxs-lookup"><span data-stu-id="5def6-251">Heavy data processing, gaming back-ends, databases as a service, and other low-latency scenarios all benefit from stateful services, which provide local state for faster access.</span></span>

<span data-ttu-id="5def6-252">Serviços sem estado e com estado são complementares.</span><span class="sxs-lookup"><span data-stu-id="5def6-252">Stateless and stateful services are complementary.</span></span> <span data-ttu-id="5def6-253">Por exemplo, um serviço com estado pode ser dividido em várias partições.</span><span class="sxs-lookup"><span data-stu-id="5def6-253">For instance, a stateful service could be split into multiple partitions.</span></span> <span data-ttu-id="5def6-254">Para acessar essas partições, talvez seja necessário um serviço sem estado que funcione como um serviço de gateway que saiba como tratar cada partição com base nas chaves de partição.</span><span class="sxs-lookup"><span data-stu-id="5def6-254">To access those partitions, you might need a stateless service acting as a gateway service that knows how to address each partition based on partition keys.</span></span>

<span data-ttu-id="5def6-258">Outras estruturas de microsserviço que permitem serviços com estado, que dão suporte a padrão de ator e que melhoram a tolerância a falhas e a latência entre os dados e a lógica de negócios são Microsoft [Orleans](https://github.com/dotnet/orleans), da Microsoft Research e [Akka.NET](https://getakka.net/).</span><span class="sxs-lookup"><span data-stu-id="5def6-258">Other microservice frameworks that allow stateful services, that support the Actor pattern, and that improve fault tolerance and latency between business logic and data are Microsoft [Orleans](https://github.com/dotnet/orleans), from Microsoft Research, and [Akka.NET](https://getakka.net/).</span></span> <span data-ttu-id="5def6-259">No momento, ambas as estruturas estão melhorando o suporte para Docker.</span><span class="sxs-lookup"><span data-stu-id="5def6-259">Both frameworks are currently improving their support for Docker.</span></span>
