## <a name="monolithic-application-deployed-as-a-container"></a><span data-ttu-id="2fb59-145">Aplicativo monolítico implantado como um contêiner</span><span class="sxs-lookup"><span data-stu-id="2fb59-145">Monolithic application deployed as a container</span></span>

<span data-ttu-id="2fb59-146">Há vantagens em usar contêineres para gerenciar implantações monolíticas.</span><span class="sxs-lookup"><span data-stu-id="2fb59-146">There are benefits to using containers to manage monolithic deployments.</span></span> <span data-ttu-id="2fb59-147">O dimensionamento das instâncias de contêineres é muito mais rápido e fácil do que a implantação de VMs adicionais.</span><span class="sxs-lookup"><span data-stu-id="2fb59-147">Scaling the instances of containers is far faster and easier than deploying additional VMs.</span></span>

<span data-ttu-id="2fb59-148">Implantar atualizações como imagens do Docker é muito mais rápido e eficiente em termos de rede.</span><span class="sxs-lookup"><span data-stu-id="2fb59-148">Deploying updates as Docker images is far faster and network efficient.</span></span> <span data-ttu-id="2fb59-149">Contêineres do docker costumam ser iniciadas em segundos, que agiliza as distribuições.</span><span class="sxs-lookup"><span data-stu-id="2fb59-149">Docker containers typically start in seconds, speeding rollouts.</span></span> <span data-ttu-id="2fb59-150">Subdividir um contêiner do Docker é tão fácil quanto invocar o `docker stop` comando, geralmente leva menos de um segundo.</span><span class="sxs-lookup"><span data-stu-id="2fb59-150">Tearing down a Docker container is as easy as invoking the `docker stop` command, typically completing in less than a second.</span></span>

<span data-ttu-id="2fb59-151">Como os contêineres são inerentemente imutáveis por design, você nunca precisa se preocupar com VMs corrompidas porque se esqueceu de um script de atualização para levar em conta alguma configuração específica ou arquivos deixados no disco.</span><span class="sxs-lookup"><span data-stu-id="2fb59-151">Because containers are inherently immutable, by design, you never need to worry about corrupted VMs because an update script forgot to account for some specific configuration or file left on disk.</span></span>
