## <a name="monolithic-application-deployed-as-a-container"></a><span data-ttu-id="ab384-137">Aplicativo monolítico implantado como um contêiner</span><span class="sxs-lookup"><span data-stu-id="ab384-137">Monolithic application deployed as a container</span></span>

<span data-ttu-id="ab384-138">Há vantagens em usar contêineres para gerenciar implantações monolíticas.</span><span class="sxs-lookup"><span data-stu-id="ab384-138">There are benefits to using containers to manage monolithic deployments.</span></span> <span data-ttu-id="ab384-139">O dimensionamento das instâncias de contêineres é muito mais rápido e fácil do que a implantação de VMs adicionais.</span><span class="sxs-lookup"><span data-stu-id="ab384-139">Scaling the instances of containers is far faster and easier than deploying additional VMs.</span></span> <span data-ttu-id="ab384-140">Embora os conjuntos de dimensionamento de VM são um ótimo recurso para dimensionar as VMs, que são necessários para hospedar seus contêineres do Docker, eles demoram para configurar.</span><span class="sxs-lookup"><span data-stu-id="ab384-140">Although VM Scale Sets are a great feature to scale VMs, which are required to host your Docker containers, they take time to set up.</span></span> <span data-ttu-id="ab384-141">Quando implantada como instâncias de aplicativo, a configuração do aplicativo é gerenciada como parte da VM.</span><span class="sxs-lookup"><span data-stu-id="ab384-141">When deployed as app instances, the configuration of the app is managed as part of the VM.</span></span>

<span data-ttu-id="ab384-142">Implantar atualizações como imagens do Docker é muito mais rápido e eficiente em termos de rede.</span><span class="sxs-lookup"><span data-stu-id="ab384-142">Deploying updates as Docker images is far faster and network efficient.</span></span> <span data-ttu-id="ab384-143">As instâncias de Vn podem ser configuradas nos hosts mesmos conforme as suas instâncias Vn-1, eliminando custos resultantes de VMs adicionais.</span><span class="sxs-lookup"><span data-stu-id="ab384-143">The Vn instances can be set up on the same hosts as your Vn-1 instances, eliminating added costs resulting from additional VMs.</span></span> <span data-ttu-id="ab384-144">Imagens do docker costumam ser iniciadas em segundos, que agiliza as distribuições.</span><span class="sxs-lookup"><span data-stu-id="ab384-144">Docker images typically start in seconds, speeding rollouts.</span></span> <span data-ttu-id="ab384-145">Subdividir uma instância do Docker é tão fácil quanto invocar o `docker stop` comando, geralmente leva menos de um segundo.</span><span class="sxs-lookup"><span data-stu-id="ab384-145">Tearing down a Docker instance is as easy as invoking the `docker stop` command, typically completing in less than a second.</span></span>

<span data-ttu-id="ab384-146">Como os contêineres são inerentemente imutáveis por design, você nunca precisa se preocupar com VMs corrompidas porque se esqueceu de um script de atualização para levar em conta alguma configuração específica ou arquivos deixados no disco.</span><span class="sxs-lookup"><span data-stu-id="ab384-146">Because containers are inherently immutable, by design, you never need to worry about corrupted VMs because an update script forgot to account for some specific configuration or file left on disk.</span></span>
