## <a name="usability-and-functionality"></a><span data-ttu-id="29290-108">Usabilidade e funcionalidade</span><span class="sxs-lookup"><span data-stu-id="29290-108">Usability and functionality</span></span>

```csharp-interactive
{
    DateTime.UtcNow.AddHours(-1),

    Console.WriteLine($"Ticks: {anonymous.Ticks}, formatted: {anonymous.Formatted}");

```

<span data-ttu-id="29290-112">Os tipos anônimos são instanciados usando o [`new`](../../csharp/language-reference/operators/new-operator.md) operador, e os nomes e tipos de propriedade são inferidos da declaração.</span><span class="sxs-lookup"><span data-stu-id="29290-112">Anonymous types are instantiated by using the [`new`](../../csharp/language-reference/operators/new-operator.md) operator, and the property names and types are inferred from the declaration.</span></span> <span data-ttu-id="29290-113">Se dois ou mais inicializadores de objeto anônimo no mesmo assembly especificarem uma sequência de propriedades que estão na mesma ordem e que têm os mesmos nomes e tipos, o compilador tratará os objetos como instâncias do mesmo tipo.</span><span class="sxs-lookup"><span data-stu-id="29290-113">If two or more anonymous object initializers in the same assembly specify a sequence of properties that are in the same order and that have the same names and types, the compiler treats the objects as instances of the same type.</span></span> <span data-ttu-id="29290-114">Eles compartilham o mesmo tipo de informação gerado pelo compilador.</span><span class="sxs-lookup"><span data-stu-id="29290-114">They share the same compiler-generated type information.</span></span>

<span data-ttu-id="29290-120">Com o <xref:System.Tuple%602?displayProperty=nameWithType> , a instância expõe as propriedades de item numeradas, como `Item1` e `Item2` .</span><span class="sxs-lookup"><span data-stu-id="29290-120">With the <xref:System.Tuple%602?displayProperty=nameWithType>, the instance exposes numbered item properties, such as `Item1`, and `Item2`.</span></span> <span data-ttu-id="29290-121">Esses nomes de propriedade podem dificultar a compreensão da intenção dos valores de propriedade, uma vez que o nome da propriedade fornece apenas o ordinal.</span><span class="sxs-lookup"><span data-stu-id="29290-121">These property names can make it more difficult to understand the intent of the property values, as the property name only provides the ordinal.</span></span> <span data-ttu-id="29290-122">Além disso, os `System.Tuple` tipos são `class` tipos de referência.</span><span class="sxs-lookup"><span data-stu-id="29290-122">Furthermore, the `System.Tuple` types are reference `class` types.</span></span> <span data-ttu-id="29290-123"><xref:System.ValueTuple%602?displayProperty=nameWithType>No entanto, é um `struct` tipo de valor.</span><span class="sxs-lookup"><span data-stu-id="29290-123">The <xref:System.ValueTuple%602?displayProperty=nameWithType> however, is a value `struct` type.</span></span> <span data-ttu-id="29290-124">O trecho de código C# a seguir usa o `ValueTuple<string, long>` para projetar no.</span><span class="sxs-lookup"><span data-stu-id="29290-124">The following C# snippet, uses `ValueTuple<string, long>` to project into.</span></span> <span data-ttu-id="29290-125">Ao fazer isso, ele atribui usando uma sintaxe literal.</span><span class="sxs-lookup"><span data-stu-id="29290-125">In doing so, it assigns using a literal syntax.</span></span>

```csharp-interactive
    DateTime.UtcNow.AddHours(1),

            dates.Select(
```
