# <a name="common-language-runtime-clr-overview"></a><span data-ttu-id="54d19-102">Visão geral do CLR (Common Language Runtime)</span><span class="sxs-lookup"><span data-stu-id="54d19-102">Common Language Runtime (CLR) overview</span></span>

<span data-ttu-id="54d19-103">O .NET Framework fornece um ambiente de tempo de execução chamado Common Language Runtime, que executa o código e fornece serviços que facilitam o processo de desenvolvimento.</span><span class="sxs-lookup"><span data-stu-id="54d19-103">The .NET Framework provides a run-time environment called the common language runtime, which runs the code and provides services that make the development process easier.</span></span>

<span data-ttu-id="54d19-104">Compiladores e ferramentas expõem as funcionalidades do Common Language Runtime e permitem que você grave um código que se beneficia desse ambiente de execução gerenciado.</span><span class="sxs-lookup"><span data-stu-id="54d19-104">Compilers and tools expose the common language runtime's functionality and enable you to write code that benefits from this managed execution environment.</span></span> <span data-ttu-id="54d19-105">O código que você desenvolve com um compilador de linguagem que visa o runtime é chamado de código gerenciado. Ele aproveita recursos como integração em qualquer idioma, tratamento de exceções em qualquer idioma, segurança aprimorada, suporte a controle de versão e implantação, um modelo simplificado para interação entre componentes, além de serviços de depuração e criação de perfil.</span><span class="sxs-lookup"><span data-stu-id="54d19-105">Code that you develop with a language compiler that targets the runtime is called managed code; it benefits from features such as cross-language integration, cross-language exception handling, enhanced security, versioning and deployment support, a simplified model for component interaction, and debugging and profiling services.</span></span>

> [!NOTE]
> <span data-ttu-id="54d19-106">Compiladores e ferramentas podem produzir saída que o Common Language Runtime pode consumir porque o sistema de tipos, o formato dos metadados e o ambiente de tempo de execução (o sistema virtual de execução) são todos definidos por um padrão público, a especificação Common Language Infrastructure ECMA.</span><span class="sxs-lookup"><span data-stu-id="54d19-106">Compilers and tools are able to produce output that the common language runtime can consume because the type system, the format of metadata, and the runtime environment (the virtual execution system) are all defined by a public standard, the ECMA Common Language Infrastructure specification.</span></span> <span data-ttu-id="54d19-107">Para obter mais informações, consulte [Especificações ECMA C# e Common Language Infrastructure](https://visualstudio.microsoft.com/license-terms/ecma-c-common-language-infrastructure-standards/).</span><span class="sxs-lookup"><span data-stu-id="54d19-107">For more information, see [ECMA C# and Common Language Infrastructure Specifications](https://visualstudio.microsoft.com/license-terms/ecma-c-common-language-infrastructure-standards/).</span></span>

<span data-ttu-id="54d19-108">Para habilitar o runtime para fornecer serviços de código gerenciado, os compiladores de linguagem devem emitir metadados que descrevam os tipos, os membros e as referências em seu código.</span><span class="sxs-lookup"><span data-stu-id="54d19-108">To enable the runtime to provide services to managed code, language compilers must emit metadata that describes the types, members, and references in your code.</span></span> <span data-ttu-id="54d19-109">Os metadados são armazenados com o código. Todo arquivo PE (Portable Executable) do Common Language Runtime carregável contém metadados.</span><span class="sxs-lookup"><span data-stu-id="54d19-109">Metadata is stored with the code; every loadable common language runtime portable executable (PE) file contains metadata.</span></span> <span data-ttu-id="54d19-110">O runtime usa metadados para localizar e carregar classes, dispor instâncias na memória, resolver invocações de método, gerar código nativo, impor segurança e definir limites de contexto do runtime.</span><span class="sxs-lookup"><span data-stu-id="54d19-110">The runtime uses metadata to locate and load classes, lay out instances in memory, resolve method invocations, generate native code, enforce security, and set run-time context boundaries.</span></span>

<span data-ttu-id="54d19-111">O runtime identifica automaticamente o layout de objeto e gerencia referências a objetos, liberando-os quando eles não estão sendo mais usados.</span><span class="sxs-lookup"><span data-stu-id="54d19-111">The runtime automatically handles object layout and manages references to objects, releasing them when they are no longer being used.</span></span> <span data-ttu-id="54d19-112">Objetos cujos tempos de vida são gerenciados dessa forma são chamados de dados gerenciados.</span><span class="sxs-lookup"><span data-stu-id="54d19-112">Objects whose lifetimes are managed in this way are called managed data.</span></span> <span data-ttu-id="54d19-113">A coleta de lixo elimina perdas de memória, bem como alguns outros erros de programação comuns.</span><span class="sxs-lookup"><span data-stu-id="54d19-113">Garbage collection eliminates memory leaks as well as some other common programming errors.</span></span> <span data-ttu-id="54d19-114">Se o código for gerenciado, você poderá usar dados gerenciados, dados não gerenciados ou ambos no seu aplicativo do .NET Framework.</span><span class="sxs-lookup"><span data-stu-id="54d19-114">If your code is managed, you can use managed data, unmanaged data, or both managed and unmanaged data in your .NET Framework application.</span></span> <span data-ttu-id="54d19-115">Como os compiladores de linguagem fornecem seus próprios tipos, como tipos primitivos, você talvez sem sempre saiba (ou precise saber) se os dados estão sendo gerenciados.</span><span class="sxs-lookup"><span data-stu-id="54d19-115">Because language compilers supply their own types, such as primitive types, you might not always know (or need to know) whether your data is being managed.</span></span>

<span data-ttu-id="54d19-116">O Common Language Runtime facilita a criação de componentes e aplicativos cujos objetos interagem entre linguagens.</span><span class="sxs-lookup"><span data-stu-id="54d19-116">The common language runtime makes it easy to design components and applications whose objects interact across languages.</span></span> <span data-ttu-id="54d19-117">Objetos gravados em linguagens diferentes podem se comunicar entre si e seus comportamentos podem estar totalmente integrados.</span><span class="sxs-lookup"><span data-stu-id="54d19-117">Objects written in different languages can communicate with each other, and their behaviors can be tightly integrated.</span></span> <span data-ttu-id="54d19-118">Por exemplo, você pode definir uma classe e usar uma linguagem diferente para derivar uma classe da classe original ou chamar um método na classe original.</span><span class="sxs-lookup"><span data-stu-id="54d19-118">For example, you can define a class and then use a different language to derive a class from your original class or call a method on the original class.</span></span> <span data-ttu-id="54d19-119">Você também pode passar uma instância de uma classe para um método de uma classe gravado em uma linguagem diferente.</span><span class="sxs-lookup"><span data-stu-id="54d19-119">You can also pass an instance of a class to a method of a class written in a different language.</span></span> <span data-ttu-id="54d19-120">Essa integração em qualquer idioma é possível porque os compiladores de linguagens e ferramentas que segmentam o runtime usam um Common Type System definido pelo runtime e seguem as regras do runtime para definir novos tipos, bem como para criar, usar, manter e associar a tipos.</span><span class="sxs-lookup"><span data-stu-id="54d19-120">This cross-language integration is possible because language compilers and tools that target the runtime use a common type system defined by the <span data-ttu-id="54d19-121">Como parte de seus metadados, todos os componentes gerenciados transportam informações sobre os componentes e os recursos com base nos quais eles foram compilados.</span><span class="sxs-lookup"><span data-stu-id="54d19-121">As part of their metadata, all managed components carry information about the components and resources they were built against.</span></span> <span data-ttu-id="54d19-122">O runtime usa essas informações para garantir que o componente ou o aplicativo tenha as versões especificadas de tudo o que precisa, o que torna seu código menos suscetível à interrupção devido a alguma dependência não atendida.</span><span class="sxs-lookup"><span data-stu-id="54d19-122">The runtime uses this information to ensure that your component or application has the specified versions of everything it needs, which makes your code less likely to break because of some unmet dependency.</span></span> <span data-ttu-id="54d19-123">As informações de registro e os dados de estado não são mais armazenados no registro onde pode ser difícil estabelecer e mantê-los.</span><span class="sxs-lookup"><span data-stu-id="54d19-123">Registration information and state data are no longer stored in the registry where they can be difficult to establish and maintain.</span></span> <span data-ttu-id="54d19-124">Em vez de isso, as informações sobre os tipos que você define (e suas dependências) são armazenadas com o código como metadados, fazendo com que as tarefas de replicação do componente e remoção sejam muito menos complicadas.</span><span class="sxs-lookup"><span data-stu-id="54d19-124">Instead, information about the types you define (and their dependencies) is stored with the code as metadata, making the tasks of component replication and removal much less complicated.</span></span>

<span data-ttu-id="54d19-125">Compiladores de linguagens e ferramentas expõem a funcionalidade do runtime da maneira como seriam úteis e intuitivas para desenvolvedores.</span><span class="sxs-lookup"><span data-stu-id="54d19-125">Language compilers and tools expose the runtime's functionality in ways that are intended to be useful and intuitive to developers.</span></span> <span data-ttu-id="54d19-126">Isso significa que alguns recursos do runtime devem ser mais perceptíveis em um ambiente do que em outro.</span><span class="sxs-lookup"><span data-stu-id="54d19-126">This means that some features of the runtime might be more noticeable in one environment than in another.</span></span> <span data-ttu-id="54d19-127">Como você usa o runtime depende de quais compiladores de linguagem ou ferramentas você usa.</span><span class="sxs-lookup"><span data-stu-id="54d19-127">How you experience the runtime depends on which language compilers or tools you use.</span></span> <span data-ttu-id="54d19-128">Por exemplo, se for um desenvolvedor do Visual Basic, você poderá observar que, com o Common Language Runtime, a linguagem do Visual Basic tem mais recursos orientados a objeto do que antes.</span><span class="sxs-lookup"><span data-stu-id="54d19-128">For example, if you are a Visual Basic developer, you might notice that with the common language runtime, the Visual Basic language has more object-oriented features than before.</span></span> <span data-ttu-id="54d19-129">O runtime oferece os seguintes benefícios:</span><span class="sxs-lookup"><span data-stu-id="54d19-129">The runtime provides the following benefits:</span></span>

- <span data-ttu-id="54d19-132">Tipos extensíveis fornecidos por uma biblioteca de classes.</span><span class="sxs-lookup"><span data-stu-id="54d19-132">Extensible types provided by a class library.</span></span>

- <span data-ttu-id="54d19-137">Coleta de lixo.</span><span class="sxs-lookup"><span data-stu-id="54d19-137">Garbage collection.</span></span>
