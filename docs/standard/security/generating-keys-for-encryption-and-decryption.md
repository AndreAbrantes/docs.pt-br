## <a name="symmetric-keys"></a><span data-ttu-id="9dc6d-111">Chaves simétricas</span><span class="sxs-lookup"><span data-stu-id="9dc6d-111">Symmetric Keys</span></span>  

 <span data-ttu-id="9dc6d-112">As classes de criptografia simétrica fornecidas pelo .NET exigem uma chave e um novo vetor de inicialização (IV) para criptografar e descriptografar dados.</span><span class="sxs-lookup"><span data-stu-id="9dc6d-112">The symmetric encryption classes supplied by .NET require a key and a new initialization vector (IV) to encrypt and decrypt data.</span></span> <span data-ttu-id="9dc6d-113">Sempre que você cria uma nova instância de uma das classes de criptografia simétricas gerenciadas usando o método sem parâmetros `Create()` , uma nova chave e IV são criadas automaticamente.</span><span class="sxs-lookup"><span data-stu-id="9dc6d-113">Whenever you create a new instance of one of the managed symmetric cryptographic classes using the parameterless `Create()` method, a new key and IV are automatically created.</span></span> <span data-ttu-id="9dc6d-114">Qualquer pessoa que você permita descriptografar seus dados deve ter a mesma chave e IV e usar o mesmo algoritmo.</span><span class="sxs-lookup"><span data-stu-id="9dc6d-114">Anyone that you allow to decrypt your data must possess the same key and IV and use the same algorithm.</span></span> <span data-ttu-id="9dc6d-115">Em geral, uma nova chave e IV devem ser criadas para cada sessão e nem a chave nem o IV devem ser armazenados para uso em uma sessão posterior.</span><span class="sxs-lookup"><span data-stu-id="9dc6d-115">Generally, a new key and IV should be created for every session, and neither the key nor IV should be stored for use in a later session.</span></span>  
  
 <span data-ttu-id="9dc6d-116">Para comunicar uma chave simétrica e um IV a uma parte remota, você normalmente criptografaria a chave simétrica usando a criptografia assimétrica.</span><span class="sxs-lookup"><span data-stu-id="9dc6d-116">To communicate a symmetric key and IV to a remote party, you would usually encrypt the symmetric key by using asymmetric encryption.</span></span> <span data-ttu-id="9dc6d-117">Enviar a chave por meio de uma rede insegura sem criptografá-la não é seguro, pois qualquer pessoa que intercepta a chave e o IV pode então descriptografar seus dados.</span><span class="sxs-lookup"><span data-stu-id="9dc6d-117">Sending the key across an insecure network without encrypting it is unsafe, because anyone who intercepts the key and IV can then decrypt your data.</span></span>

```csharp  
Aes aes = Aes.Create();
```  

 <span data-ttu-id="9dc6d-119">Quando o código anterior é executado, uma nova chave e IV são gerados e colocados nas propriedades **Key** e **IV** , respectivamente.</span><span class="sxs-lookup"><span data-stu-id="9dc6d-119">When the previous code is executed, a new key and IV are generated and placed in the **Key** and **IV** properties, respectively.</span></span>  
  
 <span data-ttu-id="9dc6d-120">Às vezes, talvez seja necessário gerar várias chaves.</span><span class="sxs-lookup"><span data-stu-id="9dc6d-120">Sometimes you might need to generate multiple keys.</span></span> <span data-ttu-id="9dc6d-121">Nessa situação, você pode criar uma nova instância de uma classe que implementa um algoritmo simétrico e, em seguida, criar uma nova chave e IV chamando os métodos **GenerateKey** e **GenerateIV** .</span><span class="sxs-lookup"><span data-stu-id="9dc6d-121">In this situation, you can create a new instance of a class that implements a symmetric algorithm and then create a new key and IV by calling the **GenerateKey** and **GenerateIV** methods.</span></span> <span data-ttu-id="9dc6d-122">O exemplo de código a seguir ilustra como criar novas chaves e IVs após uma nova instância da classe de criptografia simétrica ter sido feita.</span><span class="sxs-lookup"><span data-stu-id="9dc6d-122">The following code example illustrates how to create new keys and IVs after a new instance of the symmetric cryptographic class has been made.</span></span>

```vb  
aes.GenerateIV()  
aes.GenerateKey()  
```  
  
```csharp  
Aes aes = Aes.Create();  
aes.GenerateIV();  
aes.GenerateKey();
```  

 <span data-ttu-id="9dc6d-123">Quando o código anterior é executado, uma chave e IV são gerados quando a nova instância do **AES** é feita.</span><span class="sxs-lookup"><span data-stu-id="9dc6d-123">When the preceding code is executed, a key and IV are generated when the new instance of **Aes** is made.</span></span> <span data-ttu-id="9dc6d-124">Outra chave e IV são criados quando os métodos **GenerateKey** e **GenerateIV** são chamados.</span><span class="sxs-lookup"><span data-stu-id="9dc6d-124">Another key and IV are created when the **GenerateKey** and **GenerateIV** methods are called.</span></span>
  
## <a name="asymmetric-keys"></a><span data-ttu-id="9dc6d-125">Chaves assimétricas</span><span class="sxs-lookup"><span data-stu-id="9dc6d-125">Asymmetric Keys</span></span>

 <span data-ttu-id="9dc6d-126">O .NET fornece a <xref:System.Security.Cryptography.RSA> classe para a criptografia assimétrica.</span><span class="sxs-lookup"><span data-stu-id="9dc6d-126">.NET provides the <xref:System.Security.Cryptography.RSA> class for asymmetric encryption.</span></span> <span data-ttu-id="9dc6d-127">Essa classe cria um par de chaves pública/privada quando você usa o método sem parâmetros `Create()` para criar uma nova instância.</span><span class="sxs-lookup"><span data-stu-id="9dc6d-127">This class creates a public/private key pair when you use the parameterless `Create()` method to create a new instance.</span></span> <span data-ttu-id="9dc6d-128">Chaves assimétricas podem ser armazenadas para uso em várias sessões ou geradas apenas para uma sessão.</span><span class="sxs-lookup"><span data-stu-id="9dc6d-128">Asymmetric keys can be either stored for use in multiple sessions or generated for one session only.</span></span> <span data-ttu-id="9dc6d-129">Embora a chave pública possa ser disponibilizada para o público geral, a chave privada deve ser bastante protegida.</span><span class="sxs-lookup"><span data-stu-id="9dc6d-129">While the public key can be made generally available, the private key should be closely guarded.</span></span>

 <span data-ttu-id="9dc6d-136">As chaves privadas assimétricas nunca devem ser armazenadas no formato textual nem como texto sem formatação no computador local.</span><span class="sxs-lookup"><span data-stu-id="9dc6d-136">Asymmetric private keys should never be stored verbatim or in plain text on the local computer.</span></span> <span data-ttu-id="9dc6d-137">Se precisar armazenar uma chave privada, você deverá usar um contêiner de chave.</span><span class="sxs-lookup"><span data-stu-id="9dc6d-137">If you need to store a private key, you should use a key container.</span></span> <span data-ttu-id="9dc6d-138">Para obter mais informações sobre como armazenar uma chave privada em um contêiner de chave, consulte [como armazenar chaves assimétricas em um contêiner de chave](how-to-store-asymmetric-keys-in-a-key-container.md).</span><span class="sxs-lookup"><span data-stu-id="9dc6d-138">For more information about how to store a private key in a key container, see [How to: Store Asymmetric Keys in a Key Container](how-to-store-asymmetric-keys-in-a-key-container.md).</span></span>  
  
 <span data-ttu-id="9dc6d-139">O exemplo de código a seguir cria uma nova instância da classe **RSA** , criando um par de chaves pública/privada e salva as informações de chave pública em uma estrutura **RSAParameters** .</span><span class="sxs-lookup"><span data-stu-id="9dc6d-139">The following code example creates a new instance of the **RSA** class, creating a public/private key pair, and saves the public key information to an **RSAParameters** structure.</span></span>
