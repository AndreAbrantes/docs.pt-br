<span data-ttu-id="a18d8-149">A abordagem a seguir é a mais poderosa e, portanto, potencialmente perigosa (se tiver sido feita incorretamente) para codificação de segurança: sua biblioteca serve como uma interface para que outro código acesse determinados recursos que não estão disponíveis de outra forma, assim como as classes .NET impõem permissões para os recursos que usam.</span><span class="sxs-lookup"><span data-stu-id="a18d8-149">The following approach is the most powerful and hence potentially dangerous (if done incorrectly) for security coding: your library serves as an interface for other code to access certain resources that aren't otherwise available, just as the .NET classes enforce permissions for the resources they use.</span></span> <span data-ttu-id="a18d8-150">Sempre que você expor um recurso, seu código deve primeiro solicitar a permissão apropriada para o recurso (ou seja, ele deve executar uma verificação de segurança) e, em seguida, declarar seus direitos para executar a operação real.</span><span class="sxs-lookup"><span data-stu-id="a18d8-150">Wherever you expose a resource, your code must first demand the permission appropriate to the resource (that is, it must perform a security check) and then typically assert its rights to perform the actual operation.</span></span>
