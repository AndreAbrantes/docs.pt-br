# <a name="attributes"></a><span data-ttu-id="60e71-102">{1&gt;{2&gt;Atributos&lt;2}&lt;1}</span><span class="sxs-lookup"><span data-stu-id="60e71-102">Attributes</span></span>
<span data-ttu-id="60e71-103"><xref:System.Attribute?displayProperty=nameWithType> é uma classe base usada para definir atributos personalizados.</span><span class="sxs-lookup"><span data-stu-id="60e71-103"><xref:System.Attribute?displayProperty=nameWithType> is a base class used to define custom attributes.</span></span>  
  
 <span data-ttu-id="60e71-104">Atributos são anotações que podem ser adicionadas a elementos de programação, como assemblies, tipos, membros e parâmetros.</span><span class="sxs-lookup"><span data-stu-id="60e71-104">Attributes are annotations that can be added to programming elements such as assemblies, types, members, and parameters.</span></span> <span data-ttu-id="60e71-105">Eles são armazenados nos metadados do assembly e podem ser acessados em tempo de execução usando as APIs de reflexão.</span><span class="sxs-lookup"><span data-stu-id="60e71-105">They are stored in the metadata of the assembly and can be accessed at runtime using the reflection APIs.</span></span> <span data-ttu-id="60e71-106">Por exemplo, a estrutura define o <xref:System.ObsoleteAttribute>, que pode ser aplicado a um tipo ou um membro para indicar que o tipo ou o membro foi preterido.</span><span class="sxs-lookup"><span data-stu-id="60e71-106">For example, the Framework defines the <xref:System.ObsoleteAttribute>, which can be applied to a type or a member to indicate that the type or member has been deprecated.</span></span>  
  
 <span data-ttu-id="60e71-107">Os atributos podem ter uma ou mais propriedades que contêm dados adicionais relacionados ao atributo.</span><span class="sxs-lookup"><span data-stu-id="60e71-107">Attributes can have one or more properties that carry additional data related to the attribute.</span></span> <span data-ttu-id="60e71-108">Por exemplo, `ObsoleteAttribute` poderia conter informações adicionais sobre a versão na qual um tipo ou um membro foi preterido e a descrição das novas APIs que substituem a API obsoleta.</span><span class="sxs-lookup"><span data-stu-id="60e71-108">For example, `ObsoleteAttribute` could carry additional information about the release in which a type or a member got deprecated and the description of the new APIs replacing the obsolete API.</span></span>  
  
 <span data-ttu-id="60e71-109">Algumas propriedades de um atributo devem ser especificadas quando o atributo é aplicado.</span><span class="sxs-lookup"><span data-stu-id="60e71-109">Some properties of an attribute must be specified when the attribute is applied.</span></span> <span data-ttu-id="60e71-110">Eles são chamados de propriedades obrigatórias ou argumentos necessários, pois são representados como parâmetros de Construtor posicionais.</span><span class="sxs-lookup"><span data-stu-id="60e71-110">These are referred to as the required properties or required arguments, because they are represented as positional constructor parameters.</span></span> <span data-ttu-id="60e71-111">Por exemplo, a propriedade <xref:System.Diagnostics.ConditionalAttribute.ConditionString%2A> da <xref:System.Diagnostics.ConditionalAttribute> é uma propriedade necessária.</span><span class="sxs-lookup"><span data-stu-id="60e71-111">For example, the <xref:System.Diagnostics.ConditionalAttribute.ConditionString%2A> property of the <xref:System.Diagnostics.ConditionalAttribute> is a required property.</span></span>  
  
 <span data-ttu-id="60e71-112">As propriedades que não precisam necessariamente ser especificadas quando o atributo é aplicado são chamadas de propriedades opcionais (ou argumentos opcionais).</span><span class="sxs-lookup"><span data-stu-id="60e71-112">Properties that do not necessarily have to be specified when the attribute is applied are called optional properties (or optional arguments).</span></span> <span data-ttu-id="60e71-113">Elas são representadas por propriedades de tabela.</span><span class="sxs-lookup"><span data-stu-id="60e71-113">They are represented by settable properties.</span></span> <span data-ttu-id="60e71-114">Os compiladores fornecem uma sintaxe especial para definir essas propriedades quando um atributo é aplicado.</span><span class="sxs-lookup"><span data-stu-id="60e71-114">Compilers provide special syntax to set these properties when an attribute is applied.</span></span> <span data-ttu-id="60e71-115">Por exemplo, a propriedade <xref:System.AttributeUsageAttribute.Inherited%2A?displayProperty=nameWithType> representa um argumento opcional.</span><span class="sxs-lookup"><span data-stu-id="60e71-115">For example, the <xref:System.AttributeUsageAttribute.Inherited%2A?displayProperty=nameWithType> property represents an optional argument.</span></span>  
  
 <span data-ttu-id="60e71-116">**✓ DO** nome classes de atributos personalizados com o sufixo "Atributo".</span><span class="sxs-lookup"><span data-stu-id="60e71-116">**✓ DO** name custom attribute classes with the suffix "Attribute."</span></span>  
  
 <span data-ttu-id="60e71-117">**✓ DO** se aplicam a <xref:System.AttributeUsageAttribute> para os atributos personalizados.</span><span class="sxs-lookup"><span data-stu-id="60e71-117">**✓ DO** apply the <xref:System.AttributeUsageAttribute> to custom attributes.</span></span>  
  
 <span data-ttu-id="60e71-118">**✓ DO** fornecem propriedades configuráveis para argumentos opcionais.</span><span class="sxs-lookup"><span data-stu-id="60e71-118">**✓ DO** provide settable properties for optional arguments.</span></span>  
  
 <span data-ttu-id="60e71-119">**✓ DO** fornecem propriedades somente obtenção para os argumentos necessários.</span><span class="sxs-lookup"><span data-stu-id="60e71-119">**✓ DO** provide get-only properties for required arguments.</span></span>  
  
 <span data-ttu-id="60e71-120">**✓ DO** fornecer parâmetros de construtor para inicializar propriedades correspondentes para os argumentos necessários.</span><span class="sxs-lookup"><span data-stu-id="60e71-120">**✓ DO** provide constructor parameters to initialize properties corresponding to required arguments.</span></span> <span data-ttu-id="60e71-121">Cada parâmetro deve ter o mesmo nome (embora com maiúsculas e minúsculas diferentes) como a propriedade correspondente.</span><span class="sxs-lookup"><span data-stu-id="60e71-121">Each parameter should have the same name (although with different casing) as the corresponding property.</span></span>  
  
 <span data-ttu-id="60e71-122">**X AVOID** fornecendo os parâmetros do construtor para inicializar propriedades correspondentes para os argumentos opcionais.</span><span class="sxs-lookup"><span data-stu-id="60e71-122">**X AVOID** providing constructor parameters to initialize properties corresponding to the optional arguments.</span></span>  
  
 <span data-ttu-id="60e71-123">Em outras palavras, não têm propriedades que podem ser definidas com um construtor e um setter.</span><span class="sxs-lookup"><span data-stu-id="60e71-123">In other words, do not have properties that can be set with both a constructor and a setter.</span></span> <span data-ttu-id="60e71-124">Essa diretriz torna muito explícito quais argumentos são opcionais e que são necessários e evita que você tenha duas maneiras de fazer a mesma coisa.</span><span class="sxs-lookup"><span data-stu-id="60e71-124">This guideline makes very explicit which arguments are optional and which are required, and avoids having two ways of doing the same thing.</span></span>  
  
 <span data-ttu-id="60e71-125">**X AVOID** sobrecarga construtores de atributo personalizado.</span><span class="sxs-lookup"><span data-stu-id="60e71-125">**X AVOID** overloading custom attribute constructors.</span></span>  
  
 <span data-ttu-id="60e71-126">Ter apenas um construtor se comunica claramente ao usuário quais argumentos são necessários e quais são opcionais.</span><span class="sxs-lookup"><span data-stu-id="60e71-126">Having only one constructor clearly communicates to the user which arguments are required and which are optional.</span></span>  
  
 <span data-ttu-id="60e71-127">**✓ DO** lacrar classes de atributo personalizado, se possível.</span><span class="sxs-lookup"><span data-stu-id="60e71-127">**✓ DO** seal custom attribute classes, if possible.</span></span> <span data-ttu-id="60e71-128">Isso torna a pesquisa para o atributo mais rápida.</span><span class="sxs-lookup"><span data-stu-id="60e71-128">This makes the look-up for the attribute faster.</span></span>
