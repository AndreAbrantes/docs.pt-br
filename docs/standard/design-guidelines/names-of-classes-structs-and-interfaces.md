 <span data-ttu-id="0f2b1-111">Isso é muito legível e explica a relação claramente.</span><span class="sxs-lookup"><span data-stu-id="0f2b1-111">This is very readable and explains the relationship clearly.</span></span> <span data-ttu-id="0f2b1-112">Alguns exemplos disso no código são: `ArgumentOutOfRangeException` , que é um tipo de `Exception` , e `SerializableAttribute` , que é um tipo de `Attribute` .</span><span class="sxs-lookup"><span data-stu-id="0f2b1-112">Some examples of this in code are: `ArgumentOutOfRangeException`, which is a kind of `Exception`, and `SerializableAttribute`, which is a kind of `Attribute`.</span></span> <span data-ttu-id="0f2b1-113">No entanto, é importante usar o julgamento razoável na aplicação dessa diretriz; por exemplo, a `Button` classe é um tipo de `Control` evento, embora `Control` não apareça em seu nome.</span><span class="sxs-lookup"><span data-stu-id="0f2b1-113">However, it is important to use reasonable judgment in applying this guideline; for example, the `Button` class is a kind of `Control` event, although `Control` doesn’t appear in its name.</span></span>

 <span data-ttu-id="0f2b1-121">✔️ os parâmetros de tipo genérico de nome com nomes descritivos, a menos que um nome de letra única seja totalmente auto-explicativo e um nome descritivo não adicione valor.</span><span class="sxs-lookup"><span data-stu-id="0f2b1-121">✔️ DO name generic type parameters with descriptive names unless a single-letter name is completely self-explanatory and a descriptive name would not add value.</span></span>

    TSession Session { get; }

 <span data-ttu-id="0f2b1-124">✔️ CONSIDERAR a indicação de restrições colocadas em um parâmetro de tipo no nome do parâmetro.</span><span class="sxs-lookup"><span data-stu-id="0f2b1-124">✔️ CONSIDER indicating constraints placed on a type parameter in the name of the parameter.</span></span>
