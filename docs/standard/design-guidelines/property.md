# <a name="property-design"></a><span data-ttu-id="d194e-102">Design de propriedade</span><span class="sxs-lookup"><span data-stu-id="d194e-102">Property Design</span></span>
<span data-ttu-id="d194e-103">Embora as propriedades sejam tecnicamente muito semelhantes aos métodos, elas são muito diferentes em termos de seus cenários de uso.</span><span class="sxs-lookup"><span data-stu-id="d194e-103">Although properties are technically very similar to methods, they are quite different in terms of their usage scenarios.</span></span> <span data-ttu-id="d194e-104">Eles devem ser vistos como campos inteligentes.</span><span class="sxs-lookup"><span data-stu-id="d194e-104">They should be seen as smart fields.</span></span> <span data-ttu-id="d194e-105">Eles têm a sintaxe de chamada de campos e a flexibilidade dos métodos.</span><span class="sxs-lookup"><span data-stu-id="d194e-105">They have the calling syntax of fields, and the flexibility of methods.</span></span>

 <span data-ttu-id="d194e-106">✔️ criar propriedades somente obtenção se o chamador não puder alterar o valor da propriedade.</span><span class="sxs-lookup"><span data-stu-id="d194e-106">✔️ DO create get-only properties if the caller should not be able to change the value of the property.</span></span>

 <span data-ttu-id="d194e-107">Tenha em mente que, se o tipo da propriedade for um tipo de referência mutável, o valor da propriedade poderá ser alterado mesmo se a propriedade for somente obtenção.</span><span class="sxs-lookup"><span data-stu-id="d194e-107">Keep in mind that if the type of the property is a mutable reference type, the property value can be changed even if the property is get-only.</span></span>

 <span data-ttu-id="d194e-108">o ❌ não fornece propriedades de somente definição ou propriedades com o setter com acessibilidade mais ampla do que o getter.</span><span class="sxs-lookup"><span data-stu-id="d194e-108">❌ DO NOT provide set-only properties or properties with the setter having broader accessibility than the getter.</span></span>

 <span data-ttu-id="d194e-109">Por exemplo, não use Propriedades com um setter público e um getter protegido.</span><span class="sxs-lookup"><span data-stu-id="d194e-109">For example, do not use properties with a public setter and a protected getter.</span></span>

 <span data-ttu-id="d194e-110">Se o getter da propriedade não puder ser fornecido, implemente a funcionalidade como um método em vez disso.</span><span class="sxs-lookup"><span data-stu-id="d194e-110">If the property getter cannot be provided, implement the functionality as a method instead.</span></span> <span data-ttu-id="d194e-111">Considere iniciar o nome do método com `Set` e siga com o que você teria chamado de propriedade.</span><span class="sxs-lookup"><span data-stu-id="d194e-111">Consider starting the method name with `Set` and follow with what you would have named the property.</span></span> <span data-ttu-id="d194e-112">Por exemplo, <xref:System.AppDomain> tem um método chamado `SetCachePath` em vez de ter uma propriedade Set-only chamada `CachePath`.</span><span class="sxs-lookup"><span data-stu-id="d194e-112">For example, <xref:System.AppDomain> has a method called `SetCachePath` instead of having a set-only property called `CachePath`.</span></span>

 <span data-ttu-id="d194e-113">✔️ fornecem valores padrão razoáveis para todas as propriedades, garantindo que os padrões não resultem em uma brecha de segurança ou em um código extremamente ineficiente.</span><span class="sxs-lookup"><span data-stu-id="d194e-113">✔️ DO provide sensible default values for all properties, ensuring that the defaults do not result in a security hole or terribly inefficient code.</span></span>

 <span data-ttu-id="d194e-114">✔️ permitir que as propriedades sejam definidas em qualquer ordem, mesmo se isso resultar em um estado temporário inválido do objeto.</span><span class="sxs-lookup"><span data-stu-id="d194e-114">✔️ DO allow properties to be set in any order even if this results in a temporary invalid state of the object.</span></span>

 <span data-ttu-id="d194e-115">É comum que duas ou mais propriedades sejam interrelacionadas a um ponto em que alguns valores de uma propriedade possam ser inválidos, dado os valores de outras propriedades no mesmo objeto.</span><span class="sxs-lookup"><span data-stu-id="d194e-115">It is common for two or more properties to be interrelated to a point where some values of one property might be invalid given the values of other properties on the same object.</span></span> <span data-ttu-id="d194e-116">Nesses casos, as exceções resultantes do estado inválido devem ser adiadas até que as propriedades inter-relacionadas sejam realmente usadas juntas pelo objeto.</span><span class="sxs-lookup"><span data-stu-id="d194e-116">In such cases, exceptions resulting from the invalid state should be postponed until the interrelated properties are actually used together by the object.</span></span>

 <span data-ttu-id="d194e-117">✔️ preservar o valor anterior se um setter de propriedade lançar uma exceção.</span><span class="sxs-lookup"><span data-stu-id="d194e-117">✔️ DO preserve the previous value if a property setter throws an exception.</span></span>

 <span data-ttu-id="d194e-118">❌ evitar lançar exceções de getters de propriedade.</span><span class="sxs-lookup"><span data-stu-id="d194e-118">❌ AVOID throwing exceptions from property getters.</span></span>

 <span data-ttu-id="d194e-119">Os getters de propriedade devem ser operações simples e não devem ter nenhuma condição.</span><span class="sxs-lookup"><span data-stu-id="d194e-119">Property getters should be simple operations and should not have any preconditions.</span></span> <span data-ttu-id="d194e-120">Se um getter puder gerar uma exceção, ele provavelmente deve ser reprojetado para ser um método.</span><span class="sxs-lookup"><span data-stu-id="d194e-120">If a getter can throw an exception, it should probably be redesigned to be a method.</span></span> <span data-ttu-id="d194e-121">Observe que essa regra não se aplica aos indexadores, onde podemos esperar exceções como resultado da validação dos argumentos.</span><span class="sxs-lookup"><span data-stu-id="d194e-121">Notice that this rule does not apply to indexers, where we do expect exceptions as a result of validating the arguments.</span></span>

### <a name="indexed-property-design"></a><span data-ttu-id="d194e-122">Design de propriedade indexada</span><span class="sxs-lookup"><span data-stu-id="d194e-122">Indexed Property Design</span></span>
 <span data-ttu-id="d194e-123">Uma propriedade indexada é uma propriedade especial que pode ter parâmetros e pode ser chamada com sintaxe especial semelhante à indexação de matriz.</span><span class="sxs-lookup"><span data-stu-id="d194e-123">An indexed property is a special property that can have parameters and can be called with special syntax similar to array indexing.</span></span>

 <span data-ttu-id="d194e-124">As propriedades indexadas são conhecidas como indexadores.</span><span class="sxs-lookup"><span data-stu-id="d194e-124">Indexed properties are commonly referred to as indexers.</span></span> <span data-ttu-id="d194e-125">Os indexadores devem ser usados somente em APIs que fornecem acesso a itens em uma coleção lógica.</span><span class="sxs-lookup"><span data-stu-id="d194e-125">Indexers should be used only in APIs that provide access to items in a logical collection.</span></span> <span data-ttu-id="d194e-126">Por exemplo, uma string é uma coleção de caracteres e o indexador em <xref:System.String?displayProperty=nameWithType> foi adicionado para acessar seus caracteres.</span><span class="sxs-lookup"><span data-stu-id="d194e-126">For example, a string is a collection of characters, and the indexer on <xref:System.String?displayProperty=nameWithType> was added to access its characters.</span></span>

 <span data-ttu-id="d194e-127">✔️ Considere o uso de indexadores para fornecer acesso aos dados armazenados em uma matriz interna.</span><span class="sxs-lookup"><span data-stu-id="d194e-127">✔️ CONSIDER using indexers to provide access to data stored in an internal array.</span></span>

 <span data-ttu-id="d194e-128">✔️ Considere fornecer indexadores em tipos que representam coleções de itens.</span><span class="sxs-lookup"><span data-stu-id="d194e-128">✔️ CONSIDER providing indexers on types representing collections of items.</span></span>

 <span data-ttu-id="d194e-129">❌ evitar o uso de propriedades indexadas com mais de um parâmetro.</span><span class="sxs-lookup"><span data-stu-id="d194e-129">❌ AVOID using indexed properties with more than one parameter.</span></span>

 <span data-ttu-id="d194e-130">Se o design exigir vários parâmetros, reconsidere se a propriedade representa realmente um acessador para uma coleção lógica.</span><span class="sxs-lookup"><span data-stu-id="d194e-130">If the design requires multiple parameters, reconsider whether the property really represents an accessor to a logical collection.</span></span> <span data-ttu-id="d194e-131">Se não tiver, use métodos em vez disso.</span><span class="sxs-lookup"><span data-stu-id="d194e-131">If it does not, use methods instead.</span></span> <span data-ttu-id="d194e-132">Considere iniciar o nome do método com `Get` ou `Set`.</span><span class="sxs-lookup"><span data-stu-id="d194e-132">Consider starting the method name with `Get` or `Set`.</span></span>

 <span data-ttu-id="d194e-133">❌ evitar indexadores com tipos de parâmetro diferentes de <xref:System.Int32?displayProperty=nameWithType>, <xref:System.Int64?displayProperty=nameWithType>, <xref:System.String?displayProperty=nameWithType>, <xref:System.Object?displayProperty=nameWithType>ou uma enumeração.</span><span class="sxs-lookup"><span data-stu-id="d194e-133">❌ AVOID indexers with parameter types other than <xref:System.Int32?displayProperty=nameWithType>, <xref:System.Int64?displayProperty=nameWithType>, <xref:System.String?displayProperty=nameWithType>, <xref:System.Object?displayProperty=nameWithType>, or an enum.</span></span>

 <span data-ttu-id="d194e-134">Se o design exigir outros tipos de parâmetros, reavalie fortemente se a API representa realmente um acessador para uma coleção lógica.</span><span class="sxs-lookup"><span data-stu-id="d194e-134">If the design requires other types of parameters, strongly reevaluate whether the API really represents an accessor to a logical collection.</span></span> <span data-ttu-id="d194e-135">Se não tiver, use um método.</span><span class="sxs-lookup"><span data-stu-id="d194e-135">If it does not, use a method.</span></span> <span data-ttu-id="d194e-136">Considere iniciar o nome do método com `Get` ou `Set`.</span><span class="sxs-lookup"><span data-stu-id="d194e-136">Consider starting the method name with `Get` or `Set`.</span></span>

 <span data-ttu-id="d194e-137">✔️ Use o nome `Item` para propriedades indexadas, a menos que haja um nome obviamente melhor (por exemplo, consulte a propriedade <xref:System.String.Chars%2A> em `System.String`).</span><span class="sxs-lookup"><span data-stu-id="d194e-137">✔️ DO use the name `Item` for indexed properties unless there is an obviously better name (e.g., see the <xref:System.String.Chars%2A> property on `System.String`).</span></span>

 <span data-ttu-id="d194e-138">No C#, os indexadores são por item nomeado padrão.</span><span class="sxs-lookup"><span data-stu-id="d194e-138">In C#, indexers are by default named Item.</span></span> <span data-ttu-id="d194e-139">O <xref:System.Runtime.CompilerServices.IndexerNameAttribute> pode ser usado para personalizar esse nome.</span><span class="sxs-lookup"><span data-stu-id="d194e-139">The <xref:System.Runtime.CompilerServices.IndexerNameAttribute> can be used to customize this name.</span></span>

 <span data-ttu-id="d194e-140">❌ não fornecem um indexador e métodos que são semanticamente equivalentes.</span><span class="sxs-lookup"><span data-stu-id="d194e-140">❌ DO NOT provide both an indexer and methods that are semantically equivalent.</span></span>

 <span data-ttu-id="d194e-141">❌ não fornecem mais de uma família de indexadores sobrecarregados em um tipo.</span><span class="sxs-lookup"><span data-stu-id="d194e-141">❌ DO NOT provide more than one family of overloaded indexers in one type.</span></span>

 <span data-ttu-id="d194e-142">Isso é imposto pelo C# compilador.</span><span class="sxs-lookup"><span data-stu-id="d194e-142">This is enforced by the C# compiler.</span></span>

 <span data-ttu-id="d194e-143">❌ não usar propriedades indexadas não padrão.</span><span class="sxs-lookup"><span data-stu-id="d194e-143">❌ DO NOT use nondefault indexed properties.</span></span>

 <span data-ttu-id="d194e-144">Isso é imposto pelo C# compilador.</span><span class="sxs-lookup"><span data-stu-id="d194e-144">This is enforced by the C# compiler.</span></span>

### <a name="property-change-notification-events"></a><span data-ttu-id="d194e-145">Eventos de notificação de alteração de propriedade</span><span class="sxs-lookup"><span data-stu-id="d194e-145">Property Change Notification Events</span></span>
 <span data-ttu-id="d194e-146">Às vezes, é útil fornecer um evento notificando o usuário sobre as alterações em um valor de propriedade.</span><span class="sxs-lookup"><span data-stu-id="d194e-146">Sometimes it is useful to provide an event notifying the user of changes in a property value.</span></span> <span data-ttu-id="d194e-147">Por exemplo, `System.Windows.Forms.Control` gera um evento de `TextChanged` depois que o valor de sua propriedade `Text` é alterado.</span><span class="sxs-lookup"><span data-stu-id="d194e-147">For example, `System.Windows.Forms.Control` raises a `TextChanged` event after the value of its `Text` property has changed.</span></span>

 <span data-ttu-id="d194e-148">✔️ Considere gerar eventos de notificação de alteração quando os valores de propriedade em APIs de alto nível (geralmente componentes do designer) forem modificados.</span><span class="sxs-lookup"><span data-stu-id="d194e-148">✔️ CONSIDER raising change notification events when property values in high-level APIs (usually designer components) are modified.</span></span>

 <span data-ttu-id="d194e-149">Se houver um bom cenário para que um usuário saiba quando uma propriedade de um objeto está sendo alterada, o objeto deverá gerar um evento de notificação de alteração para a propriedade.</span><span class="sxs-lookup"><span data-stu-id="d194e-149">If there is a good scenario for a user to know when a property of an object is changing, the object should raise a change notification event for the property.</span></span>

 <span data-ttu-id="d194e-150">No entanto, é improvável que haja sobrecarga para gerar tais eventos para APIs de nível baixo, como tipos de base ou coleções.</span><span class="sxs-lookup"><span data-stu-id="d194e-150">However, it is unlikely to be worth the overhead to raise such events for low-level APIs such as base types or collections.</span></span> <span data-ttu-id="d194e-151">Por exemplo, <xref:System.Collections.Generic.List%601> não geraria esses eventos quando um novo item fosse adicionado à lista e a propriedade `Count` for alterada.</span><span class="sxs-lookup"><span data-stu-id="d194e-151">For example, <xref:System.Collections.Generic.List%601> would not raise such events when a new item is added to the list and the `Count` property changes.</span></span>

 <span data-ttu-id="d194e-152">✔️ Considere gerar eventos de notificação de alteração quando o valor de uma propriedade for alterado por meio de forças externas.</span><span class="sxs-lookup"><span data-stu-id="d194e-152">✔️ CONSIDER raising change notification events when the value of a property changes via external forces.</span></span>

 <span data-ttu-id="d194e-153">Se um valor de propriedade for alterado por meio de alguma força externa (de uma maneira diferente de chamar métodos no objeto), os eventos de geração indicam ao desenvolvedor que o valor está mudando e foi alterado.</span><span class="sxs-lookup"><span data-stu-id="d194e-153">If a property value changes via some external force (in a way other than by calling methods on the object), raise events indicate to the developer that the value is changing and has changed.</span></span> <span data-ttu-id="d194e-154">Um bom exemplo é a propriedade `Text` de um controle de caixa de texto.</span><span class="sxs-lookup"><span data-stu-id="d194e-154">A good example is the `Text` property of a text box control.</span></span> <span data-ttu-id="d194e-155">Quando o usuário digita texto em um `TextBox`, o valor da propriedade é alterado automaticamente.</span><span class="sxs-lookup"><span data-stu-id="d194e-155">When the user types text in a `TextBox`, the property value automatically changes.</span></span>
