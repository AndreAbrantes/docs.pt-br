# <a name="operator-overloads"></a><span data-ttu-id="9c8eb-102">Sobrecargas de operador</span><span class="sxs-lookup"><span data-stu-id="9c8eb-102">Operator Overloads</span></span>

<span data-ttu-id="9c8eb-103">As sobrecargas de operador permitem que os tipos de estrutura apareçam como se fossem primitivos de linguagem internos.</span><span class="sxs-lookup"><span data-stu-id="9c8eb-103">Operator overloads allow framework types to appear as if they were built-in language primitives.</span></span>

 <span data-ttu-id="9c8eb-104">Embora seja permitido e útil em algumas situações, as sobrecargas de operador devem ser usadas com cuidado.</span><span class="sxs-lookup"><span data-stu-id="9c8eb-104">Although allowed and useful in some situations, operator overloads should be used cautiously.</span></span> <span data-ttu-id="9c8eb-105">Há muitos casos em que o sobrecarga de operador foi feito com abuso, como quando designers de estrutura começaram a usar operadores para operações que devem ser métodos simples.</span><span class="sxs-lookup"><span data-stu-id="9c8eb-105">There are many cases in which operator overloading has been abused, such as when framework designers started to use operators for operations that should be simple methods.</span></span> <span data-ttu-id="9c8eb-106">As diretrizes a seguir devem ajudá-lo a decidir quando e como usar a sobrecarga de operador.</span><span class="sxs-lookup"><span data-stu-id="9c8eb-106">The following guidelines should help you decide when and how to use operator overloading.</span></span>

 <span data-ttu-id="9c8eb-107">❌ Evite definir sobrecargas de operador, exceto em tipos que devem se sentir como tipos primitivos (internos).</span><span class="sxs-lookup"><span data-stu-id="9c8eb-107">❌ AVOID defining operator overloads, except in types that should feel like primitive (built-in) types.</span></span>

 <span data-ttu-id="9c8eb-108">✔️ Considere definir sobrecargas de operador em um tipo que deve parecer um tipo primitivo.</span><span class="sxs-lookup"><span data-stu-id="9c8eb-108">✔️ CONSIDER defining operator overloads in a type that should feel like a primitive type.</span></span>

 <span data-ttu-id="9c8eb-109">Por exemplo, <xref:System.String?displayProperty=nameWithType> tem `operator==` e `operator!=` definido.</span><span class="sxs-lookup"><span data-stu-id="9c8eb-109">For example, <xref:System.String?displayProperty=nameWithType> has `operator==` and `operator!=` defined.</span></span>

 <span data-ttu-id="9c8eb-110">✔️ definem sobrecargas de operador em structs que representam números (como <xref:System.Decimal?displayProperty=nameWithType> ).</span><span class="sxs-lookup"><span data-stu-id="9c8eb-110">✔️ DO define operator overloads in structs that represent numbers (such as <xref:System.Decimal?displayProperty=nameWithType>).</span></span>

 <span data-ttu-id="9c8eb-111">❌ Não seja graciosos ao definir sobrecargas de operador.</span><span class="sxs-lookup"><span data-stu-id="9c8eb-111">❌ DO NOT be cute when defining operator overloads.</span></span>

 <span data-ttu-id="9c8eb-112">A sobrecarga de operador é útil em casos nos quais é imediatamente óbvio qual será o resultado da operação.</span><span class="sxs-lookup"><span data-stu-id="9c8eb-112">Operator overloading is useful in cases in which it is immediately obvious what the result of the operation will be.</span></span> <span data-ttu-id="9c8eb-113">Por exemplo, faz sentido ser capaz de subtrair um <xref:System.DateTime> de outro `DateTime` e obter um <xref:System.TimeSpan> .</span><span class="sxs-lookup"><span data-stu-id="9c8eb-113">For example, it makes sense to be able to subtract one <xref:System.DateTime> from another `DateTime` and get a <xref:System.TimeSpan>.</span></span> <span data-ttu-id="9c8eb-114">No entanto, não é apropriado usar o operador lógico Union para unir duas consultas de banco de dados ou usar o operador SHIFT para gravar em um fluxo.</span><span class="sxs-lookup"><span data-stu-id="9c8eb-114">However, it is not appropriate to use the logical union operator to union two database queries, or to use the shift operator to write to a stream.</span></span>

 <span data-ttu-id="9c8eb-115">❌ Não forneça sobrecargas de operador, a menos que pelo menos um dos operandos seja do tipo que define a sobrecarga.</span><span class="sxs-lookup"><span data-stu-id="9c8eb-115">❌ DO NOT provide operator overloads unless at least one of the operands is of the type defining the overload.</span></span>

 <span data-ttu-id="9c8eb-116">✔️ os operadores de sobrecarga de maneira simétrica.</span><span class="sxs-lookup"><span data-stu-id="9c8eb-116">✔️ DO overload operators in a symmetric fashion.</span></span>

 <span data-ttu-id="9c8eb-117">Por exemplo, se você sobrecarregar o `operator==` , também deverá sobrecarregar o `operator!=` .</span><span class="sxs-lookup"><span data-stu-id="9c8eb-117">For example, if you overload the `operator==`, you should also overload the `operator!=`.</span></span> <span data-ttu-id="9c8eb-118">Da mesma forma, se você sobrecarregar o `operator<` , também deverá sobrecarregar o `operator>` e assim por diante.</span><span class="sxs-lookup"><span data-stu-id="9c8eb-118">Similarly, if you overload the `operator<`, you should also overload the `operator>`, and so on.</span></span>

 <span data-ttu-id="9c8eb-119">✔️ Considere fornecer métodos com nomes amigáveis que correspondam a cada operador sobrecarregado.</span><span class="sxs-lookup"><span data-stu-id="9c8eb-119">✔️ CONSIDER providing methods with friendly names that correspond to each overloaded operator.</span></span>

 <span data-ttu-id="9c8eb-120">Muitas linguagens não dão suporte à sobrecarga de operador.</span><span class="sxs-lookup"><span data-stu-id="9c8eb-120">Many languages do not support operator overloading.</span></span> <span data-ttu-id="9c8eb-121">Por esse motivo, é recomendável que os tipos que sobrecarregam operadores incluam um método secundário com um nome específico de domínio apropriado que forneça funcionalidade equivalente.</span><span class="sxs-lookup"><span data-stu-id="9c8eb-121">For this reason, it is recommended that types that overload operators include a secondary method with an appropriate domain-specific name that provides equivalent functionality.</span></span>

### <a name="overloading-operator-"></a><span data-ttu-id="9c8eb-126">Operador de sobrecarga = =</span><span class="sxs-lookup"><span data-stu-id="9c8eb-126">Overloading Operator ==</span></span>

### <a name="conversion-operators"></a><span data-ttu-id="9c8eb-129">Operadores de conversão</span><span class="sxs-lookup"><span data-stu-id="9c8eb-129">Conversion Operators</span></span>

 <span data-ttu-id="9c8eb-130">Os operadores de conversão são operadores unários que permitem a conversão de um tipo para outro.</span><span class="sxs-lookup"><span data-stu-id="9c8eb-130">Conversion operators are unary operators that allow conversion from one type to another.</span></span> <span data-ttu-id="9c8eb-131">Os operadores devem ser definidos como membros estáticos no operando ou no tipo de retorno.</span><span class="sxs-lookup"><span data-stu-id="9c8eb-131">The operators must be defined as static members on either the operand or the return type.</span></span> <span data-ttu-id="9c8eb-132">Há dois tipos de operadores de conversão: implícito e explícito.</span><span class="sxs-lookup"><span data-stu-id="9c8eb-132">There are two types of conversion operators: implicit and explicit.</span></span>

 <span data-ttu-id="9c8eb-133">❌ Não forneça um operador de conversão se tal conversão não for claramente esperada pelos usuários finais.</span><span class="sxs-lookup"><span data-stu-id="9c8eb-133">❌ DO NOT provide a conversion operator if such conversion is not clearly expected by the end users.</span></span>

 <span data-ttu-id="9c8eb-134">❌ Não defina operadores de conversão fora do domínio de um tipo.</span><span class="sxs-lookup"><span data-stu-id="9c8eb-134">❌ DO NOT define conversion operators outside of a type’s domain.</span></span>

 <span data-ttu-id="9c8eb-135">Por exemplo, <xref:System.Int32> , <xref:System.Double> e <xref:System.Decimal> são todos os tipos numéricos, enquanto <xref:System.DateTime> não é.</span><span class="sxs-lookup"><span data-stu-id="9c8eb-135">For example, <xref:System.Int32>, <xref:System.Double>, and <xref:System.Decimal> are all numeric types, whereas <xref:System.DateTime> is not.</span></span> <span data-ttu-id="9c8eb-136">Portanto, não deve haver nenhum operador de conversão para converter um `Double(long)` para um `DateTime` .</span><span class="sxs-lookup"><span data-stu-id="9c8eb-136">Therefore, there should be no conversion operator to convert a `Double(long)` to a `DateTime`.</span></span> <span data-ttu-id="9c8eb-137">Um construtor é preferencial nesse caso.</span><span class="sxs-lookup"><span data-stu-id="9c8eb-137">A constructor is preferred in such a case.</span></span>

 <span data-ttu-id="9c8eb-138">❌ Não forneça um operador de conversão implícita se a conversão tiver potencialmente perda.</span><span class="sxs-lookup"><span data-stu-id="9c8eb-138">❌ DO NOT provide an implicit conversion operator if the conversion is potentially lossy.</span></span>

 <span data-ttu-id="9c8eb-139">Por exemplo, não deve haver uma conversão implícita de `Double` para `Int32` porque `Double` o tem um intervalo maior do que `Int32` .</span><span class="sxs-lookup"><span data-stu-id="9c8eb-139">For example, there should not be an implicit conversion from `Double` to `Int32` because `Double` has a wider range than `Int32`.</span></span> <span data-ttu-id="9c8eb-140">Um operador de conversão explícita pode ser fornecido mesmo que a conversão tenha potencialmente perda.</span><span class="sxs-lookup"><span data-stu-id="9c8eb-140">An explicit conversion operator can be provided even if the conversion is potentially lossy.</span></span>

 <span data-ttu-id="9c8eb-141">❌ Não lance exceções de conversões implícitas.</span><span class="sxs-lookup"><span data-stu-id="9c8eb-141">❌ DO NOT throw exceptions from implicit casts.</span></span>

 <span data-ttu-id="9c8eb-142">É muito difícil para os usuários finais entenderem o que está acontecendo, pois eles podem não estar cientes de que uma conversão está ocorrendo.</span><span class="sxs-lookup"><span data-stu-id="9c8eb-142">It is very difficult for end users to understand what is happening, because they might not be aware that a conversion is taking place.</span></span>

 <span data-ttu-id="9c8eb-143">✔️ gerar <xref:System.InvalidCastException?displayProperty=nameWithType> se uma chamada para um operador cast resultar em uma conversão com perdas e o contrato do operador não permitir conversões com perdas.</span><span class="sxs-lookup"><span data-stu-id="9c8eb-143">✔️ DO throw <xref:System.InvalidCastException?displayProperty=nameWithType> if a call to a cast operator results in a lossy conversion and the contract of the operator does not allow lossy conversions.</span></span>
