# <a name="base-classes-for-implementing-abstractions"></a><span data-ttu-id="feb4c-102">Classes base para implementar abstrações</span><span class="sxs-lookup"><span data-stu-id="feb4c-102">Base Classes for Implementing Abstractions</span></span>
<span data-ttu-id="feb4c-103">Estritamente falando, uma classe torna-se uma classe base quando outra classe é derivada.</span><span class="sxs-lookup"><span data-stu-id="feb4c-103">Strictly speaking, a class becomes a base class when another class is derived from it.</span></span> <span data-ttu-id="feb4c-104">Para fins desta seção, no entanto, uma classe base é uma classe projetada principalmente para fornecer uma abstração comum ou para outras classes de reutilizar alguns implementação padrão embora herança.</span><span class="sxs-lookup"><span data-stu-id="feb4c-104">For the purpose of this section, however, a base class is a class designed mainly to provide a common abstraction or for other classes to reuse some default implementation though inheritance.</span></span> <span data-ttu-id="feb4c-105">Classes base geralmente ficam no meio de hierarquias de herança entre uma abstração na raiz de uma hierarquia e várias implementações personalizadas na parte inferior.</span><span class="sxs-lookup"><span data-stu-id="feb4c-105">Base classes usually sit in the middle of inheritance hierarchies, between an abstraction at the root of a hierarchy and several custom implementations at the bottom.</span></span>  
  
 <span data-ttu-id="feb4c-106">Eles servem como auxiliares de implementação para implementar abstrações.</span><span class="sxs-lookup"><span data-stu-id="feb4c-106">They serve as implementation helpers for implementing abstractions.</span></span> <span data-ttu-id="feb4c-107">Por exemplo, uma das abstrações do Framework para conjuntos ordenados de itens é o <xref:System.Collections.Generic.IList%601> interface.</span><span class="sxs-lookup"><span data-stu-id="feb4c-107">For example, one of the Framework’s abstractions for ordered collections of items is the <xref:System.Collections.Generic.IList%601> interface.</span></span> <span data-ttu-id="feb4c-108">Implementando <xref:System.Collections.Generic.IList%601> não é trivial, e, portanto, o Framework fornece várias classes base, como <xref:System.Collections.ObjectModel.Collection%601> e <xref:System.Collections.ObjectModel.KeyedCollection%602>, que servirá como auxiliares para coleções personalizadas de implementação.</span><span class="sxs-lookup"><span data-stu-id="feb4c-108">Implementing <xref:System.Collections.Generic.IList%601> is not trivial, and therefore the Framework provides several base classes, such as <xref:System.Collections.ObjectModel.Collection%601> and <xref:System.Collections.ObjectModel.KeyedCollection%602>, which serve as helpers for implementing custom collections.</span></span>  
  
 <span data-ttu-id="feb4c-109">Classes base geralmente não são adequados para servir como abstrações por si só, porque eles tendem a contêm muita implementação.</span><span class="sxs-lookup"><span data-stu-id="feb4c-109">Base classes are usually not suited to serve as abstractions by themselves, because they tend to contain too much implementation.</span></span> <span data-ttu-id="feb4c-110">Por exemplo, o `Collection<T>` classe base contém muitas relacionado ao fato de que ele implementa não genérica de implementação `IList` interface (para integrar melhor coleções não genéricas) e ao fato de que é uma coleção de itens armazenados em memória em um de seus campos.</span><span class="sxs-lookup"><span data-stu-id="feb4c-110">For example, the `Collection<T>` base class contains lots of implementation related to the fact that it implements the nongeneric `IList` interface (to integrate better with nongeneric collections) and to the fact that it is a collection of items stored in memory in one of its fields.</span></span>  
  
 <span data-ttu-id="feb4c-111">Como discutido anteriormente, as classes base podem fornecer ajuda inestimável para usuários que precisam implementar abstrações, mas ao mesmo tempo podem ser um risco significativo.</span><span class="sxs-lookup"><span data-stu-id="feb4c-111">As previously discussed, base classes can provide invaluable help for users who need to implement abstractions, but at the same time they can be a significant liability.</span></span> <span data-ttu-id="feb4c-112">Eles Adicionar área da superfície e aumentam a profundidade das hierarquias de herança e portanto conceitualmente complicam a estrutura.</span><span class="sxs-lookup"><span data-stu-id="feb4c-112">They add surface area and increase the depth of inheritance hierarchies and so conceptually complicate the framework.</span></span> <span data-ttu-id="feb4c-113">Portanto, as classes base devem ser usados somente se eles fornecem valor significativo para os usuários do framework.</span><span class="sxs-lookup"><span data-stu-id="feb4c-113">Therefore, base classes should be used only if they provide significant value to the users of the framework.</span></span> <span data-ttu-id="feb4c-114">Eles devem ser evitados se eles fornecem valor apenas para os implementadores do framework, no qual caso delegação para uma implementação interna em vez de herança de uma classe base deve ser considerada altamente.</span><span class="sxs-lookup"><span data-stu-id="feb4c-114">They should be avoided if they provide value only to the implementers of the framework, in which case delegation to an internal implementation instead of inheritance from a base class should be strongly considered.</span></span>  
  
 <span data-ttu-id="feb4c-115">**✓ CONSIDER** abstrato as classes base tornando mesmo se eles não contêm membros abstratos.</span><span class="sxs-lookup"><span data-stu-id="feb4c-115">**✓ CONSIDER** making base classes abstract even if they don’t contain any abstract members.</span></span> <span data-ttu-id="feb4c-116">Isso claramente se comunica com os usuários que a classe destina-se exclusivamente a ser herdada.</span><span class="sxs-lookup"><span data-stu-id="feb4c-116">This clearly communicates to the users that the class is designed solely to be inherited from.</span></span>  
  
 <span data-ttu-id="feb4c-117">**✓ CONSIDER** colocando classes base em um namespace separado dos tipos de cenário principal.</span><span class="sxs-lookup"><span data-stu-id="feb4c-117">**✓ CONSIDER** placing base classes in a separate namespace from the mainline scenario types.</span></span> <span data-ttu-id="feb4c-118">Por definição, as classes base destinam-se para cenários de extensibilidade avançada e, portanto, não são interessantes para a maioria dos usuários.</span><span class="sxs-lookup"><span data-stu-id="feb4c-118">By definition, base classes are intended for advanced extensibility scenarios and therefore are not interesting to the majority of users.</span></span>  
  
 <span data-ttu-id="feb4c-119">**X AVOID** nomeando classes base com um sufixo "Base" se a classe é destinada para uso em APIs públicas.</span><span class="sxs-lookup"><span data-stu-id="feb4c-119">**X AVOID** naming base classes with a "Base" suffix if the class is intended for use in public APIs.</span></span>
