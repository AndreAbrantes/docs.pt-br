## <a name="word-choice"></a><span data-ttu-id="3d487-104">Opção de palavra</span><span class="sxs-lookup"><span data-stu-id="3d487-104">Word Choice</span></span>  
 <span data-ttu-id="3d487-105">**✓ DO** Escolha nomes de identificador facilmente legível.</span><span class="sxs-lookup"><span data-stu-id="3d487-105">**✓ DO** choose easily readable identifier names.</span></span>  
  
 <span data-ttu-id="3d487-106">Por exemplo, uma propriedade chamada `HorizontalAlignment` é mais legível para o inglês do que `AlignmentHorizontal`.</span><span class="sxs-lookup"><span data-stu-id="3d487-106">For example, a property named `HorizontalAlignment` is more English-readable than `AlignmentHorizontal`.</span></span>  
  
 <span data-ttu-id="3d487-107">**✓ DO** favorecer legibilidade por questão de brevidade.</span><span class="sxs-lookup"><span data-stu-id="3d487-107">**✓ DO** favor readability over brevity.</span></span>  
  
 <span data-ttu-id="3d487-108">O nome da propriedade `CanScrollHorizontally` é melhor do que `ScrollableX` (uma referência obscura para o eixo x).</span><span class="sxs-lookup"><span data-stu-id="3d487-108">The property name `CanScrollHorizontally` is better than `ScrollableX` (an obscure reference to the X-axis).</span></span>  
  
 <span data-ttu-id="3d487-109">**X DO NOT** usar outros caracteres não alfanuméricos, hífens ou sublinhados.</span><span class="sxs-lookup"><span data-stu-id="3d487-109">**X DO NOT** use underscores, hyphens, or any other nonalphanumeric characters.</span></span>  
  
 <span data-ttu-id="3d487-110">**X DO NOT** usar notação húngara.</span><span class="sxs-lookup"><span data-stu-id="3d487-110">**X DO NOT** use Hungarian notation.</span></span>  
  
 <span data-ttu-id="3d487-111">**X AVOID** usando identificadores que entrem em conflito com palavras-chave de amplamente usados linguagens de programação.</span><span class="sxs-lookup"><span data-stu-id="3d487-111">**X AVOID** using identifiers that conflict with keywords of widely used programming languages.</span></span>  
  
 <span data-ttu-id="3d487-112">De acordo com a regra 4 do Common Language Specification (CLS), todas as linguagens compatíveis com devem fornecer um mecanismo que permite o acesso a itens nomeados que usar uma palavra-chave de idioma como um identificador.</span><span class="sxs-lookup"><span data-stu-id="3d487-112">According to Rule 4 of the Common Language Specification (CLS), all compliant languages must provide a mechanism that allows access to named items that use a keyword of that language as an identifier.</span></span> <span data-ttu-id="3d487-113">C#, por exemplo, usa o sinal como um mecanismo de escape, neste caso, @.</span><span class="sxs-lookup"><span data-stu-id="3d487-113">C#, for example, uses the @ sign as an escape mechanism in this case.</span></span> <span data-ttu-id="3d487-114">No entanto, ainda é uma boa ideia evitar palavras-chave comuns, porque ele é muito mais difícil de usar um método com a sequência de escape de um sem ele.</span><span class="sxs-lookup"><span data-stu-id="3d487-114">However, it is still a good idea to avoid common keywords because it is much more difficult to use a method with the escape sequence than one without it.</span></span>

 <span data-ttu-id="3d487-117">Por exemplo, use `GetWindow` em vez de `GetWin`.</span><span class="sxs-lookup"><span data-stu-id="3d487-117">For example, use `GetWindow` rather than `GetWin`.</span></span>  
  
 <span data-ttu-id="3d487-118">**X DO NOT** usar os acrônimos que não são amplamente aceitas e até mesmo se elas forem, somente quando necessário.</span><span class="sxs-lookup"><span data-stu-id="3d487-118">**X DO NOT** use any acronyms that are not widely accepted, and even if they are, only when necessary.</span></span>

 <span data-ttu-id="3d487-188">Isso ajuda a realçar a relação entre as APIs.</span><span class="sxs-lookup"><span data-stu-id="3d487-188">This helps to highlight the relationship between the APIs.</span></span>
