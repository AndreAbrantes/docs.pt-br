# <a name="nested-types"></a><span data-ttu-id="1437a-102">Tipos aninhados</span><span class="sxs-lookup"><span data-stu-id="1437a-102">Nested Types</span></span>
<span data-ttu-id="1437a-103">Um tipo aninhado é um tipo definido dentro do escopo de outro tipo, que é chamado de tipo de delimitador.</span><span class="sxs-lookup"><span data-stu-id="1437a-103">A nested type is a type defined within the scope of another type, which is called the enclosing type.</span></span> <span data-ttu-id="1437a-104">Um tipo aninhado tem acesso a todos os membros do seu tipo delimitador.</span><span class="sxs-lookup"><span data-stu-id="1437a-104">A nested type has access to all members of its enclosing type.</span></span> <span data-ttu-id="1437a-105">Por exemplo, ele tem acesso a campos privados, definidos no tipo delimitador e protegidos campos definidos em todos os ascendentes do tipo delimitador.</span><span class="sxs-lookup"><span data-stu-id="1437a-105">For example, it has access to private fields defined in the enclosing type and to protected fields defined in all ascendants of the enclosing type.</span></span>  
  
 <span data-ttu-id="1437a-106">Em geral, os tipos aninhados devem ser usados com moderação.</span><span class="sxs-lookup"><span data-stu-id="1437a-106">In general, nested types should be used sparingly.</span></span> <span data-ttu-id="1437a-107">Há vários motivos para isso.</span><span class="sxs-lookup"><span data-stu-id="1437a-107">There are several reasons for this.</span></span> <span data-ttu-id="1437a-108">Alguns desenvolvedores não estão totalmente familiarizados com o conceito.</span><span class="sxs-lookup"><span data-stu-id="1437a-108">Some developers are not fully familiar with the concept.</span></span> <span data-ttu-id="1437a-109">Esses desenvolvedores podem, por exemplo, ter problemas com a sintaxe de declaração de variáveis de tipos aninhados.</span><span class="sxs-lookup"><span data-stu-id="1437a-109">These developers might, for example, have problems with the syntax of declaring variables of nested types.</span></span> <span data-ttu-id="1437a-110">Tipos aninhados são também muito estreitamente ligados aos seus tipos delimitadores e como tal, não são adequados para serem tipos de uso geral.</span><span class="sxs-lookup"><span data-stu-id="1437a-110">Nested types are also very tightly coupled with their enclosing types, and as such are not suited to be general-purpose types.</span></span>  
  
 <span data-ttu-id="1437a-111">Tipos aninhados são mais adequados para modelar os detalhes de implementação de seus tipos delimitadores.</span><span class="sxs-lookup"><span data-stu-id="1437a-111">Nested types are best suited for modeling implementation details of their enclosing types.</span></span> <span data-ttu-id="1437a-112">O usuário final deve ter raramente declarar variáveis de um tipo aninhado e quase nunca deve ter que instanciar explicitamente os tipos aninhados.</span><span class="sxs-lookup"><span data-stu-id="1437a-112">The end user should rarely have to declare variables of a nested type and almost never should have to explicitly instantiate nested types.</span></span> <span data-ttu-id="1437a-113">Por exemplo, o enumerador de uma coleção pode ser um tipo aninhado dessa coleção.</span><span class="sxs-lookup"><span data-stu-id="1437a-113">For example, the enumerator of a collection can be a nested type of that collection.</span></span> <span data-ttu-id="1437a-114">Enumeradores normalmente são instanciados por meio de seu tipo delimitador e, como muitas linguagens oferecem suporte a instrução foreach, variáveis de enumerador raramente tem que ser declarado pelo usuário final.</span><span class="sxs-lookup"><span data-stu-id="1437a-114">Enumerators are usually instantiated by their enclosing type, and because many languages support the foreach statement, enumerator variables rarely have to be declared by the end user.</span></span>  
  
 <span data-ttu-id="1437a-115">**✓ DO** usar tipos aninhados quando a relação entre o tipo aninhado e seu tipo externo é tal que a semântica de acessibilidade de membros é desejável.</span><span class="sxs-lookup"><span data-stu-id="1437a-115">**✓ DO** use nested types when the relationship between the nested type and its outer type is such that member-accessibility semantics are desirable.</span></span>  
  
 <span data-ttu-id="1437a-116">**X DO NOT** tipos aninhados públicos em uso como um agrupamento lógico construir; use namespaces para isso.</span><span class="sxs-lookup"><span data-stu-id="1437a-116">**X DO NOT** use public nested types as a logical grouping construct; use namespaces for this.</span></span>  
  
 <span data-ttu-id="1437a-117">**X AVOID** exposto publicamente tipos aninhados.</span><span class="sxs-lookup"><span data-stu-id="1437a-117">**X AVOID** publicly exposed nested types.</span></span> <span data-ttu-id="1437a-118">A única exceção a isso é se as variáveis do tipo aninhado precisam ser declarada somente em cenários raros, como subclasses ou em outros cenários de personalização avançada.</span><span class="sxs-lookup"><span data-stu-id="1437a-118">The only exception to this is if variables of the nested type need to be declared only in rare scenarios such as subclassing or other advanced customization scenarios.</span></span>  
  
 <span data-ttu-id="1437a-119">**X DO NOT** usar tipos aninhados, se o tipo é provavelmente serão mencionados fora do tipo recipiente.</span><span class="sxs-lookup"><span data-stu-id="1437a-119">**X DO NOT** use nested types if the type is likely to be referenced outside of the containing type.</span></span>  
  
 <span data-ttu-id="1437a-120">Por exemplo, uma enumeração passada para um método definido em uma classe não deve ser definida como um tipo aninhado na classe.</span><span class="sxs-lookup"><span data-stu-id="1437a-120">For example, an enum passed to a method defined on a class should not be defined as a nested type in the class.</span></span>  
  
 <span data-ttu-id="1437a-121">**X DO NOT** usar tipos aninhados se eles precisam ser instanciado pelo código do cliente.</span><span class="sxs-lookup"><span data-stu-id="1437a-121">**X DO NOT** use nested types if they need to be instantiated by client code.</span></span>  <span data-ttu-id="1437a-122">Se um tipo tem um construtor público, ele provavelmente não deve ser aninhado.</span><span class="sxs-lookup"><span data-stu-id="1437a-122">If a type has a public constructor, it should probably not be nested.</span></span>  
  
 <span data-ttu-id="1437a-123">Se um tipo pode ser instanciado, parece que indicam o tipo tem um lugar no framework por conta própria (você pode criá-lo, trabalhar com ele e destruí-lo sem nunca, usando o tipo externo) e, portanto, não devem ser aninhados.</span><span class="sxs-lookup"><span data-stu-id="1437a-123">If a type can be instantiated, that seems to indicate the type has a place in the framework on its own (you can create it, work with it, and destroy it without ever using the outer type), and thus should not be nested.</span></span> <span data-ttu-id="1437a-124">Tipos internos não devem ser reutilizados amplamente fora do tipo externo sem qualquer relação qualquer que seja o tipo externo.</span><span class="sxs-lookup"><span data-stu-id="1437a-124">Inner types should not be widely reused outside of the outer type without any relationship whatsoever to the outer type.</span></span>  
  
 <span data-ttu-id="1437a-125">**X DO NOT** definir um tipo aninhado como um membro de uma interface.</span><span class="sxs-lookup"><span data-stu-id="1437a-125">**X DO NOT** define a nested type as a member of an interface.</span></span> <span data-ttu-id="1437a-126">Muitas linguagens não dão suporte a um constructo.</span><span class="sxs-lookup"><span data-stu-id="1437a-126">Many languages do not support such a construct.</span></span>
