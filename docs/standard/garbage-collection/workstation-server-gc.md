## <a name="performance-considerations"></a><span data-ttu-id="48990-119">Considerações sobre o desempenho</span><span class="sxs-lookup"><span data-stu-id="48990-119">Performance considerations</span></span>

### <a name="workstation-gc"></a><span data-ttu-id="48990-120">GC da estação de trabalho</span><span class="sxs-lookup"><span data-stu-id="48990-120">Workstation GC</span></span>

<span data-ttu-id="48990-121">Veja a seguir considerações de desempenho e de threading para a coleta de lixo da estação de trabalho:</span><span class="sxs-lookup"><span data-stu-id="48990-121">The following are threading and performance considerations for workstation garbage collection:</span></span>

- <span data-ttu-id="48990-122">A coleção ocorre no thread do usuário que disparou a coleta de lixo e permanece com a mesma prioridade.</span><span class="sxs-lookup"><span data-stu-id="48990-122">The collection occurs on the user thread that triggered the garbage collection and remains at the same priority.</span></span> <span data-ttu-id="48990-123">Como os threads de usuário normalmente são executados com prioridade normal, o coletor de lixo (que é executado em um thread de prioridade normal) deve disputam competir outros threads por tempo da CPU.</span><span class="sxs-lookup"><span data-stu-id="48990-123">Because user threads typically run at normal priority, the garbage collector (which runs on a normal priority thread) must compete with other threads for CPU time.</span></span> <span data-ttu-id="48990-124">(Os threads que executam código nativo não são suspensos na coleta de lixo do servidor ou da estação de trabalho.)</span><span class="sxs-lookup"><span data-stu-id="48990-124">(Threads that run native code are not suspended on either server or workstation garbage collection.)</span></span>

- <span data-ttu-id="48990-125">A coleta de lixo de estação de trabalho é sempre usada em um computador que tem apenas um processador, independentemente da [definição de configuração](../../core/run-time-config/garbage-collector.md#workstation-vs-server).</span><span class="sxs-lookup"><span data-stu-id="48990-125">Workstation garbage collection is always used on a computer that has only one processor, regardless of the [configuration setting](../../core/run-time-config/garbage-collector.md#workstation-vs-server).</span></span>

### <a name="server-gc"></a><span data-ttu-id="48990-126">GC do servidor</span><span class="sxs-lookup"><span data-stu-id="48990-126">Server GC</span></span>

<span data-ttu-id="48990-127">Veja a seguir considerações de desempenho e de threading para a coleta de lixo de servidor:</span><span class="sxs-lookup"><span data-stu-id="48990-127">The following are threading and performance considerations for server garbage collection:</span></span>

- <span data-ttu-id="48990-128">A coleta ocorre em vários threads dedicados que estão em execução no nível de prioridade `THREAD_PRIORITY_HIGHEST`.</span><span class="sxs-lookup"><span data-stu-id="48990-128">The collection occurs on multiple dedicated threads that are running at `THREAD_PRIORITY_HIGHEST` priority level.</span></span>

- <span data-ttu-id="48990-129">Fornece-se um heap e um thread dedicado para executar a coleta de lixo para cada CPU, e os heaps são coletados ao mesmo tempo.</span><span class="sxs-lookup"><span data-stu-id="48990-129">A heap and a dedicated thread to perform garbage collection are provided for each CPU, and the heaps are collected at the same time.</span></span> <span data-ttu-id="48990-130">Cada heap contém um heap de objeto pequeno e um heap de objeto grande, e todos os heaps podem ser acessados pelo código do usuário.</span><span class="sxs-lookup"><span data-stu-id="48990-130">Each heap contains a small object heap and a large object heap, and all heaps can be accessed by user code.</span></span> <span data-ttu-id="48990-131">Objetos em heaps diferentes podem fazer referência entre si.</span><span class="sxs-lookup"><span data-stu-id="48990-131">Objects on different heaps can refer to each other.</span></span>

- <span data-ttu-id="48990-132">Como vários threads de coleta de lixo funcionam em conjunto, a coleta de lixo de servidor é mais rápida do que a coleta de lixo de estação de trabalho no heap de mesmo tamanho.</span><span class="sxs-lookup"><span data-stu-id="48990-132">Because multiple garbage collection threads work together, server garbage collection is faster than workstation garbage collection on the same size heap.</span></span>

- <span data-ttu-id="48990-133">Geralmente, a coleta de lixo de servidor tem segmentos maiores.</span><span class="sxs-lookup"><span data-stu-id="48990-133">Server garbage collection often has larger size segments.</span></span> <span data-ttu-id="48990-134">No entanto, essa é apenas uma generalização: o tamanho do segmento é específico da implementação e está sujeito a alterações.</span><span class="sxs-lookup"><span data-stu-id="48990-134">However, this is only a generalization: segment size is implementation-specific and is subject to change.</span></span> <span data-ttu-id="48990-135">Não faça suposições sobre o tamanho dos segmentos alocados pelo coletor de lixo ao ajustar seu aplicativo.</span><span class="sxs-lookup"><span data-stu-id="48990-135">Don't make assumptions about the size of segments allocated by the garbage collector when tuning your app.</span></span>

- <span data-ttu-id="48990-136">A coleta de lixo de servidor pode usar bastante recursos.</span><span class="sxs-lookup"><span data-stu-id="48990-136">Server garbage collection can be resource-intensive.</span></span> <span data-ttu-id="48990-137">Por exemplo, imagine que haja 12 processos que usam GC de servidor em execução em um computador com quatro processadores.</span><span class="sxs-lookup"><span data-stu-id="48990-137">For example, imagine that there are 12 processes that use server GC running on a computer that has four processors.</span></span> <span data-ttu-id="48990-138">Se todos os processos ocorrerem para coletar lixo ao mesmo tempo, eles interferirão uns com os outros, pois haveria 12 threads agendados no mesmo processador.</span><span class="sxs-lookup"><span data-stu-id="48990-138">If all the processes happen to collect garbage at the same time, they would interfere with each other, as there would be 12 threads scheduled on the same processor.</span></span> <span data-ttu-id="48990-139">Se os processos estiverem ativos, não é uma boa ideia fazer com que todos usem o GC do servidor.</span><span class="sxs-lookup"><span data-stu-id="48990-139">If the processes are active, it's not a good idea to have them all use server GC.</span></span>

<span data-ttu-id="48990-140">Se você estiver executando centenas de instâncias de um aplicativo, considere usar a coleta de lixo da estação de trabalho com a coleta de lixo simultânea desabilitada.</span><span class="sxs-lookup"><span data-stu-id="48990-140">If you're running hundreds of instances of an application, consider using workstation garbage collection with concurrent garbage collection disabled.</span></span> <span data-ttu-id="48990-141">Isso resultará na redução da comutação de contexto, o que pode melhorar o desempenho.</span><span class="sxs-lookup"><span data-stu-id="48990-141">This will result in less context switching, which can improve performance.</span></span>
