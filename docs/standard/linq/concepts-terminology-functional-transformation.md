## <a name="what-is-pure-functional-transformation"></a><span data-ttu-id="5dc30-108">O que é transformação funcional pura</span><span class="sxs-lookup"><span data-stu-id="5dc30-108">What is pure functional transformation</span></span>

<span data-ttu-id="5dc30-109">Na *transformação funcional pura*, um conjunto de funções, chamadas *funções puras*, define como transformar um conjunto de dados estruturados de sua forma original em outra forma.</span><span class="sxs-lookup"><span data-stu-id="5dc30-109">In *pure functional transformation*, a set of functions, called *pure functions*, define how to transform a set of structured data from its original form into another form.</span></span> <span data-ttu-id="5dc30-110">A palavra "puro" indica que as funções são *combináveis*, o que exige que elas sejam:</span><span class="sxs-lookup"><span data-stu-id="5dc30-110">The word "pure" indicates that the functions are *composable*, which requires that they're:</span></span>

- <span data-ttu-id="5dc30-111">*Autocontidas*, de modo que elas possam ser ordenadas e reorganizadas livremente sem complicação ou interdependências com o restante do programa.</span><span class="sxs-lookup"><span data-stu-id="5dc30-111">*Self-contained*, so that they can be freely ordered and rearranged without entanglement or interdependencies with the rest of the program.</span></span> <span data-ttu-id="5dc30-112">As transformações puras não têm nenhum conhecimento de efetuam-no ou em cima do seu ambiente.</span><span class="sxs-lookup"><span data-stu-id="5dc30-112">Pure transformations have no knowledge of or effect upon their environment.</span></span> <span data-ttu-id="5dc30-113">Isto é, as funções usadas na transformação não têm *efeito colateral*.</span><span class="sxs-lookup"><span data-stu-id="5dc30-113">That is, the functions used in the transformation have no *side effects*.</span></span>

### <a name="functional-programming"></a><span data-ttu-id="5dc30-120">Programação funcional</span><span class="sxs-lookup"><span data-stu-id="5dc30-120">Functional programming</span></span>
