# <a name="collection-type-dependency-properties"></a><span data-ttu-id="ef267-102">Propriedades de dependência do tipo de coleção</span><span class="sxs-lookup"><span data-stu-id="ef267-102">Collection-Type Dependency Properties</span></span>

## <a name="implementing-a-collection-type-dependency-property"></a><span data-ttu-id="ef267-104">Implementando uma Propriedade de Dependência do Tipo de Coleção</span><span class="sxs-lookup"><span data-stu-id="ef267-104">Implementing a Collection-Type Dependency Property</span></span>  
 <span data-ttu-id="ef267-105">Para uma propriedade de dependência em geral, a implementação padrão que você segue é que você definir um [!INCLUDE[TLA2#tla_clr](../../../../includes/tla2sharptla-clr-md.md)] wrapper de propriedade, em que essa propriedade é feita por um <xref:System.Windows.DependencyProperty> identificador em vez de um campo ou outra construção.</span><span class="sxs-lookup"><span data-stu-id="ef267-105">For a dependency property in general, the implementation pattern that you follow is that you define a [!INCLUDE[TLA2#tla_clr](../../../../includes/tla2sharptla-clr-md.md)] property wrapper, where that property is backed by a <xref:System.Windows.DependencyProperty> identifier rather than a field or other construct.</span></span> <span data-ttu-id="ef267-106">Você segue esse mesmo padrão quando implementa uma propriedade de tipo de coleção.</span><span class="sxs-lookup"><span data-stu-id="ef267-106">You follow this same pattern when you implement a collection-type property.</span></span> <span data-ttu-id="ef267-107">No entanto, uma propriedade do tipo coleção apresenta alguma complexidade ao padrão sempre que o tipo que está contido dentro da coleção em si for uma <xref:System.Windows.DependencyObject> ou <xref:System.Windows.Freezable> classe derivada.</span><span class="sxs-lookup"><span data-stu-id="ef267-107">However, a collection-type property introduces some complexity to the pattern whenever the type that is contained within the collection is itself a <xref:System.Windows.DependencyObject> or <xref:System.Windows.Freezable> derived class.</span></span>

## <a name="initializing-the-collection-beyond-the-default-value"></a><span data-ttu-id="ef267-108">Inicializando a coleção além do valor padrão</span><span class="sxs-lookup"><span data-stu-id="ef267-108">Initializing the Collection Beyond the Default Value</span></span>  
 <span data-ttu-id="ef267-109">Quando você cria uma propriedade de dependência, você não especificar o valor padrão da propriedade como o valor do campo inicial.</span><span class="sxs-lookup"><span data-stu-id="ef267-109">When you create a dependency property, you do not specify the property default value as the initial field value.</span></span> <span data-ttu-id="ef267-110">Em vez disso, você pode especificar o valor padrão por meio de metadados de propriedade de dependência.</span><span class="sxs-lookup"><span data-stu-id="ef267-110">Instead, you specify the default value through the dependency property metadata.</span></span> <span data-ttu-id="ef267-111">Se a propriedade for um tipo de referência, o valor padrão especificado nos metadados de propriedade de dependência não será um valor padrão por instância; em vez disso, será um valor padrão que se aplica a todas as instâncias do tipo.</span><span class="sxs-lookup"><span data-stu-id="ef267-111">If your property is a reference type, the default value specified in dependency property metadata is not a default value per instance; ## <a name="reporting-binding-value-changes-from-collection-properties"></a><span data-ttu-id="ef267-129">Relatório de alterações de valor de associação de propriedades de coleção</span><span class="sxs-lookup"><span data-stu-id="ef267-129">Reporting Binding Value Changes from Collection Properties</span></span>  
 <span data-ttu-id="ef267-130">Uma propriedade de coleção que é uma propriedade de dependência não relata automaticamente alterações para suas subpropriedades.</span><span class="sxs-lookup"><span data-stu-id="ef267-130">A collection property that is itself a dependency property does not automatically report changes to its subproperties.</span></span> <span data-ttu-id="ef267-131">Se você estiver criando vínculos em uma coleção, isso poderá impedir que a associação relate alterações, portanto invalidando alguns cenários de vinculação de dados.</span><span class="sxs-lookup"><span data-stu-id="ef267-131">If you are creating bindings into a collection, this can prevent the binding from reporting changes, thus invalidating some data binding scenarios.</span></span> <span data-ttu-id="ef267-132">No entanto, se você usar o tipo de coleção <xref:System.Windows.FreezableCollection%601> como o tipo de coleção, então alterações de subpropriedades em elementos contidos na coleção são relatadas corretamente, e associação funciona conforme o esperado.</span><span class="sxs-lookup"><span data-stu-id="ef267-132">However, if you use the collection type <xref:System.Windows.FreezableCollection%601> as your collection type, then subproperty changes to contained elements in the collection are properly reported, and binding works as expected.</span></span>  
  
 <span data-ttu-id="ef267-133">Para habilitar a associação de subpropriedade em uma coleção de objetos de dependência, crie a propriedade de coleção como tipo <xref:System.Windows.FreezableCollection%601>, com uma restrição de tipo para essa coleção para qualquer <xref:System.Windows.DependencyObject> classe derivada.</span><span class="sxs-lookup"><span data-stu-id="ef267-133">To enable subproperty binding in a dependency object collection, create the collection property as type <xref:System.Windows.FreezableCollection%601>, with a type constraint for that collection to any <xref:System.Windows.DependencyObject> derived class.</span></span>
