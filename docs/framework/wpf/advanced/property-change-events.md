## <a name="property-triggers"></a><span data-ttu-id="c1b04-132">Gatilhos de propriedade</span><span class="sxs-lookup"><span data-stu-id="c1b04-132">Property Triggers</span></span>  
 <span data-ttu-id="c1b04-133">Um conceito fortemente relacionado a um evento de propriedade alterado é um gatilho de propriedade.</span><span class="sxs-lookup"><span data-stu-id="c1b04-133">A closely related concept to a property changed event is a property trigger.</span></span> <span data-ttu-id="c1b04-134">Um gatilho de propriedade é criado em um estilo ou modelo e permite que você crie um comportamento condicional com base no valor da propriedade na qual o gatilho de propriedade é designado.</span><span class="sxs-lookup"><span data-stu-id="c1b04-134">A property trigger is created within a style or template and enables you to create a conditional behavior based on the value of the property where the property trigger is assigned.</span></span>  
  
 <span data-ttu-id="c1b04-135">A propriedade de um gatilho de propriedade deve ser uma propriedade de dependência.</span><span class="sxs-lookup"><span data-stu-id="c1b04-135">The property for a property trigger must be a dependency property.</span></span> <span data-ttu-id="c1b04-136">Ela pode ser (e frequentemente é) uma propriedade de dependência somente leitura.</span><span class="sxs-lookup"><span data-stu-id="c1b04-136">It can be (and frequently is) a read-only dependency property.</span></span> <span data-ttu-id="c1b04-137">Um bom indicador para quando uma propriedade de dependência exposta por um controle é, pelo menos, parcialmente projetada para ser um gatilho de propriedade é se o nome da propriedade começar com "Is".</span><span class="sxs-lookup"><span data-stu-id="c1b04-137">A good indicator for when a dependency property exposed by a control is at least partially designed to be a property trigger is if the property name begins with "Is".</span></span> <span data-ttu-id="c1b04-138">As propriedades que têm essa nomenclatura são, geralmente, propriedade de dependência booliana somente leitura em que o cenário principal para a propriedade é o estado de controle de relatório que pode ter consequências para a interface do usuário em tempo real e, portanto, é um candidato a gatilho de propriedade.</span><span class="sxs-lookup"><span data-stu-id="c1b04-138">Properties that have this naming are often a read-only Boolean dependency property where the primary scenario for the property is reporting control state that might have consequences to the real-time UI and is thus a property trigger candidate.</span></span>  
  
 <span data-ttu-id="c1b04-139">Algumas dessas propriedades também têm um evento de propriedade alterado dedicado.</span><span class="sxs-lookup"><span data-stu-id="c1b04-139">Some of these properties also have a dedicated property changed event.</span></span> <span data-ttu-id="c1b04-140">Por exemplo, a propriedade <xref:System.Windows.UIElement.IsMouseCaptured%2A> tem um evento de alteração de propriedade <xref:System.Windows.UIElement.IsMouseCapturedChanged>.</span><span class="sxs-lookup"><span data-stu-id="c1b04-140">For instance, the property <xref:System.Windows.UIElement.IsMouseCaptured%2A> has a property changed event <xref:System.Windows.UIElement.IsMouseCapturedChanged>.</span></span> <span data-ttu-id="c1b04-141">A propriedade em si é somente leitura, com seu valor ajustado pelo sistema de entrada e o sistema de entrada gera <xref:System.Windows.UIElement.IsMouseCapturedChanged> em cada alteração em tempo real.</span><span class="sxs-lookup"><span data-stu-id="c1b04-141">The property itself is read-only, with its value adjusted by the input system, and the input system raises <xref:System.Windows.UIElement.IsMouseCapturedChanged> on each real-time change.</span></span>  
  
 <span data-ttu-id="c1b04-142">Em comparação com um verdadeiro evento de alteração de propriedade, usar um gatilho de propriedade para atuar em uma alteração de propriedade tem algumas limitações.</span><span class="sxs-lookup"><span data-stu-id="c1b04-142">Compared to a true property changed event, using a property trigger to act on a property change has some limitations.</span></span>  
  
 <span data-ttu-id="c1b04-143">Os gatilhos de propriedade funcionam por uma lógica de correspondência.</span><span class="sxs-lookup"><span data-stu-id="c1b04-143">Property triggers work through an exact match logic.</span></span> <span data-ttu-id="c1b04-144">É possível especificar uma propriedade e um valor que indicam o valor específico para o qual o gatilho atuará.  <span data-ttu-id="c1b04-147">A sintaxe do gatilho de propriedade é análoga a uma instrução "if" em programação.</span><span class="sxs-lookup"><span data-stu-id="c1b04-147">The property trigger syntax is analogous to an "if" statement in programming.</span></span> <span data-ttu-id="c1b04-148">Se a condição do gatilho for verdadeira, então, o "corpo" do gatilho de propriedade será "executado".</span><span class="sxs-lookup"><span data-stu-id="c1b04-148">If the trigger condition is true, then the "body" of the property trigger is "executed".</span></span> <span data-ttu-id="c1b04-149">O "corpo" de um gatilho de propriedade não é código, é marcação.</span><span class="sxs-lookup"><span data-stu-id="c1b04-149">The "body" of a property trigger is not code, it is markup.</span></span> <span data-ttu-id="c1b04-150">Essa marcação é limitada ao uso de um ou mais <xref:System.Windows.Setter> elementos para definir outras propriedades do objeto em que o estilo ou modelo está sendo aplicado.</span><span class="sxs-lookup"><span data-stu-id="c1b04-150">That markup is limited to using one or more <xref:System.Windows.Setter> elements to set other properties of the object where the style or template is being applied.</span></span>

 <span data-ttu-id="c1b04-153">Os gatilhos de propriedade são geralmente apropriados para cenários em que uma ou mais propriedades de aparência devem ser alteradas, com base no estado de outra propriedade no mesmo elemento.</span><span class="sxs-lookup"><span data-stu-id="c1b04-153">Property triggers are generally appropriate for scenarios where one or more appearance properties should change, based on the state of another property on the same element.</span></span>
