# <a name="version-compatibility"></a><span data-ttu-id="6ae45-103">Compatibilidade de versões</span><span class="sxs-lookup"><span data-stu-id="6ae45-103">Version compatibility</span></span>

<span data-ttu-id="6ae45-104">Compatibilidade com versões anteriores significa que um aplicativo desenvolvido para uma versão específica de uma plataforma será executado em versões posteriores dessa plataforma.</span><span class="sxs-lookup"><span data-stu-id="6ae45-104">Backward compatibility means that an app that was developed for a particular version of a platform will run on later versions of that platform.</span></span> <span data-ttu-id="6ae45-105">.NET Framework tenta maximizar a compatibilidade com versões anteriores: o código-fonte escrito para uma versão do .NET Framework deve ser compilado em versões posteriores do .NET Framework, e os binários executados em uma versão do .NET Framework devem se comportar de forma idêntica em versões posteriores do .NET Framework.</span><span class="sxs-lookup"><span data-stu-id="6ae45-105">.NET Framework tries to maximize backward compatibility: Source code written for one version of .NET Framework should compile on later versions of .NET Framework, and binaries that run on one version of .NET Framework should behave identically on later versions of .NET Framework.</span></span>

## <a name="version-compatibility-for-apps"></a><a name="Apps"></a> <span data-ttu-id="6ae45-106">Compatibilidade de versão para aplicativos</span><span class="sxs-lookup"><span data-stu-id="6ae45-106">Version compatibility for apps</span></span>

<span data-ttu-id="6ae45-107">Por padrão, um aplicativo é executado na versão do .NET Framework para o qual foi criado.</span><span class="sxs-lookup"><span data-stu-id="6ae45-107">By default, an app runs on the version of .NET Framework that it was built for.</span></span> <span data-ttu-id="6ae45-108">Se essa versão não estiver presente e o arquivo de configuração de aplicativos não definir versões compatíveis, um erro de inicialização do .NET Framework poderá ocorrer.</span><span class="sxs-lookup"><span data-stu-id="6ae45-108">If that version isn't present and the app configuration file doesn't define supported versions, a .NET Framework initialization error may occur.</span></span> <span data-ttu-id="6ae45-109">Nesse caso, a tentativa de executar o aplicativo falhará.</span><span class="sxs-lookup"><span data-stu-id="6ae45-109">In this case, the attempt to run the app will fail.</span></span>

<span data-ttu-id="6ae45-110">Para definir as versões específicas nas quais seu aplicativo é executado, adicione um ou mais [\<supportedRuntime>](../configure-apps/file-schema/startup/supportedruntime-element.md) elementos ao arquivo de configuração do aplicativo.</span><span class="sxs-lookup"><span data-stu-id="6ae45-110">To define the specific versions on which your app runs, add one or more [\<supportedRuntime>](../configure-apps/file-schema/startup/supportedruntime-element.md) elements to your app's configuration file.</span></span> <span data-ttu-id="6ae45-111">Cada elemento `<supportedRuntime>` lista uma versão compatível do runtime, com o primeiro especificando a versão mais preferida e o último especificando a versão menos preferida.</span><span class="sxs-lookup"><span data-stu-id="6ae45-111">Each `<supportedRuntime>` element lists a supported version of the runtime, with the first specifying the most preferred version and the last specifying the least preferred version.</span></span>

```xml

   <startup>
      <supportedRuntime version="v2.0.50727" />
      <supportedRuntime version="v4.0" />
   </startup>
</configuration>
```

<span data-ttu-id="6ae45-112">Para saber mais, confira [How to: Configure an App to Support .NET Framework 4 or 4.x](how-to-configure-an-app-to-support-net-framework-4-or-4-5.md) (Como configurar um aplicativo para dar suporte ao .NET Framework 4 ou 4.x).</span><span class="sxs-lookup"><span data-stu-id="6ae45-112">For more information, see [How to: Configure an App to Support .NET Framework 4 or 4.x](how-to-configure-an-app-to-support-net-framework-4-or-4-5.md).</span></span>

## <a name="backward-compatibility"></a><span data-ttu-id="6ae45-119">Compatibilidade com versões anteriores</span><span class="sxs-lookup"><span data-stu-id="6ae45-119">Backward compatibility</span></span>

<span data-ttu-id="6ae45-124">Na prática, essa compatibilidade pode ser desfeita por alterações aparentemente inconsequentes feitas no .NET Framework e por alterações em técnicas de programação.</span><span class="sxs-lookup"><span data-stu-id="6ae45-124">In practice, this compatibility can be broken by seemingly inconsequential changes in the .NET Framework and changes in programming techniques.</span></span> <span data-ttu-id="6ae45-125">Por exemplo, as melhorias de desempenho no .NET Framework 4.5 podem expor uma condição de corrida que não ocorria em versões anteriores.</span><span class="sxs-lookup"><span data-stu-id="6ae45-125">For example, performance improvements in the .NET Framework 4.5 can expose a race condition that did not occur on earlier versions.</span></span> <span data-ttu-id="6ae45-126">Da mesma forma, o uso de um caminho codificado para assemblies do .NET Framework, a execução de uma comparação de igualdade com uma versão específica do .NET Framework e a obtenção do valor de um campo particular usando-se reflexão não são práticas compatíveis com versões anteriores.</span><span class="sxs-lookup"><span data-stu-id="6ae45-126">Similarly, using a hard-coded path to .NET Framework assemblies, performing an equality comparison with a particular version of the .NET Framework, and getting the value of a private field by using reflection are not backward-compatible practices.</span></span> <span data-ttu-id="6ae45-127">Além disso, cada versão do .NET Framework inclui correções de bug e mudanças relacionadas à segurança que podem afetar a compatibilidade de alguns aplicativos e componentes.</span><span class="sxs-lookup"><span data-stu-id="6ae45-127">In addition, each version of the .NET Framework includes bug fixes and security-related changes that can affect the compatibility of some apps and components.</span></span>
