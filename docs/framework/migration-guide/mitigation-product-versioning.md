# <a name="mitigation-product-versioning"></a><span data-ttu-id="51bb2-102">Mitigação: Controle de versão do produto</span><span class="sxs-lookup"><span data-stu-id="51bb2-102">Mitigation: Product Versioning</span></span>
<span data-ttu-id="51bb2-103">Na [!INCLUDE[net_v46](../../../includes/net-v46-md.md)] e posterior, o controle de versão de produto foi alterada em relação às versões anteriores do .NET Framework (o .NET Framework 4, 4.5, 4.5.1 e 4.5.2).</span><span class="sxs-lookup"><span data-stu-id="51bb2-103">In the [!INCLUDE[net_v46](../../../includes/net-v46-md.md)] and later, product versioning has changed from the previous releases of the .NET Framework (the .NET Framework 4, 4.5, 4.5.1, and 4.5.2).</span></span>  
  
## <a name="product-versioning-changes"></a><span data-ttu-id="51bb2-104">Alterações no controle de versão de produto</span><span class="sxs-lookup"><span data-stu-id="51bb2-104">Product versioning changes</span></span>  
 <span data-ttu-id="51bb2-105">Veja a seguir as alterações em detalhes:</span><span class="sxs-lookup"><span data-stu-id="51bb2-105">The following are the detailed changes:</span></span>  
  
- <span data-ttu-id="51bb2-106">O valor da entrada `Version` na chave `HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\NET Framework Setup\NDP\v4\Full` foi alterado para `4.6.`*xxxxx* para o .NET Framework 4.6 e suas versões pontuais, e para `4.7.`*xxxxx* para o .NET Framework 4.7.</span><span class="sxs-lookup"><span data-stu-id="51bb2-106">The value of the `Version` entry in the `HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\NET Framework Setup\NDP\v4\Full` key has changed to `4.6.`*xxxxx* for the .NET Framework 4.6 and its point releases, and to `4.7.`*xxxxx* for the .NET Framework 4.7.</span></span> <span data-ttu-id="51bb2-107">No .NET Framework 4.5, 4.5.1 e 4.5.2, ele tinha o formato `4.5.`*xxxxx*.</span><span class="sxs-lookup"><span data-stu-id="51bb2-107">In the .NET Framework 4.5, 4.5.1, and 4.5.2, it had the format `4.5.`*xxxxx*.</span></span>  
  
- <span data-ttu-id="51bb2-108">O controle de versão de arquivo e produto para arquivos do .NET Framework foi alterado do esquema de controle de versão anterior de `4.0.30319.x` para o de `4.6.X.0` para o .NET Framework 4.6 e suas versões pontuais, e para o de `4.7.X.0` para o .NET Framework 4.7 e suas versões pontuais.</span><span class="sxs-lookup"><span data-stu-id="51bb2-108">The file and product versioning for .NET Framework files has changed from the earlier versioning scheme of `4.0.30319.x` to `4.6.X.0` for the .NET Framework 4.6 and its point releases, and to `4.7.X.0` for the .NET Framework 4.7 and its point releases.</span></span> <span data-ttu-id="51bb2-109">Você pode ver esses novos valores quando exibe as **Propriedades** do arquivo depois de clicar com o botão direito do mouse em um arquivo.</span><span class="sxs-lookup"><span data-stu-id="51bb2-109">You can see these new values when you view the file's **Properties** after right-clicking on a file.</span></span>  
  
- <span data-ttu-id="51bb2-110">Os atributos <xref:System.Reflection.AssemblyFileVersionAttribute> e <xref:System.Reflection.AssemblyInformationalVersionAttribute> para assemblies gerenciados têm valores de <xref:System.Version> no formulário `4.6.X.0` para o .NET Framework 4.6 e suas versões pontuais, e `4.7.X.0` para o .NET Framework 4.7.</span><span class="sxs-lookup"><span data-stu-id="51bb2-110">The <xref:System.Reflection.AssemblyFileVersionAttribute> and <xref:System.Reflection.AssemblyInformationalVersionAttribute> attributes for managed assemblies have <xref:System.Version> values in the form `4.6.X.0` for the .NET Framework 4.6 and its point releases, and `4.7.X.0` for the .NET Framework 4.7.</span></span>  
  
- <span data-ttu-id="51bb2-111">No [!INCLUDE[net_v46](../../../includes/net-v46-md.md)], 4.6.1 4.6.2 e 4.7, a propriedade <xref:System.Environment.Version%2A?displayProperty=nameWithType> retorna a cadeia de caracteres de versão fixa `4.0.30319.42000`.</span><span class="sxs-lookup"><span data-stu-id="51bb2-111">In the [!INCLUDE[net_v46](../../../includes/net-v46-md.md)], 4.6.1, 4.6.2, and 4.7, the <xref:System.Environment.Version%2A?displayProperty=nameWithType> property returns the fixed version string `4.0.30319.42000`.</span></span> <span data-ttu-id="51bb2-112">No .NET Framework 4, 4.5, 4.5.1 e 4.5.2, ela retorna as cadeias de caracteres de versão no formato `4.0.30319.xxxxx` (por exemplo, "4.0.30319.18010").</span><span class="sxs-lookup"><span data-stu-id="51bb2-112">In the .NET Framework 4, 4.5, 4.5.1, and 4.5.2, it returns version strings in the format `4.0.30319.xxxxx` (for example, "4.0.30319.18010").</span></span> <span data-ttu-id="51bb2-113">Não é recomendável criar nova dependência de código de aplicativo na propriedade <xref:System.Environment.Version%2A?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="51bb2-113">Note that we do not recommend application code taking any new dependency on the <xref:System.Environment.Version%2A?displayProperty=nameWithType> property.</span></span>  
  
### <a name="handling-the-product-versioning-changes"></a><span data-ttu-id="51bb2-114">Como lidar com as alterações de controle de versão de produto</span><span class="sxs-lookup"><span data-stu-id="51bb2-114">Handling the product versioning changes</span></span>  
 <span data-ttu-id="51bb2-115">Em geral, os aplicativos devem depender das técnicas recomendadas para detecção de itens como a versão de tempo de execução do .NET Framework e o diretório de instalação:</span><span class="sxs-lookup"><span data-stu-id="51bb2-115">In general, applications should depend on the recommended techniques for detecting such things as the runtime version of the .NET Framework and the installation directory:</span></span>  
  
- <span data-ttu-id="51bb2-116">Para detectar a versão de tempo de execução do .NET Framework, confira [Como: Determinar quais versões do .NET Framework estão instaladas](../../../docs/framework/migration-guide/how-to-determine-which-versions-are-installed.md).</span><span class="sxs-lookup"><span data-stu-id="51bb2-116">To detect the runtime version of the .NET Framework, see [How to: Determine Which .NET Framework Versions Are Installed](../../../docs/framework/migration-guide/how-to-determine-which-versions-are-installed.md).</span></span>  
  
- <span data-ttu-id="51bb2-117">Para determinar o caminho de instalação do .NET Framework, use o valor da entrada `InstallPath` na chave `HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\NET Framework Setup\NDP\v4\Full`.</span><span class="sxs-lookup"><span data-stu-id="51bb2-117">To determine the installation path for the .NET Framework, use the value of the `InstallPath` entry in the `HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\NET Framework Setup\NDP\v4\Full` key.</span></span>

- <span data-ttu-id="51bb2-120">Para obter a versão do CLR, chame o método <xref:System.Runtime.InteropServices.RuntimeEnvironment.GetSystemVersion%2A?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="51bb2-120">To get the CLR version, call the <xref:System.Runtime.InteropServices.RuntimeEnvironment.GetSystemVersion%2A?displayProperty=nameWithType> method.</span></span>   <span data-ttu-id="51bb2-121">Para o .NET Framework 4 e suas versões pontuais (o .NET Framework 4.5, 4.5.1, 4.5.2 e [!INCLUDE[net_v46](../../../includes/net-v46-md.md)], 4.6.1, 4.6.2 e 4.7), ele retorna a cadeia de caracteres `v4.0.30319`.</span><span class="sxs-lookup"><span data-stu-id="51bb2-121">For the .NET Framework 4 and its point releases (the .NET Framework 4.5, 4.5.1, 4.5.2, and [!INCLUDE[net_v46](../../../includes/net-v46-md.md)], 4.6.1, 4.6.2, and 4.7), it returns the string `v4.0.30319`.</span></span>
