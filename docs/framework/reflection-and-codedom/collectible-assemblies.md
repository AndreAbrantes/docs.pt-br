# <a name="collectible-assemblies-for-dynamic-type-generation"></a><span data-ttu-id="1a7ff-102">Assemblies de coleção para geração de tipos dinâmicos</span><span class="sxs-lookup"><span data-stu-id="1a7ff-102">Collectible assemblies for dynamic type generation</span></span>

<span data-ttu-id="1a7ff-103">*Assemblies de coleção* são assemblies dinâmicos que podem ser descarregados sem descarregar o domínio do aplicativo no qual eles foram criados.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-103">*Collectible assemblies* are dynamic assemblies that can be unloaded without unloading the application domain in which they were created.</span></span> <span data-ttu-id="1a7ff-104">Toda a memória gerenciada e não gerenciada usada por um assembly de coleção e os tipos que ele contém pode ser recuperada.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-104">All managed and unmanaged memory used by a collectible assembly and the types it contains can be reclaimed.</span></span> <span data-ttu-id="1a7ff-105">Informações como o nome do assembly são removidas das tabelas internas.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-105">Information such as the assembly name is removed from internal tables.</span></span>

<span data-ttu-id="1a7ff-106">Para habilitar o descarregamento, use o sinalizador <xref:System.Reflection.Emit.AssemblyBuilderAccess.RunAndCollect?displayProperty=nameWithType> ao criar um assembly dinâmico.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-106">To enable unloading, use the <xref:System.Reflection.Emit.AssemblyBuilderAccess.RunAndCollect?displayProperty=nameWithType> flag when you create a dynamic assembly.</span></span> <span data-ttu-id="1a7ff-107">O assembly é transitório (ou seja, ele não pode ser salvo) e está sujeito às limitações descritas na seção [Restrições de assemblies de coleção](#restrictions-on-collectible-assemblies).</span><span class="sxs-lookup"><span data-stu-id="1a7ff-107">The assembly is transient (that is, it cannot be saved) and is subject to limitations described in the [Restrictions on Collectible Assemblies](#restrictions-on-collectible-assemblies) section.</span></span> <span data-ttu-id="1a7ff-108">O CLR (Common Language Runtime) descarrega um assembly de coleção automaticamente quando todos os objetos associados ao assembly são liberados.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-108">The common language runtime (CLR) unloads a collectible assembly automatically when you release all objects associated with the assembly.</span></span> <span data-ttu-id="1a7ff-109">Em todos os outros aspectos, os assemblies de coleção são criados e usados da mesma maneira de outros assemblies dinâmicos.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-109">In all other respects, collectible assemblies are created and used in the same way as other dynamic assemblies.</span></span>

## <a name="lifetime-of-collectible-assemblies"></a><span data-ttu-id="1a7ff-110">Tempo de vida dos assemblies de coleção</span><span class="sxs-lookup"><span data-stu-id="1a7ff-110">Lifetime of collectible assemblies</span></span>

<span data-ttu-id="1a7ff-111">O tempo de vida de um assembly de coleção é controlado pela existência de referências aos tipos que ele contém e aos objetos que são criados com base nesses tipos.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-111">The lifetime of a collectible assembly is controlled by the existence of references to the types it contains and the objects that are created from those types.</span></span> <span data-ttu-id="1a7ff-112">O Common Language Runtime não descarregará um assembly contanto que uma ou mais das seguintes condições exista (`T` é qualquer tipo definido no assembly):</span><span class="sxs-lookup"><span data-stu-id="1a7ff-112">The common language runtime does not unload an assembly as long as one or more of the following exist (`T` is any type that is defined in the assembly):</span></span> 

- <span data-ttu-id="1a7ff-113">Uma instância de `T`.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-113">An instance of `T`.</span></span>

- <span data-ttu-id="1a7ff-114">Uma instância de uma matriz de `T`.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-114">An instance of an array of `T`.</span></span>
 
- <span data-ttu-id="1a7ff-115">Uma instância de um tipo genérico que tem `T` como um de seus argumentos de tipo.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-115">An instance of a generic type that has `T` as one of its type arguments.</span></span> <span data-ttu-id="1a7ff-116">Isso inclui coleções genéricas de `T`, mesmo se a coleção estiver vazia.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-116">This includes generic collections of `T`, even if that collection is empty.</span></span>

- <span data-ttu-id="1a7ff-117">Uma instância de <xref:System.Type> ou <xref:System.Reflection.Emit.TypeBuilder> que representa `T`.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-117">An instance of <xref:System.Type> or <xref:System.Reflection.Emit.TypeBuilder> that represents `T`.</span></span> 

   > [!IMPORTANT]
   > <span data-ttu-id="1a7ff-118">É necessário liberar todos os objetos que representam partes do assembly.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-118">You must release all objects that represent parts of the assembly.</span></span> <span data-ttu-id="1a7ff-119">O <xref:System.Reflection.Emit.ModuleBuilder> que define `T` mantém uma referência ao <xref:System.Reflection.Emit.TypeBuilder> e o objeto <xref:System.Reflection.Emit.AssemblyBuilder> mantém uma referência ao <xref:System.Reflection.Emit.ModuleBuilder>; portanto, as referências a esses objetos devem ser liberadas.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-119">The <xref:System.Reflection.Emit.ModuleBuilder> that defines `T` keeps a reference to the <xref:System.Reflection.Emit.TypeBuilder>, and the <xref:System.Reflection.Emit.AssemblyBuilder> object keeps a reference to the <xref:System.Reflection.Emit.ModuleBuilder>, so references to these objects must be released.</span></span> <span data-ttu-id="1a7ff-120">Até mesmo a existência de um <xref:System.Reflection.Emit.LocalBuilder> ou um <xref:System.Reflection.Emit.ILGenerator> usado na construção de `T` impede o descarregamento.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-120">Even the existence of a <xref:System.Reflection.Emit.LocalBuilder> or an <xref:System.Reflection.Emit.ILGenerator> used in the construction of `T` prevents unloading.</span></span>

- <span data-ttu-id="1a7ff-121">Uma referência estática a `T` por outro tipo definido de forma dinâmica `T1` que ainda pode ser acessado pelo código em execução.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-121">A static reference to `T` by another dynamically defined type `T1` that is still reachable by executing code.</span></span> <span data-ttu-id="1a7ff-122">Por exemplo, `T1` pode derivar de `T` ou `T` pode ser o tipo de um parâmetro em um método de `T1`.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-122">For example, `T1` might derive from `T`, or `T` might be the type of a parameter in a method of `T1`.</span></span>
 
- <span data-ttu-id="1a7ff-123">Uma **ByRef** a um campo estático que pertence a `T`.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-123">A **ByRef** to a static field that belongs to `T`.</span></span>

- <span data-ttu-id="1a7ff-124">Um <xref:System.RuntimeTypeHandle>, <xref:System.RuntimeFieldHandle> ou <xref:System.RuntimeMethodHandle> que se refere a `T` ou a um componente de `T`.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-124">A <xref:System.RuntimeTypeHandle>, <xref:System.RuntimeFieldHandle>, or <xref:System.RuntimeMethodHandle> that refers to `T` or to a component of `T`.</span></span>

- <span data-ttu-id="1a7ff-125">Uma instância de qualquer objeto de reflexão que poderá ser usada indireta ou diretamente para acessar o objeto <xref:System.Type> que representa `T`.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-125">An instance of any reflection object that could be used indirectly or directly to access the <xref:System.Type> object that represents `T`.</span></span> <span data-ttu-id="1a7ff-126">Por exemplo, o objeto <xref:System.Type> de `T` pode ser obtido de um tipo de matriz cujo tipo de elemento é `T` ou de um tipo genérico que tem `T` como um argumento de tipo.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-126">For example, the <xref:System.Type> object for `T` can be obtained from an array type whose element type is `T`, or from a generic type that has `T` as a type argument.</span></span> 

- <span data-ttu-id="1a7ff-127">Um método `M` na pilha de chamadas de qualquer thread, em que `M` é um método de `T` ou um método de nível de módulo definido no assembly.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-127">A method `M` on the call stack of any thread, where `M` is a method of `T` or a module-level method that is defined in the assembly.</span></span>

- <span data-ttu-id="1a7ff-128">Um representante de um método estático definido em um módulo do assembly.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-128">A delegate to a static method that is defined in a module of the assembly.</span></span>

<span data-ttu-id="1a7ff-129">Se houver um único item dessa lista para apenas um tipo ou um método no assembly, o tempo de execução não poderá descarregar o assembly.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-129">If only one item from this list exists for only one type or one method in the assembly, the runtime cannot unload the assembly.</span></span>

> [!NOTE]
> <span data-ttu-id="1a7ff-130">Na verdade, o tempo de execução não descarrega o assembly até que os finalizadores tenham sido executados para todos os itens da lista.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-130">The runtime does not actually unload the assembly until finalizers have run for all items in the list.</span></span>

<span data-ttu-id="1a7ff-131">Para fins de acompanhamento do tempo de vida, um tipo genérico construído, como `List<int>` (no C#) ou `List(Of Integer)` (no Visual Basic), que é criado e usado na geração de um assembly de coleção é considerado como tendo sido definido no assembly que contém a definição de tipo genérico ou em um assembly que contém a definição de um de seus argumentos de tipo.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-131">For purposes of tracking lifetime, a constructed generic type such as `List<int>` (in C#) or `List(Of Integer)` (in Visual Basic) that is created and used in the generation of a collectible assembly is considered to have been defined either in the assembly that contains the generic type definition or in an assembly that contains the definition of one of its type arguments.</span></span> <span data-ttu-id="1a7ff-132">O assembly exato usado é um detalhe de implementação e está sujeito a alterações.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-132">The exact assembly that is used is an implementation detail and subject to change.</span></span>
 
## <a name="restrictions-on-collectible-assemblies"></a><span data-ttu-id="1a7ff-133">Restrições de assemblies de coleção</span><span class="sxs-lookup"><span data-stu-id="1a7ff-133">Restrictions on collectible assemblies</span></span>

<span data-ttu-id="1a7ff-134">As seguintes restrições se aplicam a assemblies de coleção:</span><span class="sxs-lookup"><span data-stu-id="1a7ff-134">The following restrictions apply to collectible assemblies:</span></span> 

- <span data-ttu-id="1a7ff-135">**Referências estáticas** </span><span class="sxs-lookup"><span data-stu-id="1a7ff-135">**Static references** </span></span>  
  <span data-ttu-id="1a7ff-136">Os tipos em um assembly dinâmico comum não podem ter referências estáticas a tipos definidos em um assembly de coleção.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-136">Types in an ordinary dynamic assembly cannot have static references to types that are defined in a collectible assembly.</span></span> <span data-ttu-id="1a7ff-137">Por exemplo, se você definir um tipo comum herdado de um tipo em um assembly de coleção, uma exceção <xref:System.NotSupportedException> será gerada.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-137">For example, if you define an ordinary type that inherits from a type in a collectible assembly, a <xref:System.NotSupportedException> exception is thrown.</span></span> <span data-ttu-id="1a7ff-138">Um tipo em um assembly de coleção pode ter referências estáticas a um tipo em outro assembly de coleção, mas isso aumenta o tempo de vida do assembly referenciado para o tempo de vida do assembly de referência.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-138">A type in a collectible assembly can have static references to a type in another collectible assembly, but this extends the lifetime of the referenced assembly to the lifetime of the referencing assembly.</span></span>

- <span data-ttu-id="1a7ff-139">**Interoperabilidade COM** </span><span class="sxs-lookup"><span data-stu-id="1a7ff-139">**COM interop** </span></span>  
   <span data-ttu-id="1a7ff-140">Nenhuma interface COM pode ser definida em um assembly de coleção e nenhuma instância de tipos em um assembly de coleção pode ser convertida em objetos COM.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-140">No COM interfaces can be defined within a collectible assembly, and no instances of types within a collectible assembly can be converted into COM objects.</span></span> <span data-ttu-id="1a7ff-141">Um tipo em um assembly de coleção não pode servir como um CCW (COM Callable Wrapper) ou RCW (Runtime Callable Wrapper).</span><span class="sxs-lookup"><span data-stu-id="1a7ff-141">A type in a collectible assembly cannot serve as a COM callable wrapper (CCW) or runtime callable wrapper (RCW).</span></span> <span data-ttu-id="1a7ff-142">No entanto, os tipos em assemblies de coleção podem usar objetos que implementam interfaces COM.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-142">However, types in collectible assemblies can use objects that implement COM interfaces.</span></span>

- <span data-ttu-id="1a7ff-143">**Invocação de plataforma** </span><span class="sxs-lookup"><span data-stu-id="1a7ff-143">**Platform invoke** </span></span>  
   <span data-ttu-id="1a7ff-144">Os métodos que têm o atributo <xref:System.Runtime.InteropServices.DllImportAttribute> não serão compilados quando forem declarados em um assembly de coleção.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-144">Methods that have the <xref:System.Runtime.InteropServices.DllImportAttribute> attribute will not compile when they are declared in a collectible assembly.</span></span> <span data-ttu-id="1a7ff-145">A instrução <xref:System.Reflection.Emit.OpCodes.Calli?displayProperty=nameWithType> não pode ser usada na implementação de um tipo em um assembly de coleção e esses tipos não podem ter marshaling para um código não gerenciado.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-145">The <xref:System.Reflection.Emit.OpCodes.Calli?displayProperty=nameWithType> instruction cannot be used in the implementation of a type in a collectible assembly, and such types cannot be marshaled to unmanaged code.</span></span> <span data-ttu-id="1a7ff-146">No entanto, você pode chamar um código nativo por meio de um ponto de entrada declarado em um assembly de não coleção.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-146">However, you can call into native code by using an entry point that is declared in a non-collectible assembly.</span></span>
 
- <span data-ttu-id="1a7ff-147">**Marshaling** </span><span class="sxs-lookup"><span data-stu-id="1a7ff-147">**Marshaling** </span></span>  
   <span data-ttu-id="1a7ff-148">Objetos (em particular, representantes) que são definidos em assemblies de coleção não podem ter marshaling.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-148">Objects (in particular, delegates) that are defined in collectible assemblies cannot be marshaled.</span></span> <span data-ttu-id="1a7ff-149">Essa é uma restrição de todos os tipos emitidos transitórios.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-149">This is a restriction on all transient emitted types.</span></span>

- <span data-ttu-id="1a7ff-150">**Carregamento de assembly** </span><span class="sxs-lookup"><span data-stu-id="1a7ff-150">**Assembly loading** </span></span>  
   <span data-ttu-id="1a7ff-151">A emissão de reflexão é o único mecanismo com suporte para o carregamento de assemblies de coleção.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-151">Reflection emit is the only mechanism that is supported for loading collectible assemblies.</span></span> <span data-ttu-id="1a7ff-152">Assemblies carregados por meio de qualquer outra forma de carregamento do assembly não podem ser descarregados.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-152">Assemblies that are loaded by using any other form of assembly loading cannot be unloaded.</span></span>
 
- <span data-ttu-id="1a7ff-153">**Objetos associados ao contexto**  </span><span class="sxs-lookup"><span data-stu-id="1a7ff-153">**Context-bound objects**  </span></span>  
   <span data-ttu-id="1a7ff-154">Não há suporte para variáveis de contexto estático.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-154">Context-static variables are not supported.</span></span> <span data-ttu-id="1a7ff-155">Tipos em um assembly de coleção não podem estender <xref:System.ContextBoundObject>.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-155">Types in a collectible assembly cannot extend <xref:System.ContextBoundObject>.</span></span> <span data-ttu-id="1a7ff-156">No entanto, um código em assemblies de coleção podem usar objetos associados a contexto definidos em outro lugar.</span><span class="sxs-lookup"><span data-stu-id="1a7ff-156">However, code in collectible assemblies can use context-bound objects that are defined elsewhere.</span></span>

- <span data-ttu-id="1a7ff-157">**Dados de thread estático**     </span><span class="sxs-lookup"><span data-stu-id="1a7ff-157">**Thread-static data**     </span></span>
