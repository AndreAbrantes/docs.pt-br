<span data-ttu-id="72b45-135">A reflexão fornece classes, como <xref:System.Type> e <xref:System.Reflection.MethodInfo>, para representar tipos, membros, parâmetros e outras entidades de código.</span><span class="sxs-lookup"><span data-stu-id="72b45-135">Reflection provides classes, such as <xref:System.Type> and <xref:System.Reflection.MethodInfo>, to represent types, members, parameters, and other code entities.</span></span> <span data-ttu-id="72b45-136">No entanto, ao usar a reflexão, você não trabalha diretamente com essas classes, a maioria das quais é abstrata (`MustInherit` no Visual Basic).</span><span class="sxs-lookup"><span data-stu-id="72b45-136">However, when you use reflection, you don't work directly with these classes, most of which are abstract (`MustInherit` in Visual Basic).</span></span> <span data-ttu-id="72b45-137">Em vez disso, você trabalha com tipos fornecidos pelo CLR (Common Language Runtime).</span><span class="sxs-lookup"><span data-stu-id="72b45-137">Instead, you work with types provided by the common language runtime (CLR).</span></span>
