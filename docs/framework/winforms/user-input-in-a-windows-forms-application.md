# <a name="user-input-in-a-windows-forms-application"></a><span data-ttu-id="81684-102">Entrada do usuário em um aplicativo do Windows Forms</span><span class="sxs-lookup"><span data-stu-id="81684-102">User Input in a Windows Forms Application</span></span>
<span data-ttu-id="81684-103">No Windows Forms, a entrada do usuário é enviada para aplicativos na forma de mensagens do Windows.</span><span class="sxs-lookup"><span data-stu-id="81684-103">In Windows Forms, user input is sent to applications in the form of Windows messages.</span></span> <span data-ttu-id="81684-104">Uma série de métodos substituíveis processam essas mensagens no nível do aplicativo, formulário e controle.</span><span class="sxs-lookup"><span data-stu-id="81684-104">A series of overridable methods process these messages at the application, form, and control level.</span></span> <span data-ttu-id="81684-105">Quando esses métodos recebem mensagens de mouse e teclado, eles geram eventos que podem ser manipulados para obter informações sobre a entrada de mouse ou de teclado.</span><span class="sxs-lookup"><span data-stu-id="81684-105">When these methods receive mouse and keyboard messages, they raise events that can be handled to get information about the mouse or keyboard input.</span></span> <span data-ttu-id="81684-106">Em muitos casos, os aplicativos do Windows Forms serão capazes de processar todas as entradas do usuário simplesmente manipulando esses eventos.</span><span class="sxs-lookup"><span data-stu-id="81684-106">In many cases, Windows Forms applications will be able to process all user input simply by handling these events.</span></span> <span data-ttu-id="81684-107">Em outros casos, um aplicativo pode precisar substituir um dos métodos que processam mensagens para interceptar uma mensagem específica antes de ela ser recebida pelo aplicativo, formulário ou controle.</span><span class="sxs-lookup"><span data-stu-id="81684-107">In other cases, an application may need to override one of the methods that process messages in order to intercept a particular message before it is received by the application, form, or control.</span></span>

 <span data-ttu-id="81684-109">Todos os controles do Windows Forms herdam um conjunto de eventos relacionados às entradas de mouse e teclado.</span><span class="sxs-lookup"><span data-stu-id="81684-109">All Windows Forms controls inherit a set of events related to mouse and keyboard input.</span></span> <span data-ttu-id="81684-110">Por exemplo, um controle pode manipular o <xref:System.Windows.Forms.Control.KeyPress> evento para determinar o código de caractere de uma chave que foi pressionado ou um controle pode manipular o <xref:System.Windows.Forms.Control.MouseClick> evento para determinar o local do mouse clique em.</span><span class="sxs-lookup"><span data-stu-id="81684-110">For example, a control can handle the <xref:System.Windows.Forms.Control.KeyPress> event to determine the character code of a key that was pressed, or a control can handle the <xref:System.Windows.Forms.Control.MouseClick> event to determine the location of a mouse click.</span></span> <span data-ttu-id="81684-111">Para obter mais informações sobre eventos de mouse e teclado, consulte [Usando Eventos de Teclado](../../../docs/framework/winforms/using-keyboard-events.md) e [Eventos de Mouse no Windows Forms](../../../docs/framework/winforms/mouse-events-in-windows-forms.md).</span><span class="sxs-lookup"><span data-stu-id="81684-111">For more information on the mouse and keyboard events, see [Using Keyboard Events](../../../docs/framework/winforms/using-keyboard-events.md) and [Mouse Events in Windows Forms](../../../docs/framework/winforms/mouse-events-in-windows-forms.md).</span></span>
