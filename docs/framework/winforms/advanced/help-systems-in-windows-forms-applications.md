# <a name="help-systems-in-windows-forms-applications"></a>Sistemas de Ajuda em aplicativos do Windows Forms
Uma das cortesias mais importantes que você, como desenvolvedor de aplicativos, pode fornecer aos seus usuários com é um sistema de Ajuda competente. Esse é o local que eles procuram quando ficam confusos ou desorientados. É fácil fornecer um sistema de Ajuda em um aplicativo baseado no Windows usando o [componente HelpProvider](../controls/helpprovider-component-windows-forms.md).  
  
## <a name="different-types-of-help"></a>Diferentes tipos de ajuda  
 O componente Windows Forms <xref:System.Windows.Forms.HelpProvider> é usado para associar um arquivo de ajuda HTML ajuda 1. x (um arquivo. chm, produzido com o HTML Help Workshop ou um arquivo. htm) com seu aplicativo baseado no Windows. O componente <xref:System.Windows.Forms.HelpProvider> pode ser usado para fornecer ajuda contextual para controles em Windows Forms ou controles específicos. Além disso, o componente <xref:System.Windows.Forms.HelpProvider> pode abrir um arquivo de ajuda para áreas específicas, como a página principal de um sumário, um índice ou uma função de pesquisa. Para obter informações gerais sobre o componente <xref:System.Windows.Forms.HelpProvider>, consulte [visão geral do componente HelpProvider](../controls/helpprovider-component-overview-windows-forms.md). Para obter informações sobre como usar o componente <xref:System.Windows.Forms.HelpProvider> para mostrar a ajuda pop-up no Windows Forms, consulte [como exibir a ajuda pop-up](how-to-display-pop-up-help.md). Para obter informações sobre como usar o componente <xref:System.Windows.Forms.ToolTip> para mostrar a ajuda específica do controle, consulte [ajuda de controle usando dicas de ferramenta](control-help-using-tooltips.md).  
  
 Você pode gerar arquivos da Ajuda HTML 1.x com o HTML Help Workshop. Para obter mais informações sobre a Ajuda HTML, veja o "HTML Help Workshop" ou outros tópicos de "Ajuda HTML" no MSDN.
