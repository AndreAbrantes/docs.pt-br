## <a name="protected-objects"></a><span data-ttu-id="92107-110">Objetos protegidos</span><span class="sxs-lookup"><span data-stu-id="92107-110">Protected objects</span></span>  
 <span data-ttu-id="92107-111">Alguns objetos mantêm o estado de segurança em si.</span><span class="sxs-lookup"><span data-stu-id="92107-111">Some objects hold security state in themselves.</span></span> <span data-ttu-id="92107-112">Esses objetos não devem ser passados para código não confiável, que, em seguida, poderia obter autorização de segurança além de suas próprias permissões.</span><span class="sxs-lookup"><span data-stu-id="92107-112">These objects should not be passed to untrusted code, which would then acquire security authorization beyond its own permissions.</span></span>  
  
 <span data-ttu-id="92107-113">Um exemplo é criar um <xref:System.IO.FileStream> objeto.</span><span class="sxs-lookup"><span data-stu-id="92107-113">One example is creating a <xref:System.IO.FileStream> object.</span></span> <span data-ttu-id="92107-114">O <xref:System.Security.Permissions.FileIOPermission> é exigida no momento da criação e, se for bem-sucedida, o objeto de arquivo é retornado.</span><span class="sxs-lookup"><span data-stu-id="92107-114">The <xref:System.Security.Permissions.FileIOPermission> is demanded at the time of creation and, if it succeeds, the file object is returned.</span></span> <span data-ttu-id="92107-115">No entanto, se essa referência de objeto é passada para código sem permissões de arquivo, o objeto será capaz de ler e gravar este arquivo específico.</span><span class="sxs-lookup"><span data-stu-id="92107-115">However, if this object reference is passed to code without file permissions, the object will be able to read and write to this particular file.</span></span>  
  
 <span data-ttu-id="92107-116">A defesa mais simples para tal objeto é solicitar o mesmo **FileIOPermission** de qualquer código que tenta obter a referência de objeto por meio de um elemento de API público.</span><span class="sxs-lookup"><span data-stu-id="92107-116">The simplest defense for such an object is to demand the same **FileIOPermission** of any code that seeks to get the object reference through a public API element.</span></span>

 <span data-ttu-id="92107-125">Normalmente, o domínio de aplicativo padrão cria o filho domínios de aplicativo com um objeto de controle em cada uma.</span><span class="sxs-lookup"><span
