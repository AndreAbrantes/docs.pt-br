## <a name="walking-the-call-stack"></a><span data-ttu-id="4a741-140">Caminhando pela Pilha de Chamadas</span><span class="sxs-lookup"><span data-stu-id="4a741-140">Walking the Call Stack</span></span>  
 <span data-ttu-id="4a741-141">Para determinar se o código está autorizado a acessar um recurso ou executar uma operação, o sistema de segurança do runtime anda pela pilha de chamadas, comparando as permissões concedidas de cada chamador com a permissão que está sendo exigida.</span><span class="sxs-lookup"><span data-stu-id="4a741-141">To determine whether code is authorized to access a resource or perform an operation, the runtime's security system walks the call stack, comparing the granted permissions of each caller to the permission being demanded.</span></span> <span data-ttu-id="4a741-142">Se qualquer chamador na pilha de chamadas não tiver a permissão exigida, uma exceção de segurança será lançada e o acesso será recusado.</span><span class="sxs-lookup"><span data-stu-id="4a741-142">If any caller in the call stack does not have the demanded permission, a security exception is thrown and access is refused.</span></span> <span data-ttu-id="4a741-143">A caminhada de pilha foi projetada para ajudar a evitar ataques de atração, nos quais o código menos confiável chama código altamente confiável e o usa para executar ações não autorizadas.</span><span class="sxs-lookup"><span data-stu-id="4a741-143">The stack walk is designed to help prevent luring attacks, in which less-trusted code calls highly trusted code and uses it to perform unauthorized actions.</span></span> <span data-ttu-id="4a741-144">Exigir permissões de todos os chamadores em tempo de execução afeta o desempenho, mas é essencial ajudar a proteger o código de atrair ataques por código menos confiável.</span><span class="sxs-lookup"><span data-stu-id="4a741-144">Demanding permissions of all callers at run time affects performance, but it is essential to help protect code from luring attacks by less-trusted code.</span></span> <span data-ttu-id="4a741-145">Para otimizar o desempenho, você pode fazer com que seu código realize menos caminhadas de pilha;
