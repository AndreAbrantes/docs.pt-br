 <span data-ttu-id="3e822-141">Para determinar se o código está autorizado a acessar um recurso ou executar uma operação, o sistema de segurança do tempo de execução percorre a pilha de chamadas, comparando as permissões concedidas de cada chamador com a permissão está sendo exigida.</span><span class="sxs-lookup"><span data-stu-id="3e822-141">To determine whether code is authorized to access a resource or perform an operation, the runtime's security system walks the call stack, comparing the granted permissions of each caller to the permission being demanded.</span></span> <span data-ttu-id="3e822-142">Se algum chamador na pilha de chamadas não tem a permissão exigida, será gerada uma exceção de segurança e acesso será recusado.</span><span class="sxs-lookup"><span data-stu-id="3e822-142">If any caller in the call stack does not have the demanded permission, a security exception is thrown and access is refused.</span></span> <span data-ttu-id="3e822-143">A movimentação da pilha é projetada para ajudar a evitar ataques de atração, em que menos confiável código chama código altamente confiável e o utiliza para executar ações não autorizadas.</span><span class="sxs-lookup"><span data-stu-id="3e822-143">The stack walk is designed to help prevent luring attacks, in which less-trusted code calls highly trusted code and uses it to perform unauthorized actions.</span></span> <span data-ttu-id="3e822-144">Exigem permissões de todos os chamadores em tempo de execução afeta o desempenho, mas é essencial para ajudar a proteger o código contra ataques de atração pelo código menos confiável.</span><span class="sxs-lookup"><span data-stu-id="3e822-144">Demanding permissions of all callers at run time affects performance, but it is essential to help protect code from luring attacks by less-trusted code.</span></span> <span data-ttu-id="3e822-145">Para otimizar o desempenho, você pode ter seu código executar menos movimentações de pilha;
