## <a name="using-managed-wrapper-classes"></a><span data-ttu-id="02b92-178">Usando classes de wrapper gerenciadas</span><span class="sxs-lookup"><span data-stu-id="02b92-178">Using Managed Wrapper Classes</span></span>

<span data-ttu-id="02b92-179">A maioria dos aplicativos e componentes (exceto bibliotecas seguras) não deve chamar diretamente código não gerenciado.</span><span class="sxs-lookup"><span data-stu-id="02b92-179">Most applications and components (except secure libraries) should not directly call unmanaged code.</span></span> <span data-ttu-id="02b92-180">Há várias razões para isso.</span><span class="sxs-lookup"><span data-stu-id="02b92-180">There are several reasons for this.</span></span> <span data-ttu-id="02b92-181">Se o código chamar código não gerenciado diretamente, ele não será permitido ser executado em muitas circunstâncias, porque o código deve ser concedido um alto nível de confiança para chamar código nativo.</span><span class="sxs-lookup"><span data-stu-id="02b92-181">If code calls unmanaged code directly, it will not be allowed to run in many circumstances because code must be granted a high level of trust to call native code.</span></span> <span data-ttu-id="02b92-182">Se a política for modificada para permitir que tal aplicativo seja executado, ele pode enfraquecer significativamente a segurança do sistema, deixando o aplicativo livre para realizar quase qualquer operação.</span><span class="sxs-lookup"><span data-stu-id="02b92-182">If policy is modified to allow such an application to run, it can significantly weaken the security of the system, leaving the application free to perform almost any operation.</span></span>
