# <a name="extending-windows-workflow-foundation"></a>Estendendo Windows Workflow Foundation
A seção a seguir descreve como estender o Windows Workflow Foundation (WF) com as atividades personalizados e designers em ambientes rehosted fora do Visual Studio 2010.

## <a name="in-this-section"></a>Nesta seção
 [Personalizando a experiência de Design de fluxo de trabalho](../../../docs/framework/windows-workflow-foundation/customizing-the-workflow-design-experience.md) indica como os cenários para criar atividades personalizados e para rehosting o [!INCLUDE[wfd1](../../../includes/wfd1-md.md)] simplificados foram bastante em [!INCLUDE[netfx40_short](../../../includes/netfx40-short-md.md)]. Desenvolvimento e implantação agora são mais fáceis e mais flexível porque o novo modelo de programação de atividade designer baseia-se ao Windows Presentation Foundation (WPF).
