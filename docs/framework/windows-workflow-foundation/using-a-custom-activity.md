# <a name="using-a-custom-activity"></a><span data-ttu-id="95229-102">Usando uma atividade personalizada</span><span class="sxs-lookup"><span data-stu-id="95229-102">Using a custom activity</span></span>

<span data-ttu-id="95229-103">As atividades que derivam de <xref:System.Activities.Activity> ou suas subclasses podem ser compostas em fluxos de trabalho maiores ou criadas diretamente no código.</span><span class="sxs-lookup"><span data-stu-id="95229-103">Activities that derive from <xref:System.Activities.Activity> or its subclasses can be composed into larger workflows, or created directly in code.</span></span> <span data-ttu-id="95229-104">Este tópico descreve como usar atividades personalizadas em fluxos de trabalho criados no código ou no designer.</span><span class="sxs-lookup"><span data-stu-id="95229-104">This topic describes how to use custom activities in workflows created either in code or in the designer.</span></span>  
  
> [!NOTE]
> <span data-ttu-id="95229-105">As atividades personalizadas podem ser usadas no mesmo projeto no qual elas são definidas, desde que a atividade personalizada e a atividade que as utiliza sejam compiladas (ou seja, carregadas por um tipo de instanciação gerado pelo processo de compilação) se a atividade de referência for carregada dinamicamente (por exemplo, usando ActivityXamlServices encontraram), o assembly referenciado deverá ser colocado em um projeto diferente ou o XAML gerado pelo designer precisa ser editado manualmente para habilitar isso.</span><span class="sxs-lookup"><span data-stu-id="95229-105">Custom activities can be used in the same project in which they are defined, as long as both the custom activity and the activity that uses it are compiled (i.e. loaded by an instantiating type generated by the build process) If the referencing activity is loaded dynamically (e.g. using ActivityXAMLServices), then the referenced assembly should be placed in a different project, or the designer-generated XAML needs to be hand-edited to enable this.</span></span>  
  
#### <a name="using-a-custom-activity-to-a-workflow-project"></a><span data-ttu-id="95229-106">Usando uma atividade personalizada para um projeto de fluxo de trabalho</span><span class="sxs-lookup"><span data-stu-id="95229-106">Using a custom activity to a workflow project</span></span>  
  
1. <span data-ttu-id="95229-107">Adicione uma referência do projeto de host para o projeto de biblioteca de atividade que contém a atividade personalizada.</span><span class="sxs-lookup"><span data-stu-id="95229-107">Add a reference from the host project to the activity library project containing the custom activity.</span></span>  
  
2. <span data-ttu-id="95229-108">Compile a solução.</span><span class="sxs-lookup"><span data-stu-id="95229-108">Build the solution.</span></span>  
  
3. <span data-ttu-id="95229-109">Para usar a atividade personalizada no designer, localize a atividade personalizada na caixa de ferramentas e arraste a atividade na superfície do designer.</span><span class="sxs-lookup"><span data-stu-id="95229-109">To use the custom activity in the designer, locate the custom activity in the toolbox, and drag the activity onto the designer surface.</span></span>  
  
4. <span data-ttu-id="95229-110">Para usar a atividade personalizada no código, adicione uma instrução Using que se refere ao projeto de atividade personalizada, e transmita uma nova instância da atividade para <xref:System.Activities.WorkflowInvoker.Invoke%2A>.</span><span class="sxs-lookup"><span data-stu-id="95229-110">To use the custom activity in code, add a Using statement that refers to the custom activity project, and pass a new instance of the activity to <xref:System.Activities.WorkflowInvoker.Invoke%2A>.</span></span>
