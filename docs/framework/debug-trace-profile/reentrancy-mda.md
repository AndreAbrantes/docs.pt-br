# <a name="reentrancy-mda"></a><span data-ttu-id="42687-103">MDA reentrancy</span><span class="sxs-lookup"><span data-stu-id="42687-103">reentrancy MDA</span></span>

<span data-ttu-id="42687-104">O MDA (Assistente de Depuração Gerenciado) de `reentrancy` é ativado quando é feita uma tentativa de transição de código nativo para gerenciado em casos nos quais um comutador anterior do código gerenciado para nativo não foi executado por meio de uma transição ordenada.</span><span class="sxs-lookup"><span data-stu-id="42687-104">The `reentrancy` managed debugging assistant (MDA) is activated when an attempt is made to transition from native to managed code in cases where a prior switch from managed to native code was not performed through an orderly transition.</span></span>  
  
## <a name="symptoms"></a><span data-ttu-id="42687-105">Sintomas</span><span class="sxs-lookup"><span data-stu-id="42687-105">Symptoms</span></span>  

 <span data-ttu-id="42687-106">O heap do objeto está corrompido ou outros erros graves estão ocorrendo durante a transição de código nativo para gerenciado.</span><span class="sxs-lookup"><span data-stu-id="42687-106">The object heap is corrupted or other serious errors are occurring when transitioning from native to managed code.</span></span>  
  
 <span data-ttu-id="42687-107">Threads que alternam entre o código nativo e o gerenciado em qualquer direção devem realizar uma transição ordenada.</span><span class="sxs-lookup"><span data-stu-id="42687-107">Threads that switch between native and managed code in either direction must perform an orderly transition.</span></span> <span data-ttu-id="42687-108">No entanto, certos pontos de extensibilidade de nível inferior no sistema operacional, tais como o manipulador de exceção em vetor, permitem mudanças de código gerenciado para código nativo sem realizar uma transição ordenada.</span><span class="sxs-lookup"><span data-stu-id="42687-108">However, certain low-level extensibility points in the operating system, such as the vectored exception handler, allow switches from managed to native code without performing an orderly transition.</span></span>  <span data-ttu-id="42687-109">Essas opções estão sob controle do sistema operacional, em vez de sob o controle do CLR (Common Language Runtime).</span><span class="sxs-lookup"><span data-stu-id="42687-109">These switches are under operating system control, rather than under common language runtime (CLR) control.</span></span>  <span data-ttu-id="42687-110">Qualquer código nativo executado dentro desses pontos de extensibilidade deve evitar retornar a chamada para o código gerenciado.</span><span class="sxs-lookup"><span data-stu-id="42687-110">Any native code that executes inside these extensibility points must avoid calling back into managed code.</span></span>  
  
## <a name="cause"></a><span data-ttu-id="42687-111">Causa</span><span class="sxs-lookup"><span data-stu-id="42687-111">Cause</span></span>  

 <span data-ttu-id="42687-112">Um ponto de extensibilidade do sistema de operacional de baixo nível, tal como o manipulador de exceção em vetor, foi ativado durante a execução de código gerenciado.</span><span class="sxs-lookup"><span data-stu-id="42687-112">A low-level operating system extensibility point, such as the vectored exception handler, has activated while executing managed code.</span></span>  <span data-ttu-id="42687-113">O código do aplicativo que é invocado por meio desse ponto de extensibilidade está tentando retornar a chamada para o código gerenciado.</span><span class="sxs-lookup"><span data-stu-id="42687-113">The application code that is invoked through that extensibility point is attempting to call back into managed code.</span></span>  
  
 <span data-ttu-id="42687-114">Esse problema é sempre causado pelo código do aplicativo.</span><span class="sxs-lookup"><span data-stu-id="42687-114">This problem is always caused by application code.</span></span>  
  
## <a name="resolution"></a><span data-ttu-id="42687-115">Resolução</span><span class="sxs-lookup"><span data-stu-id="42687-115">Resolution</span></span>  

 <span data-ttu-id="42687-116">Examine o rastreamento de pilha do thread que ativou esse MDA.</span><span class="sxs-lookup"><span data-stu-id="42687-116">Examine the stack trace for the thread that has activated this MDA.</span></span>  <span data-ttu-id="42687-117">O thread está tentando fazer uma chamada ilegal para o código gerenciado.</span><span class="sxs-lookup"><span data-stu-id="42687-117">The thread is attempting to illegally call into managed code.</span></span>  <span data-ttu-id="42687-118">O rastreamento de pilha deve revelar o código do aplicativo usando esse ponto de extensibilidade, o código de sistema operacional que fornece esse ponto de extensibilidade e o código gerenciado que foi interrompido pelo ponto de extensibilidade.</span><span class="sxs-lookup"><span data-stu-id="42687-118">The stack trace should reveal the application code using this extensibility point, the operating system code that provides this extensibility point, and the managed code that was interrupted by the extensibility point.</span></span>  
  
 <span data-ttu-id="42687-119">Por exemplo, você verá o MDA ativado em uma tentativa de chamar código gerenciado de dentro de um manipulador de exceção em vetor.</span><span class="sxs-lookup"><span data-stu-id="42687-119">For example, you will see the MDA activated in an attempt to call managed code from inside a vectored exception handler.</span></span>  <span data-ttu-id="42687-120">Na pilha, você verá o código de tratamento de exceção do sistema operacional e algum código gerenciado disparando uma exceção, tal como uma <xref:System.DivideByZeroException> ou uma <xref:System.AccessViolationException>.</span><span class="sxs-lookup"><span data-stu-id="42687-120">On the stack you will see the operating system exception handling code and some managed code triggering an exception such as a <xref:System.DivideByZeroException> or an <xref:System.AccessViolationException>.</span></span>  
  
 <span data-ttu-id="42687-121">Neste exemplo, a resolução correta é implementar o manipulador de exceção em vetor completamente em código não gerenciado.</span><span class="sxs-lookup"><span data-stu-id="42687-121">In this example, the correct resolution is to implement the vectored exception handler completely in unmanaged code.</span></span>

```csharp
public delegate int ExceptionHandler(IntPtr ptrExceptionInfo);

{  
    public static ExceptionHandler keepAlive;  
  
    [System.Runtime.InteropServices.DllImport("kernel32", ExactSpelling=true,
        CharSet=System.Runtime.InteropServices.CharSet.Auto)]  
    public static extern IntPtr AddVectoredExceptionHandler(int bFirst,
        ExceptionHandler handler);  
  
    static int MyHandler(IntPtr ptrExceptionInfo)
    {  
        // EXCEPTION_CONTINUE_SEARCH  
        return 0;  
    }  
    void Run() {}  
  
    static void Main()
    {  
        keepAlive = new ExceptionHandler(Reenter.MyHandler);  
        IntPtr ret = AddVectoredExceptionHandler(1, keepAlive);  
        try
        {  
            // Dispatch on null should AV.  
            Reenter r = null;
            r.Run();  
        }
        catch { }  
    }  
}  
```
