## <a name="what-is-the-net-framework"></a><span data-ttu-id="cb042-105">O que é o .NET Framework?</span><span class="sxs-lookup"><span data-stu-id="cb042-105">What is the .NET Framework?</span></span>

<span data-ttu-id="cb042-106">O .NET Framework é um ambiente de execução gerenciado que oferece uma variedade de serviços aos aplicativos em execução.</span><span class="sxs-lookup"><span data-stu-id="cb042-106">The .NET Framework is a managed execution environment that provides a variety of services to its running apps.</span></span> <span data-ttu-id="cb042-107">Ele consiste em dois componentes principais: o CLR (Common Language Runtime), o mecanismo de execução que manipula aplicativos em execução, e a biblioteca de classes .NET Framework, que oferece uma biblioteca de códigos testados e reutilizáveis que os desenvolvedores podem chamar de seus próprios aplicativos.</span><span class="sxs-lookup"><span data-stu-id="cb042-107">It consists of two major components: the common language runtime (CLR), which is the execution engine that handles running apps, and the .NET Framework Class Library, which provides a library of tested, reusable code that developers can call from their own apps.</span></span> <span data-ttu-id="cb042-108">Entre os serviços que o .NET Framework fornece aos aplicativos em execução estão os seguintes:</span><span class="sxs-lookup"><span data-stu-id="cb042-108">The services that the .NET Framework provides to running apps include the following:</span></span>

- <span data-ttu-id="cb042-109">Gerenciamento de memória.</span><span class="sxs-lookup"><span data-stu-id="cb042-109">Memory management.</span></span> <span data-ttu-id="cb042-110">Em muitas linguagens de programação, os programadores são responsáveis por alocar e liberar memória e por identificar o tempo de vida do objeto.</span><span class="sxs-lookup"><span data-stu-id="cb042-110">In many programming languages, programmers are responsible for allocating and releasing memory and for handling object lifetimes.</span></span> <span data-ttu-id="cb042-111">Em aplicativos do .NET Framework, o CLR fornece esses serviços em nome do aplicativo.</span><span class="sxs-lookup"><span data-stu-id="cb042-111">In .NET Framework apps, the CLR provides these services on behalf of the app.</span></span>

- <span data-ttu-id="cb042-112">Um Common Type System.</span><span class="sxs-lookup"><span data-stu-id="cb042-112">A common type system.</span></span> <span data-ttu-id="cb042-113">Em linguagens de programação tradicionais, os tipos básicos são definidos pelo compilador, que complica a interoperabilidade entre linguagens.</span><span class="sxs-lookup"><span data-stu-id="cb042-113">In traditional programming languages, basic types are defined by the compiler, which complicates cross-language interoperability.</span></span> <span data-ttu-id="cb042-114">No .NET Framework, os tipos básicos são definidos pelo sistema de tipos do .NET Framework e são comuns a todas as linguagens que segmentam o .NET Framework.</span><span class="sxs-lookup"><span data-stu-id="cb042-114">In the .NET Framework, basic types are defined by the .NET Framework type system and are common to all languages that target the .NET Framework.</span></span>

- <span data-ttu-id="cb042-115">Uma biblioteca de classes abrangente.</span><span class="sxs-lookup"><span data-stu-id="cb042-115">An extensive class library.</span></span> <span data-ttu-id="cb042-116">Em vez de precisar gravar grandes volumes de código para manipular operações de programação comuns de baixo nível, os programadores usam uma biblioteca de tipos facilmente acessível e seus membros da biblioteca de classes .NET Framework.</span><span class="sxs-lookup"><span data-stu-id="cb042-116">Instead of having to write vast amounts of code to handle common low-level programming operations, programmers use a readily accessible library of types and their members from the .NET Framework Class Library.</span></span>

- <span data-ttu-id="cb042-117">Estruturas e tecnologias de desenvolvimento.</span><span class="sxs-lookup"><span data-stu-id="cb042-117">Development frameworks and technologies.</span></span> <span data-ttu-id="cb042-118">O .NET Framework inclui bibliotecas para áreas específicas do desenvolvimento de aplicativos, como o ASP.NET para aplicativos Web, o ADO.NET para acesso a dados e o Windows Communication Foundation para aplicativos orientados a serviços.</span><span class="sxs-lookup"><span data-stu-id="cb042-118">The .NET Framework includes libraries for specific areas of app development, such as ASP.NET for web apps, ADO.NET for data access, and Windows Communication Foundation for service-oriented apps.</span></span>

- <span data-ttu-id="cb042-119">Interoperabilidade da linguagem.</span><span class="sxs-lookup"><span data-stu-id="cb042-119">Language interoperability.</span></span> <span data-ttu-id="cb042-120">Compiladores de linguagens que direcionam o .NET Framework emitem um código intermediário chamado de CIL (Common Intermediate Language), que, por sua vez, é compilado no tempo de execução pelo Common Language Runtime.</span><span class="sxs-lookup"><span data-stu-id="cb042-120">Language compilers that target the .NET Framework emit an intermediate code named Common Intermediate Language (CIL), which, in turn, is compiled at runtime by the common language runtime.</span></span> <span data-ttu-id="cb042-121">Com esse recurso, as rotinas gravadas em uma linguagem são acessíveis a outras linguagens, e os programadores privilegiam a criação de aplicativos em suas linguagens preferidas.</span><span class="sxs-lookup"><span data-stu-id="cb042-121">With this feature, routines written in one language are accessible to other languages, and programmers focus on creating apps in their preferred languages.</span></span>

- <span data-ttu-id="cb042-122">Compatibilidade de versões.</span><span class="sxs-lookup"><span data-stu-id="cb042-122">Version compatibility.</span></span> <span data-ttu-id="cb042-123">Com raras exceções, os aplicativos desenvolvidos usando uma versão específica do .NET Framework são executados sem modificação em uma versão posterior.</span><span class="sxs-lookup"><span data-stu-id="cb042-123">With rare exceptions, apps that are developed by using a particular version of the .NET Framework run without modification on a later version.</span></span>

- <span data-ttu-id="cb042-124">Execução lado a lado.</span><span class="sxs-lookup"><span data-stu-id="cb042-124">Side-by-side execution.</span></span> <span data-ttu-id="cb042-125">O .NET Framework ajuda a resolver conflitos de versão permitindo que várias versões do CLR existam no mesmo computador.</span><span class="sxs-lookup"><span data-stu-id="cb042-125">The .NET Framework helps resolve version conflicts by allowing multiple versions of the common language runtime to exist on the same computer.</span></span> <span data-ttu-id="cb042-126">Isso significa que várias versões dos aplicativos podem coexistir e que um aplicativo pode ser executado na versão do .NET Framework com a qual foi compilada.</span><span class="sxs-lookup"><span data-stu-id="cb042-126">This means that multiple versions of apps can coexist and that an app can run on the version of the .NET Framework with which it was built.</span></span> <span data-ttu-id="cb042-127">A execução lado a lado se aplica aos grupos de versão do .NET Framework 1.0/1.1, 2.0/3.0/3.5 e 4/4.5.x/4.6.x/4.7.</span><span class="sxs-lookup"><span data-stu-id="cb042-127">Side-by-side execution applies to the .NET Framework version groups 1.0/1.1, 2.0/3.0/3.5, and 4/4.5.x/4.6.x/4.7.x.</span></span>

- <span data-ttu-id="cb042-128">Multiplataforma.</span><span class="sxs-lookup"><span data-stu-id="cb042-128">Multitargeting.</span></span> <span data-ttu-id="cb042-129">Ao direcionar o [.NET Standard](~/docs/standard/net-standard.md), os desenvolvedores criam assemblies que funcionam em várias plataformas .NET Framework, como o Windows 7, Windows 8, Windows 8.1, Windows 10, Windows Phone e Xbox 360.</span><span class="sxs-lookup"><span data-stu-id="cb042-129">By targeting [.NET Standard](~/docs/standard/net-standard.md), developers create assemblies that work on multiple .NET Framework platforms, such as Windows 7, Windows 8, Windows 8.1, Windows 10, Windows Phone, and Xbox 360.</span></span>
