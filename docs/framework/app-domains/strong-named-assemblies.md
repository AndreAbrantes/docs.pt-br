# <a name="strong-named-assemblies"></a><span data-ttu-id="20ea3-102">Assemblies de nomes fortes</span><span class="sxs-lookup"><span data-stu-id="20ea3-102">Strong-Named Assemblies</span></span>
<span data-ttu-id="20ea3-103">Uma nomeação forte na assembly cria uma única identidade para assembly e pode evitar conflitos de assembly.</span><span class="sxs-lookup"><span data-stu-id="20ea3-103">Strong-naming an assembly creates a unique identity for the assembly, and can prevent assembly conflicts.</span></span>  
  
## <a name="what-makes-a-strong-named-assembly"></a><span data-ttu-id="20ea3-104">Como é criado um assembly de nomeação forte?</span><span class="sxs-lookup"><span data-stu-id="20ea3-104">What makes a strong-named assembly?</span></span>  
 <span data-ttu-id="20ea3-105">Um assembly de nome forte é gerado por meio da utilização de uma chave privada que corresponde a uma chave pública distribuída com o assembly e por ele mesmo.</span><span class="sxs-lookup"><span data-stu-id="20ea3-105">A strong named assembly is generated by using the private key that corresponds to the public key distributed with the assembly, and the assembly itself.</span></span> <span data-ttu-id="20ea3-106">O assembly inclui o manifesto assembly que contém os nomes e os hashes de todos os arquivos que compõem o assembly.</span><span class="sxs-lookup"><span data-stu-id="20ea3-106">The assembly includes the assembly manifest, which contains the names and hashes of all the files that make up the assembly.</span></span> <span data-ttu-id="20ea3-107">Assemblies que têm o mesmo nome forte devem ser idênticos.</span><span class="sxs-lookup"><span data-stu-id="20ea3-107">Assemblies that have the same strong name should be identical.</span></span>  
  
 <span data-ttu-id="20ea3-108">Você pode usar nomes fortes nos assemblies usando o Visual Studio ou uma ferramenta de linha de comando.</span><span class="sxs-lookup"><span data-stu-id="20ea3-108">You can strong-name assemblies by using Visual Studio or a command-line tool.</span></span> <span data-ttu-id="20ea3-109">Para saber mais, confira [Como assinar um assembly com um nome forte](../../../docs/framework/app-domains/how-to-sign-an-assembly-with-a-strong-name.md) ou [Sn.exe (Ferramenta de Nome Forte)](../../../docs/framework/tools/sn-exe-strong-name-tool.md).</span><span class="sxs-lookup"><span data-stu-id="20ea3-109">For more information, see [How to: Sign an Assembly with a Strong Name](../../../docs/framework/app-domains/how-to-sign-an-assembly-with-a-strong-name.md) or [Sn.exe (Strong Name Tool)](../../../docs/framework/tools/sn-exe-strong-name-tool.md).</span></span>  
  
 <span data-ttu-id="20ea3-110">Quando um assembly de nome forte é criado, ele contém o nome de texto simples do assembly, o número de versão, informação cultural opcional, uma assinatura digital e uma chave pública que corresponde a uma chave privada utilizada para assinar.</span><span class="sxs-lookup"><span data-stu-id="20ea3-110">When a strong-named assembly is created, it contains the simple text name of the assembly, the version number, optional culture information, a digital signature, and the public key that corresponds to the private key used for signing.</span></span>  
  
> [!WARNING]
>  <span data-ttu-id="20ea3-111">Por segurança, não confie em nomes fortes.</span><span class="sxs-lookup"><span data-stu-id="20ea3-111">Do not rely on strong names for security.</span></span> <span data-ttu-id="20ea3-112">Eles apenas fornecem uma identidade exclusiva.</span><span class="sxs-lookup"><span data-stu-id="20ea3-112">They provide a unique identity only.</span></span>  
  
## <a name="why-strong-name-your-assemblies"></a><span data-ttu-id="20ea3-113">Por que usar um nome forte em seus assemblies?</span><span class="sxs-lookup"><span data-stu-id="20ea3-113">Why strong-name your assemblies?</span></span>  
 <span data-ttu-id="20ea3-114">Quando você usa um assembly de nome forte como referência, espera obter determinados benefícios, como controle de versão e proteção de nomenclatura.</span><span class="sxs-lookup"><span data-stu-id="20ea3-114">When you reference a strong-named assembly, you can expect certain benefits, such as versioning and naming protection.</span></span> <span data-ttu-id="20ea3-115">Assemblies com nomes fortes podem ser instalados no Cache de Assembly Global, que é necessário para habilitar alguns cenários.</span><span class="sxs-lookup"><span data-stu-id="20ea3-115">Strong-named assemblies can be installed in the Global Assembly Cache, which is required to enable some scenarios.</span></span>  
  
 <span data-ttu-id="20ea3-116">Assemblies de nome forte são úteis nos seguintes cenários:</span><span class="sxs-lookup"><span data-stu-id="20ea3-116">Strong-named assemblies are useful in the following scenarios:</span></span>  
  
-   <span data-ttu-id="20ea3-117">Você deseja habilitar seus assemblies a serem referenciados por assemblies de nome forte ou para dar acesso `friend` aos seus assemblies de outros assemblies de nome forte.</span><span class="sxs-lookup"><span data-stu-id="20ea3-117">You want to enable your assemblies to be referenced by strong-named assemblies, or you want to give `friend` access to your assemblies from other strong-named assemblies.</span></span>  
  
-   <span data-ttu-id="20ea3-118">Um aplicativo precisa acessar versões diferentes do mesmo assembly.</span><span class="sxs-lookup"><span data-stu-id="20ea3-118">An app needs access to different versions of the same assembly.</span></span> <span data-ttu-id="20ea3-119">Isso significa que você precisa de versões diferentes de um assembly para carregar lado a lado no mesmo domínio de aplicativo sem conflitos.</span><span class="sxs-lookup"><span data-stu-id="20ea3-119">This means  you need different versions of an assembly to load side by side in the same app domain without conflict.</span></span> <span data-ttu-id="20ea3-120">Por exemplo, se extensões diferentes de uma API existirem em assemblies que tenham o mesmo nome simples, nomeações fortes fornecem uma identidade única para cada versão de assembly.</span><span class="sxs-lookup"><span data-stu-id="20ea3-120">For example, if different extensions of an API exist in assemblies that have the same simple name, strong-naming provides a unique identity for each version of the assembly.</span></span>  
  
-   <span data-ttu-id="20ea3-121">Você não deseja afetar negativamente o desempenho de aplicativos usando o assembly, por isso assembly deve ser neutros quanto ao domínio.</span><span class="sxs-lookup"><span data-stu-id="20ea3-121">You do not want to negatively affect performance of apps using your assembly, so you want the assembly to be domain neutral.</span></span> <span data-ttu-id="20ea3-122">Isso exige uma nomeação forte porque um assembly com neutro com relação ao domínio deve ser instalado no cache de assembly global.</span><span class="sxs-lookup"><span data-stu-id="20ea3-122">This requires strong-naming because a domain-neutral assembly must be installed in the global assembly cache.</span></span>  
  
-   <span data-ttu-id="20ea3-123">Quando você deseja centralizar o serviço para o aplicativo aplicando políticas de versão, isso significa que o assembly deve ser instalado no cache de assembly global.</span><span class="sxs-lookup"><span data-stu-id="20ea3-123">When you want to centralize servicing for your app by applying publisher policy, which means the assembly must be installed in the  global assembly cache.</span></span>  
  
 <span data-ttu-id="20ea3-124">Se você é um desenvolvedor de código aberto e deseja os benefícios da identidade de um assembly de nome forte, considere inserir na chave privada um assembly no sistema de controle de fonte.</span><span class="sxs-lookup"><span data-stu-id="20ea3-124">If you are an open-source developer and you want the identity benefits of a strong-named assembly, consider checking in the private key associated with an assembly into your source control system.</span></span>
