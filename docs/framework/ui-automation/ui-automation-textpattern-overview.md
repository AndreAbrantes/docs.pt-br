[!INCLUDE[TLA#tla_tsf](../../../includes/tlasharptla-tsf-md.md)] <span data-ttu-id="51c39-117">é uma estrutura de sistema simples e escalonável que permite que os serviços de linguagem natural e na área de trabalho e em aplicativos de entrada de texto avançada.</span><span class="sxs-lookup"><span data-stu-id="51c39-117">is a simple and scalable system framework that enables natural language services and advanced text input on the desktop and within applications.</span></span> <span data-ttu-id="51c39-118">Além de fornecer interfaces para aplicativos para expor seu repositório de texto também suporta metadados para esse armazenamento de texto.</span><span class="sxs-lookup"><span data-stu-id="51c39-118">In addition to providing interfaces for applications to expose their text store it also supports metadata for that text store.</span></span>

<span data-ttu-id="51c39-119">No entanto, [!INCLUDE[TLA2#tla_tsf](../../../includes/tla2sharptla-tsf-md.md)] foi projetado para aplicativos que precisam para inserir dados de entrada em cenários sensíveis ao contexto enquanto <xref:System.Windows.Automation.TextPattern> é uma solução somente leitura (com a alternativa limitada indicada acima) deve fornecer acesso otimizado a um armazenamento de texto para leitores de tela e dispositivos de Braille.</span><span class="sxs-lookup"><span data-stu-id="51c39-119">However, [!INCLUDE[TLA2#tla_tsf](../../../includes/tla2sharptla-tsf-md.md)] was designed for applications that need to inject input into context-aware scenarios whereas <xref:System.Windows.Automation.TextPattern> is a read-only solution (with the limited workaround noted above) meant to provide optimized access to a text store for screen-readers and Braille devices.</span></span>
