# <a name="iclrtaskyieldtask-method"></a>Método ICLRTask::YieldTask
Solicita que o Common Language Runtime (CLR) separe a tarefa que a instância [ICLRTask](iclrtask-interface.md) atual representa e disponibilize o tempo do processador para outras tarefas.  
  
## <a name="syntax"></a>Sintaxe  
  
```cpp  
HRESULT YieldTask ();  
```  
  
## <a name="return-value"></a>Valor retornado  
  
|HRESULT|Description|  
|-------------|-----------------|  
|S_OK|`YieldTask`retornado com êxito.|  
|HOST_E_CLRNOTAVAILABLE|O CLR não foi carregado em um processo ou o CLR está em um estado no qual não pode executar código gerenciado ou processar a chamada com êxito.|  
|HOST_E_TIMEOUT|A chamada atingiu o tempo limite.|  
|HOST_E_NOT_OWNER|O chamador não possui o bloqueio.|  
|HOST_E_ABANDONED|Um evento foi cancelado enquanto um thread ou uma fibra bloqueada estava esperando.|  
|E_FAIL|Ocorreu uma falha catastrófica desconhecida. Quando um método retorna E_FAIL, o CLR não é mais utilizável no processo. As chamadas subsequentes para métodos de hospedagem retornam HOST_E_CLRNOTAVAILABLE.|  
  
## <a name="remarks"></a>Comentários  
 Um host chama `YieldTask` para solicitar recursos do processador para outras tarefas ou processos. Esse método destina-se principalmente a permitir o código de execução longa para desistir do tempo de CPU. O tempo de execução tenta colocar a tarefa que a `ICLRTask` instância atual representa em um estado em que pode gerar tempo de processamento, mas não garante o sucesso.
