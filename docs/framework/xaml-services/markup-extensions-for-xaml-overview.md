 <span data-ttu-id="e505b-240">Se você estiver trabalhando com um fluxo do nó XAML no caminho de carga, o uso de uma extensão de marcação aparece no fluxo do nó como um objeto.</span><span class="sxs-lookup"><span data-stu-id="e505b-240">If you are working with a XAML node stream on the load path, a markup extension usage appears in the node stream as an object.</span></span>  
  
 <span data-ttu-id="e505b-241">Se o uso da extensão de marcação usa argumentos posicionais, ele é representado como um objeto de início com um valor de inicialização.</span><span class="sxs-lookup"><span data-stu-id="e505b-241">If the markup extension usage uses positional arguments, it is represented as a start object with an initialization value.</span></span> <span data-ttu-id="e505b-242">Como uma representação de texto aproximada, fluxo do nó é semelhante à seguinte:</span><span class="sxs-lookup"><span data-stu-id="e505b-242">As a rough text representation, the node stream resembles the following:</span></span>

 <span data-ttu-id="e505b-244">`StartMember` (nome do <xref:System.Xaml.XamlMember> é `_InitializationText`)</span><span class="sxs-lookup"><span data-stu-id="e505b-244">`StartMember` (name of the <xref:System.Xaml.XamlMember> is `_InitializationText`)</span></span>

 `EndMember`  
  
 `EndObject`  
  
 <span data-ttu-id="e505b-246">Um uso de extensão de marcação com argumentos nomeados é representado como um objeto com membros dos nomes de relevantes, cada conjunto com valores de cadeia de caracteres de texto.</span><span class="sxs-lookup"><span data-stu-id="e505b-246">A markup extension usage with named arguments is represented as an object with members of the relevant names, each set with text string values.</span></span>  
  
 <span data-ttu-id="e505b-247">Na verdade, invocar o `ProvideValue` a implementação de uma extensão de marcação exige o contexto do esquema XAML porque que requer o mapeamento de tipo e criando uma instância de tipo de suporte de extensão de marcação.</span><span class="sxs-lookup"><span data-stu-id="e505b-247">Actually invoking the `ProvideValue` implementation of a markup extension requires the XAML schema context because that requires type-mapping and creating a markup extension support type instance.</span></span> <span data-ttu-id="e505b-248">Esse é um motivo por que usos de extensão de marcação são preservados dessa maneira nos fluxos de nó de serviços XAML do .NET Framework padrão – a parte do leitor de um caminho de carga geralmente não tem o contexto de esquema XAML necessário disponível.</span><span class="sxs-lookup"><span data-stu-id="e505b-248">This is one reason why markup extension usages are preserved this way in the default .NET Framework XAML Services node streams -  the reader part of a load path often does not have the necessary XAML schema context available.</span></span>  
  
 <span data-ttu-id="e505b-249">Se você estiver trabalhando com um fluxo do nó XAML em Salvar caminho, geralmente não há nada presente em uma representação de gráfico de objeto que pode informar que o markup extension usage and a `ProvideValue` result.</span></span> <span data-ttu-id="e505b-250">Entrada de XAML em cenários que deve persistir usos de extensão de marcação para o ciclo, enquanto class="sxs-lookup"><span data-stu-id="e505b-250">Scenarios that need to persist markup extension usages for round-tripping while also capturing other changes in the object graph must devise their own
