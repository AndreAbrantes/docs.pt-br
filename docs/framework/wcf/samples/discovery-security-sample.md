## <a name="secure-channel-factory"></a><span data-ttu-id="713dd-116">Fábrica de canais seguros</span><span class="sxs-lookup"><span data-stu-id="713dd-116">Secure Channel Factory</span></span>  
 <span data-ttu-id="713dd-117">A fábrica de canais seguros cria saída ou canais duplex que adicionar uma assinatura compact para cabeçalhos de mensagem.</span><span class="sxs-lookup"><span data-stu-id="713dd-117">The secure channel factory creates output or duplex channels that add a compact signature to message headers.</span></span> <span data-ttu-id="713dd-118">Para manter as mensagens tão pequena quanto possível o formato de assinatura compact é usado.</span><span class="sxs-lookup"><span data-stu-id="713dd-118">To keep messages as small as possible the compact signature format is used.</span></span> <span data-ttu-id="713dd-119">A estrutura de uma assinatura compact é mostrada no exemplo a seguir.</span><span class="sxs-lookup"><span data-stu-id="713dd-119">The structure of a compact signature is shown in the following example.</span></span>  
  
```xml  
<d:Security ... >   
  [<d:Sig Scheme="xs:anyURI"   
         [KeyId="xs:base64Binary"]?  
          Refs="..."  
         [PrefixList]="xs:NMTOKENS"   
          Sig="xs:base64Binary"   
          ... />]?  
  ...   
</d:Security>  
```  
  
> [!NOTE]
>  <span data-ttu-id="713dd-120">O `PrefixList` foi adicionado no protocolo de versão de descoberta de 2008.</span><span class="sxs-lookup"><span data-stu-id="713dd-120">The `PrefixList` was added in the 2008 Discovery version protocol.</span></span>  
  
 <span data-ttu-id="713dd-121">Para calcular a assinatura, o exemplo determina os itens de assinatura expandida.</span><span class="sxs-lookup"><span data-stu-id="713dd-121">To compute the signature, the sample determines the expanded signature items.</span></span> <span data-ttu-id="713dd-122">Uma assinatura XML (`SignedInfo`) é criado usando o `ds` prefixo de namespace, conforme exigido pela especificação WS-Discovery.</span><span class="sxs-lookup"><span data-stu-id="713dd-122">An XML signature (`SignedInfo`) is created, using the `ds` namespace prefix, as required by the WS-Discovery specification.</span></span> <span data-ttu-id="713dd-123">O corpo e todos os cabeçalhos de descoberta e a resolução de namespaces são referenciados na assinatura, para que eles não possam ser violados.</span><span class="sxs-lookup"><span data-stu-id="713dd-123">The body and all the headers in discovery and addressing namespaces are referenced in the signature, so they cannot be tampered with.</span></span> <span data-ttu-id="713dd-124">Cada elemento referenciado é transformado usando a canonicalização exclusivo (http://www.w3.org/2001/10/xml-exc-c14n# ), e, em seguida, um valor de digest SHA-1 é computado (http://www.w3.org/2000/09/xmldsig#sha1 ).</span><span class="sxs-lookup"><span data-stu-id="713dd-124">Each referenced element is transformed using the Exclusive Canonicalization (http://www.w3.org/2001/10/xml-exc-c14n# ), and then an SHA-1 digest value is computed (http://www.w3.org/2000/09/xmldsig#sha1 ).</span></span> <span data-ttu-id="713dd-125">Com base em todos os elementos e seus valores de resumo, o valor de assinatura é calculado usando o algoritmo RSA (http://www.w3.org/2000/09/xmldsig#rsa-sha1 ).</span><span class="sxs-lookup"><span data-stu-id="713dd-125">Based on all referenced elements and their digest values, the signature value is computed using the RSA algorithm (http://www.w3.org/2000/09/xmldsig#rsa-sha1 ).</span></span>  
  
 <span data-ttu-id="713dd-126">As mensagens são assinadas com um certificado de cliente especificado.</span><span class="sxs-lookup"><span data-stu-id="713dd-126">The messages are signed with a client-specified certificate.</span></span> <span data-ttu-id="713dd-127">O local do repositório, o nome e o nome de assunto do certificado devem ser especificados quando o elemento de associação é criado.</span><span class="sxs-lookup"><span data-stu-id="713dd-127">The store location, name and the certificate subject name must be specified when the binding element is created.</span></span> <span data-ttu-id="713dd-128">O `KeyId` na assinatura compact representa o identificador de chave de token de assinatura e é o assunto chave identificador SKI () do token de assinatura ou (se o SKI não existir) um hash SHA-1 da chave pública do token de assinatura.</span><span class="sxs-lookup"><span data-stu-id="713dd-128">The `KeyId` in the compact signature represents the key identifier of the signing token and is the Subject Key Identifier (SKI) of the signing token or (if the SKI does not exist) a SHA-1 hash of the public key of the signing token.</span></span>
