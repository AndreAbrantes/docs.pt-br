## <a name="discussion"></a><span data-ttu-id="36629-107">Discussão</span><span class="sxs-lookup"><span data-stu-id="36629-107">Discussion</span></span>  
 <span data-ttu-id="36629-108">Quando aplicativos autenticarem usando Kerberos, Digest ou NTLM usando HTTPS, primeiro é estabelecer um canal de segurança de nível de transporte (TLS) e, em seguida, a autenticação é feita usando o canal seguro.</span><span class="sxs-lookup"><span data-stu-id="36629-108">When applications authenticate using Kerberos, Digest or NTLM using HTTPS, a Transport Level Security (TLS) channel is first established and then authentication takes place using the secure channel.</span></span> <span data-ttu-id="36629-109">No entanto, não há nenhuma associação entre a chave de sessão gerado pelo SSL e a chave de sessão gerada durante a autenticação.</span><span class="sxs-lookup"><span data-stu-id="36629-109">However, there is no binding between the session key generated by SSL and the session key generated during authentication.</span></span> <span data-ttu-id="36629-110">Qualquer MITM pode estabelecer entre o cliente e o servidor e iniciar a encaminhar as solicitações do cliente, mesmo quando o canal de transporte é seguro, porque o servidor não tem como saber se o canal seguro de estabelecimento do cliente ou um MITM.</span><span class="sxs-lookup"><span data-stu-id="36629-110">Any MITM can establish itself between the client and the server and start forwarding the requests from the client, even when the transport channel itself is secure, because the server has no way of knowing whether the secure channel has been established from the client or a MITM.</span></span> <span data-ttu-id="36629-111">A solução neste cenário é associar o canal TLS externo com o canal de autenticação interno, de modo que o servidor pode detectar se há um MITM.</span><span class="sxs-lookup"><span data-stu-id="36629-111">The solution in this scenario is to bind the outer TLS channel with the inner authentication channel such that the server can detect if there is a MITM.</span></span>
