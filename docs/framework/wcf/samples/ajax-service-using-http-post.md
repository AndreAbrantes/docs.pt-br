<span data-ttu-id="42f6b-110">Se o <xref:System.ServiceModel.Web.WebInvokeAttribute> atributo for aplicado em uma operação ou o <xref:System.ServiceModel.Web.WebGetAttribute> atributo não for aplicado, o verbo HTTP padrão ("post") será usado.</span><span class="sxs-lookup"><span data-stu-id="42f6b-110">If the <xref:System.ServiceModel.Web.WebInvokeAttribute> attribute is applied on an operation, or the <xref:System.ServiceModel.Web.WebGetAttribute> attribute is not applied, the default HTTP verb ("POST") is used.</span></span> <span data-ttu-id="42f6b-111">Solicitações POST são mais difíceis de construir que solicitações GET, mas não são armazenadas em cache; Use solicitações POST para todas as operações em que o Caching não é apropriado.</span><span class="sxs-lookup"><span data-stu-id="42f6b-111">POST requests are harder to construct than GET requests, but they are not cached; use POST requests for all operations where caching is not appropriate.</span></span>

```csharp

public interface ICalculator
{
    [WebInvoke]
    double Add(double n1, double n2);
    //Other operations omitted…
}
```

<span data-ttu-id="42f6b-112">Crie um ponto de extremidade AJAX no serviço usando o <xref:System.ServiceModel.Activation.WebScriptServiceHostFactory> , assim como no exemplo de serviço básico Ajax.</span><span class="sxs-lookup"><span data-stu-id="42f6b-112">Create an AJAX endpoint on the service by using the <xref:System.ServiceModel.Activation.WebScriptServiceHostFactory>, just as in the Basic AJAX Service sample.</span></span>

<span data-ttu-id="42f6b-113">Ao contrário das solicitações GET, não é possível invocar serviços POST do navegador.</span><span class="sxs-lookup"><span data-stu-id="42f6b-113">Unlike GET requests, you cannot invoke POST services from the browser.</span></span> <span data-ttu-id="42f6b-114">Por exemplo, navegar até `http://localhost/ServiceModelSamples/service.svc/Add?n1=100&n2=200` resulta em um erro, pois o serviço post espera que `n1` os `n2` parâmetros e sejam enviados no corpo da mensagem no formato JSON, e não na URL.</span><span class="sxs-lookup"><span data-stu-id="42f6b-114">For example, navigating to `http://localhost/ServiceModelSamples/service.svc/Add?n1=100&n2=200` results in an error, because the POST service expects the `n1` and `n2` parameters to be sent in the message body in the JSON format, and not in the URL.</span></span>
