 <span data-ttu-id="27c4b-109">O formato da resposta retornado pelo serviço é determinado pela primeira vez por um `format` parâmetro de cadeia de caracteres de consulta e o segundo por um cabeçalho aceitar HTTP fornecido com a solicitação.</span><span class="sxs-lookup"><span data-stu-id="27c4b-109">The response format returned by the service is determined first by a `format` query-string parameter and second by an HTTP Accept header supplied with the request.</span></span> <span data-ttu-id="27c4b-110">Se o valor de `format` parâmetro de cadeia de caracteres de consulta é um dos formatos anteriores, em seguida, a resposta é retornada no formato.</span><span class="sxs-lookup"><span data-stu-id="27c4b-110">If the value of `format` query-string parameter is one of the preceding formats, then the response is returned in that format.</span></span> <span data-ttu-id="27c4b-111">Se o `format` cadeia de caracteres de consulta não está presente, em seguida, o serviço itera por meio dos elementos de cabeçalho Accept da solicitação e retorna o formato do primeiro tipo de conteúdo que o serviço oferece suporte.</span><span class="sxs-lookup"><span data-stu-id="27c4b-111">If the `format` query-string is not present, then the service iterates through the Accept header elements from the request and returns the format of the first content-type that the service supports.</span></span>  
  
 <span data-ttu-id="27c4b-112">O tipo de retorno da operação é vale a pena observar.</span><span class="sxs-lookup"><span data-stu-id="27c4b-112">The return type of the operation is worth noting.</span></span> <span data-ttu-id="27c4b-113">O WCF REST modelo de programação somente modo nativo oferece suporte aos formatos de resposta XML e JSON quando uma operação retorna um tipo diferente de <xref:System.ServiceModel.Channels.Message>.</span><span class="sxs-lookup"><span data-stu-id="27c4b-113">The WCF REST programming model only natively supports XML and JSON response formats when an operation returns a type other than <xref:System.ServiceModel.Channels.Message>.</span></span> <span data-ttu-id="27c4b-114">No entanto, ao usar <xref:System.ServiceModel.Channels.Message> como o tipo de retorno, o desenvolvedor tenha controle completo sobre como o conteúdo da mensagem deve ser formatado.</span><span class="sxs-lookup"><span data-stu-id="27c4b-114">However, when using <xref:System.ServiceModel.Channels.Message> as the return type, the developer has complete control over how the content of the message should be formatted.</span></span>

TT permite que um desenvolvedor gravar uma resposta em um formulário de modelo que contém variáveis e estruturas de controle.</span><span class="sxs-lookup"><span data-stu-id="27c4b-118">The .tt file allows a developer to write a response in a template form that contains variables and control structures.</span></span> <span data-ttu-id="27c4b-119">Para obter mais informações sobre T4, consulte [gerando artefatos por usando modelos de texto](http://go.microsoft.com/fwlink/?LinkId=166023).</span><span class="sxs-lookup"><span data-stu-id="27c4b-119">For more information about T4, see [Generating Artifacts By Using Text Templates](http://go.microsoft.com/fwlink/?LinkId=166023).</span></span>
