 <span data-ttu-id="966ce-111">Por exemplo, uma API de serviço Web pública retorna o seguinte objeto JSON, que descreve algumas informações sobre um usuário do serviço.</span><span class="sxs-lookup"><span data-stu-id="966ce-111">For example, a public Web service API returns the following JSON object, which describes some information about a user of the service.</span></span>  
  
```json  
{"personal": {"name": "Paul", "age": 23, "height": 1.7, "isSingle": true, "luckyNumbers": [5,17,21]}, "favoriteBands": ["Band ABC", "Band XYZ"]}  
```  
  
 <span data-ttu-id="966ce-112">Para desserializar esse objeto, um cliente WCF deve implementar os seguintes tipos definidos pelo usuário.</span><span class="sxs-lookup"><span data-stu-id="966ce-112">To deserialize this object, a WCF client must implement the following user-defined types.</span></span>  
  
```  
[DataContract]  
 public class MemberProfile  
 {  
     [DataMember]  
     public PersonalInfo personal;  
  
     [DataMember]  
     public string[] favoriteBands;  
 }  
  
 [DataContract]  
 public class PersonalInfo  
 {  
     [DataMember]  
     public string name;  
  
     [DataMember]  
     public int age;  
  
     [DataMember]  
     public double height;  
  
     [DataMember]  
     public bool isSingle;  
  
     [DataMember]  
     public int[] luckyNumbers;  
 }  
```  
  
 <span data-ttu-id="966ce-113">Isso pode ser complicado, especialmente se o cliente tiver que lidar com mais de um tipo de objeto JSON.</span><span class="sxs-lookup"><span data-stu-id="966ce-113">This can be cumbersome, especially if the client has to handle more than one type of JSON object.</span></span>  
  
 <span data-ttu-id="966ce-114">O `JsonObject` tipo fornecido por este exemplo apresenta uma representação de tipo fraco do objeto JSON desserializado.</span><span class="sxs-lookup"><span data-stu-id="966ce-114">The `JsonObject` type provided by this sample introduces a weakly-typed representation of the deserialized JSON object.</span></span> <span data-ttu-id="966ce-115">`JsonObject`o depende do mapeamento natural entre objetos JSON e dicionários de .NET Framework e o mapeamento entre matrizes JSON e matrizes de .NET Framework.</span><span class="sxs-lookup"><span data-stu-id="966ce-115">`JsonObject` relies on the natural mapping between JSON objects and .NET Framework dictionaries, and the mapping between JSON arrays and .NET Framework arrays.</span></span> <span data-ttu-id="966ce-116">O código a seguir mostra `JsonObject` o tipo.</span><span class="sxs-lookup"><span data-stu-id="966ce-116">The following code shows the `JsonObject` type.</span></span>  
  
```  
// Instantiation of JsonObject json omitted  
  
string name = json["root"]["personal"]["name"];  
int age = json["root"]["personal"]["age"];  
double height = json["root"]["personal"]["height"];  
bool isSingle = json["root"]["personal"]["isSingle"];  
int[] luckyNumbers = {  
                                     json["root"]["personal"]["luckyNumbers"][0],  
                                     json["root"]["personal"]["luckyNumbers"][1],  
                                     json["root"]["personal"]["luckyNumbers"][2]   
                                 };  
string[] favoriteBands = {  
                                        json["root"]["favoriteBands"][0],  
                                        json["root"]["favoriteBands"][1]  
                                    };  
```  
  
 <span data-ttu-id="966ce-117">Observe que você pode "procurar" objetos JSON e matrizes sem a necessidade de declarar seu tipo no momento da compilação.</span><span class="sxs-lookup"><span data-stu-id="966ce-117">Note that you can "browse" JSON objects and arrays without the need to declare their type at compile time.</span></span> <span data-ttu-id="966ce-118">Para obter uma explicação sobre o requisito para o objeto de `["root"]` nível superior, consulte o tópico [mapeamento entre JSON e XML](../../../../docs/framework/wcf/feature-details/mapping-between-json-and-xml.md).</span><span class="sxs-lookup"><span data-stu-id="966ce-118">For an explanation of the requirement for the top-level `["root"]` object, see the topic [Mapping Between JSON and XML](../../../../docs/framework/wcf/feature-details/mapping-between-json-and-xml.md).</span></span>

> <span data-ttu-id="966ce-119">A `JsonObject` classe é fornecida apenas como exemplo.</span><span class="sxs-lookup"><span data-stu-id="966ce-119">The `JsonObject` class is provided as an example only.</span></span> <span data-ttu-id="966ce-120">Ele não foi totalmente testado e não deve ser usado em ambientes de produção.</span><span class="sxs-lookup"><span data-stu-id="966ce-120">It has not been thoroughly tested, and should not be used in production environments.</span></span> <span data-ttu-id="966ce-121">Uma implicação óbvia de serialização JSON de tipo fraco é a falta de segurança de tipos ao trabalhar com `JsonObject`o.</span><span class="sxs-lookup"><span data-stu-id="966ce-121">An obvious implication of weakly-typed JSON serialization is the lack of type-safety when working with `JsonObject`.</span></span>  
  
 <span data-ttu-id="966ce-122">Para usar o `JsonObject` tipo, o contrato de operação do cliente <xref:System.ServiceModel.Channels.Message> deve usar como seu tipo de retorno.</span><span class="sxs-lookup"><span data-stu-id="966ce-122">To use the `JsonObject` type, the client operation contract must use <xref:System.ServiceModel.Channels.Message> as its return type.</span></span>  
  
```

    interface IClientSideProfileService  
    {  
        // There is no need to write a DataContract for the complex type returned by the service.  
        // The client will use a JsonObject to browse the JSON in the received message.  
  
        [OperationContract]  
        [WebGet(ResponseFormat = WebMessageFormat.Json)]  
        Message GetMemberProfile();  
    }  
```  
  
 <span data-ttu-id="966ce-123">O `JsonObject` é então instanciado, conforme mostrado no código a seguir.</span><span class="sxs-lookup"><span data-stu-id="966ce-123">The `JsonObject` is then instantiated as shown in the following code.</span></span>

  
 <span data-ttu-id="966ce-124">O `JsonObject` construtor usa um <xref:System.Xml.XmlDictionaryReader>, que é obtido por meio <xref:System.ServiceModel.Channels.Message.GetReaderAtBodyContents%2A> do método.</span><span class="sxs-lookup"><span data-stu-id="966ce-124">The `JsonObject` constructor takes a <xref:System.Xml.XmlDictionaryReader>, which is obtained through the <xref:System.ServiceModel.Channels.Message.GetReaderAtBodyContents%2A> method.</span></span> <span data-ttu-id="966ce-125">O leitor contém uma representação XML da mensagem JSON recebida pelo cliente.</span><span class="sxs-lookup"><span data-stu-id="966ce-125">The reader contains an XML representation of the JSON message received by the client.</span></span> <span data-ttu-id="966ce-126">Para obter mais informações, consulte o tópico [mapeamento entre JSON e XML](../../../../docs/framework/wcf/feature-details/mapping-between-json-and-xml.md).</span><span class="sxs-lookup"><span data-stu-id="966ce-126">For more information, see the topic [Mapping Between JSON and XML](../../../../docs/framework/wcf/feature-details/mapping-between-json-and-xml.md).</span></span>
