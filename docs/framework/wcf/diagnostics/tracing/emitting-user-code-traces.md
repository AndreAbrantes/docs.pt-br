## <a name="propagating-the-activity-id-to-a-service"></a><span data-ttu-id="70465-123">Propagando a ID de atividade a um serviço</span><span class="sxs-lookup"><span data-stu-id="70465-123">Propagating the Activity ID to A Service</span></span>

<span data-ttu-id="70465-124">Se você definir a `propagateActivity` de atributo para `true` para o `System.ServiceModel` arquivos de origem de rastreamento na configuração de serviço e cliente, o serviço de processamento para a solicitação para adicionar ocorre na mesma atividade como aquele definido no cliente.</span><span class="sxs-lookup"><span data-stu-id="70465-124">If you set the `propagateActivity` attribute to `true` for the `System.ServiceModel` trace source in both the client and service configuration files, the service processing for the Add request occurs in the same activity as the one defined in the client.</span></span> <span data-ttu-id="70465-125">Se o serviço define suas próprias atividades e transferências, os rastreamentos de serviço não aparecem na atividade propagada pelo cliente.</span><span class="sxs-lookup"><span data-stu-id="70465-125">If the service defines its own activities and transfers, the service traces do not appear in the client-propagated activity.</span></span> <span data-ttu-id="70465-126">Em vez disso, eles aparecem em uma atividade correlacionada por rastreamentos de transferência para a atividade cuja ID é propagada pelo cliente.</span><span class="sxs-lookup"><span data-stu-id="70465-126">Instead, they appear in an activity correlated by transfer traces to the activity whose ID is propagated by the client.</span></span>

> [!NOTE]
> <span data-ttu-id="70465-127">Se o `propagateActivity` atributo é definido como `true` sobre o cliente e o serviço, a atividade de ambiente no escopo de operação do serviço é definida pelo WCF.</span><span class="sxs-lookup"><span data-stu-id="70465-127">If the `propagateActivity` attribute is set to `true` on both the client and service, the ambient activity in the operation scope of the service is set by WCF.</span></span>

```csharp

// equal to Guid.Empty), create a new one.

{
    Guid newGuid = Guid.NewGuid();
    Trace.CorrelationManager.ActivityId = newGuid;
}
// Emit your Start trace.

// Emit the processing traces for that request.

                            + n1 + ", " + n2);
// double result = n1 + n2;

// Emit the Stop trace and exit the method scope.
ts.TraceEvent(TraceEventType.Stop, 0, "Add Activity");
// return result;
```
