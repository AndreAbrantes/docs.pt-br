<span data-ttu-id="ab59a-106">Em todos os aplicativos gerenciados, os erros de processamento são representados por objetos <xref:System.Exception>.</span><span class="sxs-lookup"><span data-stu-id="ab59a-106">In all managed applications, processing errors are represented by <xref:System.Exception> objects.</span></span> <span data-ttu-id="ab59a-107">Em aplicativos baseados em SOAP, como aplicativos WCF, os métodos de serviço comunicam informações de erro de processamento usando mensagens de falha de SOAP.</span><span class="sxs-lookup"><span data-stu-id="ab59a-107">In SOAP-based applications such as WCF applications, service methods communicate processing error information using SOAP fault messages.</span></span> <span data-ttu-id="ab59a-108">Falhas de SOAP são tipos de mensagem que são incluídos nos metadados de uma operação de serviço e, portanto, criam um contrato de falha que os clientes podem usar para tornar suas operações mais robustas ou interativas.</span><span class="sxs-lookup"><span data-stu-id="ab59a-108">SOAP faults are message types that are included in the metadata for a service operation and therefore create a fault contract that clients can use to make their operation more robust or interactive.</span></span> <span data-ttu-id="ab59a-109">Além disso, como as falhas de SOAP são expressas para clientes em formato XML, é um sistema de tipo altamente interoperável que os clientes em qualquer plataforma SOAP podem usar, aumentando o alcance de seu aplicativo WCF.</span><span class="sxs-lookup"><span data-stu-id="ab59a-109">In addition, because SOAP faults are expressed to clients in XML form, it is a highly interoperable type system that clients on any SOAP platform can use, increasing the reach of your WCF application.</span></span>

<span data-ttu-id="ab59a-110">Como os aplicativos WCF são executados em ambos os tipos de sistemas de erro, todas as informações de exceção gerenciadas enviadas ao cliente devem ser convertidas de exceções em falhas de SOAP no serviço, enviadas e convertidas de falhas de SOAP para exceções de falha em clientes WCF.</span><span class="sxs-lookup"><span data-stu-id="ab59a-110">Because WCF applications run under both types of error systems, any managed exception information that is sent to the client must be converted from exceptions into SOAP faults on the service, sent, and converted from SOAP faults to fault exceptions in WCF clients.</span></span> <span data-ttu-id="ab59a-111">No caso de clientes duplex, os contratos de cliente também podem enviar falhas SOAP de volta para um serviço.</span><span class="sxs-lookup"><span data-stu-id="ab59a-111">In the case of duplex clients, client contracts can also send SOAP faults back to a service.</span></span> <span data-ttu-id="ab59a-112">Em ambos os casos, você pode usar os comportamentos de exceção de serviço padrão ou pode controlar explicitamente se — e como — as exceções são mapeadas para as mensagens de falha.</span><span class="sxs-lookup"><span data-stu-id="ab59a-112">In either case, you can use the default service exception behaviors, or you can explicitly control whether—and how—exceptions are mapped to fault messages.</span></span>

<span data-ttu-id="ab59a-116">É altamente recomendável que as operações de serviço declarem suas falhas usando o atributo <xref:System.ServiceModel.FaultContractAttribute> para especificar formalmente todas as falhas SOAP que um cliente pode esperar receber no curso normal de uma operação.</span><span class="sxs-lookup"><span data-stu-id="ab59a-116">It is strongly recommended that service operations declare their faults by using the <xref:System.ServiceModel.FaultContractAttribute> attribute to formally specify all SOAP faults that a client can expect to receive in the normal course of an operation.</span></span> <span data-ttu-id="ab59a-117">Também é recomendável que você retorne em uma falha SOAP apenas as informações que um cliente deve saber para minimizar a divulgação de informações.</span><span class="sxs-lookup"><span data-stu-id="ab59a-117">It is also recommended that you return in a SOAP fault only the information that a client must know to minimize information disclosure.</span></span>

<span data-ttu-id="ab59a-118">Normalmente, os serviços (e clientes duplex) realizam as seguintes etapas para integrar com êxito o tratamento de erros em seus aplicativos:</span><span class="sxs-lookup"><span data-stu-id="ab59a-118">Typically, services (and duplex clients) take the following steps to successfully integrate error handling into their applications:</span></span>
