# <a name="extending-the-channel-layer"></a>Estendendo a camada do canal
A camada do canal é responsável para a troca de mensagens entre clientes e serviços. Extensões de canal podem implementar a nova funcionalidade de protocolo, como segurança ou funcionalidade de transporte, como implementar um novo transporte de rede para transportar mensagens SOAP.  
  
## <a name="in-this-section"></a>Nesta seção  
 [Visão geral do modelo de canal](../../../../docs/framework/wcf/extending/channel-model-overview.md)  
 Fornece uma visão geral de quais canais serão, os recursos que eles fornecem e como elas funcionam em um serviço e um aplicativo cliente.  
  
 [Desenvolvimento de canais](../../../../docs/framework/wcf/extending/developing-channels.md)  
 Descreve detalhadamente as funções que têm vários tipos de infraestrutura de canal, como funciona o ciclo de vida de mecanismo e o estado de estado, a manipulação de exceções e falhas, como implementar o suporte a metadados e como canais funcionam com codificadores de mensagem.  
  
 [Codificadores personalizados](../../../../docs/framework/wcf/extending/custom-encoders.md)  
 Descreve a função codificadores de mensagem em canais e como criar um.  
  
 [Upgrades de fluxo personalizados](../../../../docs/framework/wcf/extending/custom-stream-upgrades.md)  
 Descreve o processo de atualização os fluxos fornecidos por transportes voltados para o fluxo.
