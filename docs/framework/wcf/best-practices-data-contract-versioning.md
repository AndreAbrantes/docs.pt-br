## <a name="note-on-schema-validation"></a><span data-ttu-id="39193-105">Observação sobre a validação de esquema</span><span class="sxs-lookup"><span data-stu-id="39193-105">Note on Schema Validation</span></span>  
 <span data-ttu-id="39193-106">Discutir o controle de versão de contrato de dados, é importante observar que o esquema de contrato de dados exportado pelo Windows Communication Foundation (WCF) não tem nenhum suporte de controle de versão, que não seja o fato de que os elementos são marcados como opcionais por padrão.</span><span class="sxs-lookup"><span data-stu-id="39193-106">In discussing data contract versioning, it is important to note that the data contract schema exported by Windows Communication Foundation (WCF) does not have any versioning support, other than the fact that elements are marked as optional by default.</span></span>  
  
 <span data-ttu-id="39193-107">Isso significa que até mesmo o cenário controle de versão mais comuns, como adicionar um novo membro de dados, não pode ser implementado de maneira que seja consistente em relação a um determinado esquema.</span><span class="sxs-lookup"><span data-stu-id="39193-107">This means that even the most common versioning scenario, such as adding a new data member, cannot be implemented in a way that is seamless with regard to a given schema.</span></span> <span data-ttu-id="39193-108">As versões mais recentes de um contrato de dados (com um novo membro de dados, por exemplo) não validam usando o esquema antigo.</span><span class="sxs-lookup"><span data-stu-id="39193-108">The newer versions of a data contract (with a new data member, for example) do not validate using the old schema.</span></span>  
  
 <span data-ttu-id="39193-109">No entanto, há muitos cenários em que a compatibilidade de esquema estrita não é necessária.</span><span class="sxs-lookup"><span data-stu-id="39193-109">However, there are many scenarios in which strict schema compliance is not required.</span></span> <span data-ttu-id="39193-110">Muitas plataformas de serviços Web, incluindo o WCF e XML Web services criado usando ASP.NET, não executar a validação de esquema por padrão e, portanto, tolerar elementos adicionais que não são descritos no esquema.</span><span class="sxs-lookup"><span data-stu-id="39193-110">Many Web services platforms, including WCF and XML Web services created using ASP.NET, do not perform schema validation by default and therefore tolerate extra elements that are not described by the schema.</span></span> <span data-ttu-id="39193-111">Ao trabalhar com essas plataformas, muitos cenários de controle de versão são mais fáceis de implementar.</span><span class="sxs-lookup"><span data-stu-id="39193-111">When working with such platforms, many versioning scenarios are easier to implement.</span></span>  
  
 <span data-ttu-id="39193-112">Assim, há diretrizes de controle de versão de contrato de dois conjuntos de dados: um conjunto para cenários em que a validade de esquema estrita é importante e outro conjunto para cenários quando não for.</span><span class="sxs-lookup"><span data-stu-id="39193-112">Thus, there are two sets of data contract versioning guidelines: one set for scenarios where strict schema validity is important, and another set for scenarios when it is not.</span></span>

 <span data-ttu-id="39193-120">Observe que as alterações em contratos de dados que são referenciados por outros contratos de dados também estendem a camada de modelo de serviço.</span><span class="sxs-lookup"><span data-stu-id="39193-120">Note that changes in data contracts that are referenced by other data contracts also extend to the service model layer.</span></span> <span data-ttu-id="39193-121">Por exemplo, no cenário anterior a `PurchaseOrder` contrato de dados não precisa alterar.</span><span class="sxs-lookup"><span data-stu-id="39193-121">For example, in the previous scenario the `PurchaseOrder` data contract does not need to change.</span></span> <span data-ttu-id="39193-122">No entanto, ele contém um membro de dados de um `Customer` contrato de dados, que por sua vez continha um membro de dados do `Address` contrato de dados, que precisam ser alteradas.</span><span class="sxs-lookup"><span data-stu-id="39193-122">However, it contains a data member of a `Customer` data contract, which in turn contained a data member of the `Address` data contract, which does need to be changed.</span></span> <span data-ttu-id="39193-123">Nesse caso, você precisaria criar um `Address2` contrato de dados com as alterações necessárias, um `Customer2` contrato de dados que contém o `Address2` membro de dados e um `PurchaseOrder2` contrato de dados que contém um `Customer2` membro de dados.</span><span class="sxs-lookup"><span data-stu-id="39193-123">In that case, you would need to create an `Address2` data contract with the required changes, a `Customer2` data contract that contains the `Address2` data member, and a `PurchaseOrder2` data contract that contains a `Customer2` data member.</span></span> <span data-ttu-id="39193-124">Como no caso anterior, o contrato de serviço precisa ser bem com controle de versão.</span><span class="sxs-lookup"><span data-stu-id="39193-124">As in the previous case, the service contract would have to be versioned as well.</span></span>  
  
 <span data-ttu-id="39193-125">Embora esses exemplos nomes são alterados (anexando "2"), a recomendação é alterar os namespaces em vez de nomes, acrescentando novos namespaces com um número de versão ou uma data.</span><span class="sxs-lookup"><span data-stu-id="39193-125">Although in these examples names are changed (by appending a "2"), the recommendation is to change namespaces instead of names by appending new namespaces with a version number or a date.</span></span> <span data-ttu-id="39193-126">Por exemplo, o `http://schemas.contoso.com/2005/05/21/PurchaseOrder` contrato de dados muda para o `http://schemas.contoso.com/2005/10/14/PurchaseOrder` contrato de dados.</span><span class="sxs-lookup"><span data-stu-id="39193-126">For example, the `http://schemas.contoso.com/2005/05/21/PurchaseOrder` data contract would change to the `http://schemas.contoso.com/2005/10/14/PurchaseOrder` data contract.</span></span>
