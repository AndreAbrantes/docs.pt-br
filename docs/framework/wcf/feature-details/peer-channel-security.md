## <a name="security-scenarios"></a><span data-ttu-id="129b7-107">Cenários de segurança</span><span class="sxs-lookup"><span data-stu-id="129b7-107">Security Scenarios</span></span>

 <span data-ttu-id="129b7-112">Cenários de colaboração como grupo colaboração de documentos geralmente requerem que cada membro que participam da sessão ser identificado individualmente e autenticado.</span><span class="sxs-lookup"><span data-stu-id="129b7-112">Collaboration scenarios like group document collaboration often require that each member participating in the session be individually identified and authenticated.</span></span> <span data-ttu-id="129b7-113">Isso significa que um mecanismo para definir grupos de usuário e autenticar em relação a esses grupos é necessário ter uma sessão segura.</span><span class="sxs-lookup"><span data-stu-id="129b7-113">This means that a mechanism to define user groups and authenticate against those groups is necessary to have a secured session.</span></span> <span data-ttu-id="129b7-114">Além disso, os aplicativos podem exigir rastreamento cada mensagem pela autenticação no nível da mensagem.</span><span class="sxs-lookup"><span data-stu-id="129b7-114">Moreover, applications might require tracing each message by authentication at the message level.</span></span> <span data-ttu-id="129b7-115">Esses tipos de aplicativos, o desempenho pode ser sacrificado para um esquema de segurança mais forte.</span><span class="sxs-lookup"><span data-stu-id="129b7-115">In these types of applications, performance can be sacrificed for a stronger security scheme.</span></span>  
  
 <span data-ttu-id="129b7-116">Uma sessão de comunicação entre um grupo de usuários casuais pode exigir modelos de segurança informal, como dados de Conhecimento de um segredo comuns entre o grupo.</span><span class="sxs-lookup"><span data-stu-id="129b7-116">A communication session among a group of casual users can require informal security models, like knowledge of a common secret among the group.</span></span> <span data-ttu-id="129b7-117">Para esses tipos de aplicativos, com um modelo de segurança que é conveniente para estabelecer e configurar é mais importante do que com a forma mais forte de autenticação ou fornecer medidas de não-repúdio.</span><span class="sxs-lookup"><span data-stu-id="129b7-117">For these types of applications, having a security model that is convenient to establish and configure is more important than having the strongest form of authentication or providing nonrepudiation measures.</span></span> <span data-ttu-id="129b7-118">Nessas situações, um mecanismo de autenticação baseada em senha ajuda a proteger a camada de comunicação enquanto ainda permite que para autenticação de mensagem.</span><span class="sxs-lookup"><span data-stu-id="129b7-118">For these scenarios, a password-based authentication mechanism helps to secure the communication layer while still allowing for message authentication.</span></span> <span data-ttu-id="129b7-119">Segurança baseada em senha é a configuração padrão de canal par.</span><span class="sxs-lookup"><span data-stu-id="129b7-119">Password-based security is the default setting for Peer Channel.</span></span>  
  
## <a name="token-types"></a><span data-ttu-id="129b7-120">Tipos de token</span><span class="sxs-lookup"><span data-stu-id="129b7-120">Token Types</span></span>  
 <span data-ttu-id="129b7-121">Canal par reconhece um único tipo de token para identificação de alta segurança, certificados x. 509, que fornecem um modelo de identidade de alta segurança com base no tipo de autenticação e autorização que pode ser implementada.</span><span class="sxs-lookup"><span data-stu-id="129b7-121">Peer Channel recognizes a single token type for strong identification, X.509 certificates, which provide a strong identity model based on the type of authentication and authorization that can be implemented.</span></span> <span data-ttu-id="129b7-122">Integridade e confidencialidade facilmente são fornecidos usando certificados.</span><span class="sxs-lookup"><span data-stu-id="129b7-122">Confidentiality and integrity are easily provided using certificates.</span></span> <span data-ttu-id="129b7-123">No entanto, os certificados x. 509 podem ser difícil de usar e implantar.</span><span class="sxs-lookup"><span data-stu-id="129b7-123">However, X.509 certificates can be difficult to use and deploy.</span></span>  
  
 <span data-ttu-id="129b7-124">Canal par também oferece suporte para aplicativos simples com o uso de senhas.</span><span class="sxs-lookup"><span data-stu-id="129b7-124">Peer Channel also provides support for simple applications through the use of passwords.</span></span> <span data-ttu-id="129b7-125">Aplicativos podem optar por configurar grupos de ponto a ponto rápido e simples com base em uma senha fornecida.</span><span class="sxs-lookup"><span data-stu-id="129b7-125">Applications can choose to set up quick and simple peer groups based on a supplied password.</span></span> <span data-ttu-id="129b7-126">Nesse caso, um proprietário do grupo decide e a senha, se comunica com membros.</span><span class="sxs-lookup"><span data-stu-id="129b7-126">In this case, a group owner decides and communicates the password to members.</span></span> <span data-ttu-id="129b7-127">Cada membro deve entrar usando essa senha antes que elas possam ingressar a sessão.</span><span class="sxs-lookup"><span data-stu-id="129b7-127">Each member must sign in using this password before they can join the session.</span></span> <span data-ttu-id="129b7-128">Senhas podem ser usadas apenas para permitir a entrada para a sessão; eles não podem ser usados para realizar a autenticação de mensagem.</span><span class="sxs-lookup"><span data-stu-id="129b7-128">Passwords can be used only to allow entry to the session; they cannot be used to perform message authentication.</span></span> <span data-ttu-id="129b7-129">Isso ocorre porque um token simétrico que compartilham um grupo de pontos é difícil e inadequada usar para autenticação de origem.</span><span class="sxs-lookup"><span data-stu-id="129b7-129">This is because a symmetric token that a group of peers share is difficult and inappropriate to use for source authentication.</span></span>  
  
## <a name="security-model"></a><span data-ttu-id="129b7-130">Modelo de segurança</span><span class="sxs-lookup"><span data-stu-id="129b7-130">Security Model</span></span>  
 <span data-ttu-id="129b7-131">Canal par fornece a capacidade de proteger os links individuais entre pares.</span><span class="sxs-lookup"><span data-stu-id="129b7-131">Peer Channel provides the ability to secure the individual links between peers.</span></span> <span data-ttu-id="129b7-132">Isso significa que a mensagem nunca flui em um link inseguro (da perspectiva do aplicativo).</span><span class="sxs-lookup"><span data-stu-id="129b7-132">This means that a message never flows on an unsecured link (from the application perspective).</span></span> <span data-ttu-id="129b7-133">Internamente, cada link (um canal de transporte entre dois pontos) é protegida usando a segurança de camada de transporte (TLS).</span><span class="sxs-lookup"><span data-stu-id="129b7-133">Internally, each link (a transport channel between two peers) is secured using Transport Layer Security (TLS).</span></span> <span data-ttu-id="129b7-134">Isso significa que, quando um remetente compõe e envia uma mensagem, ela é enviada por transporte seguro para cada um dos seus pares de imediatos, que a mensagem de acesso e, por sua vez enviar a mensagem para seus colegas imediatas através de conexões seguras.</span><span class="sxs-lookup"><span data-stu-id="129b7-134">This means that when a sender composes and sends a message, it is sent over secure transport to each of its immediate peers, who access the message, and in turn send the message to their immediate peers over secure connections.</span></span> <span data-ttu-id="129b7-135">Essa segurança só funciona com transporte nível e é independente dos modelos de segurança de mensagem.</span><span class="sxs-lookup"><span data-stu-id="129b7-135">This security only works at the transport level and is independent of the message security models.</span></span>  
  
 <span data-ttu-id="129b7-136">Canal par também fornece uma maneira de proteger mensagens independentemente a segurança de transporte usado.</span><span class="sxs-lookup"><span data-stu-id="129b7-136">Peer Channel also provides a way to secure messages independently of the transport security used.</span></span> <span data-ttu-id="129b7-137">Neste modelo, a mensagem é protegida na fonte, usando o token de segurança da fonte, embora os certificados x. 509 no momento, somente têm suporte.</span><span class="sxs-lookup"><span data-stu-id="129b7-137">In this model, the message is secured at the source using the source’s security token, although currently only X.509 certificates are supported.</span></span> <span data-ttu-id="129b7-138">A mensagem protegida, em seguida, é transmitida pela rede ponto a ponto.</span><span class="sxs-lookup"><span data-stu-id="129b7-138">The secured message is then transmitted over the peer network.</span></span> <span data-ttu-id="129b7-139">Cada ponto de recebimento pode verificar a autenticidade da origem.</span><span class="sxs-lookup"><span data-stu-id="129b7-139">Each receiving peer can verify the authenticity of the source.</span></span> <span data-ttu-id="129b7-140">Observe que a mensagem é protegida para que não podem violar intermediários com ele.</span><span class="sxs-lookup"><span data-stu-id="129b7-140">Note that the message is secured so that intermediaries cannot tamper with it.</span></span>  
  
 <span data-ttu-id="129b7-141">Para obter a confidencialidade, aplicativos podem empregar segurança de transporte com esquemas de associação de grupo de alta segurança para evitar acesso não autorizado à mensagem.</span><span class="sxs-lookup"><span data-stu-id="129b7-141">To achieve confidentiality, applications can employ transport security with strong group membership schemes to prevent unauthorized access to the message.</span></span>  
  
 <span data-ttu-id="129b7-142">Canal par não exigem um modelo de identidade específico como o aplicativo escolhe um dos tipos de token com suporte.</span><span class="sxs-lookup"><span data-stu-id="129b7-142">Peer Channel does not require a specific identity model as long as the application chooses one of the supported token types.</span></span> <span data-ttu-id="129b7-143">Aplicativos possuem completamente o ciclo de vida dessas identidades e decisões de autenticação.</span><span class="sxs-lookup"><span data-stu-id="129b7-143">Applications completely own the life cycle of these identities and authentication decisions.</span></span>
