# <a name="peer-channel-security"></a><span data-ttu-id="d6099-102">Segurança de canal par</span><span class="sxs-lookup"><span data-stu-id="d6099-102">Peer Channel Security</span></span>

<span data-ttu-id="d6099-103">O Peer Channel permite uma variedade de tipos de aplicativos distribuídos que dependem de mensagens de vários partes.</span><span class="sxs-lookup"><span data-stu-id="d6099-103">Peer Channel enables a variety of distributed application types that depend on multiparty messaging.</span></span> <span data-ttu-id="d6099-104">Alguns exemplos incluem distribuição de conteúdo de escala da Internet, em que uma fonte confiável distribui conteúdo (como atualizações de mídia ou software), um grupo de amigos que troca músicas e fotos, ou uma equipe de colegas que edita um documento.</span><span class="sxs-lookup"><span data-stu-id="d6099-104">Some examples include Internet-scale content distribution, where a trusted source distributes content (such as media or software updates), a group of friends exchange music and photos, or a team of colleagues collaboratively edit a document.</span></span> <span data-ttu-id="d6099-105">Cada um desses cenários requer um modelo de segurança exclusivo.</span><span class="sxs-lookup"><span data-stu-id="d6099-105">Each of these scenarios requires a unique security model.</span></span> <span data-ttu-id="d6099-106">O modelo de segurança de canal par é projetado para lidar com esses cenários e fornece um modelo de segurança de som para as respectivas necessidades de identidade, autenticação e modelos de autorização diferentes.</span><span class="sxs-lookup"><span data-stu-id="d6099-106">The Peer Channel security model is designed to address these scenarios and provides a sound security model for the respective needs of different identity, authentication, and authorization models.</span></span>  
  
## <a name="security-scenarios"></a><span data-ttu-id="d6099-107">Cenários de segurança</span><span class="sxs-lookup"><span data-stu-id="d6099-107">Security Scenarios</span></span>  

 <span data-ttu-id="d6099-108">Um cenário de distribuição de conteúdo requer que cada destinatário de conteúdo identifique a fonte de conteúdo.</span><span class="sxs-lookup"><span data-stu-id="d6099-108">A content-distribution scenario requires that each content recipient identify the content source.</span></span> <span data-ttu-id="d6099-109">Devido à natureza distribuída do cenário, nem sempre é possível conhecer e confiar nos intermediários que processam ou interceptam mensagens.</span><span class="sxs-lookup"><span data-stu-id="d6099-109">Due to the distributed nature of the scenario, it is not always possible to know and trust the intermediaries that process or intercept messages.</span></span> <span data-ttu-id="d6099-110">Para reduzir efetivamente a ameaça de que um intermediário não confiável pode violar as mensagens, os aplicativos podem proteger a mensagem no remetente para que as tentativas de violação sejam detectadas facilmente.</span><span class="sxs-lookup"><span data-stu-id="d6099-110">To effectively mitigate the threat that an untrusted intermediary might tamper with the messages, applications can secure the message at the sender so that any tampering attempts are easily detected.</span></span> <span data-ttu-id="d6099-111">Nesse caso, dependendo da confidencialidade do conteúdo, a criptografia pode ser necessária.</span><span class="sxs-lookup"><span data-stu-id="d6099-111">In this case, depending on the confidentiality of the content, encryption can be necessary.</span></span>  
  
 <span data-ttu-id="d6099-112">Cenários de colaboração como a colaboração de documentos de grupo geralmente exigem que cada membro que participa da sessão seja identificado e autenticado individualmente.</span><span class="sxs-lookup"><span data-stu-id="d6099-112">Collaboration scenarios like group document collaboration often require that each member participating in the session be individually identified and authenticated.</span></span> <span data-ttu-id="d6099-113">Isso significa que um mecanismo para definir grupos de usuários e autenticar nesses grupos é necessário para ter uma sessão segura.</span><span class="sxs-lookup"><span data-stu-id="d6099-113">This means that a mechanism to define user groups and authenticate against those groups is necessary to have a secured session.</span></span> <span data-ttu-id="d6099-114">Além disso, os aplicativos podem exigir o rastreamento de cada mensagem por meio da autenticação no nível da mensagem.</span><span class="sxs-lookup"><span data-stu-id="d6099-114">Moreover, applications might require tracing each message by authentication at the message level.</span></span> <span data-ttu-id="d6099-115">Nesses tipos de aplicativos, o desempenho pode ser sacrificado por um esquema de segurança mais forte.</span><span class="sxs-lookup"><span data-stu-id="d6099-115">In these types of applications, performance can be sacrificed for a stronger security scheme.</span></span>  
  
 <span data-ttu-id="d6099-116">Uma sessão de comunicação entre um grupo de usuários casuais pode exigir modelos de segurança informais, como o conhecimento de um segredo comum entre o grupo.</span><span class="sxs-lookup"><span data-stu-id="d6099-116">A communication session among a group of casual users can require informal security models, like knowledge of a common secret among the group.</span></span> <span data-ttu-id="d6099-117">Para esses tipos de aplicativos, ter um modelo de segurança que seja conveniente para estabelecer e configurar é mais importante do que ter a forma mais forte de autenticação ou fornecer medidas de não-repúdio.</span><span class="sxs-lookup"><span data-stu-id="d6099-117">For these types of applications, having a security model that is convenient to establish and configure is more important than having the strongest form of authentication or providing nonrepudiation measures.</span></span> <span data-ttu-id="d6099-118">Para esses cenários, um mecanismo de autenticação baseado em senha ajuda a proteger a camada de comunicação e ainda permitir a autenticação de mensagens.</span><span class="sxs-lookup"><span data-stu-id="d6099-118">For these scenarios, a password-based authentication mechanism helps to secure the communication layer while still allowing for message authentication.</span></span> <span data-ttu-id="d6099-119">A segurança baseada em senha é a configuração padrão para o canal par.</span><span class="sxs-lookup"><span data-stu-id="d6099-119">Password-based security is the default setting for Peer Channel.</span></span>  
  
## <a name="token-types"></a><span data-ttu-id="d6099-120">Tipos de token</span><span class="sxs-lookup"><span data-stu-id="d6099-120">Token Types</span></span>  

 <span data-ttu-id="d6099-121">O canal par reconhece um único tipo de token para certificados de identificação forte, X. 509, que fornecem um modelo de identidade forte baseado no tipo de autenticação e autorização que podem ser implementados.</span><span class="sxs-lookup"><span data-stu-id="d6099-121">Peer Channel recognizes a single token type for strong identification, X.509 certificates, which provide a strong identity model based on the type of authentication and authorization that can be implemented.</span></span> <span data-ttu-id="d6099-122">A confidencialidade e a integridade são facilmente fornecidas usando certificados.</span><span class="sxs-lookup"><span data-stu-id="d6099-122">Confidentiality and integrity are easily provided using certificates.</span></span> <span data-ttu-id="d6099-123">No entanto, os certificados X. 509 podem ser difíceis de usar e implantar.</span><span class="sxs-lookup"><span data-stu-id="d6099-123">However, X.509 certificates can be difficult to use and deploy.</span></span>  
  
 <span data-ttu-id="d6099-124">O canal par também fornece suporte para aplicativos simples por meio do uso de senhas.</span><span class="sxs-lookup"><span data-stu-id="d6099-124">Peer Channel also provides support for simple applications through the use of passwords.</span></span> <span data-ttu-id="d6099-125">Os aplicativos podem optar por configurar grupos de pares rápidos e simples com base em uma senha fornecida.</span><span class="sxs-lookup"><span data-stu-id="d6099-125">Applications can choose to set up quick and simple peer groups based on a supplied password.</span></span> <span data-ttu-id="d6099-126">Nesse caso, um proprietário de grupo decide e comunica a senha aos membros.</span><span class="sxs-lookup"><span data-stu-id="d6099-126">In this case, a group owner decides and communicates the password to members.</span></span> <span data-ttu-id="d6099-127">Cada membro deve entrar usando essa senha antes que possa ingressar na sessão.</span><span class="sxs-lookup"><span data-stu-id="d6099-127">Each member must sign in using this password before they can join the session.</span></span> <span data-ttu-id="d6099-128">As senhas só podem ser usadas para permitir a entrada na sessão; Eles não podem ser usados para executar a autenticação de mensagens.</span><span class="sxs-lookup"><span data-stu-id="d6099-128">Passwords can be used only to allow entry to the session; they cannot be used to perform message authentication.</span></span> <span data-ttu-id="d6099-129">Isso ocorre porque um token simétrico que um grupo de colegas compartilha é difícil e inadequado para a autenticação de origem.</span><span class="sxs-lookup"><span data-stu-id="d6099-129">This is because a symmetric token that a group of peers share is difficult and inappropriate to use for source authentication.</span></span>  
  
## <a name="security-model"></a><span data-ttu-id="d6099-130">Modelo de segurança</span><span class="sxs-lookup"><span data-stu-id="d6099-130">Security Model</span></span>  

 <span data-ttu-id="d6099-131">O canal par fornece a capacidade de proteger os links individuais entre os pares.</span><span class="sxs-lookup"><span data-stu-id="d6099-131">Peer Channel provides the ability to secure the individual links between peers.</span></span> <span data-ttu-id="d6099-132">Isso significa que uma mensagem nunca flui em um link não seguro (da perspectiva do aplicativo).</span><span class="sxs-lookup"><span data-stu-id="d6099-132">This means that a message never flows on an unsecured link (from the application perspective).</span></span> <span data-ttu-id="d6099-133">Internamente, cada link (um canal de transporte entre dois pares) é protegido usando TLS (Transport Layer Security).</span><span class="sxs-lookup"><span data-stu-id="d6099-133">Internally, each link (a transport channel between two peers) is secured using Transport Layer Security (TLS).</span></span> <span data-ttu-id="d6099-134">Isso significa que, quando um remetente compõe e envia uma mensagem, ele é enviado pelo transporte seguro para cada um de seus pares imediatos, quem acessa a mensagem e, por sua vez, envia a mensagem para seus pares imediatos em conexões seguras.</span><span class="sxs-lookup"><span data-stu-id="d6099-134">This means that when a sender composes and sends a message, it is sent over secure transport to each of its immediate peers, who access the message, and in turn send the message to their immediate peers over secure connections.</span></span> <span data-ttu-id="d6099-135">Essa segurança só funciona no nível de transporte e é independente dos modelos de segurança de mensagem.</span><span class="sxs-lookup"><span data-stu-id="d6099-135">This security only works at the transport level and is independent of the message security models.</span></span>  
  
 <span data-ttu-id="d6099-136">O canal par também fornece uma maneira de proteger mensagens independentemente da segurança de transporte usada.</span><span class="sxs-lookup"><span data-stu-id="d6099-136">Peer Channel also provides a way to secure messages independently of the transport security used.</span></span> <span data-ttu-id="d6099-137">Nesse modelo, a mensagem é protegida na origem usando o token de segurança da origem, embora atualmente haja suporte apenas para certificados X. 509.</span><span class="sxs-lookup"><span data-stu-id="d6099-137">In this model, the message is secured at the source using the source’s security token, although currently only X.509 certificates are supported.</span></span> <span data-ttu-id="d6099-138">A mensagem protegida é então transmitida pela rede de mesmo nível.</span><span class="sxs-lookup"><span data-stu-id="d6099-138">The secured message is then transmitted over the peer network.</span></span> <span data-ttu-id="d6099-139">Cada ponto de recebimento pode verificar a autenticidade da origem.</span><span class="sxs-lookup"><span data-stu-id="d6099-139">Each receiving peer can verify the authenticity of the source.</span></span> <span data-ttu-id="d6099-140">Observe que a mensagem é protegida para que os intermediários não possam adulterar isso.</span><span class="sxs-lookup"><span data-stu-id="d6099-140">Note that the message is secured so that intermediaries cannot tamper with it.</span></span>  
  
 <span data-ttu-id="d6099-141">Para obter confidencialidade, os aplicativos podem empregar segurança de transporte com esquemas de associação de grupo fortes para impedir o acesso não autorizado à mensagem.</span><span class="sxs-lookup"><span data-stu-id="d6099-141">To achieve confidentiality, applications can employ transport security with strong group membership schemes to prevent unauthorized access to the message.</span></span>  
  
 <span data-ttu-id="d6099-142">O canal par não requer um modelo de identidade específico, desde que o aplicativo escolha um dos tipos de token com suporte.</span><span class="sxs-lookup"><span data-stu-id="d6099-142">Peer Channel does not require a specific identity model as long as the application chooses one of the supported token types.</span></span> <span data-ttu-id="d6099-143">Os aplicativos são completamente proprietários do ciclo de vida dessas identidades e decisões de autenticação.</span><span class="sxs-lookup"><span data-stu-id="d6099-143">Applications completely own the life cycle of these identities and authentication decisions.</span></span>
