# <a name="adopt-windows-communication-foundation"></a><span data-ttu-id="210dc-102">Adote Windows Communication Foundation</span><span class="sxs-lookup"><span data-stu-id="210dc-102">Adopt Windows Communication Foundation</span></span>

<span data-ttu-id="210dc-103">Você pode optar por usar o Windows Communication Foundation (WCF) para um novo desenvolvimento enquanto continua a manter os aplicativos existentes desenvolvidos usando o ASP.NET.</span><span class="sxs-lookup"><span data-stu-id="210dc-103">You can choose to use Windows Communication Foundation (WCF) for new development while continuing to maintain existing applications developed using ASP.NET.</span></span> <span data-ttu-id="210dc-104">Como o WCF deve ser a opção mais adequada para facilitar a comunicação com aplicativos criados com o .NET Framework em qualquer cenário, ele pode servir como uma ferramenta padrão para resolver uma ampla variedade de problemas de comunicação de software de uma maneira que o ASP.NET não pode.</span><span class="sxs-lookup"><span data-stu-id="210dc-104">Because WCF is intended to be the most suitable choice for facilitating communication with applications built with the .NET Framework in any scenario, it can serve as a standard tool for solving a wide variety of software communications problems in a way that ASP.NET cannot.</span></span>

<span data-ttu-id="210dc-105">Novos aplicativos WCF podem ser implantados nos mesmos computadores que os serviços Web ASP.NET existentes.</span><span class="sxs-lookup"><span data-stu-id="210dc-105">New WCF applications can be deployed on the same machines as existing ASP.NET Web services.</span></span> <span data-ttu-id="210dc-106">Se os serviços Web do ASP.NET existentes usarem uma versão do .NET Framework anterior à versão 2,0, você poderá usar a ferramenta de registro do IIS ASP.NET para implantar seletivamente o .NET Framework 2,0 em aplicativos do IIS nos quais novos aplicativos WCF devem ser hospedados.</span><span class="sxs-lookup"><span data-stu-id="210dc-106">If the existing ASP.NET Web services use a version of the .NET Framework prior to version 2.0, then you can use the ASP.NET IIS Registration Tool to selectively deploy the .NET Framework 2.0 to IIS applications in which new WCF applications are to be hosted.</span></span> <span data-ttu-id="210dc-107">Essa ferramenta está documentada na [ASP.net (ferramenta de registro Aspnet_regiis.exe do IIS)](/previous-versions/dotnet/netframework-3.5/k6h9cz8h(v=vs.90))e tem uma interface do usuário incorporada ao console de gerenciamento do IIS 6,0.</span><span class="sxs-lookup"><span data-stu-id="210dc-107">That tool is documented at [ASP.NET IIS Registration Tool (Aspnet_regiis.exe)](/previous-versions/dotnet/netframework-3.5/k6h9cz8h(v=vs.90)), and has a user interface built into the IIS 6.0 management console.</span></span>

<span data-ttu-id="210dc-108">O WCF pode ser usado para adicionar novos recursos aos serviços Web ASP.NET existentes adicionando os serviços WCF configurados para serem executados no modo de compatibilidade ASP.NET para aplicativos de serviço Web ASP.NET existentes no IIS.</span><span class="sxs-lookup"><span data-stu-id="210dc-108">WCF can be used to add new features to existing ASP.NET Web services by adding WCF services configured to run in ASP.NET compatibility mode to existing ASP.NET Web service applications in IIS.</span></span> <span data-ttu-id="210dc-109">Devido ao modo de compatibilidade do ASP.NET, o código para os novos serviços WCF pode acessar e atualizar as mesmas informações de estado do aplicativo que o código ASP.NET pré-existente, usando a <xref:System.Web.HttpContext> classe.</span><span class="sxs-lookup"><span data-stu-id="210dc-109">Because of ASP.NET compatibility mode, the code for the new WCF services can access and update the same application state information as the pre-existing ASP.NET code, by using the <xref:System.Web.HttpContext> class.</span></span> <span data-ttu-id="210dc-110">Os aplicativos também podem compartilhar as mesmas bibliotecas de classes.</span><span class="sxs-lookup"><span data-stu-id="210dc-110">The applications can also share the same class libraries.</span></span>

<span data-ttu-id="210dc-111">Os clientes WCF podem usar os serviços Web do ASP.NET.</span><span class="sxs-lookup"><span data-stu-id="210dc-111">WCF clients can use ASP.NET Web services.</span></span> <span data-ttu-id="210dc-112">Os serviços WCF que são configurados com o <xref:System.ServiceModel.BasicHttpBinding> podem ser usados por clientes do serviço Web ASP.net.</span><span class="sxs-lookup"><span data-stu-id="210dc-112">WCF services that are configured with the <xref:System.ServiceModel.BasicHttpBinding> can be used by ASP.NET Web service clients.</span></span> <span data-ttu-id="210dc-113">Os serviços Web do ASP.NET podem coexistir com os aplicativos WCF, e o WCF pode até ser usado para adicionar recursos aos serviços Web existentes do ASP.NET.</span><span class="sxs-lookup"><span data-stu-id="210dc-113">ASP.NET Web services can coexist with WCF applications, and WCF can even be used to add features to existing ASP.NET Web services.</span></span> <span data-ttu-id="210dc-114">Considerando todas essas maneiras em que os serviços Web do WCF e do ASP.NET podem ser usados juntos, talvez você queira migrar os serviços Web do ASP.NET para o WCF somente se precisar de recursos fornecidos pelo WCF e não pelo ASP.NET Web Services.</span><span class="sxs-lookup"><span data-stu-id="210dc-114">Given all of these ways in which WCF and ASP.NET Web services can be used together, you may want to migrate ASP.NET Web services to WCF only if you require features that are provided by WCF and not ASP.NET Web services.</span></span>

<span data-ttu-id="210dc-115">Mesmo nos poucos casos em que é necessário, a migração de código de uma tecnologia para outra raramente é a abordagem correta.</span><span class="sxs-lookup"><span data-stu-id="210dc-115">Even in the few cases where it is necessary, migrating code from one technology to another is seldom the correct approach.</span></span> <span data-ttu-id="210dc-116">O motivo para adotar a nova tecnologia é atender a novos requisitos que não podem ser atendidos com a tecnologia anterior e, nesse caso, a coisa certa a fazer é criar uma nova solução para atender ao conjunto de requisitos recentemente expandido.</span><span class="sxs-lookup"><span data-stu-id="210dc-116">The reason for adopting the new technology is to meet new requirements that cannot be met with the earlier technology, and in that case, the correct thing to do is to design a new solution to meet the newly expanded set of requirements.</span></span> <span data-ttu-id="210dc-117">O novo design beneficia-se da sua experiência com o sistema existente e da sabedoria obtida desde que o sistema foi projetado.</span><span class="sxs-lookup"><span data-stu-id="210dc-117">The new design benefits from your experience with the existing system and from wisdom gained since that system was designed.</span></span> <span data-ttu-id="210dc-118">O novo design também pode usar os recursos completos das novas tecnologias em vez de reproduzir o design antigo na nova plataforma.</span><span class="sxs-lookup"><span data-stu-id="210dc-118">The new design can also use the full capabilities of the new technologies rather than reproducing the old design on the new platform.</span></span> <span data-ttu-id="210dc-119">Após a criação de protótipos dos principais elementos do novo design, fica mais fácil reutilizar o código do sistema existente dentro do novo.</span><span class="sxs-lookup"><span data-stu-id="210dc-119">After prototyping key elements of the new design, it becomes easier to reuse code from the existing system within the new one.</span></span>
