# <a name="adopting-windows-communication-foundation"></a><span data-ttu-id="5a150-102">Adotando o Windows Communication Foundation</span><span class="sxs-lookup"><span data-stu-id="5a150-102">Adopting Windows Communication Foundation</span></span>

<span data-ttu-id="5a150-103">Você pode optar por usar o Windows Communication Foundation (WCF) para novos desenvolvimentos, enquanto continua a manter os aplicativos existentes desenvolvidos usando o ASP.NET.</span><span class="sxs-lookup"><span data-stu-id="5a150-103">You can choose to use Windows Communication Foundation (WCF) for new development, while continuing to maintain existing applications developed using ASP.NET.</span></span> <span data-ttu-id="5a150-104">Porque o WCF deve ser a escolha mais adequada para facilitar a comunicação com os aplicativos criados com o .NET Framework em qualquer cenário, ele pode servir como uma ferramenta padrão para resolver uma ampla variedade de problemas de comunicação de software de forma que o ASP.NET não é possível.</span><span class="sxs-lookup"><span data-stu-id="5a150-104">Because WCF is intended to be the most suitable choice for facilitating communication with applications built with the .NET Framework in any scenario, it can serve as a standard tool for solving a wide variety of software communications problems in a way that ASP.NET cannot.</span></span>

<span data-ttu-id="5a150-105">Novos aplicativos do WCF podem ser implantados nos mesmos computadores como serviços Web do ASP.NET existentes.</span><span class="sxs-lookup"><span data-stu-id="5a150-105">New WCF applications can be deployed on the same machines as existing ASP.NET Web services.</span></span> <span data-ttu-id="5a150-106">Se os serviços Web do ASP.NET existentes usam uma versão do .NET Framework anteriores à versão 2.0, você pode usar a ferramenta de registro ASP.NET IIS implantar seletivamente o .NET Framework 2.0 em aplicativos do IIS em que novos aplicativos do WCF devem ser hospedadas.</span><span class="sxs-lookup"><span data-stu-id="5a150-106">If the existing ASP.NET Web services use a version of the .NET Framework prior to version 2.0, then you can use the ASP.NET IIS Registration Tool to selectively deploy the .NET Framework 2.0 to IIS applications in which new WCF applications are to be hosted.</span></span> <span data-ttu-id="5a150-107">Essa ferramenta está documentada em [ferramenta de registro ASP.NET IIS (Aspnet_regiis.exe)](https://go.microsoft.com/fwlink/?LinkId=94687), e tem uma interface de usuário integrada no console de gerenciamento do IIS 6.0.</span><span class="sxs-lookup"><span data-stu-id="5a150-107">That tool is documented at [ASP.NET IIS Registration Tool (Aspnet_regiis.exe)](https://go.microsoft.com/fwlink/?LinkId=94687), and has a user interface built into the IIS 6.0 management console.</span></span>

<span data-ttu-id="5a150-108">O WCF pode ser usado para adicionar novos recursos para os serviços Web do ASP.NET existentes, adicionando os serviços WCF configurados para ser executado no modo de compatibilidade do ASP.NET para aplicativos de serviço Web ASP.NET existentes no IIS.</span><span class="sxs-lookup"><span data-stu-id="5a150-108">WCF can be used to add new features to existing ASP.NET Web services by adding WCF services configured to run in ASP.NET compatibility mode to existing ASP.NET Web service applications in IIS.</span></span> <span data-ttu-id="5a150-109">Devido ao modo de compatibilidade do ASP.NET, o código para os novos serviços WCF pode acessar e atualizar as informações de estado do aplicativo mesmo que o código ASP.NET já existente, usando o <xref:System.Web.HttpContext> classe.</span><span class="sxs-lookup"><span data-stu-id="5a150-109">Because of ASP.NET compatibility mode, the code for the new WCF services can access and update the same application state information as the pre-existing ASP.NET code, by using the <xref:System.Web.HttpContext> class.</span></span> <span data-ttu-id="5a150-110">Os aplicativos também podem compartilhar as mesmas bibliotecas de classe.</span><span class="sxs-lookup"><span data-stu-id="5a150-110">The applications can also share the same class libraries.</span></span>

<span data-ttu-id="5a150-111">Clientes do WCF podem usar os serviços Web do ASP.NET.</span><span class="sxs-lookup"><span data-stu-id="5a150-111">WCF clients can use ASP.NET Web services.</span></span> <span data-ttu-id="5a150-112">Os serviços WCF que são configurados com o <xref:System.ServiceModel.BasicHttpBinding> pode ser usado por clientes de serviço Web do ASP.NET.</span><span class="sxs-lookup"><span data-stu-id="5a150-112">WCF services that are configured with the <xref:System.ServiceModel.BasicHttpBinding> can be used by ASP.NET Web service clients.</span></span> <span data-ttu-id="5a150-113">Serviços Web do ASP.NET podem coexistir com os aplicativos do WCF e WCF ainda pode ser usado para adicionar recursos a serviços Web do ASP.NET existentes.</span><span class="sxs-lookup"><span data-stu-id="5a150-113">ASP.NET Web services can co-exist with WCF applications, and WCF can even be used to add features to existing ASP.NET Web services.</span></span> <span data-ttu-id="5a150-114">Considerando todas essas formas em que os serviços WCF e ASP.NET Web podem ser usados juntos, você talvez queira migrar serviços Web do ASP.NET ao WCF somente se você precisar de recursos que são fornecidos pelos serviços WCF e não da Web do ASP.NET.</span><span class="sxs-lookup"><span data-stu-id="5a150-114">Given all of these ways in which WCF and ASP.NET Web services can be used together, you may want to migrate ASP.NET Web services to WCF only if you require features that are provided by WCF and not ASP.NET Web services.</span></span>

<span data-ttu-id="5a150-115">Até mesmo em alguns casos em que é necessário, migrando o código de uma tecnologia para outro raramente é a abordagem correta.</span><span class="sxs-lookup"><span para adotar a nova tecnologia é para atender aos novos requisitos que não podem ser atendidos com a tecnologia anterior e, nesse caso, a coisa correta a fazer é criar uma nova solução para atender o conjunto de requisitos recentemente expandido.</span><span class="sxs-lookup"><span data-stu-id="5a150-116">The reason for adopting the new technology is to meet new requirements that cannot be met with the earlier technology, and in that case, the correct thing to do is to design a new solution to meet the newly-expanded set of requirements.</span></span> <span data-ttu-id="5a150-117">Os novos benefícios de design de sua experiência com o sistema existente e de sabedoria ganhou uma vez que esse sistema foi projetado.</span><span class="sxs-lookup"><span data-stu-id="5a150-117">The new capabilities of the new technologies rather than reproducing the old design on the new platform.</span></span> <span data-ttu-id="5a150-119">Após a criação de protótipos elementos-chave o novo design, fica mais fácil para reutilização de código do sistema existente dentro de um novo.</span><span class="sxs-lookup"><span data-stu-id="5a150-119">After prototyping key elements of the new design, it
