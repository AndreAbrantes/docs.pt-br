 <span data-ttu-id="e82cc-106">O sistema de mensagens confiável tem os seguintes aspectos principais:</span><span class="sxs-lookup"><span data-stu-id="e82cc-106">Reliable messaging has the following key aspects:</span></span>

## <a name="reliable-sessions"></a><span data-ttu-id="e82cc-114">Sessões confiáveis</span><span class="sxs-lookup"><span data-stu-id="e82cc-114">Reliable Sessions</span></span>  
 <span data-ttu-id="e82cc-115">As sessões confiáveis fornecem transferência de mensagens confiável de ponta a ponta entre uma origem e um destino usando o protocolo WS-ReliableMessaging, independentemente do número ou tipo de intermediários que separam os pontos de extremidade de mensagens (origem e destino).</span><span class="sxs-lookup"><span data-stu-id="e82cc-115">Reliable sessions provide end-to-end reliable transfer of messages between a source and a destination using the WS-ReliableMessaging protocol regardless of the number or type of intermediaries that separate the messaging (source and destination) endpoints.</span></span> <span data-ttu-id="e82cc-116">Isso inclui os intermediários de transporte que não usam SOAP (por exemplo, proxies HTTP) ou intermediários que usam SOAP (por exemplo, roteadores ou pontes baseados em SOAP) que são necessários para que as mensagens fluam entre os pontos de extremidade.</span><span class="sxs-lookup"><span data-stu-id="e82cc-116">This includes any transport intermediaries that do not use SOAP (for example, HTTP proxies) or intermediaries that use SOAP (for example, SOAP-based routers or bridges) that are required for messages to flow between the endpoints.</span></span> <span data-ttu-id="e82cc-117">As sessões confiáveis usam uma janela de transferência na memória para mascarar falhas de nível de mensagem SOAP e restabelecer conexões no caso de falhas de transporte.</span><span class="sxs-lookup"><span data-stu-id="e82cc-117">Reliable sessions use an in-memory transfer window to mask SOAP message-level failures and re-establish connections in the case of transport failures.</span></span>  
  
 <span data-ttu-id="e82cc-118">As sessões confiáveis fornecem transferências de mensagens confiáveis de baixa latência.</span><span class="sxs-lookup"><span data-stu-id="e82cc-118">Reliable sessions provide low-latency reliable message transfers.</span></span> <span data-ttu-id="e82cc-119">Eles fornecem mensagens SOAP sobre quaisquer proxies ou intermediários, equivalentes ao que o TCP fornece para pacotes em pontes IP.</span><span class="sxs-lookup"><span data-stu-id="e82cc-119">They provide for SOAP messages over any proxies or intermediaries, equivalent to what TCP provides for packets over IP bridges.</span></span> <span data-ttu-id="e82cc-120">Para obter mais informações sobre sessões confiáveis, consulte [sessões confiáveis](reliable-sessions.md).</span><span class="sxs-lookup"><span data-stu-id="e82cc-120">For more information about reliable sessions, see [Reliable Sessions](reliable-sessions.md).</span></span>  
  
## <a name="queues"></a><span data-ttu-id="e82cc-121">Filas</span><span class="sxs-lookup"><span data-stu-id="e82cc-121">Queues</span></span>  
 <span data-ttu-id="e82cc-122">As filas no WCF fornecem transferências confiáveis de mensagens e separação entre origens e destinos ao custo de alta latência.</span><span class="sxs-lookup"><span data-stu-id="e82cc-122">Queues in WCF provide both reliable transfers of messages and separation between sources and destinations at the cost of high latency.</span></span> <span data-ttu-id="e82cc-123">A comunicação na fila do WCF é criada com base no enfileiramento de mensagens (também conhecido como MSMQ).</span><span class="sxs-lookup"><span data-stu-id="e82cc-123">WCF queued communication is built on top of Message Queuing (also known as MSMQ).</span></span>  
  
 <span data-ttu-id="e82cc-124">O MSMQ é fornecido como uma opção com o Windows que é executado como um serviço NT.</span><span class="sxs-lookup"><span data-stu-id="e82cc-124">MSMQ is shipped as an option with Windows that runs as an NT service.</span></span> <span data-ttu-id="e82cc-125">Ele captura mensagens para transmissão em uma fila de transmissão em nome da origem e a entrega a uma fila de destino.</span><span class="sxs-lookup"><span data-stu-id="e82cc-125">It captures messages for transmission in a transmission queue on behalf of the source and delivers it to a target queue.</span></span> <span data-ttu-id="e82cc-126">A fila de destino aceita mensagens em nome do destino para entrega posterior sempre que o destino solicita mensagens.</span><span class="sxs-lookup"><span data-stu-id="e82cc-126">The target queue accepts messages on behalf of the destination for later delivery whenever the destination requests for messages.</span></span> <span data-ttu-id="e82cc-127">Os gerenciadores de filas do MSMQ implementam um protocolo de transferência de mensagens confiável para que as mensagens não sejam perdidas na transmissão.</span><span class="sxs-lookup"><span data-stu-id="e82cc-127">The MSMQ queue managers implement a reliable message-transfer protocol so that messages are not lost in transmission.</span></span> <span data-ttu-id="e82cc-128">O protocolo pode ser baseado em SOAP ou nativo, como o protocolo SRMP (SOAP Reliable Messaging Protocol).</span><span class="sxs-lookup"><span data-stu-id="e82cc-128">The protocol can be native or SOAP-based, such as Soap Reliable Messaging Protocol (SRMP).</span></span>  
  
 <span data-ttu-id="e82cc-129">A separação, combinada com transferências de mensagens confiáveis entre filas, permite que os aplicativos que são livremente acoplados se comuniquem de forma confiável.</span><span class="sxs-lookup"><span data-stu-id="e82cc-129">The separation, coupled with reliable message transfers between queues, enables applications that are loosely coupled to communicate reliably.</span></span> <span data-ttu-id="e82cc-130">Ao contrário de sessões confiáveis, a origem e o destino não precisam estar em execução ao mesmo tempo.</span><span class="sxs-lookup"><span data-stu-id="e82cc-130">Unlike reliable sessions, the source and destination do not have to be running at the same time.</span></span> <span data-ttu-id="e82cc-131">Isso habilita implicitamente cenários em que as filas são, em vigor, usadas como um mecanismo de nivelamento de carga quando há uma incompatibilidade entre a taxa de produção de mensagem pela origem e a taxa de consumo de mensagem pelo destino.</span><span class="sxs-lookup"><span data-stu-id="e82cc-131">This implicitly enables scenarios where queues are, in effect, used as a load-leveling mechanism when there is a mismatch between the rate of message production by the source and the rate of the message consumption by the destination.</span></span> <span data-ttu-id="e82cc-132">Para obter mais informações sobre filas, consulte [filas no WCF](queues-in-wcf.md).</span><span class="sxs-lookup"><span data-stu-id="e82cc-132">For more information about queues, see [Queues in WCF](queues-in-wcf.md).</span></span>
