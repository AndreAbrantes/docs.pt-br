>  <span data-ttu-id="689a6-106">O moniker de serviço usa um canal de comunicação WCF para todas as comunicações.</span><span class="sxs-lookup"><span data-stu-id="689a6-106">The service moniker uses a WCF communication channel for all communication.</span></span> <span data-ttu-id="689a6-107">Os mecanismos de segurança e identidade para esse canal diferem daqueles usados no padrão proxies COM e DCOM.</span><span class="sxs-lookup"><span data-stu-id="689a6-107">The security and identity mechanisms for that channel differ from those used in standard COM and DCOM proxies.</span></span> <span data-ttu-id="689a6-108">Além disso, como o moniker de serviço usa um canal de comunicação WCF o período de tempo limite padrão é um minuto para todas as chamadas.</span><span class="sxs-lookup"><span data-stu-id="689a6-108">In addition, because the service moniker uses a WCF communication channel the default timeout period is one minute for all calls.</span></span>  
  
 <span data-ttu-id="689a6-109">O moniker de serviço é usado com o `GetObject` função para fornecer o desenvolvedor não gerenciado com uma abordagem fortemente tipada COM específicas para chamar os serviços Web WCF.</span><span class="sxs-lookup"><span data-stu-id="689a6-109">The service moniker is used with the `GetObject` function to provide the unmanaged developer with a strongly-typed, COM-specific approach for calling WCF Web services.</span></span> <span data-ttu-id="689a6-110">Isso requer um local, visível em COM definição de contrato de serviço Web WCF e a associação a ser usado.</span><span class="sxs-lookup"><span data-stu-id="689a6-110">This requires a local, COM-visible definition of the WCF Web service contract and the binding that is to be used.</span></span> <span data-ttu-id="689a6-111">Como outros clientes do WCF, o moniker de serviço deve construir um canal inserido para o serviço, embora essa construção de canal ocorre de forma transparente para o programador COM na primeira chamada do método.</span><span class="sxs-lookup"><span data-stu-id="689a6-111">Like other WCF clients, the service moniker must construct a typed channel to the service, though this channel construction occurs transparently to the COM programmer on the first method call.</span></span>  
  
 <span data-ttu-id="689a6-112">Em comum com outros clientes do WCF, ao usar o identificador de origem, os aplicativos especificar o endereço, associação e contrato para se comunicar com um serviço.</span><span class="sxs-lookup"><span data-stu-id="689a6-112">In common with other WCF clients, when using the moniker, applications specify the address, binding and contract to communicate with a service.</span></span> <span data-ttu-id="689a6-113">O contrato pode ser especificado em uma das seguintes maneiras:</span><span class="sxs-lookup"><span data-stu-id="689a6-113">The contract can be specified in one of the following ways:</span></span>

-   <span data-ttu-id="689a6-116">Contrato MEX – o contrato é recuperado em tempo de execução de um ponto de extremidade de troca de metadados (MEX).</span><span class="sxs-lookup"><span data-stu-id="689a6-116">MEX contract – the contract is retrieved at runtime from a Metadata Exchange (MEX) endpoint.</span></span>
