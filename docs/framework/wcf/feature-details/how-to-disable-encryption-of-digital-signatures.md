# <a name="how-to-disable-encryption-of-digital-signatures"></a><span data-ttu-id="e19b5-102">Como: desabilitar criptografia de assinaturas digitais</span><span class="sxs-lookup"><span data-stu-id="e19b5-102">How to: Disable Encryption of Digital Signatures</span></span>
<span data-ttu-id="e19b5-103">Por padrão, uma mensagem é assinada e a assinatura digital é criptografada.</span><span class="sxs-lookup"><span data-stu-id="e19b5-103">By default, a message is signed and the signature is digitally encrypted.</span></span> <span data-ttu-id="e19b5-104">Isso é controlado pela criação de uma associação personalizada com uma instância da <xref:System.ServiceModel.Channels.AsymmetricSecurityBindingElement> ou o <xref:System.ServiceModel.Channels.SymmetricSecurityBindingElement> e definindo o `MessageProtectionOrder` propriedade de qualquer classe para um <xref:System.ServiceModel.Security.MessageProtectionOrder> valor de enumeração.</span><span class="sxs-lookup"><span data-stu-id="e19b5-104">This is controlled by creating a custom binding with an instance of the <xref:System.ServiceModel.Channels.AsymmetricSecurityBindingElement> or the <xref:System.ServiceModel.Channels.SymmetricSecurityBindingElement> and setting the `MessageProtectionOrder` property of either class to a <xref:System.ServiceModel.Security.MessageProtectionOrder> enumeration value.</span></span> <span data-ttu-id="e19b5-105">O padrão é <xref:System.ServiceModel.Security.MessageProtectionOrder.SignBeforeEncryptAndEncryptSignature>.</span><span class="sxs-lookup"><span data-stu-id="e19b5-105">The default is <xref:System.ServiceModel.Security.MessageProtectionOrder.SignBeforeEncryptAndEncryptSignature>.</span></span> <span data-ttu-id="e19b5-106">Esse processo consome até 30 por cento mais tempo do que data-ttu-id="e19b5-107">Desabilitar a criptografia da assinatura, no entanto, pode permitir que um invasor adivinhar o conteúdo da mensagem.</span><span class="sxs-lookup"><span data-stu-id="e19b5-107">Disabling encryption of the signature, however, might allow an attacker to guess the content of the message.</span></span> <span data-ttu-id="e19b5-108">Isso é possível porque o elemento de assinatura contém o código hash do texto sem formatação de todas as partes na mensagem assinada.</span><span class="sxs-lookup"><span data-stu-id="e19b5-108">This is possible because the assinatura atenua esse vetor de ataque.</span><span class="sxs-lookup"><span data-stu-id="e19b5-111">Encrypting the signature mitigates this attack vector.</span></span>  
  
 <span data-ttu-id="e19b5-112">Portanto, desabilite a criptografia da assinatura somente quando o valor do conteúdo é baixo ou o conjunto de valores possíveis de conteúdo é grande e não determinísticas, e o ganho de desempenho é mais importante do que reduzir o ataque descrito acima.</span><span class="sxs-lookup"><span data-stu-id="e19b5-112">Therefore, disable encryption of the signature only when the value of the content is low or the set of possible content values is large and nondeterministic, and the performance gain is more important than mitigating the attack described above.</span></span>  
  
> [!NOTE]
>  <span data-ttu-id="e19b5-113">Se não houver nada na mensagem que é criptografada, o elemento de assinatura não for criptografado, mesmo quando o <xref:System.ServiceModel.Channels.AsymmetricSecurityBindingElement.MessageProtectionOrder%2A?displayProperty=nameWithType> ou <xref:System.ServiceModel.Channels.SymmetricSecurityBindingElement.MessageProtectionOrder%2A?displayProperty=nameWithType> estiver definida como <xref:System.ServiceModel.Security.MessageProtectionOrder.SignBeforeEncryptAndEncryptSignature>.</span><span class="sxs-lookup"><span data-stu-id="e19b5-113">If there is nothing in the message that is encrypted, the signature element is not encrypted, even when the <xref:System.ServiceModel.Channels.AsymmetricSecurityBindingElement.MessageProtectionOrder%2A?displayProperty=nameWithType> or <xref:System.ServiceModel.Channels.SymmetricSecurityBindingElement.MessageProtectionOrder%2A?displayProperty=nameWithType> property is set to <xref:System.ServiceModel.Security.MessageProtectionOrder.SignBeforeEncryptAndEncryptSignature>.</span></span> <span data-ttu-id="e19b5-114">Esse comportamento ocorre mesmo com associações fornecidas pelo sistema; todas as associações fornecidas pelo sistema que a ordem de proteção de mensagem definida como `SignBeforeEncryptAndEncryptSignature`.</span><span class="sxs-lookup"><span data-stu-id="e19b5-114">This behavior occurs even with system-provided bindings; all system-provided bindings have the message protection order set to `SignBeforeEncryptAndEncryptSignature`.</span></span> <span data-ttu-id="e19b5-115">No entanto, a descrição de linguagem WSDL (Web Services) WCF gera será ainda contêm a `<sp:EncryptSignature>` asserção.</span><span class="sxs-lookup"><span data-stu-id="e19b5-115">However, the Web Services Description Language (WSDL) WCF generates will still contain the `<sp:EncryptSignature>` assertion.</span></span>  
  
### <a name="to-disable-digital-signing"></a><span data-ttu-id="e19b5-116">Para desabilitar a assinatura digital</span><span class="sxs-lookup"><span data-stu-id="e19b5-116">To disable digital signing</span></span>
