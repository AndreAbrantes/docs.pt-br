 <span data-ttu-id="733a5-104">Considere um cenário com três participantes: um cliente, servidor e invasor.</span><span class="sxs-lookup"><span data-stu-id="733a5-104">Consider a scenario with three participants: a client, server, and attacker.</span></span> <span data-ttu-id="733a5-105">O servidor tem a URL `https://server` , enquanto o invasor tem a URL `https://attacker` .</span><span class="sxs-lookup"><span data-stu-id="733a5-105">The server has the URL `https://server`, whereas the attacker has the URL `https://attacker`.</span></span> <span data-ttu-id="733a5-106">O invasor vaza o cliente para acessar o invasor como se fosse o servidor.</span><span class="sxs-lookup"><span data-stu-id="733a5-106">The attacker tricks the client into accessing the attacker as if it were WWW-Authenticate.</span><span class="sxs-lookup"><span data-stu-id="733a5-108">If the attacker is trying to access a secure resource, the server replies to the attacker with a WWW-Authenticate header.</span></span> <span data-ttu-id="733a5-109">O invasor não tem as informações de autenticação e, portanto, envia o cabeçalho WWW-Authenticate para o cliente.</span><span  <span data-ttu-id="733a5-111">Atualmente, quando um aplicativo cliente se autentica no servidor usando Kerberos, Digest ou NTLM usando HTTPS, um canal de TLS (segurança de nível de transporte) é class="sxs-lookup"><span data-stu-id="733a5-112">However, there is no binding between the session key generated by Secure Sockets Layer (SSL) and the session key that is generated during authentication.</span></span> <span data-ttu-id="733a5-113">Portanto, no cenário anterior, se a comunicação assumir o lugar de um TLS (como um canal HTTPS), há dois canais SSL criados: um entre o cliente e o invasor e outro entre o invasor e o servidor.</span><span class="sxs-lookup"><span data-stu-id="733a5-113">So, in the previous scenario, if the communication takes places over a TLS (such credenciais do cliente são enviadas do cliente para o servidor primeiro por meio do canal SSL entre o cliente e o invasor e, em seguida, pelo canal entre o invasor e o servidor.</span><span class="sxs-lookup"><span data-stu-id="733a5-114">The client’s credentials are sent from the client to the server first over the SSL channel between the client and the attacker and then over the server, the server verifies the credentials without detecting that the channel over which those credentials were sent originated with the attacker and not the client.</span></span>  
  
 <span data-ttu-id="733a5-116">A solução é usar um canal externo protegido por TLS e um canal interno autenticado pelo cliente e passar um token de associação de canal (CBT) para o servidor.</span><span class="sxs-lookup"><span data-stu-id="733a5-116">The solution is to use a TLS-secured outer channel and a client-authenticated inner channel, and to pass a Channel Binding Token (CBT) to the server.</span></span> <span data-ttu-id="733a5-117">O CBT é uma propriedade do canal externo protegido por TLS e é usado para associar o canal externo a uma conversa no canal interno autenticado pelo cliente.</span><span class="sxs-lookup"><span data-stu-id="733a5-117">The CBT is a property of the TLS-secured outer channel, and is used to bind the outer channel to a conversation over the client-authenticated inner channel.</span></span>  
  
 <span data-ttu-id="733a5-118">No cenário anterior, o CBT do canal do cliente-invasor TLS é mesclado com as informações de autorização que são enviadas para o servidor.</span><span class="sxs-lookup"><span data-stu-id="733a5-118">In the previous scenario, the CBT of the client-attacker TLS channel is merged with the authorization information that is sent to the server.</span></span> <span data-ttu-id="733a5-119">Um servidor com reconhecimento de CBT compara o CBT contido nas informações de autenticação do cliente, que corresponde ao canal do invasor do cliente, para o CBT conectado ao canal do servidor do invasor.</span><span class="sxs-lookup"><span data-stu-id="733a5-119">A CBT-aware server compares the CBT contained in the client authentication information, which corresponds to the client-attacker channel, to the CBT attached to the attacker-server channel.</span></span> <span data-ttu-id="733a5-120">Um CBT é específico para o destino de um canal, portanto, o CBT do invasor do cliente não corresponde ao invasor-servidor CBT.</span><span class="sxs-lookup"><span data-stu-id="733a5-120">A CBT is specific to a channel’s destination, so the client-attacker CBT does not match the attacker-server CBT.</span></span> <span data-ttu-id="733a5-121">Isso permite que o servidor detecte o ataque MITM e recuse a solicitação de autenticação.</span><span class="sxs-lookup"><span data-stu-id="733a5-121">This lets the server detect the MITM attack and refuse the authentication request.</span></span>  
  
 <span data-ttu-id="733a5-122">O lado do cliente não requer nenhuma definição de configuração.</span><span class="sxs-lookup"><span data-stu-id="733a5-122">The client side does not require any configuration setting.</span></span> <span data-ttu-id="733a5-123">Depois que o cliente tiver sido atualizado para passar o CBT para o servidor, ele sempre fará isso.</span><span class="sxs-lookup"><span data-stu-id="733a5-123">Once the client has been updated to pass the CBT to the server, it always does so.</span></span> <span data-ttu-id="733a5-124">Se o servidor também tiver sido atualizado, ele poderá ser configurado para usar o CBT ou ignorá-lo.</span><span class="sxs-lookup"><span data-stu-id="733a5-124">If the server has also been updated, it can be configured to use the CBT or ignore it.</span></span> <span data-ttu-id="733a5-125">Se ele não tiver sido atualizado, ele o ignorará.</span><span class="sxs-lookup"><span data-stu-id="733a5-125">If it has not been updated, it ignores it.</span></span>  
  
 <span data-ttu-id="733a5-126">O servidor pode ter os seguintes níveis de proteção:</span><span class="sxs-lookup"><span data-stu-id="733a5-126">The server can have the following levels of protection:</span></span>  
  
- <span data-ttu-id="733a5-127">Nenhum.</span><span class="sxs-lookup"><span data-stu-id="733a5-127">None.</span></span> <span data-ttu-id="733a5-128">Nenhuma validação de associação de canal é executada.</span><span class="sxs-lookup"><span data-stu-id="733a5-128">No channel binding validation is performed.</span></span> <span data-ttu-id="733a5-129">Esse é o comportamento de todos os servidores que não foram atualizados.</span><span class="sxs-lookup"><span data-stu-id="733a5-129">This is the behavior of all servers that have not been updated.</span></span>  
  
- <span data-ttu-id="733a5-130">Parcial.</span><span class="sxs-lookup"><span data-stu-id="733a5-130">Partial.</span></span> <span data-ttu-id="733a5-131">Todos os clientes que foram atualizados devem fornecer informações de associação de canal para o servidor.</span><span class="sxs-lookup"><span data-stu-id="733a5-131">All clients that have been updated must provide channel binding information to the server.</span></span> <span data-ttu-id="733a5-132">Os clientes que não foram atualizados não precisam fazer isso.</span><span class="sxs-lookup"><span data-stu-id="733a5-132">Clients that have not been updated do not have to do so.</span></span> <span data-ttu-id="733a5-133">Essa é uma opção intermediária que permite a compatibilidade de aplicativos.</span><span class="sxs-lookup"><span data-stu-id="733a5-133">This is an intermediate option that allows for application compatibility.</span></span>
