# <a name="replay-attacks"></a><span data-ttu-id="f95b3-102">Ataques por repetição</span><span class="sxs-lookup"><span data-stu-id="f95b3-102">Replay Attacks</span></span>
<span data-ttu-id="f95b3-103">Um *ataque de reprodução* ocorre quando um invasor copia um fluxo de mensagens entre duas partes e repete o fluxo a um ou mais das partes.</span><span class="sxs-lookup"><span data-stu-id="f95b3-103">A *replay attack* occurs when an attacker copies a stream of messages between two parties and replays the stream to one or more of the parties.</span></span> <span data-ttu-id="f95b3-104">A menos que atenuado, os computadores sujeita a ataque processam o fluxo como mensagens legítimas, resultando em um intervalo de consequências incorretas, como com redundância de pedidos de um item.</span><span class="sxs-lookup"><span data-stu-id="f95b3-104">Unless mitigated, the computers subject to the attack process the stream as legitimate messages, resulting in a range of bad consequences, such as redundant orders of an item.</span></span>  
  
## <a name="bindings-may-be-subject-to-reflection-attacks"></a><span data-ttu-id="f95b3-105">Associações podem estar sujeito a ataques de reflexão</span><span class="sxs-lookup"><span data-stu-id="f95b3-105">Bindings May Be Subject to Reflection Attacks</span></span>  
 <span data-ttu-id="f95b3-106">*Ataques de reflexão* são repetições de mensagens de volta para um remetente como se eles vieram o destinatário, como a resposta.</span><span class="sxs-lookup"><span data-stu-id="f95b3-106">*Reflection attacks* are replays of messages back to a sender as if they came from the receiver as the reply.</span></span> <span data-ttu-id="f95b3-107">O padrão *detecção de reprodução* no Windows Communication Foundation (WCF) mecanismo não lidam automaticamente com isso.</span><span class="sxs-lookup"><span data-stu-id="f95b3-107">The standard *replay detection* in the Windows Communication Foundation (WCF) mechanism does not automatically handle this.</span></span>  
  
 <span data-ttu-id="f95b3-108">Ataques de reflexão são atenuados por padrão porque o modelo de serviço do WCF adiciona uma ID de mensagem assinadas para mensagens de solicitação e espera-se com um sinal `relates-to` cabeçalho nas mensagens de resposta.</span><span class="sxs-lookup"><span data-stu-id="f95b3-108">Reflection attacks are mitigated by default because the WCF service model adds a signed message ID to request messages and expects a signed `relates-to` header on response messages.</span></span> <span data-ttu-id="f95b3-109">Consequentemente, a mensagem de solicitação não pode ser reproduzida como uma resposta.</span><span class="sxs-lookup"><span data-stu-id="f95b3-109">Consequently, the request message cannot be replayed as a response.</span></span> <span data-ttu-id="f95b3-110">Em cenários de mensagens confiável e segura de (RM), ataques de reflexão são atenuados porque:</span><span class="sxs-lookup"><span data-stu-id="f95b3-110">In secure reliable message (RM) scenarios, reflection attacks are mitigated because:</span></span>  
  
-   <span data-ttu-id="f95b3-111">A sequência de criação e criar esquemas de mensagens de resposta de sequência são diferentes.</span><span class="sxs-lookup"><span data-stu-id="f95b3-111">The create sequence and create sequence response message schemas are different.</span></span>  
  
-   <span data-ttu-id="f95b3-112">Para as sequências de simples, mensagens de sequência, que o cliente envia não podem ser repetidas volta para ele, porque o cliente não é possível entender essas mensagens.</span><span class="sxs-lookup"><span data-stu-id="f95b3-112">For simplex sequences, sequence messages the client sends cannot be replayed back to it because the client cannot understand such messages.</span></span>  
  
-   <span data-ttu-id="f95b3-113">Para as sequências de duplex, a sequência de duas IDs deve ser exclusivos.</span><span class="sxs-lookup"><span data-stu-id="f95b3-113">For duplex sequences, the two sequence IDs must be unique.</span></span> <span data-ttu-id="f95b3-114">Portanto, uma mensagem de sequência de saída não pode ser reproduzida volta como uma mensagem de sequência de entrada (todos os cabeçalhos de sequência e corpos são assinados, muito).</span><span class="sxs-lookup"><span data-stu-id="f95b3-114">Thus, an outgoing sequence message cannot be replayed back as an incoming sequence message (all sequence headers and bodies are signed, too).</span></span>  
  
 <span data-ttu-id="f95b3-115">As apenas associações que são suscetíveis a ataques de reflexão são aqueles sem WS-Addressing: ligações personalizadas que têm o WS-Addressing desabilitado e usam a segurança de baseada em chave simétrica.</span><span class="sxs-lookup"><span data-stu-id="f95b3-115">The only bindings that are susceptible to reflection attacks are those without WS-Addressing: custom bindings that have WS-Addressing disabled and use the symmetric key-based security.</span></span> <span data-ttu-id="f95b3-116">O <xref:System.ServiceModel.BasicHttpBinding> não usa WS-Addressing por padrão, mas não usa segurança baseada em chave simétrica de uma maneira que permite que ele seja vulnerável a esse ataque.</span><span class="sxs-lookup"><span data-stu-id="f95b3-116">The <xref:System.ServiceModel.BasicHttpBinding> does not use WS-Addressing by default, but it does not use symmetric key-based security in a way that allows it to be vulnerable to this attack.</span></span>  
  
 <span data-ttu-id="f95b3-117">A mitigação para associações personalizadas é não estabelecer o contexto de segurança ou para solicitar que os cabeçalhos de WS-Addressing.</span><span class="sxs-lookup"><span data-stu-id="f95b3-117">The mitigation for custom bindings is to not establish security context or to require WS-Addressing headers.</span></span>  
  
## <a name="web-farm-attacker-replays-request-to-multiple-nodes"></a><span data-ttu-id="f95b3-118">Web Farm: Solicitação de repetições do invasor para vários nós</span><span class="sxs-lookup"><span data-stu-id="f95b3-118">Web Farm: Attacker Replays Request to Multiple Nodes</span></span>  
 <span data-ttu-id="f95b3-119">Um cliente usa um serviço que é implementado em uma Web farm.</span><span class="sxs-lookup"><span data-stu-id="f95b3-119">A client uses a service that is implemented on a Web farm.</span></span> <span data-ttu-id="f95b3-120">Um invasor repetirá uma solicitação enviada a um nó no farm para outro nó no farm.</span><span class="sxs-lookup"><span data-stu-id="f95b3-120">An attacker replays a request that was sent to one node in the farm to another node in the farm.</span></span> <span data-ttu-id="f95b3-121">Além disso, se um serviço for reiniciado, o cache de reprodução é liberado, permitindo que um invasor repetir a solicitação.</span><span class="sxs-lookup"><span data-stu-id="f95b3-121">In addition, if a service is restarted, the replay cache is flushed, allowing an attacker to replay the request.</span></span> <span data-ttu-id="f95b3-122">(O cache contém valores de assinatura de mensagem usado, Vista anteriormente e impede a repetição para que essas assinaturas podem ser usadas apenas uma vez.</span><span class="sxs-lookup"><span data-stu-id="f95b3-122">(The cache contains used, previously seen message signature values and prevents replays so those signatures can be used only once.</span></span> <span data-ttu-id="f95b3-123">Reprodução caches não são compartilhados entre um Web farm.)</span><span class="sxs-lookup"><span data-stu-id="f95b3-123">Replay caches are not shared across a Web farm.)</span></span>  
  
 <span data-ttu-id="f95b3-124">Atenuantes incluem:</span><span class="sxs-lookup"><span data-stu-id="f95b3-124">Mitigations include:</span></span>  
  
-   <span data-ttu-id="f95b3-125">Use a segurança de modo de mensagem com tokens de contexto de segurança com monitoração de estado (com ou sem conversa segura habilitada).</span><span class="sxs-lookup"><span data-stu-id="f95b3-125">Use message mode security with stateful security context tokens (with or without secure conversation enabled).</span></span> <span data-ttu-id="f95b3-126">Para obter mais informações, confira [Como: Criar um contexto de segurança para uma sessão segura Token](../../../../docs/framework/wcf/feature-details/how-to-create-a-security-context-token-for-a-secure-session.md).</span><span class="sxs-lookup"><span data-stu-id="f95b3-126">For more information, see [How to: Create a Security Context Token for a Secure Session](../../../../docs/framework/wcf/feature-details/how-to-create-a-security-context-token-for-a-secure-session.md).</span></span>  
  
-   <span data-ttu-id="f95b3-127">Configure o serviço para usar a segurança de nível de transporte.</span><span class="sxs-lookup"><span data-stu-id="f95b3-127">Configure the service to use transport-level security.</span></span>
