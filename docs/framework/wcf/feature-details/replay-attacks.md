# <a name="replay-attacks"></a><span data-ttu-id="7ae80-102">Ataques por repetição</span><span class="sxs-lookup"><span data-stu-id="7ae80-102">Replay Attacks</span></span>
<span data-ttu-id="7ae80-103">Um *ataque de reprodução* ocorre quando um invasor copia um fluxo de mensagens entre duas partes e repete o fluxo para uma ou mais das partes.</span><span class="sxs-lookup"><span data-stu-id="7ae80-103">A *replay attack* occurs when an attacker copies a stream of messages between two parties and replays the stream to one or more of the parties.</span></span> <span data-ttu-id="7ae80-104">A menos que seja atenuado, os computadores sujeitos ao ataque processam o fluxo como mensagens legítimas, resultando em uma variedade de consequências inadequadas, como ordens redundantes de um item.</span><span class="sxs-lookup"><span data-stu-id="7ae80-104">Unless mitigated, the computers subject to the attack process the stream as legitimate messages, resulting in a range of bad consequences, such as redundant orders of an item.</span></span>  
  
## <a name="bindings-may-be-subject-to-reflection-attacks"></a><span data-ttu-id="7ae80-105">As associações podem estar sujeitas a ataques de reflexo</span><span class="sxs-lookup"><span data-stu-id="7ae80-105">Bindings May Be Subject to Reflection Attacks</span></span>  
 <span data-ttu-id="7ae80-106">Os *ataques de reflexo* são repetições de mensagens de volta para um remetente como se tivessem vindo do receptor como resposta.</span><span class="sxs-lookup"><span data-stu-id="7ae80-106">*Reflection attacks* are replays of messages back to a sender as if they came from the receiver as the reply.</span></span> <span data-ttu-id="7ae80-107">A *detecção de reprodução* padrão no mecanismo de Windows Communication Foundation (WCF) não trata isso automaticamente.</span><span class="sxs-lookup"><span data-stu-id="7ae80-107">The standard *replay detection* in the Windows Communication Foundation (WCF) mechanism does not automatically handle this.</span></span>  
  
 <span data-ttu-id="7ae80-108">Os ataques de reflexão são mitigados por padrão porque o modelo de serviço do WCF adiciona uma ID de mensagem assinada para solicitar mensagens e espera um `relates-to` cabeçalho assinado em mensagens de resposta.</span><span class="sxs-lookup"><span data-stu-id="7ae80-108">Reflection attacks are mitigated by default because the WCF service model adds a signed message ID to request messages and expects a signed `relates-to` header on response messages.</span></span> <span data-ttu-id="7ae80-109">Consequentemente, a mensagem de solicitação não pode ser repetida como uma resposta.</span><span class="sxs-lookup"><span data-stu-id="7ae80-109">Consequently, the request message cannot be replayed as a response.</span></span> <span data-ttu-id="7ae80-110">Em cenários de RM (mensagens confiáveis seguras), os ataques de reflexo são mitigados porque:</span><span class="sxs-lookup"><span data-stu-id="7ae80-110">In secure reliable message (RM) scenarios, reflection attacks are mitigated because:</span></span>  
  
- <span data-ttu-id="7ae80-111">Os esquemas de mensagem de resposta Create Sequence e Create Sequence são diferentes.</span><span class="sxs-lookup"><span data-stu-id="7ae80-111">The create sequence and create sequence response message schemas are different.</span></span>  
  
- <span data-ttu-id="7ae80-112">Para sequências simplex, as mensagens de sequência que o cliente envia não podem ser reproduzidas para ele porque o cliente não pode entender essas mensagens.</span><span class="sxs-lookup"><span data-stu-id="7ae80-112">For simplex sequences, sequence messages the client sends cannot be replayed back to it because the client cannot understand such messages.</span></span>  
  
- <span data-ttu-id="7ae80-113">Para sequências duplex, as duas IDs de sequência devem ser exclusivas.</span><span class="sxs-lookup"><span data-stu-id="7ae80-113">For duplex sequences, the two sequence IDs must be unique.</span></span> <span data-ttu-id="7ae80-114">Portanto, uma mensagem de sequência de saída não pode ser repetida como uma mensagem de sequência de entrada (todos os cabeçalhos e corpos de sequência também são assinados).</span><span class="sxs-lookup"><span data-stu-id="7ae80-114">Thus, an outgoing sequence message cannot be replayed back as an incoming sequence message (all sequence headers and bodies are signed, too).</span></span>  
  
 <span data-ttu-id="7ae80-115">As únicas associações que são suscetíveis a ataques de reflexão são aquelas sem WS-Addressing: associações personalizadas que têm o WS-Addressing desabilitado e usam a segurança baseada em chave simétrica.</span><span class="sxs-lookup"><span data-stu-id="7ae80-115">The only bindings that are susceptible to reflection attacks are those without WS-Addressing: custom bindings that have WS-Addressing disabled and use the symmetric key-based security.</span></span> <span data-ttu-id="7ae80-116">O <xref:System.ServiceModel.BasicHttpBinding> não usa o WS-Addressing por padrão, mas não usa a segurança baseada em chave simétrica de forma a permitir que ela fique vulnerável a esse ataque.</span><span class="sxs-lookup"><span data-stu-id="7ae80-116">The <xref:System.ServiceModel.BasicHttpBinding> does not use WS-Addressing by default, but it does not use symmetric key-based security in a way that allows it to be vulnerable to this attack.</span></span>  
  
 <span data-ttu-id="7ae80-117">A mitigação de associações personalizadas é não estabelecer o contexto de segurança ou exigir cabeçalhos WS-Addressing.</span><span class="sxs-lookup"><span data-stu-id="7ae80-117">The mitigation for custom bindings is to not establish security context or to require WS-Addressing headers.</span></span>  
  
## <a name="web-farm-attacker-replays-request-to-multiple-nodes"></a><span data-ttu-id="7ae80-118">Web farm: o invasor repete a solicitação para vários nós</span><span class="sxs-lookup"><span data-stu-id="7ae80-118">Web Farm: Attacker Replays Request to Multiple Nodes</span></span>  
 <span data-ttu-id="7ae80-119">Um cliente usa um serviço que é implementado em um Web farm.</span><span class="sxs-lookup"><span data-stu-id="7ae80-119">A client uses a service that is implemented on a Web farm.</span></span> <span data-ttu-id="7ae80-120">Um invasor repete uma solicitação que foi enviada para um nó no farm para outro nó no farm.</span><span class="sxs-lookup"><span data-stu-id="7ae80-120">An attacker replays a request that was sent to one node in the farm to another node in the farm.</span></span> <span data-ttu-id="7ae80-121">Além disso, se um serviço for reiniciado, o cache de reprodução será liberado, permitindo que um invasor reproduza a solicitação.</span><span class="sxs-lookup"><span data-stu-id="7ae80-121">In addition, if a service is restarted, the replay cache is flushed, allowing an attacker to replay the request.</span></span> <span data-ttu-id="7ae80-122">(O cache contém os valores de assinatura de mensagem usados anteriormente e que impede as repetições para que essas assinaturas possam ser usadas apenas uma vez.</span><span class="sxs-lookup"><span data-stu-id="7ae80-122">(The cache contains used, previously seen message signature values and prevents replays so those signatures can be used only once.</span></span> <span data-ttu-id="7ae80-123">Os caches de reprodução não são compartilhados em um Web farm.)</span><span class="sxs-lookup"><span data-stu-id="7ae80-123">Replay caches are not shared across a Web farm.)</span></span>  
  
 <span data-ttu-id="7ae80-124">Atenuantes incluem:</span><span class="sxs-lookup"><span data-stu-id="7ae80-124">Mitigations include:</span></span>  
  
- <span data-ttu-id="7ae80-125">Use a segurança do modo de mensagem com tokens de contexto de segurança com estado (com ou sem conversa segura habilitada).</span><span class="sxs-lookup"><span data-stu-id="7ae80-125">Use message mode security with stateful security context tokens (with or without secure conversation enabled).</span></span> <span data-ttu-id="7ae80-126">Para obter mais informações, consulte [como: criar um token de contexto de segurança para uma sessão segura](how-to-create-a-security-context-token-for-a-secure-session.md).</span><span class="sxs-lookup"><span data-stu-id="7ae80-126">For more information, see [How to: Create a Security Context Token for a Secure Session](how-to-create-a-security-context-token-for-a-secure-session.md).</span></span>  
  
- <span data-ttu-id="7ae80-127">Configure o serviço para usar a segurança em nível de transporte.</span><span class="sxs-lookup"><span data-stu-id="7ae80-127">Configure the service to use transport-level security.</span></span>
