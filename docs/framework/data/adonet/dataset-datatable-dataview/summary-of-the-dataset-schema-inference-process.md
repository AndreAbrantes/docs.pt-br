# <a name="summary-of-the-dataset-schema-inference-process"></a><span data-ttu-id="db3f3-102">Resumo do processo de inferência de esquema de conjunto de dados</span><span class="sxs-lookup"><span data-stu-id="db3f3-102">Summary of the DataSet Schema Inference Process</span></span>
<span data-ttu-id="db3f3-103">O processo de inferência primeiro determina, do documento XML, quais elementos serão inferidos como tabelas.</span><span class="sxs-lookup"><span data-stu-id="db3f3-103">The inference process first determines, from the XML document, which elements will be inferred as tables.</span></span> <span data-ttu-id="db3f3-104">O XML restantes, o processo de inferência determina as colunas para as tabelas.</span><span class="sxs-lookup"><span data-stu-id="db3f3-104">From the remaining XML, the inference process determines the columns for those tables.</span></span> <span data-ttu-id="db3f3-105">Para tabelas aninhadas, o processo de inferência gera aninhada <xref:System.Data.DataRelation> e <xref:System.Data.ForeignKeyConstraint> objetos.</span><span class="sxs-lookup"><span data-stu-id="db3f3-105">For nested tables, the inference process generates nested <xref:System.Data.DataRelation> and <xref:System.Data.ForeignKeyConstraint> objects.</span></span>  
  
 <span data-ttu-id="db3f3-106">A seguir é um resumo das regras de inferência de tipos:</span><span class="sxs-lookup"><span data-stu-id="db3f3-106">Following is a brief summary of inference rules:</span></span>  
  
-   <span data-ttu-id="db3f3-107">Elementos que têm atributos são inferidos como tabelas.</span><span class="sxs-lookup"><span data-stu-id="db3f3-107">Elements that have attributes are inferred as tables.</span></span>  
  
-   <span data-ttu-id="db3f3-108">Elementos que têm elementos filho são inferidos como tabelas.</span><span class="sxs-lookup"><span data-stu-id="db3f3-108">Elements that have child elements are inferred as tables.</span></span>  
  
-   <span data-ttu-id="db3f3-109">Elementos que se repetem são inferidos como uma única tabela.</span><span class="sxs-lookup"><span data-stu-id="db3f3-109">Elements that repeat are inferred as a single table.</span></span>  
  
-   <span data-ttu-id="db3f3-110">Se o elemento do documento ou raiz, nenhum atributo e não há elementos filho que poderiam ser inferidos como colunas, ele é inferido como um <xref:System.Data.DataSet>.</span><span class="sxs-lookup"><span data-stu-id="db3f3-110">If the document, or root, element has no attributes, and no child elements that would be inferred as columns, it is inferred as a <xref:System.Data.DataSet>.</span></span> <span data-ttu-id="db3f3-111">Caso contrário, o elemento do documento será inferido como uma tabela.</span><span class="sxs-lookup"><span data-stu-id="db3f3-111">Otherwise, the document element is inferred as a table.</span></span>  
  
-   <span data-ttu-id="db3f3-112">Atributos são inferidos como colunas.</span><span class="sxs-lookup"><span data-stu-id="db3f3-112">Attributes are inferred as columns.</span></span>  
  
-   <span data-ttu-id="db3f3-113">Elementos que não têm atributos ou elementos filho, e que não se repetem, são inferidos como colunas.</span><span class="sxs-lookup"><span data-stu-id="db3f3-113">Elements that have no attributes or child elements, and that do not repeat, are inferred as columns.</span></span>  
  
-   <span data-ttu-id="db3f3-114">Para elementos que são inferidos como tabelas aninhadas dentro de outros elementos que também são inferidos como tabelas, uma aninhada **DataRelation** é criada entre as duas tabelas.</span><span class="sxs-lookup"><span data-stu-id="db3f3-114">For elements that are inferred as nested tables within other elements that are also inferred as tables, a nested **DataRelation** is created between the two tables.</span></span> <span data-ttu-id="db3f3-115">Uma coluna de chave primária, novo chamada **TableName_Id** é adicionado a ambas as tabelas e usado pelo **DataRelation**.</span><span class="sxs-lookup"><span data-stu-id="db3f3-115">A new, primary key column named **TableName_Id** is added to both tables and used by the **DataRelation**.</span></span> <span data-ttu-id="db3f3-116">Um **ForeignKeyConstraint** é criada entre as duas tabelas usando o **TableName_Id** coluna.</span><span class="sxs-lookup"><span data-stu-id="db3f3-116">A **ForeignKeyConstraint** is created between the two tables using the **TableName_Id** column.</span></span>  
  
-   <span data-ttu-id="db3f3-117">Para elementos que são inferidos como tabelas e que contêm texto, mas que nenhum elemento filho, uma nova coluna chamada **TableName_Text** é criado para o texto de cada um dos elementos.</span><span class="sxs-lookup"><span data-stu-id="db3f3-117">For elements that are inferred as tables and that contain text but have no child elements, a new column named **TableName_Text** is created for the text of each of the elements.</span></span> <span data-ttu-id="db3f3-118">Se um elemento é inferido como uma tabela e tem o texto, mas também tem elementos filho, o texto será ignorado.</span><span class="sxs-lookup"><span data-stu-id="db3f3-118">If an element is inferred as a table and has text, but also has child elements, the text is ignored.</span></span>
