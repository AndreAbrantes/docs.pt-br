# <a name="navigation-property"></a><span data-ttu-id="e3450-102">propriedade de navegação</span><span class="sxs-lookup"><span data-stu-id="e3450-102">navigation property</span></span>
<span data-ttu-id="e3450-103">Um *propriedade de navegação* é uma propriedade opcional em uma [tipo de entidade](../../../../docs/framework/data/adonet/entity-type.md) que permite a navegação de um [final](../../../../docs/framework/data/adonet/association-end.md) de um [associação](../../../../docs/framework/data/adonet/association-type.md) para a outra extremidade.</span><span class="sxs-lookup"><span data-stu-id="e3450-103">A *navigation property* is an optional property on an [entity type](../../../../docs/framework/data/adonet/entity-type.md) that allows for navigation from one [end](../../../../docs/framework/data/adonet/association-end.md) of an [association](../../../../docs/framework/data/adonet/association-type.md) to the other end.</span></span> <span data-ttu-id="e3450-104">Ao contrário de outras [propriedades](../../../../docs/framework/data/adonet/property.md), propriedades de navegação não contêm dados.</span><span class="sxs-lookup"><span data-stu-id="e3450-104">Unlike other [properties](../../../../docs/framework/data/adonet/property.md), navigation properties do not carry data.</span></span>  
  
 <span data-ttu-id="e3450-105">Uma definição de propriedade de navegação inclui o seguinte:</span><span class="sxs-lookup"><span data-stu-id="e3450-105">A navigation property definition includes the following:</span></span>  
  
-   <span data-ttu-id="e3450-106">Um nome.</span><span class="sxs-lookup"><span data-stu-id="e3450-106">A name.</span></span> <span data-ttu-id="e3450-107">(Necessário)</span><span class="sxs-lookup"><span data-stu-id="e3450-107">(Required)</span></span>  
  
-   <span data-ttu-id="e3450-108">A associação que navega.</span><span class="sxs-lookup"><span data-stu-id="e3450-108">The association that it navigates.</span></span> <span data-ttu-id="e3450-109">(Necessário)</span><span class="sxs-lookup"><span data-stu-id="e3450-109">(Required)</span></span>  
  
-   <span data-ttu-id="e3450-110">Termina de associação que navega.</span><span class="sxs-lookup"><span data-stu-id="e3450-110">The ends of the association that it navigates.</span></span> <span data-ttu-id="e3450-111">(Necessário)</span><span class="sxs-lookup"><span data-stu-id="e3450-111">(Required)</span></span>  
  
 <span data-ttu-id="e3450-112">Observe que as propriedades de navegação são opcionais em ambos os tipos de entidade termina de uma associação.</span><span class="sxs-lookup"><span data-stu-id="e3450-112">Note that navigation properties are optional on both entity types at the ends of an association.</span></span> <span data-ttu-id="e3450-113">Se você definir uma propriedade de navegação em um tipo de entidade no final de uma associação, você não precisa definir uma propriedade de navegação no tipo de entidade no outro extremo de associação.</span><span class="sxs-lookup"><span data-stu-id="e3450-113">If you define a navigation property on one entity type at the end of an association, you do not have to define a navigation property on the entity type at the other end of the association.</span></span>  
  
 <span data-ttu-id="e3450-114">O tipo de dados de uma propriedade de navegação é determinado pelo [multiplicidade](../../../../docs/framework/data/adonet/association-end-multiplicity.md) do seu remote [end de associação](../../../../docs/framework/data/adonet/association-end.md).</span><span class="sxs-lookup"><span data-stu-id="e3450-114">The data type of a navigation property is determined by the [multiplicity](../../../../docs/framework/data/adonet/association-end-multiplicity.md) of its remote [association end](../../../../docs/framework/data/adonet/association-end.md).</span></span> <span data-ttu-id="e3450-115">Por exemplo, suponha uma propriedade de navegação, `OrdersNavProp`, existe em um tipo de entidade de `Customer` e navegar em um para muitos associação entre `Customer` e `Order`.</span><span class="sxs-lookup"><span data-stu-id="e3450-115">For example, suppose a navigation property, `OrdersNavProp`, exists on a `Customer` entity type and navigates a one-to-many association between `Customer` and `Order`.</span></span> <span data-ttu-id="e3450-116">Porque o fim remoto da associação para a propriedade de navegação tem a multiplicidade de muitos (\*), seu tipo de dados é uma coleção (de `Order`).</span><span class="sxs-lookup"><span data-stu-id="e3450-116">Because the remote association end for the navigation property has multiplicity of many (\*), its data type is a collection (of `Order`).</span></span> <span data-ttu-id="e3450-117">Da mesma forma, se uma propriedade de navegação, `CustomerNavProp`, existe no tipo de entidade de `Order` , seu tipo de dados deve ser `Customer`, porque a multiplicidade de extremidade remoto é um (1).</span><span class="sxs-lookup"><span data-stu-id="e3450-117">Similarly, if a navigation property, `CustomerNavProp`, exists on the `Order` entity type, its data type would be `Customer`, because the multiplicity of the remote end is one (1).</span></span>
