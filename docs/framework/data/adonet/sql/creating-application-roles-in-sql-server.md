## <a name="application-role-features"></a><span data-ttu-id="87f3a-109">Recursos da função de aplicativo</span><span class="sxs-lookup"><span data-stu-id="87f3a-109">Application Role Features</span></span>  
 <span data-ttu-id="87f3a-110">As funções de aplicativo têm os seguintes recursos:</span><span class="sxs-lookup"><span data-stu-id="87f3a-110">Application roles have the following features:</span></span>  
  
-   <span data-ttu-id="87f3a-111">Ao contrário das funções de banco de dados, as funções de aplicativo não contêm nenhum membro.</span><span class="sxs-lookup"><span data-stu-id="87f3a-111">Unlike database roles, application roles contain no members.</span></span>  
  
-   <span data-ttu-id="87f3a-112">As funções de aplicativo são ativadas quando um aplicativo fornece o nome da função de aplicativo e uma senha para o procedimento armazenado do sistema `sp_setapprole`.</span><span class="sxs-lookup"><span data-stu-id="87f3a-112">Application roles are activated when an application supplies the application role name and a password to the `sp_setapprole` system stored procedure.</span></span>  
  
-   <span data-ttu-id="87f3a-113">A senha deve ser armazenado no computador cliente e ser fornecida em tempo de execução; uma função de aplicativo não pode ser ativada de dentro do SQL Server.</span><span class="sxs-lookup"><span data-stu-id="87f3a-113">The password must be stored on the client computer and supplied at run time; an application role cannot be activated from inside of SQL Server.</span></span>  
  
-   <span data-ttu-id="87f3a-114">A senha não é criptografada.</span><span class="sxs-lookup"><span data-stu-id="87f3a-114">The password is not encrypted.</span></span> <span data-ttu-id="87f3a-115">A senha do parâmetro é armazenada como um hash unidirecional.</span><span class="sxs-lookup"><span data-stu-id="87f3a-115">The parameter password is stored as a one-way hash.</span></span>  
  
-   <span data-ttu-id="87f3a-116">Quando forem ativadas, as permissões adquiridas por meio da função de aplicativo permanecem aplicadas para a duração da conexão.</span><span class="sxs-lookup"><span data-stu-id="87f3a-116">Once activated, permissions acquired through the application role remain in effect for the duration of the connection.</span></span>  
  
-   <span data-ttu-id="87f3a-117">A função de aplicativo herda as permissões concedidas para a função `public`.</span><span class="sxs-lookup"><span data-stu-id="87f3a-117">The application role inherits permissions granted to the `public` role.</span></span>  
  
-   <span data-ttu-id="87f3a-118">Se um membro da função de servidor fixa do `sysadmin` ativar uma função de aplicativo, o contexto de segurança mudará para a função de aplicativo para a duração da conexão.</span><span class="sxs-lookup"><span data-stu-id="87f3a-118">If a member of the `sysadmin` fixed server role activates an application role, the security context switches to that of the application role for the duration of the connection.</span></span>  
  
-   <span data-ttu-id="87f3a-119">Se você criar uma conta de `guest` em um banco de dados que tenha uma função de aplicativo, não precisará criar um usuário de banco de dados para a função de aplicativo ou para nenhum dos logons que a chamam.</span><span class="sxs-lookup"><span data-stu-id="87f3a-119">If you create a `guest` account in a database that has an application role, you do not need to create a database user account for the application role or for any of the logins that invoke it.</span></span> <span data-ttu-id="87f3a-120">As funções de aplicativo poderão acessar diretamente outro banco de dados somente se uma conta de `guest` existir no segundo banco de dados</span><span class="sxs-lookup"><span data-stu-id="87f3a-120">Application roles can directly access another database only if a `guest` account exists in the second database</span></span>  
  
-   <span data-ttu-id="87f3a-121">As funções internas que retornam os nomes de logon, como SYSTEM_USER, retornam o nome do logon que chamou a função de aplicativo.</span><span class="sxs-lookup"><span data-stu-id="87f3a-121">Built-in functions that return login names, such as SYSTEM_USER, return the name of the login that invoked the application role.</span></span> <span data-ttu-id="87f3a-122">As funções internas que retornam os nomes de usuário de banco de dados retornam o nome da função de aplicativo.</span><span class="sxs-lookup"><span data-stu-id="87f3a-122">Built-in functions that return database user names return the name of the application role.</span></span>  
  
### <a name="the-principle-of-least-privilege"></a><span data-ttu-id="87f3a-123">O princípio de privilégios mínimos</span><span class="sxs-lookup"><span data-stu-id="87f3a-123">The Principle of Least Privilege</span></span>  
 <span data-ttu-id="87f3a-124">As funções de aplicativo devem receber somente as permissões necessárias no caso de a senha ser comprometida.</span><span class="sxs-lookup"><span data-stu-id="87f3a-124">Application roles should be granted only required permissions in case the password is compromised.</span></span> <span data-ttu-id="87f3a-125">As permissões para a função `public` devem ser revogadas em qualquer banco de dados usando uma função de aplicativo.</span><span class="sxs-lookup"><span data-stu-id="87f3a-125">Permissions to the `public` role should be revoked in any database using an application role.</span></span> <span data-ttu-id="87f3a-126">Desative a conta de `guest` em qualquer banco de dados ao qual você não queira que chamadores da função de aplicativo tenham acesso.</span><span class="sxs-lookup"><span data-stu-id="87f3a-126">Disable the `guest` account in any database you do not want callers of the application role to have access to.</span></span>  
  
### <a name="application-role-enhancements"></a><span data-ttu-id="87f3a-127">Aprimoramentos da função de aplicativo</span><span class="sxs-lookup"><span data-stu-id="87f3a-127">Application Role Enhancements</span></span>  
 <span data-ttu-id="87f3a-128">O contexto de execução pode ser alternado de volta para o chamador original após ter ativado uma função do aplicativo, eliminando a necessidade de desativar o pool de conexões.</span><span class="sxs-lookup"><span data-stu-id="87f3a-128">The execution context can be switched back to the original caller after activating an application role, removing the need to disable connection pooling.</span></span> <span data-ttu-id="87f3a-129">O procedimento `sp_setapprole` tem uma nova opção que cria um cookie, que contém informações de contexto sobre o chamador.</span><span class="sxs-lookup"><span data-stu-id="87f3a-129">The `sp_setapprole` procedure has a new option that creates a cookie, which contains context information about the caller.</span></span> <span data-ttu-id="87f3a-130">Você pode reverter a sessão chamando o procedimento `sp_unsetapprole`, passando o cookie.</span><span class="sxs-lookup"><span data-stu-id="87f3a-130">You can revert the session by calling the `sp_unsetapprole` procedure, passing it the cookie.</span></span>
