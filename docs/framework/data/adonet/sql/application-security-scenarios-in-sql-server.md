## <a name="common-threats"></a><span data-ttu-id="e4203-109">Ameaças comuns</span><span class="sxs-lookup"><span data-stu-id="e4203-109">Common Threats</span></span>

### <a name="sql-injection"></a><span data-ttu-id="e4203-113">Injeção de SQL</span><span class="sxs-lookup"><span data-stu-id="e4203-113">SQL Injection</span></span>

 <span data-ttu-id="e4203-118">Os ataques de elevação de privilégio ocorrem quando um usuário é capaz de assumir os privilégios de uma conta confiável, como um proprietário ou administrador.</span><span class="sxs-lookup"><span data-stu-id="e4203-118">Elevation of privilege attacks occur when a user is able to assume the privileges of a trusted account, such as an owner or administrator.</span></span> <span data-ttu-id="e4203-119">Sempre execute em contas de usuário com o mínimo de privilégios e atribua apenas as permissões necessárias.</span><span class="sxs-lookup"><span data-stu-id="e4203-119">Always run under least-privileged user accounts and assign only needed permissions.</span></span> <span data-ttu-id="e4203-120">Evite usar contas administrativas ou de proprietário para executar o código.</span><span class="sxs-lookup"><span data-stu-id="e4203-120">Avoid using administrative or owner accounts for executing code.</span></span> <span data-ttu-id="e4203-121">Isso limitará a quantidade de danos que podem ocorrer se um ataque for realizado com sucesso.</span><span class="sxs-lookup"><span data-stu-id="e4203-121">This limits the amount of damage that can occur if an attack succeeds.</span></span> <span data-ttu-id="e4203-122">Ao executar tarefas que exigem permissões adicionais, use a assinatura ou representação de procedimento somente pela duração da tarefa.</span><span class="sxs-lookup"><span data-stu-id="e4203-122">When performing tasks that require additional permissions, use procedure signing or impersonation only for the duration of the task.</span></span> <span data-ttu-id="e4203-123">Você pode assinar procedimentos armazenados com certificados ou usar a representação para atribuir permissões temporariamente.</span><span class="sxs-lookup"><span data-stu-id="e4203-123">You can sign stored procedures with certificates or use impersonation to temporarily assign permissions.</span></span>

 <span data-ttu-id="e4203-128">Um ataque de injeção de cadeia de conexão pode ocorrer ao usar logons do SQL Server se uma cadeia de conexão baseada na entrada do usuário for construída em tempo de execução.</span><span class="sxs-lookup"><span data-stu-id="e4203-128">A connection string injection attack can occur when using SQL Server logins if a connection string based on user input is constructed at run time.</span></span> <span data-ttu-id="e4203-129">Se a cadeia de conexão não for verificada em busca de pares de palavras-chave válidos, um invasor poderá inserir caracteres extras, potencialmente acessando dados confidenciais ou outros recursos no servidor.</span><span class="sxs-lookup"><span data-stu-id="e4203-129">If the connection string is not checked for valid keyword pairs, an attacker can insert extra characters, potentially accessing sensitive data or other resources on the server.</span></span> <span data-ttu-id="e4203-130">Use a autenticação do Windows sempre que possível.</span><span class="sxs-lookup"><span data-stu-id="e4203-130">Use Windows authentication wherever possible.</span></span> <span data-ttu-id="e4203-131">Se você precisar usar logons do SQL Server, use o <xref:System.Data.SqlClient.SqlConnectionStringBuilder> para criar e validar cadeias de conexão em tempo de execução.</span><span class="sxs-lookup"><span data-stu-id="e4203-131">If you must use SQL Server logins, use the <xref:System.Data.SqlClient.SqlConnectionStringBuilder> to create and validate connection strings at run time.</span></span>
