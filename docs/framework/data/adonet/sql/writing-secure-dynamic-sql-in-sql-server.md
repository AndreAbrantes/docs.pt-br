# <a name="writing-secure-dynamic-sql-in-sql-server"></a><span data-ttu-id="8afdf-102">Escrevendo SQL dinâmico seguro no SQL Server</span><span class="sxs-lookup"><span data-stu-id="8afdf-102">Writing Secure Dynamic SQL in SQL Server</span></span>

 <span data-ttu-id="8afdf-105">Qualquer procedimento que construa instruções SQL deve ser verificado quanto a vulnerabilidades de injeção porque o SQL Server executará todas as consultas sintaticamente válidas que receber.</span><span class="sxs-lookup"><span data-stu-id="8afdf-105">Any procedure that constructs SQL statements should be reviewed for injection vulnerabilities because SQL Server will execute all syntactically valid queries that it receives.</span></span> <span data-ttu-id="8afdf-106">Mesmo dados com parâmetros podem ser manipulados por um invasor qualificado e determinado.</span><span class="sxs-lookup"><span data-stu-id="8afdf-106">Even parameterized data can be manipulated by a skilled and determined attacker.</span></span> <span data-ttu-id="8afdf-107">Se você usar SQL dinâmico, não deixe de parametrizar seus comandos e nunca inclua valores de parâmetro diretamente na cadeia de caracteres de consulta.</span><span class="sxs-lookup"><span data-stu-id="8afdf-107">If you use dynamic SQL, be sure to parameterize your commands, and never include parameter values directly into the query string.</span></span>  
  
## <a name="anatomy-of-a-sql-injection-attack"></a><span data-ttu-id="8afdf-108">Anatomia de um ataque de injeção de SQL</span><span class="sxs-lookup"><span data-stu-id="8afdf-108">Anatomy of a SQL Injection Attack</span></span>  

 <span data-ttu-id="8afdf-109">O processo de injeção funciona encerrando prematuramente uma cadeia de caracteres de texto e anexando um novo comando.</span><span class="sxs-lookup"><span data-stu-id="8afdf-109">The injection process works by prematurely terminating a text string and appending a new command.</span></span> <span data-ttu-id="8afdf-110">Como o comando inserido pode ter cadeias de caracteres adicionais anexadas a ele antes de ser executado, o malfeitor encerra a cadeia de caracteres injetada com uma marca de comentário "--".</span><span class="sxs-lookup"><span data-stu-id="8afdf-110">Because the inserted command may have additional strings appended to it before it is executed, the malefactor terminates the injected string with a comment mark "--".</span></span> <span data-ttu-id="8afdf-111">O texto subsequente é ignorado no momento da execução.</span><span class="sxs-lookup"><span data-stu-id="8afdf-111">Subsequent text is ignored at execution time.</span></span> <span data-ttu-id="8afdf-112">Vários comandos podem ser inseridos usando um delimitador de ponto e vírgula (;).</span><span class="sxs-lookup"><span data-stu-id="8afdf-112">Multiple commands can be inserted using a semicolon (;) delimiter.</span></span>  
  
 <span data-ttu-id="8afdf-113">Contanto que o código SQL injetado esteja sintaticamente correto, a violação não poderá ser detectada programaticamente.</span><span class="sxs-lookup"><span data-stu-id="8afdf-113">As long as injected SQL code is syntactically correct, tampering cannot be detected programmatically.</span></span> <span data-ttu-id="8afdf-114">Portanto, é necessário validar todas as entradas de usuário e verificar com cuidado o código que executa comandos SQL construídos no servidor que você está usando.</span><span class="sxs-lookup"><span data-stu-id="8afdf-114">Therefore, you must validate all user input and carefully review code that executes constructed SQL commands in the server that you are using.</span></span> <span data-ttu-id="8afdf-115">Nunca concatene entrada de usuário que não seja validada.</span><span class="sxs-lookup"><span data-stu-id="8afdf-115">Never concatenate user input that is not validated.</span></span> <span data-ttu-id="8afdf-116">A concatenação de cadeia de caracteres é o ponto principal de entrada de injeção de script.</span><span class="sxs-lookup"><span data-stu-id="8afdf-116">String concatenation is the primary point of entry for script injection.</span></span>  
  
 <span data-ttu-id="8afdf-117">Veja algumas diretrizes úteis:</span><span class="sxs-lookup"><span data-stu-id="8afdf-117">Here are some helpful guidelines:</span></span>  
  
- <span data-ttu-id="8afdf-118">Nunca compile instruções Transact-SQL diretamente da entrada do usuário; use procedimentos armazenados para validar a entrada do usuário.</span><span class="sxs-lookup"><span data-stu-id="8afdf-118">Never build Transact-SQL statements directly from user input; use stored procedures to validate user input.</span></span>  
  
- <span data-ttu-id="8afdf-119">Valide entrada de usuário testando tipo, comprimento, formato e intervalo.</span><span class="sxs-lookup"><span data-stu-id="8afdf-119">Validate user input by testing type, length, format, and range.</span></span> <span data-ttu-id="8afdf-120">Use a função QUOTENAME() do Transact-SQL para escapar nomes do sistema ou a função REPLACE() para escapar qualquer caractere em uma cadeia de caracteres.</span><span class="sxs-lookup"><span data-stu-id="8afdf-120">Use the Transact-SQL QUOTENAME() function to escape system names or the REPLACE() function to escape any character in a string.</span></span>  
  
- <span data-ttu-id="8afdf-121">Implemente várias camadas de validação em cada nível do aplicativo.</span><span class="sxs-lookup"><span data-stu-id="8afdf-121">Implement multiple layers of validation in each tier of your application.</span></span>  
  
- <span data-ttu-id="8afdf-122">Teste o tamanho e tipo de dados de entrada e imponha limites apropriados.</span><span class="sxs-lookup"><span data-stu-id="8afdf-122">Test the size and data type of input and enforce appropriate limits.</span></span> <span data-ttu-id="8afdf-123">Isso pode ajudar a impedir excesso deliberado de buffer.</span><span class="sxs-lookup"><span data-stu-id="8afdf-123">This can help prevent deliberate buffer overruns.</span></span>  
  
- <span data-ttu-id="8afdf-124">Teste o conteúdo de variáveis de cadeia de caracteres e só aceite valores esperados.</span><span class="sxs-lookup"><span data-stu-id="8afdf-124">Test the content of string variables and accept only expected values.</span></span> <span data-ttu-id="8afdf-125">Rejeite entradas que contenham dados binários, sequências de escape e caracteres de comentário.</span><span class="sxs-lookup"><span data-stu-id="8afdf-125">Reject entries that contain binary data, escape sequences, and comment characters.</span></span>  
  
- <span data-ttu-id="8afdf-126">Quando você estiver trabalhando com documentos XML, valide todos os dados em seu esquema à medida que são inseridos.</span><span class="sxs-lookup"><span data-stu-id="8afdf-126">When you are working with XML documents, validate all data against its schema as it is entered.</span></span>  
  
- <span data-ttu-id="8afdf-127">Em ambientes de várias camadas, todos os dados devem ser validados antes de serem admitidos à zona confiável.</span><span class="sxs-lookup"><span data-stu-id="8afdf-127">In multi-tiered environments, all data should be validated before admission to the trusted zone.</span></span>  
  
- <span data-ttu-id="8afdf-128">Não aceite as seguintes cadeias de caracteres nos campos dos quais os nomes de arquivo podem ser criados: AUX, CLOCK$, COM1 a COM8, CON, CONFIG$, LPT1 a LPT8, NUL e PRN.</span><span class="sxs-lookup"><span data-stu-id="8afdf-128">Do not accept the following strings in fields from which file names can be constructed: AUX, CLOCK$, COM1 through COM8, CON, CONFIG$, LPT1 through LPT8, NUL, and PRN.</span></span>  
  
- <span data-ttu-id="8afdf-129">Use objetos <xref:System.Data.SqlClient.SqlParameter> com procedimentos armazenados e comandos para fornecer verificação de tipo e validação de comprimento.</span><span class="sxs-lookup"><span data-stu-id="8afdf-129">Use <xref:System.Data.SqlClient.SqlParameter> objects with stored procedures and commands to provide type checking and length validation.</span></span>  
  
- <span data-ttu-id="8afdf-130">Use expressões <xref:System.Text.RegularExpressions.Regex> no código do cliente para filtrar caracteres inválidos.</span><span class="sxs-lookup"><span data-stu-id="8afdf-130">Use <xref:System.Text.RegularExpressions.Regex> expressions in client code to filter invalid characters.</span></span>  
  
## <a name="dynamic-sql-strategies"></a><span data-ttu-id="8afdf-131">Estratégias dinâmicas do SQL</span><span class="sxs-lookup"><span data-stu-id="8afdf-131">Dynamic SQL Strategies</span></span>  

 <span data-ttu-id="8afdf-132">A execução de instruções SQL criadas dinamicamente no seu código de procedimento quebra a cadeia de propriedade, fazendo com que SQL Server verifique as permissões do chamador em relação aos objetos que estão sendo acessados pelo SQL dinâmico.</span><span class="sxs-lookup"><span data-stu-id="8afdf-132">Executing dynamically created SQL statements in your procedural code breaks the ownership chain, causing SQL Server to check the permissions of the caller against the objects being accessed by the dynamic SQL.</span></span>
