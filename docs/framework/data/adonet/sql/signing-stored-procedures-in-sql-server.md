# <a name="signing-stored-procedures-in-sql-server"></a><span data-ttu-id="bb5c2-102">Assinando procedimentos armazenados no SQL Server</span><span class="sxs-lookup"><span data-stu-id="bb5c2-102">Signing Stored Procedures in SQL Server</span></span>

<span data-ttu-id="bb5c2-103">A assinatura digital é um resumo de dados criptografados com a chave privada do signatário.</span><span class="sxs-lookup"><span data-stu-id="bb5c2-103">A digital signature is a data digest encrypted with the private key of the signer.</span></span> <span data-ttu-id="bb5c2-104">A chave privada assegura que a assinatura digital seja exclusiva de seu portador ou proprietário.</span><span class="sxs-lookup"><span data-stu-id="bb5c2-104">The private key ensures that the digital signature is unique to its bearer or owner.</span></span> <span data-ttu-id="bb5c2-105">Você pode assinar procedimentos armazenados, funções (exceto para funções com valor de tabela embutida), gatilhos e assemblies.</span><span class="sxs-lookup"><span data-stu-id="bb5c2-105">You can sign stored procedures, functions (except for inline table-valued functions), triggers, and assemblies.</span></span>

<span data-ttu-id="bb5c2-106">Você pode assinar um procedimento armazenado com um certificado ou uma chave assimétrica.</span><span class="sxs-lookup"><span data-stu-id="bb5c2-106">You can sign a stored procedure with a certificate or an asymmetric key.</span></span> <span data-ttu-id="bb5c2-107">Isso é criado para cenários quando as permissões não podem ser herdadas pelo encadeamento de propriedade ou quando a cadeia de propriedade é quebrada, como SQL dinâmico.</span><span class="sxs-lookup"><span data-stu-id="bb5c2-107">This is designed for scenarios when permissions cannot be inherited through ownership chaining or when the ownership chain is broken, such as dynamic SQL.</span></span> <span data-ttu-id="bb5c2-108">Em seguida, você pode criar um usuário mapeado para o certificado, concedendo permissões de usuário de certificado nos objetos que o procedimento armazenado precisa acessar.</span><span class="sxs-lookup"><span data-stu-id="bb5c2-108">You can then create a user mapped to the certificate, granting the certificate user permissions on the objects the stored procedure needs to access.</span></span>

<span data-ttu-id="bb5c2-109">Você também pode criar um logon mapeado para o mesmo certificado e, em seguida, conceder permissões de nível de servidor necessárias para esse logon ou adicionar o logon a uma ou mais das funções de servidor fixas.</span><span class="sxs-lookup"><span data-stu-id="bb5c2-109">You can also create a login mapped to the same certificate, and then grant any necessary server-level permissions to that login, or add the login to one or more of the fixed server roles.</span></span> <span data-ttu-id="bb5c2-110">Isso é projetado para evitar a habilitação da configuração `TRUSTWORTHY` banco de dados para cenários nos quais são necessárias permissões de nível superior.</span><span class="sxs-lookup"><span data-stu-id="bb5c2-110">This is designed to avoid enabling the `TRUSTWORTHY` database setting for scenarios in which higher level permissions are needed.</span></span>

<span data-ttu-id="bb5c2-111">Quando o procedimento armazenado é executado, SQL Server combina as permissões do usuário do certificado e/ou o logon com aqueles do chamador.</span><span class="sxs-lookup"><span data-stu-id="bb5c2-111">When the stored procedure is executed, SQL Server combines the permissions of the certificate user and/or login with those of the caller.</span></span> <span data-ttu-id="bb5c2-112">Ao contrário da cláusula `EXECUTE AS`, ela não altera o contexto de execução do procedimento.</span><span class="sxs-lookup"><span data-stu-id="bb5c2-112">Unlike the `EXECUTE AS` clause, it does not change the execution context of the procedure.</span></span> <span data-ttu-id="bb5c2-113">As funções internas que retornam o logon e os nomes de usuário retornam o nome do chamador, não o nome do usuário do certificado.</span><span class="sxs-lookup"><span data-stu-id="bb5c2-113">Built-in functions that return login and user names return the name of the caller, not the certificate user name.</span></span>

## <a name="creating-certificates"></a><span data-ttu-id="bb5c2-114">Criando certificados</span><span class="sxs-lookup"><span data-stu-id="bb5c2-114">Creating Certificates</span></span>

<span data-ttu-id="bb5c2-115">Quando você assina um procedimento armazenado com um certificado ou uma chave assimétrica, um resumo de dados que consiste no hash criptografado do código de procedimento armazenado, juntamente com o usuário execute-as, é criado usando a chave privada.</span><span class="sxs-lookup"><span data-stu-id="bb5c2-115">When you sign a stored procedure with a certificate or asymmetric key, a data digest consisting of the encrypted hash of the stored procedure code, along with the execute-as user, is created using the private key.</span></span> <span data-ttu-id="bb5c2-116">Em tempo de execução, o resumo dos dados é criptografado com a chave pública e comparado com o valor de hash do procedimento armazenado.</span><span class="sxs-lookup"><span data-stu-id="bb5c2-116">At run time, the data digest is decrypted with the public key and compared with the hash value of the stored procedure.</span></span> <span data-ttu-id="bb5c2-117">Alterar o usuário executar como invalida o valor de hash para que a assinatura digital não corresponda mais.</span><span class="sxs-lookup"><span data-stu-id="bb5c2-117">Changing the execute-as user invalidates the hash value so that the digital signature no longer matches.</span></span> <span data-ttu-id="bb5c2-118">A modificação do procedimento armazenado descarta totalmente a assinatura, o que impede que alguém que não tenha acesso à chave privada altere o código do procedimento armazenado.</span><span class="sxs-lookup"><span data-stu-id="bb5c2-118">Modifying the stored procedure drops the signature entirely, which prevents someone who does not have access to the private key from changing the stored procedure code.</span></span> <span data-ttu-id="bb5c2-119">Em ambos os casos, você deve assinar novamente o procedimento sempre que alterar o código ou o usuário executar como.</span><span class="sxs-lookup"><span data-stu-id="bb5c2-119">In either case, you must re-sign the procedure each time you change the code or the execute-as user.</span></span>

<span data-ttu-id="bb5c2-120">Há duas etapas necessárias envolvidas na assinatura de um módulo:</span><span class="sxs-lookup"><span data-stu-id="bb5c2-120">There are two required steps involved in signing a module:</span></span>

1. <span data-ttu-id="bb5c2-121">Criar um certificado usando a instrução Transact-SQL `CREATE CERTIFICATE [certificateName]`.</span><span class="sxs-lookup"><span data-stu-id="bb5c2-121">Create a certificate using the Transact-SQL `CREATE CERTIFICATE [certificateName]` statement.</span></span> <span data-ttu-id="bb5c2-122">Essa instrução tem várias opções para definir uma data inicial e final e uma senha.</span><span class="sxs-lookup"><span data-stu-id="bb5c2-122">This statement has several options for setting a start and end date and a password.</span></span> <span data-ttu-id="bb5c2-123">A data de validade padrão é de um ano.</span><span class="sxs-lookup"><span data-stu-id="bb5c2-123">The default expiration date is one year.</span></span>

1. <span data-ttu-id="bb5c2-124">Assinar o procedimento armazenado com o certificado usando a instrução Transact-SQL `ADD SIGNATURE TO [procedureName] BY CERTIFICATE [certificateName]`.</span><span class="sxs-lookup"><span data-stu-id="bb5c2-124">Sign the procedure with the certificate using the Transact-SQL `ADD SIGNATURE TO [procedureName] BY CERTIFICATE [certificateName]` statement.</span></span>

<span data-ttu-id="bb5c2-125">Depois que o módulo tiver sido assinado, uma ou mais entidades de segurança precisarão ser criadas para manter as permissões adicionais que devem ser associadas ao certificado.</span><span class="sxs-lookup"><span data-stu-id="bb5c2-125">Once the module has been signed, one or more principals needs to be created in order to hold the additional permissions that should be associated with the certificate.</span></span>

<span data-ttu-id="bb5c2-126">Se o módulo precisar de permissões adicionais no nível do banco de dados:</span><span class="sxs-lookup"><span data-stu-id="bb5c2-126">If the module needs additional database-level permissions:</span></span>

1. <span data-ttu-id="bb5c2-127">Crie um usuário de banco de dados associado ao certificado usando a instrução Transact-SQL `CREATE USER [userName] FROM CERTIFICATE [certificateName]`.</span><span class="sxs-lookup"><span data-stu-id="bb5c2-127">Create a database user associated with that certificate using the Transact-SQL `CREATE USER [userName] FROM CERTIFICATE [certificateName]` statement.</span></span> <span data-ttu-id="bb5c2-128">Esse usuário existe somente no banco de dados e não está associado a um logon, a menos que um logon também tenha sido criado a partir desse mesmo certificado.</span><span class="sxs-lookup"><span data-stu-id="bb5c2-128">This user exists in the database only and is not associated with a login unless a login has also been created from that same certificate.</span></span>

1. <span data-ttu-id="bb5c2-129">Conceda ao usuário do certificado as permissões de nível de banco de dados necessárias.</span><span class="sxs-lookup"><span data-stu-id="bb5c2-129">Grant the certificate user the required database-level permissions.</span></span>

<span data-ttu-id="bb5c2-130">Se o módulo precisar de permissões adicionais no nível do servidor:</span><span class="sxs-lookup"><span data-stu-id="bb5c2-130">If the module needs additional server-level permissions:</span></span>

1. <span data-ttu-id="bb5c2-131">Copie o certificado para o banco de dados `master`.</span><span class="sxs-lookup"><span data-stu-id="bb5c2-131">Copy the certificate to the `master` database.</span></span>

1. <span data-ttu-id="bb5c2-132">Crie um logon associado a esse certificado usando a instrução Transact-SQL `CREATE LOGIN [userName] FROM CERTIFICATE [certificateName]`.</span><span class="sxs-lookup"><span data-stu-id="bb5c2-132">Create a login associated with that certificate using the Transact-SQL `CREATE LOGIN [userName] FROM CERTIFICATE [certificateName]` statement.</span></span>

1. <span data-ttu-id="bb5c2-133">Conceda ao certificado logon as permissões necessárias no nível do servidor.</span><span class="sxs-lookup"><span data-stu-id="bb5c2-133">Grant the certificate login the required server-level permissions.</span></span>

> [!NOTE]
> <span data-ttu-id="bb5c2-134">Um certificado não pode conceder permissões a um usuário que tem permissões revogadas usando a instrução DENY.</span><span class="sxs-lookup"><span data-stu-id="bb5c2-134">A certificate cannot grant permissions to a user that has had permissions revoked using the DENY statement.</span></span> <span data-ttu-id="bb5c2-135">DENY sempre terá precedência sobre GRANT, impedindo que o chamador herde as permissões concedidas ao usuário do certificado.</span><span class="sxs-lookup"><span data-stu-id="bb5c2-135">DENY always takes precedence over GRANT, preventing the caller from inheriting permissions granted to the certificate user.</span></span>
