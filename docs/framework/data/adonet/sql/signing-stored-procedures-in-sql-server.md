# <a name="signing-stored-procedures-in-sql-server"></a><span data-ttu-id="1add5-102">Assinando procedimentos armazenados no SQL Server</span><span class="sxs-lookup"><span data-stu-id="1add5-102">Signing Stored Procedures in SQL Server</span></span>
 <span data-ttu-id="1add5-103">Uma assinatura digital é um resumo dos dados criptografados com a chave privada do assinante.</span><span class="sxs-lookup"><span data-stu-id="1add5-103">A digital signature is a data digest encrypted with the private key of the signer.</span></span> <span data-ttu-id="1add5-104">A chave privada garante que a assinatura digital seja exclusiva de seu portador ou proprietário.</span><span class="sxs-lookup"><span data-stu-id="1add5-104">The private key ensures that the digital signature is unique to its bearer or owner.</span></span> <span data-ttu-id="1add5-105">Você pode assinar assemblies, gatilhos, procedimentos armazenados e funções (exceto funções com valor de tabela do embutidas).</span><span class="sxs-lookup"><span data-stu-id="1add5-105">You can sign stored procedures, functions (except for inline table-valued functions), triggers, and assemblies.</span></span>  
  
 <span data-ttu-id="1add5-106">Você pode assinar um procedimento armazenado com um certificado ou uma chave assimétrica.</span><span class="sxs-lookup"><span data-stu-id="1add5-106">You can sign a stored procedure with a certificate or an asymmetric key.</span></span> <span data-ttu-id="1add5-107">Isso é criado para cenários quando as permissões não podem ser herdadas pelo encadeamento de propriedade ou quando a cadeia de propriedade é quebrada, como SQL dinâmico.</span><span class="sxs-lookup"><span data-stu-id="1add5-107">This is designed for scenarios when permissions cannot be inherited through ownership chaining or when the ownership chain is broken, such as dynamic SQL.</span></span> <span data-ttu-id="1add5-108">Em seguida, você pode criar um usuário mapeado para o certificado, concedendo a permissões de usuário nos objetos de que procedimento armazenado precisa acessar o certificado.</span><span class="sxs-lookup"><span data-stu-id="1add5-108">You can then create a user mapped to the certificate, granting the certificate user permissions on the objects the stored procedure needs to access.</span></span>  

 <span data-ttu-id="1add5-109">Você pode também criar um logon mapeado para o mesmo certificado e, em seguida, conceder quaisquer permissões de nível de servidor necessárias para que o logon ou adicionar o logon a uma ou mais das funções de servidor fixas.</span><span class="sxs-lookup"><span data-stu-id="1add5-109">You can also create a login mapped to the same certificate, and then grant any necessary server-level permissions to that login, or add the login to one or more of the fixed server roles.</span></span> <span data-ttu-id="1add5-110">Isso foi projetado para evitar habilitar o `TRUSTWORTHY` configuração para situações em que as permissões de nível superior são necessárias de banco de dados.</span><span class="sxs-lookup"><span data-stu-id="1add5-110">This is designed to avoid enabling the `TRUSTWORTHY` database setting for scenarios in which higher level permissions are needed.</span></span>

## <a name="creating-certificates"></a><span data-ttu-id="1add5-114">Criando certificados</span><span class="sxs-lookup"><span data-stu-id="1add5-114">Creating Certificates</span></span>  
 <span data-ttu-id="1add5-115">Quando você assinar um procedimento armazenado com um certificado ou chave assimétrica, um resumo de dados consiste em hash criptografado do código de procedimento data-ttu-id="1add5-116">Em tempo de execução, o resumo dos dados é criptografado com a chave pública e comparado com o valor de hash do procedimento armazenado.</span><span class="sxs-lookup"><span data-stu-id="1add5-116">At run time, the data digest is decrypted with the public key and compared with the hash value of the stored procedure.</span></span> <span data-ttu-id="1add5-117">Alterando o executar-como usuário invalida o valor de hash para que a assinatura digital não corresponda.</span><span class="sxs-lookup"><span data-stu-id="1add5-117">Changing the execute-as user invalidates the hash value so that the digital signature no longer matches.</span></span> <span data-ttu-id="1add5-118">Modificar o procedimento armazenado remove a assinatura totalmente, que impede que alguém que procedimento sempre que você alterar o código ou o executar-como usuário.</span><span class="sxs-lookup"><span data-stu-id="1add5-119">In either case, you must re-sign the procedure each time you  <span data-ttu-id="1add5-120">Há duas etapas necessárias envolvidas na assinatura de um módulo:</span><span class="sxs-lookup"><span data-stu-id="1add5-120">There are two required steps involved in signing a module:</span></span>  
  
1. <span data-ttu-id="1add5-121">Criar um certificado usando a instrução Transact-SQL `CREATE CERTIFICATE [certificateName]`.</span><span class="sxs-lookup"><span data-stu-id="1add5-121">Create a certificate using the Transact-SQL `CREATE CERTIFICATE [certificateName]` statement.</span></span> <span data-ttu-id="1add5-122">Essa instrução tem várias opções para definir uma data inicial e final e uma senha.</span><span class="sxs-lookup"><span data-stu-id="1add5-122">This statement has several options for setting a start and end date and a password.</span></span> <span data-ttu-id="1add5-123">A data de expiração padrão é um ano.</span><span class="sxs-lookup"><span data-stu-id="1add5-123">The default expiration date is one year.</span></span>  
  
1. <span data-ttu-id="1add5-124">Assinar o procedimento armazenado com o certificado usando a instrução Transact-SQL `ADD SIGNATURE TO [procedureName] BY CERTIFICATE [certificateName]`.</span><span class="sxs-lookup"><span data-stu-id="1add5-124">Sign the procedure with the certificate using the Transact-SQL `ADD SIGNATURE TO [procedureName] BY CERTIFICATE [certificateName]` statement.</span></span>  

<span data-ttu-id="1add5-125">Depois que o módulo foi assinado, uma ou mais entidades precisa ser criada para manter as permissões adicionais que devem ser associadas ao certificado.</span><span class="sxs-lookup"><span data-stu-id="1add5-125">Once the module has been signed, one or more principals needs to be created in order to hold the additional permissions that should be associated with the certificate.</span></span>  

<span data-ttu-id="1add5-126">Se o módulo precisa de permissões de nível de banco de dados adicionais:</span><span class="sxs-lookup"><span data-stu-id="1add5-126">If the module needs additional database-level permissions:</span></span>  
  
1. <span data-ttu-id="1add5-127">Crie um usuário de banco de dados associado ao certificado usando a instrução Transact-SQL `CREATE USER [userName] FROM CERTIFICATE [certificateName]`.</span><span class="sxs-lookup"><span data-stu-id="1add5-127">Create a database user associated with that certificate using the Transact-SQL `CREATE USER [userName] FROM CERTIFICATE [certificateName]` statement.</span></span> <span data-ttu-id="1add5-128">Esse usuário existe no banco de dados somente e não está associado um logon, a menos que também foi criado um logon do mesmo certificado.</span><span class="sxs-lookup"><span data-stu-id="1add5-128">This user exists in the database only and is not associated with a login unless a login has also been created from that same certificate.</span></span>  
  
1. <span data-ttu-id="1add5-129">Conceda ao usuário de certificado as permissões de nível de banco de dados necessárias.</span><span class="sxs-lookup"><span data-stu-id="1add5-129">Grant the certificate user the required database-level permissions.</span></span>  
  
<span data-ttu-id="1add5-130">Se o módulo precisa de permissões de nível de servidor adicionais:</span><span class="sxs-lookup"><span data-stu-id="1add5-130">If the module needs additional server-level permissions:</span></span>  
  
1. <span data-ttu-id="1add5-131">Copie o certificado para o `master` banco de dados.</span><span class="sxs-lookup"><span data-stu-id="1add5-131">Copy the certificate to the `master` database.</span></span>  
 
1. <span data-ttu-id="1add5-132">Crie um logon associado ao certificado usando o Transact-SQL `CREATE LOGIN [userName] FROM CERTIFICATE [certificateName]` instrução.</span><span class="sxs-lookup"><span data-stu-id="1add5-132">Create a login associated with that certificate using the Transact-SQL `CREATE LOGIN [userName] FROM CERTIFICATE [certificateName]` statement.</span></span>  
  
1. <span data-ttu-id="1add5-133">Concede ao logon de certificado as permissões de nível de servidor necessárias.</span><span class="sxs-lookup"><span data-stu-id="1add5-133">Grant the certificate login the required server-level permissions.</span></span>  
  
> [!NOTE]  
>  <span data-ttu-id="1add5-134">Um certificado não pode conceder permissões a um usuário que tem permissões revogadas usando a instrução DENY.</span><span class="sxs-lookup"><span data-stu-id="1add5-134">A certificate cannot grant permissions to a user that has had permissions revoked using the DENY statement.</span></span> <span data-ttu-id="1add5-135">DENY sempre terá precedência sobre GRANT, impedindo que o chamador herde as permissões concedidas ao usuário do certificado.</span><span class="sxs-lookup"><span data-stu-id="1add5-135">DENY always takes precedence over GRANT, preventing the caller from inheriting permissions granted to the certificate user.</span></span>
