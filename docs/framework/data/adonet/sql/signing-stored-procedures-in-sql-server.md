# <a name="signing-stored-procedures-in-sql-server"></a><span data-ttu-id="3030e-102">Assinando procedimentos armazenados no SQL Server</span><span class="sxs-lookup"><span data-stu-id="3030e-102">Signing Stored Procedures in SQL Server</span></span>
<span data-ttu-id="3030e-103">Você pode assinar um procedimento armazenado com um certificado ou uma chave assimétrica.</span><span class="sxs-lookup"><span data-stu-id="3030e-103">You can sign a stored procedure with a certificate or an asymmetric key.</span></span> <span data-ttu-id="3030e-104">Isso é criado para cenários quando as permissões não podem ser herdadas pelo encadeamento de propriedade ou quando a cadeia de propriedade é quebrada, como SQL dinâmico.</span><span class="sxs-lookup"><span data-stu-id="3030e-104">This is designed for scenarios when permissions cannot be inherited through ownership chaining or when the ownership chain is broken, such as dynamic SQL.</span></span> <span data-ttu-id="3030e-105">Você cria um usuário mapeado para o certificado, concedendo permissões de usuário do certificado nos objetos que o procedimento armazenado precisa acessar.</span><span class="sxs-lookup"><span data-stu-id="3030e-105">You then create a user mapped to the certificate, granting the certificate user permissions on the objects the stored procedure needs to access.</span></span>  
  
 <span data-ttu-id="3030e-106">Quando o procedimento armazenado é executado, o SQL Server combina as permissões do usuário do certificado com as do chamador.</span><span class="sxs-lookup"><span data-stu-id="3030e-106">When the stored procedure is executed, SQL Server combines the permissions of the certificate user with those of the caller.</span></span> <span data-ttu-id="3030e-107">Ao contrário da cláusula EXECUTE AS, ele não altera o contexto de execução do procedimento.</span><span class="sxs-lookup"><span data-stu-id="3030e-107">Unlike the EXECUTE AS clause, it does not change the execution context of the procedure.</span></span> <span data-ttu-id="3030e-108">As funções internas que retornam o logon e os nomes de usuário retornam o nome do chamador, não o nome do usuário do certificado.</span><span class="sxs-lookup"><span data-stu-id="3030e-108">Built-in functions that return login and user names return the name of the caller, not the certificate user name.</span></span>  
  
 <span data-ttu-id="3030e-109">Uma assinatura digital é um resumo dos dados criptografados com a chave privada do assinante.</span><span class="sxs-lookup"><span data-stu-id="3030e-109">A digital signature is a data digest encrypted with the private key of the signer.</span></span> <span data-ttu-id="3030e-110">A chave privada garante que a assinatura digital seja exclusiva de seu portador ou proprietário.</span><span class="sxs-lookup"><span data-stu-id="3030e-110">The private key ensures that the digital signature is unique to its bearer or owner.</span></span> <span data-ttu-id="3030e-111">Você pode assinar procedimentos armazenados, funções ou gatilhos.</span><span class="sxs-lookup"><span data-stu-id="3030e-111">You can sign stored procedures, functions, or triggers.</span></span>

## <a name="creating-certificates"></a><span data-ttu-id="3030e-113">Criando certificados</span><span class="sxs-lookup"><span data-stu-id="3030e-113">Creating Certificates</span></span>  
 <span data-ttu-id="3030e-114">Quando você assina um procedimento armazenado com um certificado, um resumo dos dados que consiste em hash criptografado do código do procedimento armazenado é criado usando a chave privada.</span><span class="sxs-lookup"><span data-stu-id="3030e-114">When you sign a stored procedure with a certificate, a data digest consisting of the encrypted hash of the stored procedure code is created using the private key.</span></span> <span data-ttu-id="3030e-115">Em tempo de execução, o resumo dos dados é criptografado com a chave pública e comparado com o valor de hash do procedimento armazenado.</span><span class="sxs-lookup"><span data-stu-id="3030e-115">At run time, the data digest is decrypted with the public key and compared with the hash value of the stored procedure.</span></span> <span data-ttu-id="3030e-116">Alterar o procedimento armazenado invalida o valor de hash de modo que a assinatura digital não corresponda.</span><span class="sxs-lookup"><span data-stu-id="3030e-116">Modifying the stored procedure invalidates the hash value so that the digital signature no longer matches.</span></span> <span data-ttu-id="3030e-117">Isso impedirá que alguém que não tenha acesso à chave privada altere o código do procedimento armazenado.</span><span class="sxs-lookup"><span data-stu-id="3030e-117">This prevents someone who does not have access to the private key from changing the stored procedure code.</span></span> <span data-ttu-id="3030e-118">Portanto, você deve reassinar o procedimento sempre que modificá-lo.</span><span class="sxs-lookup"><span data-stu-id="3030e-118">Therefore you must re-sign the procedure each time you modify it.</span></span>

3.  <span data-ttu-id="3030e-125">Conceder ao usuário do certificado as permissões necessárias nos objetos de banco de dados.</span><span class="sxs-lookup"><span data-stu-id="3030e-125">Grant the certificate user the required permissions on the database objects.</span></span>
