 <span data-ttu-id="facce-106">Quando você fizer essa chamada, <xref:System.Data.Linq.DataContext> tenta converter suas alterações em comandos SQL equivalentes.</span><span class="sxs-lookup"><span data-stu-id="facce-106">When you make this call, the <xref:System.Data.Linq.DataContext> tries to translate your changes into equivalent SQL commands.</span></span> <span data-ttu-id="facce-107">Você pode usar sua própria lógica personalizada para substituir essas ações, mas a ordem de envio é orquestrada por um serviço do <xref:System.Data.Linq.DataContext> conhecido como o *alterar processador*.</span><span class="sxs-lookup"><span data-stu-id="facce-107">You can use your own custom logic to override these actions, but the order of submission is orchestrated by a service of the <xref:System.Data.Linq.DataContext> known as the *change processor*.</span></span> <span data-ttu-id="facce-108">A sequência de eventos é a seguinte:</span><span class="sxs-lookup"><span data-stu-id="facce-108">The sequence of events is as follows:</span></span>  
  
1.  <span data-ttu-id="facce-109">Quando você chama <xref:System.Data.Linq.DataContext.SubmitChanges%2A>, [!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)] examina o conjunto de objetos conhecidos para determinar se as novas instâncias eles foram anexadas.</span><span class="sxs-lookup"><span data-stu-id="facce-109">When you call <xref:System.Data.Linq.DataContext.SubmitChanges%2A>, [!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)] examines the set of known objects to determine whether new instances have been attached to them.</span></span> <span data-ttu-id="facce-110">Se eles tiverem, essas novas instâncias são adicionadas ao conjunto de objetos rastreadas.</span><span class="sxs-lookup"><span data-stu-id="facce-110">If they have, these new instances are added to the set of tracked objects.</span></span>

3.  <span data-ttu-id="facce-113">Imediatamente antes todas as alterações reais são passadas, [!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)] inicia uma transação para encapsular a série de comandos individuais.</span><span class="sxs-lookup"><span data-stu-id="facce-113">Immediately before any actual changes are transmitted, [!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)] starts a transaction to encapsulate the series of individual commands.</span></span>  
  
4.  <span data-ttu-id="facce-114">As alterações para os objetos são traduzidas um por um para comandos SQL e enviadas ao servidor.</span><span class="sxs-lookup"><span data-stu-id="facce-114">The changes to the objects are translated one by one to SQL commands and sent to the server.</span></span>  
  
 <span data-ttu-id="facce-115">Neste ponto, todos os erros detectados por base de dados fazem com que o processo de envio parar, e uma exceção é gerada.</span><span class="sxs-lookup"><span data-stu-id="facce-115">At this point, any errors detected by the database cause the submission process to stop, and an exception is raised.</span></span> <span data-ttu-id="facce-116">Todas as alterações a base de dados são revertidas como se nenhuma envio ocorreu nunca.</span><span class="sxs-lookup"><span data-stu-id="facce-116">All changes to the database are rolled back as if no submissions ever occurred.</span></span> <span data-ttu-id="facce-117"><xref:System.Data.Linq.DataContext> ainda tem uma gravação completa de todas as alterações.</span><span class="sxs-lookup"><span data-stu-id="facce-117">The <xref:System.Data.Linq.DataContext> still has a full recording of all changes.</span></span> <span data-ttu-id="facce-118">Portanto você pode tentar corrigir o problema e chamar novamente <xref:System.Data.Linq.DataContext.SubmitChanges%2A> , como no exemplo de código que segue.</span><span class="sxs-lookup"><span data-stu-id="facce-118">You can therefore try to correct the problem and call <xref:System.Data.Linq.DataContext.SubmitChanges%2A> again, as in the code example that follows.</span></span>
