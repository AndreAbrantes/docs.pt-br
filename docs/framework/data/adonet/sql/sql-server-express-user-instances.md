 <span data-ttu-id="10784-166">Por padrão, todos os membros do grupo Builtin\Users do Windows recebem permissões para conectar-se na instância local, assim como permissões de leitura e execução nos binários do SQL Server.</span><span class="sxs-lookup"><span data-stu-id="10784-166">By default all members of the Windows Builtin\Users group are granted permissions to connect on the local instance as well as read and execute permissions on the SQL Server binaries.</span></span> <span data-ttu-id="10784-167">Depois que as credenciais do usuário chamador que hospeda a instância de usuário tiverem sido verificadas, o usuário se tornará o `sysadmin` nessa instância.</span><span class="sxs-lookup"><span data-stu-id="10784-167">Once the credentials of the calling user hosting the user instance have been verified, that user becomes the `sysadmin` on that instance.</span></span> <span data-ttu-id="10784-168">Somente a memória compartilhada está habilitada para instâncias de usuário, o que significa que são possíveis apenas operações no computador local.</span><span class="sxs-lookup"><span data-stu-id="10784-168">Only shared memory is enabled for user instances, which means that only operations on the local machine are possible.</span></span>  
  
 <span data-ttu-id="10784-169">Os usuários devem receber permissões de leitura e gravação nos arquivos .mdf e .ldf especificados na cadeia de conexão.</span><span class="sxs-lookup"><span data-stu-id="10784-169">Users must be granted both read and write permissions on the .mdf and .ldf files specified in the connection string.</span></span>  
  
> [!NOTE]

O banco de dados não será aberto se for acoplado com o arquivo de log errado.</span><span class="sxs-lookup"><span data-stu-id="10784-172">The database file contains information about the exact version of the log file, and the database will not open if it is coupled with the wrong log file.</span></span>  
  
 <span data-ttu-id="10784-173">Para evitar dados corrompidos, um banco de dados na instância de usuário é aberto com acesso exclusivo.</span><span class="sxs-lookup"><span data-stu-id="10784-173">To avoid data corruption, a database in the user instance is opened with exclusive access.</span></span> <span data-ttu-id="10784-174">Se duas instâncias de usuário diferentes compartilharem o mesmo banco de dados no mesmo computador, o usuário na primeira instância deverá fechar o banco de dados antes que ele possa ser aberto em uma segunda instância.</span><span class="sxs-lookup"><span data-stu-id="10784-174">If two different user instances share the same database on the same computer, the user on the first instance must close the database before it can be opened in a second instance.</span></span>

 <span data-ttu-id="10784-176">As instâncias de usuário fornecem aos desenvolvedores de aplicativos de banco de dados um armazenamento de dados do SQL Server que não depende de os desenvolvedores sem precisar de intervenção de um administrador do sistema para conceder permissões.</span><span class="sxs-lookup"><span data-stu-id="10784-177">User instances are based on the Access/Jet model,
