 <span data-ttu-id="8cec4-107">A sequência de eventos simplificada a seguir ocorre para um objeto <xref:System.Data.SqlClient.SqlConnection> direcionado a um banco de dados configurado para espelhamento:</span><span class="sxs-lookup"><span data-stu-id="8cec4-107">The following simplified sequence of events occurs for a <xref:System.Data.SqlClient.SqlConnection> object that targets a database configured for mirroring:</span></span>

## <a name="specifying-the-failover-partner-in-the-connection-string"></a><span data-ttu-id="8cec4-113">Especificando o parceiro de failover na cadeia de conexão</span><span class="sxs-lookup"><span data-stu-id="8cec4-113">Specifying the Failover Partner in the Connection String</span></span>  

 <span data-ttu-id="8cec4-114">Se você fornecer o nome de um servidor de parceiro de failover na cadeia de conexão, o cliente tentará uma conexão de forma transparente com o parceiro de failover se o banco de dados principal estiver indisponível quando o aplicativo cliente se conectar pela primeira vez.</span><span class="sxs-lookup"><span data-stu-id="8cec4-114">If you supply the name of a failover partner server in the connection string, the client will transparently attempt a connection with the failover partner if the principal database is unavailable when the client application first connects.</span></span>

  
 <span data-ttu-id="8cec4-115">Se você omitir o nome do servidor do parceiro de failover e o banco de dados principal estiver indisponível quando o aplicativo cliente se conectar pela primeira vez, uma <xref:System.Data.SqlClient.SqlException> será gerada.</span><span class="sxs-lookup"><span data-stu-id="8cec4-115">If you omit the name of the failover partner server and the principal database is unavailable when the client application first connects then a <xref:System.Data.SqlClient.SqlException> is raised.</span></span>  
  
 <span data-ttu-id="8cec4-116">Quando uma <xref:System.Data.SqlClient.SqlConnection> for aberta com êxito, o nome do parceiro de failover será retornado ao servidor e substituirá qualquer valor fornecido na cadeia de conexão.</span><span class="sxs-lookup"><span data-stu-id="8cec4-116">When a <xref:System.Data.SqlClient.SqlConnection> is successfully opened, the failover partner name is returned by the server and supersedes any values supplied in the connection string.</span></span>  
  
> [!NOTE]
> <span data-ttu-id="8cec4-117">Especifique, de forma explícita, o nome do banco de dados ou do catálogo inicial na cadeia de conexão para cenários de espelhamento de banco de dados.</span><span class="sxs-lookup"><span data-stu-id="8cec4-117">You must explicitly specify the initial catalog or database name in the connection string for database mirroring scenarios.</span></span> <span data-ttu-id="8cec4-118">Se o cliente receber informações de failover em uma conexão que não tem um banco de dados ou um catálogo inicial explicitamente especificado, as informações de failover não serão armazenadas em cache e o aplicativo não tentará fazer o failover se o servidor principal falhar.</span><span class="sxs-lookup"><span data-stu-id="8cec4-118">If the client receives failover information on a connection that doesn't have an explicitly specified initial catalog or database, the failover information is not cached and the application does not attempt to fail over if the principal server fails.</span></span> <span data-ttu-id="8cec4-119">Se uma cadeia de conexão tiver um valor para o parceiro de failover, mas nenhum valor para o banco de dados ou o catálogo inicial, uma `InvalidArgumentException` será gerada.</span><span class="sxs-lookup"><span data-stu-id="8cec4-119">If a connection string has a value for the failover partner, but no value for the initial catalog or database, an `InvalidArgumentException` is raised.</span></span>
