# <a name="customizing-permissions-with-impersonation-in-sql-server"></a><span data-ttu-id="6baca-102">Personalizando permissões com representação no SQL Server</span><span class="sxs-lookup"><span data-stu-id="6baca-102">Customizing Permissions with Impersonation in SQL Server</span></span>
<span data-ttu-id="6baca-103">Muitos aplicativos usam procedimentos armazenados para acessar os dados, dependendo do encadeamento de propriedade para restringir o acesso a tabelas base.</span><span class="sxs-lookup"><span data-stu-id="6baca-103">Many applications use stored procedures to access data, relying on ownership chaining to restrict access to base tables.</span></span> <span data-ttu-id="6baca-104">Você pode conceder permissões EXECUTE em procedimentos armazenados, revogando ou negando permissões nas tabelas base.</span><span class="sxs-lookup"><span data-stu-id="6baca-104">You can grant EXECUTE permissions on stored procedures, revoking or denying permissions on the base tables.</span></span> <span data-ttu-id="6baca-105">O SQL Server não verifica as permissões do chamador se o procedimento armazenado e as tabelas têm o mesmo proprietário.</span><span class="sxs-lookup"><span data-stu-id="6baca-105">SQL Server does not check the permissions of the caller if the stored procedure and tables have the same owner.</span></span> <span data-ttu-id="6baca-106">No entanto, o encadeamento de propriedades não funcionará se os objetos tiverem proprietários diferentes ou no caso de SQL dinâmico.</span><span class="sxs-lookup"><span data-stu-id="6baca-106">However, ownership chaining doesn't work if objects have different owners or in the case of dynamic SQL.</span></span>  
  
 <span data-ttu-id="6baca-107">Você pode usar a cláusula EXECUTE AS em um procedimento armazenado quando o chamador não tiver permissões nos objetos de banco de dados referenciados.</span><span class="sxs-lookup"><span data-stu-id="6baca-107">You can use the EXECUTE AS clause in a stored procedure when the caller doesn't have permissions on the referenced database objects.</span></span> <span data-ttu-id="6baca-108">O efeito da cláusula EXECUTE AS é que o contexto de execução é alternado para o usuário do proxy.</span><span class="sxs-lookup"><span data-stu-id="6baca-108">The effect of the EXECUTE AS clause is that the execution context is switched to the proxy user.</span></span> <span data-ttu-id="6baca-109">Qualquer código, assim como as chamadas para os procedimentos armazenados ou gatilhos aninhados, é executado sob o contexto de segurança do usuário do proxy.</span><span class="sxs-lookup"><span data-stu-id="6baca-109">All code, as well as any calls to nested stored procedures or triggers, executes under the security context of the proxy user.</span></span> <span data-ttu-id="6baca-110">O contexto de execução é revertido para o chamador original somente após a execução do procedimento ou quando uma instrução REVERT for emitida.</span><span class="sxs-lookup"><span data-stu-id="6baca-110">Execution context is reverted to the original caller only after execution of the procedure or when a REVERT statement is issued.</span></span>  
  
## <a name="context-switching-with-the-execute-as-statement"></a><span data-ttu-id="6baca-111">Alternância de contexto com a instrução EXECUTE AS</span><span class="sxs-lookup"><span data-stu-id="6baca-111">Context Switching with the EXECUTE AS Statement</span></span>  
 <span data-ttu-id="6baca-112">A instrução EXECUTE AS do Transact-SQL permite a troca do contexto de execução de uma instrução representando outro logon ou usuário do banco de dados.</span><span class="sxs-lookup"><span data-stu-id="6baca-112">The Transact-SQL EXECUTE AS statement allows you to switch the execution context of a statement by impersonating another login or database user.</span></span> <span data-ttu-id="6baca-113">Essa é uma técnica útil para testar consultas e procedimentos como outro usuário.</span><span class="sxs-lookup"><span data-stu-id="6baca-113">This is a useful technique for testing queries and procedures as another user.</span></span>  
  
```  
EXECUTE AS LOGIN = 'loginName';  
EXECUTE AS USER = 'userName';  
```  
  
 <span data-ttu-id="6baca-114">Você deve ter permissões IMPERSONATE no logon ou usuário que você está representando.</span><span class="sxs-lookup"><span data-stu-id="6baca-114">You must have IMPERSONATE permissions on the login or user you are impersonating.</span></span> <span data-ttu-id="6baca-115">Essa permissão é implícita para `sysadmin` para todos os bancos de dados e os membros de função `db_owner` nos bancos de dados que eles possuem.</span><span class="sxs-lookup"><span data-stu-id="6baca-115">This permission is implied for `sysadmin` for all databases, and `db_owner` role members in databases that they own.</span></span>  
  
## <a name="granting-permissions-with-the-execute-as-clause"></a><span data-ttu-id="6baca-116">Concedendo permissões com a cláusula EXECUTE AS</span><span class="sxs-lookup"><span data-stu-id="6baca-116">Granting Permissions with the EXECUTE AS Clause</span></span>  
 <span data-ttu-id="6baca-117">Você pode usar a cláusula EXECUTE AS no cabeçalho da definição de um procedimento armazenado, gatilho, ou uma função definida pelo usuário (com exceção das funções com valor de tabela embutida).</span><span class="sxs-lookup"><span data-stu-id="6baca-117">You can use the EXECUTE AS clause in the definition header of a stored procedure, trigger, or user-defined function (except for inline table-valued functions).</span></span> <span data-ttu-id="6baca-118">Isso faz o procedimento ser executado no contexto do nome de usuário ou palavra-chave especificada na cláusula EXECUTE AS.</span><span class="sxs-lookup"><span data-stu-id="6baca-118">This causes the procedure to execute in the context of the user name or keyword specified in the EXECUTE AS clause.</span></span> <span data-ttu-id="6baca-119">Você pode criar um usuário de proxy no banco de dados que não seja mapeado para um logon, permitindo apenas as permissões necessárias sobre os objetos acessados pelo procedimento.</span><span class="sxs-lookup"><span data-stu-id="6baca-119">You can create a proxy user in the database that is not mapped to a login, granting it only the necessary permissions on the objects accessed by the procedure.</span></span> <span data-ttu-id="6baca-120">Apenas o usuário do proxy especificado na cláusula EXECUTE AS deve ter permissões em todos os objetos acessados pelo módulo.</span><span class="sxs-lookup"><span data-stu-id="6baca-120">Only the proxy user specified in the EXECUTE AS clause must have permissions on all objects accessed by the module.</span></span>  
  
> [!NOTE]
>  <span data-ttu-id="6baca-121">Algumas ações, como TRUNCATE TABLE, não têm permissões concessíveis.</span><span class="sxs-lookup"><span data-stu-id="6baca-121">Some actions, such as TRUNCATE TABLE, do not have grantable permissions.</span></span> <span data-ttu-id="6baca-122">Incorporando a instrução dentro de um procedimento e especificando um usuário de proxy que tenha permissões ALTER TABLE, você pode estender as permissões para truncar a tabela para chamadores que têm somente as permissões EXECUTE no procedimento.</span><span class="sxs-lookup"><span data-stu-id="6baca-122">By incorporating the statement within a procedure and specifying a proxy user who has ALTER TABLE permissions, you can extend the permissions to truncate the table to callers who have only EXECUTE permissions on the procedure.</span></span>  
  
 <span data-ttu-id="6baca-123">O contexto especificado na cláusula EXECUTE AS é válido para a duração do procedimento, inclusive procedimentos aninhados e gatilhos.</span><span class="sxs-lookup"><span data-stu-id="6baca-123">The context specified in the EXECUTE AS clause is valid for the duration of the procedure, including nested procedures and triggers.</span></span> <span data-ttu-id="6baca-124">O contexto será revertido para o chamador quando a execução estiver concluída ou a instrução REVERT for emitida.</span><span class="sxs-lookup"><span data-stu-id="6baca-124">Context reverts to the caller when execution is complete or the REVERT statement is issued.</span></span>  
  
 <span data-ttu-id="6baca-125">Há três etapas envolvidas no uso da cláusula EXECUTE AS em um procedimento.</span><span class="sxs-lookup"><span data-stu-id="6baca-125">There are three steps involved in using the EXECUTE AS clause in a procedure.</span></span>  
  
1.  <span data-ttu-id="6baca-126">Crie um usuário de proxy no banco de dados que não seja mapeado para um logon.</span><span class="sxs-lookup"><span data-stu-id="6baca-126">Create a proxy user in the database that is not mapped to a login.</span></span> <span data-ttu-id="6baca-127">Isso não é obrigatório, mas ajuda a gerenciar permissões.</span><span class="sxs-lookup"><span data-stu-id="6baca-127">This is not required, but it helps when managing permissions.</span></span>  
  
```  
CREATE USER proxyUser WITHOUT LOGIN  
```  
  
1.  <span data-ttu-id="6baca-128">Conceda ao usuário de proxy as permissões necessárias.</span><span class="sxs-lookup"><span data-stu-id="6baca-128">Grant the proxy user the necessary permissions.</span></span>  
  
2.  <span data-ttu-id="6baca-129">Adicione a cláusula EXECUTE AS no procedimento armazenado ou função definida pelo usuário.</span><span class="sxs-lookup"><span data-stu-id="6baca-129">Add the EXECUTE AS clause to the stored procedure or user-defined function.</span></span>  
  
```

```  
  
> [!NOTE]
>  <span data-ttu-id="6baca-130">Os aplicativos que exigem a auditoria podem ser interrompidos porque o contexto de segurança original do chamador não é retido.</span><span class="sxs-lookup"><span data-stu-id="6baca-130">Applications that require auditing can break because the original security context of the caller is not retained.</span></span> <span data-ttu-id="6baca-131">As funções internas que retornam a identidade do usuário atual, como SESSION_USER, USER ou USER_NAME retornarão o usuário associado com a cláusula EXECUTE AS, não o chamador original.</span><span class="sxs-lookup"><span data-stu-id="6baca-131">Built-in functions that return the identity of the current user, such as SESSION_USER, USER, or USER_NAME, return the user associated with the EXECUTE AS clause, not the original caller.</span></span>
