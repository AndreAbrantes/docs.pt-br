## <a name="use-windows-authentication"></a><span data-ttu-id="bdbdb-108">Usar Autenticação do Windows</span><span class="sxs-lookup"><span data-stu-id="bdbdb-108">Use Windows Authentication</span></span>  
 <span data-ttu-id="bdbdb-109">Para ajudar a limitar o acesso a sua fonte de dados, você deverá proteger as informações de conexão como a identificação do usuário, senha e nome da fonte de dados.</span><span class="sxs-lookup"><span data-stu-id="bdbdb-109">To help limit access to your data source, you must secure connection information such as user ID, password, and data source name.</span></span> <span data-ttu-id="bdbdb-110">Para evitar expor as informações do usuário, recomendamos o uso da autenticação do Windows (às vezes referida como *segurança integrada)* sempre que possível.</span><span class="sxs-lookup"><span data-stu-id="bdbdb-110">In order to avoid exposing user information, we recommend using Windows authentication (sometimes referred to as *integrated security*) wherever possible.</span></span> <span data-ttu-id="bdbdb-111">A autenticação do Windows é especificada em uma cadeia de conexão usando as palavras-chave `Integrated Security` ou `Trusted_Connection`, eliminando a necessidade de usar uma identificação de usuário e senha.</span><span class="sxs-lookup"><span data-stu-id="bdbdb-111">Windows authentication is specified in a connection string by using the `Integrated Security` or `Trusted_Connection` keywords, eliminating the need to use a user ID and password.</span></span> <span data-ttu-id="bdbdb-112">Ao usar a autenticação do Windows, os usuários são autenticados pelo Windows e o acesso ao servidor e os recursos de banco de dados são determinados concedendo permissões a usuários e grupos do Windows.</span><span class="sxs-lookup"><span data-stu-id="bdbdb-112">When using Windows authentication, users are authenticated by Windows, and access to server and database resources is determined by granting permissions to Windows users and groups.</span></span>  
  
 <span data-ttu-id="bdbdb-113">Para situações onde não é possível usar a autenticação do Windows, você deverá ter cuidado adicional porque as credenciais do usuário são expostas na cadeia de conexão.</span><span class="sxs-lookup"><span data-stu-id="bdbdb-113">For situations where it is not possible to use Windows authentication, you must use extra care because user credentials are exposed in the connection string.</span></span> <span data-ttu-id="bdbdb-114">Em um aplicativo do ASP.NET, você pode configurar uma conta do Windows como uma identidade fixa que é usada para se conectar a bancos de dados e outros recursos de rede.</span><span class="sxs-lookup"><span data-stu-id="bdbdb-114">In an ASP.NET application, you can configure a Windows account as a fixed identity that is used to connect to databases and other network resources.</span></span> <span data-ttu-id="bdbdb-115">Você habilita a personificação no elemento de identidade no arquivo **Web.config** e especifica um nome de usuário e senha.</span><span class="sxs-lookup"><span data-stu-id="bdbdb-115">You enable impersonation in the identity element in the **web.config** file and specify a user name and password.</span></span>  
  
```xml

        userName="MyDomain\UserAccount"
        password="*****" />  
```  
  
 <span data-ttu-id="bdbdb-116">A conta de identidade fixa deve ser uma conta de privilégios baixos que tem concedidas somente as permissões necessárias no banco de dados.</span><span class="sxs-lookup"><span data-stu-id="bdbdb-116">The fixed identity account should be a low-privilege account that has been granted only necessary permissions in the database.</span></span> <span data-ttu-id="bdbdb-117">Além disso, você deve criptografar o arquivo de configuração de modo que o nome de usuário e a senha não sejam expostos em texto não criptografado.</span><span class="sxs-lookup"><span data-stu-id="bdbdb-117">In addition, you should encrypt the configuration file so that the user name and password are not exposed in clear text.</span></span>

## <a name="use-persist-security-infofalse"></a><span data-ttu-id="bdbdb-127">Use segurança de persistência Info=False</span><span class="sxs-lookup"><span data-stu-id="bdbdb-127">Use Persist Security Info=False</span></span>
