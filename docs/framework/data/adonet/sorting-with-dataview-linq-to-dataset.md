 <span data-ttu-id="0ff1f-109">Um objeto <xref:System.Data.DataView> pode ser criado a partir de uma consulta [!INCLUDE[linq_dataset](../../../../includes/linq-dataset-md.md)].</span><span class="sxs-lookup"><span data-stu-id="0ff1f-109">A <xref:System.Data.DataView> object can be created from a [!INCLUDE[linq_dataset](../../../../includes/linq-dataset-md.md)] query.</span></span> <span data-ttu-id="0ff1f-110">Se essa consulta contiver uma <xref:System.Linq.Enumerable.OrderBy%2A>, <xref:System.Linq.Enumerable.OrderByDescending%2A>, <xref:System.Linq.Enumerable.ThenBy%2A>, ou <xref:System.Linq.Enumerable.ThenByDescending%2A> cláusula as expressões nessas cláusulas são usadas como base para classificar os dados a <xref:System.Data.DataView>.</span><span class="sxs-lookup"><span data-stu-id="0ff1f-110">If that query contains an <xref:System.Linq.Enumerable.OrderBy%2A>, <xref:System.Linq.Enumerable.OrderByDescending%2A>, <xref:System.Linq.Enumerable.ThenBy%2A>, or <xref:System.Linq.Enumerable.ThenByDescending%2A> clause the expressions in these clauses are used as the basis for sorting the data in the <xref:System.Data.DataView>.</span></span> <span data-ttu-id="0ff1f-111">Por exemplo, se a consulta contiver a `Order By…`e `Then By…` cláusulas, resultante <xref:System.Data.DataView> deverá solicitar os dados por ambas as colunas especificadas.</span><span class="sxs-lookup"><span data-stu-id="0ff1f-111">For example, if the query contains the `Order By…`and `Then By…` clauses, the resulting <xref:System.Data.DataView> would order the data by both columns specified.</span></span>  
  
 <span data-ttu-id="0ff1f-112">A classificação baseada em expressão oferece uma classificação mais avançada e complexa do que a classificação mais simples baseada em cadeia de caracteres.</span><span class="sxs-lookup"><span data-stu-id="0ff1f-112">Expression-based sorting offers more powerful and complex sorting than the simpler string-based sorting.</span></span> <span data-ttu-id="0ff1f-113">Observe que a classificação baseada em cadeia de caracteres e a classificação baseada em expressão são mutuamente excludentes.</span><span class="sxs-lookup"><span data-stu-id="0ff1f-113">Note that string-based and expression-based sorting are mutually exclusive.</span></span> <span data-ttu-id="0ff1f-114">Se a <xref:System.Data.DataView.Sort%2A> baseada em cadeia de caracteres for definida após <xref:System.Data.DataView> ser criado a partir de uma consulta, o filtro baseado em expressão inferido da consulta será limpo e não poderá ser redefinido.</span><span class="sxs-lookup"><span data-stu-id="0ff1f-114">If the string-based <xref:System.Data.DataView.Sort%2A> is set after a <xref:System.Data.DataView> is created from a query, the expression-based filter inferred from the query is cleared and cannot be reset.</span></span>  
  
 <span data-ttu-id="0ff1f-115">O índice de um <xref:System.Data.DataView> é compilado quando <xref:System.Data.DataView> é criado e quando qualquer informação de classificação ou de filtragem é modificada.</span><span class="sxs-lookup"><span data-stu-id="0ff1f-115">The index for a <xref:System.Data.DataView> is built both when the <xref:System.Data.DataView> is created and when any of the sorting or filtering information is modified.</span></span> <span data-ttu-id="0ff1f-116">Você obtém melhor desempenho fornecendo critérios de classificação na consulta [!INCLUDE[linq_dataset](../../../../includes/linq-dataset-md.md)] da qual <xref:System.Data.DataView> é criado e não modificando informações de classificação posteriormente.</span><span class="sxs-lookup"><span data-stu-id="0ff1f-116">You get the best performance by supplying sorting criteria in the [!INCLUDE[linq_dataset](../../../../includes/linq-dataset-md.md)] query that the <xref:System.Data.DataView> is created from and not modifying the sorting information, later.</span></span> <span data-ttu-id="0ff1f-117">Para obter mais informações, consulte [DataView desempenho](../../../../docs/framework/data/adonet/dataview-performance.md).</span><span class="sxs-lookup"><span data-stu-id="0ff1f-117">For more information, see [DataView Performance](../../../../docs/framework/data/adonet/dataview-performance.md).</span></span>  
  
> [!NOTE]
>  <span data-ttu-id="0ff1f-118">Na maioria dos casos, as expressões usadas para classificação não devem ter efeitos colaterais e devem ser determinísticas.</span><span class="sxs-lookup"><span data-stu-id="0ff1f-118">In most cases, the expressions used for sorting should not have side effects and must be deterministic.</span></span> <span data-ttu-id="0ff1f-119">Além disso, as expressões não devem conter lógica que dependa de um número definido de execuções, pois as operações de classificação podem ser executadas qualquer número de vezes.</span><span class="sxs-lookup"><span data-stu-id="0ff1f-119">Also, the expressions should not contain any logic that depends on a set number of executions, because the sorting operations might be executed any number of times.</span></span>
