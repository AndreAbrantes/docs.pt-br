<span data-ttu-id="2b51c-103">O ADO.NET fornece acesso consistente a fontes de dados como o SQL Server e o XML, e a fontes de dados expostas através do OLE DB e do ODBC.</span><span class="sxs-lookup"><span data-stu-id="2b51c-103">ADO.NET provides consistent access to data sources such as SQL Server and XML, and to data sources exposed through OLE DB and ODBC.</span></span> <span data-ttu-id="2b51c-104">Os aplicativos do consumidor de compartilhamento de dados podem usar o ADO.NET para se conectar a essas fontes de dados, e para recuperar, manipular e atualizar os dados nelas contidos.</span><span class="sxs-lookup"><span data-stu-id="2b51c-104">Data-sharing consumer applications can use ADO.NET to connect to these data sources and retrieve, handle, and update the data that they contain.</span></span>

 <span data-ttu-id="2b51c-111">O ADO.NET fornece a funcionalidade para os desenvolvedores que gravam um código gerenciado semelhante à funcionalidade fornecida aos desenvolvedores COM nativos pelos objetos ActiveX Data Objects (ADO).</span><span class="sxs-lookup"><span data-stu-id="2b51c-111">ADO.NET provides functionality to developers who write managed code similar to the functionality provided to native component object model (COM) developers by ActiveX Data Objects (ADO).</span></span> <span data-ttu-id="2b51c-112">É recomendável que você use o ADO.NET, e não o ADO, para acessar dados nos aplicativos .NET.</span><span class="sxs-lookup"><span data-stu-id="2b51c-112">We recommend that you use ADO.NET, not ADO, for accessing data in your .NET applications.</span></span>

 <span data-ttu-id="2b51c-115">**Declaração de privacidade**: Os assemblies de DLL, privacidade&lt;1, OracleClient, dll, System, SqlServerCe. dll e DataSetExtensions. dll não fazem distinção entre um usuário dados particulares e não privados.</span><span class="sxs-lookup"><span data-stu-id="2b51c-115">**Privacy Statement**: The System.Data.dll, System.Data.Design.dll, System.Data.OracleClient.dll, System.Data.SqlXml.dll, System.Data.Linq.dll, System.Data.SqlServerCe.dll, and System.Data.DataSetExtensions.dll assemblies do not distinguish between a user's private data and non-private data.</span></span>  <span data-ttu-id="2b51c-116">Esses assemblies não coletam, não armazenam nem transmitem dados privados de nenhum usuário.</span><span class="sxs-lookup"><span data-stu-id="2b51c-116">These assemblies do not collect, store, or transport any user's private data.</span></span> <span data-ttu-id="2b51c-117">No entanto, os aplicativos de terceiros podem coletar, armazenar ou transmitir dados privados de um usuário usando esses assemblies.</span><span class="sxs-lookup"><span data-stu-id="2b51c-117">However, third-party applications might collect, store, or transport a user's private data using these assemblies.</span></span>
