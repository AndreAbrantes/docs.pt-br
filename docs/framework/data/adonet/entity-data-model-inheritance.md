data-ttu-id="ccffc-106">No entanto, diferentemente das classes em programação orientada a objeto, em um modelo conceitual o tipo derivado herda sempre qualquer os class="sxs-lookup"><span data-stu-id="ccffc-106">However, unlike classes in object-oriented programming, in a conceptual model the derived type always inherits all the data-ttu-id="ccffc-107">Você não pode substituir propriedades herdadas em um tipo derivado.</span><span class="sxs-lookup"><span data-stu-id="ccffc-107">You cannot override inherited properties in a  <span data-ttu-id="ccffc-108">Em um modelo conceitual você pode criar hierarquias de herança em que um tipo derivado herda de outro tipo derivado.</span><span class="sxs-lookup"><span data-stu-id="ccffc-108">In a conceptual model you can build inheritance hierarchies in which a derived type inherits from another derived type.</span></span> <span data-ttu-id="ccffc-109">O tipo na parte superior da hierarquia (um tipo na hierarquia que não é um tipo derivado) é chamado de *raiz do tipo*.</span><span class="sxs-lookup"><span data-stu-id="ccffc-109">The type at the top of the hierarchy (the one type in the hierarchy that is not a derived type) is called the *root type*.</span></span> <span data-ttu-id="ccffc-110">Em uma hierarquia de herança, a [chave de entidade](../../../../docs/framework/data/adonet/entity-key.md) deve ser definido no tipo de raiz.</span><span class="sxs-lookup"><span data-stu-id="ccffc-110">In an inheritance hierarchy, the [entity key](../../../../docs/framework/data/adonet/entity-key.md) must be defined on the root type.</span></span>  
  
 <span data-ttu-id="ccffc-111">Você não pode criar hierarquias de herança em que um tipo derivado herda de mais de um tipo.</span><span class="sxs-lookup"><span data-stu-id="ccffc-111">You cannot build inheritance hierarchies in which a derived type inherits from more than one type.</span></span> <span data-ttu-id="ccffc-112">Por exemplo, em um modelo conceitual com um tipo de entidade de `Book` , você pode definir tipos derivados `FictionBook` e `NonFictionBook` que cada herda de `Book`.</span><span class="sxs-lookup"><span data-stu-id="ccffc-112">For example, in a conceptual model with a `Book` entity type, you could define derived types `FictionBook` and `NonFictionBook` that each inherit from `Book`.</span></span> <span data-ttu-id="ccffc-113">No entanto, você não pode então definir um tipo que herdasse dos tipos de `FictionBook` e de `NonFictionBook` .</span><span class="sxs-lookup"><span data-stu-id="ccffc-113">However, you could not then define a type that inherits from both the `FictionBook` and `NonFictionBook` types.</span></span>
