# <a name="entity-data-model-inheritance"></a><span data-ttu-id="12d5a-102">Modelo de Dados de Entidade: Herança</span><span class="sxs-lookup"><span data-stu-id="12d5a-102">Entity Data Model: Inheritance</span></span>

<span data-ttu-id="12d5a-103">O Modelo de Dados de Entidade (EDM) dá suporte à herança para [tipos de entidade](entity-type.md).</span><span class="sxs-lookup"><span data-stu-id="12d5a-103">The Entity Data Model (EDM) supports inheritance for [entity types](entity-type.md).</span></span> <span data-ttu-id="12d5a-104">Herança em EDM é semelhante à herança para classes em idiomas de programação orientada a objeto.</span><span class="sxs-lookup"><span data-stu-id="12d5a-104">Inheritance in the EDM is similar to inheritance for classes in object-oriented programming languages.</span></span> <span data-ttu-id="12d5a-105">Assim como acontece com classes em linguagens orientadas a objeto, em um modelo conceitual, você pode definir um tipo de entidade (um *tipo derivado*) que herda de outro tipo de entidade (o *tipo base*).</span><span class="sxs-lookup"><span data-stu-id="12d5a-105">Like with classes in object-oriented languages, in a conceptual model you can define an entity type (a *derived type*) that inherits from another entity type (the *base type*).</span></span> <span data-ttu-id="12d5a-106">No entanto, ao contrário das classes na programação orientada a objeto, em um modelo conceitual, o tipo derivado sempre herda todas as [Propriedades](property.md) e [Propriedades de navegação](navigation-property.md) do tipo base.</span><span class="sxs-lookup"><span data-stu-id="12d5a-106">However, unlike classes in object-oriented programming, in a conceptual model the derived type always inherits all the [properties](property.md) and [navigation properties](navigation-property.md) of the base type.</span></span> <span data-ttu-id="12d5a-107">Você não pode substituir propriedades herdadas em um tipo derivado.</span><span class="sxs-lookup"><span data-stu-id="12d5a-107">You cannot override inherited properties in a derived type.</span></span>  
  
 <span data-ttu-id="12d5a-108">Em um modelo conceitual você pode criar hierarquias de herança em que um tipo derivado herda de outro tipo derivado.</span><span class="sxs-lookup"><span data-stu-id="12d5a-108">In a conceptual model you can build inheritance hierarchies in which a derived type inherits from another derived type.</span></span> <span data-ttu-id="12d5a-109">O tipo na parte superior da hierarquia (aquele tipo na hierarquia que não é um tipo derivado) é chamado de *tipo raiz*.</span><span class="sxs-lookup"><span data-stu-id="12d5a-109">The type at the top of the hierarchy (the one type in the hierarchy that is not a derived type) is called the *root type*.</span></span> <span data-ttu-id="12d5a-110">Em uma hierarquia de herança, a [chave de entidade](entity-key.md) deve ser definida no tipo de raiz.</span><span class="sxs-lookup"><span data-stu-id="12d5a-110">In an inheritance hierarchy, the [entity key](entity-key.md) must be defined on the root type.</span></span>  
  
 <span data-ttu-id="12d5a-111">Você não pode criar hierarquias de herança em que um tipo derivado herda de mais de um tipo.</span><span class="sxs-lookup"><span data-stu-id="12d5a-111">You cannot build inheritance hierarchies in which a derived type inherits from more than one type.</span></span> <span data-ttu-id="12d5a-112">Por exemplo, em um modelo conceitual com um tipo de entidade de `Book` , você pode definir tipos derivados `FictionBook` e `NonFictionBook` que cada herda de `Book`.</span><span class="sxs-lookup"><span data-stu-id="12d5a-112">For example, in a conceptual model with a `Book` entity type, you could define derived types `FictionBook` and `NonFictionBook` that each inherit from `Book`.</span></span> <span data-ttu-id="12d5a-113">No entanto, você não pode então definir um tipo que herdasse dos tipos de `FictionBook` e de `NonFictionBook` .</span><span class="sxs-lookup"><span data-stu-id="12d5a-113">However, you could not then define a type that inherits from both the `FictionBook` and `NonFictionBook` types.</span></span>
