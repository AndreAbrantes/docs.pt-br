# <a name="deferred-execution-and-lazy-evaluation-in-linq-to-xml-c"></a><span data-ttu-id="c13fd-102">Execução adiada e avaliação lenta em LINQ to XML (C#)</span><span class="sxs-lookup"><span data-stu-id="c13fd-102">Deferred Execution and Lazy Evaluation in LINQ to XML (C#)</span></span>
<span data-ttu-id="c13fd-103">As operações de consulta e eixo são geralmente implementadas para usar a execução adiada.</span><span class="sxs-lookup"><span data-stu-id="c13fd-103">Query and axis operations are often implemented to use deferred execution.</span></span> <span data-ttu-id="c13fd-104">Este tópico explica os requisitos e as vantagens da execução adiada, além de algumas considerações sobre a implementação.</span><span class="sxs-lookup"><span data-stu-id="c13fd-104">This topic explains the requirements and advantages of deferred execution, and some implementation considerations.</span></span>  
  
## <a name="deferred-execution"></a><span data-ttu-id="c13fd-105">Execução Adiada</span><span class="sxs-lookup"><span data-stu-id="c13fd-105">Deferred Execution</span></span>  
 <span data-ttu-id="c13fd-106">A execução adiada significa que a avaliação de uma expressão será atrasada até que seu valor *realizado* seja realmente necessário.</span><span class="sxs-lookup"><span data-stu-id="c13fd-106">Deferred execution means that the evaluation of an expression is delayed until its *realized* value is actually required.</span></span> <span data-ttu-id="c13fd-107">A execução adiada pode aumentar muito o desempenho quando você precisa manipular grandes coleções de dados, especialmente em programas que contêm uma série de consultas ou manipulações encadeadas.</span><span class="sxs-lookup"><span data-stu-id="c13fd-107">Deferred execution can greatly improve performance when you have to manipulate large data collections, especially in programs that contain a series of chained queries or manipulations.</span></span> <span data-ttu-id="c13fd-108">Na melhor das hipóteses, a execução adiada permite apenas uma única iteração pela coleção de origem.</span><span class="sxs-lookup"><span data-stu-id="c13fd-108">In the best case, deferred execution enables only a single iteration through the source collection.</span></span>  
  
 <span data-ttu-id="c13fd-109">As tecnologias LINQ usam de modo intenso a execução adiada em ambos os membros das classes <xref:System.Linq?displayProperty=nameWithType> centrais e nos métodos de extensão nos diversos namespaces LINQ, como <xref:System.Xml.Linq.Extensions?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="c13fd-109">The LINQ technologies make extensive use of deferred execution in both the members of core <xref:System.Linq?displayProperty=nameWithType> classes and in the extension methods in the various LINQ namespaces, such as <xref:System.Xml.Linq.Extensions?displayProperty=nameWithType>.</span></span>  
  
 <span data-ttu-id="c13fd-110">A execução adiada tem suporte direto na linguagem C# pela palavra-chave [yield](../../../../csharp/language-reference/keywords/yield.md) (na forma da instrução `yield-return`) quando usada dentro de um bloco de iterador.</span><span class="sxs-lookup"><span data-stu-id="c13fd-110">Deferred execution is supported directly in the C# language by the [yield](../../../../csharp/language-reference/keywords/yield.md) keyword (in the form of the `yield-return` statement) when used within an iterator block.</span></span> <span data-ttu-id="c13fd-111">Tal iterador deve retornar uma coleção do tipo <xref:System.Collections.IEnumerator> ou <xref:System.Collections.Generic.IEnumerator%601> (ou um tipo derivado).</span><span class="sxs-lookup"><span data-stu-id="c13fd-111">Such an iterator must return a collection of type <xref:System.Collections.IEnumerator> or <xref:System.Collections.Generic.IEnumerator%601> (or a derived type).</span></span>  
  
## <a name="eager-vs-lazy-evaluation"></a><span data-ttu-id="c13fd-112">Avaliação ansiosa vs. lenta</span><span class="sxs-lookup"><span data-stu-id="c13fd-112">Eager vs. Lazy Evaluation</span></span>  
 <span data-ttu-id="c13fd-113">Quando você escreve um método que implementa a execução adiada, também precisa decidir se deve implementar o método usando a avaliação lenta ou a avaliação ansiosa.</span><span class="sxs-lookup"><span data-stu-id="c13fd-113">When you write a method that implements deferred execution, you also have to decide whether to implement the method using lazy evaluation or eager evaluation.</span></span>  
  
- <span data-ttu-id="c13fd-114">Na *avaliação lenta*, um único elemento da coleção de origem é processado durante cada chamada ao iterador.</span><span class="sxs-lookup"><span data-stu-id="c13fd-114">In *lazy evaluation*, a single element of the source collection is processed during each call to the iterator.</span></span> <span data-ttu-id="c13fd-115">Essa é a forma comum de implementação de iteradores.</span><span class="sxs-lookup"><span data-stu-id="c13fd-115">This is the typical way in which iterators are implemented.</span></span>  
  
- <span data-ttu-id="c13fd-116">Na *avaliação adiantada*, a primeira chamada ao iterador resultará no processamento da coleção inteira.</span><span class="sxs-lookup"><span data-stu-id="c13fd-116">In *eager evaluation*, the first call to the iterator will result in the entire collection being processed.</span></span> <span data-ttu-id="c13fd-117">Uma cópia temporária da coleção de origem também pode ser necessária.</span><span class="sxs-lookup"><span data-stu-id="c13fd-117">A temporary copy of the source collection might also be required.</span></span> <span data-ttu-id="c13fd-118">Por exemplo, o método <xref:System.Linq.Enumerable.OrderBy%2A> precisa classificar toda a coleção antes de retornar o primeiro elemento.</span><span class="sxs-lookup"><span data-stu-id="c13fd-118">For example, the <xref:System.Linq.Enumerable.OrderBy%2A> method has to sort the entire collection before it returns the first element.</span></span>  
  
 <span data-ttu-id="c13fd-119">A avaliação lenta normalmente gera um melhor desempenho porque distribui a sobrecarga de processamento uniformemente por toda a avaliação da coleção e minimiza o uso de dados temporários.</span><span class="sxs-lookup"><span data-stu-id="c13fd-119">Lazy evaluation usually yields better performance because it distributes overhead processing evenly throughout the evaluation of the collection and minimizes the use of temporary data.</span></span> <span data-ttu-id="c13fd-120">Naturalmente, para algumas operações, não há nenhuma outra opção que não seja materializar resultados intermediários.</span><span class="sxs-lookup"><span data-stu-id="c13fd-120">Of course, for some operations, there is no other option than to materialize intermediate results.</span></span>
