## <a name="what-is-pure-functional-transformation"></a><span data-ttu-id="234ad-107">Transformação que é funcional pura?</span><span class="sxs-lookup"><span data-stu-id="234ad-107">What Is Pure Functional Transformation?</span></span>  
 <span data-ttu-id="234ad-108">Na *transformação funcional pura*, um conjunto de funções, chamadas *funções puras*, define como transformar um conjunto de dados estruturados de sua forma original em outra forma.</span><span class="sxs-lookup"><span data-stu-id="234ad-108">In *pure functional transformation*, a set of functions, called *pure functions*, define how to transform a set of structured data from its original form into another form.</span></span> <span data-ttu-id="234ad-109">A palavra "pura" indica que as funções são *combináveis*, que exigem que elas sejam:</span><span class="sxs-lookup"><span data-stu-id="234ad-109">The word "pure" indicates that the functions are *composable*, which requires that they are:</span></span>  
  
-   <span data-ttu-id="234ad-110">*Autocontidas*, de modo que elas possam ser ordenadas e reorganizadas livremente sem complicação ou interdependências com o restante do programa.</span><span class="sxs-lookup"><span data-stu-id="234ad-110">*Self-contained*, so that they can be freely ordered and rearranged without entanglement or interdependencies with the rest of the program.</span></span> <span data-ttu-id="234ad-111">As transformações puras não têm nenhum conhecimento de efetuam-no ou em cima do seu ambiente.</span><span class="sxs-lookup"><span data-stu-id="234ad-111">Pure transformations have no knowledge of or effect upon their environment.</span></span> <span data-ttu-id="234ad-112">Isto é, as funções usadas na transformação não têm *efeito colateral*.</span><span class="sxs-lookup"><span data-stu-id="234ad-112">That is, the functions used in the transformation have no *side effects*.</span></span>  
  
-   <span data-ttu-id="234ad-113">*Sem monitoração de estado*, de modo que a execução da mesma função ou conjunto específico de funções, na mesma entrada, sempre resultará na mesma saída.</span><span class="sxs-lookup"><span data-stu-id="234ad-113">*Stateless*, so that executing the same function or specific set of functions on the same input will always result in the same output.</span></span> <span data-ttu-id="234ad-114">As transformações puras não têm memória do seu uso prévio.</span><span class="sxs-lookup"><span data-stu-id="234ad-114">Pure transformations have no memory of their prior use.</span></span>  
  
> [!IMPORTANT]
>  <span data-ttu-id="234ad-115">No restante deste tutorial, o termo “function” pura é usado em um sentido geral indicar uma abordagem de programação, e não em um recurso de linguagem específica.</span><span class="sxs-lookup"><span data-stu-id="234ad-115">In the rest of this tutorial, the term "pure function" is used in a general sense to indicate a programming approach, and not a specific language feature.</span></span>

>   
>  <span data-ttu-id="234ad-117">Além disso, você não deve confundir funções puras com os métodos puros virtuais em C++.</span><span class="sxs-lookup"><span data-stu-id="234ad-117">Also, you should not confuse pure functions with pure virtual methods in C++.</span></span> <span data-ttu-id="234ad-118">O último indica que a classe recipiente é abstrato e que nenhum corpo do método é fornecido.</span><span class="sxs-lookup"><span data-stu-id="234ad-118">The latter indicates that the containing class is abstract and that no method body is supplied.</span></span>  
  
### <a name="functional-programming"></a><span data-ttu-id="234ad-119">Programação funcional</span><span class="sxs-lookup"><span data-stu-id="234ad-119">Functional Programming</span></span>  
 <span data-ttu-id="234ad-120">A *programação funcional* é uma abordagem de programação que dá suporte diretamente à transformação funcional pura.</span><span class="sxs-lookup"><span data-stu-id="234ad-120">*Functional programming* is a programming approach that directly supports pure functional transformation.</span></span>  
  
 <span data-ttu-id="234ad-121">Historicamente, as linguagens de programação funcionais de uso geral, como o ML, Scheme, Haskell e F#, têm sido, principalmente, de interesse da comunidade acadêmica.</span><span class="sxs-lookup"><span data-stu-id="234ad-121">Historically, general-purpose functional programming languages, such as ML, Scheme, Haskell, and F#, have been primarily of interest to the academic community.</span></span> <span data-ttu-id="234ad-122">Embora sempre foi possível escrever transformações funcionais puras no C#, a dificuldade em fazê-lo não tornou essa opção atrativa à maioria dos programadores.</span><span class="sxs-lookup"><span data-stu-id="234ad-122">Although it has always been possible to write pure functional transformations in C#, the difficulty of doing so has not made it an attractive option to most programmers.</span></span> <span data-ttu-id="234ad-123">Nas versões recentes do C#, no entanto, os novos constructos de linguagem como expressões lambda e inferência de tipos tornaram a programação funcional muito mais fácil e mais produtiva.</span><span class="sxs-lookup"><span data-stu-id="234ad-123">In recent versions of C#, however, new language constructs such as lambda expressions and type inference make it functional programming much easier and more productive.</span></span>
