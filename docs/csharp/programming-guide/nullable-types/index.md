# <a name="nullable-types-c-programming-guide"></a><span data-ttu-id="1bc47-103">Tipos que permitem valor nulo (Guia de Programação em C#)</span><span class="sxs-lookup"><span data-stu-id="1bc47-103">Nullable types (C# Programming Guide)</span></span>

<span data-ttu-id="1bc47-104">Os tipos que permitem valor nulo são instâncias do struct <xref:System.Nullable%601?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="1bc47-104">Nullable types are instances of the <xref:System.Nullable%601?displayProperty=nameWithType> struct.</span></span> <span data-ttu-id="1bc47-105">Os tipos que permitem valor nulo podem representar todos os valores de um tipo subjacente `T` e um valor [null](../../language-reference/keywords/null.md) adicional.</span><span class="sxs-lookup"><span data-stu-id="1bc47-105">Nullable types can represent all the values of an underlying type `T`, and an additional [null](../../language-reference/keywords/null.md) value.</span></span> <span data-ttu-id="1bc47-106">O tipo subjacente `T` pode ser qualquer [tipo de valor](../../language-reference/keywords/value-types.md) que não permite valor nulo.</span><span class="sxs-lookup"><span data-stu-id="1bc47-106">The underlying type `T` can be any non-nullable [value type](../../language-reference/keywords/value-types.md).</span></span> <span data-ttu-id="1bc47-107">`T` não pode ser um tipo de referência.</span><span class="sxs-lookup"><span data-stu-id="1bc47-107">`T` cannot be a reference type.</span></span>

<span data-ttu-id="1bc47-108">Por exemplo, é possível atribuir `null` ou qualquer valor inteiro de <xref:System.Int32.MinValue?displayProperty=nameWithType> até <xref:System.Int32.MaxValue?displayProperty=nameWithType> a um `Nullable<int>` e [true](../../language-reference/keywords/true-literal.md), [false](../../language-reference/keywords/false-literal.md) ou `null` a um `Nullable<bool>`.</span><span class="sxs-lookup"><span data-stu-id="1bc47-108">For example, you can assign `null` or any integer value from <xref:System.Int32.MinValue?displayProperty=nameWithType> to <xref:System.Int32.MaxValue?displayProperty=nameWithType> to a `Nullable<int>` and [true](../../language-reference/keywords/true-literal.md), [false](../../language-reference/keywords/false-literal.md), or `null` to a `Nullable<bool>`.</span></span>

<span data-ttu-id="1bc47-109">Use um tipo que permite valor nulo quando precisar representar o valor indefinido de um tipo subjacente.</span><span class="sxs-lookup"><span data-stu-id="1bc47-109">You use a nullable type when you need to represent the undefined value of an underlying type.</span></span> <span data-ttu-id="1bc47-110">Uma variável booliana pode ter apenas dois valores: true e false.</span><span class="sxs-lookup"><span data-stu-id="1bc47-110">A Boolean variable can have only two values: true and false.</span></span> <span data-ttu-id="1bc47-111">Não há nenhum valor "indefinido".</span><span class="sxs-lookup"><span data-stu-id="1bc47-111">There is no "undefined" value.</span></span> <span data-ttu-id="1bc47-112">Em muitos aplicativos de programação, principalmente nas interações de banco de dados, um valor de variável pode estar indefinido ou ausente.</span><span class="sxs-lookup"><span data-stu-id="1bc47-112">In many programming applications, most notably database interactions, a variable value can be undefined or missing.</span></span> <span data-ttu-id="1bc47-113">Por exemplo, um campo em um banco de dados pode conter os valores true ou false ou pode não conter nenhum valor.</span><span class="sxs-lookup"><span data-stu-id="1bc47-113">For example, a field in a database may contain the values true or false, or it may contain no value at all.</span></span> <span data-ttu-id="1bc47-114">Use um tipo `Nullable<bool>` nesse caso.</span><span class="sxs-lookup"><span data-stu-id="1bc47-114">You use a `Nullable<bool>` type in that case.</span></span>

<span data-ttu-id="1bc47-115">Os tipos que permitem valor nulo têm as seguintes características:</span><span class="sxs-lookup"><span data-stu-id="1bc47-115">Nullable types have the following characteristics:</span></span>
  
- <span data-ttu-id="1bc47-116">Os tipos que permitem valor nulo representam variáveis de tipo de valor que podem ser atribuídas ao valor `null`.</span><span class="sxs-lookup"><span data-stu-id="1bc47-116">Nullable types represent value-type variables that can be assigned the `null` value.</span></span> <span data-ttu-id="1bc47-117">Você não pode criar um tipo que permite valor nulo com base em um tipo de referência.</span><span class="sxs-lookup"><span data-stu-id="1bc47-117">You cannot create a nullable type based on a reference type.</span></span> <span data-ttu-id="1bc47-118">(Os tipos de referência já dão suporte ao valor `null`.)</span><span class="sxs-lookup"><span data-stu-id="1bc47-118">(Reference types already support the `null` value.)</span></span>  
  
- <span data-ttu-id="1bc47-119">A sintaxe `T?` é uma abreviação para `Nullable<T>`.</span><span class="sxs-lookup"><span data-stu-id="1bc47-119">The syntax `T?` is shorthand for `Nullable<T>`.</span></span> <span data-ttu-id="1bc47-120">As duas formas são intercambiáveis.</span><span class="sxs-lookup"><span data-stu-id="1bc47-120">The two forms are interchangeable.</span></span>  
  
- <span data-ttu-id="1bc47-121">Atribua um valor a um tipo que permite valor nulo como você faria para um tipo de valor subjacente: `int? x = 10;` ou `double? d = 4.108;`.</span><span class="sxs-lookup"><span data-stu-id="1bc47-121">Assign a value to a nullable type just as you would for an underlying value type: `int? x = 10;` or `double? d = 4.108;`.</span></span> <span data-ttu-id="1bc47-122">Também é possível atribuir o valor `null`: `int? x = null;`.</span><span class="sxs-lookup"><span data-stu-id="1bc47-122">You also can assign the `null` value: `int? x = null;`.</span></span>  
  
- <span data-ttu-id="1bc47-123">Use as propriedades somente leitura <xref:System.Nullable%601.HasValue%2A?displayProperty=nameWithType> e <xref:System.Nullable%601.Value%2A?displayProperty=nameWithType> para testar em relação ao valor nulo e recuperar o valor, conforme mostrado no exemplo a seguir: `if (x.HasValue) y = x.Value;`</span><span class="sxs-lookup"><span data-stu-id="1bc47-123">Use the <xref:System.Nullable%601.HasValue%2A?displayProperty=nameWithType> and <xref:System.Nullable%601.Value%2A?displayProperty=nameWithType> readonly properties to test for null and retrieve the value, as shown in the following example: `if (x.HasValue) y = x.Value;`</span></span>  
  
  - <span data-ttu-id="1bc47-124">A propriedade <xref:System.Nullable%601.HasValue%2A> retornará `true` se a variável contiver um valor ou `false` se for `null`.</span><span class="sxs-lookup"><span data-stu-id="1bc47-124">The <xref:System.Nullable%601.HasValue%2A> property returns `true` if the variable contains a value, or `false` if it's `null`.</span></span>
  
  - <span data-ttu-id="1bc47-125">A propriedade <xref:System.Nullable%601.Value%2A> retornará um valor se <xref:System.Nullable%601.HasValue%2A> retornar `true`.</span><span class="sxs-lookup"><span data-stu-id="1bc47-125">The <xref:System.Nullable%601.Value%2A> property returns a value if <xref:System.Nullable%601.HasValue%2A> returns `true`.</span></span> <span data-ttu-id="1bc47-126">Caso contrário, um <xref:System.InvalidOperationException> será gerado.</span><span class="sxs-lookup"><span data-stu-id="1bc47-126">Otherwise, an <xref:System.InvalidOperationException> is thrown.</span></span>  
  
- <span data-ttu-id="1bc47-127">Também é possível usar os operadores `==` e `!=` com um tipo que permite valor nulo, conforme mostrado no exemplo a seguir: `if (x != null) y = x.Value;`.</span><span class="sxs-lookup"><span data-stu-id="1bc47-127">You can also use the `==` and `!=` operators with a nullable type, as shown in the following example: `if (x != null) y = x.Value;`.</span></span> <span data-ttu-id="1bc47-128">Se `a` e `b` forem ambos nulos, `a == b` será avaliado como `true`.</span><span class="sxs-lookup"><span data-stu-id="1bc47-128">If `a` and `b` are both null, `a == b` evaluates to `true`.</span></span>  

- <span data-ttu-id="1bc47-129">A partir do C# 7.0, você pode usar [correspondência de padrões](../../pattern-matching.md#the-is-type-pattern-expression) para examinar e obter um valor de um tipo que permite valor nulo: `if (x is int valueOfX) y = valueOfX;`.</span><span class="sxs-lookup"><span data-stu-id="1bc47-129">Beginning with C# 7.0, you can use [pattern matching](../../pattern-matching.md#the-is-type-pattern-expression) to both examine and get a value of a nullable type: `if (x is int valueOfX) y = valueOfX;`.</span></span>
  
- <span data-ttu-id="1bc47-130">O valor padrão de `T?` é uma instância cuja propriedade <xref:System.Nullable%601.HasValue%2A> retorna `false`.</span><span class="sxs-lookup"><span data-stu-id="1bc47-130">The default value of `T?` is an instance whose <xref:System.Nullable%601.HasValue%2A> property returns `false`.</span></span>  

- <span data-ttu-id="1bc47-131">Use o método <xref:System.Nullable%601.GetValueOrDefault> para retornar o valor atribuído ou o valor [padrão](../../language-reference/keywords/default-values-table.md) do tipo de valor subjacente se o valor do tipo que permite valor nulo for `null`.</span><span class="sxs-lookup"><span data-stu-id="1bc47-131">Use the <xref:System.Nullable%601.GetValueOrDefault> method to return either the assigned value, or the [default](../../language-reference/keywords/default-values-table.md) value of the underlying value type if the value of the nullable type is `null`.</span></span>  

- <span data-ttu-id="1bc47-132">Use o método <xref:System.Nullable%601.GetValueOrDefault(%600)> para retornar o valor atribuído ou o valor padrão fornecido se o valor do tipo que permite valor nulo for `null`.</span><span class="sxs-lookup"><span data-stu-id="1bc47-132">Use the <xref:System.Nullable%601.GetValueOrDefault(%600)> method to return either the assigned value, or the provided default value if the value of the nullable type is `null`.</span></span>
  
- <span data-ttu-id="1bc47-133">Use o [operador de coalescência nula](../../language-reference/operators/null-coalescing-operator.md), `??`, para atribuir um valor a um tipo subjacente baseado em um valor do tipo que permite valor nulo: `int? x = null; int y = x ?? -1;`.</span><span class="sxs-lookup"><span data-stu-id="1bc47-133">Use the [null-coalescing operator](../../language-reference/operators/null-coalescing-operator.md), `??`, to assign a value to an underlying type based on a value of the nullable type: `int? x = null; int y = x ?? -1;`.</span></span> <span data-ttu-id="1bc47-134">No exemplo, como `x` é null, o valor de resultado `y` é `-1`.</span><span class="sxs-lookup"><span data-stu-id="1bc47-134">In the example, since `x` is null, the result value of `y` is `-1`.</span></span>

- <span data-ttu-id="1bc47-135">Se uma conversão definida pelo usuário for definida entre dois tipos de dados, a mesma conversão também poderá ser usada com as versões que permitem valor nulo desses tipos de dados.</span><span class="sxs-lookup"><span data-stu-id="1bc47-135">If a user-defined conversion is defined between two data types, the same conversion can also be used with the nullable versions of these data types.</span></span>
  
- <span data-ttu-id="1bc47-136">Os tipos que permitem valor nulo aninhados não são permitidos.</span><span class="sxs-lookup"><span data-stu-id="1bc47-136">Nested nullable types are not allowed.</span></span> <span data-ttu-id="1bc47-137">A linha a seguir não é compilada: `Nullable<Nullable<int>> n;`</span><span class="sxs-lookup"><span data-stu-id="1bc47-137">The following line doesn't compile: `Nullable<Nullable<int>> n;`</span></span>
