## <a name="value-equality"></a><span data-ttu-id="8ac0d-120">Igualdade de valor</span><span class="sxs-lookup"><span data-stu-id="8ac0d-120">Value equality</span></span>

 <span data-ttu-id="8ac0d-121">Igualdade de valor significa que dois objetos contêm o mesmo valor ou valores.</span><span class="sxs-lookup"><span data-stu-id="8ac0d-121">Value equality means that two objects contain the same value or values.</span></span> <span data-ttu-id="8ac0d-122">Para tipos de valor primitivos, como [int](../../language-reference/builtin-types/integral-numeric-types.md) ou [bool](../../language-reference/builtin-types/bool.md), os testes de igualdade de valor são simples.</span><span class="sxs-lookup"><span data-stu-id="8ac0d-122">For primitive value types such as [int](../../language-reference/builtin-types/integral-numeric-types.md) or [bool](../../language-reference/builtin-types/bool.md), tests for value equality are straightforward.</span></span> <span data-ttu-id="8ac0d-123">É possível usar o operador [==](../../language-reference/operators/equality-operators.md#equality-operator-), conforme mostrado no exemplo a seguir.</span><span class="sxs-lookup"><span data-stu-id="8ac0d-123">You can use the [==](../../language-reference/operators/equality-operators.md#equality-operator-) operator, as shown in the following example.</span></span>  
  
```csharp  
int a = GetOriginalValue();  
int b = GetCurrentValue();  
  
// Test for value equality.   
if( b == a)   
{  
    // The two integers are equal.  
}  
```  
  
 <span data-ttu-id="8ac0d-124">Para a maioria dos outros tipos, o teste de igualdade de valor é mais complexo, pois é necessário entender como o tipo o define.</span><span class="sxs-lookup"><span data-stu-id="8ac0d-124">For most other types, testing for value equality is more complex because it requires that you understand how the type defines it.</span></span> <span data-ttu-id="8ac0d-125">Para classes e structs que têm vários campos ou propriedades, a igualdade de valor geralmente é definida para determinar que todos os campos ou propriedades tenham o mesmo valor.</span><span class="sxs-lookup"><span data-stu-id="8ac0d-125">For classes and structs that have multiple fields or properties, value equality is often defined to mean that all fields or properties have the same value.</span></span> <span data-ttu-id="8ac0d-126">Por exemplo, dois objetos `Point` podem ser definidos para serem equivalentes se pointA.X for igual a pointB.X e pointA.Y for igual a pointB.Y.</span><span class="sxs-lookup"><span data-stu-id="8ac0d-126">For example, two `Point` objects might be defined to be equivalent if pointA.X is equal to pointB.X and pointA.Y is equal to pointB.Y.</span></span>  
  
<span data-ttu-id="8ac0d-127">No entanto, não há nenhuma exigência de que a equivalência seja baseada em todos os campos em um tipo.</span><span class="sxs-lookup"><span data-stu-id="8ac0d-127">However, there is no requirement that equivalence be based on all the fields in a type.</span></span> <span data-ttu-id="8ac0d-128">Ela pode ser baseada em um subconjunto.</span><span class="sxs-lookup"><span data-stu-id="8ac0d-128">It can be based on a subset.</span></span> <span data-ttu-id="8ac0d-129">Ao comparar tipos que não são de sua propriedade, certifique-se de que a forma como a equivalência é definida especificamente para esse tipo foi entendida.</span><span class="sxs-lookup"><span data-stu-id="8ac0d-129">When you compare types that you do not own, you should make sure to understand specifically how equivalence is defined for that type.</span></span> <span data-ttu-id="8ac0d-130">Para obter mais informações sobre como definir a igualdade de valor em suas próprias classes e estruturas, consulte [como definir a igualdade de valor para um tipo](./how-to-define-value-equality-for-a-type.md).</span><span class="sxs-lookup"><span data-stu-id="8ac0d-130">For more information about how to define value equality in your own classes and structs, see [How to define value equality for a type](./how-to-define-value-equality-for-a-type.md).</span></span>
