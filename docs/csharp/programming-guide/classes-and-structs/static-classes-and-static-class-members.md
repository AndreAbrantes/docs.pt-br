# <a name="static-classes-and-static-class-members-c-programming-guide"></a><span data-ttu-id="9ac1b-102">Classes static e membros de classes static (Guia de Programação em C#)</span><span class="sxs-lookup"><span data-stu-id="9ac1b-102">Static Classes and Static Class Members (C# Programming Guide)</span></span>
<span data-ttu-id="9ac1b-103">Uma classe [static](../../../csharp/language-reference/keywords/static.md) é basicamente o mesmo que uma classe não estática, mas há uma diferença: uma classe estática não pode ser instanciada.</span><span class="sxs-lookup"><span data-stu-id="9ac1b-103">A [static](../../../csharp/language-reference/keywords/static.md) class is basically the same as a non-static class, but there is one difference: a static class cannot be instantiated.</span></span> <span data-ttu-id="9ac1b-104">Em outras palavras, você não pode usar a palavra-chave [new](../../../csharp/language-reference/keywords/new.md) para criar uma variável do tipo de classe.</span><span class="sxs-lookup"><span data-stu-id="9ac1b-104">In other words, you cannot use the [new](../../../csharp/language-reference/keywords/new.md) keyword to create a variable of the class type.</span></span> <span data-ttu-id="9ac1b-105">Como não há nenhuma variável de instância, você acessa os membros de uma classe estática usando o próprio nome de classe.</span><span class="sxs-lookup"><span data-stu-id="9ac1b-105">Because there is no instance variable, you access the members of a static class by using the class name itself.</span></span> <span data-ttu-id="9ac1b-106">Por exemplo, se você tiver uma classe estática denominada `UtilityClass` que tem um método público chamado `MethodA`, chame o método, conforme mostrado no exemplo a seguir:</span><span class="sxs-lookup"><span data-stu-id="9ac1b-106">For example, if you have a static class that is named `UtilityClass` that has a public method named `MethodA`, you call the method as shown in the following example:</span></span>  
  
```csharp  
UtilityClass.MethodA();  
```  
  
 <span data-ttu-id="9ac1b-107">Uma classe estática pode ser usada como um contêiner conveniente para conjuntos de métodos que operam apenas em parâmetros de entrada e não precisam obter ou definir campos de instância internos.</span><span class="sxs-lookup"><span data-stu-id="9ac1b-107">A static class can be used as a convenient container for sets of methods that just operate on input parameters and do not have to get or set any internal instance fields.</span></span> <span data-ttu-id="9ac1b-108">Por exemplo, na biblioteca de classes .NET Framework, a classe estática <xref:System.Math?displayProperty=nameWithType> contém métodos que executam operações matemáticas, sem a necessidade de armazenar ou recuperar dados que são exclusivos de uma determinada instância da classe <xref:System.Math>.</span><span class="sxs-lookup"><span data-stu-id="9ac1b-108">For example, in the .NET Framework Class Library, the static <xref:System.Math?displayProperty=nameWithType> class contains methods that perform mathematical operations, without any requirement to store or retrieve data that is unique to a particular instance of the <xref:System.Math> class.</span></span> <span data-ttu-id="9ac1b-109">Ou seja, você aplica os membros da classe especificando o nome de classe e o nome do método, conforme mostrado no exemplo a seguir.</span><span class="sxs-lookup"><span data-stu-id="9ac1b-109">That is, you apply the members of the class by specifying the class name and the method name, as shown in the following example.</span></span>  
  
```csharp  
double dub = -3.14;  
Console.WriteLine(Math.Abs(dub));  
Console.WriteLine(Math.Floor(dub));  
Console.WriteLine(Math.Round(Math.Abs(dub)));  
  
// Output:  
// 3.14  
// -4  
// 3  
```  
  
 <span data-ttu-id="9ac1b-110">Como é o caso com todos os tipos de classe, as informações de tipo de uma classe estática são carregadas pelo CLR (Common Language Runtime) [!INCLUDE[dnprdnshort](~/includes/dnprdnshort-md.md)] quando o programa que faz referência à classe é carregado.</span><span class="sxs-lookup"><span data-stu-id="9ac1b-110">As is the case with all class types, the type information for a static class is loaded by the [!INCLUDE[dnprdnshort](~/includes/dnprdnshort-md.md)] common language runtime (CLR) when the program that references the class is loaded.</span></span> <span data-ttu-id="9ac1b-111">O programa não pode especificar exatamente quando a classe é carregada.</span><span class="sxs-lookup"><span data-stu-id="9ac1b-111">The program cannot specify exactly when the class is loaded.</span></span> <span data-ttu-id="9ac1b-112">No entanto, é garantido que ela será carregada e terá seus campos inicializados e seu construtor estático chamado antes que a classe seja referenciada pela primeira vez em seu programa.</span><span class="sxs-lookup"><span data-stu-id="9ac1b-112">However, it is guaranteed to be loaded and to have its fields initialized and its static constructor called before the class is referenced for the first time in your program.</span></span> <span data-ttu-id="9ac1b-113">Um construtor estático é chamado apenas uma vez e uma classe estática permanece na memória pelo tempo de vida do domínio do aplicativo em que seu programa reside.</span><span class="sxs-lookup"><span data-stu-id="9ac1b-113">A static constructor is only called one time, and a static class remains in memory for the lifetime of the application domain in which your program resides.</span></span>  
  
> [!NOTE]
>  <span data-ttu-id="9ac1b-114">Para criar uma classe não estática que permite que apenas uma instância de si mesma seja criada, consulte [Implementando singleton no C#](https://msdn.microsoft.com/library/ms998558.aspx).</span><span class="sxs-lookup"><span data-stu-id="9ac1b-114">To create a non-static class that allows only one instance of itself to be created, see [Implementing Singleton in C#](https://msdn.microsoft.com/library/ms998558.aspx).</span></span>  
  
 <span data-ttu-id="9ac1b-115">A lista a seguir fornece os principais recursos de uma classe estática:</span><span class="sxs-lookup"><span data-stu-id="9ac1b-115">The following list provides the main features of a static class:</span></span>  
  
-   <span data-ttu-id="9ac1b-116">Contém apenas membros estáticos.</span><span class="sxs-lookup"><span data-stu-id="9ac1b-116">Contains only static members.</span></span>  
  
-   <span data-ttu-id="9ac1b-117">Não pode ser instanciada.</span><span class="sxs-lookup"><span data-stu-id="9ac1b-117">Cannot be instantiated.</span></span>  
  
-   <span data-ttu-id="9ac1b-118">É lacrada.</span><span class="sxs-lookup"><span data-stu-id="9ac1b-118">Is sealed.</span></span>  
  
-   <span data-ttu-id="9ac1b-119">Não pode conter [Construtores de instância](../../../csharp/programming-guide/classes-and-structs/instance-constructors.md).</span><span class="sxs-lookup"><span data-stu-id="9ac1b-119">Cannot contain [Instance Constructors](../../../csharp/programming-guide/classes-and-structs/instance-constructors.md).</span></span>  
  
 <span data-ttu-id="9ac1b-120">Criar uma classe estática é, portanto, basicamente o mesmo que criar uma classe que contém apenas membros estáticos e um construtor particular.</span><span class="sxs-lookup"><span data-stu-id="9ac1b-120">Creating a static class is therefore basically the same as creating a class that contains only static members and a private constructor.</span></span> <span data-ttu-id="9ac1b-121">Um construtor particular impede que a classe seja instanciada.</span><span class="sxs-lookup"><span data-stu-id="9ac1b-121">A private constructor prevents the class from being instantiated.</span></span> <span data-ttu-id="9ac1b-122">A vantagem de usar uma classe estática é que o compilador pode verificar se nenhum membro de instância foi adicionado acidentalmente.</span><span class="sxs-lookup"><span data-stu-id="9ac1b-122">The advantage of using a static class is that the compiler can check to make sure that no instance members are accidentally added.</span></span> <span data-ttu-id="9ac1b-123">O compilador garantirá que as instâncias dessa classe não possam ser criadas.</span><span class="sxs-lookup"><span data-stu-id="9ac1b-123">The compiler will guarantee that instances of this class cannot be created.</span></span>  
  
 <span data-ttu-id="9ac1b-124">Classes estáticas são lacradas e, portanto, não podem ser herdadas.</span><span class="sxs-lookup"><span data-stu-id="9ac1b-124">Static classes are sealed and therefore cannot be inherited.</span></span> <span data-ttu-id="9ac1b-125">Elas não podem herdar de qualquer classe, exceto por <xref:System.Object>.</span><span class="sxs-lookup"><span data-stu-id="9ac1b-125">They cannot inherit from any class except <xref:System.Object>.</span></span> <span data-ttu-id="9ac1b-126">Classes estáticas não podem conter um construtor de instância. No entanto, elas podem conter um construtor estático.</span><span class="sxs-lookup"><span data-stu-id="9ac1b-126">Static classes cannot contain an instance constructor; however, they can contain a static constructor.</span></span> <span data-ttu-id="9ac1b-127">Classes não estáticas também devem definir um construtor estático se a classe contiver membros estáticos que exigem inicialização não trivial.</span><span class="sxs-lookup"><span data-stu-id="9ac1b-127">Non-static classes should also define a static constructor if the class contains static members that require non-trivial initialization.</span></span> <span data-ttu-id="9ac1b-128">Para obter mais informações, consulte [Construtores estáticos](../../../csharp/programming-guide/classes-and-structs/static-constructors.md).</span><span class="sxs-lookup"><span data-stu-id="9ac1b-128">For more information, see [Static Constructors](../../../csharp/programming-guide/classes-and-structs/static-constructors.md).</span></span>

 <span data-ttu-id="9ac1b-132">Uma classe não estática não pode conter métodos, campos, propriedades ou eventos estáticos.</span><span class="sxs-lookup"><span data-stu-id="9ac1b-132">A non-static class can contain static methods, fields, properties, or events.</span></span> <span data-ttu-id="9ac1b-133">O membro estático pode ser chamado em uma classe, mesmo quando nenhuma instância da classe foi criada.</span><span class="sxs-lookup"><span data-stu-id="9ac1b-133">The static member is callable on a class even when no instance of the class has been created.</span></span> <span data-ttu-id="9ac1b-134">O membro estático sempre é acessado pelo nome de classe, não pelo nome da instância.</span><span class="sxs-lookup"><span data-stu-id="9ac1b-134">The static member is always accessed by the class name, not the instance name.</span></span> <span data-ttu-id="9ac1b-135">Existe apenas uma cópia de um membro estático, independentemente de quantas instâncias da classe forem criadas.</span><span class="sxs-lookup"><span data-stu-id="9ac1b-135">Only one copy of a static member exists, regardless of how many instances of the class are created.</span></span> <span data-ttu-id="9ac1b-136">Propriedades e métodos estáticos não podem acessar eventos e campos não estáticos no tipo que os contêm e não podem acessar uma variável de instância de nenhum objeto, a menos que ele seja passado explicitamente em um parâmetro de método.</span><span class="sxs-lookup"><span data-stu-id="9ac1b-136">Static methods and properties cannot access non-static fields and events in their containing type, and they cannot access an instance variable of any object unless it is explicitly passed in a method parameter.</span></span>

 <span data-ttu-id="9ac1b-139">Métodos estáticos podem ser sobrecarregados, mas não substituídos, porque pertencem à classe e não a qualquer instância da classe.</span><span class="sxs-lookup"><span data-stu-id="9ac1b-139">Static methods can be overloaded but not overridden, because they belong to the class, and not to any instance of the class.</span></span>
