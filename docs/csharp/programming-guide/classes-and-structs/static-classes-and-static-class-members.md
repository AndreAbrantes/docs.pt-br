<span data-ttu-id="cc31a-105">Uma classe [static](../../language-reference/keywords/static.md) é basicamente o mesmo que uma classe não estática, mas há uma diferença: uma classe estática não pode ser instanciada.</span><span class="sxs-lookup"><span data-stu-id="cc31a-105">A [static](../../language-reference/keywords/static.md) class is basically the same as a non-static class, but there is one difference: a static class cannot be instantiated.</span></span> <span data-ttu-id="cc31a-106">Em outras palavras, você não pode usar o operador [new](../../language-reference/operators/new-operator.md) para criar uma variável do tipo de classe.</span><span class="sxs-lookup"><span data-stu-id="cc31a-106">In other words, you cannot use the [new](../../language-reference/operators/new-operator.md) operator to create a variable of the class type.</span></span> <span data-ttu-id="cc31a-107">Como não há nenhuma variável de instância, você acessa os membros de uma classe estática usando o próprio nome de classe.</span><span class="sxs-lookup"><span data-stu-id="cc31a-107">Because there is no instance variable, you access the members of a static class by using the class name itself.</span></span> <span data-ttu-id="cc31a-108">Por exemplo, se houver uma classe estática chamada `UtilityClass` com um método público chamado `MethodA`, chame o método, como mostra o exemplo a seguir:</span><span class="sxs-lookup"><span data-stu-id="cc31a-108">For example, if you have a static class that is named `UtilityClass` that has a public static method named `MethodA`, you call the method as shown in the following example:</span></span>

```csharp  
UtilityClass.MethodA();  
```  
  
 <span data-ttu-id="cc31a-109">Uma classe estática pode ser usada como um contêiner conveniente para conjuntos de métodos que operam apenas em parâmetros de entrada e não precisam obter ou definir campos de instância internos.</span><span class="sxs-lookup"><span data-stu-id="cc31a-109">A static class can be used as a convenient container for sets of methods that just operate on input parameters and do not have to get or set any internal instance fields.</span></span> <span data-ttu-id="cc31a-110">Por exemplo, na biblioteca de classes do .NET, a <xref:System.Math?displayProperty=nameWithType> classe estática contém métodos que executam operações matemáticas, sem nenhum requisito para armazenar ou recuperar dados que sejam exclusivos de uma instância específica da <xref:System.Math> classe.</span><span class="sxs-lookup"><span data-stu-id="cc31a-110">For example, in the .NET Class Library, the static <xref:System.Math?displayProperty=nameWithType> class contains methods that perform mathematical operations, without any requirement to store or retrieve data that is unique to a particular instance of the <xref:System.Math> class.</span></span> <span data-ttu-id="cc31a-111">Ou seja, você aplica os membros da classe especificando o nome de classe e o nome do método, conforme mostrado no exemplo a seguir.</span><span class="sxs-lookup"><span data-stu-id="cc31a-111">That is, you apply the members of the class by specifying the class name and the method name, as shown in the following example.</span></span>  
  
```csharp

Console.WriteLine(Math.Abs(dub));  
Console.WriteLine(Math.Floor(dub));  
Console.WriteLine(Math.Round(Math.Abs(dub)));  
  
// Output:  
// 3.14  
// -4  
// 3  
```  
  
 <span data-ttu-id="cc31a-112">Como é o caso com todos os tipos de classe, as informações de tipo de uma classe estática são carregadas pelo tempo de execução do .NET quando o programa que faz referência à classe é carregado.</span><span class="sxs-lookup"><span data-stu-id="cc31a-112">As is the case with all class types, the type information for a static class is loaded by the .NET runtime when the program that references the class is loaded.</span></span> <span data-ttu-id="cc31a-113">O programa não pode especificar exatamente quando a classe é carregada.</span><span class="sxs-lookup"><span data-stu-id="cc31a-113">The program cannot specify exactly when the class is loaded.</span></span> <span data-ttu-id="cc31a-114">No entanto, é garantido que ela será carregada e terá seus campos inicializados e seu construtor estático chamado antes que a classe seja referenciada pela primeira vez em seu programa.</span><span class="sxs-lookup"><span data-stu-id="cc31a-114">However, it is guaranteed to be loaded and to have its fields initialized and its static constructor called before the class is referenced for the first time in your program.</span></span> <span data-ttu-id="cc31a-115">Um construtor estático é chamado apenas uma vez e uma classe estática permanece na memória pelo tempo de vida do domínio do aplicativo em que seu programa reside.</span><span class="sxs-lookup"><span data-stu-id="cc31a-115">A static constructor is only called one time, and a static class remains in memory for the lifetime of the application domain in which your program resides.</span></span>

 <span data-ttu-id="cc31a-122">Criar uma classe estática é, portanto, basicamente o mesmo que criar uma classe que contém apenas membros estáticos e um construtor particular.</span><span class="sxs-lookup"><span data-stu-id="cc31a-122">Creating a static class is therefore basically the same as creating a class that contains only static members and a private constructor.</span></span> <span data-ttu-id="cc31a-123">Um construtor particular impede que a classe seja instanciada.</span><span class="sxs-lookup"><span data-stu-id="cc31a-123">A private constructor prevents the class from being instantiated.</span></span> <span data-ttu-id="cc31a-124">A vantagem de usar uma classe estática é que o compilador pode verificar se nenhum membro de instância foi adicionado acidentalmente.</span><span class="sxs-lookup"><span data-stu-id="cc31a-124">The advantage of using a static class is that the compiler can check to make sure that no instance members are accidentally added.</span></span> <span data-ttu-id="cc31a-125">O compilador garantirá que as instâncias dessa classe não possam ser criadas.</span><span class="sxs-lookup"><span data-stu-id="cc31a-125">The compiler will guarantee that instances of this class cannot be created.</span></span>  
  
 <span data-ttu-id="cc31a-126">Classes estáticas são lacradas e, portanto, não podem ser herdadas.</span><span class="sxs-lookup"><span data-stu-id="cc31a-126">Static classes are sealed and therefore cannot be inherited.</span></span> <span data-ttu-id="cc31a-127">Elas não podem herdar de qualquer classe, exceto por <xref:System.Object>.</span><span class="sxs-lookup"><span data-stu-id="cc31a-127">They cannot inherit from any class except <xref:System.Object>.</span></span> <span data-ttu-id="cc31a-128">Classes estáticas não podem conter um construtor de instância.</span><span class="sxs-lookup"><span data-stu-id="cc31a-128">Static classes cannot contain an instance constructor.</span></span> <span data-ttu-id="cc31a-129">No entanto, eles podem conter um construtor estático.</span><span class="sxs-lookup"><span data-stu-id="cc31a-129">However, they can contain a static constructor.</span></span> <span data-ttu-id="cc31a-130">Classes não estáticas também devem definir um construtor estático se a classe contiver membros estáticos que exigem inicialização não trivial.</span><span class="sxs-lookup"><span data-stu-id="cc31a-130">Non-static classes should also define a static constructor if the class contains static members that require non-trivial initialization.</span></span> <span data-ttu-id="cc31a-131">Para obter mais informações, consulte [Construtores estáticos](./static-constructors.md).</span><span class="sxs-lookup"><span data-stu-id="cc31a-131">For more information, see [Static Constructors](./static-constructors.md).</span></span>
