<span data-ttu-id="9f3bd-105">Os recursos de manipulação de exceção da linguagem C# ajudam você a lidar com quaisquer situações excepcionais ou inesperadas que ocorram quando um programa for executado.</span><span class="sxs-lookup"><span data-stu-id="9f3bd-105">The C# language's exception handling features help you deal with any unexpected or exceptional situations that occur when a program is running.</span></span> <span data-ttu-id="9f3bd-106">A manipulação de exceção usa as palavras-chave `try`, `catch` e `finally` para executar ações que podem não ser bem-sucedidas, lidar com falhas quando decidir se é razoável fazer isso e limpar recursos posteriormente.</span><span class="sxs-lookup"><span data-stu-id="9f3bd-106">Exception handling uses the `try`, `catch`, and `finally` keywords to try actions that may not succeed, to handle failures when you decide that it is reasonable to do so, and to clean up resources afterward.</span></span> <span data-ttu-id="9f3bd-107">As exceções podem ser geradas pelo Common Language Runtime (CLR), por bibliotecas .NET ou de terceiros, ou por código de aplicativo.</span><span class="sxs-lookup"><span data-stu-id="9f3bd-107">Exceptions can be generated by the common language runtime (CLR), by .NET or third-party libraries, or by application code.</span></span> <span data-ttu-id="9f3bd-108">As exceções são criadas usando a palavra-chave `throw`.</span><span class="sxs-lookup"><span data-stu-id="9f3bd-108">Exceptions are created by using the `throw` keyword.</span></span>

## <a name="exceptions-overview"></a><span data-ttu-id="9f3bd-114">Visão geral sobre exceções</span><span class="sxs-lookup"><span data-stu-id="9f3bd-114">Exceptions Overview</span></span>

<span data-ttu-id="9f3bd-115">As exceções têm as seguintes propriedades:</span><span class="sxs-lookup"><span data-stu-id="9f3bd-115">Exceptions have the following properties:</span></span>

- <span data-ttu-id="9f3bd-116">As exceções são tipos que derivam, por fim, de `System.Exception`.</span><span class="sxs-lookup"><span data-stu-id="9f3bd-116">Exceptions are types that all ultimately derive from `System.Exception`.</span></span>
- <span data-ttu-id="9f3bd-117">Use um bloco `try` nas instruções que podem lançar exceções.</span><span class="sxs-lookup"><span data-stu-id="9f3bd-117">Use a `try` block around the statements that might throw exceptions.</span></span>
- <span data-ttu-id="9f3bd-118">Quando ocorre uma exceção no bloco `try`, o fluxo de controle vai para o primeiro manipulador de exceção associada que está presente em qualquer lugar na pilha de chamadas.</span><span class="sxs-lookup"><span data-stu-id="9f3bd-118">Once an exception occurs in the `try` block, the flow of control jumps to the first associated exception handler that is present anywhere in the call stack.</span></span> <span data-ttu-id="9f3bd-119">No C#, a palavra-chave `catch` é usada para definir um manipulador de exceção.</span><span class="sxs-lookup"><span data-stu-id="9f3bd-119">In C#, the `catch` keyword is used to define an exception handler.</span></span>
- <span data-ttu-id="9f3bd-120">Se nenhum manipulador de exceção para uma determinada exceção estiver presente, o programa interromperá a execução com uma mensagem de erro.</span><span class="sxs-lookup"><span data-stu-id="9f3bd-120">If no exception handler for a given exception is present, the program stops executing with an error message.</span></span>
- <span data-ttu-id="9f3bd-121">Não capture uma exceção a menos que você possa manipulá-la e deixar o aplicativo em um estado conhecido.</span><span class="sxs-lookup"><span data-stu-id="9f3bd-121">Do not catch an exception unless you can handle it and leave the application in a known state.</span></span> <span data-ttu-id="9f3bd-122">Se você detectar `System.Exception` , recrie-a usando a `throw` palavra-chave no final do `catch` bloco.</span><span class="sxs-lookup"><span data-stu-id="9f3bd-122">If you catch `System.Exception`, re-throw it using the `throw` keyword at the end of the `catch` block.</span></span>
- <span data-ttu-id="9f3bd-123">Se um bloco `catch` define uma variável de exceção, você pode usá-lo para obter mais informações sobre o tipo de exceção que ocorreu.</span><span class="sxs-lookup"><span data-stu-id="9f3bd-123">If a `catch` block defines an exception variable, you can use it to obtain more information about the type of exception that occurred.</span></span>
- <span data-ttu-id="9f3bd-124">As exceções podem ser geradas explicitamente por um programa usando a palavra-chave `throw`.</span><span class="sxs-lookup"><span data-stu-id="9f3bd-124">Exceptions can be explicitly generated by a program by using the `throw` keyword.</span></span>
- <span data-ttu-id="9f3bd-125">Os objetos de exceção contêm informações detalhadas sobre o erro, como o estado da pilha de chamadas e uma descrição de texto do erro.</span><span class="sxs-lookup"><span data-stu-id="9f3bd-125">Exception objects contain detailed information about the error, such as the state of the call stack and a text description of the error.</span></span>
- <span data-ttu-id="9f3bd-126">O código em um bloco `finally` será executado mesmo se uma exceção for lançada.</span><span class="sxs-lookup"><span data-stu-id="9f3bd-126">Code in a `finally` block is executed even if an exception is thrown.</span></span> <span data-ttu-id="9f3bd-127">Use um bloco `finally` para liberar recursos, por exemplo, para fechar todos os fluxos ou arquivos que foram abertos no bloco `try`.</span><span class="sxs-lookup"><span data-stu-id="9f3bd-127">Use a `finally` block to release resources, for example to close any streams or files that were opened in the `try` block.</span></span>
- <span data-ttu-id="9f3bd-128">As exceções gerenciadas no .NET são implementadas sobre o mecanismo de manipulação de exceção estruturada do Win32.</span><span class="sxs-lookup"><span data-stu-id="9f3bd-128">Managed exceptions in .NET are implemented on top of the Win32 structured exception handling mechanism.</span></span> <span data-ttu-id="9f3bd-129">Para obter mais informações, consulte [Manipulação de exceções estruturadas (C/C++)](/cpp/cpp/structured-exception-handling-c-cpp) e [Curso rápido sobre a manipulação de exceções estruturadas do Win32](http://bytepointer.com/resources/pietrek_crash_course_depths_of_win32_seh.htm).</span><span class="sxs-lookup"><span data-stu-id="9f3bd-129">For more information, see [Structured Exception Handling (C/C++)](/cpp/cpp/structured-exception-handling-c-cpp) and [A Crash Course on the Depths of Win32 Structured Exception Handling](http://bytepointer.com/resources/pietrek_crash_course_depths_of_win32_seh.htm).</span></span>
