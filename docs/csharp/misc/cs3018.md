# <a name="compiler-warning-level-1-cs3018"></a>Compilador CS3018 de aviso (nível 1)
'type' não pode ser marcado como compatível com CLS porque é um membro de tipo não compatível com CLS 'type'  
  
 Este aviso ocorre se uma classe aninhada com o atributo CLSCompliant para `true` é declarado como um membro de uma classe declarada com o atributo CLSCompliant para `false`. Isso não é permitido, como uma classe aninhada não pode ser compatível com CLS se ele é um membro de uma classe externa que não é compatível com CLS. Para resolver este aviso, remova o atributo CLSCompliant da classe aninhada ou alterá-lo no `true` para `false`. Para obter mais informações sobre a Conformidade com CLS, consulte [Gravando Código em Conformidade com CLS](http://msdn.microsoft.com/en-us/4c705105-69a2-4e5e-b24e-0633bc32c7f3) e [Componentes de Independência de Linguagem e Componentes Independentes da Linguagem](https://msdn.microsoft.com/library/12a7a7h3).  
  
## <a name="example"></a>Exemplo  
 O exemplo a seguir gera CS3018.  
  
```  
// CS3018.cs  
// compile with: /target:library  
using System;  
  
[assembly: CLSCompliant(true)]  
[CLSCompliant(false)]  
public class Outer  
{  
   [CLSCompliant(true)]   // CS3018  
   public class Nested {}  
  
   // OK  
   public class Nested2 {}  
  
   [CLSCompliant(false)]  
   public class Nested3 {}  
}  
```
