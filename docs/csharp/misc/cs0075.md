# <a name="compiler-error-cs0075"></a><span data-ttu-id="09c80-103">Erro do Compilador CS0075</span><span class="sxs-lookup"><span data-stu-id="09c80-103">Compiler Error CS0075</span></span>

<span data-ttu-id="09c80-104">Para converter um valor negativo, você deve colocar o valor entre parênteses</span><span class="sxs-lookup"><span data-stu-id="09c80-104">To cast a negative value, you must enclose the value in parentheses</span></span>  
  
 <span data-ttu-id="09c80-105">Se você estiver convertendo usando uma palavra-chave que identifica um tipo predefinido, não precisará de parênteses.</span><span class="sxs-lookup"><span data-stu-id="09c80-105">If you are casting using a keyword that identifies a predefined type, then you do not need parentheses.</span></span> <span data-ttu-id="09c80-106">Caso contrário, você deve colocar os parênteses porque (x) – y não será considerado uma expressão de conversão.</span><span class="sxs-lookup"><span data-stu-id="09c80-106">Otherwise, you must put the parentheses because (x) –y will not be considered a cast expression.</span></span> <span data-ttu-id="09c80-107">Na especificação do C#, a seção 7.6.6:</span><span class="sxs-lookup"><span data-stu-id="09c80-107">From the C# Specification, Section 7.6.6:</span></span>  
  
 <span data-ttu-id="09c80-108">*A partir da regra de desambiguidade, ela segue que, se x e y são identificadores, (x) y, (x) (y) e (x) (-y) são expressões Cast, mas (x)-y não é, mesmo que x identifique um tipo. No entanto, se x for uma palavra-chave que identifica um tipo predefinido (como int), todas as quatro formas serão expressões Cast (porque essa palavra-chave não poderia ser uma expressão por si só).*</span><span class="sxs-lookup"><span data-stu-id="09c80-108">*From the disambiguation rule it follows that, if x and y are identifiers, (x)y, (x)(y), and (x)(-y) are cast-expressions, but (x)-y is not, even if x identifies a type. However, if x is a keyword that identifies a predefined type (such as int), then all four forms are cast-expressions (because such a keyword could not possibly be an expression by itself).*</span></span>  
  
 <span data-ttu-id="09c80-109">O código a seguir gera CS0075:</span><span class="sxs-lookup"><span data-stu-id="09c80-109">The following code generates CS0075:</span></span>  
  
```csharp  
// CS0075  
namespace MyNamespace  
{  
    enum MyEnum { }  
    public class MyClass  
    {  
        public static void Main()  
        {  
            // To fix the error, place the negative  
            // values below in parentheses  
            int i = (System.Int32) - 4; //CS0075  
            MyEnum e = (MyEnum) - 1;    //CS0075  
            System.Console.WriteLine(i); //to avoid warning  
            System.Console.WriteLine(e); //to avoid warning  
        }  
    }  
}  
```
