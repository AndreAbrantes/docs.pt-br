# <a name="compiler-error-cs0075"></a><span data-ttu-id="902b5-102">Erro do compilador CS0075</span><span class="sxs-lookup"><span data-stu-id="902b5-102">Compiler Error CS0075</span></span>
<span data-ttu-id="902b5-103">Para converter um valor negativo, você deve colocar o valor entre parênteses</span><span class="sxs-lookup"><span data-stu-id="902b5-103">To cast a negative value, you must enclose the value in parentheses</span></span>  
  
 <span data-ttu-id="902b5-104">Se você está convertendo usando uma palavra-chave que identifica um tipo predefinido, você não precisa parênteses.</span><span class="sxs-lookup"><span data-stu-id="902b5-104">If you are casting using a keyword that identifies a predefined type, then you do not need parentheses.</span></span> <span data-ttu-id="902b5-105">Caso contrário, você deve colocar os parênteses porque (x) – y não será considerada uma expressão de conversão.</span><span class="sxs-lookup"><span data-stu-id="902b5-105">Otherwise, you must put the parentheses because (x) –y will not be considered a cast expression.</span></span> <span data-ttu-id="902b5-106">Do C# especificação, seção 7.6.6:</span><span class="sxs-lookup"><span data-stu-id="902b5-106">From the C# Specification, Section 7.6.6:</span></span>  
  
 <span data-ttu-id="902b5-107">*Da regra de Desambiguidade segue que, se x e y são identificadores, (y, x) (x)(y), e (x)(-y) são expressões de conversão, mas (x)-y é não, mesmo se x identifica um tipo. No entanto, se x for uma palavra-chave que identifica um tipo predefinido (como int), em seguida, todos os quatro formas são expressões de conversão (porque tal uma palavra-chave não pode ser uma expressão, possivelmente, por si só).*</span><span class="sxs-lookup"><span data-stu-id="902b5-107">*From the disambiguation rule it follows that, if x and y are identifiers, (x)y, (x)(y), and (x)(-y) are cast-expressions, but (x)-y is not, even if x identifies a type. However, if x is a keyword that identifies a predefined type (such as int), then all four forms are cast-expressions (because such a keyword could not possibly be an expression by itself).*</span></span>  
  
 <span data-ttu-id="902b5-108">O código a seguir gera CS0075:</span><span class="sxs-lookup"><span data-stu-id="902b5-108">The following code generates CS0075:</span></span>  
  
```csharp  
// CS0075  
namespace MyNamespace  
{  
    enum MyEnum { }  
    public class MyClass  
    {  
        public static void Main()  
        {  
            // To fix the error, place the negative  
            // values below in parentheses  
            int i = (System.Int32) - 4; //CS0075  
            MyEnum e = (MyEnum) - 1;    //CS0075  
            System.Console.WriteLine(i); //to avoid warning  
            System.Console.WriteLine(e); //to avoid warning  
        }  
    }  
}  
```
