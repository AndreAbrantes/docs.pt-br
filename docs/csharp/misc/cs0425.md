# <a name="compiler-error-cs0425"></a>CS0425 de erro do compilador
As restrições de parâmetro de tipo 'parâmetro de tipo' do método 'method' devem corresponder às restrições para parâmetro 'tipo parâmetro de tipo' do método de interface 'method'. Considere usar uma implementação de interface explícita.  
  
 Esse erro ocorre se um método genérico virtual é substituído em uma classe derivada, e as restrições sobre o método na classe derivada não coincidem as restrições sobre o método na classe base. Para evitar esse erro, verifique se o `where` cláusula é idêntica em declarações ou implementar a interface explicitamente.  
  
## <a name="example"></a>Exemplo  
 O exemplo a seguir gera CS0425:  
  
```csharp  
// CS0425.cs  
  
class C1  
{  
}  
  
class C2  
{  
}  
  
interface IBase  
{  
    void F<ItemType>(ItemType item) where ItemType : C1;  
}  
  
class Derived : IBase  
{  
    public void F<ItemType>(ItemType item) where ItemType : C2  // CS0425  
    {  
    }  
}  
  
class CMain  
{  
    public static void Main()  
    {  
    }  
}  
```  
  
## <a name="example"></a>Exemplo  
 As restrições não têm uma correspondência literal, como o conjunto de restrições tem o mesmo significado. Por exemplo, o seguinte é okey:  
  
```csharp  
// CS0425b.cs  
  
interface J<Z>  
{  
}  
  
interface I<S>  
{  
    void F<T>(S s, T t) where T: J<S>, J<int>;  
}  
  
class C : I<int>  
{  
    public void F<X>(int s, X x) where X : J<int>  
    {  
    }  
  
    public static void Main()  
    {  
    }  
}  
```
