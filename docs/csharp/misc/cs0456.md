# <a name="compiler-error-cs0456"></a><span data-ttu-id="8d41b-103">Erro do Compilador CS0456</span><span class="sxs-lookup"><span data-stu-id="8d41b-103">Compiler Error CS0456</span></span>

<span data-ttu-id="8d41b-104">O parâmetro de tipo ' nome do parâmetro de tipo 1 ' tem a restrição ' struct ' e, portanto, ' nome do parâmetro de tipo 1 ' não pode ser usado como uma restrição para ' nome do parâmetro de tipo 2 '</span><span class="sxs-lookup"><span data-stu-id="8d41b-104">Type parameter 'Type Parameter Name 1' has the 'struct' constraint so 'Type Parameter Name 1' cannot be used as a constraint for 'Type Parameter Name 2'</span></span>  
  
 <span data-ttu-id="8d41b-105">As restrições de tipo de valor são seladas implicitamente, portanto, essas restrições não podem ser usadas como restrições em um parâmetro de segundo tipo.</span><span class="sxs-lookup"><span data-stu-id="8d41b-105">Value type constraints are implicitly sealed so those constraints cannot be used as constraints on a second type parameter.</span></span> <span data-ttu-id="8d41b-106">Isso ocorre porque os tipos de valor não podem ser substituídos.</span><span class="sxs-lookup"><span data-stu-id="8d41b-106">This is because value types cannot be overridden.</span></span> <span data-ttu-id="8d41b-107">Para resolver esse erro, coloque uma restrição de tipo de valor diretamente no segundo parâmetro de tipo, em vez de fazer isso indiretamente por meio do primeiro parâmetro de tipo.</span><span class="sxs-lookup"><span data-stu-id="8d41b-107">To resolve this error, put a value type constraint directly on the second type parameter, instead of doing so indirectly by means of the first type parameter.</span></span>

 <span data-ttu-id="8d41b-109">O exemplo a seguir gera CS0456.</span><span class="sxs-lookup"><span data-stu-id="8d41b-109">The following sample generates CS0456.</span></span>  
  
```csharp  
// CS0456.cs

{  
   public class G5<T> where T : struct

   }

```
