# <a name="compiler-error-cs0457"></a><span data-ttu-id="fb16c-102">CS0457 de erro do compilador</span><span class="sxs-lookup"><span data-stu-id="fb16c-102">Compiler Error CS0457</span></span>
<span data-ttu-id="fb16c-103">Um usuário ambíguo definiu conversões 'nome de método de conversão 1' e 'nome de método de conversão 2' ao converter de ' nome de tipo 1' para ' nome de tipo 2'</span><span class="sxs-lookup"><span data-stu-id="fb16c-103">Ambiguous user defined conversions 'Conversion method name 1' and 'Conversion method name 2' when converting from 'type name 1' to 'type name 2'</span></span>  
  
 <span data-ttu-id="fb16c-104">Dois métodos de conversão são aplicáveis, e o compilador não pode decidir qual delas usar.</span><span class="sxs-lookup"><span data-stu-id="fb16c-104">Two conversion methods are applicable, and the compiler is unable to decide which one to use.</span></span>  
  
 <span data-ttu-id="fb16c-105">Um cenário que pode causar esse erro é o seguinte:</span><span class="sxs-lookup"><span data-stu-id="fb16c-105">One scenario that can cause this error is as follows:</span></span>  
  
-   <span data-ttu-id="fb16c-106">Você deseja converter a classe A, classe B onde A e B estão relacionados.</span><span class="sxs-lookup"><span data-stu-id="fb16c-106">You want to convert class A to class B where A and B are unrelated.</span></span>  
  
-   <span data-ttu-id="fb16c-107">Um é derivado do A0, e há um método que converte de A0 em B.</span><span class="sxs-lookup"><span data-stu-id="fb16c-107">A is derived from A0, and there is a method that converts from A0 to B.</span></span>  
  
-   <span data-ttu-id="fb16c-108">B tem uma subclasse B1 e há um método que converte de um para B1.</span><span class="sxs-lookup"><span data-stu-id="fb16c-108">B has a subclass B1 and there is a method that converts from A to B1.</span></span>  
  
 <span data-ttu-id="fb16c-109">O compilador será ponderar os dois métodos de conversão igualmente, porque a primeira conversão fornece o melhor tipo de destino, e a segunda conversão fornece o melhor tipo de origem.</span><span class="sxs-lookup"><span data-stu-id="fb16c-109">The compiler will weight the two conversion methods equally, because the first conversion provides the best destination type, and the second conversion provides the best source type.</span></span> <span data-ttu-id="fb16c-110">Desde que o compilador não poderão escolher, esse erro será gerado.</span><span class="sxs-lookup"><span data-stu-id="fb16c-110">Since the compiler will be unable to choose, this error will be generated.</span></span> <span data-ttu-id="fb16c-111">Para resolver, gravar um novo método explícito convertendo A para B.</span><span class="sxs-lookup"><span data-stu-id="fb16c-111">To resolve, write a new explicit method converting A to B.</span></span>  
  
 <span data-ttu-id="fb16c-112">Outro cenário que causa esse erro é se há dois métodos que convertem A para B. Para corrigir, especifique qual conversão a ser usado por meio de uma conversão explícita.</span><span class="sxs-lookup"><span data-stu-id="fb16c-112">Another scenario that causes this error is if there are two methods that convert A to B. ## <a name="example"></a><span data-ttu-id="fb16c-113">Exemplo</span><span class="sxs-lookup"><span data-stu-id="fb16c-113">Example</span></span>  
 <span data-ttu-id="fb16c-114">O exemplo a seguir gera CS0457.</span><span class="sxs-lookup"><span data-stu-id="fb16c-114">The following sample generates CS0457.</span></span>  
  
```csharp  
// CS0457.cs  
using System;  
public class A { }  
  
public class G0 {  }  
public class G1<R> : G0 {  }  
  
public class H0 {  
   public static implicit operator G0(H0 h) {  
      return new G0();  
   }  
}  
public class H1<R> : H0 {  
   public static implicit operator G1<R>(H1<R> h) {  
      return new G1<R>();  
   }  
}  
  
public class Test   
{  
   public static void F0(G0 g) {  }  
   public static void Main()   
   {  
      H1<A> h1a = new H1<A>();  
      F0(h1a);   // CS0457  
   }  
}  
```
