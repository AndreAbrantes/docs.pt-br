# <a name="compiler-error-cs5001"></a>Erro do compilador CS5001

O programa ' Program ' não contém um método ' Main ' estático adequado para um ponto de entrada  
  
Esse erro ocorre quando nenhum método `Main` estático com uma assinatura correta é encontrado no código que produz um arquivo executável. Esse erro também ocorrerá se a função de ponto `Main`de entrada,, for definida com o caso incorreto, como `main`em letras minúsculas. Para obter informações sobre as regras que se aplicam ao `Main` método, consulte [Main () e argumentos de linha de comando](../programming-guide/main-and-command-args/index.md).

Se o `Main` método tiver um `async` modificador, certifique-se de que a [versão de idioma selecionada C# ](../language-reference/configure-language-version.md) seja 7,1 ou superior.

O `Main` método só é necessário ao compilar um arquivo executável, ou seja, quando [-target: exe](/dotnet/csharp/language-reference/compiler-options/target-exe-compiler-option) ou [-target: a](/dotnet/csharp/language-reference/compiler-options/target-winexe-compiler-option) opção do compilador winexe é especificada. Os seguintes tipos de projeto do Visual Studio especificam uma dessas opções por padrão:

- Aplicativo de console
- ASP.NET Core aplicativo
- aplicativo WPF
- aplicativo Windows Forms

## <a name="example"></a>Exemplo  
 O exemplo a seguir gera CS5001:  
  
```csharp  
// CS5001.cs  
// CS5001 expected when compiled with -target:exe or -target:winexe 
public class a  
{  
   // Uncomment the following line to resolve.  
   // static void Main() {}  
}  
```
