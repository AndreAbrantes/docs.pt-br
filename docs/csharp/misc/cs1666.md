 <span data-ttu-id="2141e-105">Esse erro ocorre se você usar o buffer de tamanho fixo em uma expressão que envolve uma classe que não fixa.</span><span class="sxs-lookup"><span data-stu-id="2141e-105">This error occurs if you use the fixed sized buffer in an expression involving a class that is not itself fixed.</span></span> <span data-ttu-id="2141e-106">O tempo de execução está livre para mover a classe unfixed para otimizar o acesso de memória, o que poderá resultar em erros ao usar fixos em tamanho de buffer.</span><span class="sxs-lookup"><span data-stu-id="2141e-106">The runtime is free to move the unfixed class around to optimize memory access, which could lead to errors when using the fixed sized buffer.</span></span> <span data-ttu-id="2141e-107">Para evitar esse erro, use o `fixed` palavra-chave na instrução.</span><span class="sxs-lookup"><span data-stu-id="2141e-107">To avoid this error, use the `fixed` keyword on the statement.</span></span>  
  
## <a name="example"></a><span data-ttu-id="2141e-108">Exemplo</span><span class="sxs-lookup"><span data-stu-id="2141e-108">Example</span></span>  
 <span data-ttu-id="2141e-109">O exemplo a seguir gera CS1666.</span><span class="sxs-lookup"><span data-stu-id="2141e-109">The following sample generates CS1666.</span></span>  
  
```csharp  
// CS1666.cs  
// compile with: /unsafe /target:library  
unsafe struct S  
{  
   public fixed int buffer[1];  
}  
  
unsafe class Test  
{  
   S field = new S();  
  
   private bool example1()  
   {  
      return (field.buffer[0] == 0);   // CS1666 error  
   }  
  
   private bool example2()  
   {  
      // OK  
      fixed (S* p = &field)  
      {  
         return (p->buffer[0] == 0);  
      }  
   }  
  
   private bool example3()  
   {  
      S local = new S();  
      return (local.buffer[0] == 0);   
   }   
}  
```
