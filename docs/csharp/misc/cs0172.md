# <a name="compiler-error-cs0172"></a><span data-ttu-id="35d54-102">Erro do compilador CS0172</span><span class="sxs-lookup"><span data-stu-id="35d54-102">Compiler Error CS0172</span></span>
<span data-ttu-id="35d54-103">Tipo de expressão condicional não pode ser determinado porque 'type1' e 'type2' se convertem implicitamente um ao outro</span><span class="sxs-lookup"><span data-stu-id="35d54-103">Type of conditional expression cannot be determined because 'type1' and 'type2' implicitly convert to one another</span></span>  
  
<span data-ttu-id="35d54-104">Em uma instrução condicional, você deve ser capaz de converter implicitamente os tipos nos dois lados do `:` token.</span><span class="sxs-lookup"><span data-stu-id="35d54-104">In a conditional statement, you must be able to implicitly convert the types on either side of the `:` token.</span></span> <span data-ttu-id="35d54-105">Além disso, não é possível haver mútuas conversões implícitas; Você só precisa de uma conversão.</span><span class="sxs-lookup"><span data-stu-id="35d54-105">Also, there cannot be mutual implicit conversions; you only need one conversion.</span></span>
  
<span data-ttu-id="35d54-106">O exemplo a seguir gera CS0172:</span><span class="sxs-lookup"><span data-stu-id="35d54-106">The following sample generates CS0172:</span></span>
  
```csharp  
// CS0172.cs  
public class Square  
{  
   public class Circle  
   {  
      public static implicit operator Circle(Square aa)  
      {  
         return null;  
      }  
  
      public static implicit operator Square(Circle aa)  
      // using explicit resolves this error  
      // public static explicit operator Square(Circle aa)  
      {  
         return null;  
      }  
   }  
  
   public static void Main()  
   {  
      Circle aa = new Circle();  
      Square ii = new Square();  
      object o = (1 == 1) ? aa : ii;   // CS0172  
      // the following cast would resolve this error  
      // (1 == 1) ? aa : (Circle)ii;  
   }  
}  
```
