## <a name="rules-for-command-line-syntax-for-the-c-compiler"></a><span data-ttu-id="2ccb9-124">Regras para sintaxe de linha de comando para o compilador do C#</span><span class="sxs-lookup"><span data-stu-id="2ccb9-124">Rules for command-line syntax for the C# compiler</span></span>

<span data-ttu-id="2ccb9-125">O compilador do C# usa as seguintes regras para interpretar os argumentos fornecidos na linha de comando do sistema operacional:</span><span class="sxs-lookup"><span data-stu-id="2ccb9-125">The C# compiler uses the following rules when it interprets arguments given on the operating system command line:</span></span>

- <span data-ttu-id="2ccb9-126">Os argumentos são delimitados por espaço em branco, que é um espaço ou uma tabulação.</span><span class="sxs-lookup"><span data-stu-id="2ccb9-126">Arguments are delimited by white space, which is either a space or a tab.</span></span>

- <span data-ttu-id="2ccb9-127">O caractere de acento circunflexo (^) não é reconhecido como um caractere de escape ou um delimitador.</span><span class="sxs-lookup"><span data-stu-id="2ccb9-127">The caret character (^) is not recognized as an escape character or delimiter.</span></span> <span data-ttu-id="2ccb9-128">O caractere é tratado pelo analisador de linha de comando no sistema operacional antes de ser passado para a matriz `argv` no programa.</span><span class="sxs-lookup"><span data-stu-id="2ccb9-128">The character is handled by the command-line parser in the operating system before it's passed to the `argv` array in the program.</span></span>

- <span data-ttu-id="2ccb9-129">Uma cadeia de caracteres entre aspas duplas ("cadeia de caracteres") é interpretada como um único argumento, independentemente do espaço em branco que está contido nela.</span><span class="sxs-lookup"><span data-stu-id="2ccb9-129">A string enclosed in double quotation marks ("string") is interpreted as a single argument, regardless of white space that is contained within.</span></span> <span data-ttu-id="2ccb9-130">Uma cadeia de caracteres entre aspas pode ser inserida em um argumento.</span><span class="sxs-lookup"><span data-stu-id="2ccb9-130">A quoted string can be embedded in an argument.</span></span>

- <span data-ttu-id="2ccb9-131">Aspas duplas precedidas por uma barra invertida (\\") são interpretados como um caractere literal de aspas duplas (").</span><span class="sxs-lookup"><span data-stu-id="2ccb9-131">A double quotation mark preceded by a backslash (\\") is interpreted as a literal double quotation mark character (").</span></span>

- <span data-ttu-id="2ccb9-132">As barras invertidas são interpretadas literalmente, a menos que precedam imediatamente as aspas duplas.</span><span class="sxs-lookup"><span data-stu-id="2ccb9-132">Backslashes are interpreted literally, unless they immediately precede a double quotation mark.</span></span>

- <span data-ttu-id="2ccb9-133">Se um número par de barras invertidas for seguido por aspas duplas, uma barra invertida será colocada na matriz `argv` para cada par de barras invertidas e aspas duplas serão interpretadas como um delimitador de cadeia de caracteres.</span><span class="sxs-lookup"><span data-stu-id="2ccb9-133">If an even number of backslashes is followed by a double quotation mark, one backslash is put in the `argv` array for every pair of backslashes, and the double quotation mark is interpreted as a string delimiter.</span></span>

- <span data-ttu-id="2ccb9-134">Se um número ímpar de barras invertidas for seguido por aspas duplas, uma barra invertida será colocada na matriz `argv` para cada par de barras invertidas e aspas duplas serão "escapadas" pela barra invertida restante.</span><span class="sxs-lookup"><span data-stu-id="2ccb9-134">If an odd number of backslashes is followed by a double quotation mark, one backslash is put in the `argv` array for every pair of backslashes, and the double quotation mark is "escaped" by the remaining backslash.</span></span> <span data-ttu-id="2ccb9-135">Isso fará com que aspas duplas (") literais sejam adicionadas na `argv`.</span><span class="sxs-lookup"><span data-stu-id="2ccb9-135">This causes a literal double quotation mark (") to be added in `argv`.</span></span>
