 <span data-ttu-id="e1e25-106">`+` &#124; `-`</span><span class="sxs-lookup"><span data-stu-id="e1e25-106">`+` &#124; `-`</span></span>  
 <span data-ttu-id="e1e25-107">Especificar `+` ou apenas **-debug** faz o compilador gerar informações de depuração e colocá-las em um de banco de dados de programa (arquivo .pdb).</span><span class="sxs-lookup"><span data-stu-id="e1e25-107">Specifying `+`, or just **-debug**, causes the compiler to generate debugging information and place it in a program database (.pdb file).</span></span> <span data-ttu-id="e1e25-108">Especificar `-`, que será aplicado se **-debug** não for especificado, faz com que nenhuma informação de depuração seja criada.</span><span class="sxs-lookup"><span data-stu-id="e1e25-108">Specifying `-`, which is in effect if you do not specify **-debug**, causes no debug information to be created.</span></span>

 <span data-ttu-id="e1e25-110">Especifica o tipo de informações de depuração geradas pelo compilador.</span><span class="sxs-lookup"><span data-stu-id="e1e25-110">Specifies the type of debugging information generated by the compiler.</span></span> <span data-ttu-id="e1e25-111">O argumento completo, que será aplicado se **-debug:pdbonly** não for especificado, permite anexar um depurador ao programa em execução.</span><span class="sxs-lookup"><span data-stu-id="e1e25-111">The full argument, which is in effect if you do not specify **-debug:pdbonly**, enables attaching a debugger to the running program.</span></span> <span data-ttu-id="e1e25-112">Especificar pdbonly habilita a depuração do código-fonte quando o programa é iniciado no depurador, mas exibirá somente o assembler quando o programa em execução for anexado ao depurador.</span><span class="sxs-lookup"><span data-stu-id="e1e25-112">Specifying pdbonly allows source code debugging when the program is started in the debugger but will only display assembler when the running program is attached to the debugger.</span></span>
