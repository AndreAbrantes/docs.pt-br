## <a name="example"></a><span data-ttu-id="67d22-120">Exemplo</span><span class="sxs-lookup"><span data-stu-id="67d22-120">Example</span></span>  

 <span data-ttu-id="67d22-121">O código a seguir faz referência às duas versões de DLL que são definidas no código anterior.</span><span class="sxs-lookup"><span data-stu-id="67d22-121">The following code references the two DLL versions that are defined in the preceding code.</span></span> <span data-ttu-id="67d22-122">`AssemblyA` refere-se à DLL criada pelo CS1705a.cs (versão 1.0).</span><span class="sxs-lookup"><span data-stu-id="67d22-122">`AssemblyA` refers to the DLL created by CS1705a.cs (version 1.0).</span></span> <span data-ttu-id="67d22-123">`AssemblyB` refere-se à DLL criada pelo CS1705b.cs (versão 2.0).</span><span class="sxs-lookup"><span data-stu-id="67d22-123">`AssemblyB` refers to the DLL created by CS1705b.cs (version 2.0).</span></span> <span data-ttu-id="67d22-124">Na `ClassC`, dois métodos são definidos.</span><span class="sxs-lookup"><span data-stu-id="67d22-124">In `ClassC`, two methods are defined.</span></span> <span data-ttu-id="67d22-125">O primeiro, `Return1A`, retorna um objeto do tipo `Class1A`, que é um alias para a `Class1` da versão 1.0 da DLL.</span><span class="sxs-lookup"><span data-stu-id="67d22-125">The first, `Return1A`, returns an object of type `Class1A`, which is an alias for `Class1` from version 1.0 of the DLL.</span></span> <span data-ttu-id="67d22-126">O segundo, `Return1B`, retorna um objeto do tipo `Class1B`, que é um alias para a `Class1` da versão 2.0 da DLL.</span><span class="sxs-lookup"><span data-stu-id="67d22-126">The second, `Return1B`, returns an object of type `Class1B`, which is an alias for `Class1` from version 2.0 of the DLL.</span></span> <span data-ttu-id="67d22-127">A definição de `Return1A` cria uma dependência na versão 1.0. A definição de `Return1B` cria uma dependência na versão 2.0.</span><span class="sxs-lookup"><span data-stu-id="67d22-127">The definition of `Return1A` creates a dependency on version 1.0; the definition of `Return1B` creates a dependency on version 2.0.</span></span>  
  
```csharp  
// CS1705c.cs  
  
// Compile by using the following command. AssemblyA refers to the DLL created by  
// CS1705a.cs (version 1.0). AssemblyB refers to the DLL created by CS1705b.cs  
// (version 2.0).  
//    csc /target:library /r:AssemblyA=C:\\CS1705.dll /r:AssemblyB=CS1705.dll CS1705c.cs  
  
extern alias AssemblyA;  
extern alias AssemblyB;  
  
// Class1A is an alias for type Class1 from VS1705a.cs, which is in version 1.0
// of the assembly. Class1B is an alias for type Class1 from CS1705b.cs, which  
// is in version 2.0 of the assembly.  
  
using Class1A = AssemblyA::Class1;  
using Class1B = AssemblyB::Class1;  
  
// Method Return1A in ClassC returns an object of type Class1A, which is  
// Class1 from version 1.0 of the DLL. Method Return1B returns an object  
// of type Class1B, which is Class1 from version 2.0 of the DLL.  
  
public class ClassC  
{  
    // The following line creates a dependency on version 1.0 of CS1705.dll.  
    // This is not the source of the problem when ClassC is accessed from  
    // CS1705d.cs because CS1705d.cs references version 1.0 of the DLL.
    // Therefore, type Class1A and the assembly have the same version.  
    public static Class1A Return1A() { return new Class1A(); }  
  
    // The following line creates a dependency on version 2.0 of CS1705.dll.  
    // This causes compiler error CS1705 when ClassC is accessed from
    // CS1705d.cs, because CS1705d.cs does not reference version 2.0 of the
    // DLL. Class1B is the alias for Class1 in version 2.0, and CS1705d.cs
    // references version 1.0.  
    public static Class1B Return1B() { return new Class1B(); }  
}  
```
