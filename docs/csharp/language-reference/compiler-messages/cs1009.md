# <a name="compiler-error-cs1009"></a><span data-ttu-id="b07c0-103">Erro do Compilador CS1009</span><span class="sxs-lookup"><span data-stu-id="b07c0-103">Compiler Error CS1009</span></span>
<span data-ttu-id="b07c0-104">Sequência de escape não reconhecida</span><span class="sxs-lookup"><span data-stu-id="b07c0-104">Unrecognized escape sequence</span></span>  
  
 <span data-ttu-id="b07c0-105">Um caractere inesperado após uma barra invertida (\\) em uma [cadeia de caracteres](../builtin-types/reference-types.md#the-string-type).</span><span class="sxs-lookup"><span data-stu-id="b07c0-105">An unexpected character follows a backslash (\\) in a [string](../builtin-types/reference-types.md#the-string-type).</span></span> <span data-ttu-id="b07c0-106">O compilador espera um dos caracteres de escape válidos.</span><span class="sxs-lookup"><span data-stu-id="b07c0-106">The compiler expects one of the valid escape characters.</span></span> <span data-ttu-id="b07c0-107">Para obter mais informações, consulte [Escapes de Caracteres](../../../standard/base-types/character-escapes-in-regular-expressions.md).</span><span class="sxs-lookup"><span data-stu-id="b07c0-107">For more information, see [Character Escapes](../../../standard/base-types/character-escapes-in-regular-expressions.md).</span></span>  
  
 <span data-ttu-id="b07c0-108">O exemplo a seguir gera o erro CS1009.</span><span class="sxs-lookup"><span data-stu-id="b07c0-108">The following sample generates CS1009.</span></span>  
  
```csharp  
// CS1009-a.cs  
class MyClass  
{  
   static void Main()  
   {  
      // The following line causes CS1009.  
      string a = "\m";
      // Try the following line instead.  
      // string a = "\t";  
   }  
}  
```  
  
 <span data-ttu-id="b07c0-109">Uma causa comum desse erro é usar o caractere de barra invertida em um nome de arquivo, conforme mostrado no exemplo a seguir.</span><span class="sxs-lookup"><span data-stu-id="b07c0-109">A common cause of this error is using the backslash character in a file name, as the following example shows.</span></span>  
  
```csharp  
string filename = "c:\myFolder\myFile.txt";  
```  
  
 <span data-ttu-id="b07c0-110">Para resolver esse erro, use "\\\\" ou o literal @-quoted da cadeia de caracteres, conforme mostrado no exemplo a seguir.</span><span class="sxs-lookup"><span data-stu-id="b07c0-110">To resolve this error, use "\\\\" or the @-quoted string literal, as the following example shows.</span></span>  
  
```csharp  
// CS1009-b.cs  
class MyClass  
{  
   static void Main()  
   {  
      // The following line causes CS1009.  
      string filename = "c:\myFolder\myFile.txt";
      // Try one of the following lines instead.  
      // string filename = "c:\\myFolder\\myFile.txt";  
      // string filename = @"c:\myFolder\myFile.txt";  
   }  
}  
```
