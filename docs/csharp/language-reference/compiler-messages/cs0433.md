<span data-ttu-id="ef45a-103">O tipo TypeName1 existe no TypeName2 e no TypeName3</span><span class="sxs-lookup"><span data-stu-id="ef45a-103">The type TypeName1 exists in both TypeName2 and TypeName3</span></span>  
  
 <span data-ttu-id="ef45a-104">Dois assemblies diferentes referenciados no aplicativo contêm o mesmo namespace e tipo, o que gera ambiguidade.</span><span class="sxs-lookup"><span data-stu-id="ef45a-104">Two different assemblies referenced in your application contain the same namespace and type, which produces ambiguity.</span></span>  
  
 <span data-ttu-id="ef45a-105">Para resolver esse erro, use o recurso de alias da opção do compilador [/reference (Opções do Compilador C#)](../../../csharp/language-reference/compiler-options/reference-compiler-option.md) ou não referencie um dos assemblies.</span><span class="sxs-lookup"><span data-stu-id="ef45a-105">To resolve this error, use the alias feature of the [/reference (C# Compiler Options)](../../../csharp/language-reference/compiler-options/reference-compiler-option.md) compiler option or do not reference one of your assemblies.</span></span>

## <a name="example"></a><span data-ttu-id="ef45a-110">Exemplo</span><span class="sxs-lookup"><span data-stu-id="ef45a-110">Example</span></span>  
 <span data-ttu-id="ef45a-111">O exemplo a seguir gera o erro CS0433.</span><span class="sxs-lookup"><span data-stu-id="ef45a-111">The following example generates CS0433.</span></span>  
  
```csharp  
// CS0433_3.cs

using TypeBindConflicts;  
public class Test   
{  
   public static void Main()   
   {  
      AggPubImpAggPubImp n6 = new AggPubImpAggPubImp();   // CS0433  
   }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="ef45a-112">Exemplo</span><span class="sxs-lookup"><span data-stu-id="ef45a-112">Example</span></span>
