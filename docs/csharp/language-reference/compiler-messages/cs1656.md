# <a name="compiler-error-cs1656"></a><span data-ttu-id="040c6-102">Erro do Compilador CS1656</span><span class="sxs-lookup"><span data-stu-id="040c6-102">Compiler Error CS1656</span></span>

 <span data-ttu-id="040c6-104">Esse erro ocorre quando uma atribuição de variável ocorre em um contexto de somente leitura.</span><span class="sxs-lookup"><span data-stu-id="040c6-104">This error occurs when an assignment to variable occurs in a read-only context.</span></span> <span data-ttu-id="040c6-105">Contextos de somente leitura incluem variáveis de iteração [foreach](../../../csharp/language-reference/keywords/foreach-in.md), variáveis [using](../../../csharp/language-reference/keywords/using-statement.md) e variáveis [fixed](../../../csharp/language-reference/keywords/fixed-statement.md).</span><span class="sxs-lookup"><span data-stu-id="040c6-105">Read-only contexts include [foreach](../../../csharp/language-reference/keywords/foreach-in.md) iteration variables, [using](../../../csharp/language-reference/keywords/using-statement.md) variables, and [fixed](../../../csharp/language-reference/keywords/fixed-statement.md) variables.</span></span> <span data-ttu-id="040c6-106">Para resolver esse erro, evite fazer atribuições a uma variável de instrução em blocos `using`, instruções `foreach` e instruções `fixed`.</span><span class="sxs-lookup"><span data-stu-id="040c6-106">To resolve this error, avoid assignments to a statement variable in `using` blocks, `foreach` statements, and `fixed` statements.</span></span>  
  
## <a name="example"></a><span data-ttu-id="040c6-107">Exemplo</span><span class="sxs-lookup"><span data-stu-id="040c6-107">Example</span></span>  
 <span data-ttu-id="040c6-108">O exemplo a seguir gera erro CS1656 porque ele tenta substituir elementos completos de uma coleção dentro de um loop `foreach`.</span><span class="sxs-lookup"><span data-stu-id="040c6-108">The following example generates error CS1656 because it tries to replace complete elements of a collection inside a `foreach` loop.</span></span> <span data-ttu-id="040c6-109">Uma maneira de contornar esse erro é alterar o loop `foreach` para um loop [for](../../../csharp/language-reference/keywords/for.md).</span><span class="sxs-lookup"><span data-stu-id="040c6-109">One way to work around the error is to change the `foreach` loop to a [for](../../../csharp/language-reference/keywords/for.md) loop.</span></span> <span data-ttu-id="040c6-110">Outra maneira, não mostrada aqui, é modificar os membros do elemento existente. Isso é possível com classes, mas não com structs.</span><span class="sxs-lookup"><span data-stu-id="040c6-110">Another way, not shown here, is to modify the members of the existing element; this is possible with classes, but not with structs.</span></span>  
  
```csharp  
using System;  
using System.Collections;  
using System.Collections.Generic;  
using System.Text;  
  
namespace CS1656_2  
{  
  
    class Book  
    {  
        public string Title;  
        public string Author;  
        public double Price;  
        public Book(string t, string a, double p)  
        {  
            Title=t;  
            Author=a;  
            Price=p;  
  
        }  
    }  
  
    class Program  
    {  
        private List<Book> list;  
        static void Main(string[] args)  
        {  
            Program prog = new Program();  
            prog.list = new List<Book>();  
            prog.list.Add(new Book ("The C# Programming Language",  
                                    "Hejlsberg, Wiltamuth, Golde",  
                                     29.95));  
            prog.list.Add(new Book ("The C++ Programming Language",  
                                    "Stroustrup",  
                                     29.95));  
            prog.list.Add(new Book ("The C Programming Language",  
                                    "Kernighan, Ritchie",  
                                    29.95));  
            foreach(Book b in prog.list)  
            {  
                // Cannot modify an entire element in a foreach loop   
                // even with reference types.  
                // Use a for or while loop instead  
                if (b.Title == "The C Programming Language")  
                    // Cannot assign to 'b' because it is a 'foreach   
                    // iteration variable'  
                    b = new Book("Programming Windows, 5th Ed.", "Petzold", 29.95); //CS1656  
            }  
  
            //With a for loop you can modify elements  
            //for(int x = 0; x < prog.list.Count; x++)  
            //{  
            //    if(prog.list[x].Title== "The C Programming Language")  
            //        prog.list[x] = new Book("Programming Windows, 5th Ed.", "Petzold", 29.95);  
            //}  
            //foreach(Book b in prog.list)  
            //    Console.WriteLine(b.Title);  
  
        }  
    }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="040c6-111">Exemplo</span><span class="sxs-lookup"><span data-stu-id="040c6-111">Example</span></span>  
 <span data-ttu-id="040c6-112">O exemplo a seguir demonstra como o erro CS1656 pode ser gerado em outros contextos, diferentes de um loop `foreach`:</span><span class="sxs-lookup"><span data-stu-id="040c6-112">The following sample demonstrates how CS1656 can be generated in other contexts besides a `foreach` loop:</span></span>  
  
```csharp  
// CS1656.cs  
// compile with: /unsafe  
using System;  
  
class C : IDisposable  
{  
    public void Dispose() { }  
}  
  
class CMain  
{  
    unsafe public static void Main()  
    {  
        using (C c = new C())  
        {  
            // Cannot assign to 'c' because it is a 'using variable'  
            c = new C(); // CS1656  
        }  
  
        int[] ary = new int[] { 1, 2, 3, 4 };  
        fixed (int* p = ary)  
        {  
            // Cannot assign to 'p' because it is a 'fixed variable'  
            p = null; // CS1656  
        }  
    }  
}  
```
