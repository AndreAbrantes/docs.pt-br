## <a name="example"></a><span data-ttu-id="d9dee-108">Exemplo</span><span class="sxs-lookup"><span data-stu-id="d9dee-108">Example</span></span>  
 <span data-ttu-id="d9dee-109">O código a seguir gera o erro CS1941 porque o operador `equals` está sendo solicitado a comparar um `int` com um `string`.</span><span class="sxs-lookup"><span data-stu-id="d9dee-109">The following code generates CS1941 because the `equals` operator is being asked to compare an `int` to a `string`.</span></span>  
  
```csharp  
// cs1941.cs  
using System.Collections;  
using System.Linq;  
class Test  
{  
    static int Main()  
    {  
        var nums = new[] { 1, 2, 3, 4, 5, 6 };  
        var words = new string[] { "lake", "mountain", "sky" };  
        IEnumerable e = from n in nums  
                        join w in words on n equals w // CS1941  
                        select w;  
        return 0;  
    }  
}  
```  
  
 <span data-ttu-id="d9dee-110">O método no qual a inferência de tipos falha é o método para o qual a cláusula de consulta é movida em tempo de compilação.</span><span class="sxs-lookup"><span data-stu-id="d9dee-110">The method in which type inference fails is the method that the query clause is translated to at compile time.</span></span>
