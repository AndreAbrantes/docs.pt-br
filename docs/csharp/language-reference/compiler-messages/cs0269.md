# <a name="compiler-error-cs0269"></a><span data-ttu-id="8f860-102">Erro do Compilador CS0269</span><span class="sxs-lookup"><span data-stu-id="8f860-102">Compiler Error CS0269</span></span>
<span data-ttu-id="8f860-103">Uso do parâmetro de saída não atribuído 'parameter'</span><span class="sxs-lookup"><span data-stu-id="8f860-103">Use of unassigned out parameter 'parameter'</span></span>  
  
 <span data-ttu-id="8f860-104">O compilador não pôde verificar se um valor foi atribuído ao parâmetro de saída antes de ele ser usado; seu valor pode ser indefinido quando atribuído.</span><span class="sxs-lookup"><span data-stu-id="8f860-104">The compiler could not verify that the out parameter was assigned a value before it was used; its value may be undefined when assigned.</span></span> <span data-ttu-id="8f860-105">Atribua um valor aos parâmetros `out` no método chamado antes de acessar o valor.</span><span class="sxs-lookup"><span data-stu-id="8f860-105">Be sure to assign a value to `out` parameters in the called method before accessing the value.</span></span> <span data-ttu-id="8f860-106">Se for necessário usar o valor da variável passada, em vez disso, use um parâmetro `ref`.</span><span class="sxs-lookup"><span data-stu-id="8f860-106">If you need to use the value of the variable passed in, use a `ref` parameter instead.</span></span> <span data-ttu-id="8f860-107">Para obter mais informações, consulte [Passando Parâmetros](../../programming-guide/classes-and-structs/passing-parameters.md).</span><span class="sxs-lookup"><span data-stu-id="8f860-107">For more information, see [Passing Parameters](../../programming-guide/classes-and-structs/passing-parameters.md).</span></span>  
  
## <a name="example"></a><span data-ttu-id="8f860-108">Exemplo</span><span class="sxs-lookup"><span data-stu-id="8f860-108">Example</span></span>  
 <span data-ttu-id="8f860-109">O exemplo a seguir gera o erro CS0269:</span><span class="sxs-lookup"><span data-stu-id="8f860-109">The following sample generates CS0269:</span></span>  
  
```csharp  
// CS0269.cs  
class C  
{  
    public static void F(out int i)  
    // One way to resolve the error is to use a ref parameter instead  
    // of an out parameter.  
    //public static void F(ref int i)  
    {  
        // The following line causes a compiler error because no value  
        // has been assigned to i.  
        int k = i;  // CS0269  
        i = 1;  
        // The error does not occur if the order of the two previous   
        // lines is reversed.  
    }  
  
    public static void Main()  
    {  
        int myInt = 1;  
        F(out myInt);  
        // If the declaration of method F is changed to require a ref  
        // parameter, ref must be specified in the call as well.  
        //F(ref myInt);  
    }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="8f860-110">Exemplo</span><span class="sxs-lookup"><span data-stu-id="8f860-110">Example</span></span>  
 <span data-ttu-id="8f860-111">Isso também poderá acontecer se a inicialização de uma variável ocorrer em um bloco de tentativa, que o compilador não pode verificar, será executada com êxito:</span><span class="sxs-lookup"><span data-stu-id="8f860-111">This could also occur if the initialization of a variable occurs in a try block, which the compiler is unable to verify will execute successfully:</span></span>  
  
```csharp  
// CS0269b.cs  
class C  
{  
    public static void F(out int i)  
    {  
        try  
        {  
            // Assignment occurs, but compiler can't verify it  
            i = 1;  
        }  
        catch  
        {  
        }  
  
        int k = i;  // CS0269  
        i = 1;  
    }  
  
    public static void Main()  
    {  
        int myInt;  
        F(out myInt);  
    }  
}  
```
