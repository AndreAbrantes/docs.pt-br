# <a name="compiler-warning-level-3-cs0675"></a><span data-ttu-id="52ce5-102">Aviso do compilador (nível 3) CS0675</span><span class="sxs-lookup"><span data-stu-id="52ce5-102">Compiler Warning (level 3) CS0675</span></span>
<span data-ttu-id="52ce5-103">Operador OR bit a bit usado em um operando de assinatura estendida; é recomendável realizar a conversão em um tipo menor sem assinatura primeiro</span><span class="sxs-lookup"><span data-stu-id="52ce5-103">Bitwise-or operator used on a sign-extended operand; consider casting to a smaller unsigned type first</span></span>  
  
 <span data-ttu-id="52ce5-104">O compilador ampliou e estendeu a assinatura de uma variável implicitamente, usando posteriormente o valor resultante em uma operação OR bit a bit.</span><span class="sxs-lookup"><span data-stu-id="52ce5-104">The compiler implicitly widened and sign-extended a variable, and then used the resulting value in a bitwise OR operation.</span></span> <span data-ttu-id="52ce5-105">Isso pode resultar em um comportamento inesperado.</span><span class="sxs-lookup"><span data-stu-id="52ce5-105">This can result in unexpected behavior.</span></span>  
  
 <span data-ttu-id="52ce5-106">O exemplo a seguir gera o erro CS0675:</span><span class="sxs-lookup"><span data-stu-id="52ce5-106">The following sample generates CS0675:</span></span>  
  
```csharp  
// CS0675.cs  
// compile with: /W:3  
using System;  
  
public class sign  
{  
   public static void Main()  
   {  
      int hi = 1;  
      int lo = 1;  
      long value = (((long)hi) << 32) | lo;              // CS0675  
      // try the following line instead  
      // long value = (((long)hi) << 32) | ((uint)lo);   // correct  
   }  
}  
```
