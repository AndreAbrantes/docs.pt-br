# <a name="getting-started-with-f"></a>Guia de Introdução com F # #

Há várias maneiras de começar a F #.  Temos vários artigos escritos para fornecer um guia para cada forma principal.  Você pode usar a tabela a seguir para ajudar na tomada de decisão.

| SISTEMA OPERACIONAL | Preferir o Visual Studio | Prefira código do Visual Studio | Preferir uma linha de comando |
| -- |------------------------|--------------------------|-----------------------------|-------------------------|
| Windows | [Introdução ao Visual Studio](get-started-visual-studio.md) | [Começar a usar o VSCode e Ionide](get-started-vscode.md) | [Introdução ao .NET Core CLI](get-started-command-line.md) |
| macOS | [Introdução ao VS para Mac](get-started-with-visual-studio-for-mac.md) | [Começar a usar o VSCode e Ionide](get-started-vscode.md) | [Introdução ao .NET Core CLI](get-started-command-line.md) |
| Linux | N/D | [Começar a usar o VSCode e Ionide](get-started-vscode.md) | [Introdução ao .NET Core CLI](get-started-command-line.md) |

Em geral, não é possível começar específico que é melhor do que o restante.  É recomendável tentar todas as maneiras de usar F # em seu computador para ver o que você deseja que o melhor!
