# <a name="getting-started-with-f"></a>Guia de IntroduçãoF# #

Há várias maneiras para começar com F#.  Temos vários artigos escritos para fornecer um guia para cada forma significativa.  Você pode usar a tabela a seguir para ajudar na tomada de decisão.

| Sistema operacional | Preferir o Visual Studio | Preferir o Visual Studio Code | Prefere uma linha de comando |
| -- |------------------------|--------------------------|-----------------------------|-------------------------|
| Windows | [Introdução ao Visual Studio](get-started-visual-studio.md) | [Introdução ao VSCode e Ionide](get-started-vscode.md) | [Comece com a CLI do .NET Core](get-started-command-line.md) |
| macOS | [Introdução ao VS para Mac](get-started-with-visual-studio-for-mac.md) | [Introdução ao VSCode e Ionide](get-started-vscode.md) | [Comece com a CLI do .NET Core](get-started-command-line.md) |
| Linux | N/D | [Introdução ao VSCode e Ionide](get-started-vscode.md) | [Comece com a CLI do .NET Core](get-started-command-line.md) |

Em geral, não há nenhuma maneira específica para começar a usar o que é melhor do que o restante.  É recomendável tentar todas as maneiras de usar F# em seu computador para ver o que você deseja o melhor!
