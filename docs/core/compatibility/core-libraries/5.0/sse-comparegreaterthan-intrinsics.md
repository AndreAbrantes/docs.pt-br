# <a name="sse-and-sse2-comparegreaterthan-methods-properly-handle-nan-inputs"></a><span data-ttu-id="05f2f-103">Os métodos SSE e SSE2 CompareGreaterThan manipulam corretamente as entradas NaN</span><span class="sxs-lookup"><span data-stu-id="05f2f-103">SSE and SSE2 CompareGreaterThan methods properly handle NaN inputs</span></span>

<span data-ttu-id="05f2f-104">Os métodos e a seguir foram <xref:System.Runtime.Intrinsics.X86.Sse?displayProperty=nameWithType> <xref:System.Runtime.Intrinsics.X86.Sse2?displayProperty=nameWithType> corrigidos para lidar corretamente com `NaN` entradas e corresponder ao comportamento de hardware dos métodos equivalentes na <xref:System.Runtime.Intrinsics.X86.Avx?displayProperty=nameWithType> classe:</span><span class="sxs-lookup"><span data-stu-id="05f2f-104">The following <xref:System.Runtime.Intrinsics.X86.Sse?displayProperty=nameWithType> and <xref:System.Runtime.Intrinsics.X86.Sse2?displayProperty=nameWithType> methods have been fixed to properly handle `NaN` inputs and match the hardware behavior of the equivalent methods in the <xref:System.Runtime.Intrinsics.X86.Avx?displayProperty=nameWithType> class:</span></span>

* `CompareGreaterThan`
* `CompareGreaterThanOrEqual`
* `CompareNotGreaterThan`
* `CompareNotGreaterThanOrEqual`
* `CompareScalarGreaterThan`
* `CompareScalarGreaterThanOrEqual`
* `CompareScalarNotGreaterThan`
* `CompareScalarNotGreaterThanOrEqual`

## <a name="change-description"></a><span data-ttu-id="05f2f-105">Descrição das alterações</span><span class="sxs-lookup"><span data-stu-id="05f2f-105">Change description</span></span>

<span data-ttu-id="05f2f-106">Anteriormente, `NaN` as entradas para os <xref:System.Runtime.Intrinsics.X86.Sse> métodos listados e <xref:System.Runtime.Intrinsics.X86.Sse2> retornaram um resultado incorreto.</span><span class="sxs-lookup"><span data-stu-id="05f2f-106">Previously, `NaN` inputs to the listed <xref:System.Runtime.Intrinsics.X86.Sse> and <xref:System.Runtime.Intrinsics.X86.Sse2> methods returned an incorrect result.</span></span> <span data-ttu-id="05f2f-107">O resultado também é diferente do resultado gerado pelo método correspondente na <xref:System.Runtime.Intrinsics.X86.Avx> classe.</span><span class="sxs-lookup"><span data-stu-id="05f2f-107">The result also differed from the result generated by the corresponding method in the <xref:System.Runtime.Intrinsics.X86.Avx> class.</span></span>

<span data-ttu-id="05f2f-108">A partir do .NET 5,0, esses métodos manipulam corretamente `NaN` as entradas e retornam os mesmos resultados que os métodos correspondentes na <xref:System.Runtime.Intrinsics.X86.Avx> classe.</span><span class="sxs-lookup"><span data-stu-id="05f2f-108">Starting in .NET 5.0, these methods correctly handle `NaN` inputs and return the same results as the corresponding methods in the <xref:System.Runtime.Intrinsics.X86.Avx> class.</span></span>

<span data-ttu-id="05f2f-109">As arquiteturas de Streaming SIMD Extensions (SSE) e Streaming SIMD Extensions 2 (SSE2) do setor (ISAs) não fornecem suporte direto de hardware para esses métodos de comparação, para que sejam implementadas no software.</span><span class="sxs-lookup"><span data-stu-id="05f2f-109">The Streaming SIMD Extensions (SSE) and Streaming SIMD Extensions 2 (SSE2) industry standard architectures (ISAs) don't provide direct hardware support for these comparison methods, so they're implemented in software.</span></span> <span data-ttu-id="05f2f-110">Anteriormente, os métodos foram implementados incorretamente e eles incorretamente manipulavam `NaN` entradas.</span><span class="sxs-lookup"><span data-stu-id="05f2f-110">Previously, the methods were improperly implemented, and they incorrectly handled `NaN` inputs.</span></span> <span data-ttu-id="05f2f-111">Para o código portado de nativo, o comportamento incorreto pode introduzir bugs.</span><span class="sxs-lookup"><span data-stu-id="05f2f-111">For code ported from native, the incorrect behavior may introduce bugs.</span></span> <span data-ttu-id="05f2f-112">Para um caminho de código de 256 bits, os métodos também podem produzir resultados diferentes para os métodos equivalentes na <xref:System.Runtime.Intrinsics.X86.Avx> classe.</span><span class="sxs-lookup"><span data-stu-id="05f2f-112">For a 256-bit code path, the methods can also produce different results to the equivalent methods in the <xref:System.Runtime.Intrinsics.X86.Avx> class.</span></span>

<span data-ttu-id="05f2f-113">Como exemplo de como os métodos estavam incorretos anteriormente, você pode implementar `CompareNotGreaterThan(x,y)` como `CompareLessThanOrEqual(x,y)` para inteiros regulares.</span><span class="sxs-lookup"><span data-stu-id="05f2f-113">As an example of how the methods were previously incorrect, you can implement `CompareNotGreaterThan(x,y)` as `CompareLessThanOrEqual(x,y)` for regular integers.</span></span> <span data-ttu-id="05f2f-114">No entanto, para `NaN` entradas, essa lógica computa o resultado errado.</span><span class="sxs-lookup"><span data-stu-id="05f2f-114">However, for `NaN` inputs, that logic computes the wrong result.</span></span> <span data-ttu-id="05f2f-115">Em vez disso, usar `CompareNotLessThan(y,x)` compara os números corretamente *e* leva as `NaN` entradas em consideração.</span><span class="sxs-lookup"><span data-stu-id="05f2f-115">Instead, using `CompareNotLessThan(y,x)` compares the numbers correctly *and* takes `NaN` inputs into consideration.</span></span>

  * `CompareScalarNotGreaterThan(x,y)` -> `CompareScalarLessThanOrEqual(x,y)`
