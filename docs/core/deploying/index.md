## <a name="self-contained-deployments-scd"></a><span data-ttu-id="b89fe-131">SCD (implantação autocontida)</span><span class="sxs-lookup"><span data-stu-id="b89fe-131">Self-contained deployments (SCD)</span></span>

<span data-ttu-id="b89fe-132">No caso de uma implantação autocontida, você implanta o aplicativo e as dependências de terceiros necessárias, juntamente com a versão do .NET Core que usou para criar o aplicativo.</span><span class="sxs-lookup"><span data-stu-id="b89fe-132">For a self-contained deployment, you deploy your app and any required third-party dependencies along with the version of .NET Core that you used to build the app.</span></span> <span data-ttu-id="b89fe-133">A criação de uma SCD não inclui as [dependências nativas do .NET Core](https://github.com/dotnet/core/blob/master/Documentation/prereqs.md) de várias plataformas, por isso elas devem estar presentes antes de executar o aplicativo.</span><span class="sxs-lookup"><span data-stu-id="b89fe-133">Creating an SCD doesn't include the [native dependencies of .NET Core](https://github.com/dotnet/core/blob/master/Documentation/prereqs.md) on various platforms, so these must be present before the app runs.</span></span>

<span data-ttu-id="b89fe-136">Implantar uma implantação autocontida apresenta duas vantagens principais:</span><span class="sxs-lookup"><span data-stu-id="b89fe-136">Deploying a Self-contained deployment has two major advantages:</span></span>

- <span data-ttu-id="b89fe-143">Implantar vários aplicativos .NET Core autocontidos em um sistema pode consumir um volume significativo de espaço em disco, visto que cada aplicativo duplica os arquivos do .NET Core.</span><span class="sxs-lookup"><span data-stu-id="b89fe-143">Deploying numerous self-contained .NET Core apps to a system can consume significant amounts of disk space, since each app duplicates .NET Core files.</span></span>
