## <a name="rules"></a><span data-ttu-id="4e713-104">Regras</span><span class="sxs-lookup"><span data-stu-id="4e713-104">Rules</span></span>  

 <span data-ttu-id="4e713-105">Um nome de elemento em Visual Basic deve observar as seguintes regras:</span><span class="sxs-lookup"><span data-stu-id="4e713-105">An element name in Visual Basic must observe the following rules:</span></span>

 <span data-ttu-id="4e713-111">O exemplo a seguir mostra alguns nomes de elemento válidos.</span><span class="sxs-lookup"><span data-stu-id="4e713-111">The following example shows some valid element names.</span></span>

 `' Three INVALID element names`  
  
 `_`  
  
 `12ABC`  
  
 `xyz$wv`

## <a name="case-sensitivity-in-names"></a><span data-ttu-id="4e713-133">Distinção de maiúsculas e minúsculas em nomes</span><span class="sxs-lookup"><span data-stu-id="4e713-133">Case Sensitivity in Names</span></span>  

 <span data-ttu-id="4e713-134">Os nomes de elementos em Visual Basic não diferenciam maiúsculas de minúsculas.</span><span class="sxs-lookup"><span data-stu-id="4e713-134">Element names in Visual Basic are case-insensitive.</span></span> <span data-ttu-id="4e713-135">Isso significa que, quando o compilador compara dois nomes que diferem somente em maiúsculas e minúsculas, ele os interpreta como o mesmo nome.</span><span class="sxs-lookup"><span data-stu-id="4e713-135">This means that when the compiler compares two names that differ in alphabetic case only, it interprets them as the same name.</span></span> <span data-ttu-id="4e713-136">Por exemplo, ele considera `ABC` e `abc` se refere ao mesmo elemento declarado.</span><span class="sxs-lookup"><span data-stu-id="4e713-136">For example, it considers `ABC` and `abc` to refer to the same declared element.</span></span>  
  
 <span data-ttu-id="4e713-137">No entanto, o Common Language Runtime (CLR) usa a associação que diferencia maiúsculas de minúsculas.</span><span class="sxs-lookup"><span data-stu-id="4e713-137">However, the common language runtime (CLR) uses case-sensitive binding.</span></span> <span data-ttu-id="4e713-138">Portanto, quando você produz um assembly ou uma DLL e o disponibiliza para outros assemblies, seus nomes não são mais sensíveis a maiúsculas e minúsculas.</span><span class="sxs-lookup"><span data-stu-id="4e713-138">Therefore, when you produce an assembly or a DLL and make it available to other assemblies, your names are no longer case-insensitive.</span></span> <span data-ttu-id="4e713-139">Por exemplo, se você definir uma classe com um elemento chamado `ABC` , e outros assemblies fizerem uso de sua classe por meio da Common Language Runtime, eles deverão se referir ao elemento como `ABC` .</span><span class="sxs-lookup"><span data-stu-id="4e713-139">For example, if you define a class with an element called `ABC`, and other assemblies make use of your class through the common language runtime, they must refer to the element as `ABC`.</span></span> <span data-ttu-id="4e713-140">Se, posteriormente, você recompilar sua classe e alterar o nome do elemento para `abc` , os outros assemblies que usam sua classe não poderão mais acessar esse elemento.</span><span class="sxs-lookup"><span data-stu-id="4e713-140">If you subsequently recompile your class and change the element's name to `abc`, the other assemblies using your class could no longer access that element.</span></span> <span data-ttu-id="4e713-141">Portanto, quando você libera uma versão atualizada de um assembly, não deve alterar o caso alfabético de quaisquer elementos públicos.</span><span class="sxs-lookup"><span data-stu-id="4e713-141">Therefore, when you release an updated version of an assembly, you should not change the alphabetic case of any public elements.</span></span>
