<span data-ttu-id="417ec-103">Quando você passa um ou mais argumentos para um procedimento, cada argumento corresponde a um elemento de programação subjacente no código de chamada.</span><span class="sxs-lookup"><span data-stu-id="417ec-103">When you pass one or more arguments to a procedure, each argument corresponds to an underlying programming element in the calling code.</span></span> <span data-ttu-id="417ec-104">Você pode passar o valor desse elemento subjacente ou uma referência a ele.</span><span class="sxs-lookup"><span data-stu-id="417ec-104">You can pass either the value of this underlying element, or a reference to it.</span></span> <span data-ttu-id="417ec-105">Isso é conhecido como o *mecanismo de passagem*.</span><span class="sxs-lookup"><span data-stu-id="417ec-105">This is known as the *passing mechanism*.</span></span>  
  
## <a name="passing-by-value"></a><span data-ttu-id="417ec-106">passagem por valor</span><span class="sxs-lookup"><span data-stu-id="417ec-106">Passing by Value</span></span>  
 <span data-ttu-id="417ec-107">Você passa um argumento *por valor* especificando a palavra-chave [ByVal](../../../../visual-basic/language-reference/modifiers/byval.md) para o parâmetro correspondente na definição do procedimento.</span><span class="sxs-lookup"><span data-stu-id="417ec-107">You pass an argument *by value* by specifying the [ByVal](../../../../visual-basic/language-reference/modifiers/byval.md) keyword for the corresponding parameter in the procedure definition.</span></span> <span data-ttu-id="417ec-108">Quando você usa esse mecanismo de passagem, o Visual Basic copia o valor do elemento de programação subjacente em uma variável local no procedimento.</span><span class="sxs-lookup"><span data-stu-id="417ec-108">When you use this passing mechanism, Visual Basic copies the value of the underlying programming element into a local variable in the procedure.</span></span> <span data-ttu-id="417ec-109">O código do procedimento não tem nenhum acesso ao elemento subjacente no código de chamada.</span><span class="sxs-lookup"><span data-stu-id="417ec-109">The procedure code does not have any access to the underlying element in the calling code.</span></span>  
  
## <a name="passing-by-reference"></a><span data-ttu-id="417ec-110">Passando por referência</span><span class="sxs-lookup"><span data-stu-id="417ec-110">Passing by Reference</span></span>  
 <span data-ttu-id="417ec-111">Você passa um argumento *por referência* especificando a palavra-chave [ByRef](../../../../visual-basic/language-reference/modifiers/byref.md) para o parâmetro correspondente na definição do procedimento.</span><span class="sxs-lookup"><span data-stu-id="417ec-111">You pass an argument *by reference* by specifying the [ByRef](../../../../visual-basic/language-reference/modifiers/byref.md) keyword for the corresponding parameter in the procedure definition.</span></span> <span data-ttu-id="417ec-112">Quando você usa esse mecanismo de passagem, Visual Basic dá ao procedimento uma referência direta para o elemento de programação subjacente no código de chamada.</span><span class="sxs-lookup"><span data-stu-id="417ec-112">When you use this passing mechanism, Visual Basic gives the procedure a direct reference to the underlying programming element in the calling code.</span></span>  
  
## <a name="passing-mechanism-and-element-type"></a><span data-ttu-id="417ec-113">Tipo de mecanismo e elemento de passagem</span><span class="sxs-lookup"><span data-stu-id="417ec-113">Passing Mechanism and Element Type</span></span>  
 <span data-ttu-id="417ec-114">A escolha do mecanismo de passagem não é a mesma que a classificação do tipo de elemento subjacente.</span><span class="sxs-lookup"><span data-stu-id="417ec-114">The choice of passing mechanism is not the same as the classification of the underlying element type.</span></span> <span data-ttu-id="417ec-115">A passagem por valor ou por referência refere-se ao que Visual Basic fornece ao código do procedimento.</span><span class="sxs-lookup"><span data-stu-id="417ec-115">Passing by value or by reference refers to what Visual Basic supplies to the procedure code.</span></span> <span data-ttu-id="417ec-116">Um tipo de valor ou tipo de referência refere-se a como um elemento de programação é armazenado na memória.</span><span class="sxs-lookup"><span data-stu-id="417ec-116">A value type or reference type refers to how a programming element is stored in memory.</span></span>  
  
 <span data-ttu-id="417ec-117">No entanto, o mecanismo de passagem e o tipo de elemento são inter-relacionados.</span><span class="sxs-lookup"><span data-stu-id="417ec-117">However, the passing mechanism and element type are interrelated.</span></span> <span data-ttu-id="417ec-118">O valor de um tipo de referência é um ponteiro para os dados em outro lugar na memória.</span><span class="sxs-lookup"><span data-stu-id="417ec-118">The value of a reference type is a pointer to the data elsewhere in memory.</span></span> <span data-ttu-id="417ec-119">Isso significa que quando você passa um tipo de referência por valor, o código do procedimento tem um ponteiro para os dados do elemento subjacente, mesmo que ele não possa acessar o próprio elemento subjacente.</span><span class="sxs-lookup"><span data-stu-id="417ec-119">This means that when you pass a reference type by value, the procedure code has a pointer to the underlying element's data, even though it cannot access the underlying element itself.</span></span> <span data-ttu-id="417ec-120">Por exemplo, se o elemento for uma variável de matriz, o código do procedimento não terá acesso à variável em si, mas poderá acessar os membros da matriz.</span><span class="sxs-lookup"><span data-stu-id="417ec-120">For example, if the element is an array variable, the procedure code does not have access to the variable itself, but it can access the array members.</span></span>
