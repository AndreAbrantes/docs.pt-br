<span data-ttu-id="ec41a-103">Quando você passa um ou mais argumentos para um procedimento, cada argumento corresponde a um elemento de programação subjacente no código de chamada.</span><span class="sxs-lookup"><span data-stu-id="ec41a-103">When you pass one or more arguments to a procedure, each argument corresponds to an underlying programming element in the calling code.</span></span> <span data-ttu-id="ec41a-104">Você pode passar o valor desse elemento ou uma referência a ele.</span><span class="sxs-lookup"><span data-stu-id="ec41a-104">You can pass either the value of this underlying element, or a reference to it.</span></span> <span data-ttu-id="ec41a-105">Isso é conhecido como o *mecanismo de passagem*.</span><span class="sxs-lookup"><span data-stu-id="ec41a-105">This is known as the *passing mechanism*.</span></span>  
  
## <a name="passing-by-value"></a><span data-ttu-id="ec41a-106">passagem por valor</span><span class="sxs-lookup"><span data-stu-id="ec41a-106">Passing by Value</span></span>  
 <span data-ttu-id="ec41a-107">Passar um argumento *pelo valor* , especificando as [ByVal](../../../../visual-basic/language-reference/modifiers/byval.md) palavra-chave para o parâmetro correspondente na definição do procedimento.</span><span class="sxs-lookup"><span data-stu-id="ec41a-107">You pass an argument *by value* by specifying the [ByVal](../../../../visual-basic/language-reference/modifiers/byval.md) keyword for the corresponding parameter in the procedure definition.</span></span> <span data-ttu-id="ec41a-108">Quando você usa este mecanismo de passagem, o Visual Basic copia o valor do elemento de programação subjacente em uma variável local no procedimento.</span><span class="sxs-lookup"><span data-stu-id="ec41a-108">When you use this passing mechanism, Visual Basic copies the value of the underlying programming element into a local variable in the procedure.</span></span> <span data-ttu-id="ec41a-109">O código do procedimento não tem qualquer acesso ao elemento subjacente no código de chamada.</span><span class="sxs-lookup"><span data-stu-id="ec41a-109">The procedure code does not have any access to the underlying element in the calling code.</span></span>  
  
## <a name="passing-by-reference"></a><span data-ttu-id="ec41a-110">Passagem por referência</span><span class="sxs-lookup"><span data-stu-id="ec41a-110">Passing by Reference</span></span>  
 <span data-ttu-id="ec41a-111">Passar um argumento *por referência* , especificando as [ByRef](../../../../visual-basic/language-reference/modifiers/byref.md) palavra-chave para o parâmetro correspondente na definição do procedimento.</span><span class="sxs-lookup"><span data-stu-id="ec41a-111">You pass an argument *by reference* by specifying the [ByRef](../../../../visual-basic/language-reference/modifiers/byref.md) keyword for the corresponding parameter in the procedure definition.</span></span> <span data-ttu-id="ec41a-112">Quando você usa este mecanismo de passagem, o Visual Basic fornece o procedimento uma referência direta ao elemento de programação subjacente no código de chamada.</span><span class="sxs-lookup"><span data-stu-id="ec41a-112">When you use this passing mechanism, Visual Basic gives the procedure a direct reference to the underlying programming element in the calling code.</span></span>  
  
## <a name="passing-mechanism-and-element-type"></a><span data-ttu-id="ec41a-113">Mecanismo de passagem e o tipo de elemento</span><span class="sxs-lookup"><span data-stu-id="ec41a-113">Passing Mechanism and Element Type</span></span>  
 <span data-ttu-id="ec41a-114">A escolha do mecanismo de passagem não é o mesmo que a classificação do tipo de elemento subjacente.</span><span class="sxs-lookup"><span data-stu-id="ec41a-114">The choice of passing mechanism is not the same as the classification of the underlying element type.</span></span> <span data-ttu-id="ec41a-115">Passagem por valor ou referência se refere a que o Visual Basic fornece ao código do procedimento.</span><span class="sxs-lookup"><span data-stu-id="ec41a-115">Passing by value or by reference refers to what Visual Basic supplies to the procedure code.</span></span> <span data-ttu-id="ec41a-116">Um tipo de valor ou tipo de referência se refere a como um elemento de programação é armazenado na memória.</span><span class="sxs-lookup"><span data-stu-id="ec41a-116">A value type or reference type refers to how a programming element is stored in memory.</span></span>  
  
 <span data-ttu-id="ec41a-117">No entanto, o mecanismo de passagem e o tipo de elemento são inter-relacionadas.</span><span class="sxs-lookup"><span data-stu-id="ec41a-117">However, the passing mechanism and element type are interrelated.</span></span> <span data-ttu-id="ec41a-118">O valor de um tipo de referência é um ponteiro para os dados em outro lugar na memória.</span><span class="sxs-lookup"><span data-stu-id="ec41a-118">The value of a reference type is a pointer to the data elsewhere in memory.</span></span> <span data-ttu-id="ec41a-119">Isso significa que, quando você passa um tipo de referência por valor, o código do procedimento tem um ponteiro para os dados do elemento, mesmo que ele não pode acessar o elemento subjacente.</span><span class="sxs-lookup"><span data-stu-id="ec41a-119">This means that when you pass a reference type by value, the procedure code has a pointer to the underlying element's data, even though it cannot access the underlying element itself.</span></span> <span data-ttu-id="ec41a-120">Por exemplo, se o elemento for uma variável de matriz, o código do procedimento não tem acesso à variável em si, mas ele pode acessar os membros da matriz.</span><span class="sxs-lookup"><span data-stu-id="ec41a-120">For example, if the element is an array variable, the procedure code does not have access to the variable itself, but it can access the array members.</span></span>
