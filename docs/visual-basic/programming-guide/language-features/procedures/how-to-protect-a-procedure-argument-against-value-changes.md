<span data-ttu-id="6d5a5-103">Se um procedimento declara um parâmetro como [ByRef](../../../../visual-basic/language-reference/modifiers/byref.md), o Visual Basic fornece ao código do procedimento uma referência direta para o elemento de programação subjacente do argumento no código de chamada.</span><span class="sxs-lookup"><span data-stu-id="6d5a5-103">If a procedure declares a parameter as [ByRef](../../../../visual-basic/language-reference/modifiers/byref.md), Visual Basic gives the procedure code a direct reference to the programming element underlying the argument in the calling code.</span></span> <span data-ttu-id="6d5a5-104">Isso permite que o procedimento altere o valor subjacente do argumento no código de chamada.</span><span class="sxs-lookup"><span data-stu-id="6d5a5-104">This permits the procedure to change the value underlying the argument in the calling code.</span></span> <span data-ttu-id="6d5a5-105">Em alguns casos, o código de chamada talvez queira estar protegido contra essa alteração.</span><span class="sxs-lookup"><span data-stu-id="6d5a5-105">In some cases the calling code might want to protect against such a change.</span></span>  
  
 <span data-ttu-id="6d5a5-106">Você pode proteger um argumento de alteração, declarando o parâmetro correspondente com a palavra-chave [ByVal](../../../../visual-basic/language-reference/modifiers/byval.md) no procedimento.</span><span class="sxs-lookup"><span data-stu-id="6d5a5-106">You can always protect an argument from change by declaring the corresponding parameter [ByVal](../../../../visual-basic/language-reference/modifiers/byval.md) in the procedure.</span></span> <span data-ttu-id="6d5a5-107">Se desejar alterar um argumento fornecido em alguns casos, mas outros não, você pode declará-la `ByRef` e permitir que o código de chamada determine o mecanismo de passagem em cada chamada.</span><span class="sxs-lookup"><span data-stu-id="6d5a5-107">If you want to be able to change a given argument in some cases but not others, you can declare it `ByRef` and let the calling code determine the passing mechanism in each call.</span></span> <span data-ttu-id="6d5a5-108">Ele faz isso colocando o argumento correspondente entre parênteses para passá-lo por valor, ou não envolvendo em parênteses para passá-lo por referência.</span><span class="sxs-lookup"><span data-stu-id="6d5a5-108">It does this by enclosing the corresponding argument in parentheses to pass it by value, or not enclosing it in parentheses to pass it by reference.</span></span> <span data-ttu-id="6d5a5-109">Para obter mais informações, consulte [como: forçar um argumento a ser passado por valor](./how-to-force-an-argument-to-be-passed-by-value.md).</span><span class="sxs-lookup"><span data-stu-id="6d5a5-109">For more information, see [How to: Force an Argument to Be Passed by Value](./how-to-force-an-argument-to-be-passed-by-value.md).</span></span>
