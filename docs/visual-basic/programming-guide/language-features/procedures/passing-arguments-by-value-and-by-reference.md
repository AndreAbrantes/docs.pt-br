## <a name="choice-of-passing-mechanism"></a><span data-ttu-id="4f832-113">Escolha do mecanismo de passagem</span><span class="sxs-lookup"><span data-stu-id="4f832-113">Choice of Passing Mechanism</span></span>

-   <span data-ttu-id="4f832-115">**Proteção**.</span><span class="sxs-lookup"><span data-stu-id="4f832-115">**Protection**.</span></span> <span data-ttu-id="4f832-116">Em escolher entre os mecanismos de passagem de dois, o critério mais importante é a exposição de chamada de variáveis para alterar.</span><span class="sxs-lookup"><span data-stu-id="4f832-116">In choosing between the two passing mechanisms, the most important criterion is the exposure of calling variables to change.</span></span> <span data-ttu-id="4f832-117">A vantagem de passar um argumento `ByRef` é que o procedimento pode retornar um valor para o código de chamada por meio desse argumento.</span><span class="sxs-lookup"><span data-stu-id="4f832-117">The advantage of passing an argument `ByRef` is that the procedure can return a value to the calling code through that argument.</span></span> <span data-ttu-id="4f832-118">A vantagem de passar um argumento `ByVal` é que ele protege uma variável que está sendo alterada pelo procedimento.</span><span class="sxs-lookup"><span data-stu-id="4f832-118">The advantage of passing an argument `ByVal` is that it protects a variable from being changed by the procedure.</span></span>
