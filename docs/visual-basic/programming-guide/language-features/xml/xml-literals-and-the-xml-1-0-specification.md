# <a name="xml-literals-and-the-xml-10-specification-visual-basic"></a>Especificação dos literais XML e do XML 1.0 (Visual Basic)
A sintaxe de literais XML em [!INCLUDE[vbprvb](../../../../csharp/programming-guide/concepts/linq/includes/vbprvb_md.md)] oferece suporte à maior parte da especificação do Extensible Markup Language (XML) 1.0. Para obter detalhes sobre a especificação do XML 1.0, consulte [Extensible Markup Language (XML) 1.0](http://go.microsoft.com/fwlink/?LinkId=73927) no site do W3C.  
  
## <a name="what-visual-basic-does-not-support"></a>O Visual Basic não dá suporte  
  
-   Um literal XML não pode conter um document type definition (DTD).  
  
-   Um literal de documento XML deve começar com uma declaração de documento XML.  
  
-   Um literal XML não pode conter mais de 65.535 caracteres em uma linha.  
  
-   Prefixos de namespace XML, nomes de elemento e nomes de atributo não podem conter mais de 1.024 caracteres.  
  
## <a name="extra-features-that-visual-basic-supports"></a>Recursos adicionais que oferece suporte ao Visual Basic  
  
-   A sintaxe de expressão inserida permitida em documentos e literais de elemento não é XML válido.
