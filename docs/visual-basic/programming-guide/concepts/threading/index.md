 <span data-ttu-id="66eb0-105">Os threads têm as seguintes propriedades:</span><span class="sxs-lookup"><span data-stu-id="66eb0-105">Threads have the following properties:</span></span>  
  
-   <span data-ttu-id="66eb0-106">Eles permitem que seu programa execute processamento simultâneo.</span><span class="sxs-lookup"><span data-stu-id="66eb0-106">Threads enable your program to perform concurrent processing.</span></span>

 <span data-ttu-id="66eb0-110">Por padrão, um programa do Visual Basic tem um thread.</span><span class="sxs-lookup"><span data-stu-id="66eb0-110">By default, a Visual Basic program has one thread.</span></span> <span data-ttu-id="66eb0-111">No entanto, threads auxiliares podem ser criados e usados para executar código em paralelo com o thread primário.</span><span class="sxs-lookup"><span data-stu-id="66eb0-111">However, auxiliary threads can be created and used to execute code in parallel with the primary thread.</span></span> <span data-ttu-id="66eb0-112">Esses segmentos são chamados de *threads de trabalho*.</span><span class="sxs-lookup"><span data-stu-id="66eb0-112">These threads are often called *worker threads*.</span></span>  
  
 <span data-ttu-id="66eb0-113">Os threads de trabalho podem ser usados para executar tarefas demoradas ou críticas sem prender o thread primário.</span><span class="sxs-lookup"><span data-stu-id="66eb0-113">Worker threads can be used to perform time-consuming or time-critical tasks without tying up the primary thread.</span></span> <span data-ttu-id="66eb0-114">Por exemplo, threads de trabalho são geralmente usados em aplicativos de servidor para atender às solicitações de entrada sem aguardar a solicitação anterior seja concluída.</span><span class="sxs-lookup"><span data-stu-id="66eb0-114">For example, worker threads are often used in server applications to fulfill incoming requests without waiting for the previous request to be completed.</span></span> <span data-ttu-id="66eb0-115">Os threads de trabalho também são usados para executar tarefas "em segundo plano" em aplicativos da área de trabalho para que o thread principal – que orienta os elementos da interface do usuário – permaneçam responsivos às ações do usuário.</span><span class="sxs-lookup"><span data-stu-id="66eb0-115">Worker threads are also used to perform "background" tasks in desktop applications so that the main thread--which drives user interface elements--remains responsive to user actions.</span></span>  
  
 <span data-ttu-id="66eb0-116">O threading resolve problemas com taxa de transferência e capacidade de resposta, mas ele também pode introduzir problemas de compartilhamento de recursos, como deadlocks e condições de corrida.</span><span class="sxs-lookup"><span data-stu-id="66eb0-116">Threading solves problems with throughput and responsiveness, but it can also introduce resource-sharing issues such as deadlocks and race conditions.</span></span> <span data-ttu-id="66eb0-117">Vários threads são melhores para tarefas que exigem recursos diferentes, como identificadores de arquivos e conexões de rede.</span><span class="sxs-lookup"><span data-stu-id="66eb0-117">Multiple threads are best for tasks that require different resources such as file handles and network connections.</span></span> <span data-ttu-id="66eb0-118">A atribuição de vários threads a um único recurso pode causar problemas de sincronização e ter threads bloqueados com frequência ao aguardar outros threads frustra o propósito do uso de vários threads.</span><span class="sxs-lookup"><span data-stu-id="66eb0-118">Assigning multiple threads to a single resource is likely to cause synchronization issues, and having threads frequently blocked when waiting for other threads defeats the purpose of using multiple threads.</span></span>  
  
 <span data-ttu-id="66eb0-119">Uma estratégia comum é usar threads de trabalho para executar tarefas demoradas ou tarefas de tempo crítico que não exigem muitos dos recursos usados por outros threads.</span><span class="sxs-lookup"><span data-stu-id="66eb0-119">A common strategy is to use worker threads to perform time-consuming or time-critical tasks that do not require many of the resources used by other threads.</span></span> <span data-ttu-id="66eb0-120">Naturalmente, alguns recursos em seu programa devem ser acessados por vários threads.</span><span class="sxs-lookup"><span data-stu-id="66eb0-120">Naturally, some resources in your program must be accessed by multiple threads.</span></span> <span data-ttu-id="66eb0-121">Nesses casos, o namespace <xref:System.Threading> fornece classes para sincronização de threads.</span><span class="sxs-lookup"><span data-stu-id="66eb0-121">For these cases, the <xref:System.Threading> namespace provides classes for synchronizing threads.</span></span> <span data-ttu-id="66eb0-122">Essas classes incluem <xref:System.Threading.Mutex>, <xref:System.Threading.Monitor>, <xref:System.Threading.Interlocked>, <xref:System.Threading.AutoResetEvent> e <xref:System.Threading.ManualResetEvent>.</span><span class="sxs-lookup"><span data-stu-id="66eb0-122">These classes include <xref:System.Threading.Mutex>, <xref:System.Threading.Monitor>, <xref:System.Threading.Interlocked>, <xref:System.Threading.AutoResetEvent>, and <xref:System.Threading.ManualResetEvent>.</span></span>  
  
 <span data-ttu-id="66eb0-123">Você pode usar algumas ou todas essas classes para sincronizar as atividades de vários threads, mas o suporte para threading é dado pela linguagem do Visual Basic.</span><span class="sxs-lookup"><span data-stu-id="66eb0-123">You can use some or all these classes to synchronize the activities of multiple threads, but some support for threading is supported by the Visual Basic language.</span></span> <span data-ttu-id="66eb0-124">Por exemplo, a [Instrução SyncLock](../../../../visual-basic/language-reference/statements/synclock-statement.md) fornece funcionalidades de sincronização por meio do uso implícito de <xref:System.Threading.Monitor>.</span><span class="sxs-lookup"><span data-stu-id="66eb0-124">For example, the [SyncLock Statement](../../../../visual-basic/language-reference/statements/synclock-statement.md) provides synchronization features through implicit use of <xref:System.Threading.Monitor>.</span></span>
