# <a name="type-argument-inference-failed-for-type-parameter-39lttypeparametername1gt39-of-39ltgenericproceduresignaturegt39"></a><span data-ttu-id="8f1c2-102">Inferência de argumento fracassada para o parâmetro de tipo '&lt;typeparametername1&gt;'de'&lt;genericproceduresignature&gt;'</span><span class="sxs-lookup"><span data-stu-id="8f1c2-102">Type argument inference failed for type parameter &#39;&lt;typeparametername1&gt;&#39; of &#39;&lt;genericproceduresignature&gt;&#39;</span></span>
<span data-ttu-id="8f1c2-103">Inferência de argumento fracassada para o parâmetro de tipo '\<typeparametername1 >' de '\<genericproceduresignature >'.</span><span class="sxs-lookup"><span data-stu-id="8f1c2-103">Type argument inference failed for type parameter '\<typeparametername1>' of '\<genericproceduresignature>'.</span></span> <span data-ttu-id="8f1c2-104">Digite argumento inferido do argumento passado ao parâmetro '\<parametername1 >' está em conflito com o argumento de tipo inferido do argumento passado ao parâmetro '\<parametername2 >'.</span><span class="sxs-lookup"><span data-stu-id="8f1c2-104">Type argument inferred from the argument passed to parameter '\<parametername1>' conflicts with the type argument inferred from the argument passed to parameter '\<parametername2>'.</span></span>  
  
 <span data-ttu-id="8f1c2-105">Um procedimento genérico é chamado sem nenhum argumento de tipo, e a inferência de tipo tentadas produziram um conflito de tipo de dados entre os parâmetros de tipo.</span><span class="sxs-lookup"><span data-stu-id="8f1c2-105">A generic procedure is called without any type arguments, and the attempted type inference has produced a data type conflict among the type parameters.</span></span>  
  
 <span data-ttu-id="8f1c2-106">Normalmente, quando você chama um procedimento genérico, você fornece um argumento de tipo para cada parâmetro de tipo que define o procedimento genérico.</span><span class="sxs-lookup"><span data-stu-id="8f1c2-106">Normally, when you call a generic procedure, you supply a type argument for each type parameter that the generic procedure defines.</span></span> <span data-ttu-id="8f1c2-107">Se você não fornecer nenhum argumento de tipo, o compilador tenta inferir os tipos a serem passados para os parâmetros de tipo.</span><span class="sxs-lookup"><span data-stu-id="8f1c2-107">If you do not supply any type arguments, then the compiler attempts to infer the types to be passed to the type parameters.</span></span> <span data-ttu-id="8f1c2-108">Se o contexto da chamada fornece conflitantes informações tipo de dados para um parâmetro de tipo, a inferência de tipo falhará.</span><span class="sxs-lookup"><span data-stu-id="8f1c2-108">If the context of the call provides conflicting data type information for a type parameter, then the type inference fails.</span></span>  
  
 <span data-ttu-id="8f1c2-109">O código a seguir pode gerar esse erro.</span><span class="sxs-lookup"><span data-stu-id="8f1c2-109">The following code can generate this error.</span></span>  
  
```  
Public Sub takeTwoValues(Of t)(ByVal x As t, ByVal y As t)  
End Sub  
Call takeTwoValues(4, 2.5)  
```  
  
 <span data-ttu-id="8f1c2-110">Como o primeiro argumento faz com que o compilador inferir `Integer` para o parâmetro de tipo `t`, enquanto o segundo argumento o leva a inferir `Double` para o mesmo parâmetro de tipo, há um conflito sobre qual tipo de dados para passar para `t`.</span><span class="sxs-lookup"><span data-stu-id="8f1c2-110">Because the first argument causes the compiler to infer `Integer` for type parameter `t`, while the second argument causes it to infer `Double` for the same type parameter, there is a conflict regarding which data type to pass to `t`.</span></span>  
  
 <span data-ttu-id="8f1c2-111">**ID do erro:** BC32116</span><span class="sxs-lookup"><span data-stu-id="8f1c2-111">**Error ID:** BC32116</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="8f1c2-112">Para corrigir este erro</span><span class="sxs-lookup"><span data-stu-id="8f1c2-112">To correct this error</span></span>  
  
-   <span data-ttu-id="8f1c2-113">Forneça os argumentos de tipo para o tipo genérico, de modo que o compilador não precise inferi-los.</span><span class="sxs-lookup"><span data-stu-id="8f1c2-113">Supply type arguments to the generic type, so that the compiler does not have to infer them.</span></span>  
  
    ```  
    Call takeTwoValues(Of Double)(4, 2.5)  
    ```  
  
     <span data-ttu-id="8f1c2-114">Observe que neste caso, quando dois argumentos normais são tipos de dados diferentes, o código de chamada deve passar um argumento de tipo que possa acomodar ambos os tipos de dados.</span><span class="sxs-lookup"><span data-stu-id="8f1c2-114">Notice that in this case, where the two normal arguments are of different data types, the calling code must pass a type argument that can accommodate both of those data types.</span></span> <span data-ttu-id="8f1c2-115">Nesse caso, `Integer` amplia a `Double`.</span><span class="sxs-lookup"><span data-stu-id="8f1c2-115">In this case, `Integer` widens to `Double`.</span></span>  
  
     <span data-ttu-id="8f1c2-116">-ou-</span><span class="sxs-lookup"><span data-stu-id="8f1c2-116">-or-</span></span>  
  
-   <span data-ttu-id="8f1c2-117">Redefina o procedimento genérico para especificar parâmetros de tipo diferente para os parâmetros normais, para que não haja conflitos inferindo os tipos.</span><span class="sxs-lookup"><span data-stu-id="8f1c2-117">Redefine the generic procedure to specify different type parameters for the normal parameters, so that there is no conflict inferring the types.</span></span>  
  
    ```  
    Public Sub takeTwoValues(Of t1, t2)(ByVal x As t1, ByVal y As t2)  
    ```
