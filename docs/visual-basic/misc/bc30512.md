# <a name="option-strict-on-disallows-implicit-conversions-from-39lttype1gt39-to-39lttype2gt39"></a><span data-ttu-id="2804b-102">Option Strict On não permite conversões implícitas de '&lt;type1&gt;'para'&lt;type2&gt;'</span><span class="sxs-lookup"><span data-stu-id="2804b-102">Option Strict On disallows implicit conversions from &#39;&lt;type1&gt;&#39; to &#39;&lt;type2&gt;&#39;</span></span>
<span data-ttu-id="2804b-103">Você tentou converter um tipo em outro tipo que não poderá conter o valor, como um `Long` para um `Integer`, enquanto a opção de verificação de tipo ([instrução Option Strict](../../visual-basic/language-reference/statements/option-strict-statement.md)) é definido como `On`.</span><span class="sxs-lookup"><span data-stu-id="2804b-103">You have tried to convert a type to another type that may not be able to contain the value, such as a `Long` to an `Integer`, while the type checking switch ([Option Strict Statement](../../visual-basic/language-reference/statements/option-strict-statement.md)) is set to `On`.</span></span>  
  
 <span data-ttu-id="2804b-104">Esse tipo de conversão é chamado um *conversão de redução*, e é possível que ele falhe em tempo de execução.</span><span class="sxs-lookup"><span data-stu-id="2804b-104">This type of conversion is called a *narrowing conversion*, and it is possible for it to fail at run time.</span></span> <span data-ttu-id="2804b-105">Por esse motivo, `Option Strict On` não permite conversões de estreitamento implícitas.</span><span class="sxs-lookup"><span data-stu-id="2804b-105">For this reason, `Option Strict On` disallows implicit narrowing conversions.</span></span>  
  
 <span data-ttu-id="2804b-106">**ID do erro:** BC30512</span><span class="sxs-lookup"><span data-stu-id="2804b-106">**Error ID:** BC30512</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="2804b-107">Para corrigir este erro</span><span class="sxs-lookup"><span data-stu-id="2804b-107">To correct this error</span></span>  
  
1.  <span data-ttu-id="2804b-108">Determinar se há uma conversão de qualquer tipo do `<type1>` para `<type2>`.</span><span class="sxs-lookup"><span data-stu-id="2804b-108">Determine whether a conversion of any type exists from `<type1>` to `<type2>`.</span></span> <span data-ttu-id="2804b-109">Se ambos estiverem [!INCLUDE[vbprvb](../../csharp/programming-guide/concepts/linq/includes/vbprvb_md.md)] tipos elementares, ou se ambos forem instâncias de classes, você geralmente pode fazer essa determinação consultando a tabela em [Widening e conversões de estreitamento](../../visual-basic/programming-guide/language-features/data-types/widening-and-narrowing-conversions.md).</span><span class="sxs-lookup"><span data-stu-id="2804b-109">If both are [!INCLUDE[vbprvb](../../csharp/programming-guide/concepts/linq/includes/vbprvb_md.md)] elementary types, or if both are instances of classes, you can usually make this determination by consulting the table in [Widening and Narrowing Conversions](../../visual-basic/programming-guide/language-features/data-types/widening-and-narrowing-conversions.md).</span></span>  
  
2.  <span data-ttu-id="2804b-110">Se houver uma conversão de redução de apenas `<type1>` para `<type2>`, você deve usar a conversão explícita.</span><span class="sxs-lookup"><span data-stu-id="2804b-110">If only a narrowing conversion exists from `<type1>` to `<type2>`, you should use explicit casting.</span></span> <span data-ttu-id="2804b-111">O [função CType](../../visual-basic/language-reference/functions/ctype-function.md) e [operador DirectCast](../../visual-basic/language-reference/operators/directcast-operator.md) palavras-chave acionar uma exceção de tempo de execução se a conversão falhar.</span><span class="sxs-lookup"><span data-stu-id="2804b-111">The [CType Function](../../visual-basic/language-reference/functions/ctype-function.md) and [DirectCast Operator](../../visual-basic/language-reference/operators/directcast-operator.md) keywords throw a run-time exception if the conversion fails.</span></span> <span data-ttu-id="2804b-112">O [operador TryCast](../../visual-basic/language-reference/operators/trycast-operator.md) palavra-chave se aplica somente aos tipos de referência e retorna [nada](../../visual-basic/language-reference/nothing.md) se a conversão falhar.</span><span class="sxs-lookup"><span data-stu-id="2804b-112">The [TryCast Operator](../../visual-basic/language-reference/operators/trycast-operator.md) keyword applies only to reference types and returns [Nothing](../../visual-basic/language-reference/nothing.md) if the conversion fails.</span></span>  
  
3.  <span data-ttu-id="2804b-113">Se existe uma conversão de restrição e seu programa pode tolerar uma falha de tempo de execução, ou você tiver certeza de que uma falha de tempo de execução não é possível, você pode especificar `Option Strict Off` no início do seu código-fonte.</span><span class="sxs-lookup"><span data-stu-id="2804b-113">If a narrowing conversion exists and your program can tolerate a run-time failure, or you are confident that a run-time failure is not possible, you can specify `Option Strict Off` at the beginning of your source code.</span></span> <span data-ttu-id="2804b-114">Mas você ainda deve colocar a conversão em um [Try... Catch... Instrução Finally](../../visual-basic/language-reference/statements/try-catch-finally-statement.md) bloco para evitar resultados inesperados ou encerramento do programa.</span><span class="sxs-lookup"><span data-stu-id="2804b-114">But you should still enclose the conversion in a [Try...Catch...Finally Statement](../../visual-basic/language-reference/statements/try-catch-finally-statement.md) block to avoid unexpected results or early termination of your program.</span></span>  
  
4.  <span data-ttu-id="2804b-115">Não se existir nenhuma conversão de `<type1>` para `<type2>`, você deve reavaliar sua lógica do programa.</span><span class="sxs-lookup"><span data-stu-id="2804b-115">If no conversion exists from `<type1>` to `<type2>`, you must re-evaluate your program logic.</span></span> <span data-ttu-id="2804b-116">Você poderá escrever código que pode atribuir valores a `<type2>` correspondente previsto valores de `<type1>`.</span><span class="sxs-lookup"><span data-stu-id="2804b-116">You might be able to write code that can assign values to `<type2>` corresponding to anticipated values of `<type1>`.</span></span>  
  
5.  <span data-ttu-id="2804b-117">Não se existir nenhuma conversão de `<type1>` para `<type2>` e um dos tipos é uma classe ou estrutura que você definiu, você poderá definir um operador de conversão de tipo para ou de outro tipo.</span><span class="sxs-lookup"><span data-stu-id="2804b-117">If no conversion exists from `<type1>` to `<type2>` and one of the types is a class or structure you have defined, you might be able to define a conversion operator from that type to or from the other type.</span></span> <span data-ttu-id="2804b-118">Para obter mais informações, consulte [como: definir um operador de conversão](../../visual-basic/programming-guide/language-features/procedures/how-to-define-a-conversion-operator.md).</span><span class="sxs-lookup"><span data-stu-id="2804b-118">For more information, see [How to: Define a Conversion Operator](../../visual-basic/programming-guide/language-features/procedures/how-to-define-a-conversion-operator.md).</span></span>  
  
6.  <span data-ttu-id="2804b-119">Em todos os casos e como diretriz geral, você deve evitar usar conversões de restrição, a menos que seja possível detectar falhas em um `Catch` bloquear e lidar com eles de forma eficiente.</span><span class="sxs-lookup"><span data-stu-id="2804b-119">In all cases and as a general guideline, you should avoid using narrowing conversions unless you can trap failures in a `Catch` block and deal with them effectively.</span></span>
