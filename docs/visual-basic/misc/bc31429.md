<span data-ttu-id="31924-103">Uma expressão acessa um elemento de programação definido em uma classe, estrutura, módulo ou interface que contém mais de um membro com o mesmo nome.</span><span class="sxs-lookup"><span data-stu-id="31924-103">An expression accesses a programming element defined in a class, structure, module, or interface that contains more than one member with the same name.</span></span>  
  
 <span data-ttu-id="31924-104">A causa mais provável desse erro é *de maiusculas e minúsculas*.</span><span class="sxs-lookup"><span data-stu-id="31924-104">The most likely cause of this error is *case sensitivity*.</span></span> <span data-ttu-id="31924-105">Nomes de Visual Basic diferenciam maiusculas de minúsculas, o que significa que você pode colocá-las diferente em diferentes locais em seu código.</span><span class="sxs-lookup"><span data-stu-id="31924-105">Visual Basic names are case-insensitive, which means you can capitalize them differently at different places in your code.</span></span> <span data-ttu-id="31924-106">Por exemplo, se você definir uma variável com o nome `XYZ` e posteriormente acessá-lo como `xyz`, o compilador considera os dois nomes sejam equivalentes.</span><span class="sxs-lookup"><span data-stu-id="31924-106">For example, if you define a variable with the name `XYZ` and later access it as `xyz`, the compiler considers the two names to be equivalent.</span></span>
