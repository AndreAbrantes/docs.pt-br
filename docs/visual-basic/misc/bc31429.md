 <span data-ttu-id="a9b7b-104">A causa mais provável desse erro é *diferenciação*.</span><span class="sxs-lookup"><span data-stu-id="a9b7b-104">The most likely cause of this error is *case sensitivity*.</span></span> <span data-ttu-id="a9b7b-105">Nomes de Visual Basic diferenciam maiusculas de minúsculas, o que significa que você pode colocá-las diferentemente em diferentes locais em seu código.</span><span class="sxs-lookup"><span data-stu-id="a9b7b-105">Visual Basic names are case-insensitive, which means you can capitalize them differently at different places in your code.</span></span> <span data-ttu-id="a9b7b-106">Por exemplo, se você definir uma variável com o nome `XYZ` e posterior acessá-lo como `xyz`, o compilador considera os dois nomes serem equivalentes.</span><span class="sxs-lookup"><span data-stu-id="a9b7b-106">For example, if you define a variable with the name `XYZ` and later access it as `xyz`, the compiler considers the two names to be equivalent.</span></span>  
  
 <span data-ttu-id="a9b7b-107">No entanto, outras linguagens, como [ C# ](../../csharp/index.md) e [Visual C++](/cpp/index), diferenciam maiusculas de minúsculas.</span><span class="sxs-lookup"><span data-stu-id="a9b7b-107">However, other languages, such as [C#](../../csharp/index.md) and [Visual C++](/cpp/index), are case-sensitive.</span></span> <span data-ttu-id="a9b7b-108">Na linguagem, `XYZ` e `xyz` não são considerados para ser o mesmo nome.</span><span class="sxs-lookup"><span data-stu-id="a9b7b-108">In such a language, `XYZ` and `xyz` are not considered to be the same name.</span></span> <span data-ttu-id="a9b7b-109">Portanto, uma classe escrita em uma linguagem tal poderia definir uma variável denominada `XYZ` e uma propriedade chamada `xyz`.</span><span class="sxs-lookup"><span data-stu-id="a9b7b-109">Therefore, a class written in such a language could define a variable named `XYZ` and a property named `xyz`.</span></span> <span data-ttu-id="a9b7b-110">O common language runtime (CLR) preserva a diferenciação de maiusculas em assemblies.</span><span class="sxs-lookup"><span data-stu-id="a9b7b-110">The common language runtime (CLR) preserves case sensitivity in assemblies.</span></span> <span data-ttu-id="a9b7b-111">No entanto, se um aplicativo Visual Basic acessa um assembly com nomes `XYZ` e `xyz`, eles são exibidos como o mesmo nome.</span><span class="sxs-lookup"><span data-stu-id="a9b7b-111">However, if a Visual Basic application accesses an assembly with names `XYZ` and `xyz`, they appear as the same name.</span></span>
