 <span data-ttu-id="55f02-104">A causa mais provável desse erro é a *diferenciação de maiúsculas e minúsculas*.</span><span class="sxs-lookup"><span data-stu-id="55f02-104">The most likely cause of this error is *case sensitivity*.</span></span> <span data-ttu-id="55f02-105">Os nomes de Visual Basic não diferenciam maiúsculas de minúsculas, o que significa que você pode capitalizar deles de forma diferente em locais diferentes em seu código.</span><span class="sxs-lookup"><span data-stu-id="55f02-105">Visual Basic names are case-insensitive, which means you can capitalize them differently at different places in your code.</span></span> <span data-ttu-id="55f02-106">Por exemplo, se você definir uma variável com o nome `XYZ` e depois acessá-la como `xyz`, o compilador considerará que os dois nomes são equivalentes.</span><span class="sxs-lookup"><span data-stu-id="55f02-106">For example, if you define a variable with the name `XYZ` and later access it as `xyz`, the compiler considers the two names to be equivalent.</span></span>  
  
 <span data-ttu-id="55f02-107">No entanto, outras linguagens [C#](../../csharp/index.yml) , [como C++Visual ](/cpp/index), diferenciam maiúsculas de minúsculas.</span><span class="sxs-lookup"><span data-stu-id="55f02-107">However, other languages, such as [C#](../../csharp/index.yml) and [Visual C++](/cpp/index), are case-sensitive.</span></span> <span data-ttu-id="55f02-108">Nesse idioma, `XYZ` e `xyz` não são considerados o mesmo nome.</span><span class="sxs-lookup"><span data-stu-id="55f02-108">In such a language, `XYZ` and `xyz` are not considered to be the same name.</span></span> <span data-ttu-id="55f02-109">Portanto, uma classe escrita em tal linguagem poderia definir uma variável chamada `XYZ` e uma propriedade chamada `xyz`.</span><span class="sxs-lookup"><span data-stu-id="55f02-109">Therefore, a class written in such a language could define a variable named `XYZ` and a property named `xyz`.</span></span> <span data-ttu-id="55f02-110">O Common Language Runtime (CLR) preserva a diferenciação de maiúsculas e minúsculas nos assemblies.</span><span class="sxs-lookup"><span data-stu-id="55f02-110">The common language runtime (CLR) preserves case sensitivity in assemblies.</span></span> <span data-ttu-id="55f02-111">No entanto, se um aplicativo Visual Basic acessar um assembly com nomes `XYZ` e `xyz`, eles aparecerão como o mesmo nome.</span><span class="sxs-lookup"><span data-stu-id="55f02-111">However, if a Visual Basic application accesses an assembly with names `XYZ` and `xyz`, they appear as the same name.</span></span>
