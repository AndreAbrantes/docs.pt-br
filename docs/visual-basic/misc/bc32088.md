# <a name="type-arguments-unexpected"></a><span data-ttu-id="b6115-102">Argumentos de tipo inesperado</span><span class="sxs-lookup"><span data-stu-id="b6115-102">Type arguments unexpected</span></span>
<span data-ttu-id="b6115-103">Um `Implements` cláusula fornece argumentos de tipo para o membro de interface que está implementando.</span><span class="sxs-lookup"><span data-stu-id="b6115-103">An `Implements` clause supplies type arguments for the interface member it is implementing.</span></span>  
  
 <span data-ttu-id="b6115-104">O `Implements` cláusula só deve identificar a interface e o membro que está implementando.</span><span class="sxs-lookup"><span data-stu-id="b6115-104">The `Implements` clause should only identify the interface and member it is implementing.</span></span> <span data-ttu-id="b6115-105">Isso significa que, se você está declarando um procedimento genérico, o `Of` cláusula e os argumentos de tipo devem aparecer na parte principal da declaração, como fariam se você não fosse implementar um procedimento de interface.</span><span class="sxs-lookup"><span data-stu-id="b6115-105">This means that if you are declaring a generic procedure, the `Of` clause and the type arguments should appear in the main part of the declaration, just as they would if you were not implementing an interface procedure.</span></span>  
  
 <span data-ttu-id="b6115-106">O código a seguir pode gerar esse erro.</span><span class="sxs-lookup"><span data-stu-id="b6115-106">The following code can generate this error.</span></span>  
  
```  
Public Interface testInterface  
    Sub testSub(Of t)()  
End Interface  
Public Class testClass  
    Implements testInterface  
    Public Sub testSub() Implements testInterface.testSub(Of t)()  
    End Sub  
End Class  
```  
  
 <span data-ttu-id="b6115-107">A declaração anterior de `Implements` cláusula deve parecer com a definição de interface, exceto para a possível adição de modificadores de acesso ou procedimento.</span><span class="sxs-lookup"><span data-stu-id="b6115-107">The declaration preceding the `Implements` clause should look like the interface definition, except for the possible addition of access or procedure modifiers.</span></span> <span data-ttu-id="b6115-108">O código a seguir evita o erro.</span><span class="sxs-lookup"><span data-stu-id="b6115-108">The following code avoids the error.</span></span>  
  
```  
Public Sub testSub(Of t)() Implements testInterface.testSub  
```  
  
 <span data-ttu-id="b6115-109">**ID do erro:** BC32088</span><span class="sxs-lookup"><span data-stu-id="b6115-109">**Error ID:** BC32088</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="b6115-110">Para corrigir este erro</span><span class="sxs-lookup"><span data-stu-id="b6115-110">To correct this error</span></span>  
  
-   <span data-ttu-id="b6115-111">Remover a lista de argumentos de tipo de `Implements` cláusula.</span><span class="sxs-lookup"><span data-stu-id="b6115-111">Remove the type argument list from the `Implements` clause.</span></span>  
  
-   <span data-ttu-id="b6115-112">Se você estiver implementando um membro genérico de interface, então coloque a lista de argumentos de tipo na parte principal da declaração, anterior a `Implements` palavra-chave.</span><span class="sxs-lookup"><span data-stu-id="b6115-112">If you are implementing a generic member of the interface, then put the type argument list in the main part of the declaration, preceding the `Implements` keyword.</span></span>
