# <a name="type-arguments-unexpected"></a><span data-ttu-id="37639-102">Argumentos de tipo inesperados</span><span class="sxs-lookup"><span data-stu-id="37639-102">Type arguments unexpected</span></span>
<span data-ttu-id="37639-103">Uma cláusula `Implements` fornece argumentos de tipo para o membro de interface que está implementando.</span><span class="sxs-lookup"><span data-stu-id="37639-103">An `Implements` clause supplies type arguments for the interface member it is implementing.</span></span>  
  
 <span data-ttu-id="37639-104">A cláusula `Implements` só deve identificar a interface e o membro que está implementando.</span><span class="sxs-lookup"><span data-stu-id="37639-104">The `Implements` clause should only identify the interface and member it is implementing.</span></span> <span data-ttu-id="37639-105">Isso significa que se você estiver declarando um procedimento genérico, a cláusula `Of` e os argumentos de tipo deverão aparecer na parte principal da declaração, assim como fariam se você não estivesse implementando um procedimento de interface.</span><span class="sxs-lookup"><span data-stu-id="37639-105">This means that if you are declaring a generic procedure, the `Of` clause and the type arguments should appear in the main part of the declaration, just as they would if you were not implementing an interface procedure.</span></span>  
  
 <span data-ttu-id="37639-106">O código a seguir pode gerar esse erro.</span><span class="sxs-lookup"><span data-stu-id="37639-106">The following code can generate this error.</span></span>  
  
```vb  
Public Interface testInterface  
    Sub testSub(Of t)()  
End Interface  
Public Class testClass  
    Implements testInterface  
    Public Sub testSub() Implements testInterface.testSub(Of t)()  
    End Sub  
End Class  
```  
  
 <span data-ttu-id="37639-107">A declaração que precede a cláusula `Implements` deve ser parecida com a definição da interface, exceto para a possível adição de modificadores de acesso ou de procedimento.</span><span class="sxs-lookup"><span data-stu-id="37639-107">The declaration preceding the `Implements` clause should look like the interface definition, except for the possible addition of access or procedure modifiers.</span></span> <span data-ttu-id="37639-108">O código a seguir evita o erro.</span><span class="sxs-lookup"><span data-stu-id="37639-108">The following code avoids the error.</span></span>  
  
```vb  
Public Sub testSub(Of t)() Implements testInterface.testSub  
```  
  
 <span data-ttu-id="37639-109">**ID do erro:** BC32088</span><span class="sxs-lookup"><span data-stu-id="37639-109">**Error ID:** BC32088</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="37639-110">Para corrigir este erro</span><span class="sxs-lookup"><span data-stu-id="37639-110">To correct this error</span></span>  
  
- <span data-ttu-id="37639-111">Remova a lista de argumentos de tipo da cláusula `Implements`.</span><span class="sxs-lookup"><span data-stu-id="37639-111">Remove the type argument list from the `Implements` clause.</span></span>  
  
- <span data-ttu-id="37639-112">Se você estiver implementando um membro genérico da interface, coloque a lista de argumentos de tipo na parte principal da declaração, precedendo a palavra-chave `Implements`.</span><span class="sxs-lookup"><span data-stu-id="37639-112">If you are implementing a generic member of the interface, then put the type argument list in the main part of the declaration, preceding the `Implements` keyword.</span></span>
