# <a name="class-39ltclassnamegt39-has-no-accessible-39sub-new39-and-cannot-be-inherited"></a><span data-ttu-id="32a37-102">Classe &#39; &lt;classname&gt;&#39; não tem nenhum &#39; Sub novo &#39; e não pode ser herdada</span><span class="sxs-lookup"><span data-stu-id="32a37-102">Class &#39;&lt;classname&gt;&#39; has no accessible &#39;Sub New&#39; and cannot be inherited</span></span>
<span data-ttu-id="32a37-103">Uma classe usa uma [herda instrução](../../visual-basic/language-reference/statements/inherits-statement.md) para especificar uma base de dados de classe, mas não pode acessar nenhum construtor na classe base.</span><span class="sxs-lookup"><span data-stu-id="32a37-103">A class uses an [Inherits Statement](../../visual-basic/language-reference/statements/inherits-statement.md) to specify a base class, but it cannot access any constructor on the intended base class.</span></span>  
  
 <span data-ttu-id="32a37-104">Isso pode acontecer se a classe base não tem nenhum construtor ou se ele tiver construtor com níveis de acesso que impeçam o acesso de outra classe.</span><span class="sxs-lookup"><span data-stu-id="32a37-104">This can happen if the intended base class has no constructors or if it has constructors with access levels that prevent access from another class.</span></span>  
  
 <span data-ttu-id="32a37-105">Quando você herda uma classe, o construtor deve chamar o construtor da classe base usando [MyBase](~/docs/visual-basic/programming-guide/program-structure/me-my-mybase-and-myclass.md#mybase).</span><span class="sxs-lookup"><span data-stu-id="32a37-105">When you inherit a class, your constructor should call the base class constructor using [MyBase](~/docs/visual-basic/programming-guide/program-structure/me-my-mybase-and-myclass.md#mybase).</span></span> <span data-ttu-id="32a37-106">Se você não fizer essa chamada, ou se você nem mesmo gravar um construtor explícito, Visual Basic gera um construtor implícito que chama `MyBase.New()`.</span><span class="sxs-lookup"><span data-stu-id="32a37-106">If you do not make this call, or if you do not even write an explicit constructor, Visual Basic generates an implicit constructor that calls `MyBase.New()`.</span></span>  
  
 <span data-ttu-id="32a37-107">**ID do erro:** BC31399</span><span class="sxs-lookup"><span data-stu-id="32a37-107">**Error ID:** BC31399</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="32a37-108">Para corrigir este erro</span><span class="sxs-lookup"><span data-stu-id="32a37-108">To correct this error</span></span>  
  
1.  <span data-ttu-id="32a37-109">Se você tiver o controle de origem sobre a classe base, altere o nível de acesso de pelo menos um dos seus construtores para que outra classe possa acessá-los.</span><span class="sxs-lookup"><span data-stu-id="32a37-109">If you have source control over the intended base class, then change the access level of at least one of its constructors so that another class can access them.</span></span>
