# <a name="class-classname-has-no-accessible-sub-new-and-cannot-be-inherited"></a><span data-ttu-id="62a22-102">Classe\<classname >' não tem nenhum 'Sub New' acessível e não pode ser herdada</span><span class="sxs-lookup"><span data-stu-id="62a22-102">Class '\<classname>' has no accessible 'Sub New' and cannot be inherited</span></span>
<span data-ttu-id="62a22-103">Uma classe usa um [instrução Inherits](../../visual-basic/language-reference/statements/inherits-statement.md) para especificar uma base de dados de classe, mas ele não pode acessar nenhum construtor na classe base.</span><span class="sxs-lookup"><span data-stu-id="62a22-103">A class uses an [Inherits Statement](../../visual-basic/language-reference/statements/inherits-statement.md) to specify a base class, but it cannot access any constructor on the intended base class.</span></span>  
  
 <span data-ttu-id="62a22-104">Isso pode acontecer se a classe base não tem nenhum construtor ou se ele tem construtores com níveis de acesso que impedem o acesso de outra classe.</span><span class="sxs-lookup"><span data-stu-id="62a22-104">This can happen if the intended base class has no constructors or if it has constructors with access levels that prevent access from another class.</span></span>  
  
 <span data-ttu-id="62a22-105">Quando você herdar uma classe, o construtor deve chamar o construtor da classe base usando [MyBase](~/docs/visual-basic/programming-guide/program-structure/me-my-mybase-and-myclass.md#mybase).</span><span class="sxs-lookup"><span data-stu-id="62a22-105">When you inherit a class, your constructor should call the base class constructor using [MyBase](~/docs/visual-basic/programming-guide/program-structure/me-my-mybase-and-myclass.md#mybase).</span></span> <span data-ttu-id="62a22-106">Se você não fizer essa chamada, ou se você nem mesmo gravar um construtor explícito, Visual Basic gera um construtor implícito que chama `MyBase.New()`.</span><span class="sxs-lookup"><span data-stu-id="62a22-106">If you do not make this call, or if you do not even write an explicit constructor, Visual Basic generates an implicit constructor that calls `MyBase.New()`.</span></span>  
  
 <span data-ttu-id="62a22-107">**ID do erro:** BC31399</span><span class="sxs-lookup"><span data-stu-id="62a22-107">**Error ID:** BC31399</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="62a22-108">Para corrigir este erro</span><span class="sxs-lookup"><span data-stu-id="62a22-108">To correct this error</span></span>  
  
1.  <span data-ttu-id="62a22-109">Se você tiver o controle do código-fonte sobre a classe base, em seguida, altere o nível de acesso de pelo menos um de seus construtores para que outra classe possa acessá-los.</span><span class="sxs-lookup"><span data-stu-id="62a22-109">If you have source control over the intended base class, then change the access level of at least one of its constructors so that another class can access them.</span></span>  
  
2.  <span data-ttu-id="62a22-110">Se você não pode alterar os níveis de acesso dos construtores de classe base, herde de uma classe diferente ou nenhum.</span><span class="sxs-lookup"><span data-stu-id="62a22-110">If you cannot change the access levels of the intended base class constructors, then inherit from a different class or not at all.</span></span>
