# <a name="class-39ltclassnamegt39-has-no-accessible-39sub-new39-and-cannot-be-inherited"></a><span data-ttu-id="95037-102">Classe&lt;classname&gt;' não tem nenhum 'Sub New' acessível e não pode ser herdada</span><span class="sxs-lookup"><span data-stu-id="95037-102">Class &#39;&lt;classname&gt;&#39; has no accessible &#39;Sub New&#39; and cannot be inherited</span></span>
<span data-ttu-id="95037-103">Uma classe usa uma [instrução Inherits](../../visual-basic/language-reference/statements/inherits-statement.md) para especificar uma base de dados de classe, mas não pode acessar nenhum construtor na classe base pretendida.</span><span class="sxs-lookup"><span data-stu-id="95037-103">A class uses an [Inherits Statement](../../visual-basic/language-reference/statements/inherits-statement.md) to specify a base class, but it cannot access any constructor on the intended base class.</span></span>  
  
 <span data-ttu-id="95037-104">Isso pode acontecer se a classe base não tem nenhum construtor ou se ele tiver construtor com níveis de acesso que impeçam o acesso de outra classe.</span><span class="sxs-lookup"><span data-stu-id="95037-104">This can happen if the intended base class has no constructors or if it has constructors with access levels that prevent access from another class.</span></span>  
  
 <span data-ttu-id="95037-105">Quando você herdar uma classe, o construtor deve chamar o construtor da classe base usando [MyBase - excluir](http://msdn.microsoft.com/en-us/52491d06-6451-4f6f-9aa6-8fab59bbc2b9).</span><span class="sxs-lookup"><span data-stu-id="95037-105">When you inherit a class, your constructor should call the base class constructor using [MyBase - delete](http://msdn.microsoft.com/en-us/52491d06-6451-4f6f-9aa6-8fab59bbc2b9).</span></span> <span data-ttu-id="95037-106">Se você não fizer essa chamada, ou se você nem mesmo gravar um construtor explícito, Visual Basic gera um construtor implícito que chama `MyBase.New()`.</span><span class="sxs-lookup"><span data-stu-id="95037-106">If you do not make this call, or if you do not even write an explicit constructor, Visual Basic generates an implicit constructor that calls `MyBase.New()`.</span></span>  
  
 <span data-ttu-id="95037-107">**ID do erro:** BC31399</span><span class="sxs-lookup"><span data-stu-id="95037-107">**Error ID:** BC31399</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="95037-108">Para corrigir este erro</span><span class="sxs-lookup"><span data-stu-id="95037-108">To correct this error</span></span>  
  
1.  <span data-ttu-id="95037-109">Se você tiver o controle de origem sobre a classe base, altere o nível de acesso de pelo menos um dos seus construtores para que outra classe possa acessá-los.</span><span class="sxs-lookup"><span data-stu-id="95037-109">If you have source control over the intended base class, then change the access level of at least one of its constructors so that another class can access them.</span></span>
