# <a name="cannot-implement-interfacename1membername-because-its-implementation-could-conflict-with-the-implementation-for-interfacename2membername-for-some-type-arguments"></a><span data-ttu-id="43d60-102">Não é possível implementar ' \<interfacename1> . \<membername> ' porque sua implementação pode entrar em conflito com a implementação de ' \<interfacename2> . \<membername> ' para alguns argumentos de tipo</span><span class="sxs-lookup"><span data-stu-id="43d60-102">Cannot implement '\<interfacename1>.\<membername>' because its implementation could conflict with the implementation for '\<interfacename2>.\<membername>' for some type arguments</span></span>

<span data-ttu-id="43d60-103">Uma classe implementa mais de uma interface genérica, uma das quais herda de outra, e duas implementações de um membro de interface podem entrar em conflito para determinados valores dos argumentos de tipo.</span><span class="sxs-lookup"><span data-stu-id="43d60-103">A class implements more than one generic interface, one of which inherits from another, and two implementations of an interface member could conflict for certain values of type arguments.</span></span>  
  
 <span data-ttu-id="43d60-104">As instruções a seguir podem gerar esse erro.</span><span class="sxs-lookup"><span data-stu-id="43d60-104">The following statements can generate this error.</span></span>  
  
```vb  
Public Interface iFace1(Of t)  
    Sub testSub()  
End Interface  
Public Interface iFace2(Of u)  
    Inherits iFace1(Of u)  
End Interface  
Public Class testClass(Of y, z)  
    Implements iFace1(Of y), iFace2(Of z)  
    Public Sub testSuby() Implements iFace1(Of y).testSub  
    End Sub  
    Public Sub testSubz() Implements iFace1(Of z).testSub  
    End Sub  
End Class  
```  
  
 <span data-ttu-id="43d60-105">Como `iFace2` herdar de `iFace1` usar seu próprio parâmetro de tipo ( `u` ), o `testClass` implementaria duas versões do `iFace1.testSub` com assinaturas idênticas se o mesmo argumento de tipo fosse passado para `y` e `z` .</span><span class="sxs-lookup"><span data-stu-id="43d60-105">Because `iFace2` inherits from `iFace1` using its own type parameter (`u`), `testClass` would implement two versions of `iFace1.testSub` with identical signatures if the same type argument were passed to `y` and `z`.</span></span> <span data-ttu-id="43d60-106">Isso produziria uma ambiguidade sobre qual versão acessar.</span><span class="sxs-lookup"><span data-stu-id="43d60-106">This would produce an ambiguity about which version to access.</span></span>  
  
 <span data-ttu-id="43d60-107">**ID do erro:** BC32125</span><span class="sxs-lookup"><span data-stu-id="43d60-107">**Error ID:** BC32125</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="43d60-108">Para corrigir este erro</span><span class="sxs-lookup"><span data-stu-id="43d60-108">To correct this error</span></span>  
  
- <span data-ttu-id="43d60-109">Altere a estrutura de herança das interfaces para que `iFace1` não possa ser implementada com dois argumentos de tipo diferentes.</span><span class="sxs-lookup"><span data-stu-id="43d60-109">Change the inheritance structure of the interfaces so that `iFace1` could not be implemented with two different type arguments.</span></span>  
  
     <span data-ttu-id="43d60-110">- ou -</span><span class="sxs-lookup"><span data-stu-id="43d60-110">-or-</span></span>  
  
- <span data-ttu-id="43d60-111">Remova da `Implements` instrução uma das interfaces que resulta no conflito de implementação.</span><span class="sxs-lookup"><span data-stu-id="43d60-111">Remove from the `Implements` statement one of the interfaces resulting in the implementation conflict.</span></span>
