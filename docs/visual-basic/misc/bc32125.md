# <a name="cannot-implement-ltinterfacename1gtltmembernamegt-because-its-implementation-could-conflict-with-the-implementation-for-ltinterfacename2gtltmembernamegt-for-some-type-arguments"></a><span data-ttu-id="1b4cd-102">Não é possível implementar '&lt;interfacename1&gt;.&lt; membername&gt;'porque sua implementação poderia entrar em conflito com a implementação para'&lt;interfacename2&gt;.&lt; membername&gt;' para alguns argumentos de tipo</span><span class="sxs-lookup"><span data-stu-id="1b4cd-102">Cannot implement '&lt;interfacename1&gt;.&lt;membername&gt;' because its implementation could conflict with the implementation for '&lt;interfacename2&gt;.&lt;membername&gt;' for some type arguments</span></span>
<span data-ttu-id="1b4cd-103">Uma classe implementa mais de uma interface genérica, um dos quais herda de outra, e duas implementações de um membro de interface poderia entrar em conflito para determinados valores de argumentos de tipo.</span><span class="sxs-lookup"><span data-stu-id="1b4cd-103">A class implements more than one generic interface, one of which inherits from another, and two implementations of an interface member could conflict for certain values of type arguments.</span></span>  
  
 <span data-ttu-id="1b4cd-104">As instruções a seguir podem gerar esse erro.</span><span class="sxs-lookup"><span data-stu-id="1b4cd-104">The following statements can generate this error.</span></span>  
  
```  
Public Interface iFace1(Of t)  
    Sub testSub()  
End Interface  
Public Interface iFace2(Of u)  
    Inherits iFace1(Of u)  
End Interface  
Public Class testClass(Of y, z)  
    Implements iFace1(Of y), iFace2(Of z)  
    Public Sub testSuby() Implements iFace1(Of y).testSub  
    End Sub  
    Public Sub testSubz() Implements iFace1(Of z).testSub  
    End Sub  
End Class  
```  
  
 <span data-ttu-id="1b4cd-105">Porque `iFace2` herda `iFace1` usando seu próprio parâmetro de tipo (`u`), `testClass` implementaria duas versões do `iFace1.testSub` com assinaturas idênticas se o mesmo argumento de tipo foi passado para `y` e `z`.</span><span class="sxs-lookup"><span data-stu-id="1b4cd-105">Because `iFace2` inherits from `iFace1` using its own type parameter (`u`), `testClass` would implement two versions of `iFace1.testSub` with identical signatures if the same type argument were passed to `y` and `z`.</span></span> <span data-ttu-id="1b4cd-106">Isso produziria uma ambiguidade sobre a versão que deseja acessar.</span><span class="sxs-lookup"><span data-stu-id="1b4cd-106">This would produce an ambiguity about which version to access.</span></span>  
  
 <span data-ttu-id="1b4cd-107">**ID do erro:** BC32125</span><span class="sxs-lookup"><span data-stu-id="1b4cd-107">**Error ID:** BC32125</span></span>

     <span data-ttu-id="1b4cd-110">-ou-</span><span class="sxs-lookup"><span data-stu-id="1b4cd-110">-or-</span></span>
