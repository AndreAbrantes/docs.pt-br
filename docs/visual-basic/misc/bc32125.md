# <a name="cannot-implement-39ltinterfacename1gtltmembernamegt39-because-its-implementation-could-conflict-with-the-implementation-for-39ltinterfacename2gtltmembernamegt39-for-some-type-arguments"></a><span data-ttu-id="80bc2-102">Não é possível implementar '&lt;interfacename1&gt;.&lt; nome do membro&gt;'porque sua implementação poderia entrar em conflito com a implementação para'&lt;interfacename2&gt;.&lt; nome do membro&gt;' para alguns argumentos de tipo</span><span class="sxs-lookup"><span data-stu-id="80bc2-102">Cannot implement &#39;&lt;interfacename1&gt;.&lt;membername&gt;&#39; because its implementation could conflict with the implementation for &#39;&lt;interfacename2&gt;.&lt;membername&gt;&#39; for some type arguments</span></span>
<span data-ttu-id="80bc2-103">Uma classe implementa mais de uma interface genérica, uma que herda de outra, e duas implementações de um membro de interface poderia entrar em conflito para certos valores de argumentos de tipo.</span><span class="sxs-lookup"><span data-stu-id="80bc2-103">A class implements more than one generic interface, one of which inherits from another, and two implementations of an interface member could conflict for certain values of type arguments.</span></span>  
  
 <span data-ttu-id="80bc2-104">As instruções a seguir podem gerar esse erro.</span><span class="sxs-lookup"><span data-stu-id="80bc2-104">The following statements can generate this error.</span></span>  
  
```  
Public Interface iFace1(Of t)  
    Sub testSub()  
End Interface  
Public Interface iFace2(Of u)  
    Inherits iFace1(Of u)  
End Interface  
Public Class testClass(Of y, z)  
    Implements iFace1(Of y), iFace2(Of z)  
    Public Sub testSuby() Implements iFace1(Of y).testSub  
    End Sub  
    Public Sub testSubz() Implements iFace1(Of z).testSub  
    End Sub  
End Class  
```  
  
 <span data-ttu-id="80bc2-105">Porque `iFace2` herda de `iFace1` usando seu próprio parâmetro de tipo (`u`), `testClass` implementaria duas versões de `iFace1.testSub` com assinaturas idênticas se o mesmo argumento de tipo foi passado para `y` e `z`.</span><span class="sxs-lookup"><span data-stu-id="80bc2-105">Because `iFace2` inherits from `iFace1` using its own type parameter (`u`), `testClass` would implement two versions of `iFace1.testSub` with identical signatures if the same type argument were passed to `y` and `z`.</span></span> <span data-ttu-id="80bc2-106">Isso produziria uma ambiguidade sobre qual versão acessar.</span><span class="sxs-lookup"><span data-stu-id="80bc2-106">This would produce an ambiguity about which version to access.</span></span>  
  
 <span data-ttu-id="80bc2-107">**ID do erro:** BC32125</span><span class="sxs-lookup"><span data-stu-id="80bc2-107">**Error ID:** BC32125</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="80bc2-108">Para corrigir este erro</span><span class="sxs-lookup"><span data-stu-id="80bc2-108">To correct this error</span></span>  
  
-   <span data-ttu-id="80bc2-109">Alterar a estrutura de herança das interfaces de modo que `iFace1` não pode ser implementado com dois argumentos de tipo diferente.</span><span class="sxs-lookup"><span data-stu-id="80bc2-109">Change the inheritance structure of the interfaces so that `iFace1` could not be implemented with two different type arguments.</span></span>  
  
     <span data-ttu-id="80bc2-110">-ou-</span><span class="sxs-lookup"><span data-stu-id="80bc2-110">-or-</span></span>  
  
-   <span data-ttu-id="80bc2-111">Remova o `Implements` instrução das interfaces que resulta no conflito de implementação.</span><span class="sxs-lookup"><span data-stu-id="80bc2-111">Remove from the `Implements` statement one of the interfaces resulting in the implementation conflict.</span></span>
