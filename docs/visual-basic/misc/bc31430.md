# <a name="expression-of-type-typename1-can-never-be-of-type-typename2"></a><span data-ttu-id="c40b0-102">A expressão do tipo ' \<typename1> ' nunca pode ser do tipo ' \<typename2> '</span><span class="sxs-lookup"><span data-stu-id="c40b0-102">Expression of type '\<typename1>' can never be of type '\<typename2>'</span></span>

<span data-ttu-id="c40b0-103">Uma `TypeOf` expressão... `Is` testa uma variável de referência de objeto para um tipo de dados que não pode conter.</span><span class="sxs-lookup"><span data-stu-id="c40b0-103">A `TypeOf`...`Is` expression tests an object reference variable to a data type that it cannot hold.</span></span>  
  
 <span data-ttu-id="c40b0-104">Em alguns casos, o compilador pode determinar que um `TypeOf` teste... `Is` só poderá falhar, por exemplo, se não houver nenhuma relação de herança entre duas classes.</span><span class="sxs-lookup"><span data-stu-id="c40b0-104">In some cases the compiler can determine that a `TypeOf`...`Is` test can only fail, for example if there is no inheritance relationship between two classes.</span></span>  
  
 <span data-ttu-id="c40b0-105">O código a seguir pode gerar esse erro.</span><span class="sxs-lookup"><span data-stu-id="c40b0-105">The following code can generate this error.</span></span>  
  
 `Dim refVar as System.Windows.Forms.Form`  
  
 `If TypeOf refVar Is System.Array`  
  
 `End If`  
  
 <span data-ttu-id="c40b0-106">Como <xref:System.Windows.Forms.Form> e <xref:System.Array> são tipos totalmente não relacionados, o compilador pode determinar que a `TypeOf` expressão... `Is` retorna `False` para qualquer valor de `refVar` .</span><span class="sxs-lookup"><span data-stu-id="c40b0-106">Because <xref:System.Windows.Forms.Form> and <xref:System.Array> are totally unrelated types, the compiler can determine that the `TypeOf`...`Is` expression returns `False` for any value of `refVar`.</span></span>
