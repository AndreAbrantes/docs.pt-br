 <span data-ttu-id="84d30-104">Quando uma variável ou expressão for avaliada como `Object`, o compilador deve executar *ligação tardia*, que faz com que operações extras em tempo de execução.</span><span class="sxs-lookup"><span data-stu-id="84d30-104">When a variable or expression evaluates to `Object`, the compiler must perform *late binding*, which causes extra operations at run time.</span></span> <span data-ttu-id="84d30-105">Ele também expõe seu aplicativo para possíveis erros de tempo de execução.</span><span class="sxs-lookup"><span data-stu-id="84d30-105">It also exposes your application to potential run-time errors.</span></span> <span data-ttu-id="84d30-106">Por exemplo, se você atribuir uma <xref:System.Windows.Forms.Form> para um `Object` variável e, em seguida, tente para compará-lo com um número, o tempo de execução lança uma <xref:System.InvalidCastException> porque o Visual Basic não é possível converter um <xref:System.Windows.Forms.Form> objeto para um valor numérico.</span><span class="sxs-lookup"><span data-stu-id="84d30-106">For example, if you assign a <xref:System.Windows.Forms.Form> to an `Object` variable and then try to compare it with a number, the runtime throws an <xref:System.InvalidCastException> because Visual Basic cannot convert a <xref:System.Windows.Forms.Form> object to a numeric value.</span></span>  
  
 <span data-ttu-id="84d30-107">As expressões em um `Select`... `Case` construção todos deve ser do mesmo tipo de dados ou de dados relacionados tipos que podem ser convertidos entre si.</span><span class="sxs-lookup"><span data-stu-id="84d30-107">The expressions in a `Select`...`Case` construction should all be of the same data type or of closely related data types that can be converted to each other.</span></span> <span data-ttu-id="84d30-108">Isso ocorre porque cada `Case` instrução compara a pelo menos um valor em relação à expressão de teste no qual o `Select`... `Case` com base em construção.</span><span class="sxs-lookup"><span data-stu-id="84d30-108">This is because each `Case` statement compares at least one value against the test expression on which the `Select`...`Case` construction is based.</span></span>  
  
 <span data-ttu-id="84d30-109">Por padrão, esta mensagem é um aviso.</span><span class="sxs-lookup"><span data-stu-id="84d30-109">By default, this message is a warning.</span></span> <span data-ttu-id="84d30-110">Para obter informações sobre como ocultar avisos ou tratar avisos como erros, consulte [Configurando avisos no Visual Basic](/visualstudio/ide/configuring-warnings-in-visual-basic).</span><span class="sxs-lookup"><span data-stu-id="84d30-110">For information on hiding warnings or treating warnings as errors, see [Configuring Warnings in Visual Basic](/visualstudio/ide/configuring-warnings-in-visual-basic).</span></span>
