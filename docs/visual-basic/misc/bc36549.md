# <a name="anonymous-type-property-39ltpropertynamegt39-cannot-be-used-in-the-definition-of-a-lambda-expression-within-the-same-initialization-list"></a><span data-ttu-id="673a8-102">Propriedade de tipo anônimo '&lt;propertyname&gt;' não pode ser usado na definição de uma expressão lambda na mesma lista de inicialização</span><span class="sxs-lookup"><span data-stu-id="673a8-102">Anonymous type property &#39;&lt;propertyname&gt;&#39; cannot be used in the definition of a lambda expression within the same initialization list</span></span>
<span data-ttu-id="673a8-103">Propriedades definidas na lista de inicialização de um tipo anônimo não podem ser parte de uma definição de expressão lambda na mesma lista.</span><span class="sxs-lookup"><span data-stu-id="673a8-103">Properties defined in the initialization list of an anonymous type cannot be part of a lambda expression definition in the same list.</span></span> <span data-ttu-id="673a8-104">Por exemplo, no código a seguir, a propriedade `Num` não podem ser incluídas na definição de `LambdaFun`.</span><span class="sxs-lookup"><span data-stu-id="673a8-104">For example, in the following code, property `Num` cannot be included in the definition of `LambdaFun`.</span></span>  
  
```vb  
' Not valid.  
'Dim anon = New With {.Num = 4, .LambdaFun = Function() .Num > 0}  
```  
  
 <span data-ttu-id="673a8-105">**ID do erro:** BC36549</span><span class="sxs-lookup"><span data-stu-id="673a8-105">**Error ID:** BC36549</span></span>  

## <a name="to-correct-this-error"></a><span data-ttu-id="673a8-106">Para corrigir este erro</span><span class="sxs-lookup"><span data-stu-id="673a8-106">To correct this error</span></span>  
  
1.  <span data-ttu-id="673a8-107">Considere dividir o tipo anônimo em duas partes:</span><span class="sxs-lookup"><span data-stu-id="673a8-107">Consider splitting the anonymous type into two parts:</span></span>  
  
```vb  
Dim anon1 = New With {.Num = 4}  
Dim anon2 = New With {.LambdaFun = Function() anon1.Num > 0}  
' - or -  
Dim anon3 = New With {.lambdaFun = Function(n As Integer) n > 0}  
Console.WriteLine((anon2.LambdaFun)())  
Console.WriteLine(anon3.lambdaFun(anon1.Num))  
anon1.Num = -5  
Console.WriteLine((anon2.LambdaFun)())  
Console.WriteLine(anon3.lambdaFun(anon1.Num))  
```  
  
<span data-ttu-id="673a8-108">Observe que, se você declarar `anon1.Num` como um `Key` propriedade, o valor não pode ser alterado.</span><span class="sxs-lookup"><span data-stu-id="673a8-108">Note that if you declare `anon1.Num` as a `Key` property, its value cannot be changed.</span></span>  
  
2.  <span data-ttu-id="673a8-109">Uma alternativa é usar uma instrução de função normal para acessar a propriedade de tipo anônimo:</span><span class="sxs-lookup"><span data-stu-id="673a8-109">An alternative is to use a regular function statement to access the anonymous type property:</span></span>  
  
```vb  
Function testNum(ByVal n As Integer) As Boolean  
    Return n > 0  
End Function  
Console.WriteLine(testNum(anon1.Num))  
```  
  
3.  <span data-ttu-id="673a8-110">Da mesma forma, você pode usar uma função lambda que é definida fora de tipo anônimo:</span><span class="sxs-lookup"><span data-stu-id="673a8-110">Similarly, you can use a lambda function that is defined outside the anonymous type:</span></span>  
  
```vb  
Dim lambdaFun1 = Function() anon1.Num > 0  
Dim lambdaFun2 = Function(n As Integer) n > 0  
```
