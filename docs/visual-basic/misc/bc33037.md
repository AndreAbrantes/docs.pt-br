# <a name="cannot-copy-the-value-of-39byref39-parameter-39ltparameternamegt39-back-to-the-matching-argument-because-type-39lttypename1gt39-cannot-be-converted-to-type-39lttypename2gt39"></a><span data-ttu-id="27d22-102">Não é possível copiar o valor de &#39;ByRef&#39; parâmetro &#39; &lt;parametername&gt; &#39; de volta para o argumento correspondente porque o tipo &#39; &lt;typename1&gt; &#39; não pode ser convertido para o tipo &#39; &lt;typename2&gt;&#39;</span><span class="sxs-lookup"><span data-stu-id="27d22-102">Cannot copy the value of &#39;ByRef&#39; parameter &#39;&lt;parametername&gt;&#39; back to the matching argument because type &#39;&lt;typename1&gt;&#39; cannot be converted to type &#39;&lt;typename2&gt;&#39;</span></span>
<span data-ttu-id="27d22-103">Um procedimento é declarado com um tipo de parâmetro que não pode ser convertido para o tipo de argumento de chamada.</span><span class="sxs-lookup"><span data-stu-id="27d22-103">A procedure is declared with a parameter type which cannot be converted back to the calling argument type.</span></span>

 <span data-ttu-id="27d22-107">Se você passar o argumento [ByRef](../../visual-basic/language-reference/modifiers/byref.md), Visual Basic, às vezes, copia o valor do argumento para uma variável local no procedimento em vez de passar uma referência.</span><span class="sxs-lookup"><span data-stu-id="27d22-107">If you pass the argument [ByRef](../../visual-basic/language-reference/modifiers/byref.md), Visual Basic sometimes copies the argument value into a local variable in the procedure instead of passing a reference.</span></span> <span data-ttu-id="27d22-108">Nesse caso, quando o procedimento retorna, Visual Basic deve, em seguida, copie o valor da variável local volta para o argumento no código de chamada.</span><span class="sxs-lookup"><span data-stu-id="27d22-108">In such a case, when the procedure returns, Visual Basic must then copy the local variable value back into the argument in the calling code.</span></span>  
  
 <span data-ttu-id="27d22-109">Se um `ByRef` o valor do argumento é copiado para o procedimento e o argumento e parâmetro são do mesmo tipo, nenhuma conversão é necessária.</span><span class="sxs-lookup"><span data-stu-id="27d22-109">If a `ByRef` argument value is copied into the procedure and the argument and parameter are of the same type, no conversion is necessary.</span></span> <span data-ttu-id="27d22-110">Mas se os tipos forem diferentes, o Visual Basic deve converter em ambas as direções.</span><span class="sxs-lookup"><span data-stu-id="27d22-110">But if the types are different, Visual Basic must convert in both directions.</span></span> <span data-ttu-id="27d22-111">Se um dos tipos é o tipo de classe ou estrutura, o Visual Basic deve convertê-lo para e o outro tipo.</span><span class="sxs-lookup"><span data-stu-id="27d22-111">If one of the types is your class or structure type, Visual Basic must convert it both to and from the other type.</span></span> <span data-ttu-id="27d22-112">Isso significa que você deve definir os operadores de conversão em ambas as direções.</span><span class="sxs-lookup"><span data-stu-id="27d22-112">This means you must define conversion operators in both directions.</span></span>

## <a name="to-correct-this-error"></a><span data-ttu-id="27d22-114">Para corrigir este erro</span><span class="sxs-lookup"><span data-stu-id="27d22-114">To correct this error</span></span>
