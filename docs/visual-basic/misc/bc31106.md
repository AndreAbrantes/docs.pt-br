# <a name="property-accessors-cannot-be-declared-accessmodifier-in-a-notoverridable-property"></a><span data-ttu-id="0c213-102">Os acessadores da propriedade não podem ser declarados como '\<accessmodifier>' em uma propriedade 'NotOverridable'</span><span class="sxs-lookup"><span data-stu-id="0c213-102">Property accessors cannot be declared '\<accessmodifier>' in a 'NotOverridable' property</span></span>

<span data-ttu-id="0c213-103">Uma instrução [Get](../language-reference/statements/get-statement.md) ou [instruções SET](../language-reference/statements/set-statement.md) em uma `NotOverridable` Propriedade incluem a `Private` palavra-chave.</span><span class="sxs-lookup"><span data-stu-id="0c213-103">A [Get Statement](../language-reference/statements/get-statement.md) or [Set Statement](../language-reference/statements/set-statement.md) in a `NotOverridable` property includes the `Private` keyword.</span></span>  
  
 <span data-ttu-id="0c213-104">A seguinte linha de raciocínio explica por que `NotOverridable` e `Private` não pode ser combinada em uma [instrução de propriedade](../language-reference/statements/property-statement.md):</span><span class="sxs-lookup"><span data-stu-id="0c213-104">The following line of reasoning explains why `NotOverridable` and `Private` cannot be combined in a [Property Statement](../language-reference/statements/property-statement.md):</span></span>  
  
1. <span data-ttu-id="0c213-105">Uma propriedade ou procedimento que não substitui uma propriedade ou procedimento de classe base tem uma configuração padrão de [NotOverridable](../language-reference/modifiers/notoverridable.md).</span><span class="sxs-lookup"><span data-stu-id="0c213-105">A property or procedure that does not override a base class property or procedure has a default setting of [NotOverridable](../language-reference/modifiers/notoverridable.md).</span></span>  
  
2. <span data-ttu-id="0c213-106">No entanto, uma propriedade ou um procedimento em uma classe derivada que substitui uma propriedade ou um procedimento de classe base tem uma configuração padrão de [substituível](../language-reference/modifiers/overridable.md).</span><span class="sxs-lookup"><span data-stu-id="0c213-106">However, a property or procedure in a derived class that overrides a base class property or procedure has a default setting of [Overridable](../language-reference/modifiers/overridable.md).</span></span> <span data-ttu-id="0c213-107">Para encerrar a hierarquia de substituição, você pode declará-la `NotOverridable` .</span><span class="sxs-lookup"><span data-stu-id="0c213-107">To terminate the hierarchy of overriding, you can declare it `NotOverridable`.</span></span> <span data-ttu-id="0c213-108">Esse é o único contexto no qual você pode usar `NotOverridable` .</span><span class="sxs-lookup"><span data-stu-id="0c213-108">This is the only context in which you can use `NotOverridable`.</span></span> <span data-ttu-id="0c213-109">Ou seja, você pode usar `NotOverridable` somente em combinação com [substituições](../language-reference/modifiers/overrides.md).</span><span class="sxs-lookup"><span data-stu-id="0c213-109">That is, you can use `NotOverridable` only in combination with [Overrides](../language-reference/modifiers/overrides.md).</span></span>  
  
3. <span data-ttu-id="0c213-110">Se uma propriedade de classe base ou um procedimento for declarado [particular](../language-reference/modifiers/private.md), uma classe derivada não poderá substituir essa propriedade ou procedimento porque ele não pode acessá-la.</span><span class="sxs-lookup"><span data-stu-id="0c213-110">If a base class property or procedure is declared [Private](../language-reference/modifiers/private.md), a derived class cannot override that property or procedure because it cannot access it.</span></span> <span data-ttu-id="0c213-111">Por isso, você não pode usar `Private` em combinação com `Overridable` .</span><span class="sxs-lookup"><span data-stu-id="0c213-111">Because of this, you cannot use `Private` in combination with `Overridable`.</span></span>  
  
4. <span data-ttu-id="0c213-112">Para substituir uma propriedade ou um procedimento, a propriedade ou o procedimento de substituição deve ter não apenas a assinatura idêntica, mas também o mesmo nível de acesso.</span><span class="sxs-lookup"><span data-stu-id="0c213-112">To override a property or procedure, the overriding property or procedure must have not only the identical signature but also the same access level.</span></span> <span data-ttu-id="0c213-113">Isso significa que uma propriedade ou procedimento de substituição não pode especificar `Private` , porque uma propriedade substituível ou um procedimento não pode especificar `Private` .</span><span class="sxs-lookup"><span data-stu-id="0c213-113">This means that an overriding property or procedure cannot specify `Private`, because an overridable property or procedure cannot specify `Private`.</span></span>  
  
5. <span data-ttu-id="0c213-114">Como você pode especificar `NotOverridable` apenas em uma propriedade ou procedimento de substituição, não é possível combiná-lo com `Private` .</span><span class="sxs-lookup"><span data-stu-id="0c213-114">Because you can specify `NotOverridable` only on an overriding property or procedure, you cannot combine it with `Private`.</span></span>  
  
 <span data-ttu-id="0c213-115">Pelo mesmo raciocínio, os procedimentos de propriedade individuais ( `Get` e `Set` ) de uma propriedade de substituição não podem ser `Private` .</span><span class="sxs-lookup"><span data-stu-id="0c213-115">By the same reasoning, the individual property procedures (`Get` and `Set`) of an overriding property cannot be `Private`.</span></span>  
  
 <span data-ttu-id="0c213-116">**ID do erro:** BC31106</span><span class="sxs-lookup"><span data-stu-id="0c213-116">**Error ID:** BC31106</span></span>
