# <a name="type-argument-typeargumentname-is-declared-mustinherit-and-does-not-satisfy-the-new-constraint-for-type-parameter-typeparametername"></a><span data-ttu-id="04768-102">O argumento de tipo '\<typeargumentname >' é declarado 'como MustInherit' e não satisfaz a restrição 'New' para parâmetro de tipo '\<typeparametername >'</span><span class="sxs-lookup"><span data-stu-id="04768-102">Type argument '\<typeargumentname>' is declared 'MustInherit' and does not satisfy the 'New' constraint for type parameter '\<typeparametername>'</span></span>
<span data-ttu-id="04768-103">Um tipo genérico é invocado com um `MustInherit` classe como um argumento de tipo, enquanto o parâmetro de tipo correspondente é declarado com o `New` restrição.</span><span class="sxs-lookup"><span data-stu-id="04768-103">A generic type is invoked with a `MustInherit` class as a type argument, while the corresponding type parameter is declared with the `New` constraint.</span></span>  
  
 <span data-ttu-id="04768-104">O `New` restrição requer que o tipo passado no argumento de tipo correspondente deve oferecer suporte a criação de objetos.</span><span class="sxs-lookup"><span data-stu-id="04768-104">The `New` constraint requires that the type passed in the corresponding type argument must support the creation of objects.</span></span> <span data-ttu-id="04768-105">No entanto, uma *abstrata* classe, ou seja, uma classe declarada como `MustInherit`, não expõe nenhum construtor porque você não pode criar todos os objetos dele.</span><span class="sxs-lookup"><span data-stu-id="04768-105">However, an *abstract* class, that is, a class declared as `MustInherit`, does not expose any constructors because you cannot create any objects from it.</span></span>  
  
 <span data-ttu-id="04768-106">**ID do erro:** BC32082</span><span class="sxs-lookup"><span data-stu-id="04768-106">**Error ID:** BC32082</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="04768-107">Para corrigir este erro</span><span class="sxs-lookup"><span data-stu-id="04768-107">To correct this error</span></span>  
  
1. <span data-ttu-id="04768-108">Se a classe usada no argumento de tipo não precisa ser abstrato e, em seguida, remova o `MustInherit` palavra-chave de sua declaração.</span><span class="sxs-lookup"><span data-stu-id="04768-108">If the class used in the type argument does not need to be abstract, then remove the `MustInherit` keyword from its declaration.</span></span>  
  
2. <span data-ttu-id="04768-109">Se a classe usada no argumento de tipo deve ser abstrato, mas não precisa ser usado para construir o tipo genérico, em seguida, passe uma classe diferente no argumento de tipo.</span><span class="sxs-lookup"><span data-stu-id="04768-109">If the class used in the type argument needs to be abstract but does not need to be used to construct the generic type, then pass a different class in the type argument.</span></span>  
  
3. <span data-ttu-id="04768-110">Se o parâmetro de tipo correspondente não precisa criar os objetos do tipo passado para ele e, em seguida, remover o `New` restrição de sua declaração.</span><span class="sxs-lookup"><span data-stu-id="04768-110">If the corresponding type parameter does not need to create any objects from the type passed to it, then remove the `New` constraint from its declaration.</span></span>
