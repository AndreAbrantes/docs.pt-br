# <a name="type-argument-39lttypeargumentnamegt39-is-declared-39mustinherit39-and-does-not-satisfy-the-39new39-constraint-for-type-parameter-39lttypeparameternamegt39"></a><span data-ttu-id="87e36-102">O argumento de tipo '&lt;typeargumentname&gt;'é declarado 'MustInherit' e não satisfaz a restrição 'New' para o parâmetro de tipo'&lt;typeparametername&gt;'</span><span class="sxs-lookup"><span data-stu-id="87e36-102">Type argument &#39;&lt;typeargumentname&gt;&#39; is declared &#39;MustInherit&#39; and does not satisfy the &#39;New&#39; constraint for type parameter &#39;&lt;typeparametername&gt;&#39;</span></span>
<span data-ttu-id="87e36-103">Um tipo genérico é invocado com um `MustInherit` classe como um argumento de tipo, enquanto o parâmetro do tipo correspondente é declarado com o `New` restrição.</span><span class="sxs-lookup"><span data-stu-id="87e36-103">A generic type is invoked with a `MustInherit` class as a type argument, while the corresponding type parameter is declared with the `New` constraint.</span></span>  
  
 <span data-ttu-id="87e36-104">O `New` restrição requer que o tipo passado no argumento de tipo correspondente deve oferecer suporte a criação de objetos.</span><span class="sxs-lookup"><span data-stu-id="87e36-104">The `New` constraint requires that the type passed in the corresponding type argument must support the creation of objects.</span></span> <span data-ttu-id="87e36-105">No entanto, uma *abstrato* classe, isto é, uma classe declarada como `MustInherit`, não expõe nenhum construtor porque você não pode criar nenhum objeto dele.</span><span class="sxs-lookup"><span data-stu-id="87e36-105">However, an *abstract* class, that is, a class declared as `MustInherit`, does not expose any constructors because you cannot create any objects from it.</span></span>  
  
 <span data-ttu-id="87e36-106">**ID do erro:** BC32082</span><span class="sxs-lookup"><span data-stu-id="87e36-106">**Error ID:** BC32082</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="87e36-107">Para corrigir este erro</span><span class="sxs-lookup"><span data-stu-id="87e36-107">To correct this error</span></span>  
  
1.  <span data-ttu-id="87e36-108">Se a classe usada no argumento de tipo não precisa ser abstrata, remova o `MustInherit` palavra-chave da sua declaração.</span><span class="sxs-lookup"><span data-stu-id="87e36-108">If the class used in the type argument does not need to be abstract, then remove the `MustInherit` keyword from its declaration.</span></span>  
  
2.  <span data-ttu-id="87e36-109">Se a classe usada no argumento de tipo precisa ser abstrata mas não precisa ser usado para construir o tipo genérico, em seguida, passe uma classe diferente no argumento de tipo.</span><span class="sxs-lookup"><span data-stu-id="87e36-109">If the class used in the type argument needs to be abstract but does not need to be used to construct the generic type, then pass a different class in the type argument.</span></span>  
  
3.  <span data-ttu-id="87e36-110">Se o parâmetro de tipo correspondente não precisa criar os objetos do tipo passado para ele e, em seguida, remover o `New` restrição de sua declaração.</span><span class="sxs-lookup"><span data-stu-id="87e36-110">If the corresponding type parameter does not need to create any objects from the type passed to it, then remove the `New` constraint from its declaration.</span></span>
