# <a name="type-or-with-expected"></a><span data-ttu-id="a547a-102">Tipo ou ' with ' esperado</span><span class="sxs-lookup"><span data-stu-id="a547a-102">Type or 'With' expected</span></span>
<span data-ttu-id="a547a-103">Quando você declara uma instância de uma classe, a palavra-chave `New` deve ser seguida por um nome de tipo ou por `With`.</span><span class="sxs-lookup"><span data-stu-id="a547a-103">When you declare an instance of a class, the `New` keyword must be followed by a type name or by `With`.</span></span> <span data-ttu-id="a547a-104">Por exemplo, as instruções a seguir declaram `client` como uma instância da classe `Customer`.</span><span class="sxs-lookup"><span data-stu-id="a547a-104">For example, the following statements each declare `client` to be an instance of the `Customer` class.</span></span> <span data-ttu-id="a547a-105">O nome do tipo `Customer` segue `New`.</span><span class="sxs-lookup"><span data-stu-id="a547a-105">The type name `Customer` follows `New`.</span></span>  
  
```vb  
' Dim client As New Customer()  
' The next declaration uses an object initializer.  
Dim client As New Customer() With {.Name = "Litware, Inc."}  
```  
  
 <span data-ttu-id="a547a-106">A partir do Visual Basic 2008, você pode declarar um objeto para ser uma instância de um tipo anônimo; nesse caso, você não especifica um tipo de dados.</span><span class="sxs-lookup"><span data-stu-id="a547a-106">Beginning with Visual Basic 2008, you can declare an object to be an instance of an anonymous type, in which case you do not specify a data type.</span></span> <span data-ttu-id="a547a-107">Em declarações de tipo anônimo, a palavra-chave `With` segue `New`.</span><span class="sxs-lookup"><span data-stu-id="a547a-107">In anonymous type declarations, the keyword `With` follows `New`.</span></span>  
  
```vb  
Dim person = New With {.Name ="Mike Nash", .Age = 27}  
```
