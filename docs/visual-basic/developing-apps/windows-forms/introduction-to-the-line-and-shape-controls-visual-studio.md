# <a name="introduction-to-the-line-and-shape-controls-visual-studio"></a><span data-ttu-id="ac5e9-102">Introdução aos controles de linha e forma (Visual Studio)</span><span class="sxs-lookup"><span data-stu-id="ac5e9-102">Introduction to the Line and Shape Controls (Visual Studio)</span></span>
<span data-ttu-id="ac5e9-103">Os controles do Visual Basic Power Packs linha e forma são um conjunto de três controles gráficos que lhe permitem desenhar linhas e formas em formulários e contêineres.</span><span class="sxs-lookup"><span data-stu-id="ac5e9-103">The Visual Basic Power Packs Line and Shape controls are a set of three graphical controls that enable you to draw lines and shapes on forms and containers.</span></span> <span data-ttu-id="ac5e9-104">O <xref:Microsoft.VisualBasic.PowerPacks.LineShape> controle é usado para desenhar linhas horizontais, verticais e diagonais.</span><span class="sxs-lookup"><span data-stu-id="ac5e9-104">The <xref:Microsoft.VisualBasic.PowerPacks.LineShape> control is used to draw horizontal, vertical, and diagonal lines.</span></span> <span data-ttu-id="ac5e9-105">O <xref:Microsoft.VisualBasic.PowerPacks.OvalShape> controle é usado para desenhar círculos e elipses e o <xref:Microsoft.VisualBasic.PowerPacks.RectangleShape> controle é usado para desenhar retângulos e quadrados.</span><span class="sxs-lookup"><span data-stu-id="ac5e9-105">The <xref:Microsoft.VisualBasic.PowerPacks.OvalShape> control is used to draw circles and ovals, and the <xref:Microsoft.VisualBasic.PowerPacks.RectangleShape> control is used to draw rectangles and squares.</span></span>  
  
## <a name="line-and-shape-controls"></a><span data-ttu-id="ac5e9-106">Controles de linha e forma</span><span class="sxs-lookup"><span data-stu-id="ac5e9-106">Line and Shape Controls</span></span>  
 <span data-ttu-id="ac5e9-107">Controles de linha e forma encapsulam vários dos métodos gráficos contidos no <xref:System.Drawing> namespace.</span><span class="sxs-lookup"><span data-stu-id="ac5e9-107">Line and Shape controls encapsulate many of the graphics methods that are contained in the <xref:System.Drawing> namespace.</span></span> <span data-ttu-id="ac5e9-108">Isso permite que você desenhar linhas e formas em uma única etapa sem ter de criar objetos gráficos, canetas e pincéis.</span><span class="sxs-lookup"><span data-stu-id="ac5e9-108">This enables you to draw lines and shapes in a single step without having to create graphics objects, pens, and brushes.</span></span> <span data-ttu-id="ac5e9-109">Técnicas gráficas complexas, como preenchimentos com gradiente podem ser feitas definindo-se apenas algumas propriedades.</span><span class="sxs-lookup"><span data-stu-id="ac5e9-109">Complex graphics techniques such as gradient fills can be accomplished by just setting some properties.</span></span>  
  
 <span data-ttu-id="ac5e9-110">Embora também seja possível desenhar linhas e formas, usando métodos gráficos, há diversas vantagens em usar os controles de linha e forma:</span><span class="sxs-lookup"><span data-stu-id="ac5e9-110">Although it is also possible to draw lines and shapes by using graphics methods, there are several advantages to using the Line and Shape controls:</span></span>  
  
-   <span data-ttu-id="ac5e9-111">Métodos gráficos podem ser chamados somente em tempo de execução.</span><span class="sxs-lookup"><span data-stu-id="ac5e9-111">Graphics methods can be called only at run time.</span></span> <span data-ttu-id="ac5e9-112">Controles de linha e forma podem ser adicionados a um formulário em tempo de design.</span><span class="sxs-lookup"><span data-stu-id="ac5e9-112">Line and Shape controls can be added to a form at design time.</span></span> <span data-ttu-id="ac5e9-113">Isso permite que você veja sua aparência e posicioná-los; Eles também podem ser adicionados em tempo de execução.</span><span class="sxs-lookup"><span data-stu-id="ac5e9-113">This enables you to see what they look like and to position them exactly; they can also be added at run time.</span></span>  
  
-   <span data-ttu-id="ac5e9-114">Controles de linha e forma são selecionáveis no tempo de execução, fornecendo a eventos, como <xref:Microsoft.VisualBasic.PowerPacks.Shape.Click> e <xref:Microsoft.VisualBasic.PowerPacks.Shape.OnDoubleClick%2A>.</span><span class="sxs-lookup"><span data-stu-id="ac5e9-114">Line and Shape controls are selectable at run time, providing events such as <xref:Microsoft.VisualBasic.PowerPacks.Shape.Click> and <xref:Microsoft.VisualBasic.PowerPacks.Shape.OnDoubleClick%2A>.</span></span> <span data-ttu-id="ac5e9-115">As saídas dos métodos gráficos não são selecionáveis e não fornecem eventos.</span><span class="sxs-lookup"><span data-stu-id="ac5e9-115">The outputs of graphics methods are not selectable and do not provide events.</span></span>  
  
-   <span data-ttu-id="ac5e9-116">Controles de linha e forma fornecem <xref:Microsoft.VisualBasic.PowerPacks.Shape.BringToFront%2A> e <xref:Microsoft.VisualBasic.PowerPacks.Shape.SendToBack%2A> métodos que permitem que você controle a ordem z em tempo de design e em tempo de execução.</span><span class="sxs-lookup"><span data-stu-id="ac5e9-116">Line and Shape controls provide <xref:Microsoft.VisualBasic.PowerPacks.Shape.BringToFront%2A> and <xref:Microsoft.VisualBasic.PowerPacks.Shape.SendToBack%2A> methods that enable you to control their z-order at design time and at run time.</span></span> <span data-ttu-id="ac5e9-117">A ordem z dos métodos gráficos pode ser controlada apenas alterando sua ordem de execução em tempo de execução.</span><span class="sxs-lookup"><span data-stu-id="ac5e9-117">The z-order of graphics methods can be controlled only by changing their order of execution at run time.</span></span>  
  
-   <span data-ttu-id="ac5e9-118">Controles de linha e forma são controles sem janelas; eles têm nenhum identificador de janela e, portanto, usar menos recursos do sistema.</span><span class="sxs-lookup"><span data-stu-id="ac5e9-118">Line and Shape controls are windowless controls; they have no window handles and therefore use less system resources.</span></span>  
  
### <a name="object-model"></a><span data-ttu-id="ac5e9-119">Modelo de objeto</span><span class="sxs-lookup"><span data-stu-id="ac5e9-119">Object Model</span></span>

 <span data-ttu-id="ac5e9-121">A ilustração a seguir mostra a hierarquia de objeto de linha e forma.</span><span class="sxs-lookup"><span data-stu-id="ac5e9-121">The following illustration shows the Line and Shape object hierarchy.</span></span>

<span data-ttu-id="ac5e9-123">Hierarquia de objetos de linha e forma</span><span class="sxs-lookup"><span data-stu-id="ac5e9-123">Line and Shape object hierarchy</span></span>

 <span data-ttu-id="ac5e9-127">O <xref:Microsoft.VisualBasic.PowerPacks.OvalShape> e <xref:Microsoft.VisualBasic.PowerPacks.RectangleShape> classes podem ser usadas para desenhar retângulos com cantos arredondados, elipses, retângulos e círculos.</span><span class="sxs-lookup"><span data-stu-id="ac5e9-127">The <xref:Microsoft.VisualBasic.PowerPacks.OvalShape> and <xref:Microsoft.VisualBasic.PowerPacks.RectangleShape> classes can be used to draw circles, ovals, rectangles, and rectangles with rounded corners.</span></span>
