# <a name="conclusion"></a>Conclusão

As seguintes conclusões-chave são as conclusões mais importantes deste guia.

**Benefícios de usar sem servidor.** As soluções sem servidor proporcionam o importante benefício da redução de custos porque o sem servidor é implementado em um modelo de pagamento por uso. O Serverless torna possível dimensionar, testar e implantar componentes individuais independentes do seu aplicativo. O Serverless é exclusivamente adequado para implementar arquiteturas de microsserviços e se integra totalmente a um pipeline DevOps.

**Código como uma unidade de implantação.** Sem servidor, o hardware, o sistema operacional e o tempo de execução se afastam do aplicativo. O Serverless permite focar na lógica de negócios em código como a unidade de implantação.

**Gatilhos e amarras.** O Serverless facilita a integração com armazenamento, APIs e outros recursos em nuvem. O Azure Functions fornece gatilhos para executar códigos e vinculações para interagir com os recursos.

**Microserviços.** A arquitetura de microsserviços está se tornando a abordagem preferida para aplicativos distribuídos e grandes ou complexos de missão crítica que são baseados em vários subsistemas independentes na forma de serviços autônomos. Em uma arquitetura baseada em microserviços, o aplicativo é construído como uma coleção de serviços que podem ser desenvolvidos, testados, versionados, implantados e dimensionados independentemente. Serverless é uma arquitetura adequada para a construção desses serviços.

**Plataformas sem servidor.** Sem servidor não é só sobre o código. Plataformas que suportam arquiteturas sem servidor incluem fluxos de trabalho e orquestração sem servidor, serviços de mensagens e eventos sem servidor e bancos de dados sem servidor.

**Desafios sem servidor.** O Serverless introduz desafios relacionados ao desenvolvimento de aplicativos distribuídos, como modelos de dados fragmentados e independentes, resiliência, versionamento e descoberta de serviços. Sem servidor pode não ser ideal para processos ou componentes de longa duração que se beneficiem de um acoplamento mais apertado.

**Sem servidor como ferramenta, não na caixa de ferramentas.** Sem servidor não é a solução exclusiva para a arquitetura de aplicativos. É uma ferramenta que pode ser aproveitada como parte de uma aplicação híbrida que pode conter camadas tradicionais, extremidades traseiras de monólito e recipientes. O Serverless pode ser usado para melhorar as soluções existentes e não é uma abordagem tudo ou nada para o desenvolvimento de aplicativos.
