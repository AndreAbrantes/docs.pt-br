# <a name="conclusions"></a>Conclusões

- As soluções baseadas em contêineres, em última análise, proporcionam benefícios de redução de custos. Os contêineres são uma solução para problemas de implantação porque removem o atrito causado pela ausência de dependências nos ambientes de produção. Ao remover esses problemas, melhora significativamente as operações de Dev/Test, DevOps e produção.

- Um contêiner do Docker está se tornando a unidade padrão de implantação para qualquer serviço ou aplicativo baseado em servidor.

- Para ambientes de produção, você deve usar um orquestrador (como kubernetes) para hospedar aplicativos dimensionáveis baseados em contêineres.

- Os contêineres de hospedagem de VMs do Azure são uma maneira rápida e simples de criar pequenos ambientes Dev/Test na nuvem.

- A instância gerenciada do banco de dados Do Azure SQL é recomendada por padrão ao migrar seus bancos de dados relacionais de aplicativos existentes para o Azure.

- Visual Studio 2017 e Image2Docker são ferramentas básicas para você começar a modernizar seus aplicativos .NET existentes com o Windows Containers, acelerando a curva de aprendizado de início.

- Ao colocar aplicativos conteilistas na produção, você sempre criará ou adotará uma cultura DevOps e ferramentas DevOps para pipelines de CI/CD, como Azure DevOps Services ou Jenkins.

- O Microsoft Azure fornece o ambiente mais abrangente e completo para modernizar seus aplicativos .NET Framework existentes com contêineres Windows, infra-estrutura de nuvem e serviços PaaS.
