# <a name="conclusions"></a>Conclusões

- As soluções baseadas em contêiner, por fim, fornecem benefícios de economia de custo. Os contêineres são uma solução para problemas de implantação porque removem o conflito causado por uma ausência de dependências em ambientes de produção. Ao remover esses problemas, ele melhora significativamente as operações de desenvolvimento/teste, DevOps e produção.

- Um contêiner do Docker está se tornando a unidade padrão de implantação para qualquer serviço ou aplicativo baseado em servidor.

- Para ambientes de produção, você deve usar um orquestrador (como o kubernetes) para hospedar aplicativos baseados em contêineres escalonáveis.

- Os contêineres de Hospedagem de VMs do Azure são uma maneira rápida e simples de criar pequenos ambientes de desenvolvimento/teste na nuvem.

- O Instância Gerenciada do Banco de Dados SQL do Azure é recomendado por padrão ao migrar seus bancos de dados relacionais de aplicativos existentes para o Azure.

- O Visual Studio 2017 e o Image2Docker são ferramentas básicas para você começar a modernizar seus aplicativos .NET existentes com contêineres do Windows acelerando a curva de aprendizado de introdução.

- Ao colocar aplicativos em contêineres em produção, você sempre criará ou adotará uma cultura DevOps e ferramentas de DevOps para pipelines de CI/CD, como Azure DevOps Services ou Jenkins.

- Microsoft Azure fornece o ambiente mais abrangente e completo para modernizar seus aplicativos .NET Framework existentes com contêineres do Windows, infraestrutura de nuvem e serviços de PaaS.
