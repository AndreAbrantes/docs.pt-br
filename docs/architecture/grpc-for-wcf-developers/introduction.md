## <a name="history"></a><span data-ttu-id="e3cc1-112">Histórico</span><span class="sxs-lookup"><span data-stu-id="e3cc1-112">History</span></span>

<span data-ttu-id="e3cc1-113">O princípio fundamental de uma rede de computadores como nada mais do que um grupo de computadores que trocam dados entre si para atingir um conjunto de tarefas inter-relacionadas não mudou desde seu início.</span><span class="sxs-lookup"><span data-stu-id="e3cc1-113">The fundamental principle of a computer network as nothing more than a group of computers exchanging data with each other to achieve a set of interrelated tasks hasn't changed since its inception.</span></span> <span data-ttu-id="e3cc1-114">No entanto, a complexidade, a escala e as expectativas cresceram exponencialmente.</span><span class="sxs-lookup"><span data-stu-id="e3cc1-114">However, the complexity, scale, and expectations have grown exponentially.</span></span>

<span data-ttu-id="e3cc1-117">No entanto, o foco em breve mudou para a melhor maneira de otimizar a comunicação entre várias plataformas, promovendo uma abordagem independente de linguagem.</span><span class="sxs-lookup"><span data-stu-id="e3cc1-117">However, the focus soon shifted to how best to optimize communication across multiple platforms promoting a language agnostic approach.</span></span> <span data-ttu-id="e3cc1-118">A SOA (arquitetura orientada a serviços) forneceu uma estrutura para unir de forma flexível uma ampla coleção de serviços que poderiam ser fornecidos a um aplicativo.</span><span class="sxs-lookup"><span data-stu-id="e3cc1-118">Service-oriented architecture (SOA) provided a structure for loosely coupling a broad collection of services that could be provided to an application.</span></span>

<span data-ttu-id="e3cc1-119">O desenvolvimento de *Serviços Web* ocorreu quando todas as principais plataformas podiam acessar a Internet, mas eles ainda não podiam interagir entre si.</span><span class="sxs-lookup"><span data-stu-id="e3cc1-119">The development of *Web Services* occurred once all major platforms could access the Internet, but they still couldn’t interact with each other.</span></span> <span data-ttu-id="e3cc1-120">Os serviços Web têm padrões e protocolos abertos, incluindo:</span><span class="sxs-lookup"><span data-stu-id="e3cc1-120">Web services have open standards and protocols including:</span></span>

- <span data-ttu-id="e3cc1-122">SOAP (Simple Object Access Protocol) para transferir dados;</span><span class="sxs-lookup"><span data-stu-id="e3cc1-122">Simple Object Access Protocol (SOAP) to transfer data;</span></span>
- <span data-ttu-id="e3cc1-123">WSDL (Web Services Definition Language) – descreve e conecta o serviço Web ao aplicativo cliente; *e*</span><span class="sxs-lookup"><span data-stu-id="e3cc1-123">Web Services Definition Language (WSDL) – describes and connects the web service to the client application; *and*</span></span>

Mensagens SOAP são maiores que mensagens em protocolos comparáveis.</span><span class="sxs-lookup"><span data-stu-id="e3cc1-127">The sacrifice for making SOAP easily understood is size; SOAP messages are larger than messages in comparable protocols.</span></span> <span data-ttu-id="e3cc1-128">O SOAP foi projetado para dividir aplicativos monolíticos em formulários de vários componentes sem perder a segurança ou o controle.</span><span class="sxs-lookup"><span data-stu-id="e3cc1-128">SOAP was designed to break down monolithic applications into multi-component form without losing security or control.</span></span> <span data-ttu-id="e3cc1-129">Portanto, o WCF foi projetado para trabalhar com esse tipo de sistema, ao contrário de gRPC, que começou como um sistema distribuído.</span><span class="sxs-lookup"><span data-stu-id="e3cc1-129">Therefore, WCF was designed to work with that kind of system, unlike gRPC, which began as a distributed system.</span></span> <span data-ttu-id="e3cc1-130">O WCF abordou algumas dessas limitações ao desenvolver e documentar protocolos de extensão proprietários para a pilha SOAP, mas com o custo de falta de suporte de outras plataformas.</span><span class="sxs-lookup"><span data-stu-id="e3cc1-130">WCF addressed some of these limitations by developing and documenting proprietary extension protocols for the SOAP stack, but at the cost of lack of support from other platforms.</span></span>

<span data-ttu-id="e3cc1-131">Windows Communication Foundation é uma estrutura para a criação de serviços.</span><span class="sxs-lookup"><span data-stu-id="e3cc1-131">Windows Communication Foundation is a framework for building services.</span></span> <span data-ttu-id="e3cc1-132">Ele foi projetado na 2000s inicial para ajudar os desenvolvedores usando a SOA inicial a gerenciar as complexidades de trabalhar com SOAP.</span><span class="sxs-lookup"><span data-stu-id="e3cc1-132">It was designed in the early 2000s to help developers using early SOA to manage the complexities of working with SOAP.</span></span> <span data-ttu-id="e3cc1-133">Embora ele remova o requisito para que o desenvolvedor escreva seus próprios protocolos SOAP, o WCF ainda usa SOAP para habilitar a interoperabilidade com outros sistemas.</span><span class="sxs-lookup"><span data-stu-id="e3cc1-133">Although it removes the requirement for the developer to write their own SOAP protocols, WCF still uses SOAP to enable interoperability with other systems.</span></span> <span data-ttu-id="e3cc1-134">O WCF também foi projetado para fornecer soluções em vários protocolos (HTTP/1.1, NetTCP e assim por diante).</span><span class="sxs-lookup"><span data-stu-id="e3cc1-134">WCF was also designed to deliver solutions across multiple protocols (HTTP/1.1, NetTCP, and so on).</span></span>

## <a name="microservices"></a><span data-ttu-id="e3cc1-135">Microsserviços</span><span class="sxs-lookup"><span data-stu-id="e3cc1-135">Microservices</span></span>

- <span data-ttu-id="e3cc1-139">As alterações e atualizações podem ser manipuladas de forma independente.</span><span class="sxs-lookup"><span data-stu-id="e3cc1-139">Changes and upgrades can be handled independently.</span></span>
- <span data-ttu-id="e3cc1-140">O tratamento de erros se torna mais eficiente, pois os problemas podem ser rastreados para serviços específicos que são, então, isolados, recriados, testados e reimplantados independentemente dos outros serviços.</span><span class="sxs-lookup"><span data-stu-id="e3cc1-140">Error handling becomes more efficient as problems can be traced to specific services that are then isolated, rebuilt, tested, and redeployed independent of the other services.</span></span>

<span data-ttu-id="e3cc1-143">A mudança para aumentar a virtualização, a computação em nuvem, os contêineres e os Internet das Coisas contribuíram para o aumento contínuo de microserviços.</span><span class="sxs-lookup"><span data-stu-id="e3cc1-143">The move towards increasing virtualization, cloud computing, containers and the Internet of Things have contributed to the ongoing rise of microservices.</span></span> <span data-ttu-id="e3cc1-144">No entanto, os microserviços não estão sem seus desafios.</span><span class="sxs-lookup"><span data-stu-id="e3cc1-144">However, microservices aren't without their challenges.</span></span> <span data-ttu-id="e3cc1-145">A infra-estrutura fragmentada/descentralizada colocou mais ênfase na necessidade de simplicidade e velocidade ao se comunicar entre os serviços.</span><span class="sxs-lookup"><span data-stu-id="e3cc1-145">The fragmented/decentralized infrastructure placed more emphasis on the need for simplicity and speed when communicating between services.</span></span> <span data-ttu-id="e3cc1-146">Isso, por sua vez, atraiu atenção para a natureza mais trabalhoso e condelito de SOAP.</span><span class="sxs-lookup"><span data-stu-id="e3cc1-146">This in turn drew attention to the sometimes laborious and contorted nature of SOAP.</span></span>
