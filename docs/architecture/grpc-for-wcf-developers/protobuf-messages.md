## <a name="declaring-a-message"></a><span data-ttu-id="4a1af-106">Declarando uma mensagem</span><span class="sxs-lookup"><span data-stu-id="4a1af-106">Declaring a message</span></span>

<span data-ttu-id="4a1af-107">No WCF, uma classe de `Stock` para um aplicativo de comércio no mercado de ações pode ser definida como o exemplo a seguir:</span><span class="sxs-lookup"><span data-stu-id="4a1af-107">In WCF, a `Stock` class for a stock market trading application might be defined like the following example:</span></span>

```csharp
namespace TraderSys
{
    [DataContract]
    public class Stock
    {
        [DataMember]
        public int Id { get; set;}
        [DataMember]
        public string Symbol { get; set;}
        [DataMember]
        public string DisplayName { get; set;}
        [DataMember]
        public int MarketId { get; set; }
    }
}
```

<span data-ttu-id="4a1af-108">Para implementar a classe equivalente em Protobuf, ela deve ser declarada no arquivo `.proto`.</span><span class="sxs-lookup"><span data-stu-id="4a1af-108">To implement the equivalent class in Protobuf, it must be declared in the `.proto` file.</span></span> <span data-ttu-id="4a1af-109">O compilador de `protoc` irá gerar a classe .NET como parte do processo de compilação.</span><span class="sxs-lookup"><span data-stu-id="4a1af-109">The `protoc` compiler will then generate the .NET class as part of the build process.</span></span>

```protobuf
syntax "proto3";

option csharp_namespace = "TraderSys";

message Stock {

    int32 id = 1;
    string symbol = 2;
    string display_name = 3;
    int32 market_id = 4;

}  
```

<span data-ttu-id="4a1af-110">A primeira linha declara a versão de sintaxe que está sendo usada.</span><span class="sxs-lookup"><span data-stu-id="4a1af-110">The first line declares the syntax version being used.</span></span> <span data-ttu-id="4a1af-111">A versão 3 da linguagem foi lançada em 2016 e é a versão recomendada para os serviços gRPCs.</span><span class="sxs-lookup"><span data-stu-id="4a1af-111">Version 3 of the language was released in 2016 and is the recommended version for gRPC services.</span></span>

<span data-ttu-id="4a1af-112">A linha de `option csharp_namespace` especifica o namespace a ser usado para os C# tipos gerados.</span><span class="sxs-lookup"><span data-stu-id="4a1af-112">The `option csharp_namespace` line specifies the namespace to be used for the generated C# types.</span></span> <span data-ttu-id="4a1af-113">Essa opção será ignorada quando o arquivo de `.proto` for compilado para outros idiomas.</span><span class="sxs-lookup"><span data-stu-id="4a1af-113">This option will be ignored when the `.proto` file is compiled for other languages.</span></span> <span data-ttu-id="4a1af-114">É comum que os arquivos Protobuf contenham opções específicas de idioma para vários idiomas.</span><span class="sxs-lookup"><span data-stu-id="4a1af-114">It is common for Protobuf files to contain language-specific options for several languages.</span></span>

<span data-ttu-id="4a1af-115">A definição da mensagem de `Stock` especifica quatro campos, cada um com um tipo, um nome e um número de campo.</span><span class="sxs-lookup"><span data-stu-id="4a1af-115">The `Stock` message definition specifies four fields, each with a type, a name, and a field number.</span></span>

## <a name="field-numbers"></a><span data-ttu-id="4a1af-116">Números de campo</span><span class="sxs-lookup"><span data-stu-id="4a1af-116">Field numbers</span></span>

<span data-ttu-id="4a1af-117">Os números de campo são uma parte importante do Protobuf.</span><span class="sxs-lookup"><span data-stu-id="4a1af-117">Field numbers are an important part of Protobuf.</span></span> <span data-ttu-id="4a1af-118">Eles são usados para identificar campos nos dados binários codificados, o que significa que eles não podem mudar de versão para versão do seu serviço.</span><span class="sxs-lookup"><span data-stu-id="4a1af-118">They're used to identify fields in the binary encoded data, which means they can't change from version to version of your service.</span></span> <span data-ttu-id="4a1af-119">A vantagem é que a compatibilidade com versões anteriores e posteriores é possível.</span><span class="sxs-lookup"><span data-stu-id="4a1af-119">The advantage is that backward and forward compatibility is possible.</span></span> <span data-ttu-id="4a1af-120">Os clientes e serviços simplesmente ignorarão os números de campo que não conhecem, contanto que a possibilidade de valores ausentes seja manipulada.</span><span class="sxs-lookup"><span data-stu-id="4a1af-120">Clients and services will simply ignore field numbers they don't know about, as long as the possibility of missing values is handled.</span></span>

<span data-ttu-id="4a1af-121">No formato binário, o número do campo é combinado com um identificador de tipo.</span><span class="sxs-lookup"><span data-stu-id="4a1af-121">In the binary format, the field number is combined with a type identifier.</span></span> <span data-ttu-id="4a1af-122">Os números de campo de 1 a 15 podem ser codificados com seu tipo como um único byte; os números de 16 a 2047 levam 2 bytes.</span><span class="sxs-lookup"><span data-stu-id="4a1af-122">Field numbers from 1 to 15 can be encoded with their type as a single byte; numbers from 16 to 2047 take 2 bytes.</span></span> <span data-ttu-id="4a1af-123">Você pode ficar mais alto se precisar de mais de 2047 campos em uma mensagem por qualquer motivo.</span><span class="sxs-lookup"><span data-stu-id="4a1af-123">You can go higher if you need more than 2047 fields on a message for any reason.</span></span> <span data-ttu-id="4a1af-124">Os identificadores de byte único para os números de campo de 1 a 15 oferecem melhor desempenho, portanto, você deve usá-los para os campos mais básicos e usados com frequência.</span><span class="sxs-lookup"><span data-stu-id="4a1af-124">The single byte identifiers for field numbers 1 to 15 offer better performance, so you should use them for the most basic, frequently used fields.</span></span>
