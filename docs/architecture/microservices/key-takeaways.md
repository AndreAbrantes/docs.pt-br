# <a name="key-takeaways"></a><span data-ttu-id="ea701-103">Principais aspectos a serem lembrados</span><span class="sxs-lookup"><span data-stu-id="ea701-103">Key Takeaways</span></span>

<span data-ttu-id="ea701-104">A seguir, encontram-se as conclusões mais importantes deste guia, que servem como um resumo dos principais aspectos a serem lembrados.</span><span class="sxs-lookup"><span data-stu-id="ea701-104">As a summary and key takeaways, the following are the most important conclusions from this guide.</span></span>

<span data-ttu-id="ea701-105">**Benefícios do uso de contêineres.**</span><span class="sxs-lookup"><span data-stu-id="ea701-105">**Benefits of using containers.**</span></span> <span data-ttu-id="ea701-106">As soluções baseadas em contêiner oferecem uma importante economia de custo porque ajudam a reduzir os problemas de implantação causados por dependências com falha em ambientes de produção.</span><span class="sxs-lookup"><span data-stu-id="ea701-106">Container-based solutions provide important cost savings because they help reduce deployment problems caused by failing dependencies in production environments.</span></span> <span data-ttu-id="ea701-107">Portanto, os contêineres melhoram significativamente as operações de produção e DevOps.</span><span class="sxs-lookup"><span data-stu-id="ea701-107">Containers significantly improve DevOps and production operations.</span></span>

<span data-ttu-id="ea701-108">**Contêineres serão onipresentes.**</span><span class="sxs-lookup"><span data-stu-id="ea701-108">**Containers will be ubiquitous.**</span></span> <span data-ttu-id="ea701-109">Os contêineres baseados no Docker estão se tornando o verdadeiro padrão no setor, com suporte dos principais fornecedores nos ecossistemas do Windows e do Linux, como Microsoft, Amazon AWS, Google e IBM.</span><span class="sxs-lookup"><span data-stu-id="ea701-109">Docker-based containers are becoming the de facto standard in the industry, supported by key vendors in the Windows and Linux ecosystems, such as Microsoft, Amazon AWS, Google, and IBM.</span></span> <span data-ttu-id="ea701-110">Provavelmente, em breve o Docker será predominante nos data centers de nuvem e locais.</span><span class="sxs-lookup"><span data-stu-id="ea701-110">Docker will probably soon be ubiquitous in both the cloud and on-premises datacenters.</span></span>

<span data-ttu-id="ea701-111">**Contêineres como uma unidade de implantação.**</span><span class="sxs-lookup"><span data-stu-id="ea701-111">**Containers as a unit of deployment.**</span></span> <span data-ttu-id="ea701-112">Um contêiner do Docker está se tornando a unidade padrão de implantação para qualquer serviço ou aplicativo baseado em servidor.</span><span class="sxs-lookup"><span data-stu-id="ea701-112">A Docker container is becoming the standard unit of deployment for any server-based application or service.</span></span>

<span data-ttu-id="ea701-113">**Microsserviços.**</span><span class="sxs-lookup"><span data-stu-id="ea701-113">**Microservices.**</span></span> <span data-ttu-id="ea701-114">A arquitetura de microsserviços está se tornando a abordagem preferencial para aplicativos críticos grandes ou complexos e distribuídos baseados em diversos subsistemas independentes na forma de serviços autônomos.</span><span class="sxs-lookup"><span data-stu-id="ea701-114">The microservices architecture is becoming the preferred approach for distributed and large or complex mission-critical applications based on many independent subsystems in the form of autonomous services.</span></span> <span data-ttu-id="ea701-115">Em uma arquitetura baseada em microsserviço, o aplicativo é criado em uma coleção de serviços que podem ser desenvolvidos, testados, implantados e ter as versões controladas de forma independente.</span><span class="sxs-lookup"><span data-stu-id="ea701-115">In a microservice-based architecture, the application is built as a collection of services that are developed, tested, versioned, deployed, and scaled independently.</span></span> <span data-ttu-id="ea701-116">Cada serviço pode incluir qualquer banco de dados autônomo relacionado.</span><span class="sxs-lookup"><span data-stu-id="ea701-116">Each service can include any related autonomous database.</span></span>

<span data-ttu-id="ea701-117">**Design controlado por domínio e SOA.**</span><span class="sxs-lookup"><span data-stu-id="ea701-117">**Domain-driven design and SOA.**</span></span> <span data-ttu-id="ea701-118">Os padrões de arquitetura de microsserviços derivam da SOA (arquitetura orientada a serviços) e do DDD (design controlado por domínio).</span><span class="sxs-lookup"><span data-stu-id="ea701-118">The microservices architecture patterns derive from service-oriented architecture (SOA) and domain-driven design (DDD).</span></span> <span data-ttu-id="ea701-119">Ao projetar e desenvolver microsserviços para ambientes com necessidades e regras dos negócios em evolução, é importante considerar as abordagens e os padrões de DDD.</span><span class="sxs-lookup"><span data-stu-id="ea701-119">When you design and develop microservices for environments with evolving business needs and rules, it's important to consider DDD approaches and patterns.</span></span>

<span data-ttu-id="ea701-120">**Desafios dos microsserviços.**</span><span class="sxs-lookup"><span data-stu-id="ea701-120">**Microservices challenges.**</span></span> <span data-ttu-id="ea701-121">Os data-stu-id="ea701-125">These include large and complex applications with multiple evolving subsystems.</span></span> <span data-ttu-id="ea701-126">Nesses casos, vale a pena investir em uma <span data-ttu-id="ea701-127">**Contêineres para qualquer aplicativo.**</span><span class="sxs-lookup"><span data-stu-id="ea701-127">**Containers for any application.**</span></span> <span data-ttu-id="ea701-128">Os contêineres são convenientes para os microsserviços, mas também podem ser úteis para aplicativos monolíticos baseados no .NET Framework tradicional, quando são usados contêineres do Windows.</span><span class="sxs-lookup"><span data-stu-id="ea701-128">Containers are convenient for microservices, but can also be useful for monolithic applications based on the traditional .NET Framework, when using Windows Containers.</span></span> <span data-ttu-id="ea701-129">Os benefícios de usar o Docker, como resolver vários problemas entre as fases de implantação e produção e fornecer ambientes de desenvolvimento e teste de última geração, aplicam-se a vários tipos de aplicativos diferentes.</span><span class="sxs-lookup"><span data-stu-id="ea701-129">The benefits of using Docker, such as solving many deployment-to-production issues and providing state-of-the-art Dev and Test environments, apply to many different types of applications.</span></span>

<span data-ttu-id="ea701-130">**CLI versus IDE.**</span><span class="sxs-lookup"><span data-stu-id="ea701-130">**CLI versus IDE.**</span></span> <span data-ttu-id="ea701-131">Com as ferramentas da Microsoft, é possível desenvolver aplicativos .NET em contêineres usando sua abordagem preferencial.</span><span class="sxs-lookup"><span data-stu-id="ea701-131">With Microsoft tools, you can develop containerized .NET applications using your preferred approach.</span></span> <span data-ttu-id="ea701-132">Você pode desenvolver com uma CLI e um ambiente baseado em editor usando a CLI do Docker e o Visual Studio Code.</span><span class="sxs-lookup"><span data-stu-id="ea701-132">You can develop with a CLI and an editor-based environment by using the Docker CLI and Visual Studio Code.</span></span> <span data-ttu-id="ea701-133">Outra opção é usar uma abordagem centrada em IDE com o Visual Studio e seus recursos exclusivos para Docker, como a depuração de aplicativos de vários contêineres.</span><span class="sxs-lookup"><span data-stu-id="ea701-133">Or you can use an IDE-focused approach with Visual Studio and its unique features for Docker, such as multi-container debugging.</span></span>

<span data-ttu-id="ea701-134">**Aplicativos de nuvem resilientes.**</span><span class="sxs-lookup"><span data-stu-id="ea701-134">**Resilient cloud applications.**</span></span> <span data-ttu-id="ea701-135">Em sistemas baseados em nuvem e sistemas distribuídos em geral, há sempre o risco de falha parcial.</span><span class="sxs-lookup"><span data-stu-id="ea701-135">In cloud-based systems and distributed systems in general, there is always the risk of partial failure.</span></span> <span data-ttu-id="ea701-136">Como clientes e serviços são processos separados (contêineres), um serviço pode não ser capaz de responder de forma oportuna a uma solicitação do cliente.</span><span class="sxs-lookup"><span data-stu-id="ea701-136">Since clients and services are separate processes (containers), a service might not be able to respond in a timely way to a client’s request.</span></span> <span data-ttu-id="ea701-137">Por exemplo, um serviço pode estar inativo por causa de uma falha parcial ou de manutenção, estar sobrecarregado e respondendo a solicitações de maneira extremamente lenta ou simplesmente não estar acessível durante um curto período devido a problemas de rede.</span><span class="sxs-lookup"><span data-stu-id="ea701-137">For example, a service might be down because of a partial failure or for maintenance;
