# <a name="when-to-choose-net-framework-for-docker-containers"></a><span data-ttu-id="a8aff-103">Quando escolher o .NET Framework para os contêineres do Docker</span><span class="sxs-lookup"><span data-stu-id="a8aff-103">When to choose .NET Framework for Docker containers</span></span>

<span data-ttu-id="a8aff-104">Embora o .NET Core ofereça benefícios significativos para novos aplicativos e padrões de aplicativo, o .NET Framework continuará sendo uma boa escolha para muitos cenários existentes.</span><span class="sxs-lookup"><span data-stu-id="a8aff-104">While .NET Core offers significant benefits for new applications and application patterns, .NET Framework will continue to be a good choice for many existing scenarios.</span></span>

## <a name="migrating-existing-applications-directly-to-a-windows-server-container"></a><span data-ttu-id="a8aff-105">Migrando aplicativos existentes diretamente para um contêiner do Windows Server</span><span class="sxs-lookup"><span data-stu-id="a8aff-105">Migrating existing applications directly to a Windows Server container</span></span>

<span data-ttu-id="a8aff-109">Na maioria dos casos desse cenário, não será necessário migrar seus aplicativos existentes para o .NET Core; é possível usar contêineres do Docker que incluem o .NET Framework tradicional.</span><span class="sxs-lookup"><span data-stu-id="a8aff-109">In most cases for this scenario, you will not need to migrate your existing applications to .NET Core; you can use Docker containers that include the traditional .NET Framework.</span></span> <span data-ttu-id="a8aff-110">Contudo, uma abordagem recomendada é usar o .NET Core ao estender um aplicativo existente, por exemplo, para gravar um novo serviço no ASP.NET Core.</span><span class="sxs-lookup"><span data-stu-id="a8aff-110">However, a recommended approach is to use .NET Core as you extend an existing application, such as writing a new service in ASP.NET Core.</span></span>

<span data-ttu-id="a8aff-112">Bibliotecas de terceiros estão rapidamente adotando [o .NET Standard](../../../standard/net-standard.md), que permite o compartilhamento de código em todos os sabores .NET, incluindo o .NET Core.</span><span class="sxs-lookup"><span data-stu-id="a8aff-112">Third-party libraries are quickly embracing [.NET Standard](../../../standard/net-standard.md), which enables code sharing across all .NET flavors, including .NET Core.</span></span> <span data-ttu-id="a8aff-113">Com o .NET Standard 2.0 e posterior, a compatibilidade de superfície da API entre diferentes frameworks tornou-se significativamente maior.</span><span class="sxs-lookup"><span data-stu-id="a8aff-113">With .NET Standard 2.0 and later, the API surface compatibility across different frameworks has become significantly larger.</span></span> <span data-ttu-id="a8aff-114">Ainda mais, o .NET Core 2.x e os aplicativos mais novos também podem referenciar diretamente as bibliotecas do .NET Framework existentes (consulte [.NET Framework 4.6.1 com suporte ao .NET Standard 2.0](https://github.com/dotnet/standard/blob/master/docs/planning/netstandard-2.0/README.md#net-framework-461-supporting-net-standard-20)).</span><span class="sxs-lookup"><span data-stu-id="a8aff-114">Even more, .NET Core 2.x and newer applications can also directly reference existing .NET Framework libraries (see [.NET Framework 4.6.1 supporting .NET Standard 2.0](https://github.com/dotnet/standard/blob/master/docs/planning/netstandard-2.0/README.md#net-framework-461-supporting-net-standard-20)).</span></span>

<span data-ttu-id="a8aff-115">Além disso, o [Pacote de Compatibilidade do Windows](../../../core/porting/windows-compat-pack.md) amplia a superfície da API disponível para o .NET Standard 2.0 no Windows.</span><span class="sxs-lookup"><span data-stu-id="a8aff-115">In addition, the [Windows Compatibility Pack](../../../core/porting/windows-compat-pack.md) extends the API surface available for .NET Standard 2.0 on Windows.</span></span> <span data-ttu-id="a8aff-116">Este pacote permite recompilar a maioria do código existente para o .NET Standard 2.x com pouca ou nenhuma modificação para ser executado no Windows.</span><span class="sxs-lookup"><span data-stu-id="a8aff-116">This pack allows recompiling most existing code to .NET Standard 2.x with little or no modification, to run on Windows.</span></span>

<span data-ttu-id="a8aff-117">No entanto, mesmo com essa progressão excepcional desde o .NET Standard 2.0 e o .NET Core 2.1, pode haver casos em que determinados pacotes NuGet precisam do Windows para serem executados e não dão suporte ao .NET Core.</span><span class="sxs-lookup"><span data-stu-id="a8aff-117">However, even with that exceptional progression since .NET Standard 2.0 and .NET Core 2.1, there might be cases where certain NuGet packages need Windows to run and might not support .NET Core.</span></span> <span data-ttu-id="a8aff-118">Se esses pacotes forem críticos ao aplicativo, será necessário usar o .NET Framework em contêineres do Windows.</span><span class="sxs-lookup"><span data-stu-id="a8aff-118">If those packages are critical for your application, then you will need to use .NET Framework on Windows Containers.</span></span>
