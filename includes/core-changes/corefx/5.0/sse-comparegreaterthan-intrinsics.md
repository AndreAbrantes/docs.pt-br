### <a name="sse-and-sse2-comparegreaterthan-methods-properly-handle-nan-inputs"></a>Os métodos SSE e SSE2 CompareGreaterThan manipulam corretamente as entradas NaN

Os métodos <xref:System.Runtime.Intrinsics.X86.Sse?displayProperty=nameWithType> e <xref:System.Runtime.Intrinsics.X86.Sse2?displayProperty=nameWithType> a seguir foram corrigidos para lidar `NaN` corretamente com entradas e corresponder ao comportamento de hardware dos métodos equivalentes na <xref:System.Runtime.Intrinsics.X86.Avx?displayProperty=nameWithType> classe:

* `CompareGreaterThan`
* `CompareGreaterThanOrEqual`
* `CompareNotGreaterThan`
* `CompareNotGreaterThanOrEqual`
* `CompareScalarGreaterThan`
* `CompareScalarGreaterThanOrEqual`
* `CompareScalarNotGreaterThan`
* `CompareScalarNotGreaterThanOrEqual`

#### <a name="change-description"></a>Descrição da alteração

Anteriormente, `NaN` as entradas para os <xref:System.Runtime.Intrinsics.X86.Sse> métodos <xref:System.Runtime.Intrinsics.X86.Sse2> listados e retornaram um resultado incorreto. O resultado também é diferente do resultado gerado pelo método correspondente na <xref:System.Runtime.Intrinsics.X86.Avx> classe.

A partir do .NET 5,0, esses métodos manipulam `NaN` corretamente as entradas e retornam os mesmos resultados que os <xref:System.Runtime.Intrinsics.X86.Avx> métodos correspondentes na classe.

As arquiteturas de Streaming SIMD Extensions (SSE) e Streaming SIMD Extensions 2 (SSE2) do setor (ISAs) não fornecem suporte direto de hardware para esses métodos de comparação, para que sejam implementadas no software. Anteriormente, os métodos foram implementados incorretamente e eles incorretamente manipulavam `NaN` entradas. Para o código portado de nativo, o comportamento incorreto pode introduzir bugs. Para um caminho de código de 256 bits, os métodos também podem produzir resultados diferentes para os métodos equivalentes <xref:System.Runtime.Intrinsics.X86.Avx> na classe.

Como exemplo de como os métodos estavam incorretos anteriormente, você pode `CompareNotGreaterThan(x,y)` implementar `CompareLessThanOrEqual(x,y)` como para inteiros regulares. No entanto `NaN` , para entradas, essa lógica computa o resultado errado. Em vez disso `CompareNotLessThan(y,x)` , usar compara os números corretamente `NaN` *e* leva as entradas em consideração.

#### <a name="version-introduced"></a>Versão introduzida

5,0 versão prévia 4

#### <a name="recommended-action"></a>Ação recomendada

- Se o comportamento anterior era um bug, nenhuma alteração é necessária.

- Se o comportamento anterior foi desejado, você pode manter esse comportamento alterando a invocação relevante da seguinte maneira:

  * `CompareGreaterThan(x,y)` -> `CompareNotLessThanOrEqual(x,y)`
  * `CompareGreaterThanOrEqual(x,y)` -> `CompareNotLessThan(x,y)`
  * `CompareNotGreaterThan(x,y)` -> `CompareLessThanOrEqual(x,y)`
  * `CompareNotGreaterThanOrEqual(x,y)` -> `CompareLessThan(x,y)`
  * `CompareScalarGreaterThan(x,y)` -> `CompareScalarNotLessThanOrEqual(x,y)`
  * `CompareScalarGreaterThanOrEqual(x,y)` -> `CompareScalarNotLessThan(x,y)`
  * `CompareScalarNotGreaterThan(x,y)` -> `CompareScalarLessThanOrEqual(x,y)`
  * `CompareScalarNotGreaterThanOrEqual(x,y)` -> `CompareScalarLessThan(x,y)`
